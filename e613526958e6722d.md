# ResourceBridge

ResourceBridge 是一个面向技术内容聚合与外部资源规范化管理的开源工具集，定位于帮助开发者、技术文档维护者以及社区运营人员高效整理、校验和呈现大规模外部链接资源。本项目不提供具体的业务数据，而是提供一套标准化的资源索引框架、链接健康度检查工具以及基于 Markdown 的文档生成流水线，适用于需要维护大量外链的技术博客、开源项目文档站或内部知识库系统。

目标用户包括开源项目维护者、技术内容运营团队、DevOps 工程师以及需要定期审计大量外部 URL 可用性的质量保障人员。ResourceBridge 通过声明式配置、自动化检查与结构化输出，将零散的 URL 列表转化为可维护、可追溯、可展示的文档资产，有效降低人工整理外链的出错率与维护成本。

## 功能概览

- **批量链接导入与去重**：支持从文本文件、CSV 或直接粘贴的原始数据中批量导入 URL，自动识别协议与域名变体，完成去重与格式化清洗。

- **链接可用性健康检查**：内置异步 HTTP 探活模块，支持配置超时与重试策略，可对每一条外链进行状态码与响应时间检测，输出健康报告。

- **分类索引与标签生成**：基于 URL 的域名、路径层级或自定义正则规则，自动为链接打上分类标签，便于后续按主题或来源分组展示。

- **Markdown 文档生成器**：根据预设模板将结构化链接数据渲染为符合项目 README 风格的 Markdown 列表或表格，支持自定义章节顺序与层级深度。

- **变更审计与版本记录**：每次运行均生成链接清单的哈希指纹，对比历史版本输出变更差异（新增、删除、URL 改动），方便纳入 Git 版本控制。

- **多格式导出支持**：除 Markdown 外，支持导出为 JSON、YAML 或 HTML 片段，方便嵌入静态站点生成器或前端页面。

- **定时任务集成示例**：提供 systemd timer 与 cron 任务配置样例，支持定期自动执行链接检查并推送更新通知。

## 应用场景

- **开源项目文档站外链维护**：开源项目通常需要在 README 或文档站中引用大量外部资源（依赖库、教程、参考文档）。ResourceBridge 可定期检查这些链接是否失效或发生重定向，帮助维护者及时发现并更新，避免用户因访问死链而获得不良体验。

- **技术社区资源聚合页构建**：技术社区或学习平台常整理“精选工具”“推荐阅读”等外链汇总页面。使用 ResourceBridge 可将运营人员收集的 URL 清单自动转化为排版统一的 Markdown 页面，同时通过分类索引功能实现按标签筛选展示。

- **企业内部知识库链接审计**：大型企业的内部 Wiki 或 Confluence 空间中包含大量指向外部供应商、技术论坛或云服务控制台的链接。ResourceBridge 可对接内部 CI 流水线，定期抓取指定页面的所有外链并执行可用性检查，生成审计报表发送给知识库管理员。

- **迁移或改版时的链接存量盘点**：当网站或文档站进行域名迁移、路径重构时，需要全面盘点所有外部引用链接，区分哪些指向自身旧域名、哪些指向第三方。ResourceBridge 的路径分析与筛选功能可快速完成存量盘点并生成迁移清单。

## 快速开始

以下命令演示如何从 Git 克隆项目、安装依赖并执行一次完整的链接处理流程。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
pip install -r requirements.txt
python run.py --input ./samples/urls.txt --output ./output/README.md --check
```

其中 `--input` 指定包含 URL 列表的文本文件（每行一个 URL），`--output` 指定生成的 Markdown 文件路径，`--check` 启用链接健康检查。运行完成后将在输出目录生成资源清单文档与一份 `health_report.json` 健康报告。

## 项目结构

```
resourcebridge/
├── run.py                         # 命令行入口，解析参数并协调各模块执行
├── config.yaml                    # 主配置文件，定义分类规则、检查参数与输出模板
├── requirements.txt               # Python 依赖清单，供 pip 安装使用
├── src/                           # 核心源码目录
│   ├── core/                      # 核心逻辑模块
│   │   ├── loader.py              # 从文件或标准输入加载原始 URL 列表，执行去重
│   │   ├── checker.py             # 异步健康检查器，封装 aiohttp 会话与重试机制
│   │   ├── classifier.py          # 基于规则或域名路径对 URL 进行分组与打标签
│   │   └── generator.py           # Markdown 渲染器，根据模板生成最终文档
│   ├── models/                    # 数据模型定义
│   │   ├── link.py                # Link 实体类，包含 url、status、tags、response_time
│   │   └── report.py              # 检查报告与变更差异的数据结构
│   ├── utils/                     # 通用工具函数
│   │   ├── http.py                # HTTP 请求辅助函数，处理重定向与超时
│   │   ├── logger.py              # 日志配置与格式化输出
```

## 常见问题

**Q：项目是否支持处理非 HTTP/HTTPS 协议的链接，例如 ftp 或 mailto？**

A：当前版本的健康检查模块仅针对 HTTP 与 HTTPS 协议实现探活逻辑。对于 ftp、mailto、tel 等协议，系统会跳过可用性检查，但依然会将其纳入文档生成的链接列表，并在健康报告中标记为“未检测”状态。后续版本计划扩展协议支持。

**Q：如何处理链接检查时遇到的网络超时或临时性不可达问题？**

A：检查器支持通过配置文件中的 `check.timeout` 与 `check.retries` 字段调整超时秒数与重试次数。默认超时为 10 秒，重试 2 次。若您的网络环境较慢或目标服务器响应延迟较高，建议适当增大这两个值。同时，检查结果报告中会区分“超时”“拒绝连接”“HTTP 错误状态码”等不同失败类型，方便定位问题。

**Q：生成的 Markdown 文档能否自定义章节顺序或标题层级？**

A：可以。ResourceBridge 的生成器模块使用 Jinja2 模板引擎渲染输出。您可以将默认模板复制到项目外部进行修改，并通过 `--template` 参数指定自定义模板文件路径。模板中可自由控制标题层级、列表样式以及是否包含健康检查结果信息。
