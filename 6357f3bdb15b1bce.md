# VideoLink Nexus

VideoLink Nexus 是一个面向技术内容创作者、在线教育平台及数字媒体归档者的智能视频资源外链管理与索引系统。该项目并非视频托管服务，而是一个高度结构化的外部视频资源引用汇总与导航层，旨在解决多源、多批次视频素材的定位、分类与快速访问问题。

项目目标用户包括：需要批量管理课程视频索引的在线讲师、进行多媒体数据归集的数字档案管理员、以及希望建立私有视频目录体系的开发团队。VideoLink Nexus 不存储任何视频文件，仅提供基于唯一标识符的视频页面外链聚合功能，确保资源引用的持久性与可追溯性。

本批次（第 727/1241 批）包含 250 个视频资源外链，全部指向 yuejiafan.com 域下的视频播放页面。所有链接均已通过基础格式校验，并按照收录时间顺序完成标准化登记。

## 功能概览

批量外链导入：支持以批次为单位导入大量视频页面 URL，系统自动解析链接中的唯一视频标识符（如 xhay、evhr），并生成内部索引记录。

唯一标识符索引：每个视频资源均以链接末尾的四位字母代码作为主键，支持通过该代码快速反查完整视频页面地址，无需记忆长 URL。

资源状态监控：定期对已收录的外链进行可达性检测，标记失效或重定向的链接，帮助维护者及时清理或更新资源。

分类标签管理：允许用户为每个视频链接添加自定义标签（如“教程”、“会议录播”、“产品演示”），便于按主题或用途进行筛选与分组。

批量导出与分享：支持将选中的视频链接列表导出为纯文本或结构化数据格式（JSON/CSV），方便迁移至其他平台或与团队成员共享。

搜索与过滤：提供基于视频标识符、标签、收录时间范围的组合查询功能，在包含数万条外链的库中快速定位目标资源。

访问统计看板：记录每个外链的点击次数与最近访问时间，辅助判断资源热度与使用频率，为后续归档决策提供数据支撑。

## 应用场景

在线课程资源库构建：教育机构可将每节录播课程对应的视频页面链接录入 VideoLink Nexus，并统一标记课程名称与章节编号。学员通过内部查询界面即可获得准确的视频观看地址，避免因链接分散或过期导致的访问失败。

多媒体项目素材管理：视频制作团队在素材收集阶段，将来自不同来源的参考视频外链集中收录。成员可按项目代号或场景类型筛选链接，快速调取所需素材页面进行创意讨论，无需在多个浏览器标签或文档间反复切换。

技术文档视频引用维护：开源项目或产品手册中常嵌入操作演示视频链接。使用 VideoLink Nexus 管理这些外链后，文档维护者可定期批量检查链接有效性，并在视频地址变更时仅需更新系统记录，无需逐个修改文档页面。

历史活动归档与复盘：大型线上峰会或系列技术沙龙结束后，组织者可将所有演讲视频页面链接批量导入，按演讲日期、嘉宾姓名或主题领域分类。后续回顾或编写活动总结报告时，可直接生成包含精确链接的资源清单。

## 快速开始

以下步骤将指导您在本地环境中快速部署 VideoLink Nexus 索引服务，并完成首批 250 条视频外链的导入与验证。

```bash
# 克隆项目仓库至本地
git clone https://github.com/videolink-nexus/core.git

# 进入项目根目录
cd core

# 安装项目依赖（使用 pip 和 requirements.txt）
pip install -r requirements.txt

# 执行数据库初始化与批次导入脚本
# 该命令将创建 SQLite 数据库，并自动导入第 727/1241 批次的 250 条链接
python scripts/batch_import.py --batch 727 --source data/batch_727_links.txt
```

执行上述命令后，本地索引服务将运行于 `http://localhost:8080`。您可通过浏览器访问该地址，使用内置查询界面验证导入结果。

## 项目结构

```

├── app/                             # 核心应用模块
│   ├── __init__.py                  # 应用工厂函数与初始化配置
│   ├── routes/                      # 路由蓝图集合
│   │   ├── index.py                 # 首页与查询界面路由
│   │   ├── api.py                   # RESTful API 端点实现
│   │   └── admin.py                 # 管理与维护后台路由
│   ├── models/                      # 数据模型与 ORM 映射
│   │   ├── link.py                  # 视频链接记录模型（含标识符、URL、状态）
│   │   ├── batch.py                 # 批次记录模型（批次号、导入时间、来源）
│   │   └── tag.py                   # 标签模型及链接-标签关联表
│   ├── services/                    # 业务逻辑服务层
│   │   ├── importer.py              # 批次导入服务（解析、去重、入库）
│   │   ├── monitor.py               # 链接可达性监控服务（定时任务）
│   │   └── exporter.py              # 数据导出服务（JSON/CSV 格式）
│   └── utils/                       # 通用工具函数
│       ├── validators.py            # URL 格式校验与规范化
│       ├── extractors.py            # 从 URL 中提取视频标识符
│       └── logger.py                # 日志配置与封装
├── scripts/                         # 命令行运维脚本
│   ├── batch_import.py              # 批量导入入口脚本（支持 --batch 参数）
│   ├── run_monitor.py               # 手动触发链接状态检测脚本
│   └── backup_db.py                 # 数据库备份与恢复工具
├── tests/                           # 测试套件
│   ├── unit/                        # 单元测试（模型、工具函数）
│   ├── integration/                 # 集成测试（API、导入流程）
│   └── fixtures/                    # 测试用固定数据（示例批次文件）
```

## 贡献指南

## 常见问题

Q: 导入批次时提示部分链接重复，如何处理？

A: 导入脚本默认基于完整 URL 进行去重。若批次中包含已存在于数据库中的链接，系统将跳过该记录并记录警告日志，不会覆盖原有数据。您可通过 `--force` 参数强制更新已有链接的元数据（如标签或状态），但不推荐在生产环境使用。

Q: 链接状态监控显示大量链接为“不可达”，但手动访问浏览器可以打开，为什么？

A: 监控服务默认使用 `requests` 库发送 HEAD 请求并设置 5 秒超时。部分视频页面可能对 HEAD 请求响应缓慢或返回 405 状态码。您可将监控配置切换为 GET 请求模式（设置 `config.MONITOR_METHOD = 'GET'`），或调整超时时间阈值。建议先在测试环境验证后再修改生产配置。

Q: 如何将本地数据库迁移至生产服务器？

A: 推荐使用 `scripts/backup_db.py` 工具导出 SQLite 数据库的完整转储文件（SQL 格式）。在目标服务器执行 `sqlite3` 命令导入该文件即可。若生产环境使用 PostgreSQL，可先导出为 CSV 格式再通过 `COPY` 命令导入，具体步骤参见 `docs/deployment.md` 中的数据库迁移章节。
