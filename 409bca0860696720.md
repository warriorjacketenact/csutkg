# EON Resource Navigator

EON Resource Navigator 是一个面向技术研究人员、开源社区贡献者以及信息分析工作者的外链资源汇总与导航系统。该项目定位于对分散在多个垂直领域的高质量外部链接进行结构化整理、分类归档与状态监控，帮助用户在复杂信息环境中快速定位所需资源。项目本身不生产内容，而是通过严格的链接筛选与分类策略，构建一个可复用、可扩展、可审计的参考信息库。

本项目服务于以下用户群体：需要定期查阅特定领域资料的研究人员、需要维护项目文档外链有效性的技术文档工程师、以及希望快速建立自身资源索引体系的开发者。通过统一的入口和规范的分类逻辑，用户能够减少重复搜索成本，提升信息获取效率。

## 功能概览

- **按主题分类的链接索引**：系统将收录的外链按照技术文档、学术参考、行业动态、工具平台、社区讨论等主题维度进行划分，每个链接附带简要描述与标签，便于快速筛选。

- **链接可用性监控**：内置定时检查机制，定期对已收录的 URL 进行 HTTP 状态探测，标记失效链接并生成报告，帮助维护人员及时更新或剔除无效资源。

- **全文检索与过滤**：支持基于标题、描述、标签、域名等字段的全文搜索，并可按分类、状态、更新时间等条件进行组合过滤，提升查找精度。

- **自定义分类标签体系**：用户可根据自身需求对链接添加自定义标签，或调整现有分类映射，系统提供标签管理界面与批量操作接口，适应个性化信息组织方式。

- **导入与导出接口**：支持以 JSON、CSV、Markdown 表格等格式导入外部链接清单，也可将当前索引导出为标准化格式，便于迁移、备份或与其他工具集成。

- **访问统计与热度排序**：记录每个链接的被访问次数与最后访问时间，提供按热度、新增时间、更新时间排序的视图，帮助用户发现近期关注度较高的资源。

## 应用场景

1. **技术文档团队维护外部参考链接库**  
   技术文档编写过程中需引用大量外部规范、API 参考、社区讨论等资源。EON Resource Navigator 可作为团队内部的链接知识库，统一管理所有外部引用，定期自动检查链接有效性，避免文档中出现死链。

2. **开源项目 README 与 Wiki 外链整理**  
   开源项目常需在 README 或 Wiki 中列出一系列相关资源链接。通过本项目的导出功能，可将整理好的链接列表直接生成为 Markdown 格式，便捷地嵌入项目文档中，保持与主仓库同步更新。

3. **学术研究文献参考信息归档**  
   研究人员在文献调研阶段收集大量网页链接。使用本项目的分类标签与检索功能，可对链接按研究主题、期刊来源、发表日期等维度进行精细化管理，并在论文写作阶段快速回溯原始资料。

4. **技术社区内容聚合与分享**  
   技术社区运营者可将社区内推荐的优质文章、工具、视频等链接统一收录，通过本项目的访问统计功能了解社区成员的兴趣偏好，定期推送热门资源，增强社区活跃度。

5. **个人知识体系外链备份**  
   个人开发者可利用本项目建立自己的技术书签系统，将散落在浏览器收藏夹、笔记软件中的链接集中管理，配合自定义标签与全文搜索，打造轻量级个人知识外挂。

## 快速开始

以下指令演示如何从 GitHub 克隆项目、安装依赖并启动开发服务。

```bash
# 克隆项目仓库
git clone https://github.com/eon-resource/eon-navigator.git

# 进入项目目录
cd eon-navigator

# 安装项目依赖（使用 npm）
npm install

# 启动本地开发服务器
npm run dev
```

执行上述命令后，服务默认运行于 localhost:3000。打开浏览器访问该地址即可开始使用 EON Resource Navigator 的本地实例。生产环境部署请参考文档导航章节中的部署指南。

| npm | 是 | 包管理器，随 Node.js 一同安装，版本需不低于 v9.0.0 |
| SQLite3 | 是 | 默认嵌入式数据库，用于存储链接信息与配置，无需额外安装服务 |
| Git | 是 | 用于版本控制和克隆仓库，推荐 v2.30.0 以上 |

## 项目结构

项目采用前后端分离的目录组织方式，后端基于 Express 框架，前端基于 React + Vite 构建。

```
eon-navigator/
├── backend/                        # 后端服务代码
│   ├── src/
│   │   ├── api/                    # RESTful API 路由定义
│   │   │   ├── links.js            # 链接增删改查接口
│   │   │   ├── categories.js       # 分类管理接口
│   │   │   ├── monitor.js          # 状态监控与报告接口
│   │   │   └── export.js           # 数据导出接口
│   │   ├── services/               # 业务逻辑层
│   │   │   ├── linkService.js      # 链接索引与检索逻辑
│   │   │   ├── monitorService.js   # 定时监控调度逻辑
│   │   │   └── statsService.js     # 访问统计聚合逻辑
│   │   ├── models/                 # 数据模型（SQLite 表结构定义）
│   │   │   ├── LinkModel.js
│   │   │   ├── CategoryModel.js
```

## 常见问题

**Q: 如何批量导入我现有的书签或收藏夹？**  
A: 系统支持导入 CSV 格式文件，需包含 url、title、category 三列。您可以从浏览器书签导出 HTML 后转换为 CSV，或直接按照示例模板整理。导入入口位于界面右上角的「导入」菜单。若导入数据量超过 500 条，建议分批次进行，避免请求超时。

**Q: 链接状态监控的频率是多少？是否会对我访问的目标网站造成压力？**  
A: 默认监控周期为每 7 天一次，每次检查采用 HEAD 请求，仅获取响应头信息，不会下载完整页面内容，对目标服务器负载影响极小。用户可在配置文件中调整监控间隔或完全关闭自动监控，改用手动检查。对于高频访问的目标站点，建议设置白名单排除监控。

**Q: 本项目是否支持分布式部署或多用户使用？**  
A: 当前版本主要面向单机部署与单用户/小团队场景。多用户支持已在 v2.0 规划中，届时将引入基于角色的访问控制。目前若需多用户使用，可部署多个独立实例，或共享同一个数据库文件并通过文件系统权限管理访问。生产环境建议配合 Nginx 进行基础认证。
