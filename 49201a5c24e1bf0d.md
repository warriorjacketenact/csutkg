# ResourceBridge 资源导航聚合系统

ResourceBridge 是一个面向开发者、技术研究者与内容创作者的轻量级资源导航与聚合平台。项目定位为技术信息的中转枢纽，通过结构化的外链管理与分类索引，帮助用户从海量网络信息中快速定位所需的技术文档、工具站点、视频教程与项目案例。

项目核心目标用户包括：需要维护大量外链资源的开源项目维护者、日常阅读技术资料但缺乏有效收藏管理手段的工程师、以及希望构建个人知识导航站点的内容创作者。ResourceBridge 不提供内容存储，而是以目录树与标签系统对第三方链接进行组织，并提供检索、分类筛选与访问状态检测等辅助功能，使分散的外部资源形成可维护、可共享的知识网络。

本批次为项目第 526/1241 批资源入库，共计收录 250 个外链条目，涵盖技术博客、视频课程、开源工具文档与行业资讯站点等多个类别。

## 功能概览

**多级分类目录体系** 支持用户自定义分类层级，对入库链接进行归属标记，可精确到三级子类，满足大规模外链管理需求。

**批量链接导入与校验** 支持从文本文件、CSV 或 JSON 格式批量导入链接列表，系统自动对每个链接进行可达性探测与状态标记，区分有效、失效与重定向三种状态。

**全文检索与标签过滤** 基于轻量级倒排索引实现链接标题、描述与标签的快速检索，同时支持按标签组合过滤，提升资源查找效率。

**访问统计与热度排序** 记录每个外链被点击的次数与最后访问时间，支持按热度、入库时间或字母序对列表进行动态排序。

**自定义视图与分享面板** 用户可为每个分类或标签生成独立的只读分享页面，方便将整理好的资源集合以干净整洁的界面分享给团队成员或公开社区。

**定时健康检查** 后台守护进程按可配置周期对所有已入库链接执行 HTTP 头检测，发现异常时通过日志或邮件通知管理员，减少死链累积。

**RESTful 管理接口** 提供完整的 JSON API 用于链接的增删改查、分类管理与批量操作，便于与其他自动化脚本或持续集成流水线集成。

## 应用场景

技术团队内部知识库建设。团队可以将日常开发中积累的官方文档链接、技术博客、调试工具地址等统一录入 ResourceBridge，按项目或技术栈分类，新成员入职时可直接通过系统了解团队常用技术资源，减少信息传递成本。

个人开发者的阅读列表管理。开发者日常浏览过程中发现的优质文章或工具，往往分散保存在浏览器收藏夹中，缺乏结构化整理。ResourceBridge 提供轻量级录入界面与标签系统，帮助个人构建长期维护的技术参考索引。

开源项目的资源附录托管。开源项目的 README 或文档站中往往需要附带大量外部参考链接，直接维护在 Markdown 中容易导致文档臃肿且难以统计访问情况。ResourceBridge 可作为独立服务为项目提供资源列表托管，并通过 API 将数据同步到文档站点。

技术资讯聚合与筛选。内容创作者或社区运营者可将关注的资讯源、视频频道、代码仓库统一入库，利用分类与标签对内容主题进行标注，再通过分享面板生成面向读者的推荐列表，简化内容分发流程。

## 快速开始

以下命令演示如何从 GitHub 克隆项目、安装依赖并启动开发服务。生产环境部署请参考文档导航章节中的运维部署指南。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
pip install -r requirements.txt
python scripts/init_db.py
python app.py --host 0.0.0.0 --port 8080
```

启动后访问 http://localhost:8080 进入管理面板，默认管理员账号为 admin，初始密码在首次启动时输出至控制台日志。请及时通过面板修改默认密码。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 项目运行时核心解释器，低于 3.9 版本将无法解析类型注解语法 |
| SQLite | 3.35 及以上 | 默认内嵌数据库，用于存储链接元数据、分类与标签；支持 JSON 字段类型 |
| Redis | 6.0 及以上 | 可选组件，用于缓存访问统计与分布式锁；生产环境强烈建议启用 |
| Node.js | 16.0 及以上 | 仅用于前端资源构建与开发服务器热重载；生产环境无需安装 |
| Nginx | 1.20 及以上 | 生产环境推荐反向代理服务器，用于静态资源服务与负载均衡 |
| supervisor | 4.0 及以上 | 生产环境进程守护工具，用于管理后台健康检查与定时任务进程 |

## 常见问题

Q: 系统检测到大量链接失效，是否需要手动逐一更新？

A: 不需要。健康检查服务默认每 24 小时对所有链接执行一次可达性检测，并在管理面板中以颜色标记状态（绿色有效、红色失效、黄色重定向）。失效链接可通过批量操作面板统一导出为 CSV 清单，也支持一键重新检测。如需调整检测频率，可在 .env 文件中修改 CHECK_INTERVAL_HOURS 参数。

Q: 如何将现有的浏览器收藏夹或书签文件迁移到 ResourceBridge？

A: 系统支持导入 Netscape 书签格式（HTML）以及 CSV 格式文件。在管理面板的「导入」页面中选择文件并指定默认分类即可。对于 Chrome 或 Firefox 导出的书签文件，系统会自动解析 URL、标题与文件夹层级，并将文件夹映射为分类标签。导入过程在后台异步执行，完成后会发送通知。

Q: 项目的数据存储是否支持从 SQLite 迁移到 PostgreSQL 或 MySQL？

A: 支持。项目使用 SQLAlchemy ORM，理论上兼容多种关系型数据库。生产环境建议使用 PostgreSQL。迁移时需在 .env 中修改 DATABASE_URL 连接串，然后执行 scripts/migrate_db.py 脚本完成表结构与数据的迁移。迁移前请务必备份原始 SQLite 文件。
