# VueBest 渠道资源聚合平台

VueBest 渠道资源聚合平台是一个面向 Vue.js 生态系统的技术资源导航与汇总工具，致力于为 Vue 开发者、架构师和技术决策者提供高质量的第三方渠道链接、社区精选内容和垂直领域的技术导航服务。本项目作为第 1045/1241 批次资源整合计划的核心输出，系统性地收录并分类了超过 250 个经过初步筛选的 Vue 相关技术渠道，涵盖组件库、工具链、性能优化、可视化、移动端适配、状态管理等多个技术维度。

本项目解决的核心问题是 Vue 开发者在海量社区资源中难以高效定位高质量技术渠道的痛点，通过建立结构化的资源索引体系，辅助开发者快速发现与当前技术栈匹配的第三方库、脚手架方案和最佳实践参考，显著降低技术选型的信息收集成本。项目定位为技术资源的外链聚合站，不存储任何第三方资源文件，仅提供定向导航服务。

## 功能概览

**渠道分类索引** 提供按技术领域、应用场景和资源类型三个维度组织的分类导航体系，支持快速筛选特定方向的 Vue 渠道资源。

**实时外链校验** 定期对收录的渠道链接进行可用性检测，标记失效链接并生成健康度报告，确保导航数据的准确性和时效性。

**标签化资源标注** 为每个渠道资源打上技术标签（如 Vue3、TypeScript、SSR、动画库等），支持多标签组合检索，满足精细化查询需求。

**社区热度排序** 基于外部社区讨论热度、GitHub 星标趋势和更新活跃度对渠道进行动态排序，突出当前社区关注度高的优质资源。

**自定义收藏夹** 允许用户将常用渠道加入个人收藏列表，支持批量导出为 JSON 或 Markdown 格式，便于团队内部共享资源清单。

**版本兼容性标注** 标注每个渠道资源与 Vue 2.x / 3.x 的兼容性状态，帮助开发者在版本升级或新项目启动时做出准确判断。

**渠道更新日志追踪** 聚合被收录渠道的版本发布动态，在导航面板中显示近期更新标记，方便开发者跟进上游变化。

**黑暗模式适配** 导航界面原生支持系统主题切换，在明亮与黑暗模式下均保持良好的可读性和视觉体验。

## 应用场景

技术选型调研阶段，架构师需要评估 Vue 生态中可用的状态管理方案、UI 组件库或构建工具。通过本平台按标签筛选同类渠道，集中对比多个候选方案的外部评价、更新频率和社区活跃度，显著提升调研效率。

日常开发过程中，开发者遇到特定功能需求（如文件上传、富文本编辑、数据可视化图表）时，可通过平台快速定位对应的社区渠道，直接跳转至官方文档或示例代码库，避免在搜索引擎中重复过滤低质量结果。

团队内部知识沉淀场景中，技术负责人可将本平台作为团队 Vue 技术栈的推荐资源入口，结合收藏夹功能整理内部认证的渠道清单，统一新成员的技术学习路径和项目脚手架选型参考。

开源项目维护者可将自己的 Vue 组件库或工具提交至平台收录队列，通过平台的渠道热度排序机制获得更多社区曝光，同时借助版本兼容性标注降低使用者的接入门槛。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/vuebest/vuebest-channel-aggregator.git

# 进入项目目录
cd vuebest-channel-aggregator

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 运行渠道链接健康度检测
npm run check:links
```

## 项目结构

```
vuebest-channel-aggregator/
├── src/
│   ├── api/                         # API 服务层，封装后端接口调用
│   │   ├── client.ts               # Axios 实例配置与拦截器
│   │   └── channel.ts              # 渠道资源相关的 CRUD 接口
│   ├── assets/                      # 静态资源，包含图标、字体和全局样式
│   │   ├── fonts/                  # 自定义字体文件
│   │   └── styles/                 # 全局 CSS 变量与重置样式
│   ├── components/                  # Vue 组件库，按功能模块分层
│   │   ├── common/                 # 通用组件，按钮、输入框、卡片、模态框
│   │   ├── layout/                 # 布局组件，导航栏、侧边栏、页脚
│   │   ├── channel/                # 渠道相关组件，列表项、详情卡片、标签组
│   │   └── dashboard/              # 仪表盘组件，统计概览与热度排行
│   ├── composables/                 # 组合式函数，封装可复用的响应式逻辑
│   │   ├── useChannelFilter.ts     # 渠道筛选与排序逻辑
│   │   ├── useLinkHealth.ts        # 链接健康度检测与缓存管理
│   │   └── useBookmark.ts          # 收藏夹的增删改查与本地持久化
│   ├── data/                        # 数据层，包含渠道分类的静态树形结构
│   │   ├── categories.json         # 分类体系的 JSON 定义
│   │   └── tags.json               # 全量标签列表与别名映射
```

## 贡献指南

提交渠道收录请求时，请通过 GitHub Issues 提交新渠道的 URL、所属分类标签以及 100 字以内的中文简介。贡献者需要确认该渠道与 Vue 生态相关且内容合法，避免提交重复或已失效的链接。收录审核通常在 48 小时内完成。

改进分类体系或标签定义时，请 fork 本仓库并在 src/data/ 目录下修改对应的 JSON 文件，提交 Pull Request 时附带修改说明和影响范围评估。分类调整需要同步更新文档导航中的相关说明，确保用户手册与实际数据结构保持一致。

报告链接失效或内容异常时，请使用平台内置的「反馈」按钮或直接在 Issues 中标注渠道 ID 和异常现象。建议附带截图或网络请求日志，便于维护团队快速定位问题根源并更新健康度记录。

开发新功能或修复缺陷时，请基于 main 分支创建 feature/ 或 fix/ 前缀的功能分支，提交前运行 npm run test 确保所有测试用例通过，并使用 npm run lint 修复代码风格问题。PR 描述需清晰说明变更动机、实现方案和测试覆盖情况。

参与国际化翻译时，请参考 docs/i18n/ 目录下的模板文件，为新增的界面文本提供中英文双语对照。翻译贡献者将在项目官网的贡献者列表中获得署名致谢。

## 常见问题

问：平台收录的渠道链接是否经过人工审核？

答：所有提交的渠道链接均经过初步的人工审核，确保内容与 Vue 生态相关且不包含恶意软件或违规信息。但鉴于资源数量庞大，平台不对链接指向的外部内容的准确性、完整性和时效性承担保证责任。用户访问外部链接时应自行判断安全风险。

问：如何获取渠道的最新更新通知？

答：平台提供渠道更新日志追踪功能，在渠道详情页点击「订阅更新」按钮后，系统会通过邮件或 Webhook 方式推送该渠道的版本发布动态。订阅通知需要提供有效的邮箱地址或 Webhook 端点，具体配置方式请参考用户手册中的通知设置章节。

问：平台的数据存储在本地还是云端？

答：本平台的渠道元数据和健康度检测记录默认存储在本地 SQLite 数据库中，所有数据仅在用户本地环境保留。开源部署版本不包含云端同步功能，用户可根据需要在 scripts/ 目录下编写自定义导出脚本，将数据备份为 JSON 或 CSV 格式。官方提供的演示站点部署于独立的云服务器，其数据库仅供演示用途，定期重置。
