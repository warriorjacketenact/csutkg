# LinkVault - 技术资源外链聚合导航系统

LinkVault 是一个面向开发者、技术研究人员与内容创作者的轻量级外链资源聚合与导航平台。该项目旨在解决个人收藏夹混乱、书签跨设备不同步、技术参考链接散落各处难以复用等问题，通过结构化的资源分类与元数据标注，帮助用户建立可维护、可分享的技术资源索引体系。

LinkVault 定位于小规模团队内部知识库的链接基座，也适合独立开发者作为个人技术雷达的存档工具。项目本身不存储任何第三方内容，仅提供链接的录入、标签化、全文检索与快照摘要展示功能，确保合规、轻量且易于自托管。

## 功能概览

**链接收录与管理** 支持手动添加、批量导入与浏览器书签导出文件解析，自动提取页面标题与元描述。

**多级标签分类体系** 允许用户自定义标签树，同一链接可归属多个分类，支持按标签筛选与排除检索。

**全文检索与快速查找** 基于标题、描述、标签与备注字段构建倒排索引，检索响应时间低于 200 毫秒。

**外部资源快照预览** 通过 iframe 沙箱或后端无头浏览器抓取首屏截图，在跳转前提供视觉参考。

**数据导入导出标准格式** 支持 JSON、CSV 与 HTML 书签文件互转，便于迁移至其他书签管理工具。

**访问频率统计与失效检测** 定期对已收录链接进行可达性检查，标记响应超时或状态码异常的条目，并生成报告。

**用户自定义视图与共享列表** 支持创建临时分享链接，允许他人以只读方式访问特定标签组合下的资源列表。

## 应用场景

**技术团队内部知识库的入口层** 团队新成员入职时，可通过 LinkVault 快速获取开发环境搭建文档、编码规范、常用工具站与内部 API 文档的导航列表，减少重复沟通成本。

**开源项目维护者的外部依赖清单管理** 开源项目常需引用大量第三方库、规范文档与社区讨论帖。维护者可将这些外链统一收录至 LinkVault，并按版发布本打上标签，方便后续回溯与更新。

**技术博主的内容素材仓库** 技术写作人员围绕特定主题（如 Kubernetes 网络插件对比、Rust 异步运行时生态）收集大量参考链接时，可用 LinkVault 建立专题列表，并在撰写文章时直接嵌入分享视图。

**个人开发者跨设备书签同步替代方案** 自托管 LinkVault 后，可在办公室、家庭与移动设备上通过同一域名访问统一的书签库，避免依赖浏览器厂商的私有同步服务。

## 快速开始

以下命令适用于 Linux / macOS / WSL2 环境，假设系统已安装 Git、Node.js 与 npm。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（使用 npm ci 确保锁定版本）
npm ci

# 以开发模式启动服务，默认监听 3000 端口
npm run dev
```

启动成功后，访问控制台输出的本地地址（通常为 http://localhost:3000）即可进入初始化向导。首次运行会自动创建 SQLite 数据库文件并生成管理员账户注册页面。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用官方预编译二进制或 nvm 管理 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖与执行脚本 |
| SQLite | 3.35 及以上 | 嵌入式数据库，无需额外安装，项目内嵌驱动 |
| Git | 2.30 及以上 | 版本控制工具，用于克隆仓库与拉取更新 |
| 操作系统 | Linux (glibc 2.28+) / macOS 11+ / Windows 10+ (WSL2) | 支持主流 POSIX 环境与 Windows 子系统 |
| 浏览器 | 现代浏览器（Chrome 90+ / Firefox 88+ / Edge 90+） | 管理界面基于 React 18，需要 ES2020 支持 |
| 内存 | 最低 512 MB，推荐 1 GB | 生产环境建议 2 GB 以容纳无头浏览器快照任务 |
| 存储 | 最低 100 MB | 数据库与日志文件随链接数量增长，每万条约占用 15 MB |

## 常见问题

**Q: LinkVault 是否支持 PostgreSQL 或 MySQL 作为生产数据库？**

当前版本仅内置 SQLite 驱动，因其零配置特性适合小规模部署。PostgreSQL 适配已在 2.0 版本的路线图中，预计支持事务隔离级别与连接池调优。若需使用外部数据库，可自行修改 db/client.js 中的连接适配器，并安装对应驱动包。

**Q: 快照预览功能是否会影响目标网站的访问统计或触发反爬机制？**

快照服务默认使用 Puppeteer 的无头模式，并设置了固定的 User-Agent 与慢速加载策略（延迟 2 秒后截图）。每次抓取均携带 `X-Forwarded-For` 为私有 IP 段，且遵守 robots.txt 规则。若目标网站明确禁止无头浏览器访问，可在配置文件中将该域名加入黑名单，跳过快照生成。

**Q: 导入浏览器书签时出现乱码或解析失败应如何处理？**

浏览器导出的 HTML 文件可能使用 UTF-8 或 GBK 编码。请在导入界面手动选择对应的字符集选项，或使用文本编辑器将文件另存为 UTF-8 格式后再尝试。解析失败时，系统会生成错误日志并跳过该条目，不影响其他链接的正常导入。
