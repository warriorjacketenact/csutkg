# TechResource Hub

TechResource Hub 是一个面向开发者与技术爱好者的高质量技术资源导航与聚合平台。本项目旨在解决技术学习过程中资源分散、质量参差不齐、检索效率低下的核心问题，通过人工筛选与社区共建的方式，持续收录并整理互联网上的优质技术文档、教程、工具与参考手册。

项目定位为技术领域的“资源中台”，不生产内容，但致力于成为最懂开发者需求的内容分发与索引系统。目标用户涵盖初学者、资深工程师、技术管理者以及开源社区贡献者。通过系统化的分类体系与多维度标签系统，帮助用户快速定位所需信息，显著降低信息筛选成本，提升学习与开发效率。

## 功能概览

**多维度资源分类体系** 按照编程语言、框架生态、基础设施、算法与数据结构、开发工具等维度对资源进行精细化分类，支持多标签交叉筛选，满足不同技术栈用户的检索需求。

**人工质量审核机制** 所有收录的资源均经过至少两名社区贡献者的人工审核，确保内容质量、时效性与技术准确性，杜绝低质量内容与过期文档。

**全文检索与模糊匹配** 内置轻量级全文检索引擎，支持资源标题、描述、标签、作者等多字段联合检索，并兼容中文分词与拼音模糊搜索。

**收藏夹与个性化订阅** 注册用户可创建多个主题收藏夹，对感兴趣的资源进行归类保存，同时支持关注特定标签或分类，当有新资源收录时系统自动推送通知。

**社区共建与版本追踪** 每个资源条目均提供版本历史记录与变更日志，社区成员可提交更新请求或内容纠错，维护者审核后合并，保证资源信息的持续准确性。

**开放数据导出接口** 提供 RESTful API 与数据导出功能，支持 JSON、CSV 格式输出，方便用户将资源数据集成至个人知识库或内部文档系统中。

## 应用场景

技术团队内部知识库建设。技术负责人可使用 TechResource Hub 的资源分类体系与审核机制，快速搭建团队内部的技术文档导航系统，将分散在个人浏览器收藏夹、即时通讯群组、邮件中的技术链接统一管理，降低新成员上手时的信息迷失感。

个人开发者日常学习路径规划。开发者可根据自身技术方向，通过平台的多维度筛选功能快速获取从入门到进阶的系统性学习资源列表，避免在海量互联网信息中盲目摸索，提升学习效率。

开源项目文档站外链接管理。开源项目维护者可将项目 README 或官网中的“相关资源”、“扩展阅读”部分托管至 TechResource Hub，利用平台的版本追踪与更新通知机制，确保外部链接的长期有效性与内容同步。

技术社区内容聚合与推荐。技术社区运营方可接入平台 API，将优质外链资源自动同步至社区内容板块，丰富社区生态，同时通过平台的社区共建机制激励用户参与资源贡献，形成良性内容循环。

## 快速开始

以下命令演示如何在本机部署并运行 TechResource Hub 开发环境。

```bash
# 克隆项目仓库
git clone https://github.com/techresource-hub/techresource-hub.git

# 进入项目目录
cd techresource-hub

# 安装项目依赖（使用 npm）
npm install

# 配置环境变量，复制示例配置文件
cp .env.example .env

# 初始化本地数据库（SQLite 默认）
npm run db:init

# 启动开发服务器
npm run dev
```

生产环境部署请参考 `docs/deployment.md` 文档，支持 Docker Compose 一键部署与 Kubernetes 集群部署两种方式。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | >= 18.0.0 | 项目运行时环境，推荐使用 LTS 版本 |
| npm | >= 9.0.0 | 包管理器，用于安装与管理项目依赖 |
| SQLite | >= 3.35.0 | 默认数据库引擎，用于本地开发与轻量部署 |
| Redis | >= 6.2.0 | 缓存与会话存储，生产环境必需 |
| Elasticsearch | >= 8.0.0 | 全文检索引擎，可选组件，不安装则使用 SQLite FTS5 降级方案 |
| Docker | >= 20.10.0 | 容器化部署工具，生产环境推荐 |
| Docker Compose | >= 2.0.0 | 多容器编排工具，配合 Docker 使用 |

## 常见问题

问：部署后首次启动，数据库迁移失败，提示“SQLite 版本过低”怎么办？

答：TechResource Hub 依赖 SQLite 3.35.0 及以上版本以支持窗口函数和部分 JSON 操作。请检查系统 SQLite 版本：`sqlite3 --version`。若版本过低，可通过源码编译升级，或使用 Docker 镜像部署以绕过宿主机版本限制。生产环境强烈推荐使用 PostgreSQL 替代 SQLite。

问：如何批量导入私有收藏夹中的链接到本平台？

答：平台管理后台提供了“批量导入”功能，支持 HTML 书签文件（Netscape 格式）、CSV 文件以及 JSON 格式的数据导入。导入前请参照 `docs/features/import.md` 文档中的字段映射说明进行格式预处理。若收藏夹数量超过 1000 条，建议使用命令行工具 `scripts/bulk-import.js` 异步导入，避免 HTTP 请求超时。

问：资源链接失效或目标网站变更，平台如何应对？

答：系统内置了定时可用性检查任务（默认每日凌晨执行），对全部资源链接发起 HEAD 请求验证可访问性。对于连续三次检查失败的链接，系统会自动标记为“疑似失效”并通知提交者与最后更新人。社区成员也可在资源详情页点击“报告链接问题”按钮主动提交失效反馈，维护者确认后将更新链接或移除条目。
