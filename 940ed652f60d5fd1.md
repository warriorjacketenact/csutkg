# LinkVault Resource Aggregator

LinkVault is a high-performance, developer-oriented resource aggregation and navigation system designed to systematically catalog, validate, and expose distributed web resources through a unified query interface. The project targets developers, technical researchers, and information architects who need to maintain large-scale external link inventories with rigorous availability monitoring and metadata extraction capabilities.

Unlike traditional bookmark managers or simple link dump repositories, LinkVault implements a multi-layered storage backend with pluggable validators, automatic content fingerprinting, and RESTful APIs for both programmatic and human-readable access. The project solves the pervasive problem of link rot, contextual drift, and organizational friction when curating hundreds or thousands of external references across team environments. By treating each URL as a first-class entity with associated tags, health status, and semantic annotations, LinkVault transforms raw URI collections into actionable knowledge assets.

## 功能概览

**批量导入与归一化** - Accepts plain-text URL lists, CSV exports, browser bookmarks, and RSS feed entries. Automatically normalizes schemes, strips tracking parameters, and deduplicates against existing indexes.

**声明式健康检查** - Configurable validation pipelines that test HTTP status codes, response times, TLS certificate validity, and robots.txt compliance. Supports HEAD, GET, and conditional request methods with retry policies.

**标签与分类引擎** - Hierarchical tag system with inheritance rules, synonym mapping, and auto-tagging based on domain patterns, content-type headers, and HTML meta analysis.

**全文检索与过滤** - Built on inverted indexes with support for boolean queries, fuzzy matching, and field-specific searches (title, description, domain, tags, last-checked timestamp).

**快照与变更追踪** - Stores historical response hashes and status transitions, enabling regression detection and change notification webhooks for monitored resources.

**导出与集成适配器** - Provides JSON, YAML, Markdown table, and HTML snippet export formats. Includes webhook integrations for Slack, Discord, and generic HTTP endpoints.

**访问控制与审计** - Basic role-based permissions (viewer, editor, admin) with per-resource visibility toggles and full mutation logging for compliance purposes.

## 应用场景

**技术文档库维护** - Documentation teams managing external reference links in API specs, user guides, or tutorial repositories can use LinkVault to automatically verify that every cited resource remains accessible and up-to-date, reducing broken-link reports by over 80 percent.

**安全研究索引** - Security analysts collecting threat intelligence feeds, CVE references, and exploit database entries benefit from LinkVaults structured tagging and change detection, allowing rapid correlation of new vulnerabilities against existing link collections.

**学术文献补充材料** - Researchers compiling supplementary online resources for papers or preprints can organize hundreds of dataset links, code repositories, and interactive demos with persistent identifiers, ensuring reviewers and readers have reliable access long after publication.

**内部知识库聚合** - Enterprise engineering teams can maintain a curated internal portal of deployment dashboards, monitoring graphs, runbook links, and CI/CD pipeline views, with automated alerts when any critical internal endpoint becomes unreachable.

**开源项目依赖导航** - Open-source maintainers can track upstream documentation, related projects, community forums, and migration guides in a single structured catalog, making onboarding new contributors significantly smoother.

## 快速开始

The following commands clone the repository, install Python dependencies via Poetry, and start the development server with a sample dataset.

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
poetry install --no-dev
poetry run python -m linkvault.cli init --sample-data
poetry run uvicorn linkvault.api:app --host 0.0.0.0 --port 8000 --reload
```

After startup, access the web dashboard at http://localhost:8000/ui or query the REST API at http://localhost:8000/api/v1/links. The sample dataset includes approximately 250 seed URLs for immediate testing.

## 项目结构

The repository follows a modular monolith design with clear separation between domain models, storage adapters, validation pipelines, API layers, and CLI entrypoints.

```
linkvault-core/
├── src/
│   └── linkvault/
│       ├── __init__.py                     # 包版本与导出声明
│       ├── cli/
│       │   ├── __init__.py                 # 命令路由 (init, import, check, export)
│       │   ├── commands.py                 # 各子命令的具体实现与参数解析
│       │   └── formatters.py               # 终端输出格式化 (表格, JSON, 彩色状态)
│       ├── domain/
│       │   ├── __init__.py                 # 领域实体导出
│       │   ├── link.py                     # Link 聚合根：url, title, tags, status, checksum
│       │   ├── check_record.py             # 每次健康检查的结果快照 (status_code, latency, error)
│       │   └── tag.py                      # 标签值对象与层级路径解析
│       ├── storage/
│       │   ├── __init__.py                 # 存储接口与工厂
│       │   ├── sqlite_repository.py        # SQLite 实现：链接 CRUD、标签关联、检查历史
│       │   ├── migrations/                 # Alembic 迁移脚本 (schema versioning)
│       │   │   ├── versions/
│       │   │   └── env.py
│       │   └── in_memory_repository.py     # 用于测试的内存存储实现
│       ├── validators/
│       │   ├── __init__.py                 # 验证器链组装
│       │   ├── http_validator.py           # 异步 HTTP 探活 (支持重试、代理、超时)
│       │   ├── html_parser.py              # BeautifulSoup 元数据提取 (title, description, lang)
│       │   └── robots_checker.py           # robots.txt 缓存与规则匹配
│       ├── api/
│       │   ├── __init__.py                 # FastAPI 应用工厂
│       │   ├── routes/
│       │   │   ├── links.py                # GET /links, POST /links, GET /links/{id}

│       │   │   └── tags.py                 # GET /tags, GET /tags/{name}/links
│       │   ├── schemas.py                  # Pydantic 请求/响应模型
│       │   └── middleware.py               # 日志、CORS、异常处理中间件
│       ├── services/
│       │   ├── __init__.py                 # 服务层依赖注入
│       │   ├── import_service.py           # 批量导入 (去重、归一化、批量插入)
│       │   ├── check_scheduler.py          # 后台任务调度器 (apscheduler 集成)
│       │   └── export_service.py           # 导出为 JSON/YAML/Markdown 格式
│       └── utils/
│           ├── __init__.py                 # 工具函数导出
│           ├── url_normalizer.py           # 协议补全、路径清理、查询参数排序
│           ├── fingerprint.py              # 内容 SHA-256 哈希计算 (基于响应体采样)
│           └── logger.py                   # structlog 结构化日志配置
├── tests/
│   ├── unit/                               # 单元测试 (mock 外部依赖)
│   ├── integration/                        # 集成测试 (真实 SQLite + HTTP 请求)
│   └── fixtures/                           # 测试用样本数据 (sample_links.json)
├── docs/                                   # 完整文档 (参见文档导航章节)
├── pyproject.toml                          # Poetry 项目配置与依赖声明
├── poetry.lock                             # 锁定依赖版本 (已提交)
├── Dockerfile                              # 多阶段构建 (生产镜像约 120MB)
├── docker-compose.yml                      # 开发环境快速启动 (含 Redis 缓存)
├── Makefile                                # 常用开发任务 (format, lint, test, migrate)
└── README.md                               # 本文件
```

## 贡献指南

We welcome contributions of all forms, including bug reports, feature proposals, documentation improvements, and code patches. Please follow the steps below to ensure a smooth collaboration process.

2.  **Set up the development environment** - Run `poetry install --with dev` to install all development dependencies, including pytest, black, mypy, and pre-commit hooks. Activate the pre-commit hook with `pre-commit install` to automatically run linters before each commit.

## 常见问题

**Q: How does LinkVault handle rate-limiting or blocking from target servers during health checks?**

A: The HTTP validator implements an exponential backoff retry strategy with configurable initial delay and maximum retries. Additionally, you can set a global requests-per-second throttle and a per-domain cooldown period to avoid overwhelming external services. For servers that return 429 or 503 status codes, the validator marks the link as degraded rather than failed and schedules a re-check after the cooldown period.

**Q: Can LinkVault import existing bookmarks from browsers or other tools?**

A: Yes, the CLI includes import adapters for Netscape HTML bookmarks (exported from Firefox, Chrome, Safari), CSV files with columns for URL, title, and tags, and simple newline-delimited text files. You can also write a custom adapter by implementing the `ImporterProtocol` interface and placing it in the `importers/` plugin directory.

**Q: What happens when a monitored link changes its content but keeps the same URL?**

A: The fingerprinting utility compares the SHA-256 hash of the response body (or a configurable byte prefix) against the previous stored hash. When a change is detected, LinkVault records a content mutation event, updates the last-modified timestamp, and optionally sends a webhook notification. The previous content hash remains in the history table for audit and rollback comparisons.
