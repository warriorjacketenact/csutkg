# WebLink Catalog System

WebLink Catalog System 是一个面向技术社区与内容聚合场景的高性能外链资源目录平台。该项目定位于为开发者、技术内容创作者以及运维人员提供一套标准化、可扩展的影视及多媒体资源链接管理方案，能够高效处理大量动态 URL 资源的收录、分类、检索与状态监控。

本项目并非一个通用爬虫或简单书签管理工具，而是一个专门针对数字资源 ID 体系设计的结构化存储与展示系统。目标用户包括需要批量维护多媒体资源入口的站点管理员、希望建立自有资源导航页面的团队，以及需要对接第三方数据源进行内容聚合的开发者。通过本系统，用户可以快速构建出具备清晰层次关系、稳定访问路径的资源列表服务。

## 功能概览

批量资源收录：支持一次性导入大量资源 ID，系统自动根据预设规则生成访问路径并完成初步可用性校验。

分类索引生成：根据资源 ID 的元数据特征，自动划分至不同的内容分类目录，便于后续检索与展示。

动态链接状态监控：定时检测已收录资源的访问状态，标记异常链接并生成报告，辅助管理员及时清理或更新无效条目。

多层级目录结构管理：提供树状目录管理界面，支持深度达 5 级以上的子分类嵌套，满足复杂资源组织需求。

访问统计与热度排序：记录每个资源链接的点击频次与访问来源，支持按热度、收录时间、ID 大小等多种维度排序输出。

数据导入导出接口：提供 JSON 与 CSV 格式的数据批量导入导出功能，便于与其他系统进行数据交换。

只读模式与公开访问控制：支持将系统配置为公开只读状态，使资源列表能够安全地对外发布，无需担心数据被意外篡改。

全文检索与快速定位：集成轻量级全文检索引擎，支持通过资源 ID 或自定义标签快速筛选目标条目。

## 应用场景

站点资源导航页构建：团队或个体站长可以使用本系统快速生成一个包含数百个多媒体资源入口的导航页面，所有链接按照自定义分类结构清晰陈列，无需手动编写 HTML 列表。

内容聚合平台数据中转：作为中间层服务，接收来自上游数据源的资源 ID 列表，经过结构化处理后提供给前端应用或第三方客户端使用，有效隔离原始数据源与最终展示层。

运维监控与链接治理：运维人员通过系统的状态监控功能，定期扫描资源链接的有效性，及时发现并处理失效条目，保证对外提供服务的资源质量。

私有化资源库管理：机构内部可将各类数字资源入口统一收录至本系统，建立内部知识库或媒体资产索引，提升资源复用效率。

## 快速开始

以下步骤将帮助您在本地环境中快速启动 WebLink Catalog System 实例。

```bash
# 克隆项目仓库至本地
git clone https://github.com/weblink-catalog/weblink-catalog-system.git

# 进入项目根目录并安装依赖
cd weblink-catalog-system
pip install -r requirements.txt

# 初始化数据库并启动开发服务器
python manage.py initdb
python manage.py runserver --host 0.0.0.0 --port 8080
```

## 贡献指南

提交 Issue 报告缺陷或功能请求：请使用 GitHub Issues 提交详细的问题描述，包括复现步骤、预期行为与实际行为对比，并附上相关日志或截图。

遵循代码风格规范：所有 Python 代码必须符合 PEP 8 标准，JavaScript 代码遵循 ESLint 默认配置。提交前请使用 pre-commit 钩子进行自动格式化。

编写测试用例：新增功能或修复缺陷时，需在 tests 目录下补充对应的单元测试或集成测试，确保测试覆盖率达到 80% 以上。

提交 Pull Request：从 develop 分支创建功能分支，完成开发后发起 PR 至 develop 分支。PR 描述中需说明变更内容、测试结果以及相关 Issue 编号。

更新文档：若变更涉及用户可见功能或配置项，须同步更新 docs 目录下的对应文档，并确保示例代码可正常运行。

## 常见问题

系统启动后无法访问资源链接，页面返回 404 错误。

此情况通常是由于资源 ID 对应的条目在数据库中不存在或未被正确收录。请首先检查数据导入步骤是否完整执行，随后可通过管理后台的「资源校验」功能手动触发单条链接的可用性检测。若批量导入后大量链接失效，建议检查原始数据源格式是否符合系统要求的 JSON 或 CSV 结构。

如何对已收录的资源进行批量更新或删除？

系统管理后台提供了资源列表的批量操作入口。管理员可以勾选多条记录后，选择「批量更新元数据」或「批量删除」动作。批量更新时需上传包含新字段值的 CSV 文件，系统将按 ID 匹配后进行覆盖。批量删除操作会二次确认，确认后数据将被软删除，保留 30 天后自动清理。

生产环境下如何配置 HTTPS 访问？

本项目本身不处理 TLS 终止，推荐使用 Nginx 作为前端反向代理，在 Nginx 配置中启用 SSL 证书并设置 proxy_pass 指向本机运行的服务端口。具体配置示例可参考 docs/deployment.md 中的「启用 HTTPS」章节，其中包含了证书申请、Nginx 配置片段以及重定向 HTTP 至 HTTPS 的完整方案。
