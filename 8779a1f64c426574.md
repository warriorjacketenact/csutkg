# VODLink Indexer

VODLink Indexer 是一个面向多媒体资源索引与外部链接治理的开源工具集，专注于对海量视频分发链路、内容标识符与资源定位符进行结构化整理与规范化输出。该项目主要服务于内容运营团队、数据中台开发者以及自动化测试工程师，帮助其在多源视频数据汇聚、链接有效性校验、资源分类归档等场景下提升处理效率。VODLink Indexer 不提供具体视频内容，也不对任何外部资源的可用性作出保证，仅作为链接元数据的整理与展示框架，便于用户快速定位和分类管理大规模资源列表。

## 功能概览

**批量链接导入与解析** 支持从纯文本、CSV 或 JSON 格式中批量导入链接，自动识别路径结构并提取资源标识符。

**自定义分类标签系统** 允许用户为每条链接添加多级标签，支持按域名、路径前缀、参数特征进行自动打标。

**链接去重与状态检测** 内置轻量级去重引擎，可对导入的链接列表进行 MD5 哈希比对；同时支持配置 HTTP 头检测，标记异常状态。

**结构化目录树生成** 基于链接路径层级自动生成 ASCII 目录树，便于可视化理解资源分布结构。

**元数据模板导出** 支持将整理后的链接列表导出为 Markdown 表格、JSON 映射或纯文本清单，适配不同下游系统。

**外部资源引用隔离** 所有用户提供的原始链接均以纯文本列表形式独立呈现，不进行任何主动请求或预取，确保引用行为透明可控。

**可扩展过滤器链** 提供中间件风格的过滤器接口，开发者可自行编写规则对链接进行筛选、排序或重写。

**命令行交互与脚本集成** 提供 CLI 入口，支持通过参数指定输入文件、输出格式和过滤条件，便于嵌入自动化流水线。

## 应用场景

**视频分发链路归档** 内容运营团队每日需处理数百条来自不同编码节点或 CDN 分发的视频片段链接。VODLink Indexer 可帮助将这些链接按域名和路径前缀分组，生成易于审阅的清单，并标记出可能存在的重复或过期条目，减少人工整理耗时。

**测试环境数据构造** 自动化测试工程师在构造模拟响应数据时，往往需要一批格式规范但无实际请求意图的占位链接。本项目提供的示例数据集可被直接引用为测试夹具，配合过滤器链快速生成符合特定路径模式的链接集合，降低测试数据准备成本。

**资源迁移前勘测** 在计划将视频存储从旧系统迁移至新对象存储服务时，运维人员可通过本工具导出当前所有引用链接的目录结构和分布统计，评估迁移范围并规划路径映射规则，避免迁移过程中遗漏关键资源。

**开源文档外链治理** 开源项目维护者在撰写文档或 README 时，经常需要引用大量外部示例链接。VODLink Indexer 的导出功能可将零散的链接整理为统一的资源列表章节，并保持原始地址的完整性，减少手动复制导致的格式错误。

## 快速开始

以下步骤演示如何获取 VODLink Indexer 源码、安装依赖并运行一次基础的链接清单生成任务。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/vodlink-indexer.git
cd vodlink-indexer

# 安装依赖（使用 pip 或 poetry，以 pip 为例）
pip install -r requirements.txt

# 运行基础解析示例，将 links.txt 中的链接按域名分组输出到 output/grouped.md
python cli.py parse --input samples/links.txt --output output/grouped.md --group-by domain
```

## 安装要求

运行 VODLink Indexer 需要以下软件环境与依赖库支持。建议在 Python 3.8 及以上版本的虚拟环境中安装。

| 依赖项 | 必需版本 | 说明 |

| pydantic | 2.0.0 或更高 | 数据模型校验，用于链接条目和配置对象的类型安全 |
| pytest | 7.0.0 或更高 | 仅在开发测试时需要，用于运行单元测试和集成测试 |
| black | 23.0.0 或更高 | 代码格式化工具，仅在提交代码前格式化时使用 |
| pre-commit | 3.0.0 或更高 | Git 钩子管理，用于在提交前自动执行代码检查 |
| mkdocs | 1.5.0 或更高 | 可选依赖，用于在本地构建项目文档站点 |

## 项目结构

项目采用分层模块化设计，核心解析引擎与输出格式化器解耦，便于替换或扩展。以下为源码目录树及其职责说明。

```
vodlink-indexer/
├── cli.py                      # CLI 入口，注册 parse、check、export 子命令
├── config/
│   ├── __init__.py             # 配置加载与合并逻辑
│   └── defaults.yaml           # 默认过滤器链与输出格式配置
├── core/
│   ├── __init__.py             # 核心模块暴露接口
│   ├── parser.py               # 链接解析器，负责提取域名、路径、参数
│   ├── loader.py               # 从文件或标准输入加载链接列表
│   └── models.py               # Pydantic 数据模型：LinkEntry, LinkGroup
├── filters/
│   ├── __init__.py             # 过滤器注册与链式调用
│   ├── base.py                 # 抽象过滤器基类
│   ├── dedup.py                # 去重过滤器
│   └── tagger.py               # 自动标签生成器（基于路径正则）
├── output/
│   ├── __init__.py             # 输出格式注册
│   ├── markdown.py             # Markdown 表格与列表渲染器
│   └── json_exporter.py        # JSON 序列化导出器
├── tests/
│   ├── test_parser.py          # 解析器单元测试
```

## 常见问题

**问：VODLink Indexer 是否会主动请求或验证外部链接的可达性？**

答：默认情况下，工具仅对链接文本进行静态解析和结构化整理，不会发起任何 HTTP 请求。若用户启用状态检测功能（通过 --check-status 参数），工具会使用 requests 库发送 HEAD 请求，但该功能默认关闭，且检测结果仅作为参考，不影响链接的存储与导出。

**问：资源列表中包含大量链接，解析性能如何？**

答：对于 250 条级别的链接列表，解析和去重操作可在毫秒级完成。性能瓶颈主要出现在输出大型 Markdown 表格时的渲染开销，但实测 1000 条以内均在可接受范围。若需要处理万级链接，建议使用 --output json 配合流式写入模式。

**问：能否自定义链接的分组逻辑？**

答：可以。您可以通过编写自定义过滤器类并注册到 filters 目录下，重写 group_key 方法即可实现按域名、路径深度、查询参数或其他任意特征分组。具体示例请参考 docs/developer/custom-filters.md。
