# LinkVault Resource Aggregator

LinkVault is a curated, version-controlled repository of high-value external resource links, designed for developers, researchers, and content curators who need programmatic access to a structured, deduplicated, and persistently maintained collection of web references. Unlike bookmark managers or sprawling wiki pages, LinkVault treats every link as a first-class data entity, storing metadata such as source domain, category tags, retrieval timestamp, and optional validation status. The project currently catalogs batch 88 of 1241, containing 250 distinct URLs, and provides tooling for link health checking, format normalization, and export to multiple interchange formats.

This project targets system administrators building internal knowledge bases, data engineers constructing ETL pipelines that require stable reference datasets, and open-source documentation maintainers who need to embed verified external references without manually curating link lists. LinkVault does not host or proxy content; it provides deterministic, machine-readable inventory of URLs with companion scripts for verification and reporting. All links are stored in plain-text manifest files with strict line-based formatting, enabling seamless integration with grep, awk, sed, and custom automation scripts.

## 功能概览

**批量链接导入** - 支持从纯文本文件、CSV 或标准输入流批量添加 URL，自动去重并分配内部唯一标识符。

**域名聚合统计** - 按根域名和二级域名分组统计链接分布，输出 JSON 格式的聚合报告，便于识别来源集中度。

**健康状态检查** - 内置异步 HTTP HEAD 请求检查器，可配置超时和重试策略，标记无法访问或重定向的链接。

**格式规范校验** - 强制校验 URL 协议头（http/https）和域名格式，拒绝不符合 RFC 3986 规范的条目并生成错误日志。

**多格式导出** - 支持将链接库导出为纯文本列表、JSON 数组、CSV 表格或 HTML 无序列表，适配不同下游使用场景。

**标签分类系统** - 允许为每个链接附加自由标签，支持按标签过滤和搜索，标签索引存储为独立的倒排文件。

**变更审计日志** - 记录每次链接的新增、删除和修改操作，包含时间戳和操作者信息，支持回滚至历史版本。

**定时同步任务** - 提供 cron 兼容的调度脚本，可定期执行健康检查并生成状态变更报告，通过邮件或 Webhook 推送通知。

## 应用场景

文档站外部引用管理：开源项目文档站往往引用大量第三方资源，包括 API 参考、教程和工具站点。LinkVault 可维护这些引用的清单，定期检查每个链接是否仍然有效，避免文档中出现死链影响用户体验。

数据采集管道种子库：构建网络爬虫或数据采集系统时，需要一个初始种子 URL 列表。LinkVault 提供结构化的种子库，支持按域名或标签筛选，并可输出为 Scrapy 或 Apache Nutch 兼容的格式，简化采集系统的初始化配置。

知识库资源索引：企业内部知识库或技术团队 Wiki 通常包含大量外部资源链接。LinkVault 可作为这些链接的后端存储引擎，提供统一的查询、分类和健康监控接口，避免资源散落在多个页面中难以维护。

自动化报告生成：运营团队需要定期生成竞品站点监控报告或行业资源汇总周报。LinkVault 的导出功能可结合模板引擎，自动生成包含链接列表和可用性状态的 HTML 或 Markdown 报告，减少手工整理工作。

## 快速开始

以下命令序列演示如何获取 LinkVault 仓库、安装依赖并执行初始数据导入：

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

pip install -r requirements.txt

python linkvault.py import --input ./manifests/batch_88.txt --format plain
python linkvault.py check --concurrency 10 --timeout 5
python linkvault.py export --format json --output ./exports/batch_88.json
```

首次运行时，系统会在项目根目录创建 `data/` 和 `logs/` 目录，分别存放链接清单和操作日志。导入命令执行后会显示去重前后的记录数对比，以及校验失败的条目列表。健康检查命令默认对全部已导入链接发起 HTTP HEAD 请求，结果更新至状态字段，可通过 `report` 子命令查看汇总统计。

2. 添加或修改链接清单：在 `manifests/` 目录下编辑对应的批次文件，每行仅包含一个 URL，不得包含多余空白字符或注释。若新增批次，需遵循 `batch_编号.txt` 的命名模式。

3. 编写或更新测试用例：在 `tests/` 目录下为新增功能编写相应的单元测试，确保代码覆盖率达到 80% 以上。运行 `pytest tests/` 验证所有测试通过。

## 常见问题

问：导入时提示 "invalid url format"，但我的链接看起来是完整的，问题出在哪里？

答：导入器严格执行 RFC 3986 校验规则，要求 URL 必须包含协议头（http:// 或 https://）且域名部分不能包含非法字符。请检查链接中是否包含未转义的空格、中文标点或缺失的斜杠。若链接包含查询参数，确保使用 `?` 分隔且参数键值对使用 `&` 连接。可使用 `validator.py` 单独测试 URL 的合规性。

问：健康检查命令运行很慢，能否加速？

答：健康检查默认使用 10 个并发工作线程，可通过 `--concurrency` 参数调整。若网络环境稳定，可增至 20-30 个并发。同时可缩短超时时间（`--timeout` 参数）至 3 秒。若仍显缓慢，可能是目标服务器限流，建议在非高峰时段执行，或使用 `--rate-limit` 参数控制每秒请求数。

问：如何从导出结果中筛选特定状态的链接？

答：导出命令支持 `--filter` 参数，接受 `status=ok`、`status=failed` 或 `status=pending` 等键值对。也可将导出结果通过管道传递给 `jq` 处理 JSON 格式，例如 `linkvault export --format json | jq '.[] | select(.status=="failed")'`。对于 CSV 导出，可使用 `grep` 或 `awk` 进行行过滤。
