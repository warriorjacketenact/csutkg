# ResourceForge

ResourceForge 是一个面向技术研究与内容聚合场景的轻量级外链资源管理与导航系统。项目定位于为开发者、技术内容创作者、数据分析师以及信息整理人员提供一套结构清晰、可扩展的 URL 资源收录与分类框架，帮助用户从海量零散链接中快速建立可维护的知识索引。

ResourceForge 本身并不存储任何实体内容，而是通过规范化的目录结构与元数据标注方式，将原始 URL 资源按来源、类型或主题进行归类整理，降低信息检索与回溯成本。项目内置了多级标签系统、资源状态标记、以及自动化的链接有效性检查接口，适合用作个人知识库、团队共享资源池或静态站点生成器的数据源。当前批次收录资源共计 250 项，涵盖技术文档、社区讨论、工具站点与数据参考等多个维度。

## 功能概览

**层级化资源目录**：支持无限层级的分类嵌套，每条资源可同时归属于多个逻辑目录，便于从不同维度检索。

**原始链接强制保留**：系统对用户输入的 URL 不做任何协议补全、域名标准化或路径改写，确保每一枚链接均以原始形态存储与展示，避免引入额外跳转或解析误差。

**元数据标注系统**：每条资源可附加来源批次、收录时间、失效风险等级、内容摘要与自定义标签，支持批量编辑与导出。

**快速过滤与排序**：基于资源 ID、域名、路径关键词或收录批次进行多条件组合过滤，并支持按时间或字母序排列。

**Markdown 原生渲染**：所有资源列表与目录结构均以纯 Markdown 形式输出，可直接集成至 GitHub、GitBook 或任何支持 Markdown 的文档平台，无需额外解析器。

**批量导入与校验**：提供 CSV 与纯文本格式的批量导入接口，导入时自动检测重复链接并生成冲突报告，同时支持定期对已收录链接进行 HTTP 状态检查。

## 应用场景

**技术博客的外部参考索引**：技术作者在撰写文章时，可将引用的官方文档、开源仓库、技术提案等外链统一收录至 ResourceForge，按主题分类后嵌入博客仓库，方便读者查阅原文且不破坏正文流畅度。

**开源项目依赖链记录**：大型开源项目往往依赖数十个外部服务或代码库，利用 ResourceForge 建立依赖资源清单，可清晰记录每个依赖项的官方主页、备用镜像与历史版本存档地址，降低维护者交接成本。

**数据分析任务的数据源登记**：数据采集或分析项目通常涉及大量数据源 URL，通过 ResourceForge 按数据源类型、更新频率、访问权限进行标注，可有效避免因链接丢失导致的分析流程中断。

**团队内部知识库的素材池**：产品、设计、运营团队可将日常调研中发现的优秀案例、设计参考、竞品动态等链接统一归档，为知识库文章提供可追溯的原始素材支撑。

## 快速开始

以下命令将 ResourceForge 项目克隆至本地，安装必要依赖并启动开发服务器。

```bash
git clone https://github.com/resourceforge/resourceforge.git
cd resourceforge
npm install
npm run dev
```

执行完毕后，项目将在本地 3000 端口启动开发环境，访问 http://localhost:3000 可查看当前资源列表与分类导航。若需构建生产版本，请使用 `npm run build` 命令，输出目录为 `dist/`。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.0+ | 是 | 项目运行时环境，推荐使用 LTS 版本 |
| npm 9.0+ | 是 | 包管理器，用于安装项目依赖 |
| Git 2.30+ | 是 | 用于克隆仓库及版本控制 |
| 现代浏览器 | 否 | 仅开发调试时需要，生产环境为纯静态输出 |
| Python 3.9+ | 否 | 若使用内置链接有效性检查脚本，需 Python 环境及 requests 库 |
| SQLite 3 | 否 | 若启用本地元数据缓存，需 SQLite 支持 |

│   │   ├── collector.js          # 资源采集与去重控制器
│   │   ├── validator.js          # URL 格式与可达性校验器
│   │   └── registry.js           # 资源注册与查询接口

│   │   └── export.js             # 数据导出命令
│   ├── web/                      # Web 展示层
│   │   ├── routes/               # 路由定义
│   │   ├── views/                # 模板视图
│   │   └── static/               # 静态资源 (CSS, JS)

## 常见问题

**问：ResourceForge 是否会对收录的 URL 进行内容缓存或代理？**

答：不会。ResourceForge 仅存储 URL 字符串及其元数据，不缓存任何页面内容，也不提供代理访问服务。所有链接均直接指向原始目标地址，用户点击后将由浏览器正常发起请求。

**问：如何导入包含大量 URL 的文本文件？**

答：使用 `npm run import -- --file=./path/to/urls.txt --batch=1219` 命令，文件每行一个 URL。系统会自动识别重复记录并生成导入报告，冲突项会记录至 `data/duplicates.log`，用户确认后可强制覆盖。

**问：链接有效性检查的原理是什么？**

答：检查脚本会向每个 URL 发送 HEAD 请求，根据返回的 HTTP 状态码判断链接是否存活。对于需要特殊 User-Agent 或 Cookie 的站点，可在 `config/default.yaml` 中配置请求头策略。检查结果按资源 ID 存储于 `data/metadata/` 目录，并生成汇总报告。
