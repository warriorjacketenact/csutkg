# LinkPilot 资源导航系统

LinkPilot 是一个面向技术社区的内容聚合与导航系统，专注于收集、分类和呈现高质量的外部技术资源链接。该项目定位为开发者与研究人员的技术外延知识库，帮助用户在文档阅读、框架选型、故障排查和前沿技术追踪等场景中快速定位可信赖的参考材料。

LinkPilot 不生产内容，而是通过人工筛选和社区贡献的方式，维护一个结构化的外链资源索引。项目内置资源健康度检查、访问时效性监控和标签化分类机制，适用于个人书签管理、团队知识沉淀以及开源文档站点的外链补充层。

## 功能概览

**资源收录与去重** 系统提供批量链接导入接口，支持基于 URL 特征和页面摘要的自动去重，避免重复条目污染索引库。

**分类标签体系** 每条资源可附加多个维度标签，包括技术领域、内容类型、适用人群和难度等级，支持多条件组合筛选。

**健康度巡检** 内置定时任务，对已收录链接进行 HTTP 状态码检测和响应时间记录，自动标记失效或响应过慢的资源。

**全文检索与过滤** 基于标题、描述、标签和 URL 关键字的全文搜索，支持按分类、状态和更新时间排序与过滤。

**访问统计与热度排序** 记录每个外链的点击次数和最近访问时间，提供按热度、新增度和稳定度的多种排序视图。

**导入导出兼容性** 支持 JSON、CSV 和 Markdown 列表格式的批量导入导出，便于与其他书签工具或文档系统进行数据交换。

**用户贡献接口** 提供标准化的资源提交表单和 API 端点，允许社区成员推荐新链接并附上推荐理由和分类建议。

**快照与备注功能** 支持为每个链接添加自定义备注、摘要快照和版本标记，方便团队内部共享上下文信息。

## 应用场景

技术文档站点外链增强。技术博客、开源项目文档或内部知识库往往需要引用大量外部参考资料。LinkPilot 可作为独立服务部署，通过 API 或嵌入组件为现有文档页面提供关联资源推荐和失效链接替换建议。

团队技术书签共享。研发团队可以将 LinkPilot 作为统一的书签管理后台，成员在开发过程中发现的优质教程、工具站点和解决方案链接均可集中收录，并自动同步给全体成员，减少重复查找和知识孤岛。

个人学习路径管理。学习者可以使用 LinkPilot 按技术栈和难度等级组织自己的学习资源，通过标签和分类体系构建清晰的学习路径，并结合健康度巡检避免收藏链接失效。

社区资源精选集运营。技术社区或开源组织可以利用 LinkPilot 维护对外推荐的资源精选列表，通过贡献审核机制保证链接质量，并以公开页面形式对外展示，增强社区的技术影响力。

## 快速开始

以下步骤帮助您在本地环境快速启动 LinkPilot 服务。

```bash
# 克隆代码仓库
git clone https://github.com/linkpilot/linkpilot-core.git
cd linkpilot-core

# 安装项目依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化配置文件并启动服务
cp .env.example .env
# 编辑 .env 文件填入必要的数据库连接信息
python manage.py migrate
python manage.py runserver
```

启动成功后，访问本地控制台端口进行初始管理员账号创建。生产环境部署请参考文档导航中的部署指南。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，建议使用 3.11 LTS |
| PostgreSQL | 14.0 及以上 | 主数据存储，存储资源条目、标签和用户数据 |
| Redis | 6.2 及以上 | 缓存与任务队列后端，用于健康度巡检和统计计数 |
| Node.js | 18.0 及以上 | 仅用于前端静态资源构建，运行时可无需 Node |
| Nginx | 1.20 及以上 | 生产环境反向代理与静态文件服务推荐 |
| Celery Worker | 5.3 及以上 | 异步任务执行器，独立进程运行健康巡检任务 |
| Elasticsearch | 8.0 及以上（可选） | 启用全文搜索增强时使用，非必需但推荐 |

## 贡献指南

提交新资源推荐。通过项目提供的提交表单或 API 接口提交您希望收录的链接，提交时需附带资源标题、简短描述以及至少一个分类标签。提交后系统会进行自动去重和基础格式校验。

参与健康度巡检维护。您可以在本地运行健康巡检脚本或通过后台手动触发巡检任务。发现失效链接后，可以通过管理后台标记为失效或更新为有效的新地址。

改进分类体系与标签库。如果您发现现有标签无法准确描述某些资源，可以在 GitHub Issues 中提出新增标签或调整分类结构的建议，并附上理由和示例资源。

完善文档与翻译。文档目录下的所有 Markdown 文件均欢迎修订，包括快速开始指南、API 参考和部署手册。您也可以将文档翻译为其他语言，以便更广泛的社区使用。

报告问题与提议功能。使用 GitHub Issues 提交 bug 报告或功能请求，请尽量提供复现步骤、环境信息和日志片段。Pull Request 请基于 dev 分支提交，并确保所有测试用例通过。

## 常见问题

Q: 系统支持同时管理多少个资源链接？是否有性能瓶颈？
A: 系统设计上单实例可管理十万级链接，健康巡检采用 Celery 异步队列分批执行，不会阻塞主请求。检索与过滤依赖 PostgreSQL 索引，数据量超过五万条时建议启用 Elasticsearch 以获得更快的全文搜索响应。

Q: 如何迁移现有书签或收藏夹数据到 LinkPilot？
A: 系统支持从 Chrome 书签导出的 HTML 文件、Raindrop.io 的 CSV 导出以及通用 JSON 格式导入。您可以使用管理后台的导入功能，或使用 scripts/ 目录下的转换脚本进行预处理后再批量导入。

Q: 健康巡检会频繁访问外部链接导致被目标站点屏蔽吗？
A: 巡检任务默认使用较低的并发数（并发 5 个请求）和随机的 User-Agent，且每个请求间隔至少 2 秒。您可以在配置中调整巡检频率和并发参数，生产环境建议每周执行一次全量巡检，每日执行增量巡检即可。
