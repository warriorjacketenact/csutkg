# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究者、内容策展人和开源项目维护者的外部资源链接管理与结构化汇总工具。该项目并非一个传统的爬虫或采集系统，而是一个基于 Markdown 与静态站点生成理念的资源索引框架。它通过标准化的文档模板和自动化校验脚本，帮助用户将散落在网络各处的优质 URL（如技术博文、系列教程、API 文档、数据面板等）整理为具备清晰分类、可检索、可版本控制的本地知识库。

LinkVault 主要服务于以下群体：需要维护大量外部参考链接的技术文档撰写者、开展大规模文献调研的科研助理、以及希望为开源社区提供高质量外链导航的站点维护人。项目核心解决的是链接失效、分类混乱、引用格式不统一以及批量导入导出效率低下的问题，提供了一套从原始 URL 清洗、元数据提取到目录树生成、静态站点预览的完整工作流。

## 功能概览

**批量链接导入与去重**：支持从纯文本、CSV 及 JSON 文件批量导入 URL，自动检测并移除重复项，同时对同一域名下的资源进行智能分组。

**元数据自动补全**：通过可插拔的解析器，尝试从目标页面标题、描述或路径结构中提取分类标签与发布时间，生成基础元数据表。

**目录树映射生成**：依据用户自定义的映射规则（如按域名、按一级路径或按文件扩展名），将链接自动归类至虚拟目录树，并输出可视化的 ASCII 结构图。

**链接状态健康检查**：内置异步 HTTP 探针，可定期校验列表中的每个链接是否可访问，对返回 4xx 或 5xx 状态的记录进行高亮标记，并生成失效报告。

**多格式导出支持**：除了标准的 Markdown 表格输出，还支持导出为 JSON 结构化数据、RSS 订阅源以及适用于 Hugo 或 VuePress 的主题数据文件。

**模板变量替换引擎**：允许用户在批量导入时使用预设模板字段（如 {category}、{year}、{source_domain}）自动填充描述或标签，减少重复手工录入。

**交互式过滤与查询**：在命令行界面下提供基于正则表达式的快速过滤功能，用户可按状态码、域名后缀或自定义标签筛选并批量编辑记录。

**变更日志审计**：每次对资源列表进行增删改操作时，自动记录操作类型、时间戳与影响条目数，便于团队协作时追溯修改历史。

## 应用场景

**技术文档中心的外部参考管理**：当企业或开源项目维护一份大型技术白皮书或用户手册时，其中引用了大量外部规范、论文链接和第三方工具主页。LinkVault 可以将这些引用链接统一托管于仓库的 references/ 目录下，并通过健康检查功能周期性提醒失效引用，确保文档的长期可验证性。

**学术研究阶段的文献汇总**：研究人员在开展系统文献综述时，通常需要整理数十至数百篇论文的 DOI 或项目主页。LinkVault 允许研究者将零散的浏览器书签导出为 CSV，一次性导入项目，并利用元数据补全功能自动抓取论文标题和摘要片段，生成结构化的待阅读清单。

**开源导航站点的数据源维护**：社区驱动的技术导航站点往往需要定期更新收录的框架、工具和博客链接。维护者可以使用 LinkVault 管理原始数据源，通过分类导出功能直接生成站点所需的 JSON 或 YAML 数据文件，避免手动编辑前端代码带来的格式错误。

**DevOps 监控面板的资源依赖梳理**：在构建微服务监控大盘时，运维人员需要罗列所有依赖的中间件管理地址、日志查询入口和告警回调 URL。LinkVault 可将这些内网地址（或公网地址）集中管理，并通过健康检查模块快速定位因版本升级或 IP 变更导致的不可用端点。

## 快速开始

以下指令演示如何通过 Git 克隆项目、安装依赖并运行一次完整的链接导入与校验流程。请确保在 Bash 或 Zsh 环境下执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault-core.git

# 进入项目根目录
cd linkvault-core

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行示例导入任务（将示例数据导入并生成资源列表）
python linkvault.py import --input samples/links.csv --output output/resource_list.md
```

执行上述指令后，程序将在 output/ 目录下生成一份 Markdown 格式的资源列表，同时输出校验日志至 console。若需启动内置的 Web 预览服务，可继续执行：

```bash
python linkvault.py serve --port 8080
```

访问 http://localhost:8080 即可查看当前资源库的 HTML 预览版。

## 项目结构

项目采用模块化分层设计，核心逻辑与用户数据完全分离。以下为 ASCII 目录树及其对应职责说明：

```
linkvault-core/
├── cli/                                 # 命令行接口模块，封装所有用户交互入口
│   ├── __init__.py                      # 注册子命令（import, check, export, serve）
│   ├── parser.py                        # 解析用户输入的参数并路由至对应处理器
│   └── formatter.py                     # 控制台输出格式化（颜色、表格、进度条）
├── core/                                # 核心业务逻辑层，与外部依赖解耦
│   ├── importer.py                      # 实现 CSV/JSON/TXT 导入，含去重算法
│   ├── checker.py                       # 异步链接状态检查器，可配置并发数与超时
```

## 贡献指南

欢迎各类形式的贡献，包括但不限于新增解析器插件、完善文档、提交链接分类建议以及报告缺陷。请遵循以下步骤以保证协作流程顺畅。

第一，查阅 issue 列表确认当前待办事项。若准备提交新功能或较大规模的修改，建议先创建一个 issue 说明你的意图，避免与其他贡献者的工作产生冲突。

第二，从仓库开发分支（dev 或 main）拉取新特性分支，分支命名规范为 feature/功能简述 或 fix/问题简述。所有代码变更需附带对应的单元测试，并确保测试套件在本地全部通过（执行 pytest 命令）。

第三，针对新增的外部站点解析器，请继承 plugins/base_parser.py 中的抽象类，并实现 fetch_title() 和 fetch_publish_date() 方法。若目标站点存在反爬机制，可在方法内部增加自定义请求头或代理配置，但禁止硬编码个人凭证。

第四，更新 docs/ 目录下的相关文档。如果新增了命令行参数，必须在 docs/commands.md 中添加对应说明；如果修改了配置文件格式，则需同步更新 docs/configuration.md 中的示例。

第五，提交 Pull Request 至主仓库。PR 描述中请清晰列出变更点、测试结果以及是否向下兼容。代码审查通过后，将由维护者合并至主分支，并自动触发 CI 流水线进行构建与发布。

## 常见问题

问：导入 CSV 文件时，程序提示“编码错误”，应如何解决？

答：该错误通常由源文件编码非 UTF-8 导致。请使用文本编辑器（如 VS Code 或 Notepad++）将 CSV 文件另存为 UTF-8 编码格式，并确保首行包含列头（如 url, title, category）。若文件过大无法直接另存，可使用 iconv 命令行工具进行转换：iconv -f GBK -t UTF-8 input.csv > output.csv。转换后重新运行导入命令即可。

问：健康检查模块对某些内网地址返回超时，但浏览器可以正常访问，怎么处理？

答：默认超时时间为 5 秒，且强制使用 IPv4 解析。若内网环境延迟较高或采用 IPv6，可修改 config.yaml 中的 checker.timeout 和 checker.force_ipv4 参数。另外，部分内网地址可能需要配置代理，请在 config.yaml 的 checker.proxy 字段中设置 HTTP/HTTPS 代理地址。调整参数后再次执行 linkvault check 命令即可。

问：生成的 Markdown 资源列表中，链接顺序与导入时的顺序不一致，如何控制排序？

答：项目默认按照链接的域名和路径进行字典序升序排列，以保证多次导出的稳定性。若需维持原始顺序，请在导入时使用 --preserve-order 标志，或在 config.yaml 中将 sort_by 字段设置为 none。同时，您也可以通过自定义排序键（如按标题长度或状态码）进行排序，具体用法请参考 docs/commands.md 中关于 --sort-key 参数的说明。
