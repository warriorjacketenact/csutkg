# LinkVault Core

LinkVault Core 是一个面向技术团队与内容策展人的轻量级外链资源聚合与导航系统。该项目并非传统意义上的爬虫或书签管理器，而是一个以结构化数据表为核心、支持多维度分类与快速检索的静态资源索引层。其目标用户包括技术文档撰写者、开源社区维护者、以及需要维护大量外部参考链接的研发团队。LinkVault Core 通过约定优于配置的目录规则，将散落在各处的 URL 转化为可维护、可追溯、可版本控制的资源清单，从而解决大型项目中外部链接混乱、失效、难以归类的问题。

## 功能概览

**基于 Markdown 的数据源驱动**：所有资源链接以 Markdown 列表形式存储于项目根目录的 /data 子目录中，支持批量导入与人工校对，无需数据库即可运行。

**多维度标签过滤机制**：每条链接可附带技术栈、所属领域、成熟度等标签，系统内置轻量级 grep 与 awk 过滤脚本，支持命令行环境下的快速筛选。

**自动化链接健康检查**：集成 cron 定时任务，每日对资源列表中的全部 URL 执行 HEAD 请求，自动标记失效链接并生成报告。

**静态站点生成器适配层**：提供与 VuePress、Hugo、Docsify 等主流静态站点生成器的数据格式转换脚本，方便将资源列表发布为内部导航页。

**版本化变更追踪**：所有资源增删改操作通过 Git 提交记录追溯，支持 Pull Request 流程审核外部链接的引入。

**交互式 TUI 管理界面**：提供基于 Dialog 或 Whiptail 的终端图形界面，便于非技术人员在服务器上完成链接的新增、编辑与分类移动。

## 应用场景

技术文档团队可将 LinkVault Core 作为 API 文档、SDK 指南、运维手册的附属参考索引，统一管理第三方依赖的官方文档地址、社区教程与问题追踪页面，避免文档正文被冗长的外链占据。

开源项目维护者可使用本系统整理生态内相关工具、插件、示例代码仓库的入口，在新人 onboarding 过程中提供清晰的资源地图，降低认知负担。

企业内部知识库管理员可利用 LinkVault Core 的分组与标签能力，将分散在不同 Wiki 页面、邮件、即时通讯记录中的关键外链集中归档，配合健康检查功能定期清理过时引用。

## 快速开始

以下命令演示了从克隆仓库到启动本地服务的完整流程。请确保已安装 Git 与 Python 3.8 及以上版本。

```bash
git clone https://github.com/linkvault/core.git linkvault-core
cd linkvault-core
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python3 bin/linkvault.py --serve --port 8080
```

如需仅生成静态资源清单而不启动服务，可执行 `python3 bin/linkvault.py --build`，输出文件位于 /dist 目录。

## 贡献指南

我们欢迎并鼓励社区贡献者通过以下标准化流程参与 LinkVault Core 的开发与数据维护。所有贡献需遵守行为准则与贡献者许可协议。

2. 对 /data 目录下的资源列表进行增删改操作时，请保持 Markdown 语法严格符合 /docs/data-spec.md 中的规范。新增链接需包含必要的标签信息，并确保 URL 有效可访问。

## 常见问题

Q: 项目默认的链接健康检查频率是多少？如何调整为每周一次而非每日一次？

A: 默认健康检查通过 crontab 每日凌晨 2:00 执行，命令为 `0 2 * * * cd /path/to/linkvault-core && make check`。如需改为每周一次，可将 crontab 表达式修改为 `0 2 * * 0`，表示每周日执行。配置文件位于 /etc/cron.d/linkvault，若使用 Docker 部署则需在容器编排文件中覆盖环境变量 `CHECK_INTERVAL`。

Q: 我拥有大量历史遗留的链接数据，如何批量导入到 LinkVault Core 中？

A: 系统支持通过 /data/raw 目录下的 Markdown 文件进行批量导入。您可以将链接按行整理为无序列表格式（每行以 `- ` 开头），然后运行 `python3 bin/linkvault.py --ingest /data/raw/your_file.md` 命令执行导入。导入过程中系统会自动去重、校验 URL 格式并生成初始标签索引。对于 CSV 或 JSON 格式的外部数据，建议先使用转换脚本（位于 /adapters/import_helpers 目录）转为标准 Markdown 列表再行导入。

Q: 如何自定义前端导航页面的样式与布局？

A: LinkVault Core 本身不捆绑特定前端框架，仅提供数据层适配脚本。您可以通过 /adapters 目录下的脚本将资源数据转换为 VuePress、Hugo 或 Docsify 所支持的数据格式，然后自由定制主题。如果您希望直接修改默认的纯文本导航页（/dist/index.html），可以编辑 /templates/index.template 文件并重新运行构建命令 `make build`。该模板使用 Jinja2 引擎渲染，支持自定义 CSS 与 JavaScript 注入。
