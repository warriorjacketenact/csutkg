# ResourceBridge 媒体资源导航聚合系统

ResourceBridge 是一个面向技术内容创作者、数字化学习研究者与媒体资源管理者的外链聚合与资源导航工具。项目定位于对高频访问的外部媒体资源链接进行结构化收录、分类索引与状态监控，解决技术团队或个人在内容生产、素材采集和引用溯源过程中面临的链接散落、检索效率低下和资源失效风险问题。

本系统以轻量化静态站点为核心交付形式，不对目标资源进行二次存储或代理转发，仅提供链接的整理、标注和分组展示能力。目标用户包括开源技术文档维护者、在线教育课程制作人、数字人文研究者以及需要长期维护大量引用链接的工程团队。ResourceBridge 通过约定的目录结构与元数据标注规范，使链接集合具备可维护性、可审计性和可扩展性，可作为持续集成流水线的一部分自动构建与部署。

## 功能概览

**多层级分类索引** 支持按资源类型、来源域名、内容主题等维度建立分类树，每条链接可归属于多个分类标签，便于从不同路径快速定位。

**链接状态健康检查** 内置可配置的链接探测模块，定时发起 HEAD 请求并记录响应状态码与耗时，对返回非 2xx 状态或超时的链接输出告警日志。

**元数据标注系统** 为每条链接扩展可选的 title、description、tags、priority 和 expiration_date 字段，所有标注信息以 YAML frontmatter 或独立 JSON 侧文件形式存储。

**静态站点生成引擎** 基于模板引擎将链接索引与标注数据渲染为 HTML 页面，输出包含分类总览、详情列表和单条资源卡片的全量静态站点，无需动态服务即可部署到任何 Web 托管环境。

**增量更新与变更审计** 记录每次链接增删改的操作日志，支持按时间范围回看变更历史，便于多人协作场景下的变更追溯与冲突仲裁。

**导入导出兼容层** 提供对 CSV、JSON 和 OPML 格式的导入导出支持，方便与其他书签管理工具或 feed 阅读器进行数据迁移。

## 应用场景

技术文档团队维护外部参考链接库。当编写产品手册或 API 文档时，需要频繁引用第三方规范文档、社区讨论帖和官方示例代码。ResourceBridge 可将这些散落的链接按文档章节或技术领域分类存放，并在文档构建流水线中集成健康检查，避免发布后出现死链。

在线课程制作人管理视频素材与参考文献。课程开发过程中涉及大量外部视频链接、学术论文页和工具下载地址。使用 ResourceBridge 对链接进行阶段标注（待审阅、已引用、已废弃），配合过期时间提醒功能，可在课程更新时快速识别需要复查的资源。

数字人文研究项目的引用溯源管理。人文研究中的网络档案引用要求长期可访问性。ResourceBridge 的链接状态监控和变更日志能够帮助研究者定期审查引用链接的有效性，并生成链接状态报告供项目审计使用。

个人知识工作者的信息消费流整理。对于每日阅读大量技术博客、新闻资讯和代码仓库的技术从业者，ResourceBridge 可作为信息摄入的暂存池，对有价值的外链进行初步分类和优先级标注，定期归档至长期笔记系统。

## 快速开始

以下命令序列演示了从源码获取到本地运行 ResourceBridge 实例的完整过程。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run build
npm start
```

执行上述命令后，系统将在本地 3000 端口启动静态站点预览服务。打开浏览器访问 http://localhost:3000 即可查看默认的链接索引页面。如需自定义资源数据，请编辑 `data/links` 目录下的 JSON 或 YAML 文件，保存后刷新页面即可看到更新后的内容。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Node.js | 18.x 或更高 LTS 版本 | 运行时环境与包管理基础 |
| npm | 9.x 或更高 | 依赖安装与脚本执行工具 |
| Git | 2.30 或更高 | 源码克隆与版本控制 |
| 文件系统读写权限 | 对 data/ 和 output/ 目录可读写 | 用于链接数据加载和站点输出 |
| 网络访问能力 | 至少可访问目标资源域名 | 链接健康检查需要外网访问 |
| 内存 | 最低 512 MB，推荐 1 GB | 处理大量链接数据时的性能保障 |
| 磁盘空间 | 最低 100 MB，推荐 500 MB | 用于存储链接数据、日志和生成的静态文件 |

## 贡献指南

ResourceBridge 欢迎外部贡献者参与改进。请遵循以下步骤提交代码或数据更新。

第一，查阅项目看板与议题列表。访问 GitHub Issues 页面了解当前待处理的缺陷报告和功能请求。对于较大的功能改动，建议先创建讨论议题，与维护者沟通设计方案后再着手实现。

第二，派生项目仓库并创建特性分支。将官方仓库派生至个人账户下，克隆派生仓库到本地，基于 main 分支创建新的分支，分支命名采用 `feature/描述` 或 `fix/描述` 格式。

第三，编写代码或更新数据并补充测试。代码修改需遵循 ESLint 配置的编码规范。新增功能需在 `tests/` 目录下补充对应的单元测试用例。数据文件的增删改需确保 JSON 或 YAML 格式合法，并更新对应的分类索引。

第四，提交变更并签署开发者原产地证书。提交信息采用约定式提交格式，例如 `feat: 添加链接过期提醒功能` 或 `fix: 修复分类页面分页偏移问题`。提交时需签署 DCO，即 `Signed-off-by: Your Name <email>`。

第五，创建拉取请求并等待代码审查。推送分支至派生仓库后，向官方仓库的 main 分支发起拉取请求。请填写 PR 模板中的检查清单，描述变更内容、测试覆盖情况和影响范围。维护者将在五个工作日内进行审查并给出反馈。

## 常见问题

Q: 健康检查模块对目标服务器造成压力如何处理？

A: 系统默认将并发请求数限制为 4，且每个请求的超时时间设为 10 秒。同一域名的请求间隔不低于 500 毫秒。用户可通过 `config.yaml` 中的 `checker.concurrency` 和 `checker.interval` 字段调整这些参数。对于高频访问的域名，建议在配置中将该域名加入 `checker.whitelist` 并手动设置较低的检查频率。

Q: 链接数据量达到数千条后，站点构建速度明显下降，如何优化？

A: 首先建议启用增量构建模式，该模式仅重新处理新增或修改的链接文件，而非全量重建。其次，可将链接数据按首字母或日期分片存储到多个 JSON 文件中，系统会自动并行加载。对于超过 5000 条链接的使用场景，推荐使用外部 Redis 缓存索引数据，相关配置参见 `docs/operations/scaling.md`。

Q: 如何将现有的浏览器书签或 Raindrop.io 集合导入 ResourceBridge？

A: 使用内置的导入命令 `npm run import -- --source=bookmarks.html --format=html` 可解析浏览器导出的书签 HTML 文件。对于 Raindrop.io，先通过其导出功能获取 CSV 文件，再执行 `npm run import -- --source=raindrop.csv --format=csv --map=raindrop-mapping.yaml`，其中 `raindrop-mapping.yaml` 用于指定 CSV 列到 ResourceBridge 元数据字段的映射关系。映射模板可在 `docs/import-export.md` 中找到。
