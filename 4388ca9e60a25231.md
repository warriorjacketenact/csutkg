# VideoLink Navigator

VideoLink Navigator 是一个面向技术内容创作者、在线教育平台运营者以及数字资源管理者的高性能视频外链聚合与导航系统。该项目旨在解决分散在多平台、多来源的视频资源难以统一收录、分类检索与状态监控的痛点，提供一套轻量化、可自托管的资源目录管理方案。

项目定位为技术资源与视频外链的汇总站，通过标准化的数据采集接口与可视化的管理面板，帮助用户将零散的视频链接转化为结构清晰、可快速检索的知识库。适用于个人知识管理、团队协作资源库以及小型教育站点的内容中台建设。

## 功能概览

批量链接导入与解析：支持从文本、CSV 及主流书签导出文件中批量导入视频链接，自动提取页面标题、描述及封面图信息，减少手动录入成本。

智能分类与标签系统：基于 URL 路径特征与自定义规则，对收录链接进行自动分类，并支持多级标签体系，便于按主题、项目或难度层级组织资源。

在线状态监控与可用性检测：定时检测已收录视频链接的访问状态，标记失效链接并生成报告，确保资源库的持续可用性。

全文检索与快速筛选：提供标题、描述、标签及自定义字段的全文检索引擎，支持多维度筛选器组合，实现毫秒级结果反馈。

数据导入导出与备份：支持 JSON、YAML 及 Markdown 格式的完整数据导出，便于版本控制、离线备份或迁移至其他系统。

用户自定义元数据扩展：允许为每条链接添加自定义字段（如作者、时长、难度等级、观看进度等），满足个性化管理需求。

响应式管理面板：提供适配桌面与移动设备的 Web 管理界面，支持暗色主题与键盘快捷键操作，提升日常维护效率。

开放 API 接口：基于 RESTful 风格提供完整的 CRUD 与查询 API，方便与其他系统（如自动化脚本、CMS、聊天机器人）集成。

## 应用场景

个人技术学习资源库：开发者可批量收录来自不同教程网站、技术会议录像及开源课程的视频链接，按编程语言、框架或主题分类，构建个人专属的视频学习导航站。

在线教育机构内容中台：中小型教育机构或培训团队可利用该系统统一管理自研课程视频、外部引用素材及往期直播回放链接，实现课程资源的集中授权与分发。

开源项目文档辅助：开源项目维护者可将相关的视频演示、社区 meetup 录像、兼容性测试录屏等链接收录进项目资源导航，丰富项目文档的外部参考材料。

数字策展与主题汇总：内容策展人可围绕特定技术领域（如云原生、人工智能、前端工程化）收集高质量视频资源，生成带注释的专题导航页并对外分享。

企业内部培训资源门户：企业技术部门可搭建内部培训视频导航，聚合来自不同供应商、开源社区及内部录制的学习材料，统一入口并实施访问追踪。

## 快速开始

以下步骤将指导您在本地环境快速启动 VideoLink Navigator 服务。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/videolink-navigator.git

# 进入项目目录
cd videolink-navigator

# 安装依赖（使用 npm）
npm install

# 配置环境变量（复制示例配置并修改）
cp .env.example .env

# 初始化数据库（SQLite 默认）
npm run db:init

# 启动开发服务器
npm run dev
```

服务启动后，访问 http://localhost:3000 即可进入管理面板。默认管理员账号为 admin@videolink.local，初始密码请在 .env 文件中查看或通过 `npm run reset:admin` 重置。

生产环境部署请参考 [文档导航](#文档导航) 章节中的部署指南。

| Node.js | 18.x 或 20.x LTS | 项目运行时环境，推荐使用最新的 LTS 版本 |

| Git | 2.40+ | 版本控制工具，用于克隆与更新代码 |
| Docker | 24.x（可选） | 容器化部署方案，支持一键构建镜像 |

## 项目结构

```
videolink-navigator/
├── apps/
│   ├── api/                         # RESTful API 服务（Express 应用）
│   │   ├── controllers/             # 请求控制器，处理路由逻辑
│   │   ├── middleware/              # 身份验证、日志、错误处理中间件
│   │   └── routes/                  # 路由定义文件（v1, v2 版本）
│   ├── web/                         # 管理面板前端（React + TypeScript）
│   │   ├── src/
│   │   │   ├── components/          # 可复用 UI 组件（表格、表单、筛选器）
│   │   │   ├── pages/               # 页面级组件（仪表盘、链接管理、设置）
│   │   │   ├── hooks/               # 自定义 React Hooks（数据请求、状态管理）
│   │   │   └── stores/              # Zustand 状态管理（用户偏好、缓存）
│   │   └── public/                  # 静态资源（图标、字体、manifest）
│   └── worker/                      # 后台任务进程（定时监控、数据同步）
│       ├── jobs/                    # 定时任务定义（链接检测、报表生成）
│       └── queues/                  # 任务队列配置（BullMQ 集成）
├── packages/
│   ├── core/                        # 核心业务逻辑与数据模型
│   │   ├── models/                  # 数据模型定义（Link, Category, Tag, User）
│   │   ├── services/                # 业务服务层（解析器、分类器、索引器）
│   │   └── validators/              # 输入验证与数据清洗规则
│   ├── database/                    # 数据库迁移与 ORM 配置（Prisma）
│   │   ├── migrations/              # 数据库迁移文件（按时间戳命名）
│   │   └── seeds/                   # 初始数据种子（默认分类、管理员账号）
│   └── shared/                      # 跨应用共享工具与类型定义
│       ├── types/                   # TypeScript 类型声明（接口、枚举）
│       └── utils/                   # 通用工具函数（日志、加密、HTTP 客户端）
```

## 常见问题

Q: 系统支持导入其他书签工具导出的 HTML 或 JSON 文件吗？

A: 支持。系统内置了针对主流浏览器书签导出格式（HTML）、Raindrop.io、Pocket 及 Instapaper 的 JSON 解析适配器。您可以在管理面板的导入页面选择对应格式并上传文件。对于自定义格式，可参考扩展开发文档编写自己的解析适配器。

Q: 定时监控任务是否会消耗大量服务器资源？检测频率如何调整？

A: 监控任务基于 BullMQ 队列实现，默认配置为每 12 小时检测一次所有链接，且通过并发控制限制同时检测的数量（默认 10 个）。对于链接数量超过 5000 条的大型实例，建议将检测间隔调整为 24 小时，并配置 Redis 持久化以避免任务丢失。检测频率可在 .env 文件中通过 CHECK_INTERVAL_HOURS 变量调整。

Q: 如何从 SQLite 迁移至 PostgreSQL 以支持更大规模的数据集？

A: 项目使用 Prisma ORM，支持无缝迁移至 PostgreSQL、MySQL 等主流关系型数据库。您只需在 .env 文件中修改 DATABASE_URL 连接串为 PostgreSQL 格式，然后执行 `npm run db:migrate` 即可自动生成表结构。数据迁移可使用 `npm run db:export` 导出 SQLite 数据为 JSON，再通过 `npm run db:import` 导入至新数据库。详细步骤请参考部署手册中的数据库迁移章节。
