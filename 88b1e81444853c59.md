# RC Theater Link Aggregator

RC Theater Link Aggregator 是一个面向遥控模型（RC）爱好者、无人机飞手以及相关行业从业者的外链资源汇总与导航系统。该项目并非一个传统的软件库或框架，而是一个精心编排的技术索引门户，旨在解决 RC 领域优质内容分散、官方文档查找困难以及评测信息碎片化的问题。

本项目通过自动化的数据抓取与人工校验相结合的方式，收录并分类了来自 RC 评测媒体、厂商支持页面及社区深度讨论的数千条有效链接。目标用户包括需要查找特定型号固件的地勤维护人员、希望对比不同车型性能的竞速玩家，以及需要获取技术参数进行二次开发的工程师。我们维护这批总量达 1241 批次的链接资源，本批次为第 390 批，涵盖了当前主流及经典车型的专属讨论串与媒体评测页面。

## 功能概览

- **按机型索引**：系统根据 URL 路径中的唯一标识符（Theater ID）自动归类至对应车型或产品线，支持快速过滤。
- **媒体评测聚合**：集中展示 RealRCReviews 等专业平台对特定型号的操控手感、耐用性及改装潜力的深度测评。
- **固件与驱动支持**：自动识别并高亮显示包含固件下载或驱动更新日志的特定资源链接。
- **社区讨论回溯**：捕获并整理各大论坛中关于特定车型的高频技术问答与通病解决方案。
- **状态监控与死链检测**：后台定期运行链接有效性检查，确保资源列表中的 URL 保持可访问状态。
- **快速跳转协议**：支持直接通过命令行参数或配置文件预设默认浏览器，实现一键打开多个相关资源。
- **标签化检索**：基于 URL 命名规则自动生成标签（如 "uvgp" 表示越野车架），支持多标签组合筛选。
- **本地缓存镜像**：对于频繁访问的热门资源，提供本地缓存机制以降低源站负载并加快内网访问速度。

## 应用场景

1. **新机型开箱准备**：当玩家购入一台全新车架（如 Theater ID: szny）时，可直接在本站聚合页中找到对应的官方说明书 PDF 链接、出厂设置推荐以及首批用户的避震调校心得，缩短摸索周期。
2. **故障排查与维修**：当车辆出现转向抖动或动力中断时，维护人员通过查找对应 ID（如 ivvznp）的链接列表，快速定位至厂商发布的 TSB（技术服务公报）或社区中关于舵机兼容性的讨论。
3. **竞速赛前调校参考**：竞技车手在赛前准备阶段，通过聚合页收集特定赛道条件下（对应不同 ID）其他车手的齿比搭配与轮胎选择策略，作为设定车辆的重要数据参考。
4. **技术选型与采购评估**：企业采购部门或车队经理通过对比不同 ID 下的评测总分与耐久性测试报告，评估不同品牌电调或电机在长期高负荷运行下的稳定性差异。

## 快速开始

以下步骤将指导您在本机环境部署该链接聚合器的静态前端与本地索引服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/rc-index/rc-theater-aggregator.git

# 进入项目根目录
cd rc-theater-aggregator

# 安装依赖（基于 Node.js 和 pnpm）
pnpm install

# 启动开发服务器（默认监听端口 4173）
pnpm run dev
```

执行完上述命令后，打开浏览器访问控制台输出的本地地址（通常为 http://localhost:4173 ）。此时系统将自动解析 data/links.json 中定义的当前批次资源，并生成可供浏览的导航页面。

## 贡献指南

我们欢迎社区成员提交新批次链接或优化现有索引逻辑。请遵循以下流程以保证数据一致性：

1.  **Fork 本项目**：将主仓库复制至您的个人 GitHub 账户下，并在本地克隆该副本。
2.  **创建特性分支**：在本地仓库中创建一个描述性分支名称，例如 `feat/add-batch-391` 或 `fix/broken-link-uvgp`。
3.  **更新数据文件**：若添加新链接，请严格按照 `data/links.json` 中定义的 JSON Schema 格式追加条目；若修正失效链接，请更新对应条目的 `status` 字段并记录验证时间。
4.  **运行验证脚本**：在提交前，务必在项目根目录执行 `pnpm run validate` 以检查新增或修改的 URL 是否可访问，并确保无 JSON 语法错误。
5.  **提交 Pull Request**：将您的分支推送至远程仓库，并向主仓库的 `main` 分支发起 PR。请务必在 PR 描述中详细说明本次变更的内容、关联的批次号以及验证结果。

## 常见问题

**问：为什么我访问列表中的某些链接会返回 404 错误？**

答：RC 评测媒体及厂商网站可能会对历史内容进行迁移或清理。虽然我们后台每 72 小时会执行一次死链检测，但部分外部站点的变更可能存在延迟。如果您发现失效链接，请通过 GitHub Issues 提交该资源的 URL 和 Theater ID，维护团队将在收到报告后的 24 小时内进行人工核实与状态标记。

**问：如何请求添加不在当前列表中的特定车型链接？**

答：您可以通过两种方式提交需求。一是按照上述贡献指南直接提交 PR 修改 `links.json`；二是直接在 Issues 页面选择 "Link Request" 模板，填写车型名称、期望添加的 URL 以及简要理由。我们会在每周的批次更新中评估并合并高价值的请求。

**问：该项目是否存储或代理了链接指向的实际内容？**

答：本项目仅存储指向第三方内容的 URL 链接和元数据（如标题、标签）。我们不会在服务器上存储、缓存或分发任何受版权保护的媒体文件或固件包。所有的内容访问请求均直接重定向至原始源站，用户需遵守各自第三方网站的使用条款。
