# ResourceBridge

ResourceBridge 是一个面向技术社区与内容创作者的轻量化外链资源归集与导航系统。项目定位为技术资料、视频内容、文档索引与动态资源的统一入口管理工具，主要服务于需要在多个内容源之间进行快速检索、分类归档和稳定引用的开发者、研究员与技术内容运营人员。

ResourceBridge 解决的核心问题包括：多源异构资源链接的集中化管理、资源有效性监控、访问路径规范化以及按主题维度的快速筛选。项目本身不存储任何实际内容数据，仅提供结构化链接管理与路由分发能力，适用于搭建个人技术书签站、团队知识库外链索引、视频或课程资源导航页等场景。

## 功能概览

- **多级路由映射**：支持基于路径参数的内容路由规则配置，可将不同子目录映射至特定资源分类。

- **资源生命周期标记**：每条资源支持录入、审核、上线、归档四个状态字段，便于运营过程追踪。

- **批量导入与导出**：支持通过 CSV 与 JSON 格式批量导入资源条目，并提供相同格式的导出能力用于备份或迁移。

- **访问计数与热度统计**：自动记录每条资源的点击次数与最后访问时间，支持按热度排序展示。

- **标签分类系统**：资源支持多标签关联，标签体系可自定义层级，便于构建个性化导航结构。

- **链接可用性检查**：内置定时任务可对已收录资源进行 HEAD 请求探测，标记异常链接并生成报告。

- **只读 API 接口**：提供基于 RESTful 风格的查询接口，支持按标签、时间范围、状态筛选资源列表。

- **暗色主题与响应式布局**：前端展示层同时支持亮色与暗色模式，并在移动端保持可用布局。

## 应用场景

- **技术博客外链整理**：技术作者可将散落在多篇文章中的参考链接统一收录至 ResourceBridge，通过标签分类后生成单独的导航页面，供读者快速查阅。

- **团队内部知识库索引**：开发团队可将项目文档、设计稿、测试报告、运维手册等外部链接统一托管，配合状态标记区分文档版本与审核进度。

- **视频课程配套资源页**：教育机构或内容创作者可将每节视频对应的示例代码、课后练习、扩展阅读链接集中管理，并通过路由规则为每门课程生成独立访问路径。

- **开源项目生态导航**：开源项目维护者可使用 ResourceBridge 整理生态内相关工具、插件、示例项目及社区讨论链接，降低新贡献者的信息查找门槛。

## 快速开始

以下步骤帮助您在本地环境快速启动 ResourceBridge 服务。

```bash
# 克隆代码仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装项目依赖
npm install

# 以开发模式运行服务
npm run dev
```

服务启动后默认监听 3000 端口，访问 http://localhost:3000 即可进入资源管理界面。首次启动将自动创建 SQLite 数据库文件并生成初始管理员账户，控制台日志中将输出初始登录凭证。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | >= 18.0.0 | 运行时环境，推荐使用 LTS 版本 |
| npm | >= 9.0.0 | 包管理器，用于安装项目依赖 |
| SQLite3 | 系统自带或 >= 3.35.0 | 默认嵌入式数据库，无需额外安装 |
| 操作系统 | Linux / macOS / Windows | 跨平台支持，生产环境推荐 Linux |
| 内存 | >= 512 MB | 最低运行内存要求，建议 1 GB 以上 |
| 磁盘空间 | >= 200 MB | 用于存放数据库文件与日志 |

## 项目结构

```
resourcebridge/
├── src/                           # 源代码主目录
│   ├── core/                      # 核心业务逻辑层
│   │   ├── resource.service.ts    # 资源增删改查与状态管理服务
│   │   ├── tag.service.ts         # 标签树构建与关联查询服务
│   │   └── health.checker.ts      # 链接可用性探测定时任务
│   ├── api/                       # HTTP 路由与控制器层
│   │   ├── routes/                # 路由定义文件
│   │   │   ├── resource.routes.ts # 资源相关接口路由
│   │   │   └── system.routes.ts   # 系统状态与配置接口
│   │   └── middlewares/           # 鉴权、日志、限流中间件
│   ├── db/                        # 数据库迁移与模型定义
│   │   ├── migrations/            # SQLite 表结构变更脚本
│   │   └── models/                # ORM 实体映射
│   ├── web/                       # 前端展示层
│   │   ├── pages/                 # 页面级组件
│   │   ├── components/            # 可复用 UI 组件
│   │   └── styles/                # 主题变量与全局样式
│   └── utils/                     # 通用工具函数
│       ├── logger.ts              # 结构化日志输出
│       └── validator.ts           # URL 格式与参数校验
├── config/                        # 配置文件目录
│   ├── default.yaml               # 默认配置项
│   └── production.yaml            # 生产环境覆盖配置
├── docs/                          # 完整文档目录

└── README.md                      # 项目说明文档
```

## 常见问题

**问：项目是否支持 PostgreSQL 或 MySQL 作为生产数据库？**

答：当前版本默认使用 SQLite 作为存储后端，便于快速部署与测试。项目数据访问层已抽象出统一接口，后续版本将提供 PostgreSQL 适配器。生产环境中如需使用其他数据库，可自行实现适配器或通过外部工具进行数据同步。

**问：如何批量更新资源的标签分类？**

答：在管理界面中选中多条资源后，可使用批量编辑功能统一追加或替换标签。亦可通过导入功能重新上传包含新标签信息的 CSV 文件，系统将根据资源唯一标识进行合并更新。

**问：链接可用性检查的频率和超时时间如何调整？**

答：可用性检查任务默认每 24 小时执行一次，单次请求超时时间为 5 秒。上述参数可在 config/default.yaml 文件中通过 healthCheck.interval 和 healthCheck.timeout 字段进行调整，修改后重启服务即可生效。
