# LinkCatalog Core

LinkCatalog Core 是一个面向开发者、技术研究团队与内容管理运营者的高性能外链资源归集与导航工具。该项目定位于对大规模、多批次、多来源的 URL 资源进行结构化整理、元数据标注、可用性检测与分类索引，帮助用户从海量散乱链接中快速提取有效信息，构建可维护的知识导航体系。

项目本身不依赖外部数据库或云服务，所有链接资源以纯文本与轻量级标记格式存储，便于版本控制、离线查阅与自动化流水线集成。LinkCatalog Core 可被视作一个本地优先、可扩展的链接治理框架，适用于需要长期跟踪和维护大量外链资源的各类技术文档项目、社区知识库或内部信息门户。

## 功能概览

**批量链接导入与解析**：支持从纯文本、CSV 及常见标记格式中批量读取 URL，自动识别协议、域名与路径结构，生成标准化条目。

**资源可用性探活**：内置异步 HTTP 检测器，可配置超时与重试策略，对每个链接返回状态码、响应时长与重定向链路，标记失效或异常资源。

**分类与标签体系**：允许用户为每条链接赋予多级分类标签，支持基于路径前缀、域名特征或自定义规则的自动打标，便于后续聚合筛选。

**结构化索引生成**：根据分类标签、来源批次或域名分组，自动生成 Markdown 表格、站点地图或 JSON 索引文件，输出可直接用于静态站点生成器或文档系统。

**差分变更追踪**：每次导入新批次时，自动比对已有资源，生成新增、删除与变更报告，支持增量更新与历史版本回溯。

**检索与过滤接口**：提供命令行过滤查询与简单 Web UI 两种交互方式，支持按关键字、状态码、分类、批次号等多维度组合筛选。

**扩展插件机制**：允许用户编写自定义解析器与检测钩子，适配非标准 URL 格式或私有链接协议。

**批次管理能力**：原生支持按项目批次与总链接数量进行进度追踪，当前批次为第 1128/1241 批，总资源量 250 条，便于大规模归集任务的拆解与审计。

## 应用场景

**技术文档站外链治理**：技术博客、开源项目文档或学习笔记中往往引用大量外部资源。LinkCatalog Core 可定期扫描所有外链，自动报告失效链接，帮助作者及时修正或替换，避免读者遇到死链。

**社区资源聚合导航**：技术社区或兴趣小组维护的“ awesome-xxx ”列表，随着贡献增多常出现重复或过时条目。项目提供去重与状态检测能力，协助维护者保持列表质量与可靠性。

**企业内部知识库构建**：企业内部团队在整理培训材料、工具手册或项目归档时，需要将散落在各处的重要链接统一归集。LinkCatalog Core 支持私有化部署，所有数据本地存储，满足安全合规要求。

**数据采集管道辅助**：数据工程师或爬虫开发者需要管理大量种子 URL 或数据源入口。项目可对种子链接进行初步健康检查与元数据提取，输出结构化的准入清单，减少无效请求对采集任务的干扰。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户可通过 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/linkcatalog/core.git
cd core

# 安装依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行初始化导入，加载当前批次资源
python bin/linkcatalog.py import --batch 1128 --source ./data/raw/urls_1128.txt
python bin/linkcatalog.py probe --batch 1128 --workers 10
python bin/linkcatalog.py export --batch 1128 --format markdown --output ./docs/links_1128.md
```

│   ├── __init__.py                      # 包版本与导出声明
│   ├── cli/                             # 命令行子命令模块
│   │   ├── import_cmd.py                # 导入逻辑，支持 txt/csv/yaml 格式

│   ├── core/                            # 核心数据模型与存储
│   │   ├── models.py                    # Link, Batch, Tag 等数据类定义
│   │   ├── storage.py                   # 本地文件读写与索引序列化
│   │   └── batch_manager.py             # 批次号管理与增量合并逻辑

│   │   ├── redirect_tracker.py          # 重定向链路追踪与循环检测

项目遵循开源社区标准协作流程，所有贡献者需遵守行为准则与提交规范。

1. 在 GitHub 上 fork 本仓库，并克隆到本地开发环境。确保已安装 Python 3.9 及以上版本，执行 `pip install -r requirements-dev.txt` 安装开发依赖。

2. 新建功能分支或修复分支，分支命名遵循 `feature/` 或 `fix/` 前缀，例如 `feature/add-sftp-detector`。所有代码需通过 `pytest` 单元测试，并保持测试覆盖率不低于 80%。

3. 对于新增的外部链接资源或元数据，请统一放置在 `data/raw/` 目录下，并按照 `urls_批次号.txt` 格式命名。若涉及分类规则变更，需同步更新 `classifiers/rule_engine.py` 中的默认规则集。

4. 提交前运行 `make lint` 或 `ruff check .` 进行代码风格检查，确保无语法警告。提交信息采用简洁英文或中文，格式为 `<类型>: <简短描述>`，例如 `feat: add batch diff command` 或 `fix: handle redirect loop in probe`.

5. 发起 Pull Request 至主仓库的 `main` 分支，并在 PR 描述中详细说明变更动机、实现方案与测试结果。项目维护者将在 3 个工作日内进行审查，必要时会通过评论或进一步提交请求修改。

## 常见问题

**Q：项目支持对非 HTTP 协议链接进行检测吗？**

A：目前内置检测器主要针对 HTTP 与 HTTPS 协议，对于 FTP、mailto、telnet 等协议仅做格式校验与存根记录，不发起实际连接。若需要扩展其他协议检测，可参考 `docs/plugin-dev.md` 编写自定义探测器并注册至插件系统。

**Q：如何处理单个批次中重复 URL 或无效格式？**

A：导入阶段会自动执行去重与格式校验。重复链接仅保留首次出现条目，后续重复项会被记录至导入日志但不会覆盖已有数据。格式校验失败的链接会被单独归入 `invalid_entries.log`，用户可手动修正后通过 `import --append` 方式补录。

**Q：能否将索引结果直接用于静态网站生成？**

A：可以。`export` 命令支持 `--format markdown` 输出可直接嵌入 MkDocs、Hugo 或 VuePress 等静态站点生成器的内容目录。对于需要 JSON 格式的场景，可使用 `--format json` 输出结构化数据，便于前端框架动态渲染。
