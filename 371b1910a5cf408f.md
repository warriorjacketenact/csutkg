# ResourceGate - 技术资源导航与聚合平台

ResourceGate 是一个面向开发者、技术研究人员与开源爱好者的外链资源聚合与导航系统。该项目不直接存储任何内容，而是通过结构化索引与分类体系，将分散在互联网各处的优质技术文档、工具链接、视频教程与参考手册进行统一收录与检索。项目定位为技术资源的入口级工具，帮助用户快速定位与其当前研究领域或问题高度相关的在线资料。

项目的主要目标用户包括：需要查阅最新技术规范与参考资料的软件开发工程师、从事技术选型的架构师、撰写技术博客或论文的研究人员，以及希望系统化学习某一技术栈的自学者。ResourceGate 通过提供清晰的主题分类、稳定的链接索引与快速的检索体验，显著降低信息筛选成本，提升研究与开发效率。

## 功能概览

**分类索引体系** 按照技术领域、应用场景、内容形式等多个维度对收录的链接进行精细化分类，支持多级筛选与组合查询。

**快速关键词检索** 基于标题与分类标签实现毫秒级关键词匹配，用户输入任意技术术语或资源简称即可获得相关链接列表。

**资源状态监测** 定期对收录的 URL 进行可用性检查，自动标记失效链接并生成报告，确保索引库的活性与可靠性。

**收藏与个人集合** 允许用户创建自定义收藏夹，将常用资源归入个人集合，支持导入导出为 JSON 或 YAML 格式。

**标签系统与关联推荐** 每个资源可附加多个自定义标签，系统基于标签重合度自动生成相关资源推荐，辅助用户进行延伸阅读。

**访问统计与热度排序** 记录每个链接的点击次数与近期访问趋势，提供按热度、更新时间、相关性等多种排序方式。

**开放数据导出** 支持将整个索引库或筛选结果导出为 CSV、Markdown 表格或静态 HTML 页面，便于离线查阅或二次分发。

**RSS 订阅与更新通知** 为每个分类提供独立的 RSS 订阅源，用户可通过订阅实时获取新增资源通知。

## 应用场景

技术选型与方案对比 当开发团队需要在前端框架、数据库系统或消息队列之间做出选型时，ResourceGate 可快速汇聚各技术的官方网站、性能对比报告、社区评价与典型用例，辅助团队做出基于充分信息的决策。

异常排查与错误码解读 开发人员遇到不熟悉的错误码或异常行为时，可通过 ResourceGate 检索相关技术社区的问题讨论、官方文档说明以及第三方分析文章，快速定位问题根源。

新技术学习路径规划 自学者进入一个全新领域（如机器学习、容器编排或区块链）时，可通过 ResourceGate 获取从入门教程、核心概念解释到高级实践案例的完整链接序列，形成清晰的学习路径。

文档版本追溯与历史参考 当查阅某个库或框架的特定版本文档时，ResourceGate 保留历史版本链接索引，用户无需在官网反复跳转即可直接访问所需版本的资料。

## 快速开始

以下命令序列适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resourcegate/resourcegate.git
cd resourcegate

# 安装项目依赖（使用 npm，需预先安装 Node.js 18+）
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

若使用 yarn 或 pnpm，请将 npm install 替换为对应包管理器的安装命令。启动成功后，在浏览器中访问 http://localhost:3000 即可进入 ResourceGate 主界面。首次启动会自动加载内置的初始索引数据，该数据包含本仓库收录的全部外链资源。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，用于执行构建脚本与开发服务器 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖与执行脚本命令 |
| SQLite | 3.35 及以上 | 嵌入式数据库，用于本地索引存储与查询（项目内置驱动） |

| curl | 7.68 及以上 | 可选但建议，用于资源可用性检测脚本的网络请求 |
| 操作系统 | Linux / macOS / WSL2 | 开发与生产环境均基于 Unix-like 系统进行优化 |

## 贡献指南

1. 提交 Issue 讨论新增分类或链接建议 在提交 Pull Request 之前，请先在 Issue 区域创建讨论帖，说明拟新增的资源分类、链接列表及其所属主题。维护者将在 48 小时内给予反馈，避免重复劳动或方向偏离。

2. 克隆项目并创建特性分支 获得 Issue 确认后，请将本项目 fork 至个人账户，然后克隆到本地。创建以 feature/ 或 fix/ 为前缀的分支，例如 feature/add-ai-resources。

3. 按照数据格式规范更新索引文件 所有新增或修改的链接必须遵循 docs/data-format.md 中定义的 JSON Schema，包含 title、url、tags、category、description 等必需字段。提交前请运行 npm run validate 校验数据格式合法性。

4. 编写或更新相应测试用例 如果贡献涉及核心检索逻辑、监测脚本或 API 接口变更，请在 tests/ 目录下补充对应的单元测试或集成测试，确保原有功能不受影响且新功能覆盖完整。

5. 发起 Pull Request 并等待 Code Review 将分支推送至个人远程仓库后，向主仓库的 main 分支发起 Pull Request。PR 描述中请关联对应的 Issue 编号，并简要说明变更内容与测试结果。维护者将在三个工作日内完成审查并合并。

## 常见问题

Q: ResourceGate 是否提供在线演示站点或公共实例？
A: 目前项目未部署公共演示实例，用户需按照快速开始章节的指引在本地环境运行。后续版本将考虑提供 Docker 镜像与一键部署脚本，降低使用门槛。

Q: 如何批量导入大量自定义链接，而非逐个手动添加？
A: 项目支持通过 CSV 或 JSON 格式进行批量导入。用户可按 docs/data-format.md 中规定的字段结构准备数据文件，然后执行 scripts/import-csv.sh 脚本进行导入。具体用法请参考管理员指南。

Q: 资源监测脚本的检测频率与超时时间是否可以调整？
A: 可以。监测频率与超时时间均在 config/default.yaml 中的 monitor 字段下进行配置，分别对应 interval（单位分钟）和 timeout（单位秒）。修改配置后需重启服务方可生效。
