# LinkVault CDN Resource Aggregator

LinkVault 是一个面向技术内容分发与资源链接管理的开源聚合工具，定位于帮助开发者、运维人员及内容运营者高效归集、校验和展示来自内容分发网络（CDN）的批量资源链接。本项目不提供具体视频或文件存储服务，而是作为链接元数据管理中间层，对上游源站的资源路径进行结构化索引、可用性探测与快速导航。

目标用户包括需要批量管理外链资源的站点管理员、进行数据采集与整理的爬虫开发者，以及希望建立自有轻量级资源导航页面的技术团队。LinkVault 通过将无序的 URL 列表转化为可查询、可分类、可监控的资产清单，显著降低人工维护成本，提升资源调度透明度。

## 功能概览

**批量链接导入解析** 支持从纯文本、CSV 或直接粘贴的 URL 列表中自动提取资源路径，识别协议、域名及路径层级结构。

**定时可达性探测** 内置基于 HTTP 状态码与响应时间的健康检查任务调度器，可标记失效或重定向链接，辅助清理死链。

**多维度标签分类** 允许用户为每条链接附加自定义标签（如地域、编码格式、内容类型），并据此生成动态分类目录。

**静态导航页生成** 根据链接元数据自动生成响应式 HTML 索引页面，可直接部署为静态站点，供内部团队或外部访客查阅。

**链接变更追踪** 记录资源路径的历史变更记录，支持回滚至任意历史版本的链接映射表。

**原始链接透传保真** 严格保留用户输入的原始 URL 字符串，不进行协议补全、大小写转换或路径规范化修改，确保与上游源站精确匹配。

**导入导出互操作性** 支持 JSON、YAML 及 Markdown 格式的链接清单导入导出，便于与其他数据处理流水线集成。

## 应用场景

**内部 CDN 资源盘点** 技术团队在迁移或重构静态资源存储架构时，可使用 LinkVault 批量录入现有资源链接，快速生成清单并与新存储路径进行对比校验。

**开源项目外链文档维护** 开源项目维护者可将项目文档中引用的所有外部资源链接集中托管于 LinkVault，利用其健康检查功能定期验证外链有效性，避免文档中出现失效引用。

**数据采集任务前置处理** 爬虫工程师在编写采集脚本前，先通过 LinkVault 对目标链接列表进行去重、格式校验和响应时间预检，筛选出稳定可用的数据源，提升采集任务成功率。

**轻量级资源导航站搭建** 内容运营人员无需开发后台，仅需维护一份 URL 列表，即可通过 LinkVault 的静态生成功能快速生成分类清晰的资源导航页面，用于部门内部共享或对外发布。

## 快速开始

以下步骤将指导您在本地环境中快速启动 LinkVault 服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault-core.git

# 进入项目根目录并安装依赖（使用 npm）
cd linkvault-core
npm install

# 以开发模式启动服务，默认监听 3000 端口
npm run dev
```

服务启动成功后，访问 http://localhost:3000 即可进入 Web 管理界面。首次启动会自动生成示例链接数据，您可在界面中清空并导入自己的 URL 列表。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，建议使用最新的长期支持版本 |
| npm | 9.x 或更高 | 包管理器，用于安装项目依赖 |
| SQLite3 | 内嵌于项目 | 默认轻量级数据库，无需额外安装，用于存储链接元数据与检查记录 |
| PostgreSQL（可选） | 14.x 或更高 | 生产环境推荐使用的数据库，需自行安装并配置连接字符串 |
| Redis（可选） | 7.x 或更高 | 可选缓存中间件，用于提升高频探测任务的调度效率 |

│   │   └── middleware/                 # 中间件集合（身份验证、日志、错误处理）
│   ├── db/                             # 数据库适配层
│   │   ├── models/                     # 数据模型定义（Link、CheckRecord、Tag）
│   │   ├── migrations/                 # 数据库迁移脚本（SQLite / PostgreSQL）

├── docs/                               # 文档目录（详见文档导航表格）

## 常见问题

**问：LinkVault 是否会对原始 URL 进行任何形式的自动补全或规范化修改？**

答：不会。本项目的核心设计原则之一是"原样透传"。所有用户输入的 URL，无论其是否包含协议前缀、是否以斜杠结尾，都会被完整保留在数据库和生成的导航页面中。系统仅在发起 HTTP 探测请求时根据标准协议进行临时拼接，但绝不会修改存储的原始值。这一设计确保与上游资源站点的精确匹配，避免因链接格式变更导致的访问异常。

**问：健康检查功能会频繁请求目标链接，是否会对源站造成压力？**

答：LinkVault 的探测任务默认采用指数退避与随机抖动策略，任务间隔不低于 60 秒，且每个链接的最小探测周期为 5 分钟。同时，系统支持用户自定义并发度与超时阈值，在生产环境中建议将并发数调至 5 以下。对于敏感或私有资源，您可以通过配置排除特定路径前缀来跳过探测。

**问：如何将 LinkVault 部署到生产环境并使用 PostgreSQL 替代默认的 SQLite？**

答：您需要先在目标服务器安装 PostgreSQL 14+ 并创建对应数据库。然后在项目根目录复制 .env.example 为 .env，修改 DATABASE_URL 为 postgresql://user:password@host:port/dbname 格式。最后执行 npm run migrate:prod 运行迁移脚本。项目已提供 Docker Compose 示例，包含 PostgreSQL 与 Redis 服务定义，可直接用于容器化部署。
