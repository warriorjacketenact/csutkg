# LinkVault 资源聚合系统

LinkVault 是一个面向技术团队与内容运营者的外链资源归集与导航平台。该项目定位为轻量级资源网关，用于将大量分散的外部链接按照既定分类、标签与元数据结构化存储，并提供可检索、可审计、可权限控制的统一访问入口。目标用户包括开源社区文档维护者、企业知识管理工程师、技术内容聚合站点运营人员以及需要批量管理外部引用资源的开发团队。LinkVault 不试图成为全文搜索引擎，而是专注于解决外链资源的生命周期管理问题：包括链接的导入、校验、分类、版本追踪、可用性监控与访问统计，填补了现有书签工具和通用数据库之间缺少工程化治理能力的空白。

## 功能概览

批量链接导入引擎：支持从纯文本列表、CSV、JSON Lines 及主流浏览器书签 HTML 导出文件中批量读取 URL，自动解析元数据并完成初步去重。

资源状态看板：以可视化面板展示全库链接总数、有效比例、失效警告数、分类覆盖率与近期新增趋势，支持按批次、域名、文件类型快速筛选。

元数据自动补全：对导入的 URL 自动发起 HEAD 请求获取 Content-Type 与最后修改时间，并依据路径特征与查询参数推断资源类型（文档、视频、API 端点、下载包等）。

自定义标签与分级体系：允许用户创建扁平标签或层级分类，支持一链接多标签，并提供基于标签组合的动态视图与排除过滤规则。

链接健康检查调度器：内置基于 Quartz 的定时任务，按可配置周期（每小时、每日、每周）对全量或指定标签下的链接发起可用性探测，记录响应时间与状态码变化。

权限与团队协作：基于角色的访问控制，区分管理员、编辑者、观察者三类角色，支持按分类目录或标签前缀划分数据可见范围，可对接 LDAP 或 OIDC 提供企业级身份认证。

审计日志与变更追溯：对所有链接的新增、修改、删除、标签变更及健康检查结果记录完整操作日志，支持按时间轴或操作者回溯，满足内部合规要求。

## 应用场景

文档站外链治理：技术文档站点维护者可将分散在多个 Markdown 文件中的外部参考链接统一导入 LinkVault，利用健康检查功能周期性发现死链，并在文档构建流程中通过 API 获取最新可用链接列表，避免用户访问失效资源。

开源项目依赖引用追踪：开源项目维护者将项目依赖的各项第三方服务、镜像源、协议文档等链接纳入管理，结合标签区分生产环境与开发环境，当外部服务地址变更时可快速定位并批量更新下游配置文件。

合规审计资源备案：企业法务或信息安全团队需要定期审查对外引用的第三方数据接口与隐私政策页面，LinkVault 的审计日志与版本对比能力可记录每次链接内容的变更时间点，为合规报告提供原始依据。

技术培训课程资料包管理：在线教育平台运营方将每期课程涉及的实验环境入口、视频回放地址、补充阅读材料等外链按课程编号与章节标签集中存放，学员可通过只读视图按需获取，运营方可监控各链接的访问热度。

## 快速开始

以下指令适用于 Linux / macOS 环境，需提前安装 Git、JDK 11+ 与 Maven 3.6+。Windows 用户可使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目主仓库
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# 使用 Maven 安装依赖并编译打包
mvn clean install -DskipTests

# 使用内置 H2 数据库快速启动（生产环境请替换为 MySQL 或 PostgreSQL）
java -jar target/linkvault-core-2.1.0.jar --spring.profiles.active=dev
```

启动成功后，访问 http://localhost:8080 即可进入初始化配置页面，默认管理员账户为 admin / admin123，首次登录强制修改密码。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| JDK | 11 或 17 (LTS) | 运行时环境，推荐使用 Eclipse Temurin 或 Oracle OpenJDK 构建 |
| Maven | 3.6.3 及以上 | 构建工具，用于依赖解析与打包，需配置国内镜像源以加速下载 |
| MySQL | 8.0.28 及以上 | 生产环境主数据库，用于存储链接元数据、标签、审计日志，需开启 binlog |
| Redis | 6.2 及以上 | 可选缓存中间件，用于提升健康检查任务状态查询与看板聚合接口的响应速度 |
| Elasticsearch | 7.17 或 8.5 | 可选，用于启用全文检索功能（按 URL 关键词、描述、标签进行高级搜索） |

## 项目结构

```
linkvault-core/
├── linkvault-api/                         # REST 接口定义与 DTO 层，包含 Swagger 注解
├── linkvault-bootstrap/                   # Spring Boot 启动类、配置加载与环境切换
├── linkvault-core-domain/                 # 领域模型（Link, Tag, CheckTask, AuditLog）与 JPA 实体映射
├── linkvault-scheduler/                   # Quartz 调度器封装，定义健康检查 Job 与触发器工厂
├── linkvault-parser/                      # URL 元数据解析器（HEAD 请求、Content-Type 推断、路径语义分析）
├── linkvault-elasticsearch/               # 全文检索实现，构建索引与高级查询 DSL
├── linkvault-security/                    # Spring Security 配置、JWT 过滤器、角色权限枚举
├── linkvault-common/                      # 公共工具类（字符串处理、日期格式化、HTTP 客户端池）
├── linkvault-integration-test/            # 集成测试套件，含 Testcontainers 配置（MySQL + Redis）
├── linkvault-benchmark/                   # JMH 性能基准测试，用于压力评估健康检查并发吞吐量
└── linkvault-assembly/                    # 打包描述符，生成可执行 fat JAR 与 Dockerfile
```

## 常见问题

Q: 导入包含上千条链接的 CSV 文件时出现超时，如何解决？

A: 默认的同步导入接口受限于 HTTP 连接超时与事务边界。推荐将大文件导入任务切换为异步模式：在 application.yml 中设置 linkvault.batch.async.enabled=true，并调整 linkvault.batch.chunk-size=200。系统会将任务提交至内部线程池，并通过 /api/v2/tasks/{taskId}/status 轮询进度。同时，确保 CSV 文件中的 URL 列不包含不可见字符（如 BOM 头或零宽空格），否则解析器可能跳过部分行。

Q: 健康检查任务报告大量链接为失效，但浏览器中能够正常访问，是什么原因？

A: LinkVault 默认使用 HEAD 方法探测，某些 Web 服务器可能对 HEAD 返回 405 或 403，而对 GET 返回 200。可在健康检查策略配置中将 method 切换为 GET，并设置 follow-redirects=true。另外，部分站点具有反爬机制，检查 User-Agent 头是否被过滤，可在 linkvault.health.user-agent 中自定义浏览器标识。若目标站点要求 TLS 1.3 或特定密码套件，需在 JVM 启动参数中补充 -Dhttps.protocols=TLSv1.3。

Q: 如何将现有生产数据从 H2 迁移至 MySQL？

A: 使用 Flyway 迁移脚本，项目已内置 /db/migration 目录下的 V1__init_schema.sql 至 V4__add_audit_index.sql。首先在 MySQL 中创建空数据库并赋予权限，然后在 application-prod.yml 中修改 spring.datasource 配置为 MySQL 驱动与连接串，启动时 Flyway 会自动执行版本校验。若 H2 中已有业务数据，可通过 linkvault-admin 模块提供的导出工具（/api/v2/export/database）生成兼容 MySQL 的 SQL 转储文件，再使用 source 命令导入。注意迁移前需停写服务，避免主键冲突。
