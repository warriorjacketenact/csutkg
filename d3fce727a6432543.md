# LinkForge

LinkForge 是一个面向技术团队与内容运营者的外链资源归集与管理工具。该项目定位于解决分散在各类文档、邮件、即时通讯中的外部链接难以统一归档、检索与共享的问题，特别适用于需要频繁引用外部技术文档、API 参考、行业报告或媒体资料的研发、运维与产品团队。通过结构化的资源索引与轻量级的元数据标记，LinkForge 将杂乱的 URL 集合转化为可维护、可追溯的知识库资产。

本项目本身即作为 LinkForge 的官方资源索引实例运行，收录了第 789/1241 批次共计 250 个来自 ninefold-group.com 的技术媒体链接。这些链接涵盖技术博客、发布说明、安全公告、性能测试报告等多种类型，可供集成至 CI/CD 文档生成、外部依赖追踪或合规审计流程中。

## 功能概览

**批量链接导入** 支持从纯文本、CSV 或 Markdown 列表批量解析 URL，自动去重并校验可访问性。

**层级化标签体系** 允许为每条资源自定义标签（如 backend、security、release-note），并支持多级标签组合过滤。

**快照元数据提取** 自动抓取目标页面的标题、摘要描述、最后修改时间及内容哈希，便于追踪变更。

**可编程查询接口** 提供 RESTful API 与命令行工具，支持按域名、路径模式、标签或时间范围进行复杂查询。

**静态站点生成** 可将当前资源索引导出为纯 HTML 静态站点，适合内网部署或嵌入 Confluence、Notion 等第三方平台。

**访问频率统计** 记录每条链接的查询次数与最近访问时间，帮助识别高频与低频资源。

**健康检查巡检** 定时对已收录链接发起 HEAD 请求，标记失效或重定向链接，生成异常报告。

**权限分级控制** 支持基于 API 密钥的读写分离权限，适用于多人协作场景下的资源管理。

## 应用场景

**技术文档中心外链治理** 技术写作团队可将所有引用的外部规范、SDK 下载地址、参考实现纳入 LinkForge 统一管理，当外部链接失效时快速定位并更新文档，避免文档中出现死链。

**微服务依赖溯源** 后端架构团队使用 LinkForge 记录各个微服务所依赖的中间件官方文档、性能基准测试报告及安全漏洞通告，在版本升级或安全应急时能够迅速获取一手信息来源。

**运维变更审核辅助** 运维人员在执行生产环境变更前，通过 LinkForge 检索相关变更涉及的官方公告或最佳实践链接，确保操作步骤与官方建议一致，降低人为失误风险。

**合规审计证据归档** 合规团队可将监管机构发布的指引、行业标准 PDF 下载页、认证机构的官方声明等外链集中存档，配合快照元数据满足审计追溯要求。

## 快速开始

以下命令演示如何在本地环境完成 LinkForge 的克隆、依赖安装与服务启动。

```bash
git clone https://github.com/linkforge/linkforge.git
cd linkforge
pip install -r requirements.txt
python -m linkforge.server --port 8080 --index ./resources/index.json
```

执行上述命令后，LinkForge 服务将在本地 8080 端口启动，默认加载当前目录下的 resources/index.json 作为初始资源索引。可通过 `--help` 查看全部运行时参数。

```

├── src/

│   │   └── query.py                  # 查询解析器与过滤条件构造器
│   ├── fetcher/                      # 外部链接抓取与解析模块
│   │   ├── client.py                 # 异步 HTTP 客户端封装（基于 aiohttp）
│   │   ├── parser.py                 # HTML 元数据抽取（标题/摘要/更新时间）
│   │   └── checker.py                # 链接健康状态检查与重定向跟踪
│   ├── server/                       # RESTful API 服务层
│   │   ├── app.py                    # Flask 应用工厂与路由注册
│   │   ├── handlers.py               # 请求参数校验与响应序列化
│   │   └── auth.py                   # API 密钥签发与权限验证中间件
│   ├── cli/                          # 命令行交互模块
│   │   ├── main.py                   # Click 命令入口与子命令分组
│   │   ├── import_cmd.py             # 批量导入链接的子命令实现
│   │   └── export_cmd.py             # 导出静态站点或 JSON 摘要的子命令
│   └── storage/                      # 持久化适配器
│       ├── sqlite_store.py           # SQLite 数据库 CRUD 操作实现
│       └── migrations/               # 数据库版本迁移脚本（使用 Alembic）
├── tests/                            # 单元测试与集成测试套件
│   ├── test_models.py                # 数据模型序列化/反序列化测试
│   ├── test_fetcher.py               # 模拟 HTTP 响应的抓取功能测试

│   └── sample_import.csv             # CSV 格式导入模板
├── scripts/                          # 开发辅助脚本
```

## 常见问题

**Q: LinkForge 是否支持非 HTML 资源（如 PDF、图片或二进制文件）的链接管理？**

A: 支持。LinkForge 的核心索引模型仅依赖 URL 字符串与用户自定义标签，不对目标资源类型做强制校验。对于非 HTML 资源，元数据抓取模块会跳过内容解析，仅记录最后一次健康检查的 HTTP 状态码与 Content-Type 头部。用户可以通过标签体系自行区分文档类型。

**Q: 如何迁移已有的书签库或浏览器收藏夹到 LinkForge？**

A: 主流浏览器支持将书签导出为 HTML 文件。LinkForge 提供了 `import --from-bookmarks` 子命令，可解析 Netscape Bookmark File 格式，自动提取 URL、标题以及文件夹层级作为标签。此外，也支持导入 Firefox 的 places.sqlite 与 Chrome 的 Bookmarks 文件，具体用法参见 docs/user-guide/import-sources.md。

**Q: 健康检查的并发数和超时时间如何调整？**

A: 健康检查参数通过 `config.yaml` 中的 `checker` 段控制，默认并发度为 10，单次请求超时为 5 秒。可通过环境变量 `LINKFORGE_CHECKER_CONCURRENCY` 与 `LINKFORGE_CHECKER_TIMEOUT` 覆盖。对于大规模索引（超过 5000 条链接），建议将并发度调高至 30，并配合 `--interval` 参数设置巡检周期，避免对源站造成压力。
