# Hydra Link Aggregator

Hydra Link Aggregator 是一个面向技术内容聚合与外部资源管理的开源链接集散系统。本项目定位于为开发者、技术内容创作者、知识管理爱好者提供一套高可读性、高可维护性的链接导航与分类展示方案，特别适用于需要集中管理大量外链资源的静态站点生成场景。

项目核心目标用户包括开源文档维护者、技术博客作者、在线教育平台运营者以及企业内部知识库管理员。通过 Hydra Link Aggregator，用户可以以结构化方式组织数百乃至数千条外部 URL，并按照自定义分类、批次、标签进行批量呈现，极大降低大规模链接维护的复杂度。本项目本身不存储任何第三方内容，仅作为导航与索引层，所有原始资源仍由源站点提供。

## 功能概览

**批量链接导入**：支持通过 CSV、JSON 或纯文本列表批量导入外部 URL，自动解析并去重，适用于大批量数据迁移场景。

**分级分类体系**：允许用户为每个链接分配多级分类标签，支持自定义分类层级，方便按主题、来源、批次进行检索与展示。

**URL 原样输出保障**：系统严格保留用户输入的 URL 原始格式，不擅自添加协议头、移除 www 前缀或修改大小写，确保链接准确性与合规性。

**自动批次管理**：内置批次编号机制，支持按导入批次对链接进行分组，便于追溯来源与版本控制，当前批次为第 43/1241 批。

**静态站点生成适配**：项目输出格式与主流静态站点生成器兼容，可直接生成 Markdown 格式的页面内容，减少二次处理成本。

**资源状态监控**：提供链接可达性检查功能，定期检测外部资源是否可访问，并在展示界面标注异常状态，帮助维护者及时清理失效链接。

**多视图呈现模式**：支持列表视图、卡片视图、分类目录视图三种展示方式，用户可根据实际使用场景自由切换。

**快速检索与过滤**：基于关键词、分类标签、批次号进行实时过滤，支持模糊搜索，提升大型链接库的查找效率。

## 应用场景

**技术文档资源导航**：开源项目维护者可以使用 Hydra Link Aggregator 管理文档中引用的所有外部参考链接，包括 API 文档、教程文章、视频资源等，确保所有引用均被清晰记录和定期检查。

**在线课程资料汇总**：教育平台运营者可将每期课程涉及的扩展阅读材料、工具网站、代码仓库统一收录，并按照课程章节或周次进行分类，方便学员集中获取所有相关资源。

**企业知识库外链管理**：企业内部知识管理团队可以将分散在多个内部 Wiki 页面中的外部参考链接集中迁移至 Hydra Link Aggregator，实现统一入口、统一监控，减少链接冗余和失效风险。

**个人书签库公开化**：技术博主或研究员可以将自己长期积累的优质技术资源以开源项目形式公开分享，通过批次记录展示资源收集的时间脉络，形成可追溯的个人学习轨迹。

## 快速开始

以下步骤帮助您在本地环境中快速启动 Hydra Link Aggregator 实例。

```bash
# 克隆项目仓库至本地
git clone https://github.com/hydra-link/hydra-link-aggregator.git

# 进入项目根目录
cd hydra-link-aggregator

# 安装项目依赖（使用 npm）
npm install

# 或使用 yarn
yarn install

# 运行本地开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产构建结果
npm run preview
```

## 常见问题

问：Hydra Link Aggregator 是否会对外部链接的内容进行缓存或存储？

答：不会。本项目仅存储 URL 字符串及其分类元数据，不抓取、不缓存、不存储任何第三方内容。所有对外部资源的访问均由用户浏览器或客户端直接发起，项目本身不充当代理或中间件。

问：导入的链接数量是否存在上限？

答：系统设计上不设硬性上限。单批次建议不超过 500 条链接以保证管理界面响应速度，但总链接数量仅受限于运行环境的存储空间。当前项目已在测试环境中稳定管理超过 1200 批次、累计 30 万条链接。

问：如何更新已有批次中的链接信息？

答：可以使用批量更新命令，通过 CSV 或 JSON 文件重新导入同一批次，系统将根据 URL 唯一标识进行覆盖更新。也可以使用交互式 CLI 逐条编辑链接的分类标签或备注字段。
