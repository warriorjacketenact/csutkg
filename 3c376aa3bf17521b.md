# StreamHub 技术资源外链聚合平台

StreamHub 是一个面向开发者与技术研究人员的轻量级外链资源聚合与导航系统，专注于收集、分类与快速检索互联网上的技术文档、开源工具、在线教程、API 参考与社区讨论帖。项目定位为“技术资源的统一入口”，通过简洁的路径映射与静态数据缓存，帮助用户从单一入口高效获取分散在各处的优质外部内容，避免在搜索引擎中反复筛选。

该项目适用于个人知识管理、团队技术文档索引、开源项目附属资源导航等场景。StreamHub 不存储外部资源内容，仅提供结构化链接映射与基础元信息描述，核心目标是降低技术信息发现成本，提升研发效率。

## 功能概览

**路径化资源映射** 将外部 URL 映射为 /stream/{slug} 形式的短路径，便于记忆、分享与书签管理。

**分类标签系统** 每个资源链接可关联多个分类标签，支持按技术领域、资源类型与适用场景进行多维筛选。

**全文检索支持** 内置基于标题与关键词的模糊搜索功能，可快速定位已收录的任意资源条目。

**元信息缓存机制** 自动抓取并缓存目标页面的标题、描述与结构化元数据，减少重复网络请求并提升响应速度。

**静态站点生成模式** 支持构建时预渲染全量资源页面，输出纯静态 HTML 文件，可部署至任意 Web 服务器或 CDN。

**RESTful 管理接口** 提供基于 JSON 的管理 API，支持资源条目的增删改查与批量导入导出。

**访问统计与热度排序** 记录资源链接的点击次数与访问来源，支持按热度、新增时间与字母顺序排列展示。

**自定义重定向规则** 允许管理员配置路径别名与跳转规则，适配已有书签或外部引用链接的迁移需求。

## 应用场景

个人开发者快速查阅常用技术文档。开发者在日常编码过程中需要频繁访问特定框架的 API 参考或函数手册，通过 StreamHub 将常用文档链接配置为短路径后，可直接在浏览器地址栏输入 /stream/{alias} 完成跳转，无需记忆完整 URL 或维护本地书签文件。

技术团队统一维护项目依赖资源。团队内部存在多个项目共用同一套第三方库文档、设计规范与运维手册的情况，StreamHub 可作为内部知识库的前端导航层，将所有相关外链集中管理，并随项目迭代同步更新，确保成员始终访问正确版本。

开源项目附属资源导航。开源项目通常需要引用外部教程、示例代码仓库、社区论坛或视频讲解，StreamHub 可作为项目 README 的补充，提供更细粒度的分类索引与检索能力，降低新贡献者的学习门槛。

技术博客与内容站点的外链整理。技术内容创作者在撰写文章或维护专题页面时，需要引用大量参考资料，StreamHub 可用于组织这些外链资源，生成清晰可维护的链接列表，同时支持按主题或系列进行分组展示。

## 快速开始

以下操作基于 Node.js 运行环境，使用 npm 作为包管理器。请确保系统已安装 Node.js 16.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/streamhub/streamhub.git
cd streamhub

# 安装项目依赖
npm install

# 启动开发服务器，默认监听端口 3000
npm run dev
```

执行上述命令后，打开浏览器访问 http://localhost:3000 即可进入 StreamHub 管理界面。首次启动会自动生成示例资源数据，可通过管理面板进行增删改查操作。

|---|---|---|
| Node.js | 16.x 或更高 | 运行时环境，用于执行服务端代码与构建脚本 |
| npm | 8.x 或更高 | 包管理器，用于安装项目依赖与运行脚本命令 |
| SQLite | 3.x 或更高 | 嵌入式数据库，用于存储资源条目与元信息，无需额外安装 |
| Git | 2.x 或更高 | 版本控制工具，用于克隆仓库与拉取更新 |

| 内存 | 512 MB 以上 | 最低运行内存要求，推荐 1 GB 以上用于生产环境 |
| 存储空间 | 200 MB 以上 | 用于存放源代码、依赖包与数据库文件 |
| 网络访问 | 外网连通 | 需要能够访问外部资源站点以完成元信息抓取与跳转 |

## 项目结构

```
streamhub/
├── src/                           # 源代码主目录
│   ├── core/                      # 核心模块：数据库连接、配置加载、缓存管理
│   ├── routes/                    # 路由层：处理 /stream/* 路径的请求分发与重定向逻辑
│   ├── services/                  # 服务层：元信息抓取、搜索索引、统计聚合等业务逻辑
│   ├── models/                    # 数据模型：资源条目、标签、访问日志的 ORM 映射
│   ├── controllers/               # 控制器：处理 HTTP 请求与响应的中间层
│   ├── middleware/                # 中间件：认证、日志、限流、错误处理等通用功能
│   ├── utils/                     # 工具函数：URL 校验、字符串处理、日期格式化等
│   └── workers/                   # 后台任务：定时刷新元信息、清理过期缓存
├── config/                        # 配置文件目录
│   ├── default.yaml               # 默认配置项，包含端口、数据库路径、缓存策略
│   ├── production.yaml            # 生产环境覆盖配置
│   └── development.yaml           # 开发环境覆盖配置
├── docs/                          # 项目文档
│   ├── user-guide/                # 用户指南
│   ├── admin/                     # 管理员手册
│   ├── api/                       # API 文档
│   ├── deployment/                # 部署方案
│   └── developer/                 # 开发者文档
├── static/                        # 静态资源输出目录
│   ├── css/                       # 样式表文件
│   ├── js/                        # 前端 JavaScript 脚本
│   └── assets/                    # 图片、字体等媒体资源
├── templates/                     # 页面模板
│   ├── layouts/                   # 基础布局模板
│   ├── partials/                  # 可复用的模板片段
│   └── pages/                     # 独立页面模板
├── data/                          # 数据存储目录
│   ├── db/                        # SQLite 数据库文件
│   ├── cache/                     # 元信息缓存文件
```

## 常见问题

如何批量导入外部链接？

StreamHub 支持通过管理 API 的批量导入接口一次性添加大量链接。请求体采用 JSON 数组格式，每个元素包含 url、title、tags 等字段。同时支持从 CSV 文件导入，可使用 scripts/seed.js 脚本配合适当格式的数据文件完成批量操作。

元信息抓取失败或超时怎么办？

系统默认使用内置的 HTTP 客户端抓取目标页面的标题与描述。若目标站点响应缓慢或屏蔽请求，可调整 config/default.yaml 中的 timeout 和 userAgent 参数。对于频繁失败的链接，可手动在管理面板中编辑其标题与描述字段，覆盖自动抓取结果。

如何将 StreamHub 部署到生产环境？

推荐使用静态导出模式进行生产部署。执行 npm run build 命令可在 static/ 目录下生成全量 HTML 页面，随后将整个目录上传至 Nginx、Apache 或任何静态托管服务。若需保留动态重定向与管理功能，可选择 Node.js 进程模式，并使用 PM2 进行进程守护与负载均衡。
