# LinkVault 聚合导航系统

LinkVault 是一个面向技术社区与内容创作者的轻量级外链资源聚合与导航平台。项目定位于将分散在各类技术博客、教程站点、工具文档中的高质量外部链接进行集中化收录、分类管理与快速检索，帮助开发者、研究者与技术写作人员高效建立个人或团队的外链资源库。

本项目不提供爬虫或自动化采集功能，而是提供清晰的数据组织规范、可扩展的分类索引结构以及简洁的静态页面渲染方案，适用于中小型技术团队、开源项目文档站、个人知识库等场景下的外链导航需求。LinkVault 强调内容所有者的自主控制权，所有链接数据以纯文本或 YAML 格式存储，支持通过 Git 进行版本管理与协作更新。

## 功能概览

- **多级分类索引**：支持按技术领域、内容类型、来源站点等多维度对链接进行标记与筛选，提供扁平化与树形结构两种浏览方式。

- **链接状态检测**：内置基于 HTTP 状态码的链接可用性检查模块，可定期输出失效链接报告，便于维护人员及时清理或更新。

- **元数据扩展支持**：每条链接可附加标题、描述、标签、收录时间、维护人、优先级等自定义元字段，满足不同场景下的精细化组织需求。

- **静态站点生成**：提供从数据目录到静态 HTML 页面的构建流程，输出可独立部署的纯前端导航站点，无需数据库或后端运行时依赖。

- **全文检索接口**：基于倒排索引原理实现轻量级本地搜索功能，支持按标题、描述、标签关键词快速定位目标链接。

- **导入导出兼容**：支持 CSV、JSON、YAML 三种格式的批量链接导入与导出，便于与其他书签管理工具或团队协作平台进行数据交换。

- **访问统计看板**：提供基于日志分析的链接点击次数与来源渠道统计，帮助管理员识别高频资源与热门内容方向。

## 应用场景

**技术文档站外链整合**：开源项目文档站通常需要引用大量外部参考链接，包括 API 文档、规范标准、相关项目仓库等。LinkVault 可为文档站维护一个独立的外链导航页面，按模块分类呈现，避免文档正文被过多外部链接打断，同时保证链接的可维护性。

**团队知识库资源聚合**：研发团队内部知识库中积累了众多工具地址、内部系统入口、运维手册链接等。使用 LinkVault 可建立团队共享的外链索引，通过元数据标注负责人与适用环境，减少新人上手时信息查找时间，并利用状态检测功能及时发现失效的内部系统入口。

**个人技术博客扩展页**：技术博主可在博客侧边栏或独立页面嵌入 LinkVault 生成的资源列表，将写作过程中参考的文章、视频、工具站点集中展示，既为读者提供延伸阅读入口，也体现内容的严谨性与可溯源特性。

**社区导航站建设**：技术社区或兴趣小组可利用 LinkVault 快速搭建垂直领域的资源导航站，如前端工具集、机器学习数据集索引、安全工具清单等，通过分类筛选与检索功能帮助社区成员高效发现优质资源。

## 快速开始

以下命令演示如何在本地环境中获取 LinkVault 源码、安装依赖并启动开发服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault.git

# 进入项目目录并安装依赖
cd linkvault
pip install -r requirements.txt

# 执行构建命令生成静态站点，输出目录默认为 ./build
python build.py

# 启动本地预览服务，默认监听 8000 端口
python serve.py --port 8000
```

执行完成后，在浏览器中访问 `http://localhost:8000` 即可查看生成的导航页面。如需自定义数据目录或输出路径，可参考 `config.yaml` 中的配置项说明。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Python | 3.9 及以上 | 构建脚本与本地服务器运行环境 |
| PyYAML | 6.0 及以上 | 用于解析 YAML 格式的链接数据文件 |
| markdown | 3.4 及以上 | 将链接描述中的 Markdown 片段渲染为 HTML |
| requests | 2.28 及以上 | 发送 HTTP 请求，用于链接状态检测模块 |
| click | 8.1 及以上 | 命令行接口参数解析，提供友好的 CLI 交互体验 |
| git | 2.30 及以上 | 用于版本管理与协作更新（非强制，建议安装） |

## 贡献指南

本项目欢迎各类贡献，包括但不限于新增功能、修复缺陷、完善文档与优化性能。请遵循以下流程提交贡献。

1. 在 GitHub 上 fork 本项目仓库，并在本地克隆 fork 后的副本。建议在开发前阅读 `docs/quickstart.md` 了解项目整体运行方式。

2. 创建新的功能分支，分支名称应简要描述本次修改的内容，例如 `feat/add-json-import` 或 `fix/search-case-sensitive`。请确保分支基于最新的 main 分支。

3. 完成代码或文档修改后，运行测试套件确保未引入回归问题。执行 `python -m pytest tests/` 验证所有测试用例通过。若新增功能，请同步补充对应的测试用例。

4. 提交 commit 时请遵循约定式提交规范，使用 `feat:`、`fix:`、`docs:`、`style:`、`refactor:` 等前缀，便于自动生成变更日志。提交信息应简洁描述修改内容与动机。

5. 向 main 分支发起 Pull Request，并在描述中关联相关 issue 编号，说明修改内容、测试覆盖情况以及影响范围。PR 合并前至少需要一位维护者进行代码评审。

## 常见问题

**Q：LinkVault 是否支持动态数据库后端，例如 MySQL 或 PostgreSQL？**

A：项目核心设计围绕静态站点生成展开，不强制依赖任何数据库系统。所有链接数据以 YAML 文件形式存储，便于版本控制和协作编辑。如需对接数据库，可自行扩展 `build.py` 中的数据读取层，项目已预留数据源抽象接口，支持通过配置文件切换至自定义数据源模块。

**Q：链接状态检测模块是否会频繁请求外部站点，导致源站压力？**

A：状态检测模块默认采用单线程顺序请求，并内置 5 秒超时与 3 次重试机制。检测频率由用户通过命令行参数控制，建议每周执行一次全量检测。对于大规模链接集，可通过 `--delay` 参数设置请求间隔，避免对源站造成突发流量压力。

**Q：如何迁移现有书签或收藏夹数据至 LinkVault？**

A：项目提供 `scripts/import_csv.py` 脚本，支持将符合列格式的 CSV 文件批量转换为 YAML 数据条目。常见浏览器导出的 HTML 书签文件可先用第三方工具转换为 CSV 格式后再导入。对于浏览器书签的直接解析支持已在规划中，预计后续版本通过新增 `import_bookmark.py` 脚本实现。
