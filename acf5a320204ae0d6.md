# ResourceBridge

ResourceBridge 是一个面向技术团队与独立开发者的外链资源归集与导航工具。该项目不存储任何实际内容，仅提供结构化的外链索引与元信息描述，帮助用户快速定位到分布在多个域名下的技术文档、工具站点、参考手册与社区讨论。项目本身以静态站点形式交付，可直接部署于任何支持 HTML 与 JavaScript 的托管环境。

ResourceBridge 解决的核心问题是：当团队或个人积累了大量的散落外链后，缺乏统一的检索入口与分类视图，导致重复查找、遗漏关键资料、新人上手成本高。本项目通过一个轻量级的索引层，将所有外链按业务域、技术栈、文档类型等多个维度进行重组，并提供本地化的标签过滤与全文检索能力。项目不依赖数据库，所有数据以 JSON 格式存储，适合 Git 版本管理，支持 CI/CD 自动构建。

## 功能概览

**多维度外链分类** 每个链接可标记多个分类标签，支持按技术领域、文档语言、适用角色（开发者/运维/产品）进行筛选。

**本地全文检索** 基于浏览器端倒排索引实现的关键词搜索，无需后端服务，支持标题、描述、标签字段的联合匹配。

**静态数据生成** 提供命令行工具，可从 CSV 或 Markdown 表格源文件自动生成标准化 JSON 索引数据，便于批量导入与更新。

**自定义元信息扩展** 每条外链记录支持扩展字段，包括维护状态（活跃/归档）、最后检查日期、备用镜像地址，方便团队进行资源生命周期管理。

**导入导出兼容** 支持将索引数据导出为 JSON、YAML 或 Markdown 表格格式，便于与其他文档系统（如 VuePress、Docusaurus）进行数据交换。

**URL 健康检查集成** 提供可选的定时检查脚本，利用 HEAD 请求验证外链可达性，并将异常结果输出至日志文件，辅助定期清理失效链接。

**用户自定义视图** 允许用户保存常用的过滤条件组合为“视图”，并在仪表盘快速切换，提升高频访问场景下的操作效率。

**暗色主题与阅读模式** 内置两套配色方案，并针对长文类外链提供去噪阅读模式，直接跳转时自动剥离广告与侧边栏。

## 应用场景

**技术团队内部知识库入口** 团队在日常开发中积累了大量外部参考链接，包括依赖库官方文档、性能分析工具、云服务控制台入口等。ResourceBridge 可作为团队知识库的统一导航层，按项目或微服务模块组织外链，新人入职时只需访问该站点即可获取全部必要外部资源。

**开源项目文档站的外链附录** 许多开源项目的 README 或文档站点中散布着大量参考链接，随着版本迭代难以维护。ResourceBridge 可作为独立的外链附录站点，与主文档站互链，既保持主站内容精简，又提供完整的外部参考资料索引。

**个人技术阅读工作台** 技术从业者需要跟踪多个技术社区的讨论、周报、博客更新。ResourceBridge 允许用户将日常阅读的 RSS 源地址、技术周刊归档、视频教程播放列表等统一归集，并通过标签快速切换关注领域，减少上下文切换成本。

**离线文档资源管理** 在网络隔离或弱网环境中，团队往往需要预先下载大量离线文档包。ResourceBridge 可记录每个离线包的下载地址、版本、校验和以及本地存储路径，帮助运维人员快速核对缺失资源。

## 快速开始

以下命令展示如何从源码启动 ResourceBridge 开发环境。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行 npm run dev 后，开发服务器默认启动在 127.0.0.1:3000。访问该地址即可看到外链索引页面，初始数据包含一组示例外链记录。如需生成生产环境静态文件，执行 npm run build，输出目录为 dist/，可直接部署到任何静态托管服务。

## 常见问题

问：ResourceBridge 是否存储外链对应的实际文件内容或快照？

答：不存储。ResourceBridge 仅记录 URL 地址及其元信息（标题、描述、标签等）。所有外链的实际内容仍由原始站点提供。项目自身不充当代理或缓存，也不对第三方内容的可用性负责。建议用户定期运行健康检查脚本以感知失效链接。

问：我可以通过 ResourceBridge 访问需要登录或具有访问限制的内部站点链接吗？

答：ResourceBridge 本身不处理身份认证或会话管理。若您添加的外链指向需要特定 Cookie 或 HTTP Header 才能访问的内部系统，点击链接时将由浏览器直接发起请求，其访问结果取决于您的网络环境和浏览器当前会话状态。本项目不干预任何请求头或凭证转发。

问：如何迁移已有的大量外链数据到 ResourceBridge 中？

答：推荐使用 CLI 工具中的 import-csv 命令。您需要准备一个 CSV 文件，至少包含 url 和 title 列，可选的列包括 description、tags（以分号分隔）、status、last_check 等。执行 `npm run import-csv -- --input your_links.csv --output data/raw/` 即可生成标准化 JSON。若您有 Markdown 表格格式的数据，可以借助 Pandoc 或自行编写脚本转换为 CSV 后再导入。
