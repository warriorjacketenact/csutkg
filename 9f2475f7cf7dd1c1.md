# LinkVault 资源聚合索引系统

LinkVault 是一个面向技术社区与内容创作者的轻量级外链资源聚合与导航平台。该项目定位于帮助开发者、研究者与内容运营者高效收集、分类、检索与展示分散在互联网各处的优质外部链接，解决信息碎片化导致的资源查找效率低下、链接丢失与缺乏统一管理入口的问题。

LinkVault 以静态站点生成器为核心构建，支持按批次、标签、来源域名与抓取时间对海量 URL 进行结构化存储与展示。项目特别适用于需要维护大型外链清单的场景，例如技术周刊、开源工具集合、论文参考附录或在线课程延伸阅读列表。通过一致的目录树与元数据约定，LinkVault 使数千条链接在数月乃至数年的运营周期中保持可维护性与可追溯性。

## 功能概览

**批量链接导入**：支持从纯文本、CSV 或 JSON 清单中一次性导入大量 URL，自动去重并校验协议格式。

**多维度分类索引**：每条链接可归属至自定义分类、项目批次与标签体系，支持按 programme、批次编号、内容主题进行筛选。

**原始地址透明保留**：系统严格记录用户提供的原始 URL 字符串，不自动补全协议或添加前缀，确保目标地址与来源完全一致。

**项目快照归档**：针对每批资源生成静态快照页面，包含该批次下所有链接的完整列表、导入时间与备注字段。

**链接状态健康检查**：内置可选的连通性探测模块，定时检测外链可访问性，并以标记形式在界面中提示失效链接。

**元数据扩展字段**：每条记录支持附加标题、来源站点描述、抓取日期与自定义备注，便于后续二次整理。

**静态页面输出**：基于模板引擎生成纯 HTML 页面，无需数据库即可部署至任何 Web 服务器或 CDN。

**全文检索支持**：集成客户端侧搜索能力，允许用户在当前批次内快速定位包含特定关键词的链接或注释。

## 应用场景

技术文档团队可利用 LinkVault 管理软件产品的外部参考链接，将分散在官方手册、社区讨论与第三方教程中的资源统一汇总，避免文档中直接嵌入冗长且易变的 URL 列表。

开源项目维护者可使用本系统整理项目依赖的相关工具、插件与衍生作品，按批次记录新收录的外部资源，方便贡献者快速了解项目生态。

教育机构或在线课程讲师可将每学期的延伸阅读材料通过 LinkVault 发布，学生能够在一个页面内获得全部外部链接，且讲师可随时追加新批次而不影响历史记录。

内容聚合类网站运营者能够借助 LinkVault 构建内部链接数据库，为编辑团队提供统一的外链录入与审核界面，最终生成对外展示的导航页或周刊专栏。

## 快速开始

以下命令演示如何在本地环境中获取 LinkVault 源码、安装依赖并启动开发服务器。

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
pip install -r requirements.txt
python manage.py runserver --batch 1153 --port 8080
```

执行上述命令后，访问 http://localhost:8080 即可查看第 1153 批资源的聚合页面。首次运行时会自动创建示例配置与空数据目录。

## 项目结构

项目目录遵循分层设计原则，将核心逻辑、数据存储、模板与输出产物清晰分离。

```
linkvault-core/
├── src/                               # 核心源码目录
│   ├── importers/                      # 导入器模块，支持 txt/csv/json 格式
│   │   ├── base.py                     # 导入器抽象基类，定义通用接口
│   │   └── factory.py                  # 根据文件扩展名返回对应导入器实例
│   ├── generators/                     # 静态页面生成器
│   │   ├── page_builder.py             # 构建单批次 HTML 页面
│   │   └── index_builder.py            # 构建所有批次的汇总首页
```

## 常见问题

问：系统是否会主动修改我导入的 URL 字符串？

答：不会。LinkVault 的核心设计原则之一就是完整保留用户提供的原始地址。无论是导入阶段、存储阶段还是页面渲染阶段，系统均不会添加 http:// 或 https:// 前缀，不会移除 www 子域名，也不会改变字母大小写或补充结尾斜杠。您看到的输出与您输入的内容完全一致。

问：如何将现有的大量链接清单一次性导入？

答：您可以将链接按行排列保存为纯文本文件，或整理成 CSV/JSON 格式，然后通过 `python manage.py import --batch 1153 --file yourlist.txt` 命令导入。系统会根据文件扩展名自动选择合适的导入器。导入完成后，所有记录会合并到对应的批次 JSON 文件中。

问：生成的静态页面能否部署到没有 Python 环境的服务器？

答：可以。生成器输出的全部是纯 HTML、CSS 与少量 JavaScript 文件，不依赖任何后端运行时。您可以将 output/ 目录下的内容完整复制到任意 Web 服务器、对象存储服务或静态托管平台，无需额外配置。
