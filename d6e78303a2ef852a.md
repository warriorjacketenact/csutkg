# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究者、内容策展人和开源开发者的高质量外链资源汇总工具与导航系统。项目定位于解决信息碎片化时代下优质技术内容分散、检索成本高、收藏管理混乱等问题，通过结构化分类与集中收录的方式，帮助用户快速定位到特定领域的高价值在线资源。

本项目不对收录内容进行二次编辑或篡改，仅提供索引与分类服务，确保每一份原始资源的完整性与可追溯性。LinkVault 适用于需要长期维护技术书签库、构建个人知识体系或进行行业信息监控的各类技术从业者。

## 功能概览

**自动分类索引**：根据资源 URL 路径结构与域名特征，对收录链接进行自动归类，支持按主题、平台、内容类型等多维度筛选。

**原始链接直出**：所有收录资源均以原始 URL 形式呈现，不附加任何追踪参数、跳转中介或重定向包装，保证链接的纯净性与可访问性。

**批量导入与导出**：支持通过 CSV 或 JSON 格式批量导入外部链接列表，同时可将当前库内资源导出为标准书签文件或结构化数据文件。

**链接可用性检测**：内置异步 HTTP 检测模块，定期对收录链接进行可达性验证，并对失效链接进行标记与通知，帮助维护资源库的健康度。

**全文检索与标签过滤**：基于倒排索引实现资源标题、描述及标签的快速全文搜索，同时支持多标签组合过滤，提升查找效率。

**项目结构可视化**：通过命令行工具一键生成项目目录树的 ASCII 图形化输出，便于开发者快速理解项目组织逻辑。

**多用户协作支持**：提供基于角色的访问控制，支持团队成员共同编辑资源元数据、添加备注与分类调整，适合机构级知识库运营。

**数据快照与回滚**：每次修改操作自动生成增量快照，支持回溯至任意历史版本，防止误删除或批量修改带来的数据损失。

## 应用场景

技术团队内部知识库建设：开发团队可利用 LinkVault 集中管理项目相关的技术文档、API 参考、设计规范及运维手册等外链资源，通过统一入口减少信息寻找时间，提高协作效率。

个人开发者学习路径管理：独立开发者或编程学习者可通过本工具分类收藏各技术栈的教程、案例源码、在线工具和社区讨论帖，构建体系化的学习资源地图，避免收藏夹混乱。

行业信息监控与竞品分析：市场分析人员或产品经理可将竞品官网、行业报告、数据看板及新闻源统一收录，利用标签和检索功能快速追踪特定领域的动态变化，辅助决策。

开源项目文档聚合：开源项目维护者可将依赖库文档、贡献指南、CI/CD 配置文件示例、社区论坛及版本发布说明等分散链接整合至单一资源库，降低新贡献者的上手门槛。

## 快速开始

以下操作基于 Linux/macOS 环境，Windows 用户可使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动服务
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

服务启动后，访问 http://localhost:8000 即可进入 LinkVault 仪表板。默认管理员账户为 admin，初始密码为 linkvault2024，首次登录后请立即修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 核心运行环境，低于此版本将无法解析类型注解与异步语法 |
| Django | 4.2.0 及以上 | Web 框架，用于提供 API 接口与后台管理界面 |
| PostgreSQL | 14.0 及以上 | 主数据库，存储资源条目、标签、用户及快照数据 |
| Redis | 7.0 及以上 | 缓存与消息队列后端，用于链接检测任务调度与会话存储 |
| Celery | 5.3.0 及以上 | 分布式任务队列，处理异步链接可达性检测与批量导入作业 |
| Node.js | 18.0 及以上 | 仅用于前端静态资源构建，生产环境可单独部署预编译产物 |
| Nginx | 1.24 及以上 | 推荐作为反向代理服务器，处理静态文件与负载均衡 |

## 项目结构

```
linkvault/
├── src/                                  # 核心源代码目录
│   ├── core/                             # 基础框架与配置模块
│   │   ├── settings.py                   # Django 项目配置（含数据库、缓存、中间件）
│   │   ├── urls.py                       # 根路由分发器
│   │   └── celery.py                     # Celery 应用实例与定时任务声明
│   ├── resources/                        # 资源管理应用（主业务模块）
│   │   ├── models.py                     # Resource, Tag, Snapshot 等数据模型定义
│   │   ├── classifiers/                  # 自动分类器子模块
│   │   │   ├── domain_parser.py          # 基于域名和路径前缀的规则分类器
│   │   │   └── ml_tagger.py              # 基于轻量级 NLP 的标签推荐器（实验性）
│   │   ├── detectors/                    # 链接可用性检测子模块
│   │   │   ├── http_checker.py           # 异步 HTTP HEAD/GET 探测实现
│   │   │   └── scheduler.py              # 周期性检测任务调度逻辑
│   │   └── views/                        # API 视图集（按功能拆分）
│   │       ├── resource_views.py         # 资源增删改查及导入导出接口
│   │       └── snapshot_views.py         # 快照列表、对比与回滚接口
│   ├── users/                            # 用户认证与权限管理
│   │   ├── models.py                     # 扩展用户模型与角色定义
│   │   └── permissions.py                # 基于对象的权限校验器
│   └── static/                           # 前端静态资源源码（Vue 3 + Vite）
│       ├── components/                   # 可复用 UI 组件（搜索栏、资源卡片、标签云）
│       └── stores/                       # Pinia 状态管理（资源列表、筛选条件、用户会话）
├── tests/                                # 单元测试与集成测试用例
│   ├── unit/                             # 针对模型方法、分类器、检测器的隔离测试
│   └── integration/                      # API 端到端测试与数据库事务回滚测试
├── scripts/                              # 运维与辅助脚本
│   ├── seed_db.py                        # 初始化演示数据与默认标签库
│   └── export_bookmarks.py               # 导出为 HTML 书签格式的转换脚本
├── docs/                                 # 完整文档源文件（Markdown + Sphinx）
├── requirements.txt                      # Python 生产环境依赖清单
├── requirements.dev.txt                  # 开发环境额外依赖（pytest, flake8, black）
├── Dockerfile                            # 多阶段构建镜像定义（分离构建与运行环境）
├── docker-compose.yml                    # 本地开发全栈编排（PostgreSQL + Redis + Nginx）
└── README.md                             # 本项目文件（当前文档）
```

## 贡献指南

1. 查阅 Issue 列表与 Project Board，选取未被指派的待办任务或提出新的功能建议。所有外部贡献需先创建对应 Issue 并获维护者确认，避免并行冲突。

2. Fork 本仓库至个人账户，在本地新建功能分支，分支命名遵循 `feature/功能简述` 或 `fix/问题简述` 格式，禁止在主分支直接提交。

3. 编写代码或文档时，严格遵守 `.editorconfig` 与 `pyproject.toml` 中定义的编码规范（行宽 88、使用 black 格式化、导入按 isort 排序）。所有新增模型字段或 API 端点必须附带对应的迁移文件与测试用例。

4. 提交前执行 `make check` 命令以运行全部 lint 工具与单元测试，确保无回归错误。提交信息采用 Conventional Commits 规范（如 `feat: 增加批量导入CSV支持` 或 `fix: 修复快照回滚时外键约束错误`）。

5. 发起 Pull Request 至 `develop` 分支，描述中需引用关联 Issue 编号并附上变更摘要。PR 通过 CI 检查后将由维护者进行 Code Review，必要时进行修改后合并。

## 常见问题

**Q: 收录的链接过多时，页面加载和搜索响应是否会变慢？**

A: LinkVault 在数据层使用 PostgreSQL 的 GIN 索引加速标签与全文检索查询，同时前端搜索结果采用分页加载（默认每页 50 条）。对于超过 10000 条资源的实例，建议开启 Redis 缓存并配置 Celery 定期刷新热门标签的预聚合结果，可保证毫秒级响应。

**Q: 如何迁移已有书签或收藏夹数据至 LinkVault？**

A: 项目提供 `scripts/import_bookmarks.py` 脚本，支持导入 Chrome/Firefox 导出的 HTML 书签文件、Raindrop.io 的 CSV 导出以及 Pocket 的 JSON 备份。具体用法参见 `docs/user-guide/import-export.md`，常见格式均可自动映射标签和文件夹结构。

**Q: 自定义分类规则如何编写？**

A: 分类器位于 `src/resources/classifiers/` 目录，默认基于域名与路径关键词匹配。您可继承 `BaseClassifier` 类并实现 `classify(url)` 方法，返回标签列表即可。修改后需在 `settings.py` 的 `CLASSIFIER_PIPELINE` 配置项中注册新类，重启服务即可生效。
