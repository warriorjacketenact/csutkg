# LinkVault Technical Resource Aggregator

LinkVault is a production-grade open-source technical resource aggregation and navigation system designed for developers, researchers, and technical writers who need to organize, categorize, and rapidly access large volumes of distributed web-based technical documentation, reference materials, and community resources. The project addresses the fundamental challenge of link rot, resource fragmentation, and inefficient bookmark management by providing a structured, version-controlled, and queryable repository of categorized technical URLs with metadata enrichment capabilities.

The system targets technical leads managing team knowledge bases, open-source maintainers curating community resources, and individual developers who routinely reference dozens of specialized technical articles, API references, and tool documentation. LinkVault transforms raw URL collections into navigable knowledge graphs through automated tagging, content fingerprinting, and periodic availability checking, ensuring that every resource remains accessible and contextually relevant to its intended technical domain.

## 功能概览

**Automated Resource Ingestion** - Bulk import URLs from plain text files, CSV exports, or browser bookmark HTML exports with automatic duplicate detection and normalization.

**Categorization Engine** - Assign resources to hierarchical taxonomies including programming languages, frameworks, infrastructure, security, databases, and DevOps with support for multi-tagging.

**Availability Monitoring** - Scheduled HTTP HEAD requests verify resource liveness, with configurable retry policies and dead link reporting via webhook notifications.

**Search and Filter** - Full-text search across resource titles, descriptions, and tags with faceted filtering by category, status code, and last verification timestamp.

**Metadata Extraction** - Automatically fetch page titles, Open Graph descriptions, and content-type headers during ingestion to enrich resource entries without manual data entry.

**Export Pipeline** - Generate static HTML navigation pages, JSON API endpoints, or Markdown-formatted resource lists for integration with existing documentation sites or static site generators.

**Access Control** - Role-based permissions distinguishing between viewers who consume resources and editors who manage the resource corpus, with audit logging for all modifications.

**Version History** - Track all changes to resource metadata and categorization with rollback capability, enabling teams to revert accidental bulk operations or category reorganizations.

## 应用场景

**Technical Onboarding Playbook** - Engineering teams use LinkVault to compile a curated list of internal and external references required for new hires, including coding standards, architecture decision records, and deployment runbooks. The categorization engine groups resources by team domain, allowing new engineers to progressively discover relevant materials as they ramp up on specific subsystems.

**Open-Source Project Documentation Hub** - Maintainers of large-scale open-source projects aggregate external tutorials, community blog posts, and third-party integration guides alongside official documentation. LinkVault's availability monitoring alerts maintainers when referenced external resources become unavailable, enabling proactive replacement or archival of dead links before users encounter broken references.

**Conference and Meetup Resource Compilation** - Event organizers collect speaker slides, workshop materials, and related reading lists from multiple sessions into a single searchable repository. Attendees access the aggregated resource collection post-event through the export pipeline, which generates a static HTML page that remains accessible without requiring backend infrastructure.

**Research Literature Management** - Academic researchers and technical writers organize preprint servers, specification documents, and standards body publications by research theme or technical domain. The version history feature supports iterative refinement of resource collections as papers are published, updated, or superseded by newer revisions.

**Compliance and Audit Preparation** - Security and compliance teams maintain auditable records of external references used in system design documents, threat models, and risk assessments. The audit logging capability provides evidence of which resources were consulted during specific compliance cycles, supporting regulatory reviews and external certification processes.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/your-org/linkvault.git
cd linkvault

# Install dependencies using pip with virtual environment
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt

# Initialize the SQLite database and run migrations
python manage.py migrate

# Start the development server
python manage.py runserver --host 0.0.0.0 --port 8080

# In a separate terminal, run the worker for scheduled monitoring tasks
celery -A linkvault worker --loglevel=info
```

## 项目结构

```
linkvault/
├── src/                                # 核心应用源代码
│   ├── core/                           # 数据模型与业务逻辑核心
│   │   ├── models.py                   # SQLAlchemy ORM 模型定义 (Resource, Category, Tag)
│   │   ├── schemas.py                  # Pydantic 验证模式，用于 API 请求/响应
│   │   └── enums.py                    # 枚举类型 (ResourceStatus, CategoryType)
│   ├── ingestion/                      # 资源导入与元数据提取模块
│   │   ├── importer.py                 # 批量导入器，支持 CSV/HTML/纯文本输入
│   │   ├── extractor.py                # 元数据提取器，抓取标题和描述
│   │   └── normalizer.py               # URL 标准化与去重逻辑
│   ├── monitoring/                     # 资源可用性监控
│   │   ├── checker.py                  # HTTP 存活检测器，含重试与超时策略
│   │   ├── scheduler.py                # Celery 定期任务调度配置
│   │   └── notifier.py                 # 告警通知器，支持邮件和 Webhook
│   ├── api/                            # RESTful API 端点
│   │   ├── routes/                     # 路由处理器按资源类型分组
│   │   │   ├── resources.py            # CRUD 操作端点
│   │   │   ├── categories.py           # 分类管理端点
│   │   │   └── search.py               # 全文搜索与过滤端点
│   │   └── middleware.py               # 认证、日志、速率限制中间件
│   └── web/                            # 管理界面 UI (FastAPI + Jinja2)
│       ├── templates/                  # HTML 模板文件
│       └── static/                     # CSS 与 JavaScript 资源
├── tests/                              # 测试套件
│   ├── unit/                           # 单元测试，覆盖核心逻辑
│   ├── integration/                    # 集成测试，验证数据库与 API 交互
│   └── fixtures/                       # 测试数据固件 (示例 URL 列表)
├── scripts/                            # 运维与开发辅助脚本
│   ├── seed_db.py                      # 用示例数据初始化数据库
│   ├── export_static.py                # 生成静态 HTML 导出

│   ├── operations.md                   # 日常运维手册
│   └── api-reference.md                # API 详细参考
├── config/                             # 配置文件
│   ├── settings.py                     # 应用设置 (环境变量映射)
│   ├── logging.yaml                    # 日志记录配置

│   ├── versions/                       # 按时间排序的迁移版本
│   └── env.py                          # 迁移环境配置
```

## 贡献指南

1.  Fork 本仓库并在本地克隆您的复刻副本。创建功能分支时使用描述性的名称，例如 feat/bulk-import-improvements 或 fix/monitoring-timeout。所有分支必须从 main 分支的最新提交切出。

2.  安装开发依赖并激活预提交钩子。运行 pip install -r requirements-dev.txt 后执行 pre-commit install，这会自动启用代码风格检查、类型注解验证和基本安全扫描。所有提交必须通过这些钩子才能被接受。

3.  编写或修改代码时，请遵循 PEP 8 风格指南并使用 Black 格式化器。所有新增或修改的功能必须包含对应的单元测试和集成测试，测试覆盖率不得低于 85%。使用 pytest 运行全部测试套件以验证您的变更未引入回归。

## 常见问题

**Q: 系统可以处理的最大资源数量是多少？**

A: LinkVault 使用 SQLite 作为默认数据库，能够稳定处理 5 万至 10 万条资源记录，全文搜索性能在 5 万条记录以下保持亚秒级响应。对于超过 10 万条记录的生产环境，建议迁移到 PostgreSQL，该配置已在测试中验证可支撑百万级资源，并配合同步 Elasticsearch 实现更高效的全文检索。监控调度器通过 Celery 分布式队列管理，worker 数量可按需水平扩展，检查吞吐量随 worker 实例线性增长。

**Q: 如何迁移已有书签或收藏夹到 LinkVault？**

A: 系统支持三种导入方式。第一种是直接粘贴 URL 列表，每行一个链接，系统自动批量提取元数据。第二种是上传浏览器导出的 HTML 书签文件，解析器会提取 Netscape Bookmark 格式中的所有 URL 和文件夹名称作为分类依据。第三种是通过 CSV 文件导入，支持自定义列映射，允许用户指定标题、描述、标签和分类列。导入前系统会执行去重检查，基于标准化 URL 和内容哈希双重机制避免重复条目。导入进度可通过管理界面实时查看，并支持暂停和恢复长时间运行的批量导入任务。

**Q: 可用性监控会对目标网站造成压力吗？**

A: 监控模块默认采用 HEAD 请求方法，仅获取响应头部而不下载页面正文，单个请求的网络开销极小。检查频率可根据资源重要性配置，默认每 24 小时检查一次所有资源，并引入抖动机制将检查时间分散到整个周期，避免在整点时刻产生请求峰值。对于大型部署，可配置为随机抽样检查，每日仅验证 20% 的资源，确保监控负载保持在合理范围内。所有请求遵循 robots.txt 规范，且 User-Agent 标识为 LinkVault-Monitor，便于目标站点识别和过滤。
