# ResourceBridge

ResourceBridge 是一个面向技术团队与内容运营者的轻量级外链资源聚合与导航系统。项目定位为“技术资源的中转枢纽”，帮助用户将分散在多个渠道的参考链接、文档地址、工具站点、影音资料等外部资源进行统一收束、分类归档与快速检索，降低资源遗忘与重复查找的成本。

ResourceBridge 不提供资源存储功能，而是专注于外链的元数据管理、标签化组织与多维度筛选。项目适用于开发者知识库维护、运营素材库管理、研究资料汇编等场景，尤其适合需要长期跟踪大量外部链接的团队或个人。

## 功能概览

- **资源条目管理**：支持添加、编辑、删除外部链接，每条记录包含标题、原始 URL、来源站点、添加时间、最后校验时间与自定义备注字段。

- **多级分类体系**：采用树形分类结构，允许用户自定义分类层级。分类可附带图标与描述，便于构建符合业务逻辑的导航树。

- **标签系统与复合筛选**：每条资源可绑定多个标签。支持按标签组合、分类路径、时间范围、来源域名等多条件复合筛选，快速定位目标资源。

- **链接可用性检测**：内置定时任务，周期性地对已收录的外链发起 HEAD 请求，检测链接是否可访问、是否发生重定向、返回状态码是否异常，并在管理面板中标记状态。

- **全文检索与模糊匹配**：基于轻量级倒排索引实现标题、备注、来源站点等字段的快速全文检索，支持中文分词与拼音首字母模糊匹配。

- **导入导出与备份**：支持批量导入链接列表（CSV / JSON 格式），导出为结构化数据文件，便于迁移、备份或与其他系统对接。

- **访问统计与热度排行**：记录每条资源的点击次数与最后访问时间，支持按访问热度排序，帮助识别高频使用的核心资源。

- **响应式管理界面**：提供适配桌面与移动设备的 Web 管理面板，基于主流前端框架构建，操作流程清晰，交互反馈即时。

## 应用场景

- **技术团队知识库维护**：技术负责人可将项目相关的设计文档、API 参考、代码仓库、部署手册等外链统一收录，按模块分类，新成员入职时即可通过 ResourceBridge 快速获取全部必要参考资料，减少口头传递与文档碎片化问题。

- **运营与市场内容素材管理**：运营人员日常收集大量行业报告、竞品动态、媒体稿件、图片素材源地址。通过 ResourceBridge 为每条素材打上活动名称、内容类型、使用状态等标签，在策划新活动时可快速筛选复用历史素材，避免重复劳动。

- **学术研究与文献资料汇编**：研究人员在阅读文献过程中积累大量 DOI 链接、预印本地址、数据存储库、工具官网。ResourceBridge 的时间戳与备注字段可记录阅读进度与核心观点，链接可用性检测功能则能及时发现失效链接，防止参考文献失联。

- **个人开发者资源订阅与整理**：独立开发者可将常用组件库、在线工具、学习教程、开源项目地址集中管理，通过分类与标签构建个人知识索引，替代浏览器书签的扁平化管理方式，提高日常开发效率。

## 快速开始

以下步骤指导您在本地环境快速启动 ResourceBridge 服务。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git

# 进入项目目录
cd resourcebridge

# 安装项目依赖（使用 npm）
npm install

# 配置环境变量，复制示例配置文件并编辑
cp .env.example .env

# 初始化数据库结构
npm run db:init

# 启动开发服务器
npm run dev
```

服务启动后，访问控制台输出的本地地址（默认 http://localhost:3000）即可进入管理界面。首次启动将自动创建管理员账户，初始密码会输出在终端日志中，请及时修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用官方 LTS 版本。 |
| npm | 9.x 或以上 | 包管理器，用于安装项目依赖。 |
| SQLite3 | 系统内置（需编译工具链） | 默认嵌入式数据库，适合小型部署。生产环境可切换至 PostgreSQL。 |
| TypeScript | 5.x（项目内锁定） | 开发语言与类型系统，项目源码基于 TypeScript 编写。 |
| Redis | 7.x（可选） | 用于会话存储与缓存加速，生产环境推荐启用。 |
| Git | 2.x 或以上 | 版本控制工具，用于克隆仓库与获取更新。 |
| 系统内存 | 最低 512 MB，推荐 1 GB | 运行服务与构建前端资源的内存需求。 |
| 磁盘空间 | 最低 200 MB | 项目文件、依赖包与数据库存储空间。 |

## 贡献指南

ResourceBridge 遵循开源社区协作规范，欢迎任何形式的贡献。请按照以下步骤参与项目开发。

1. 查阅问题追踪列表与项目看板
   访问 GitHub Issues 页面，查看当前待解决的问题与功能请求。对于尚无对应 Issue 的贡献提议，请先创建新 Issue 描述您的想法或缺陷，等待维护者反馈后再行开发，避免无效工作。

2. 复刻项目仓库并创建特性分支
   将主仓库复刻至您的个人账户下，然后克隆本地。请基于 main 分支创建新的特性分支，分支命名建议采用 `feat/功能简述` 或 `fix/问题简述` 格式，保持与主分支的清晰边界。

3. 编写代码并遵守项目规范
   后端代码请遵循 ESLint 与 Prettier 配置，确保 TypeScript 类型检查通过。前端组件请遵循 Vue 官方风格指南。所有新增功能需附带对应的单元测试或集成测试，测试覆盖率不得低于现有基线。提交信息请采用约定式提交格式。

4. 发起合并请求
   将您的特性分支推送到复刻仓库，然后向主仓库的 main 分支发起 Pull Request。PR 描述中请明确关联相关 Issue 编号，简述实现方案与测试结果。PR 将通过自动化 CI 流程进行构建与测试校验。

5. 代码审查与合并
   维护者将对 PR 进行审查，可能会提出修改意见。请及时响应反馈并更新代码。审查通过后由维护者完成合并。合并后的代码将自动触发开发环境部署，供进一步验证。

## 常见问题

**Q: ResourceBridge 支持哪些数据库后端？**

A: 默认使用 SQLite 作为嵌入式数据库，适合单机部署与功能试用。生产环境推荐使用 PostgreSQL，项目通过 ORM 层适配了 PostgreSQL 与 MySQL。您只需修改环境变量中的数据库连接字符串，并运行一次数据库迁移命令即可完成切换。切换前请确保目标数据库已创建且字符集设置为 UTF-8。

**Q: 链接可用性检测是否会对目标站点造成压力？**

A: 检测机制采用 HEAD 请求而非 GET 请求，仅获取响应头信息，不下载响应体内容，因此对目标站点的资源消耗极低。检测任务默认配置为每 24 小时执行一次全量检测，且检测请求之间设置了随机延迟（100-500 毫秒），避免在短时间内对同一域名发起大量请求。对于高频访问的资源，您可以在配置文件中单独设置检测白名单或调整检测频率。

**Q: 如何迁移现有书签或收藏夹数据到 ResourceBridge？**

A: 项目提供了导入功能，支持 CSV 与 JSON 两种格式。您可以从浏览器导出书签为 HTML 文件，然后使用社区提供的转换工具将 HTML 转换为 CSV 格式。导入时请确保 CSV 包含 title 与 url 两列，其他列如 tags、category、remark 为可选。导入后系统会自动进行链接可用性初始检测，您可以在管理界面中查看检测结果并批量修正失效链接。
