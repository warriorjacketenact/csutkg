# LinkSphere

LinkSphere 是一个面向开发者与技术研究人员的开源外链资源聚合平台，专注于收集、分类与存档互联网上的高质量技术文档、工具站点、在线服务与参考数据源。项目本身不生成内容，而是通过人工筛选与社区提交的方式，建立可长期维护的 URL 索引库，解决个人书签分散、链接失效、检索困难等问题。

项目定位为“技术资源的元仓库”，适用于需要快速查阅第三方依赖、环境配置参考、在线 API 测试工具或临时性技术方案的用户。LinkSphere 不对收录链接的内容质量做二次审核，但提供版本快照记录与访问可用性检测脚本，帮助用户判断资源的时效性与可靠性。当前批次为第 214/1241 批，共计收录 250 个资源链接。

## 功能概览

**批量链接导入** 支持通过纯文本列表或 CSV 文件一次性导入大量 URL，自动去重并生成唯一标识符。

**分类标签系统** 每个链接可绑定多个自定义标签，支持按技术领域、文件类型、协议版本等维度进行筛选与分组。

**可用性监控** 内置基于 Head 请求的存活检测任务，每日定时扫描所有收录链接，标记异常状态并生成变动报表。

**快照归档** 对重要链接自动抓取页面标题、描述与最后修改时间，形成本地元数据缓存，避免原始页面下线后信息丢失。

**只读 API 接口** 提供 RESTful 风格的查询端点，支持按标签、域名、收录时间范围等参数获取链接列表，便于第三方工具集成。

**Markdown 报告生成** 可将当前数据库内容导出为结构化 Markdown 文档，直接用于项目文档、周报或知识库沉淀。

## 应用场景

**技术选型参考** 当团队需要评估某类工具或库的生态活跃度时，可通过 LinkSphere 检索相关领域的社区推荐链接，快速获取官方文档、性能对比文章与实战案例。

**自动化文档构建** 结合 CI 流程，在每次构建项目文档时自动拉取 LinkSphere 中标记为“参考手册”类型的链接，生成最新的外部资源附录。

**离线环境准备** 运维人员可在有网络的环境下使用 LinkSphere 导出所有依赖下载地址，然后在隔离网络中批量获取安装包，避免逐个手动寻找。

**学习路线组织** 初学者可以按标签组合（如“Python + 异步 + 实战”）筛选出经过社区验证的学习资料，形成系统化的自学路径。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash。

```bash
# 克隆仓库
git clone https://github.com/linksphere/linksphere.git
cd linksphere

# 安装依赖（使用 pip 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动本地服务
python manage.py migrate
python manage.py runserver --host 0.0.0.0 --port 8080
```

访问 http://localhost:8080 即可进入管理面板。首次启动会自动生成管理员账户，初始密码见控制台日志输出。

| 依赖组件 | 必需版本 | 说明 |

| Python | 3.9 及以上 | 核心运行环境，低于此版本将无法解析类型注解 |

| gunicorn | 20.1.0 及以上 | 生产环境推荐的 WSGI 服务器 |
| virtualenv | 20.0 及以上 | 推荐用于隔离 Python 依赖环境 |

## 项目结构

```
linksphere/
├── cmd/                          # 命令行入口与调度脚本
│   ├── server/                   # HTTP 服务启动入口
│   └── monitor/                  # 独立运行的可用性检测守护进程
├── internal/                     # 内部核心包，不对外暴露
│   ├── storage/                  # 数据库抽象层（支持 SQLite 与 PostgreSQL）
│   ├── indexer/                  # 链接解析、标签提取与元数据规范化
│   ├── probe/                    # HTTP 探测器，含超时重试与 TLS 指纹模拟
│   ├── cache/                    # Redis 连接池与缓存策略实现
│   └── scheduler/                # 基于 cron 表达式的任务调度器
├── pkg/                          # 可复用的公共库
│   ├── api/                      # RESTful 路由定义与请求校验中间件
│   ├── model/                    # 数据实体定义（Link, Tag, Snapshot）
│   ├── utils/                    # 字符串处理、时间转换、哈希计算工具
│   └── exporter/                 # Markdown / JSON / CSV 格式导出器
├── web/                          # 前端静态资源与模板
│   ├── static/                   # 编译后的 CSS 与 JavaScript 文件
│   └── templates/                # Jinja2 模板，用于管理后台页面渲染
├── scripts/                      # 开发与运维辅助脚本
│   ├── import_batch.py           # 批量导入 URL 列表的脚本
│   ├── check_health.sh           # 手动触发全量检测的 Shell 包装器
│   └── migrate_db.sh             # 数据库版本迁移快捷命令
├── config/                       # 环境配置文件
│   ├── development.yaml          # 开发环境参数（调试模式、本地端口）
│   ├── production.yaml           # 生产环境参数（日志级别、数据库连接串）
│   └── staging.yaml              # 预发布环境配置
├── tests/                        # 单元测试与集成测试用例
│   ├── unit/                     # 对 internal 与 pkg 各模块的细粒度测试
│   └── integration/              # API 端点与数据库事务的端到端测试
├── docs/                         # 完整文档源文件，使用 Markdown 编写
├── go.mod                        # Go 模块依赖定义（项目使用 Go 语言开发）
├── go.sum                        # 依赖校验和
├── Makefile                      # 统一构建入口（编译、测试、打包、部署）
└── README.md                     # 本文档
```

## 贡献指南

我们欢迎任何形式的贡献，包括但不限于新增链接、修复检测逻辑、优化前端界面以及完善文档。请遵循以下步骤提交变更。

第一步：在 GitHub 上 fork 本仓库，并克隆到本地开发环境。确保你的 Go 版本不低于 1.21，且已安装 golangci-lint 工具。

第二步：创建新的功能分支，分支命名采用 `feature/xxx` 或 `fix/xxx` 格式。对于新增链接，请在 `data/batch/` 目录下按批次追加文本文件，不要直接修改已有的数据库导出文件。

第三步：编写或更新对应的单元测试，确保新增代码的覆盖率不低于 80%。所有测试用例必须能通过 `make test` 命令验证。

## 常见问题

**问：收录的链接如果失效了怎么办？**

系统每天凌晨 2:00 会自动执行全量存活检测，失效链接会被标记为 `unreachable` 状态并在管理面板高亮显示。用户也可以手动点击检测按钮立即重试。如果确认链接已永久关闭，拥有编辑权限的成员可将其归档至 `deprecated` 类别，不会从数据库中删除，以便保留历史记录。

**问：能否提交私有网络内部的链接？**

LinkSphere 原则上只收录公网可访问的资源。如果链接处于企业内网或需要 VPN 才能访问，请在提交时附加 `internal` 标签，并在备注中说明访问条件。监控检测任务会自动跳过带有 `internal` 标签的链接，避免误报。

**问：项目会存储链接对应的页面内容吗？**

不会。LinkSphere 只存储 URL、标题、描述标签以及最后一次访问的时间戳和 HTTP 状态码，不缓存任何页面正文、脚本或图片。所有访问请求均遵循 robots.txt 规则，且 User-Agent 会明确标识为 `LinkSphereBot/1.0`。
