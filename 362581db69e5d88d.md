# Weike Series Indexer

Weike Series Indexer 是一个系统化的技术资源导航与系列文章索引工具，专注于对 wei-ke.net 平台上的系列技术文档进行结构化整理和快速检索。该项目面向技术研究者、开发工程师以及文档维护人员，帮助其从大量分散的系列文章中快速定位所需内容，降低信息查找成本，提升知识获取效率。

该项目通过将系列文章的路径标识符映射为可读的元信息条目，构建轻量级的知识索引层。项目本身不存储文章正文，仅维护指向原始资源的引用关系与分类标签，确保版权归属清晰，同时为后续的自动化爬取、全文检索或推荐系统提供数据基础。

## 功能概览

**系列文章映射** 将形如 /series/xxxxx.html 的短标识符映射为完整可读的系列名称与简介，支持人工维护与批量导入。

**分类标签系统** 为每个系列赋予技术领域、难度等级、适用人群等多维度标签，支持按类别筛选与聚合。

**快速定位检索** 提供基于系列名称关键词、标识符前缀、更新时间等多条件的组合查询接口，实现秒级定位。

**外部资源引用清单** 集中展示项目所收录的全部原始 URL，确保用户可直接跳转至源站阅读完整内容。

**索引状态看板** 以表格形式展示各系列的收录状态、最后校验日期、失效链接标记，便于持续维护。

**批量导入与导出** 支持 CSV 与 JSON 格式的系列条目批量导入导出，方便与其他文档管理系统对接。

**变更日志追踪** 记录每次索引条目的增删改操作，支持回溯历史版本，确保索引变更可审计。

**静态站点生成** 提供内置模板引擎，可将索引数据渲染为静态 HTML 页面，便于部署为独立的导航站点。

## 应用场景

**技术团队内部知识库管理** 团队技术负责人可使用 Weike Series Indexer 将团队积累的系列技术文档统一编目，新成员入职时可快速浏览所有系列概览，按需深入学习，缩短上手周期。

**个人技术阅读规划** 开发者可将自己关注的系列文章标记为“待读”、“在读”或“已读”，系统根据标签推荐相关联的系列，辅助构建系统化的学习路径，避免碎片化阅读。

**文档自动化流水线前置环节** 在 CI/CD 流程中集成该索引工具，作为文档发布前的元数据录入步骤，确保每次新增系列文章均被正确索引，最终生成可供下游搜索服务使用的结构化数据文件。

**开源项目外链资源聚合** 开源项目维护者可将项目文档中引用的所有外部系列文章统一纳入该索引，避免链接散落在各处导致维护困难，同时提供统一的链接可用性检查入口。

## 快速开始

以下步骤帮助您在本地环境快速启动 Weike Series Indexer 服务。

```bash
# 克隆代码仓库
git clone https://github.com/weike-indexer/weike-series-indexer.git
cd weike-series-indexer

# 安装项目依赖（使用 npm）
npm install

# 启动开发服务器，默认监听端口 3000
npm run start
```

执行上述命令后，访问 http://localhost:3000 即可看到索引管理界面。首次启动时系统会自动生成示例索引数据，您可通过界面右上角的“导入”按钮上传自定义的系列映射文件。

若使用 yarn 作为包管理器，可将 npm install 替换为 yarn install，启动命令替换为 yarn start。项目同时提供 Docker 部署方式，详见文档导航章节中的部署指南。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或更高版本 | 是 | 项目运行时环境，推荐使用 LTS 版本，较低版本可能不支持 ES Module 语法 |
| npm 9.x 或 yarn 1.22+ | 是 | 包管理工具，用于安装项目依赖及执行脚本命令，两者择一即可 |
| SQLite 3 | 是 | 默认内嵌数据库，用于存储索引条目与元数据，无需额外安装，项目启动时自动创建 |
| 现代 Web 浏览器 | 是 | 访问管理界面，推荐 Chrome 110+、Firefox 109+ 或 Edge 110+，不支持 IE 系列 |
| 网络连接 | 否 | 仅当需要校验外部链接可用性时需连接互联网，本地离线模式可正常管理索引数据 |
| Docker 24.0+ | 否 | 如需使用容器化部署方式，需提前安装 Docker 引擎及 docker-compose 插件 |
| Git 2.30+ | 否 | 仅当从源码构建或参与贡献时需要，普通用户可直接下载发行版压缩包 |

│   │   ├── routes.js              # 所有 API 端点的路由注册与中间件挂载

│   │   ├── client.js              # SQLite 连接池初始化与关闭
│   │   ├── migrations/            # 数据库迁移脚本，按版本号命名
│   │   │   └── 001_initial_schema.sql  # 初始表结构：series, tags, series_tags
│   │   └── queries/               # 预编译 SQL 查询模板
│   ├── static/                    # 静态资源目录

## 贡献指南

项目遵循开源社区协作规范，欢迎任何形式的贡献，包括但不限于新增系列条目映射、修复链接失效、改进文档以及提交代码优化。

第一步，在 GitHub 上 Fork 本仓库至您的个人账户，随后将 Fork 后的仓库克隆至本地开发环境。请确保本地 Git 配置了正确的 user.name 与 user.email，以便提交记录可追溯。

第二步，创建新的功能分支，分支命名遵循 feat/xxx 或 fix/xxx 格式。若为新增系列条目，请在 src/core/mapper.js 中按照现有格式添加映射条目，并补充相应的单元测试至 tests/unit 目录。

第三步，完成修改后，运行 npm run lint 检查代码风格，运行 npm test 确保所有现有测试用例通过。新增功能需附带对应测试用例，测试覆盖率不得低于 80%。

第四步，提交代码并推送到您的远程分支，随后在 GitHub 上向本仓库的 main 分支发起 Pull Request。PR 标题需简明扼要描述变更内容，正文中请引用相关的 Issue 编号（如有）。

第五步，等待维护者进行 Code Review。如 CI 流水线检查失败或维护者提出修改意见，请及时在您的分支上补充修正并推送更新。PR 合并后，您的贡献将被记录在 CONTRIBUTORS 列表中。

## 常见问题

**问：项目启动后提示 SQLITE_ERROR: no such table: series，应如何解决？**

答：该错误表明数据库尚未完成初始化迁移。请检查是否在项目根目录下执行过 npm run migrate 命令。若执行后仍报错，可手动删除 data/ 目录下的 sqlite 文件，重新执行 npm run setup 命令，该命令会依次执行迁移并填充示例数据。若使用 Docker 部署，请确认 docker-compose 文件中已正确挂载数据卷。

**问：资源列表中的部分链接访问时返回 404，项目是否提供自动检测机制？**

答：项目内置了链接校验脚本 scripts/validate-links.js，您可通过 npm run validate 命令手动触发对所有已收录链接的 HTTP HEAD 请求检测。校验结果将输出至控制台并以表格形式汇总失效链接。该项目计划在后续版本中集成定时任务与 Webhook 通知功能，实现失效链接的自动告警。

**问：能否将索引数据从 SQLite 迁移至 PostgreSQL 或 MySQL？**

答：可以。项目在 src/db/client.js 中抽象了数据库适配层，目前默认使用 SQLite。若需切换至其他关系型数据库，您需要修改该文件中的数据库驱动引入与连接配置，并重写 queries/ 目录下的 SQL 语法以适配目标数据库的方言。社区已有部分用户贡献了 PostgreSQL 适配示例，可参考 docs/development.md 中的数据库迁移指南。
