# ResourceBridge

ResourceBridge 是一个面向技术社区与开发者的外链资源聚合与导航系统。该项目定位于解决技术信息碎片化、优质资源散落于各处难以检索的问题，通过人工筛选与结构化分类，将高频使用的开发文档、学习材料、运维手册、架构方案等外链资源整合为统一的检索与导航入口。目标用户包括软件工程师、运维工程师、技术架构师、技术社区运营者以及教育培训机构。ResourceBridge 本身不存储资源内容，仅提供链接索引与分类导航，所有资源指向第三方站点，确保版权合规与内容更新的实时性。

## 功能概览

**分类导航体系** 按照技术领域、适用场景、受众层级等多维度对资源进行标签化分类，支持快速定位目标资源类型。

**全文检索与模糊匹配** 内置标题与描述检索能力，支持关键词模糊搜索，帮助用户在海量外链中精准命中所需文档或工具。

**资源状态健康检查** 定期对收录的外链进行可达性探测，标记失效链接并生成告警日志，保证导航库的可用性。

**自定义分类标签管理** 允许用户或管理员新增、修改、删除分类标签，适配不同团队或社区的资源组织习惯。

**访问热度统计与排序** 记录每个外链的点击频次，支持按热度排序，便于发现当前社区关注的技术热点。

**导入导出与批量操作** 支持 CSV 与 JSON 格式的资源列表批量导入导出，便于迁移、备份或与第三方工具集成。

## 应用场景

**技术团队内部文档导航** 研发团队可将日常使用的 CI/CD 流水线文档、云平台控制台入口、代码规范、设计文档等外链统一收录至 ResourceBridge，作为团队首页，减少查找时间。

**开源社区资源聚合页** 开源项目维护者可将项目依赖的上下游工具、社区论坛、会议录像、贡献者指南等资源整合为导航站，降低新贡献者的上手门槛。

**技术培训机构课程外链管理** 培训机构或教育平台可将每门课程涉及的参考资料、在线 IDE、实验环境入口、作业提交系统等外链分类归档，学员通过导航页一站式访问。

**个人开发者的知识库枢纽** 个人开发者可将日常关注的技术博客、API 参考、组件仓库、在线转换工具等资源集中管理，形成个人化的技术门户。

## 快速开始

以下命令演示了从代码仓库克隆、安装依赖到启动开发服务器的完整流程。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行完毕后，访问控制台输出的本地地址即可预览导航首页。生产环境部署请参考 `docs/deployment.md` 中的 Nginx 与 Docker 配置示例。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，建议使用 nvm 管理版本 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖 |
| SQLite | 3.35 及以上 | 默认嵌入式数据库，无需额外安装 |
| Redis | 7.0 及以上 | 可选组件，用于缓存热度统计与会话管理 |
| Nginx | 1.22 及以上 | 生产环境反向代理与静态资源服务推荐 |
| PM2 | 5.x | 生产环境进程守护，可选但强烈建议 |
| Git | 2.30 及以上 | 用于克隆仓库与版本管理 |
| Docker | 20.10 及以上 | 容器化部署方式依赖，可选 |

## 项目结构

项目采用模块化单体架构，前端基于 React + Vite，后端基于 Node.js + Express，数据层使用 SQLite 与 Redis 混合存储。以下为关键目录与文件说明。

```
resourcebridge/
├── backend/                          # 后端服务代码
│   ├── src/
│   │   ├── controllers/              # 请求控制器，处理路由入参校验与响应封装
│   │   ├── services/                 # 业务逻辑层，包含链接管理、分类管理、健康检查等服务
│   │   ├── models/                   # 数据模型定义，映射 SQLite 表结构
│   │   ├── middleware/               # 中间件，包含鉴权、日志、错误处理、限流等
│   │   ├── routes/                   # RESTful 路由定义，按资源类型拆分
│   │   ├── utils/                    # 工具函数，包含 URL 标准化、时间格式化、哈希生成等
│   │   └── app.js                    # Express 应用入口，注册中间件与路由
│   ├── tests/                        # 后端单元测试与集成测试，基于 Jest
│   ├── package.json                  # 后端依赖与脚本定义
│   └── .env.example                  # 环境变量模板，包含端口、数据库路径、Redis 连接串
├── frontend/                         # 前端页面代码
│   ├── src/
│   │   ├── pages/                    # 页面级组件，包含首页、分类详情页、搜索页、管理后台页
```

## 贡献指南

欢迎社区开发者参与 ResourceBridge 的建设。请遵循以下步骤提交贡献。

1. 查阅问题列表与路线图。访问 GitHub Issues 页面，筛选标记为 `help-wanted` 或 `good-first-issue` 的任务，确认无重复工作后留言认领。

2. 派生代码仓库并创建功能分支。从主分支派生至个人账户，克隆至本地后，基于 `main` 分支创建新的分支，分支命名遵循 `feature/描述` 或 `fix/描述` 格式。

3. 编写代码并确保测试通过。前端修改需通过 ESLint 与 Prettier 检查，后端修改需补充对应的单元测试，确保 `npm run test` 全部通过且覆盖率不低于原有水平。

4. 提交变更并签署开发者原产地证书。提交信息遵循 Conventional Commits 规范，每个提交需包含明确的变更类型与范围描述。首次提交时需在 Commit Message 中附带 Signed-off-by 行，表明同意 DCO 协议。

5. 发起拉取请求并等待审核。前往 GitHub 创建 Pull Request，填写模板中的变更摘要、测试情况、关联 Issue 编号。项目维护者将在 3 个工作日内进行 Code Review，通过后合并至主分支。

## 常见问题

**问：ResourceBridge 与普通浏览器书签栏或收藏夹有何本质区别？**

答：浏览器书签是个人维度的扁平列表，缺乏分类层级、检索能力与协作特性。ResourceBridge 提供多维分类体系、全文搜索、热度排序、健康检查以及多用户管理能力，适用于团队共享与社区运营场景。同时，ResourceBridge 不依赖特定浏览器或设备，所有资源通过 Web 界面统一访问。

**问：如何保证收录外链的长期有效性？**

答：项目内置了定时健康检查脚本，默认每 24 小时对所有收录链接发起 HEAD 请求，检测 HTTP 状态码。失效链接会标记为 `unreachable` 并在管理后台高亮提示。管理员可通过管理界面批量编辑或移除失效资源。用户访问时如果检测到链接失效，前端会给出即时提示并建议管理员处理。

**问：是否可以导入现有的收藏夹数据或书签文件？**

答：支持。管理员可在管理后台通过「批量导入」功能上传 CSV 或 JSON 格式的文件，文件需包含 `title`、`url`、`category` 等必要字段。同时项目提供了 `scripts/import-csv.js` 命令行工具，支持从浏览器导出的 HTML 书签文件转换后导入，具体映射规则请参考 `docs/admin-guide.md` 中的导入章节。
