# LinkVault

LinkVault 是一个面向技术团队与内容运营者的轻量级外链资源聚合与管理平台。该项目定位于解决多源分散链接的集中存储、元数据标注、状态监控与快速检索问题，尤其适用于需要高频维护大量外联资源列表的文档站点、技术博客聚合页或项目推荐清单。LinkVault 不只是一个静态链接列表，它内置了链接可达性检测、访问频率统计、自定义标签分类与 Markdown 格式批量导出能力，帮助用户从杂乱的收藏夹或零散笔记中构建出结构化、可维护、可追踪的外链资源库。

目标用户包括开源项目维护者、技术内容创作者、在线教育课程运营人员以及企业知识库管理员。LinkVault 通过命令行界面与配置文件驱动，支持本地部署与轻量级云服务器运行，无需依赖重型数据库或复杂前端框架，即可完成每日数百条链接的定时校验与更新。项目以实用主义为设计核心，强调数据透明性与操作可复现性，所有资源记录均以纯文本 Markdown 表格存储，便于版本控制系统追踪变更历史。

## 功能概览

**批量链接导入** 支持从纯文本列表、CSV 文件或标准输入流中一次性导入大量 URL，自动去重并生成内部唯一标识符。

**自定义标签与分类系统** 允许用户为每个链接分配多个层级标签，支持按标签组合进行快速过滤与分组统计。

**定时可达性检测** 内置异步 HTTP 检测器，可配置检测频率与超时阈值，自动标记失效链接并记录响应状态码与响应时间。

**Markdown 格式导出** 支持将当前资源库按选定标签或全量导出为结构化 Markdown 表格，便于嵌入 README、文档站点或博客文章。

**检索与筛选接口** 提供基于链接关键字、标签组合、添加时间范围的多条件检索，结果以表格或纯列表形式输出。

**链接变更日志** 记录每次链接信息修改（包括标签变更、状态变化、备注更新），支持回溯任意时间段内的操作历史。

## 应用场景

开源项目文档外链管理：开源项目维护者可以使用 LinkVault 管理项目 README 中引用的所有外部资源链接，包括依赖文档、参考教程、社区论坛和工具站点。每日自动检测可及时发现失效链接，避免用户因访问死链而产生困惑。

技术博客聚合页维护：技术博客作者或聚合站点运营者可使用 LinkVault 整理每周推荐阅读列表、工具清单或嘉宾访谈链接。通过标签系统按主题（如"前端"、"后端"、"DevOps"）分类，便于读者按需浏览。

在线课程参考资料库：在线教育平台或培训机构的课程运营人员可将每期课程涉及的延伸阅读材料、视频资源、代码仓库链接统一录入 LinkVault，并按课时或模块打标。学生可通过检索快速定位特定章节的外部资料。

企业知识库外链监控：企业内部知识库中往往包含大量指向外部文档、供应商门户或行业标准站点的链接。LinkVault 可定期扫描这些外链的可达性，在链接变更或失效时生成报告，发送给知识库管理员进行更新。

## 快速开始

以下命令演示了从克隆仓库到启动服务的完整过程。确保系统已安装 Git 与 Python 3.10 及以上版本。

```bash
git clone https://github.com/linkvault/linkvault.git
cd linkvault
pip install -r requirements.txt
python linkvault.py --init --import ./samples/links.txt --start
```

上述命令执行完成后，LinkVault 将在当前目录生成 `data/` 文件夹，包含资源数据库文件、配置文件与日志文件。默认情况下，HTTP 检测服务将在后台启动，并按照 `config.yaml` 中设定的间隔执行检测任务。用户可通过 `--help` 查看所有命令行选项。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.10 或更高版本 | 是 | 核心运行环境，所有功能均依赖 Python 解释器 |
| pip 包管理工具 | 是 | 用于安装 requirements.txt 中列出的第三方库 |
| aiohttp 3.9.0 及以上 | 是 | 异步 HTTP 客户端，用于并发链接检测 |
| pyyaml 6.0 及以上 | 是 | 解析配置文件 `config.yaml` |
| click 8.1.0 及以上 | 是 | 命令行界面解析框架，提供子命令与参数校验 |
| pytest 8.0（开发依赖） | 否 | 仅当需要运行单元测试套件时安装 |
| mypy 1.8（开发依赖） | 否 | 仅当需要进行静态类型检查时安装 |
| 网络连接（出站） | 是 | 执行链接可达性检测时需访问目标域名 |
| 文件系统写入权限 | 是 | 用于创建 `data/`、`logs/` 目录及存储数据库文件 |

## 项目结构

```
linkvault/
├── linkvault.py               # 命令行入口文件，注册所有子命令并调度核心流程
├── config.yaml                # 主配置文件，包含检测间隔、超时阈值、日志级别等参数
├── requirements.txt           # 生产环境依赖列表，固定各第三方库版本号
├── data/                      # 数据存储目录，所有运行时生成的文件均存放于此
│   ├── links.db               # SQLite 数据库文件，存储链接元数据、标签、状态历史
│   └── imports/               # 导入缓存目录，存放最近一次批量导入的原始文件副本
├── src/                       # 核心源代码目录
│   ├── repository.py          # LinkRepository 类，封装数据库增删改查与事务管理
│   ├── checker.py             # HealthChecker 类，实现异步 HTTP 检测与结果回调
│   ├── exporter.py            # Exporter 类，支持 Markdown、CSV、JSON 三种格式导出
│   ├── tagger.py              # 标签管理模块，提供标签合并、拆分与重命名功能
│   └── utils/                 # 通用工具函数集合
│       ├── validators.py      # URL 格式校验、超时重试策略、指数退避算法
│       └── logger.py          # 日志初始化与滚动日志文件管理
├── tests/                     # 单元测试与集成测试套件
│   ├── test_repository.py     # 覆盖 LinkRepository 所有公共方法的测试用例
│   ├── test_checker.py        # 模拟 HTTP 服务，测试检测器的异常处理与并发控制
│   └── fixtures/              # 测试固定数据，包括示例链接列表与模拟配置文件
```

## 贡献指南

LinkVault 遵循开源社区协作规范，所有贡献者需遵守行为准则。欢迎提交功能增强、缺陷修复、文档改进与测试用例补充。

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境。确保本地已安装 Python 3.10 及以上版本，并执行 `pip install -r requirements-dev.txt` 安装开发依赖。

2. 新建以 `feature/` 或 `fix/` 为前缀的分支，例如 `feature/add-ftp-checker`。所有代码变更需保持与现有代码风格一致，并编写对应的单元测试覆盖新增或修改的逻辑。

3. 提交前运行 `pytest tests/` 确保全部测试用例通过，同时执行 `mypy src/` 进行静态类型检查，消除所有类型错误与警告。

4. 提交 Pull Request 时需填写标准模板，清晰描述变更目的、实现方式、测试结果以及是否影响向后兼容性。PR 至少需要一位项目维护者审核通过后方可合并。

5. 文档类贡献（如修正拼写错误、补充示例、翻译章节）可直接提交 Pull Request，无需关联 issue。功能类贡献建议先创建 issue 与维护者讨论设计方向，避免重复劳动或方案偏离。

## 常见问题

**问：LinkVault 能否处理需要登录或带有验证码的链接检测？**

答：LinkVault 内置的 HealthChecker 仅执行标准的 HEAD 和 GET 请求，不处理 Cookie、Session 或 JavaScript 渲染。对于需要登录态的链接，建议配置检测超时后将其标记为"需人工确认"状态，或通过自定义扩展脚本实现特定站点的认证逻辑。未来版本计划支持插件机制以扩展检测行为。

**问：如何将现有书签或收藏夹导入 LinkVault？**

答：大部分浏览器支持将书签导出为 HTML 文件，您可以使用 `scripts/parse_bookmarks.py` 辅助脚本将 HTML 书签转换为 LinkVault 可识别的纯文本列表格式，再通过 `--import` 参数进行批量导入。若使用 Pinboard、Raindrop 等网络书签服务，可先导出为 CSV 格式，然后通过 `--import-csv` 命令直接映射字段。

**问：检测频率过高是否会影响源站性能？**

答：LinkVault 默认检测间隔为 24 小时，且使用异步并发控制，单次检测任务中的并发数可在 `config.yaml` 中通过 `max_concurrent` 字段调节（默认 10）。对于大型资源库（超过 500 条链接），建议将检测间隔调整为 48 小时或 72 小时，并设置 `random_delay` 参数为 true，使请求在时间轴上随机分散，避免对特定源站造成突发流量压力。
