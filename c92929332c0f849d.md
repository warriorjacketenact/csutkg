# Resource Catalog Gateway

Resource Catalog Gateway is a curated navigation and metadata aggregation system designed for technical researchers, content curators, and infrastructure engineers who need to maintain large-scale external link inventories. The project addresses the common challenge of organizing, validating, and presenting hundreds of externally hosted resources across multiple programme areas without building a full content management system from scratch.

The system provides a lightweight gateway layer that consumes structured URL lists, enriches them with availability probes, content-type inference, and keyword extraction, then renders them through a configurable presentation pipeline. It is not a search engine nor a web crawler; it is a catalog middleware that sits between static link collections and human-readable documentation or API-driven consumption.

## 功能概览

**批量链接注入** 支持一次性载入包含数百个外部 URL 的原始数据集，自动去重并规范化条目格式，保留原始协议与主机信息。

**存活状态探测** 对每条链接执行异步 HEAD 请求检测，记录响应状态码、响应时间及内容类型，区分可访问、重定向、超时与失效状态。

**分类标签生成** 基于 URL 路径结构、文件名特征及预设规则库自动生成分类标签，支持按节目、主题、文件类型等维度分组。

**元数据快照** 提取每个资源页面的标题元素、描述 meta 标签以及正文前若干字符作为摘要，形成可检索的索引条目。

**多格式输出** 将整理后的资源目录输出为 Markdown 表格、JSON API 响应或纯文本清单，适应文档站、前端应用与命令行工具等不同使用场景。

**定时刷新任务** 内置调度器支持按小时、日或周周期重新验证链接状态，自动标记变动条目并生成变更报告。

**查询过滤接口** 提供基于状态、分类、关键词组合的过滤查询能力，支持精确匹配与模式匹配两种模式。

**结构校验工具** 检查原始 URL 列表是否违反输出规则，包括协议一致性、域名大小写、结尾斜杠及多余 HTML 标签等常见错误。

## 应用场景

**技术文档库维护** 当开源项目文档中需要引用大量第三方资源时，维护人员可使用 Resource Catalog Gateway 生成统一的外链附录，并定期检查链接有效性，避免文档中出现死链。

**内容聚合站后端** 运营人员收集了来自不同来源的活动节目链接后，通过本系统进行去重、分类和状态预检，再将干净数据推送到前端展示页面，减少人工整理时间。

**内部知识库索引** 企业或研究机构内部积累了大量分散在各部门的共享资源链接，利用该网关建立集中式索引，并提供按状态或分类筛选的访问接口，提升内部资源利用率。

**自动化监控告警** 运维团队将关键业务依赖的外部链接接入系统，配置高频探测任务，当链接不可用时通过 Webhook 发送告警，实现被动监控的补充。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/resource-catalog/gateway.git

# 进入项目目录并安装依赖
cd gateway
pip install -r requirements.txt

# 执行快速启动命令，载入示例链接并生成报告
python gateway.py --input ./data/links.txt --output ./reports/status.md
```

## 项目结构

```
gateway/
├── gateway.py                 # 主入口，解析命令行参数并调度核心流程
├── requirements.txt           # 生产环境依赖锁，记录精确版本号
├── config/
│   ├── default.yaml           # 默认配置，含超时、并发数、重试策略
│   └── schema.json            # 配置文件的 JSON Schema 校验定义
├── core/
│   ├── loader.py              # 链接载入模块，支持 txt、csv、json 格式
│   ├── probe.py               # 异步探测引擎，管理连接池与请求会话
│   ├── parser.py              # HTML 元数据解析器，依赖 beautifulsoup4
│   └── classifier.py          # 分类标签生成器，基于路径与正则规则
```

## 常见问题

问：系统如何处理输入中带有协议缺失或格式异常的 URL 条目？

答：载入模块会尝试对每个条目进行规范化修复，包括补充缺失的协议头、移除尾部多余字符、解码百分号编码等操作。对于无法修复的条目，系统会将其标记为格式错误并跳过探测流程，同时生成警告日志记录在报告中，用户可在输出报告中查看被跳过的条目列表。

问：探测大量链接时如何控制资源消耗，避免被目标服务器限流？

答：系统提供多项调节参数，包括最大并发数、每个请求的延迟间隔、单次扫描总超时以及重试退避策略。用户可在配置文件中根据目标服务器的响应特征调整这些参数。默认配置采用保守策略，并发数限制为 20，请求间隔为 200 毫秒，适合绝大多数公开站点。

问：能否将网关集成到现有的 CI/CD 流水线中？

答：可以。系统支持以非交互模式运行，接受输入文件路径和输出格式作为命令行参数，返回码区分执行成功、部分失败与完全失败三种状态，适合在持续集成环境中作为静态检查步骤使用。同时支持通过环境变量覆盖配置项，便于容器化部署场景。
