# ResourceBridge

ResourceBridge 是一个面向开发者与技术研究人员的结构化外链资源汇总与导航工具。项目定位于对分散于互联网各处的优质技术文档、工具站点、参考手册及社区讨论进行系统性归集，通过可检索、可分类、可版本化的方式降低信息发现成本。项目本身不生产内容，但提供一套标准化的资源引用与组织框架，帮助团队或个人建立私有的技术知识索引。

目标用户包括：需要维护团队技术周报的工程负责人、建立个人知识库的研发工程师、以及需要快速定位特定领域参考资料的架构师。ResourceBridge 通过统一的条目格式、标签体系与周期性更新策略，解决技术书签碎片化、链接失效不可追溯、以及跨项目参考资料难以复用的问题。

## 功能概览

**结构化资源条目** 每个外链以固定格式收录，包含标题、原始 URL、收录日期、所属类别及简要描述，支持自由扩展元数据字段。

**分类与标签系统** 内置多级分类（如后端架构、前端工程、运维监控、算法理论），每条资源可归属多个标签，便于交叉筛选。

**全文检索与过滤** 基于标题、描述、标签及 URL 关键词提供实时搜索，支持按类别、收录时间范围、来源域名进行多维度过滤。

**批量导入与导出** 支持通过 CSV 或 JSON 格式批量导入现有书签集合，也可将全部索引导出为静态 HTML 或 Markdown 文档，方便离线查阅。

**链接健康检查** 定时对已收录资源发起 HEAD 请求，标记失效链接并生成报告，辅助维护者清理或更新条目。

**版本化快照** 每次资源列表更新时自动生成变更日志，记录新增、删除及修改操作，支持回溯任意历史状态。

## 应用场景

**团队技术周报素材管理** 技术负责人每周需要汇总组内成员分享的文章、工具和视频。ResourceBridge 可作为统一收件箱，成员通过约定格式提交条目，维护者审核后发布为当周资源集，同时自动归档到历史库中。

**个人学习路径规划** 开发者围绕某一主题（如 Rust 异步编程、Kubernetes 调度器原理）收集数十个参考链接。利用分类与标签功能，可将学习资料按难度层级、官方文档与第三方分析、实践案例等维度组织，形成渐进式阅读清单。

**离线文档站外链补充** 内部文档系统（如 Confluence、GitLab Wiki）中的技术方案往往需要引用外部标准或社区讨论。通过 ResourceBridge 生成稳定的资源短码，在内部文档中仅需引用短码，后续 URL 变更时只需在 ResourceBridge 中更新一处映射。

**开源项目 README 外链托管** 开源项目常见问题中需要列出大量相关工具或扩展。将外链清单托管在 ResourceBridge 中，项目 README 仅保留指向 ResourceBridge 对应分类页面的链接，降低维护负担。

## 快速开始

以下命令演示如何获取 ResourceBridge 源码、安装依赖并启动开发服务。

```bash
# 克隆仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装依赖（使用 npm，Node.js 18+）
npm install

# 运行开发服务器（默认监听 3000 端口）
npm run dev
```

启动后访问控制台输出中显示的本地地址（通常为 http://localhost:3000），即可进入资源管理面板。首次运行将自动生成示例数据，包含预设分类与若干测试条目。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 nvm 管理版本 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖 |
| SQLite3 | 系统自带或由 better-sqlite3 捆绑 | 内置数据库引擎，无需额外安装，用于存储资源条目与元数据 |
| Git | 2.30 以上 | 用于克隆仓库及版本控制，若通过 zip 下载可忽略 |
| 现代浏览器 | Chrome 110+ / Firefox 110+ / Edge 110+ | 管理界面运行于浏览器端，需支持 ES2022 语法 |

生产环境部署时建议额外准备 PostgreSQL 14+ 作为可选的替代数据库，但 SQLite 足以支撑个人或中小团队的使用规模。

## 项目结构

```
resourcebridge/
├── src/
│   ├── core/                     # 核心数据模型与业务逻辑
│   │   ├── resource.ts           # 资源条目实体定义，包含字段校验
│   │   ├── tag.ts                # 标签聚合根，管理标签与资源的关联
│   │   └── health.ts             # 链接健康检查调度器与结果存储
│   ├── storage/                  # 数据库适配层
│   │   ├── sqlite-adapter.ts     # SQLite 实现，负责初始化表结构与迁移
│   │   └── repository.ts         # 通用 CRUD 仓储接口
│   ├── web/                      # 前端管理界面
│   │   ├── pages/                # 页面组件：列表、详情、编辑、搜索
│   │   ├── components/           # 可复用 UI 组件：表格、标签输入、搜索框
│   │   └── static/               # CSS 样式与客户端 JavaScript 工具
```

## 贡献指南

ResourceBridge 欢迎各类贡献，包括但不限于新增资源条目、改进文档、修复缺陷及添加新特性。请遵循以下步骤：

1. 在 GitHub 上 fork 本仓库，并 clone 到本地开发环境。确保本地 Node.js 版本符合安装要求，执行 npm install 安装所有依赖。

2. 新建特性分支，分支名称应简要描述改动内容，例如 feature/add-category-filter 或 fix/health-check-timeout。所有改动请确保通过现有测试，并为新增功能编写相应的测试用例。

3. 若为新增资源条目，请按照 src/core/resource.ts 中定义的字段规范填写，确保 URL 可访问且描述准确。若为代码改动，请运行 npm run lint 和 npm run test 确认无风格警告及测试失败。

4. 提交变更时使用清晰且语义化的 commit message，推荐采用 Conventional Commits 格式（如 feat: 或 fix:）。推送分支后在 GitHub 上发起 Pull Request，并在描述中关联相关 Issue（若有）。

5. Pull Request 将由维护者进行代码审查，可能会要求补充测试或调整实现细节。合并后您的贡献将出现在下一版本的更新日志中。

## 常见问题

**问：收录的资源链接失效了怎么办？**

答：ResourceBridge 内置了链接健康检查功能，默认每 24 小时自动扫描所有已收录资源。失效链接会在管理界面的“健康报告”面板中以红色高亮显示。您可以直接编辑对应条目更新 URL，或标记为“已废弃”并添加备注说明。若希望手动触发检查，可在命令行执行 npm run health:check。

**问：如何迁移或备份我的资源数据？**

答：数据存储于 SQLite 数据库文件（默认为 data/resourcebridge.db）。您可以直接复制该文件完成完整备份。如需迁移到其他环境，建议使用导出功能（npm run export -- --format=json）生成可读的 JSON 文件，再在新环境中通过导入功能（npm run import -- --file=backup.json）恢复。若从 SQLite 迁移至 PostgreSQL，需修改环境变量中的数据库连接字符串，并运行 npm run migrate 执行模式迁移。

**问：能否在团队内多人协同管理资源？**

答：ResourceBridge 当前版本为单用户设计，但支持多实例共享同一个数据库文件（如放置于网络共享存储）。如需完整的协同功能（如权限划分、操作审计、审核流程），建议搭配外部认证代理（如 OAuth2 Proxy）并提供只读/读写分离的 API 密钥。团队协同的完整原生支持已在 v2.0 路线图中规划。
