# EMHA Programme Resource Aggregator

EMHA Programme Resource Aggregator 是一个专注于聚合、整理和检索 EMHA 演出与节目相关技术资源的开源工具链项目。本项目并非 EMHA 官方项目，而是由社区开发者发起，旨在为研究者、档案管理员、开发者和兴趣爱好者提供一套结构化的外链资源汇总方案，通过标准化的数据组织方式，降低对分散节目页面信息的追踪与整合成本。

项目定位为技术资源与外链汇总站，其核心目标用户包括：从事数字人文研究的学者、需要批量处理节目数据的爬虫开发者、对 EMHA 节目历史感兴趣的内容策展人，以及希望复用结构化节目链接进行二次应用开发的技术人员。项目本身不存储任何节目内容，仅提供链接整理规范、检索辅助脚本与资源导航框架。

## 功能概览

**结构化外链索引** - 提供基于节目代码的链接分类与层级化索引，支持按节目代码前缀、页面类型或内容主题进行初步筛选与定位。

**自动化链接健康检查** - 内置链接可用性检测脚本，可定期校验资源列表中各节目页面的 HTTP 状态码，辅助识别失效或迁移的页面。

**批量元数据提取辅助** - 提供基于页面标题与元标签的批量信息提取示例代码，帮助开发者快速从节目页面中获取结构化字段。

**多维度标签分类建议** - 根据 URL 路径特征与页面内容模式，提供标签生成规则与分类映射表，便于用户建立个性化的资源库体系。

**快速部署与离线镜像支持** - 支持通过本地克隆方式部署完整的资源导航站点，并可配合离线归档工具生成节目页面的静态备份。

**扩展性接口设计** - 预留了基于 JSON 与 YAML 格式的资源导入导出接口，便于与其他数据管理工具或自动化流水线集成。

## 应用场景

数字人文项目中的节目资料整理。研究团队在分析某一时期节目趋势时，需要频繁查阅多个节目页面。本项目的链接索引与分类功能可以帮助团队快速定位相关页面，减少重复检索时间。

爬虫开发中的初始种子集构建。当开发者需要编写针对 emha-shows.org 的节目信息爬虫时，本项目提供的资源列表可直接作为种子 URL 集合，简化爬虫初始配置流程。

个人知识库中的外链管理。内容策展人可以使用本项目的结构框架，配合个人笔记工具（如 Obsidian、Notion）建立节目页面的注释与标签体系，形成可追溯的知识网络。

自动化监控与变更检测。运维人员可利用项目中的链接健康检查模块，定期监测节目页面的可访问性与内容变更状态，及时发现异常页面。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/emha-community/emha-resource-aggregator.git
cd emha-resource-aggregator

# 安装依赖（Python 3.8+ 与 pip）
pip install -r requirements.txt

# 执行资源索引构建与链接健康检查
python build_index.py --input resources/emha_links.txt --output dist/index.html
python check_health.py --source dist/links.json --timeout 5
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.8 或更高版本 | 是 | 核心脚本运行环境，推荐使用 3.10 以上版本以获得更好的类型提示支持 |
| pip 20.0 以上 | 是 | Python 包管理工具，用于安装 requirements.txt 中列出的第三方库 |
| requests 2.28.0 以上 | 是 | 用于发送 HTTP 请求，支持链接健康检查与页面元数据提取功能 |
| beautifulsoup4 4.11.0 以上 | 否 | 如启用页面标题自动提取功能，则需要安装此库以解析 HTML 文档结构 |
| pyyaml 6.0 以上 | 否 | 如需使用 YAML 格式的导入导出接口，则必须安装该库 |
| git 2.25 以上 | 否 | 仅当需要通过版本控制拉取最新资源列表或提交更新时必需 |

## 常见问题

问：项目中的链接健康检查脚本是否会频繁访问 emha-shows.org 导致被限制访问？

答：健康检查脚本默认启用请求间隔控制（--delay 参数，默认 1 秒），并支持通过 --max-workers 限制并发请求数。建议用户在使用时保持默认延迟设置，避免对目标服务器造成过大压力。对于大规模检查，可考虑在非高峰时段运行。

问：我能否将本项目生成的导航页面部署为静态网站？

答：完全可以。build_index.py 生成的 dist/index.html 为纯静态页面，所有样式与脚本均内联或引用自本地相对路径。你可以将其部署到任何支持静态文件托管的服务上，如 GitHub Pages、Netlify 或 Vercel。

问：如何自定义链接分类标签？

答：修改 config/categories.json 文件即可。该文件采用前缀匹配规则，你可以新增或调整节目代码前缀对应的分类名称。修改后重新运行 build_index.py 即可生成带有新分类的导航页面。
