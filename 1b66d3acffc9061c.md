# Water Resource Intelligence Navigation System

Water Resource Intelligence Navigation System (WRINS) 是一个面向水利信息化领域的技术资源聚合与导航平台，专注于收录、分类和索引与水资源管理、水文监测、水质分析、防洪调度及智慧水利相关的技术文档、数据接口、开源工具和学术资源。本项目定位为水利行业开发者和研究人员的起点站，解决水利技术资源分散、标准不统一、检索效率低下的问题，通过结构化索引和定期更新机制，为用户提供可追溯、可验证、高可用性的外链资源池。

本项目并非传统意义上的代码框架，而是一个持续维护的元数据索引项目。项目核心资产为经过人工筛选和机器校验的 URL 集合，每个条目均附带分类标签、可用性状态和简要描述。项目采用 Markdown 驱动的文档体系，所有资源列表以纯文本形式存储，便于版本控制和协作编辑。本项目适用于水利信息化项目选型、技术方案设计、学术文献检索和行业标准跟踪等场景。

## 功能概览

**结构化资源索引** 提供按资源类型、技术领域和数据格式三维分类的导航体系，支持快速定位特定类别的水利技术资源。

**自动化可用性检测** 内置基于 HTTP 状态码和响应时间的链接可用性检查脚本，每日定时检测资源列表中的每个 URL，标记异常条目并生成健康报告。

**资源元数据增强** 为每个收录的 URL 补充标题、描述、关键词和内容摘要等元数据字段，通过语义分析提升检索精度和浏览体验。

**版本化变更追踪** 所有资源的新增、删除和更新操作均通过 Git 提交记录追踪，支持回溯任意时间点的完整资源列表状态。

**社区贡献工作流** 提供标准化的资源提交通道和审核流程，允许外部贡献者通过 Pull Request 提交新资源或更新已有资源信息。

**多维度检索接口** 支持按关键字、分类标签、数据格式和更新时间等条件进行组合检索，提供命令行工具和 RESTful API 两种查询方式。

**资源关系图谱** 基于资源间的引用关系和共现分析，构建可视化知识图谱，揭示水利技术生态中的隐含关联和热点趋势。

**定期快照归档** 每月生成资源列表的完整快照，包括所有 URL 的元数据和可用性状态，为长期趋势分析提供数据基础。

## 应用场景

**水利信息化项目技术选型** 水利系统集成商或设计院在启动新项目时，可通过本导航系统快速获取物联网数据采集平台、GIS 地图服务、水文模型引擎等各类组件的候选方案列表，通过对比资源描述和可用性数据，缩短技术调研周期。

**学术研究文献资料收集** 高校和科研机构的研究人员在进行水文预报、水资源承载力评估、气候变化影响分析等课题时，可利用本项目归集的数据库链接、标准文档索引和开源代码仓库，系统性地构建文献综述和实验环境。

**防汛应急指挥系统运维** 防汛部门的运维人员需要定期检查所依赖的外部数据源和服务接口是否正常，本项目的可用性检测功能可作为辅助监控手段，提前发现数据源失效或响应异常，保障系统稳定运行。

**开源水利工具生态调研** 开发者希望了解当前水利领域有哪些可用的开源软件库和工具链，本项目按功能领域对开源资源进行归类，便于横向比较不同工具的特性、活跃度和社区规模。

**行业标准与规范查询** 水利工程咨询人员需要查阅 SL 系列、GB/T 系列等技术标准时，可通过本项目的标准文档索引快速定位官方发布页面或权威解读文章，避免搜索引擎返回的无关信息和失效链接。

## 快速开始

以下步骤帮助您在本地环境部署本项目，获取完整的资源列表和辅助工具。

```bash
# 克隆项目仓库
git clone https://github.com/water-resource/wrins.git
cd wrins

# 安装依赖（Python 3.8+ 环境）
pip install -r requirements.txt

# 初始化本地资源索引并执行首次可用性检测
python scripts/init_index.py --check
python scripts/health_check.py --full-scan
```

完成上述步骤后，您可以在 `data/` 目录下查看完整的资源列表文件，在 `reports/` 目录下查看可用性检测报告。

## 安装要求

本项目作为纯文档和脚本集合，运行环境要求较低，主要依赖 Python 运行时和少量第三方库用于辅助功能。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | 3.8 及以上 | 运行检测脚本和工具链的核心解释器 |
| Git | 2.25 及以上 | 版本控制和贡献协作的基础工具 |
| requests | 2.28.0 及以上 | 用于发送 HTTP 请求进行可用性检测 |

| pandas | 1.5.0 及以上 | 处理和导出资源列表的表格数据 |
| pytest | 7.0.0 及以上 | 运行单元测试和集成测试（仅开发环境） |
| mkdocs | 1.4.0 及以上 | 生成项目文档站点（仅文档构建环境） |

## 贡献指南

本项目的持续运营依赖于社区贡献者的积极参与。以下步骤说明如何为本项目新增或更新资源条目。

**提交新资源** 在 `data/index.json` 文件中按照现有格式添加新的 URL 条目，包含完整的元数据字段（标题、分类、描述、关键词）。提交前请运行 `scripts/validate_entry.py --check` 校验格式合规性。

**更新现有条目** 若发现已有资源的元数据过期（如标题变更、描述不准确或分类错误），请直接在 `data/index.json` 中修改对应字段，并在提交信息中详细说明修改原因。

**报告失效链接** 通过 GitHub Issues 提交失效链接报告，使用 `broken_link_report.md` 模板填写资源 URL、检测时间和异常现象描述。项目维护者将定期处理并更新索引。

**参与分类体系讨论** 若认为现有分类无法准确描述某类资源，可在 Discussions 中发起分类调整提案，提供新增分类的定义、适用范围和示例资源，经过社区讨论后合并。

**本地验证贡献** 所有 Pull Request 必须通过 CI 流水线中的格式校验和单元测试。请确保在提交前本地执行 `pytest tests/` 通过全部测试用例，并运行 `python scripts/health_check.py --sample` 对新添加的资源进行抽样检测。

## 常见问题

**问：本项目提供的链接是否保证永久有效？**

答：本项目不持有或维护外部链接指向的内容，因此无法保证任何外部链接的永久有效性。但项目通过每日自动化检测机制监控每个 URL 的可用性，并在资源列表中标注状态。对于连续三次检测失败的 URL，将标记为疑似失效并提交 Issue 通知维护者。用户在使用前可参考最近的健康检测报告获取当前状态。

**问：如何请求添加特定类别的资源？**

答：请在 GitHub Issues 中使用资源提交模板填写请求，包含资源 URL、所属分类（若现有分类不适用请说明理由）、简短描述及推荐理由。项目维护者会评估资源的相关性、权威性和实用性后决定是否收录。通常审核周期为 3 个工作日。

**问：本项目与普通收藏夹或书签管理工具有何区别？**

答：本项目提供三个核心差异化能力：一是结构化分类体系和元数据增强，使得资源可按技术领域、功能类型等多维度检索；二是自动化可用性检测，持续监控资源健康状态；三是版本化变更追踪和社区协作机制，确保资源列表的可追溯性和持续更新能力。这些特性使其更接近于一个面向生产环境使用的资源目录，而非个人的收藏夹工具。
