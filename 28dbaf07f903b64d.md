# LinkVault

LinkVault 是一个面向技术研究人员、开发者与内容策展人的轻量级外链资源聚合与导航系统。该项目并非一个传统的内容管理系统，而是一个专注于高密度 URL 资源的结构化存储、分类展示与快速检索的前端应用框架。其核心定位是帮助个人或小团队将分散的优质外部链接（包括技术博客、工具站点、文档镜像、数据源等）整理为具备清晰分类层级与可读性列表的静态站点，从而解决收藏夹杂乱、链接失效、缺乏上下文说明等常见信息管理痛点。

LinkVault 不依赖数据库，基于纯静态 Markdown 与 HTML 构建，所有资源条目通过约定式目录与元数据模板进行管理，便于版本控制与自动化部署。目标用户包括需要维护团队知识库的技术负责人、运营技术内容聚合页面的社区管理员，以及希望公开分享个人阅读清单的独立开发者。

## 功能概览

- **多层级资源分类**：支持按主题、来源域名、内容类型等维度对链接进行任意嵌套分类，并自动生成分类索引页。
- **原始链接透传展示**：所有外链资源均以原始 URL 形式完整呈现，保留原域名与路径结构，确保引用可追溯性。
- **元数据扩展字段**：每条资源可附带标题、描述、标签、收录日期与维护状态（有效/失效/待审核），便于精细化运营。
- **模糊搜索与过滤**：内置基于页面静态 JSON 索引的即时搜索功能，支持按 URL、标题、标签关键词进行模糊匹配。
- **批量导入与校验**：提供脚本工具支持从 CSV、Markdown 列表或书签 HTML 批量导入链接，并自动执行可达性探测与协议一致性检查。
- **响应式列表视图**：桌面端与移动端自适应布局，资源列表以表格或卡片形式呈现，支持按更新时间或域名排序。
- **自定义主题与布局**：通过 CSS 变量和布局模板配置，可快速调整站点配色、列表密度与头部导航结构，适配不同品牌风格。
- **静态站点生成（SSG）**：基于 Node.js 构建脚本，将 Markdown 数据源编译为纯静态 HTML 文件，可直接托管于任何 Web 服务器或 CDN。

## 应用场景

1. 个人技术阅读清单管理：开发者可将日常积累的技术文章、API 文档、开源项目仓库链接统一收录，并按语言或框架分类，形成个人知识检索入口。
2. 团队内部资源共享：技术团队可将常用的运维手册、设计规范、内部工具地址、云服务控制台链接汇总于一处，新成员入职时可直接查阅，减少重复沟通。
3. 开源项目生态导航：开源项目维护者可在项目文档中嵌入 LinkVault 生成的资源页，列出周边工具、插件列表、社区博客与视频教程，丰富项目生态展示。
4. 垂直领域内容聚合站点：面向特定行业（如数据科学、嵌入式开发、金融量化）的编辑者，可利用 LinkVault 构建小型的领域资源导航站，定期更新并对外公开。
5. 链接失效监控与备份：结合周期性 CI 任务，可对收录链接进行自动访问检查，标记失效资源，并提供历史快照查看功能，防止重要引用丢失。

## 快速开始

以下步骤将指导您在本地环境快速启动 LinkVault 开发服务器，并生成示例资源站点。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装依赖（使用 npm 或 yarn）
npm install

# 构建静态站点并启动开发预览服务（默认端口 3000）
npm run build
npm run serve
```

执行上述命令后，打开浏览器访问 `http://localhost:3000` 即可查看生成的示例资源列表页面。若要导入自定义链接数据，请将您的 URL 列表按照 `data/sources/` 目录下的示例模板格式（YAML 或 JSON）添加，然后重新运行构建命令。

## 项目结构

```
linkvault/
├── config/                       # 站点全局配置目录
│   ├── site.yaml                 # 站点标题、描述、导航菜单配置
│   └── categories.yaml           # 资源分类层级定义（树形结构）
├── data/                         # 数据源目录（核心）
│   ├── sources/                  # 原始资源数据存放处
│   │   ├── huangliang/           # 按域名分组的资源数据
│   │   │   └── entries.yaml      # 该域名下所有条目（含标题、标签、状态）
│   │   ├── zohao/                # 另一个域名的资源数据
│   │   │   └── entries.yaml
│   │   └── custom/               # 用户自定义混合数据
│   │       └── manual.json       # 手动编辑的额外资源
│   ├── schemas/                  # 数据校验 Schema 定义
│   │   └── entry.schema.json     # JSON Schema 用于验证条目格式
│   └── templates/                # 资源展示模板（可覆写）
```

## 常见问题

**问：LinkVault 与普通书签管理器或在线导航站有何区别？**

答：LinkVault 更侧重于将链接资源作为可维护的“数据资产”进行管理，而非单纯的快捷入口。它通过静态站点生成的方式，使得链接集合可以直接部署为独立的网站，并且所有数据都保存在本地文本文件中，便于使用 Git 进行版本追踪、协作审阅和回滚。同时，项目内置了链接校验和元数据扩展能力，适合对链接长期有效性及上下文信息有较高要求的用户。

**问：部署生成的静态站点后，能否支持动态提交新链接或在线编辑？**

答：核心设计上，LinkVault 定位为静态站点，不包含服务端 API 或数据库，因此不支持在线直接增删改操作。所有数据变更均需要通过修改本地数据源文件后重新构建并部署。这种方式虽然牺牲了实时性，但换来了极高的安全性和部署灵活性，同时避免了后端维护成本。若需要在线编辑功能，可以考虑结合 Git 托管平台（如 GitHub）的在线编辑能力，或使用第三方 CMS 作为前端。

**问：如何处理原始链接中的特殊字符或非标准协议（如 magnet:、ftp:）？**

答：LinkVault 在数据层对 URL 格式不做强制校验，理论上支持任意协议字符串。但在链接校验脚本中，仅对 `http://` 和 `https://` 协议执行可达性检查。对于其他协议，校验步骤会自动跳过并标记为“特殊协议-跳过检测”。在展示时，所有链接均以原始字符串直接渲染为 HTML 的 `href` 属性，浏览器会根据协议自动调用对应处理程序（如 magnet 链接会唤起下载工具）。
