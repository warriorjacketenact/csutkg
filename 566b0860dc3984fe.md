# LinkVault Resource Aggregator

LinkVault 是一个面向开发者、技术研究人员与内容策展人的轻量级外链资源汇总与导航系统。该项目定位于解决分散在各类技术文档、社区讨论与内部知识库中的有价值外部链接难以统一管理、检索与共享的问题，通过结构化的分类索引与静态站点生成机制，帮助团队或个人将零散的书签、参考文档、工具站与学习资源转化为可维护、可版本控制、可部署的知识资产。

目标用户包括：需要维护技术选型参考库的架构师、整理实验环境依赖链接的研究人员、搭建团队知识导航页的工程效率负责人，以及希望公开分享收藏夹的技术博主。LinkVault 不依赖数据库，所有链接数据以 YAML 或 JSON 格式存储于仓库中，支持通过 CI/CD 自动构建为静态 HTML 仪表板，或直接以 Markdown 表格形式嵌入现有文档体系。

## 功能概览

- **批量链接导入与去重**：支持从 CSV、浏览器书签 HTML 或纯文本列表批量导入外部链接，自动识别重复条目并合并标签。

- **多维度分类与标签系统**：每条资源可标记多个层级分类（如 云原生 / 监控 / 日志）与自由标签，支持无限级子目录嵌套。

- **状态标记与健康检查**：可标记链接状态（有效/失效/需复审），内置轻量级 HTTP 头检查工具，定期扫描并输出失效报告。

- **全文检索与快速过滤**：基于链接标题、描述、标签与分类的本地全文搜索，支持按状态、分类、添加时间范围过滤。

- **自定义元数据扩展**：每条记录支持键值对形式的扩展字段，用于存储作者、收费模式、版本适配信息或内部备注。

- **静态站点生成器集成**：提供预设模板，可将链接数据渲染为响应式 HTML 导航页，支持暗色主题与移动端适配。

- **变更日志与审计追踪**：每次增删改操作自动记录时间戳与操作者（基于 Git 提交元数据），便于回溯历史状态。

- **开放 API 与 Webhook 触发器**：提供 RESTful API 用于外部系统集成，支持通过 Webhook 在链接库变更时通知下游系统。

## 应用场景

**技术团队内部知识库导航**：开发团队可将常用中间件管理面板、监控图表链接、日志查询入口与运维手册外部引用统一收归至 LinkVault，替代分散的浏览器书签，实现知识库与导航一体化，降低新成员上手信息寻找成本。

**开源项目外部依赖与参考资料索引**：开源项目维护者可使用 LinkVault 整理项目依赖的协议规范、上游库文档、基准测试数据源及社区讨论帖，作为项目 README 或文档站的外部资源附录，帮助贡献者快速定位上下文。

**技术博客或课程配套资源中心**：技术博主或讲师可为每篇博文或每节课程创建独立的链接集合，存放延伸阅读材料、实验环境地址、代码仓库镜像与在线工具，通过 LinkVault 生成静态页面挂载于站点子路径，提升内容服务完整性。

**个人书签管理与迁移枢纽**：个人开发者可将长期积累的浏览器书签导出后导入 LinkVault，利用标签与分类进行二次整理，并通过变更日志跟踪书签增减，避免浏览器同步导致的丢失或混乱，同时可作为迁移至其他浏览器的中间格式。

**合规审计与外部引用审查**：企业在对外发布技术方案或合规文档时，需审查所有引用的外部链接的有效性与内容适配性。LinkVault 的状态标记与批量检查功能可辅助快速生成外部引用清单及健康度报告，满足内部审计流程要求。

## 快速开始

以下步骤将在本地克隆项目、安装依赖并启动开发服务器，默认监听 1313 端口。

```bash
git clone https://github.com/linkvault/linkvault.git
cd linkvault
pip install -r requirements.txt
python linkvault.py serve --port 1313
```

访问 `http://127.0.0.1:1313` 即可浏览默认示例链接库。如需导入自有数据，请将链接文件放置于 `data/imports/` 目录下，执行 `python linkvault.py import --format csv --path data/imports/bookmarks.csv`，随后重新启动服务。

## 贡献指南

LinkVault 遵循常规开源贡献流程，所有提交需经过代码审查与测试验证。请按以下步骤操作。

1. 查阅问题列表与项目看板：访问 GitHub Issues 与 Projects 面板，确认待处理任务或提议的新功能，避免重复工作。对于较大变更，建议先创建讨论议题进行方案对齐。

2. 派生仓库并创建功能分支：从主仓库派生副本至个人账户，克隆本地后基于 main 分支创建命名规范的分支，例如 feature/import-notion 或 fix/check-timeout。

3. 编写代码与单元测试：遵循 PEP 8 风格指南，所有新增或修改的核心逻辑需附带对应 pytest 单元测试，确保测试覆盖率达到 80% 以上。运行 tox 以验证多版本兼容性。

4. 更新文档与变更日志：若修改用户可见功能（如新增配置项、变更命令行参数），必须同步更新 docs/ 下对应手册，并在 CHANGELOG.md 中记录变更条目，格式参照 Keep a Changelog。

5. 发起拉取请求并参与评审：推送分支后向主仓库 main 分支发起 PR，填写模板中的检查项与测试结果。评审通过且所有 CI 检查（含测试、代码风格、文档构建）为绿色后方可合并。

## 常见问题

**Q: 导入浏览器书签 HTML 时中文分类名称显示乱码，如何处理？**

A: 此问题通常由 HTML 文件的字符编码声明缺失或与实际编码不一致导致。请在导入前确认书签文件为 UTF-8 编码，或在导入命令中显式指定编码参数：`python linkvault.py import --format html --encoding gbk --path bookmarks.html`。若仍存在乱码，可使用文本编辑器将文件另存为 UTF-8 格式后重试。

**Q: 健康检查返回大量超时错误，但链接在浏览器中可以正常访问，原因是什么？**

A: 默认健康检查采用 HEAD 请求并设置 3 秒超时。部分服务器不支持 HEAD 方法或响应较慢，也可能因防火墙拦截非浏览器 UA 导致。可调整检查策略：使用 `--method get` 切换为 GET 请求，或通过 `--timeout 10` 增加超时阈值。若仍不通过，可在链接元数据中设置 `skip_check: true` 跳过特定链接。

**Q: 如何将 LinkVault 部署为团队内部服务，并实现多用户管理？**

A: LinkVault 本身不内建多用户与权限系统，但可通过以下方式实现团队协作：将数据目录 `data/vault/` 存放在 Git 仓库中，团队成员通过分支与 PR 提交变更，利用 CI 自动检查冲突与健康状态。若需 Web 界面下的权限控制，可配置反向代理（如 oauth2-proxy）在应用前增加身份验证层，或使用 API 网关管理访问密钥。
