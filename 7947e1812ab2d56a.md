# LinkForge

LinkForge 是一个面向开发者、技术研究人员与内容策展人的轻量级外链资源归集与导航系统。该项目定位于解决个人或团队在浏览、整理、分享大量外部 URL 时面临的碎片化问题，提供一套结构化的资源索引框架，帮助用户将离散的网页链接转化为可检索、可分类、可协作的知识资产。LinkForge 不依赖复杂的外部数据库，核心数据基于 Markdown 与 YAML 格式存储，适合部署在静态站点环境或作为个人知识库的补充模块。

## 功能概览

- **批量链接导入与清洗**：支持从文本列表、浏览器书签导出文件或简易 CSV 中批量导入 URL，自动去除重复项、识别失效链接并生成标准化条目。
- **自定义分类与标签体系**：用户可为每条链接分配多级分类和自由标签，支持基于分类路径和标签组合的快速筛选，便于构建个性化导航树。
- **全文检索与元数据过滤**：内置倒排索引机制，支持按标题、描述、分类、标签及备注内容进行全文搜索，同时可按添加时间、访问频率、来源批次等元数据维度过滤。
- **链接状态健康检查**：提供可配置的周期性 HTTP 探测，自动检测链接可访问性、响应状态码及重定向链，并以可视化标记提示失效或异常资源。
- **快照与备注管理**：支持为重要链接保存文本快照或截图引用，并允许添加多行备注、评分或阅读状态，辅助后续内容回顾与深度阅读。
- **多格式数据导出**：支持将当前筛选或全量资源列表导出为 HTML 导航页、JSON 结构化数据、OPML 订阅列表或纯文本清单，便于嵌入其他系统或离线分发。
- **协作与审核工作流**：内置简单的多用户权限区分（管理员、编辑者、访客），支持链接提交、审核发布、变更历史记录，适用于团队知识库共建场景。

## 应用场景

- **技术文档库外链管理**：技术团队在撰写项目文档或技术白皮书时，可将外部参考链接、标准规范、相关工具网站统一收录至 LinkForge，并在文档中通过短引用标识快速调用，避免外链散落于各章节导致维护困难。
- **个人研究素材整理**：研究人员在文献调研阶段，可批量保存论文预印本、数据集页面、开源代码仓库及学术博客链接，利用标签和备注记录阅读进度与核心观点，形成可追溯的研究素材库。
- **开源项目推荐页生成**：开源社区维护者可使用 LinkForge 维护生态项目推荐列表，定期检查依赖项目或插件的官网可访问性，并一键生成优雅的推荐页嵌入项目官网或 README，提升社区资源曝光度。
- **运营内容策展**：内容运营人员可围绕特定主题（如 AI 工具周报、前端框架更新汇总）收集外链，通过分类和标签快速组织专题合集，导出为 HTML 页面后发布至公司博客或邮件通讯，提高内容生产效率。
- **内部知识库导航**：企业内部可将 LinkForge 作为部门知识导航的入口，整合常用管理后台、运维监控面板、设计规范文档、培训视频链接等，配合健康检查功能及时发现因域名变更或权限调整导致的失效链接。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，假设已安装 Git 与 Node.js（v18 及以上）。

```bash
git clone https://github.com/your-org/linkforge.git
cd linkforge
npm install
npm run start
```

执行完成后，服务默认监听本地 3000 端口。通过浏览器访问 http://localhost:3000 即可进入初始化向导，根据提示创建管理员账户并导入第一批链接数据。若需使用后台守护模式，可替换启动命令为 `npm run start:daemon`。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用官方预编译二进制或 nvm 管理 |
| npm | 9.x 或 10.x | 随 Node.js 一同安装，用于管理项目依赖包 |
| SQLite3 | 3.40 及以上 | 默认内嵌轻量级数据库，无需额外安装；若使用 MySQL 或 PostgreSQL 需额外配置连接参数 |
| Git | 2.30 及以上 | 用于克隆仓库及后续拉取更新，Windows 用户建议配合 Git Bash 使用 |
| curl / wget | 任意稳定版本 | 可选组件，用于外部链接健康检查的备选探测方式；若缺失则使用内置 http 模块 |
| Tesseract OCR | 5.0 及以上 | 可选组件，仅在启用图片内文字识别功能时需要，默认关闭 |
| ffmpeg | 6.0 及以上 | 可选组件，用于处理视频链接的封面截取与元数据读取，默认关闭 |
| Docker | 20.10 及以上 | 可选组件，若采用容器化部署方式则需准备 Docker Engine 及 Docker Compose |

## 项目结构

```

│   ├── default.yaml               # 默认配置项（端口、数据库连接、日志级别）
│   ├── production.yaml            # 生产环境覆盖配置，由部署脚本按需引用
│   └── schema.graphql             # GraphQL API 类型定义，供前端查询引擎使用

│   │   └── import-export.ts       # 批量导入导出处理器，支持多种格式适配器
```

## 贡献指南

我们欢迎任何形式的贡献，包括但不限于功能提议、代码修复、文档完善与问题反馈。为确保协作顺畅，请遵循以下步骤：

1. 查阅议题列表：在提交新功能或修复前，请先浏览 GitHub Issues 及 Projects 看板，确认当前是否存在相关讨论或进行中的工作。若议题不存在，请新建一个议题，清晰描述提议内容、背景及预期行为，并等待维护者反馈后再着手编码。
2. 分支管理规范：从主分支 `main` 拉取最新代码，并基于此创建以 `feature/` 或 `fix/` 为前缀的特性分支。提交信息遵循常规提交规范（Conventional Commits），即 `type(scope): subject` 格式，其中 type 可选 feat、fix、docs、style、refactor、test、chore。
3. 代码质量要求：所有新增或修改的代码必须通过项目配置的 ESLint 与 Prettier 检查，并补充相应的单元测试覆盖核心逻辑。提交前执行 `npm run lint` 与 `npm run test` 确保无回归问题。若涉及数据库变更，需在 `scripts/migrate-db.js` 中添加对应迁移脚本。
4. 提交拉取请求：推送分支至远程仓库后，通过 GitHub 界面发起 Pull Request 至 `main` 分支。请在 PR 描述中关联相关议题编号，并逐项填写 PR 模板中的检查清单（包括测试状态、文档更新情况、是否破坏兼容性等）。维护者将在 3 个工作日内进行 Review 并给出合并意见。
5. 文档与示例更新：若贡献内容影响用户可见功能，请在 `docs/` 目录下同步更新对应的指南文档，并在必要处补充代码示例或配置片段。对于新增配置项，务必在 `config/default.yaml` 中添加注释说明。

## 常见问题

**Q：启动时提示 SQLite3 数据库无法写入，应如何排查？**

A：此问题通常源于数据目录权限不足。请检查项目根目录下的 `data/` 文件夹是否存在且当前运行用户拥有读写权限。若该目录不存在，可手动创建并赋予 755 权限。若使用 Docker 部署，请确保将宿主机的数据目录正确挂载至容器内 `/app/data`，并确认 SELinux 或 AppArmor 未阻止写操作。此外，检查 `config/default.yaml` 中的 `database.path` 配置项是否指向有效路径。

**Q：健康检查功能频繁报告超时，但浏览器可以正常访问目标链接，是什么原因？**

A：健康检查模块默认使用 Node.js 内置 `http` 模块发送请求，并遵循系统代理设置。若目标站点对 User-Agent 或 TLS 加密套件有特殊要求，可调整 `config/default.yaml` 中的 `healthCheck.userAgent` 与 `healthCheck.tlsOptions` 字段。此外，部分站点会屏蔽非浏览器流量，建议将 User-Agent 设置为常见浏览器标识。若超时阈值过短（默认 5000ms），可在配置中适当增大 `healthCheck.timeout`。对于需要携带 Cookie 或特定 Header 的链接，可在链接条目的扩展字段中自定义请求配置。

**Q：导入包含大量链接的 HTML 书签文件时，页面响应缓慢甚至卡顿，如何优化？**

A：对于超过 500 条记录的批量导入，建议使用命令行方式执行，避免浏览器超时。项目提供了 `npm run import:bulk -- --file=bookmarks.html --format=html` 命令，该命令在后台进程中处理数据，并将进度输出至终端日志。若需进一步优化，可在配置中调整批量写入的 `batchSize` 参数（默认 50），降低数据库事务压力。同时，确保 SQLite 数据库启用了 WAL 模式（Write-Ahead Logging），该模式可显著提升并发写入性能，启用方式见 `config/default.yaml` 中的 `database.wal` 开关。
