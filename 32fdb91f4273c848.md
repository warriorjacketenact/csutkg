# EON Resource Navigator

EON Resource Navigator 是一个面向技术研究人员、数据采集工程师和信息分析团队的高密度外链资源汇总系统。本项目以 EONCN 知识库为基础，对分散在多个领域的技术文档、数据快照、政策解读与行业分析链接进行结构化整理与分类索引，解决信息碎片化导致的检索效率低下问题。项目本身不生产原始内容，而是通过人工筛选与自动化标注相结合的方式，为特定技术方向提供高价值的入口导航。

本项目适用于需要持续追踪特定技术领域动态、进行大规模信息采集规划或建立内部知识图谱的工程团队。通过统一的资源视图，用户可以在数秒内定位到原本需要数小时才能挖掘到的深层页面，显著降低信息发现成本。项目所有资源均以裸链接形式收录，保持原始访问路径，便于直接集成到各类采集脚本或自动化处理管道中。

## 功能概览

**多维分类索引**：对收录的每一份资源根据内容特征标注所属技术领域、信息类型与时效性等级，支持按类别快速筛选。

**裸链接直出模式**：所有资源链接保持原始域名与路径格式，不附加任何跟踪参数或跳转中间页，确保访问速度与链路透明性。

**批量导入与导出**：提供标准化的链接列表导入接口，支持 CSV 与 JSON 格式输出，方便与其他数据分析工具对接。

**时效性标记**：对每一条资源记录自动提取最后更新时间戳，并标记活跃度状态，帮助用户识别可能失效或过时的链接。

**标签体系与全文检索**：为每条资源分配多个层级的技术标签，配合轻量级全文检索引擎，支持模糊匹配与组合条件查询。

**访问状态监控**：定期对收录链接进行可用性探测，在管理界面中直观展示响应码与响应时间，辅助维护决策。

**权限分级管理**：支持多用户角色划分，不同级别的成员可查看不同密级的资源条目，适用于企业内部知识共享场景。

**自定义分类规则**：允许高级用户编写正则表达式或基于路径模式的自动归类规则，实现新入库资源的自动打标与归档。

## 应用场景

内部技术文库建设：技术团队可将本项目作为内部知识库的链接上游，定期拉取更新后的资源列表，经审核后合并入团队文档系统，确保团队成员始终访问到最新且经过验证的外部参考资料。

定向数据采集规划：数据采集工程师在进行特定领域的数据抓取前，可通过本项目的分类索引快速获取目标站点的入口列表，避免在搜索引擎中反复试错，大幅提高采集任务的前期准备效率。

技术动态追踪报告：行业分析师可利用本项目的时效性标记与标签筛选功能，定期导出特定主题下的新增资源清单，结合外部变化撰写趋势分析报告，辅助决策层把握技术演进脉络。

自动化巡检管道集成：运维人员可将本项目的链接列表作为输入源，纳入现有的自动化巡检系统，定时检测各项资源的可用性与内容变更，及时发现异常并触发告警流程。

## 快速开始

以下步骤指导您在本地环境中快速部署 EON Resource Navigator 的核心模块，并导入项目收录的资源索引。

```bash
# 克隆项目仓库至本地
git clone https://github.com/eon-resource/eon-navigator.git

# 进入项目工作目录
cd eon-navigator

# 安装项目依赖（使用 pip 基于 requirements.txt）
pip install -r requirements.txt

# 运行数据库初始化与资源导入脚本
python scripts/init_db.py --import-links ./data/eoncn_links.json

# 启动本地开发服务器，默认监听 8000 端口
python manage.py runserver 0.0.0.0:8000
```

| Nginx | 1.18 或更高版本 | 生产环境推荐的反向代理与静态文件服务组件 |

## 贡献指南

1. 复刻项目仓库至个人账户，在本地新建功能分支。分支命名建议采用 feat/xxx 或 fix/xxx 格式，并确保分支基于最新的 main 分支创建。

## 常见问题

问：项目是否提供对收录链接内容的本地缓存或镜像备份？

答：本项目仅提供链接导航与元数据索引服务，不存储任何外部页面的完整内容。用户访问具体资源时直接跳转至原始站点。如需离线访问或内容存档，建议用户自行使用第三方爬虫或网页归档工具进行处理。

问：如何批量导出当前收录的全部链接列表以供其他系统使用？

答：可通过调用 API 接口 /api/resources/export 获取 JSON 或 CSV 格式的完整资源列表。该接口支持按分类、标签和更新时间范围进行筛选，并支持分页下载。详细参数请参考文档中的 API 参考章节。

问：如果发现收录的链接已失效或内容严重偏离，如何处理？

答：用户可通过项目仓库的 Issue 模块提交链接失效报告，或直接在管理面板中标记该资源为异常状态。项目维护者会定期处理报告，并在确认后更新资源状态或从索引中移除。具备贡献权限的用户也可直接提交 Pull Request 修正链接。
