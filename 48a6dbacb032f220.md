# ResourceBridge

ResourceBridge 是一个面向技术团队与个人开发者的外链资源归集与分发系统。项目定位于解决多源技术文档、学习资料、社区讨论及工具站点分散管理的问题，通过统一的资源索引与结构化输出，帮助用户高效检索、分类和引用外部信息资产。

项目采用静态资源映射与动态路由解析相结合的架构，将所有外部链接以标准化流标识符进行管理，并提供可嵌入的访问入口。ResourceBridge 适用于需要频繁引用外部技术资料的内容创作者、文档维护者以及知识库运营方。

## 功能概览

**流式资源标识符映射** 每个外部链接均分配唯一流标识符，支持短链访问与永久引用。

**批量资源导入与校验** 支持从 CSV、JSON 及纯文本列表批量导入 URL，自动去重并校验可访问性。

**分类标签管理系统** 为每个资源赋予多级分类标签，支持按主题、来源、文件类型进行过滤与聚合。

**访问统计与热度分析** 记录每个资源被请求的次数与时间分布，生成访问趋势报表。

**结构化输出生成器** 将资源列表输出为 Markdown、HTML、JSON 及 RSS 订阅源格式，便于嵌入文档或站点。

**权限分级与私有资源标记** 支持对特定资源设置访问级别，区分公开可见与团队内部可见。

**定时健康检查与失效告警** 周期性检测链接可用性，自动标记失效链接并发送告警通知。

**全文检索与模糊匹配** 基于资源标题、描述、标签及 URL 路径构建倒排索引，支持快速全文搜索。

## 应用场景

技术文档站点维护者可使用 ResourceBridge 统一管理所有外部引用链接，当引用源迁移或失效时，系统自动提示并允许批量替换，避免文档中出现死链。

开源项目维护团队可将项目依赖的代码仓库、讨论区、CI/CD 服务面板等资源统一录入系统，新成员加入时通过单一入口即可获取全部开发所需的外部链接。

技术社区运营方可利用资源分类与访问统计功能，定期整理热门资料榜单，并通过 RSS 输出向订阅者推送最新收录的高质量内容。

企业内部知识库管理员可配置私有资源标记，将内部系统链接与外部公开链接分开管理，确保敏感入口仅对授权人员可见。

## 快速开始

以下命令演示了从克隆代码仓库到启动本地服务的完整流程。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run start
```

执行完成后，访问控制台输出的本地地址（默认为 http://localhost:3000）即可进入管理界面。首次启动会自动创建示例资源列表与默认管理员账户，初始密码在启动日志中打印。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.17.0 及以上 | 运行时环境，建议使用 LTS 版本 |
| npm | 9.0.0 及以上 | 包管理器，用于安装依赖项 |
| SQLite3 | 系统自带或 5.1.0 及以上 | 默认内置数据库，无需额外部署。生产环境可切换至 PostgreSQL |
| Redis | 7.0.0 及以上（可选） | 用于缓存热点资源数据与分布式会话管理。非必需但强烈推荐 |
| Nginx | 1.24.0 及以上（生产环境推荐） | 反向代理与静态资源缓存，提升并发访问能力 |

## 项目结构

```text
resourcebridge/
├── src/                           # 核心源代码目录
│   ├── core/                      # 核心模块：资源映射引擎、路由解析器
│   ├── api/                       # RESTful API 控制器与路由注册
│   ├── services/                  # 业务服务层：资源管理、分类服务、统计服务
│   ├── models/                    # 数据模型定义（SQLite / PostgreSQL ORM 映射）
│   ├── workers/                   # 后台任务：健康检查、统计聚合、告警发送
│   └── utils/                     # 工具函数：URL 校验、日志封装、加密工具
├── config/                        # 配置文件目录（包含默认配置与环境覆盖示例）
├── public/                        # 静态资源目录（前端管理界面 HTML / CSS / JS）
├── docs/                          # 完整文档（入门、操作、架构、API、部署、贡献）
├── scripts/                       # 运维脚本：数据库迁移、种子数据填充、备份恢复
├── tests/                         # 单元测试与集成测试用例（覆盖核心模块与 API）
├── logs/                          # 运行时日志存储目录（按日期轮转）
├── .env.example                   # 环境变量模板（数据库连接、Redis 地址、密钥配置）
├── package.json                   # npm 项目清单（依赖列表与脚本命令）
├── ecosystem.config.js            # PM2 进程管理配置（生产环境守护启动）
└── README.md                      # 本文件
```

## 常见问题

**Q: 导入大量 URL 时页面响应缓慢或超时，如何处理？**

A: 系统默认单次导入上限为 1000 条。若超过此数量，建议将文件分割后分批导入，或使用命令行工具执行批量导入（参考 `scripts/batch-import.js`）。同时可调整 `config/default.yaml` 中的 `import.batchSize` 与 `import.timeout` 参数以适配服务器性能。

**Q: 健康检查任务标记了某个链接为失效，但该链接在浏览器中可正常访问，原因是什么？**

A: 健康检查默认使用 HEAD 请求并遵循重定向，但部分站点会拒绝 HEAD 请求或要求特定的 User-Agent。可在配置文件中将检查方法改为 GET，或为特定域名设置自定义请求头。另外，检查超时时间过短也可能导致误判，建议将 `healthCheck.timeout` 调整为 5000 毫秒以上。

**Q: 如何从 SQLite 迁移到 PostgreSQL 生产环境？**

A: 首先在 `config/production.yaml` 中将 `database.dialect` 修改为 `postgres`，并填写正确的连接字符串。随后执行 `npm run migrate:up` 自动创建表结构，再使用 `scripts/export-sqlite.js` 导出 SQLite 数据为 JSON，最后通过 `scripts/import-postgres.js` 导入至 PostgreSQL。迁移前请务必备份原始数据文件。
