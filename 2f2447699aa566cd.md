# ResourceBridge 聚合索引系统

ResourceBridge 是一个面向技术文档检索、开源镜像站导航与在线资源归档的元数据聚合系统。项目定位于为开发者、研究人员与内容创作者提供结构化的外链索引与管理能力，通过标准化的目录映射和分类标签体系，将分散于不同源站的主题资源整合为统一的访问入口。目标用户包括需要维护内部知识库的团队、构建技术导航站点的运维人员以及希望快速定位特定主题原始资料的终端读者。系统不直接存储实体文件，而是通过可配置的抓取规则与链接校验机制，生成可验证、可更新的资源映射表，解决因链接散落、命名模糊、来源不可靠导致的信息检索效率问题。

## 功能概览

批量链接导入与自动分类 系统支持从文本文件或表格批量导入原始 URL，依据路径特征与关键词库自动划分至预设分类（如影音资料、节目档案、技术手册），降低人工整理成本。

链接可用性健康检查 内置异步 HTTP 探测器，支持自定义超时与重试策略，定期对已收录链接执行状态码校验，输出不可达或重定向链接报告，保障索引表的时效性。

结构化目录树生成器 根据链接的路径层级与命名约定，自动生成 ASCII 目录树视图，并允许用户添加注释字段，便于快速理解各子目录的职能归属。

多维度标签过滤 每一链接可附加多个自定义标签（如语言、格式、地域、主题），前端查询接口支持标签组合筛选，返回精确匹配结果。

Markdown 文档一键导出 将当前索引库的全部或部分条目导出为符合项目 README 规范的 Markdown 表格或列表，方便集成至静态站点生成器或 GitHub 仓库。

增量更新与变更日志 记录每次链接增删改的操作日志，支持回滚至历史版本，满足团队协作场景下的追溯需求。

原始地址严格保真输出 针对用户提供的任何 URL，系统在展示、导出、复制流程中均维持原始字符串形态，不自动补全协议头、不调整域名大小写、不添加尾部斜杠，确保与上游数据源完全一致。

## 应用场景

技术文档站点的外链附录管理 开源项目维护者可利用 ResourceBridge 管理 README 中引用的所有外部参考链接，通过健康检查功能提前发现失效引用，避免文档中充斥死链影响用户体验。

企业内部知识库的导航索引构建 企业技术团队可将分散于 Confluence、Notion 或内部 Wiki 的各类资源链接统一录入系统，利用分类与标签功能生成部门共享的导航页面，降低新成员的信息搜寻时间。

在线课程或讲座的资料汇总 教育机构或培训讲师可将每期课程涉及的扩展阅读、视频资料、工具网站等链接通过 ResourceBridge 整理为结构化列表，并导出为静态页面供学员查阅，同时保留原始域名信息以符合引用规范。

开源镜像站或 CDN 源站映射管理 运维人员可将多个地域或备用源站的地址录入系统，通过健康检查对比不同源站的可用性，动态调整流量调度策略，确保服务高可用。

## 快速开始

以下操作以 Linux/macOS 环境为例，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resource-bridge/resource-bridge.git
cd resource-bridge

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地数据库并导入示例链接
python manage.py migrate
python manage.py load_links --source sample_links.csv
python manage.py check_links --timeout 5
python manage.py export_readme --output README.md
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，低于此版本将无法解析类型注解与异步语法 |
| SQLite | 3.31 及以上 | 默认轻量级数据库，用于存储链接元数据与健康检查记录 |
| aiohttp | 3.8.0 及以上 | 异步 HTTP 客户端库，驱动链接健康检查并发请求 |

| pytest | 7.0.0 及以上 | 单元测试框架，仅在开发环境中需要 |
| Git | 2.25 及以上 | 版本控制工具，用于克隆仓库与提交变更 |

## 项目结构

```
resource-bridge/
├── manage.py                     # 项目入口命令行工具，聚合所有子命令
├── requirements.txt              # Python 依赖清单，锁定核心库版本
├── config/
│   ├── settings.py               # 全局配置（数据库路径、超时阈值、日志级别）
│   └── categories.yaml           # 分类规则与关键词映射表，可热加载           [注释: 用户可在此定制分类逻辑]
├── core/
│   ├── __init__.py
│   ├── fetcher.py                # 异步 HTTP 请求封装，含重试与代理支持       [注释: 核心健康检查驱动模块]
```

## 贡献指南

我们欢迎并感谢任何形式的贡献，包括但不限于功能提议、缺陷报告、代码实现与文档改进。请遵循以下流程以确保协作顺畅。

1. 查阅现有 Issue 与 Pull Request

2. Fork 仓库并创建特性分支
将主仓库 Fork 至个人账户，然后基于最新 main 分支创建以 feature/ 或 fix/ 为前缀的分支，例如 feature/link-tag-batch-edit。

任何新增功能或缺陷修复均需在 tests/ 目录下补充对应的单元测试或集成测试，确保覆盖率不低于原有水平。

从您的特性分支向主仓库 main 分支发起 PR，清晰描述变更内容、测试结果与相关 Issue 编号。评审通过后将由维护者合并。

## 常见问题

Q: 健康检查命令返回大量超时错误，应如何调整？
A: 可尝试通过 --timeout 参数增加单次请求的超时阈值（默认 5 秒），或通过 --retry 参数提高重试次数。若目标站点存在访问限制，请在 config/settings.py 中配置代理地址。同时建议检查网络环境是否稳定，部分境外站点可能需要调整 DNS 或使用 VPN。

Q: 导入链接时提示格式校验失败，但链接在浏览器中可正常访问，为什么？
A: 系统默认要求链接必须以 http:// 或 https:// 开头，且不得包含未转义的空格或中文字符。若原始链接为裸域名（如 example.com），系统不会自动补全协议头，请手动添加前缀后再导入。此外，部分链接包含非 ASCII 路径，需先进行 Punycode 编码或 URL 转义处理。

Q: 如何自定义分类标签，使新增链接自动匹配特定类别？
A: 编辑 config/categories.yaml 文件，在其中添加新的关键词映射规则。例如，若希望所有包含 manual 或 guide 路径的链接归入文档类，可在对应分类下添加正则表达式规则。修改后无需重启服务，执行 manage.py reload-categories 即可生效。
