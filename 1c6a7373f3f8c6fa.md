# LinkVault — 技术资源聚合与导航系统

LinkVault 是一个面向开发者与技术研究人员的轻量级外链资源聚合平台，专注于对高质量技术文章、工具文档、开源项目与社区讨论进行结构化收录与分类导航。项目本身不生产内容，而是通过人工筛选与社区贡献相结合的方式，持续维护一批高价值外部链接，帮助技术从业者在信息过载的环境中快速定位到可靠、相关、有深度的参考资料。

本项目适用于需要建立个人知识库、团队技术周报、开源项目文档站或社区资源导航页的场景。LinkVault 提供完整的链接管理后台、标签分类系统、全文检索接口以及静态站点生成能力，既可作为独立站点运行，也可嵌入现有技术文档体系作为外链补充模块。

## 功能概览

**链接收录与分类管理**：支持多级标签体系与自定义分类，每一条外链均可绑定多个标签，支持按分类、标签、发布时间、热度排序。

**全文检索与高级筛选**：内置基于倒排索引的轻量级检索引擎，支持标题、描述、标签、正文摘要的多字段联合检索，并提供按域名、语言、资源类型（文档/工具/视频/代码库）的筛选面板。

**定期健康检查与死链检测**：后台定时任务对已收录链接进行 HTTP 状态检查，自动标记失效链接并生成报告，支持配置重试策略与通知告警。

**用户贡献与审核工作流**：注册用户可提交新链接或对已有链接进行信息补充，管理员通过审核面板完成内容审核，整个流程支持邮件通知与操作日志追溯。

**静态站点导出与部署**：提供一键生成静态 HTML 站点的命令，输出适配移动端和桌面端的响应式页面，可直接部署至 Nginx、S3 或 GitHub Pages 等任意静态托管服务。

**开放 API 与数据导出**：提供 RESTful API 供外部系统调用，支持 JSON、RSS、OPML 三种数据导出格式，便于与其他阅读器或知识管理工具集成。

## 应用场景

**技术团队内部知识库外链模块**：技术团队可在内部 Wiki 或文档站中嵌入 LinkVault 作为外链导航，集中存放团队常用的框架文档、API 参考、运维手册、设计规范等外部资源，避免重复收藏与信息分散。

**开源项目社区资源导航页**：开源项目维护者可使用 LinkVault 搭建社区资源导航站，收录相关的插件列表、教程文章、视频课程、周边工具，降低新贡献者的学习门槛。

**个人技术博客的延伸资源站**：技术博主可将 LinkVault 作为博客的附属站点，整理每篇博文引用的参考资料、延伸阅读链接及工具推荐，形成体系化的知识外延网络。

**技术会议与培训材料配套系统**：技术大会或企业内训的组织者可利用 LinkVault 快速生成会议资料导航页，集中放置演讲稿、示例代码库、相关论文、视频回放地址，方便参会者事后查阅。

## 快速开始

以下步骤指导您在本地环境快速启动 LinkVault 服务。

```bash
# 克隆代码仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装依赖（使用 pip 和 npm）
pip install -r requirements.txt
npm install --prefix frontend

# 初始化数据库并创建管理员账户
python manage.py migrate
python manage.py createsuperuser

# 导入示例链接数据（可选）
python manage.py loaddata sample_links.json

# 启动开发服务器
python manage.py runserver
```

启动后访问 http://127.0.0.1:8000 即可使用前端界面，管理后台位于 /admin 路径。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 后端运行环境，核心业务逻辑与 API 服务 |
| Node.js | 18.x LTS | 前端构建工具与静态资源编译依赖 |
| PostgreSQL | 14.x 及以上 | 主数据库，存储链接、用户、分类、标签等结构化数据 |
| Redis | 6.x 及以上 | 缓存与消息队列，用于会话管理及异步任务 |
| Elasticsearch | 8.x | 可选依赖，用于增强全文检索性能，未安装时回退至内置检索引擎 |

## 常见问题

**Q：LinkVault 与普通书签管理器或收藏夹工具的区别是什么？**

A：LinkVault 面向的是团队协作与知识管理场景，而不仅仅是个人收藏。它提供用户权限控制、审核工作流、健康检查、全文检索、静态站点导出以及开放 API，这些能力使其能够作为技术团队知识库的基础设施组件，而非简单的网址收藏工具。

**Q：健康检查会影响已收录链接的访问频率或触发反爬机制吗？**

A：健康检查默认以较低频率（每日一次）运行，并采用 User-Agent 标识为 LinkVault/1.0 (HealthCheck)，同时遵循 robots.txt 规则。检查超时时间设置为 10 秒，仅验证 HTTP 状态码是否为 2xx 或 3xx，不下载完整页面内容，对目标站点几乎无影响。

**Q：如何从其他链接管理工具迁移数据至 LinkVault？**

A：项目提供了一套通用的数据导入接口，支持 CSV、JSON、OPML 三种格式。您可以在管理后台的「数据导入」页面中上传文件，系统会自动进行字段映射与校验。对于大规模迁移（超过 10000 条链接），建议使用命令行脚本 python manage.py import_links --file data.json 以提升效率。
