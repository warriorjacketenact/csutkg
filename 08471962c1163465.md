# LinkVault

LinkVault is a high-performance, schema-agnostic external link aggregation and cataloging system designed for technical documentation portals, knowledge bases, and research repositories. It addresses the critical challenge of managing large volumes of external references—such as legal case episodes, technical RFCs, or regulatory filings—by providing a structured, version-controlled, and queryable index layer over raw URL collections.

Target users include documentation engineers, legal tech researchers, compliance officers, and open-source project maintainers who need to maintain persistent, auditable, and navigable link registries without coupling to a specific content management system or database vendor. LinkVault does not scrape or cache external content; it provides deterministic metadata extraction, link health monitoring, and hierarchical categorization suitable for batch sizes up to 10,000 entries per manifest.

## 功能概览

**Deterministic Slug Generation** - Each URL is assigned a stable, reversible slug based on the hostname and path components, enabling predictable referencing across documentation versions without relying on external service availability.

**Batch Manifest Processing** - Ingest up to 1,000 URLs per operation with parallel validation, duplicate detection, and schema inference, producing a normalized JSON manifest suitable for downstream pipelines.

**Health Check Orchestration** - Scheduled or on-demand HEAD/GET requests with configurable timeouts, retry policies, and response status tracking, outputting health reports in both human-readable and machine-parsable formats.

**Categorical Tagging Engine** - Apply rule-based or LLM-assisted tag suggestions based on URL patterns, path segments, and query parameters, with support for user-defined taxonomy overrides.

**Versioned Snapshot Export** - Export the entire link registry as a timestamped archive (JSON, YAML, or Markdown table) for compliance auditing, backup, or integration with static site generators.

**Query Interface** - Filter, sort, and search the link collection by domain, status code, last-checked timestamp, custom tags, or free-text patterns using a lightweight CLI or embedded RESTful endpoint.

## 应用场景

**Legal Case Episode Tracking** - Legal research teams managing references to court episode transcripts or procedural filings can use LinkVault to maintain a master index of episode URLs, automatically checking for broken links and categorizing by jurisdiction, court level, or subject matter. The deterministic slug ensures that citation keys remain stable even if the underlying URL scheme changes.

**Technical Specification Registry** - Open-source projects that depend on external RFCs, IETF drafts, or vendor API documentation can integrate LinkVault into their CI pipeline to validate that all referenced specifications remain accessible before each release, reducing documentation rot and broken-link churn.

**Compliance Evidence Collection** - Organizations subject to regulatory audits can aggregate links to evidence artifacts, policy documents, or third-party attestations into a single manifest, with timestamped snapshots providing a tamper-evident trail of what was referenced at a given point in time.

**Educational Resource Indexing** - Academic courseware or training portals can leverage LinkVault to curate weekly reading lists, external lab environments, or video lecture URLs, offering students a single point of access while allowing instructors to monitor link availability across semesters.

**Internal Knowledge Base Curation** - Enterprise wiki maintainers can use LinkVault to decouple the link inventory from the wiki markup, enabling bulk updates, link validation, and categorization without editing individual pages, then re-import the enriched manifest via the wiki's API.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/your-org/linkvault.git
cd linkvault

# Install dependencies using pip and the provided requirements.txt
pip install -r requirements.txt

# Run the initial batch import with the sample manifest
python linkvault.py import --manifest samples/episode_manifest.json --output registry.json

# Perform a health check on all imported links
python linkvault.py health --registry registry.json --concurrency 20 --timeout 5

# Generate a Markdown catalog from the registry
python linkvault.py export --registry registry.json --format markdown --output CATALOG.md
```

## 常见问题

**Q: How does LinkVault handle URL changes or redirects?**

A: The health check module follows up to five redirects by default, recording the final resolved URL in the registry. If a permanent redirect (301 or 308) is detected, the registry entry is updated with the new target URL, and the old URL is preserved in a `redirect_history` field. Users can configure the maximum redirect depth and choose to enforce strict status code expectations via the `--require-status` flag.

**Q: Can I use LinkVault with private or authenticated endpoints?**

A: Yes. The health checker supports custom headers and bearer tokens via the `--headers` and `--token` CLI options, or through environment variables (`LINKVAULT_HEADERS`, `LINKVAULT_TOKEN`). For more complex authentication flows (OAuth2, client certificates), you can extend the `BaseFetcher` class in `src/linkvault/health.py` and register your custom fetcher in the configuration.

**Q: What happens when a link in the manifest is malformed or unreachable?**

A: During import, malformed URLs are flagged with a `validation_error` field and skipped from the registry unless the `--strict` flag is provided. Unreachable links (timeouts, DNS failures, 4xx/5xx status codes) are recorded with their error details and a `last_status` of `failed`. You can re-run health checks later with increased timeouts or different retry strategies, and optionally filter out or archive failed entries using the `--exclude-failed` export option.
