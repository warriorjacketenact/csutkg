# LinkVault 资源导航系统

LinkVault 是一个面向技术社区与内容创作者的轻量化外链资源汇总与导航平台。该项目定位于解决个人或团队在信息收集、分类整理与快速检索过程中面临的碎片化问题，提供结构化的资源索引能力。目标用户包括开源项目维护者、技术博主、教育培训机构以及需要长期维护大量外部链接的运营人员。

LinkVault 以静态站点生成方式运行，将原始链接数据与分类元信息通过模板引擎渲染为可浏览的 HTML 页面，同时支持输出 JSON 格式的结构化数据供其他系统调用。项目采用模块化设计，核心逻辑与主题样式完全解耦，便于二次开发和定制部署。

## 功能概览

**多级分类索引**：支持对资源链接进行无限层级的分类与子分类划分，每个分类可独立配置图标、描述与排序权重。

**批量导入导出**：提供标准 CSV 与 JSON 格式的链接批量导入接口，支持从主流书签管理工具无缝迁移数据，导出格式兼容多种静态站点生成器。

**全文检索引擎**：基于倒排索引实现标题、标签与描述字段的模糊搜索，检索响应时间控制在 200 毫秒以内，支持中文分词与拼音首字母匹配。

**链接健康巡检**：定时对已收录的域名发起 HEAD 请求检测，自动标记响应超时或状态码异常的链接，并提供离线链接清单导出功能。

**访问统计看板**：集成轻量级点击计数模块，按日、周、月维度展示热门资源访问趋势，数据存储采用 SQLite 本地数据库，无需外部依赖。

**自定义主题系统**：内置三套配色方案（亮色、暗色、高对比度），支持通过 CSS 变量覆盖基础样式，满足不同品牌视觉需求。

**权限分级管理**：提供管理员、编辑者、访客三级权限体系，编辑者仅可修改分类与链接元数据，管理员拥有系统配置与用户管理权限。

**数据快照备份**：每次发布构建前自动生成完整数据快照，支持一键回滚至任意历史版本，快照文件以时间戳命名存储于独立目录。

## 应用场景

技术博客与知识库的链接附录管理：技术作者在撰写系列教程时，常需引用大量外部文档、工具站或 API 参考链接。LinkVault 可将这些引用统一收纳并按章节分类，生成独立的资源附录页面，避免文章正文被长串 URL 干扰阅读体验。更新时只需修改数据源文件，全站引用链接自动同步刷新。

开源项目文档站的外链整合：开源项目的 README 或 Wiki 页面中通常包含贡献指南、社区论坛、CI 配置文件示例等外部链接。LinkVault 可作为子站点与主文档并行部署，将分散在不同 Markdown 文件中的链接集中管理，并通过 iframe 或侧边栏嵌入现有文档框架内，降低文档维护复杂度。

企业内部知识库的资源归类：企业行政或技术支撑团队需要向员工分发内部工具入口、审批表单链接、培训视频地址等。LinkVault 可部署在内网服务器，通过分级权限控制不同部门可见的资源范围，结合健康巡检功能自动提醒失效的内部系统入口。

教育机构的课程参考资料汇编：教师在开设编程或设计类课程时，需要为学生提供课外阅读材料、在线编译器、设计素材站等清单。LinkVault 允许按学期或课程单元建立独立分类，支持导出打印版 PDF 目录，方便无网络环境下的资料分发。

个人书签的规范化整理与跨设备同步：互联网从业者长期积累的浏览器书签常因缺乏分类体系而沦为杂乱列表。LinkVault 提供标签化整理方案，可通过关键词过滤快速定位特定技术栈或话题的资源，构建后的静态页面可托管于任意对象存储服务，实现跨设备访问。

## 快速开始

执行以下命令在本地环境中完成 LinkVault 的克隆、依赖安装与开发服务器启动。

```bash
# 克隆代码仓库至本地
git clone https://github.com/linkvault/linkvault.git

# 进入项目工作目录
cd linkvault

# 安装 Python 依赖包（推荐使用虚拟环境）
pip install -r requirements.txt

# 初始化 SQLite 数据库并创建基础表结构
python manage.py initdb

# 导入示例资源数据
python manage.py import --source fixtures/sample_links.json

# 启动开发调试服务器，默认监听 127.0.0.1:8000
python manage.py runserver
```

访问 http://127.0.0.1:8000 即可浏览本地运行的 LinkVault 实例。管理员后台入口为 /admin，默认账户为 admin，密码在首次启动时由终端输出打印。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 至 3.12 | 核心运行时，低于 3.9 版本不支持类型注解语法，高于 3.12 存在部分库兼容性风险 |
| SQLite | 3.35.0 及以上 | 内嵌数据库引擎，用于存储链接元数据、分类树与访问日志，3.35.0 版本引入对递归 CTE 的完整支持 |
| pip | 22.0 及以上 | Python 包管理器，用于安装 requirements.txt 中声明的全部依赖项 |
| Node.js | 16.0 及以上 | 仅用于前端资源构建（CSS 与 JavaScript 压缩），若使用预编译主题包可跳过此项 |
| Git | 2.25 及以上 | 用于版本控制与自动更新检测，生产环境部署时可通过 ZIP 包方式绕过 |
| 操作系统 | Linux / macOS / Windows WSL2 | 生产环境推荐 Debian 11 或 Ubuntu 22.04 LTS，Windows 原生支持有限 |
| 内存 | 最低 512 MB，推荐 1 GB | 构建大型资源站点（链接数超过 5000 条）时，内存需求可能上升至 2 GB |
| 存储 | 最低 200 MB 可用空间 | 主要用于存储 SQLite 数据库文件与静态资源缓存，链接数每增加 1000 条约消耗 5 MB |

## 常见问题

Q: 导入大量链接时出现超时或内存错误，应如何优化？

A: 对于超过 5000 条的批量导入，建议使用命令行工具 import 命令并添加 --chunk 参数，该参数将数据集拆分为每批 200 条的事务进行提交，显著降低内存占用。同时可关闭开发服务器的自动重载功能，避免导入过程中触发多次进程重启。若仍存在性能问题，可检查 SQLite 的 PRAGMA journal_mode 是否为 WAL 模式，该模式能提升并发写入吞吐量。

Q: 健康巡检功能对目标服务器会造成负载压力吗？

A: LinkVault 的巡检模块采用 HEAD 请求而非 GET 请求，仅获取响应头部信息，不传输响应体，单次请求开销极小。巡检间隔默认设置为 24 小时，且通过随机抖动算法将具体执行时间分散在每小时的前后 15 分钟内，避免所有链接在同一时刻集中请求目标域名。对于内网或私有地址段链接，可在配置文件中关闭巡检或添加白名单排除。

Q: 如何将现有浏览器书签（Chrome / Firefox / Edge）迁移至 LinkVault？

A: 各主流浏览器均支持将书签导出为 HTML 文件（书签管理器中的导出功能）。LinkVault 提供了专用的转换脚本 scripts/convert_bookmarks.py，该脚本可解析浏览器导出的 Netscape 格式书签文件，自动提取 URL、标题、添加时间与文件夹层级，并生成符合导入规范的 JSON 文件。执行转换后使用 manage.py import 命令完成数据入库。Safari 用户需先将书签导出为 CSV 格式，再通过通用导入器处理。
