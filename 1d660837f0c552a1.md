# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究、数据分析和内容挖掘领域的结构化外链资源汇总系统。本项目并非传统的爬虫或采集工具，而是一套人工筛选与自动化校验结合的 URL 收纳框架，旨在解决研究人员在分散来源中反复定位高质量外部链接的效率问题。项目定位为技术资源导航索引，核心用户包括安全分析师、SEO 研究员、内容运营人员以及开源情报（OSINT）方向的数据工程师。LinkVault 通过分层目录、状态监控和标签化分类，将大量原始 URL 转化为可检索、可审计、可版本化的外链资产。

## 功能概览

**批量链接导入与解析**：支持从纯文本、CSV 或 JSON 格式批量导入 URL 列表，自动识别协议、域名及路径结构，并生成初始分类标签。

**链接可用性健康检查**：内置异步 HTTP 探针，可配置超时与重试策略，定期对收录链接执行状态码检测、响应时间记录及 TLS 证书有效期校验。

**多维标签与全文检索**：每个链接支持自定义标签（如「视频源」「技术博客」「数据面板」），同时保留原始 URL 路径中的关键词索引，支持模糊搜索与正则表达式匹配。

**结构化目录树展示**：根据域名与路径深度自动生成逻辑目录树，方便用户从站点层级视角浏览资源分布，支持折叠、展开与节点注释。

**资源快照与版本对比**：每次更新导入会生成链接集合的快照，支持对比不同批次之间的增删差异，输出变更报告。

**开放 API 与 Webhook 通知**：提供 RESTful API 进行链接查询、状态更新和批量导出，并可配置 Webhook 在链接状态异常时推送告警。

## 应用场景

**安全威胁情报源管理**：安全团队可将大量可疑域名或第三方报告中的外链统一纳入 LinkVault，通过健康检查监控这些源的存活状态，一旦关键情报源不可用，系统立即发出告警，避免情报盲区。

**SEO 反向链接审计**：SEO 从业者定期导出第三方外链工具提供的引用域名列表，导入 LinkVault 后按域名分组，标记「已收录」「待审核」「已失效」等状态，大幅提升外链质量评估效率。

**学术文献数据溯源**：研究人员在撰写文献综述时，将参考文献中的网络资源地址集中收纳，系统记录每个链接的访问时间与响应状态，确保论文附录中的链接在审稿周期内保持可访问。

**开源情报交叉验证**：OSINT 分析师将来自不同信源的 URL 进行去重与分类，利用标签体系标注可信度、地理区域和语言，便于快速筛选特定维度的信息源。

## 快速开始

使用 Git 克隆项目仓库至本地环境，并通过 Python 虚拟环境安装依赖，最后启动开发服务器。

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
python -m venv venv
source venv/bin/activate  # Windows 环境请使用 venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

| pytest | 8.0 及以上 | 单元测试与集成测试框架（开发依赖） |

## 项目结构

```
linkvault-core/
├── src/                                 # 核心源代码目录
│   ├── core/                            # 项目配置与基础模块
│   │   ├── settings.py                  # Django 主配置，含数据库、缓存、中间件
│   │   ├── urls.py                      # 根路由分发，注册 API 与 admin
│   │   └── celery.py                    # Celery 应用实例与定时任务定义
│   ├── links/                           # 链接管理主应用
│   │   ├── models.py                    # 定义 Link、Tag、Snapshot、CheckRecord 四个 ORM 模型
│   │   ├── admin.py                     # Django 后台定制，支持批量导入与筛选
│   │   ├── api/                         # RESTful API 视图集（ViewSet）
```

## 常见问题

**问：导入大量 URL 时系统响应变慢，如何优化？**
答：建议使用命令行脚本 scripts/import_batch.py 进行后台导入，该脚本会分批提交任务至 Celery 队列，避免阻塞 Web 请求。同时可调整 settings.py 中的 BATCH_SIZE 参数（默认 1000），根据服务器内存适当降低该值以缓解数据库压力。

**问：健康检查任务如何自定义超时和重试次数？**
答：在 Django 管理后台的「系统配置」模块中，存在 CheckerConfig 模型，管理员可在线修改 default_timeout（单位秒）和 max_retries 字段。修改后保存，Celery 任务会在下一次运行时自动加载新配置，无需重启服务。

**问：能否将链接数据导出为其他格式供外部工具使用？**
答：API 端点 /api/links/export/ 支持查询参数 format=csv 或 format=json，并可配合标签过滤。此外，scripts/health_report.py 可直接生成包含状态码、响应时间和最后检查日期的 HTML 表格，方便嵌入监控看板。
