# LinkVault — 技术资源外链聚合与导航系统

LinkVault 是一个面向开发者、技术研究人员与内容创作者的轻量级外链资源聚合与导航平台。该项目旨在解决个人与团队在浏览、收藏、检索大量外部技术资源链接时面临的碎片化与低效问题，通过结构化的分类索引与元数据标记，将原本离散的 URL 转化为可检索、可分类、可共享的知识库。

项目目标用户包括开源项目维护者、技术博主、DevOps 工程师以及需要系统化整理学习资料的技术管理者。LinkVault 不提供资源内容本身，而是为每个链接赋予上下文标签、分类与状态标识，帮助用户在信息过载的环境中快速定位高价值内容。项目本身可作为静态站点部署，也可通过 API 模式与现有工作流集成。

## 功能概览

**批量链接导入与标准化**：支持从文件、剪贴板或书签导出文件批量导入链接，自动识别 URL 格式并剔除重复项，对无效链接进行初步过滤。

**多维度分类与标签系统**：每条链接可归属至自定义分类（如视频教程、技术文档、工具站点、社区讨论），并附加多个自由标签，支持层级化分类结构。

**全文检索与高级过滤**：基于标题、描述、标签、分类和备注内容进行全文搜索，支持按域名、文件类型、最后验证状态等多条件组合过滤。

**链接健康状态监测**：后台定时任务对已收录链接发起 HEAD 请求，检测可达性与响应状态码，标记失效或重定向链接，提供状态变化通知。

**收藏夹与阅读列表管理**：用户可将重要链接加入收藏夹或创建临时阅读列表，支持拖拽排序、备注添加与导出为 Markdown 或 JSON 格式。

**API 优先的设计**：提供 RESTful API 接口，支持第三方脚本、浏览器扩展或 CI/CD 流水线对链接数据进行增删改查，便于与其他工具链集成。

**数据快照与版本回溯**：每日自动生成全量数据快照，支持按时间点回溯链接库状态，防止误删或批量修改错误时数据丢失。

**公开分享页面生成**：可将指定分类或标签下的链接生成为只读公开页面，无需登录即可访问，适用于团队知识共享或项目文档引用。

## 应用场景

**技术团队内部知识库构建**：研发团队在日常开发中积累大量外部参考链接（API 文档、调试工具、性能分析案例）。LinkVault 可作为团队共享的链接中心，按项目或技术栈分类，新成员入职时可快速访问经过验证的资源列表，减少重复搜索成本。

**开源项目文档的扩展资源索引**：开源项目维护者通常需要在 README 或官方文档中引用大量外部参考链接。使用 LinkVault 管理这些链接后，可通过 API 自动生成链接列表并嵌入文档站点，确保引用链接的可维护性与一致性。

**技术内容创作素材管理**：技术博主或视频创作者在选题与资料搜集阶段需要收集大量参考资料。LinkVault 的标签与分类系统可帮助按主题、优先级、使用状态组织素材链接，创作时直接调用列表，避免反复切换浏览器标签页。

**个人技能学习路径规划**：学习者可将在线课程、教程文章、练习题集等链接按学习阶段（入门、进阶、实战）分类管理，结合健康状态监测功能及时发现失效资源，保证学习路径的持续可用性。

## 快速开始

以下步骤将在本地环境启动 LinkVault 开发或生产实例。

```bash
# 克隆代码仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装项目依赖（使用 pip 与虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化数据库与配置文件
cp .env.example .env
python scripts/init_db.py

# 启动开发服务器
python app.py --port 8080
```

启动后访问 http://localhost:8080 即可进入 LinkVault 仪表盘。首次运行将自动创建默认管理员账户，登录凭据在控制台输出中显示，请及时修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.9 及以上 | 核心运行环境，建议使用 3.11 或 3.12 以获得性能优化 |
| SQLite | 3.35 及以上 | 默认嵌入式数据库，用于元数据存储与索引；生产环境可切换至 PostgreSQL |
| Redis | 6.0 及以上 | 用于缓存与后台任务队列，非必需但推荐启用以提升查询性能 |
| Node.js | 18.x 及以上 | 仅用于前端静态资源构建，后端服务运行不依赖 |
| git | 2.25 及以上 | 用于版本管理与插件安装；生产部署可使用发布包跳过此依赖 |
| 系统内存 | 512 MB 及以上 | 最低运行内存，建议 1 GB 以上以支持并发任务；存储空间取决于链接总数及快照保留策略 |

## 项目结构

```
linkvault/
├── app/                           # 核心应用模块
│   ├── api/                       # RESTful API 路由与控制器
│   │   ├── v1/                    # API 版本 v1 端点实现
│   │   │   ├── links.py           # 链接增删改查接口
│   │   │   ├── categories.py      # 分类管理接口
│   │   │   └── health.py          # 健康检查与状态报告接口
│   │   └── middleware/            # 认证、限流、日志中间件
│   ├── models/                    # 数据模型层 (SQLAlchemy ORM)
│   │   ├── link.py                # 链接实体模型，包含 URL、标题、描述、状态字段
│   │   ├── tag.py                 # 标签实体与多对多关联表
```

## 常见问题

**Q: 导入大量链接时页面响应缓慢，如何处理？**

A: 导入操作默认在前台同步执行，当单次导入超过 500 条链接时可能影响页面响应。建议使用命令行导入模式：`python scripts/bulk_import.py --file links.json`，该命令绕过 Web 请求周期，直接操作数据库，完成后可通过仪表盘查看导入结果。同时可在配置中调整批量导入的批次大小，降低单次写入压力。

**Q: 健康检查功能是否会对外部站点造成压力？**

A: 健康检查采用异步队列与指数退避策略，默认并发请求数为 5，间隔时间至少 60 秒。检查仅发送 HEAD 请求，不下载响应体，对目标站点的资源消耗极小。用户可在 `config/default.py` 中调整 `HEALTH_CHECK_INTERVAL` 与 `HEALTH_CHECK_TIMEOUT` 参数，或针对特定域名配置白名单与黑名单规则。

**Q: 如何从旧版本的 SQLite 迁移至 PostgreSQL？**

A: 项目内置了迁移脚本 `scripts/migrate_db.py`，支持从 SQLite 导出数据并导入至 PostgreSQL。操作前请确保 PostgreSQL 数据库已创建且 `DATABASE_URL` 环境变量已正确配置。执行迁移命令：`python scripts/migrate_db.py --source sqlite:///data.db --target postgresql://user:pass@localhost/linkvault`。迁移完成后建议运行 `python scripts/verify_migration.py` 进行数据一致性校验。
