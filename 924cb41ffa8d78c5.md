# ResourceBridge

ResourceBridge 是一个面向技术社区与独立开发者的外链资源归集与导航系统。项目定位于解决分散在网络各处的优质技术文档、工具链页面、社区讨论与参考实现难以被系统化检索与复用的问题。ResourceBridge 本身不生产内容，而是通过结构化方式将外部高质量链接组织为可浏览、可分享、可继承的知识索引，适用于个人技术知识库构建、团队 onboarding 资料整理、开源项目外部依赖说明等场景。项目以静态站点形式交付，支持快速部署与低维护成本，目标用户包括技术作者、开源维护者、技术培训讲师以及需要长期维护外部引用列表的研发团队。

## 功能概览

- **链接分类归集** 支持按技术领域、内容类型、来源域名等维度对收录的 URL 进行手工标签与分类，便于按主题浏览。

- **元信息提取占位** 每条链接可记录标题、摘要描述、收录时间与失效检测状态，为后续自动化巡检预留数据接口。

- **静态索引生成** 基于 Markdown 与 YAML Frontmatter 生成可读性良好的索引页面，无需后端数据库即可部署至 GitHub Pages、Cloudflare Pages 或任何静态托管服务。

- **批量导入与校验** 提供脚本工具支持从 CSV、JSON 或纯文本列表批量导入链接，并自动校验 URL 格式合法性，避免无效条目混入索引。

- **失效链接标记** 内置基于 HTTP 状态码的链接可达性检查模块，可定期输出失效报告，帮助维护者清理或更新已断开的资源。

- **全文检索支持** 集成轻量级客户端搜索能力，基于 Lunr.js 或 MiniSearch 实现标题与标签的实时过滤，提升大型索引的查找效率。

- **自定义视图模板** 开放主题定制接口，允许用户修改页面布局、配色与列表展示密度，适配个人站点或团队文档站的整体风格。

- **版本化快照备注** 每条链接可关联多个版本备注字段，用于记录该资源在特定时间点的用途说明，例如项目依赖版本、对应代码分支或适配环境。

## 应用场景

- **技术文档站点外部引用管理** 当技术文档中需要引用大量第三方规范、SDK 下载页或社区教程时，ResourceBridge 可作为独立引用仓库，文档正文仅保留短引用标识，由 ResourceBridge 统一维护 URL 的有效性与更新记录。

- **开源项目 README 资源附录生成** 开源项目维护者可使用 ResourceBridge 管理项目生态相关的插件列表、示例工程或视频讲解链接，定期生成标准化的附录章节，减少手工维护 README 的负担。

- **团队知识库初始化物料整理** 新团队成员入职时需要访问一系列内部工具链地址、设计规范文档与学习路径资源，ResourceBridge 可快速生成按角色分类的链接看板，配合静态站点实现零登录访问。

- **技术播客或 Newsletter 过往链接归档** 内容创作者可将每期节目或邮件中提及的全部 URL 导入 ResourceBridge，按期数或主题建立索引，为受众提供可回溯的参考信息库。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，假设系统已安装 Git 与 Node.js 18.x 或以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装依赖（使用 npm）
npm install

# 执行本地构建与预览服务器
npm run build
npm run serve
```

执行完毕后，访问控制台输出的本地服务地址（通常为 http://localhost:8080 ）即可查看生成的索引首页。如需导入初始链接数据，请将 URL 列表按示例格式写入 `data/sources.json` 后重新运行 `npm run import`。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境与包管理基础 |
| npm | 9.x 或 10.x | 依赖安装与脚本执行工具 |
| Git | 2.30 及以上 | 版本控制与克隆操作 |
| 静态托管服务（可选） | 无版本要求 | 用于部署生成后的静态文件，如 GitHub Pages、Cloudflare Pages、Vercel |
| 现代浏览器 | 支持 ES2020 | 用于访问生成的索引界面，包括 Chrome 90+、Firefox 88+、Edge 90+ |
| 磁盘空间 | 至少 50 MB | 存放源码、依赖及构建产物 |
| 网络连接 | 外网可访问 | 用于初次安装依赖及后续链接可达性检测 |

## 常见问题

**问：ResourceBridge 是否提供在线演示站点？**

项目本身不维护公共演示实例，但用户可通过 `npm run serve` 在本地快速启动预览。如需长期对外展示，建议参考部署文档将构建产物托管至任意静态服务。项目源码中包含一组示例数据用于快速体验核心功能。

**问：导入的链接数量很大时，构建性能如何？**

在 250 个链接规模下，完整构建时间通常在 2 秒以内。当条目数超过 5000 时，建议启用增量构建模式（通过配置 `incremental: true`），该模式仅重新处理发生变更的数据文件。链接可达性检测采用并发请求控制，默认并发数为 10，可根据网络环境调整 `config/default.yaml` 中的 `checker.concurrency` 参数。

**问：如何为链接添加自定义字段，例如优先级或阅读状态？**

数据源 `sources.json` 支持灵活扩展，用户可在每个条目下自由添加 `meta` 对象，例如 `"meta": {"priority": "high", "read": false}`。模板层默认不会渲染未知字段，但可通过修改 `templates/default/list.hbs` 加入自定义展示逻辑，无需改动核心代码。
