# EMHA Showcase Index

EMHA Showcase Index 是一个面向技术内容策展与资源聚合的静态站点生成工具。该项目定位于帮助开发者、技术布道师与文档工程师快速构建结构化的外链资源目录，将分散于不同活动日程、节目表单与会议安排中的链接统一收录，并以可检索、可分类、可版本控制的方式进行管理。

该项目不提供爬虫或自动采集功能，而是基于 Markdown 与 YAML Frontmatter 的静态站点生成逻辑，将用户维护的 URL 清单转化为具备分类导航、标签过滤与全文搜索能力的资源列表页。EMHA Showcase Index 适用于需要长期维护大量外链引用、节目单存档、会议资料汇总或课程参考资料集成的场景，尤其适合与现有文档站点或静态博客系统并行使用。

## 功能概览

- **批量链接收录与去重**：支持从 CSV、JSON 或纯文本列表中批量导入 URL，自动识别重复条目并生成唯一标识符，确保资源列表的整洁性。

- **多级分类与标签系统**：每个资源条目可关联多个分类和标签，支持无限层级嵌套，便于按主题、年份、活动类型或技术领域进行精细化组织。

- **全文搜索与实时过滤**：内置基于 Lunr.js 或 Pagefind 的静态搜索索引，用户可在前端页面中按关键词、分类、标签或来源日期快速筛选目标链接。

- **自定义元数据扩展**：每条资源支持扩展元数据字段，包括但不限于发布日期、最后审查时间、维护者、摘要描述、优先级标记和关联项目编号。

- **自动生成多视图导航**：可根据资源元数据自动生成分类视图、标签云视图、时间线视图和项目批次视图，满足不同用户的浏览习惯。

- **周期性链接可用性检查**：项目提供命令行工具，支持对已收录链接进行 HTTP 状态检查，标记失效链接并生成报告，便于定期维护。

- **静态站点输出与主题定制**：生成完全静态的 HTML 文件，无需后端服务；内置默认响应式主题，并支持通过 Handlebars 或 EJS 模板进行完全自定义。

## 应用场景

- **技术会议与活动节目单存档**：组织者可使用 EMHA Showcase Index 汇总多届活动的演讲链接、工作坊资料和日程页面，按年份和会场分类，为参会者提供统一的回顾入口。

- **开源社区资源导航站**：开源项目维护者可构建项目生态资源导航，收录相关工具库、学习教程、视频演讲和社区博客，帮助新贡献者快速了解社区技术栈。

- **企业内部知识库外链管理**：企业技术团队可将散落在 Wiki、文档中心、邮件列表中的参考链接统一收录至 EMHA Showcase Index，配合定期可用性检查，避免知识库出现死链。

- **课程与培训材料参考目录**：教育机构或培训讲师可为每门课程维护独立的参考链接清单，按教学周次或主题模块分类，方便学员课后拓展阅读。

## 快速开始

以下命令演示了如何获取项目源码、安装依赖并启动开发服务器。

```bash
git clone https://github.com/emha-showcase/index.git
cd index
npm install
npm run dev
```

执行上述命令后，项目将在本地 3000 端口启动开发服务器。用户可通过编辑 `./data/links.csv` 或 `./data/links.json` 文件来管理资源列表，修改保存后页面将自动热重载。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Node.js | 是 | 版本需为 18.x 或 20.x LTS，用于运行构建脚本和开发服务器 |
| npm 或 yarn | 是 | 包管理器，用于安装项目依赖，推荐使用 npm 9.x 以上版本 |
| Git | 是 | 用于克隆仓库和管理版本，建议使用 2.30 及以上版本 |
| 现代浏览器 | 否 | 仅用于本地预览生成站点，推荐 Chrome 110+ 或 Firefox 110+ |
| 静态托管服务 | 否 | 生产部署时可选用 Netlify、Vercel 或 Cloudflare Pages，无需额外后端 |
| 可选：Python 3.9+ | 否 | 若需要使用额外提供的链接可用性检查脚本（基于 requests 库） |
| 可选：Docker | 否 | 若希望以容器化方式运行构建环境，需安装 Docker Engine 20.10+ |

## 常见问题

**问：项目是否支持动态后端或数据库？**

答：EMHA Showcase Index 为纯静态站点生成器，不依赖任何后端服务或数据库。所有数据均以 JSON 或 YAML 文件形式存储在 `data/` 目录下，构建时生成完整的 HTML 文件，可直接托管至任何静态文件服务。若需要动态更新数据，建议结合 CI/CD 流程定时触发重新构建。

**问：如何管理大量链接并避免重复？**

答：项目在导入数据时会依据 URL 的规范化形式（去除末尾斜杠、统一协议为 HTTPS）生成唯一指纹，若检测到重复条目，会在控制台输出警告信息，并在生成的 manifest.json 中记录重复计数。用户也可在导入时指定 `--dedupe` 参数，自动保留最新条目并合并标签与分类信息。

**问：生成站点的搜索索引是否支持中文分词？**

答：默认搜索索引基于 Lunr.js 构建，其内置分词器对中文支持有限。为获得更好的中文搜索体验，项目提供了可选的 Pagefind 集成方案，用户可在配置文件中启用 `search.provider: "pagefind"`，该方案使用更完善的中文分词逻辑。具体配置方法请参考文档导航中的主题定制章节。
