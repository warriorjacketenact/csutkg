# VideoLink Navigator

VideoLink Navigator 是一个面向技术内容创作者、在线教育平台运营者以及数字资源管理者的高性能视频外链聚合与导航系统。该项目并非视频托管服务，而是专注于解决视频资源分散、链接失效、分类混乱等痛点，通过建立结构化的外链索引与健康度监控机制，帮助用户将散落在各处的视频引用集中管理，提升资源复用效率与访问稳定性。

本项目特别适用于需要处理大量视频素材引用（如在线课程大纲、技术会议存档、游戏录像回顾、影视评论素材库等）的团队或个人。通过统一的条目管理与批量导入导出功能，用户可以快速构建专属的视频链接门户，并借助内置的链接可达性检查模块，定期自动验证所有外链的有效性，从而大幅降低维护成本。

## 功能概览

**批量链接导入** 支持从 CSV、JSON 以及纯文本列表批量导入视频链接，系统自动解析 URL 并提取域名、路径等元信息。

**智能分类标签** 允许用户为每条链接添加自定义标签和备注，支持多级分类目录，便于按项目、主题或日期进行筛选与检索。

**链接健康度监控** 内置异步任务调度器，可定期对所有存储链接发起 HEAD 请求，检测返回状态码并记录响应时间，异常链接将自动高亮标记。

**快速检索与过滤** 提供基于标题、标签、域名、状态码等多维度的全文检索能力，支持布尔运算符组合查询，结果可按相关性或时间排序。

**数据导入导出** 支持将全部链接数据导出为结构化格式（JSON、CSV、Markdown 列表），便于备份或迁移至其他系统。

**访问统计看板** 提供链接被点击次数、最后访问时间、来源域名分布等基础统计视图，帮助用户识别高频资源与冷门素材。

**团队协作支持** 支持多用户角色划分（管理员、编辑、只读访客），每条链接可记录创建者与最后修改者，便于团队内分工管理。

**开放 API 接口** 提供 RESTful 风格的 JSON API，允许第三方系统以编程方式查询、新增或删除链接条目，方便集成到现有工作流中。

## 应用场景

在线教育平台运营者可利用 VideoLink Navigator 统一管理课程中引用的外部视频素材链接。教学团队在制作课件时，经常需要引用不同视频平台上的片段作为案例。本系统允许课程策划人员提前将所有引用链接录入并按课程编号分类，同时在每次开课前自动触发链接健康度检查，确保所有素材在学员访问时仍然有效，避免因链接失效导致的负面教学体验。

技术会议与黑客松活动组织者可使用本系统归档演讲录像与演示视频。活动结束后，组织方需要将多个讲师的视频文件（通常托管在不同对象存储或视频平台上）整合成一份可公开访问的会议资料索引。通过本项目的批量导入与标签功能，可以快速生成按演讲主题、讲师姓名或时间排序的导航列表，并对外提供只读分享链接。

游戏内容创作者与录像回顾团队能够利用本系统维护个人精彩片段或比赛复盘视频的引用库。由于游戏录像往往分布在多个第三方平台，创作者可以通过本系统为每段录像添加地图、英雄、对阵队伍等标签，实现高效的素材检索与剪辑辅助，同时借助访问统计功能分析哪些类型的回放内容更受观众欢迎。

影视评论与媒体分析团队可使用本系统构建电影片段引用库。影评人在撰写分析文章时，需要反复引用特定电影桥段的在线链接。本系统允许按电影名称、导演、年代等维度组织链接，并提供备注字段记录时间码和场景描述，极大方便了跨影片的对比分析与引用查证。

## 快速开始

以下步骤帮助您在本地环境中快速启动 VideoLink Navigator 实例。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/video-link-navigator.git

# 进入项目根目录
cd video-link-navigator

# 安装项目依赖（使用 npm，确保 Node.js 版本 >= 18.0）
npm install

# 复制环境变量配置文件并填充必要参数
cp .env.example .env

# 执行数据库迁移脚本并启动开发服务器
npm run migrate
npm run dev
```

生产环境部署请参考 `docs/deployment.md` 文档，推荐使用 PM2 或 Docker 容器化方案。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | >= 18.0.0 | 项目运行时环境，推荐使用 LTS 版本 |
| npm | >= 9.0.0 | 包管理器，用于安装项目依赖 |
| PostgreSQL | >= 14.0 | 主数据库，存储链接元数据与用户信息 |
| Redis | >= 7.0 | 缓存与任务队列后端，用于健康度检查调度 |
| Elasticsearch | >= 8.0（可选） | 全文检索引擎，不安装时可降级为 SQL 模糊查询 |
| PM2 | >= 5.0（生产环境推荐） | Node.js 进程守护工具，确保服务持续运行 |

## 项目结构

```
video-link-navigator/
├── src/
│   ├── api/                        # RESTful API 路由定义
│   │   ├── v1/
│   │   │   ├── links.js            # 链接增删改查接口
│   │   │   ├── tags.js             # 标签管理接口
│   │   │   ├── health.js           # 健康度检查触发与查询接口
│   │   │   └── auth.js             # 用户认证与令牌刷新接口
│   ├── core/                       # 核心业务逻辑层
│   │   ├── linkService.js          # 链接条目业务逻辑（去重、分类、分页）
│   │   ├── healthChecker.js        # 异步健康度检查调度器（含指数退避重试）
│   │   ├── tagService.js           # 标签关联与合并逻辑
│   │   └── statsService.js         # 访问统计与聚合计算
│   ├── models/                     # 数据模型与 ORM 映射
│   │   ├── Link.js                 # 链接表模型（字段：url, title, status, lastChecked）
│   │   ├── Tag.js                  # 标签表模型
│   │   ├── User.js                 # 用户表模型（角色、密码哈希）
│   │   └── AuditLog.js             # 操作审计日志模型
│   ├── lib/                        # 基础设施与工具函数
│   │   ├── db.js                   # PostgreSQL 连接池与查询构造器
│   │   ├── redis.js                # Redis 客户端封装（缓存与 Bull 队列）
│   │   ├── elastic.js              # Elasticsearch 客户端封装（可选）
│   │   ├── logger.js               # 结构化日志（基于 Winston）
│   │   └── validators.js           # URL 格式校验、标签名净化
│   ├── queue/                      # 任务队列定义
│   │   ├── healthQueue.js          # Bull 队列配置，定义健康检查任务
│   │   └── worker.js               # 队列消费者，执行实际 HTTP 请求
```

## 常见问题

**问：健康度检查任务是否会因为目标服务器限流而导致本地 IP 被封禁？**

答：系统默认开启请求速率控制，每个目标域名在 60 秒内最多发起 10 次 HEAD 请求。同时支持配置 `respectRobotsTxt` 选项，启用后会尝试读取目标站点的 robots.txt 并遵守 Crawl-delay 指令。若仍需更高并发，建议通过代理池或分布式 worker 分散请求来源。

**问：如果外部视频链接发生了域名迁移或路径变更，系统能否自动修正？**

答：本系统不提供自动重定向追踪与永久修正功能，但健康度检查模块会记录每次请求的最终重定向链（3xx 跳转历史）。管理员可在后台查看重定向目标，并手动更新为新的 URL。未来版本计划引入基于机器学习的变化模式识别，辅助用户批量更新同类域名下的路径。

**问：能否将本系统部署为完全离线使用，即不依赖任何外部 CDN 或字体库？**

答：可以。项目前端资源（AdminLTE 主题、Chart.js 图表库）均已打包至 `public/vendor` 目录，无需外网加载。同时所有 API 响应均为纯 JSON 格式，不嵌入任何外站资源。生产部署时只需确保 PostgreSQL 与 Redis 在内网可达即可，无需互联网出口。
