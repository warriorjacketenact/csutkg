# LinkVault —— 结构化技术资源导航与链接治理平台

LinkVault 是一个面向开发者、技术团队与内容策展人的轻量级链接聚合与治理工具。项目定位为“技术资源的外链数据中台”，核心目标是将分散、孤立、易失效的外部链接纳入统一的生命周期管理体系，并提供标签分类、可用性巡检、访问统计与分享追踪等辅助能力。

LinkVault 并非传统的书签管理器或收藏夹工具，而是一套围绕“链接即资产”理念设计的工程化方案。它适用于需要长期维护大量外链资源的文档站、知识库、技术周报、项目官网或内部开发者门户。通过结构化的元数据描述与插件化的检测机制，LinkVault 帮助团队降低链接维护成本，减少死链污染，提升资源复用效率。

## 功能概览

- **链接入库与自动元数据提取**：提交任意 HTTP/HTTPS 链接后，系统自动抓取页面标题、描述、关键词及 Open Graph 信息，并生成标准化的资源卡片。

- **多级标签与自定义分类树**：支持无限层级的标签体系与分类目录，允许同时按技术领域、适用人群、使用频率、质量等级等多维度交叉组织链接。

- **定时可用性探测与状态变更通知**：内置异步巡检任务调度器，可每日或每周对全部外链执行 HEAD/GET 探测，自动标记异常状态（死链、重定向、超时、证书过期），并通过 Webhook 或邮件通知维护人。

- **链接使用分析面板**：统计每个外链的点击次数、最后访问时间、来源页面及引用频率，辅助判断资源的实际价值与淘汰优先级。

- **批量导入与导出机制**：支持 CSV、JSON 及 OPML 格式的链接批量导入，同时提供按筛选条件导出为 Markdown 表格或 HTML 列表的能力，便于嵌入外部文档。

- **用户权限与协作审批流**：提供基于角色的访问控制，支持“提交者-审核者-管理员”三级权限，所有新增或修改操作均生成审计日志，便于团队协作治理。

- **开放 API 与 Webhook 触发器**：对外提供 RESTful API 接口，允许第三方工具（如 CI/CD 流水线、IM 机器人、静态站点生成器）调用查询或写入链接数据，并可配置变更事件回调。

## 应用场景

1. **技术文档站点外链管理**：大型开源项目或企业技术文档库通常包含数百个参考链接。LinkVault 可定期检测文档内的所有外链状态，在版本发布前自动生成链接健康报告，避免用户访问失效资源。

2. **技术周报与资讯聚合平台**：编辑团队每日收集数十篇优质技术文章或工具发布信息。使用 LinkVault 统一入库后，可一键生成带分类和摘要的周报草稿，同时追踪每篇文章的读者点击热度。

3. **开发者门户导航页建设**：企业内部开发者门户需要维护一套稳定的常用工具链、镜像站、API 文档入口。LinkVault 的权限审批流确保任何链接变更都经过技术负责人确认，避免误删或误改关键入口。

4. **开源项目 README 资源清单治理**：许多开源项目的 README 会附带有用的社区资源列表。LinkVault 可定期同步这些清单并检测有效性，当发现大量失效链接时自动提醒维护者更新，提升项目专业形象。

## 快速开始

以下操作基于 Linux / macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash。

```bash
# 1. 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 2. 安装依赖（使用 Python 3.10+ 及 Poetry）
poetry install --no-dev

# 3. 初始化配置并启动服务
cp .env.example .env
# 编辑 .env 填写数据库连接、巡检间隔等关键参数
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

生产环境部署建议搭配 Gunicorn + Nginx + PostgreSQL，并配置 systemd 或 supervisor 保证进程常驻。详细部署文档参见 `docs/deployment.md`。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 或 3.11 | 核心运行环境，3.12 暂未完成兼容性测试 |
| PostgreSQL | 14.x 或 15.x | 主数据库，用于存储链接元数据、标签、审计日志 |
| Redis | 7.x | 缓存与异步任务队列后端，用于巡检任务调度与限流 |
| Node.js | 18.x | 仅前端管理面板构建时需要，后端运行时无需 |
| Poetry | 1.8.x | Python 依赖管理与虚拟环境隔离工具 |
| Nginx | 1.24+ | 生产环境反向代理与静态资源服务（可选但推荐） |
| Supervisor | 4.x | 进程守护工具，用于保持 worker 与 beat 进程稳定（生产推荐） |

内存最低要求 1GB，推荐 2GB 以上；磁盘空间取决于链接总数及历史巡检记录，按每条链接每年约 50KB 增量估算。

│   └── workflows/                   # 单元测试、代码扫描、镜像构建
├── backend/                         # 核心后端代码（Python Django）
│   ├── apps/                        # 模块化应用集合
│   │   ├── core/                    # 全局配置、基础模型、异常定义
│   │   ├── links/                   # 链接管理核心：模型、序列化器、业务逻辑
│   │   ├── checks/                  # 可用性探测引擎：调度器、执行器、结果记录
│   │   ├── tags/                    # 标签与分类树管理
│   │   ├── stats/                   # 点击统计、访问分析、报表聚合
│   │   └── users/                   # 用户认证、权限、团队协作
│   ├── libs/                        # 公共工具库（HTTP 客户端、缓存封装、加密）
│   ├── settings/                    # 多环境配置（development, staging, production）
│   └── manage.py                    # Django 管理入口
├── frontend/                        # 管理面板 SPA（React + TypeScript）
│   ├── src/
│   │   ├── pages/                   # 链接列表、详情、仪表盘、审批队列
│   │   ├── components/              # 复用 UI 组件（表格、筛选器、图表）
│   │   └── services/                # API 调用封装与 WebSocket 订阅
│   └── package.json

│   ├── tasks/                       # 巡检、通知、导入导出等延迟任务

## 常见问题

**Q：LinkVault 与普通书签管理器或浏览器收藏夹有何本质区别？**

A：书签管理器以个人使用为中心，缺乏协作能力、状态检测和数据洞察。LinkVault 面向团队和项目级资源治理，提供自动化的死链检测、访问统计、审批流程和开放 API。它不只是一个存储容器，更是一个持续运营的管理系统，旨在帮助组织降低外链风险、提升资源发现效率。

**Q：巡检任务是否会影响目标站点的正常访问？会不会被目标服务器封禁？**

A：巡检默认使用低频率（每日一次）和短超时（5 秒）的 HEAD 请求，仅获取响应头状态码，不下载完整页面内容。请求头中携带 `User-Agent: LinkVault-HealthCheck/1.0` 并附带项目主页链接，便于目标站点管理员识别。对于敏感或高价值资源，用户可在配置中单独设置白名单或关闭自动探测。

**Q：能否将 LinkVault 作为静态站点生成器的数据源？**

A：可以。LinkVault 提供只读的 REST API，支持按标签、分类、创建时间等条件筛选并导出 JSON 格式数据。用户可通过脚本在站点构建阶段拉取链接列表，渲染为导航页、友情链接模块或资源汇总表格。项目自身也提供了一个轻量级 CLI 工具 `linkvault-export`，可直接在终端执行导出操作，便于嵌入 Makefile 或 CI 流程。
