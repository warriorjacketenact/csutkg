# Nexus Index

Nexus Index 是一个面向技术研究、数据分析与数字资产管理领域的高密度外链资源汇总系统。项目定位于为开发者、研究员及运维工程师提供结构化、可检索的媒体资源索引服务，解决分散链接难以系统化管理与快速复用的问题。本仓库不存储任何实体文件，仅维护指向外部资源的稳定引用关系，通过元数据标签与分类树实现高效导航。Nexus Index 适用于需要批量引用外部文档、镜像文件、API 端点或日志素材的工作流，尤其注重链接的完整性和原始性输出。

## 功能概览

- **原始链接强制保真**：所有收录 URL 严格按用户原始输入存储与展示，不添加协议头，不修改域名大小写，不追加尾部斜杠，不转换为 markdown 超链接语法。

- **多层级分类标签**：每条链接可附加技术领域、文件类型、地域归属、时间范围等维度标签，支持组合筛选与正则检索。

- **批量导入导出**：支持 CSV、JSON 与纯文本列表格式的批量链接导入，同时提供按分类或标签的过滤导出功能。

- **链接状态巡检**：内置 HTTP 状态码检查工具，可定期检测资源可用性，并标记失效或重定向链接。

- **元数据扩展字段**：每条记录支持自定义键值对扩展，用于记录资源摘要、版本信息、访问凭证提示或校验哈希。

- **只读镜像发布**：支持生成静态站点或 JSON API 形式的只读镜像，便于嵌入其他文档系统或自动化脚本调用。

- **变更审计日志**：记录每次链接增删改的操作人、时间戳与变更原因，满足团队协作与合规追溯需求。

- **模板化引用输出**：提供针对不同下游系统（如 curl 命令、Python requests、wget 脚本）的引用模板生成器。

## 应用场景

1. 技术文档编写与维护：当编写涉及大量外部参考资料（如 RFC 文档、开源项目发布页、镜像站地址）的技术手册时，Nexus Index 作为中央链接仓库可确保引用的一致性和可更新性，避免硬编码导致的链接漂移问题。

2. 数据采集管道配置：在构建网络爬虫或数据集成作业时，需要维护目标 URL 列表及对应的解析规则。Nexus Index 的元数据扩展字段可用于存储解析模板、频率限制参数及备选端点。

3. 数字资产清单管理：对于拥有大量媒体文件、备份归档或日志分发的团队，可通过本索引系统登记存储桶地址、CDN 路径及历史版本链接，配合巡检功能主动发现访问异常。

4. 合规与审计准备：在需要对外披露数据来源或合作方资源清单的场景下，Nexus Index 的审计日志和只读镜像可快速生成可供审查的链接目录，保留完整来源证据链。

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/nexus-index/core.git
cd nexus-index

# 安装依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 初始化本地索引数据库并导入示例链接
python cli.py init --db nexus.db
python cli.py import --source data/sample_urls.csv --table media_links
```

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python >= 3.10 | 是 | 核心运行环境，用于 CLI 工具及巡检脚本 |
| SQLite >= 3.35 | 是 | 默认元数据存储引擎，支持 JSON 扩展 |
| pip / setuptools | 是 | 安装第三方 Python 依赖库 |
| Git >= 2.25 | 是 | 用于克隆仓库及版本控制 |

| jq >= 1.6 | 可选 | 用于命令行 JSON 输出格式化 |
| docker / docker-compose | 可选 | 仅当需要容器化部署静态镜像服务时需要 |
| make >= 4.2 | 可选 | 用于自动化构建文档和测试任务 |

## 项目结构

```

├── cli.py                  # 主命令行入口，整合导入、导出、巡检子命令
├── cli.py                  # 主命令行入口，整合导入、导出、巡检子命令
├── core/
│   ├── db.py               # SQLite 数据库连接池与表结构管理
│   ├── models.py           # 链接记录、标签、审计日志的数据类定义
│   ├── importer.py         # 从 CSV/JSON/纯文本解析链接并写入数据库

│   ├── routes/

│   └── templates/

│   ├── test_db.py          # 数据库 CRUD 单元测试
│   ├── test_importer.py    # 导入模块边界情况测试
│   └── test_checker.py     # 巡检工具模拟请求与超时测试
├── config/
│   ├── default.yaml        # 默认配置（并发数、超时秒数、白名单标签）
│   └── production.yaml     # 生产环境覆盖配置（日志级别、数据库路径）
├── docs/
│   ├── user-guide.md

│   ├── api-reference.md
```

## 贡献指南

1. 阅读设计文档 docs/design.md 理解核心数据模型与扩展约束，并在 GitHub Issues 中搜索是否存在相似提议或已确认的缺陷。

2. 从 `main` 分支创建新的功能分支，命名规范为 `feature/简述变更` 或 `fix/问题编号`，确保分支基于最新主干代码。

3. 实现功能或修复后，补充对应的单元测试（位于 tests/ 目录），并确保所有现有测试通过（执行 `make test`）。

## 常见问题

**问：链接列表中的 URL 是否会被自动补全协议头或标准化？**

答：不会。Nexus Index 严格保留用户输入的原始 URL 字符串，包括协议头、域名大小写、端口号及尾部路径。输出时也不进行任何自动补全或跳转解析，确保引用精确性。

**问：如何自定义链接的元数据扩展字段？**

答：每条记录在数据库中预留了一个 JSON 类型的 `extra` 列。用户可通过 CLI 的 `update` 命令传递 `--extra` 参数，或通过 API 的 PATCH 方法修改该字段。扩展字段不会影响链接的原始输出，但可用于存储下游消费所需的附加信息。

**问：静态镜像页面与 API 数据源是否保持同步？**

答：静态镜像页面生成于 `server/templates/mirror.html`，通过调用内部 API 获取实时数据。若需生成完全静态的 HTML 快照，可运行 `scripts/freeze_mirror.py` 将当前索引渲染为独立文件，适用于无动态后端的发布场景。
