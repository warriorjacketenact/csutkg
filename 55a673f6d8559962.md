# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的轻量级外链资源聚合与管理平台。该项目定位于解决分散化链接管理、资源可发现性差以及链接失效追踪困难等问题，适用于需要系统化维护大量外部技术资源链接的内部知识库构建、开源项目文档站外链整合以及技术写作中的引用管理场景。

LinkVault Core 以 YAML 配置文件驱动，支持链接的标签化分类、自动快照记录与状态检测，并提供静态站点生成能力，帮助用户将零散的 URL 集合转化为结构清晰、可检索、可监控的 Web 资源目录。项目本身不依赖数据库，所有数据存储于文件系统，适用于轻量级部署与快速迭代。

## 功能概览

- 批量链接导入与规范化校验：支持从 CSV 与 Markdown 列表批量导入链接，自动去重并补全缺失协议头，输出标准化 URL 列表。

- 基于标签与分类的分层组织：用户可为每个链接赋予多个标签与分类层级，系统自动生成分类视图与标签云，提升资源检索效率。

- 定时可用性检测与状态标记：内置轻量级 HTTP 健康检查模块，可配置定时任务对已收录链接进行可达性验证，并标记异常状态。

- 静态站点生成与多格式导出：支持将链接库一键渲染为静态 HTML 目录页，并提供 JSON 与 YAML 格式导出，便于集成到现有文档站点。

- 链接变更历史追溯与快照对比：记录每次链接属性变更与内容快照，支持差异对比，便于审计与回溯。

- 权限分级与协作审核机制：提供管理员、编辑者、访客三级权限控制，编辑者修改链接后需提交审核方可生效。

- 全文检索与模糊匹配搜索：基于倒排索引实现标题、描述及标签的全文搜索，支持拼音模糊匹配与拼写容错。

## 应用场景

1. 开源项目文档站外链管理：开源项目维护者可使用 LinkVault Core 统一管理项目 README、Wiki 及官网中的所有外部引用链接，确保文档中的资源链接长期有效且易于更新。

2. 技术团队内部知识库资源聚合：技术团队可将内部常用的开发文档、工具站点、API 参考等链接集中收录并分类，搭建团队专属的技术导航页，减少成员重复搜索时间。

3. 技术写作与博客引用管理：技术博主或文档写作者在创作过程中积累的大量参考链接可通过 LinkVault Core 进行结构化存储，并自动生成引用列表或脚注，提升写作效率。

4. 运维监控与外部依赖链路跟踪：运维团队可录入业务系统依赖的所有外部服务地址与配置仓库链接，利用定时检测功能及时发现不可用资源，辅助故障排查。

5. 教育培训课程资源索引：培训机构或高校教师可将课程涉及的所有在线资料、视频链接、代码仓库按课时整理为独立目录，随课程更新而同步维护。

## 快速开始

以下操作指南适用于 Linux / macOS / WSL 环境，要求已安装 Git 与 Node.js 18.x 或以上版本。

```bash
# 克隆仓库
git clone https://github.com/yourorg/linkvault-core.git
cd linkvault-core

# 安装依赖
npm install

# 执行快速启动脚本（包含初始化配置、导入示例数据并启动开发服务器）
npm run init
npm run dev
```

执行完毕后，访问终端输出的本地地址（默认 http://127.0.0.1:5173）即可查看链接目录界面。首次启动将自动生成 `.env` 文件与示例数据，用户可根据需要编辑 `config/links.yaml` 进行自定义。

## 常见问题

Q: 导入大量链接时出现内存占用过高或超时，如何解决？

A: 对于超过 5000 条链接的批量导入，建议使用 `--batch` 参数分批处理，每批 500 条。同时可调整 Node.js 内存限制，例如执行 `NODE_OPTIONS="--max-old-space-size=2048" npm run import -- --file data.csv --batch 500`。若链接中包含大量重复条目，开启 `--dedupe` 选项可有效减少数据量。

Q: 定时检测功能如何配置？是否支持自定义检测超时与重试策略？

A: 定时检测由 `config/settings.json` 中的 `checker.interval` 字段控制，单位为分钟，默认值为 1440（即每日一次）。检测超时时间（`timeout`）与重试次数（`retries`）同样可在该配置文件中调整。检测任务基于 node-cron 实现，支持标准的 cron 表达式，用户可按需修改 `checker.schedule` 字段覆盖默认间隔。

Q: 生成的静态站点如何部署到 Nginx 或对象存储服务？

A: 执行 `npm run build` 后，所有静态文件将输出至 `dist/` 目录。将该目录下的全部文件上传至 Web 服务器根目录或对象存储桶（如 AWS S3、阿里云 OSS）即可。如需配置自定义域名或启用 HTTPS，请参考 `docs/deployment.md` 中的详细说明，其中包含 Nginx 配置示例与 CDN 加速建议。
