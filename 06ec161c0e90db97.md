# LinkSphere

LinkSphere 是一个面向技术团队与内容创作者的轻量级外链资源聚合与导航系统。该项目定位于将分散的、临时生成的短链引用、文档锚点、以及外部参考链接统一收录、分类、版本化存档，并提供可编程的检索与重定向接口。目标用户包括技术文档维护者、API 网关管理员、以及需要长期管理大量外部引用资源的开发团队。

LinkSphere 解决的核心问题是：在项目生命周期中，文档内嵌的短链、第三方平台引用、以及临时协作链接往往缺乏统一管理，导致链接失效、引用混乱、以及审计困难。本项目通过提供结构化的资源清单、静态站点生成能力、以及基于路径前缀的路由规则，帮助团队建立可维护的外链资产库。

## 功能概览

- **资源清单结构化存储**：以 Markdown 表格和分类小节的形式组织所有外链，支持人工审阅与版本对比。
- **短链路径映射引擎**：基于路径前缀 /x/ 的静态路由表，支持将短链请求重定向至原始目标 URL，同时记录命中次数与最后访问时间。
- **链接健康检查守护进程**：周期性探测所有收录资源的 HTTP 状态码，自动标记异常链接并生成报告。
- **分类标签与全文检索**：为每条资源分配技术领域、文档类型、以及业务归属标签，支持基于标签的过滤和基于标题的模糊搜索。
- **静态站点导出模式**：支持将当前资源库导出为纯静态 HTML 文档，方便部署到 CDN 或内部文档站点。
- **访问日志聚合分析**：记录每次重定向请求的客户端 IP、User-Agent 与 Referer，提供基础的流量来源分析面板。
- **权限分级管理接口**：支持只读访客、资源编辑者、以及管理员三级权限，通过 API Token 进行身份验证。
- **变更审计日志**：对所有资源的增删改操作保留完整审计轨迹，满足内部合规要求。

## 应用场景

- **技术文档外链管理**：技术团队在编写产品文档、API 参考或运维手册时，需要引用大量外部资源（如规范标准、依赖仓库、第三方工具页）。LinkSphere 能够统一收录这些链接，并为每个链接生成稳定的内部短链，避免因外部链接变更导致文档失效。

- **运营活动落地页聚合**：市场运营团队在开展多轮推广活动时，会生成大量带有 UTM 参数的临时投放链接。LinkSphere 可作为内部投放链接的中转层，统一记录每个活动的链接去向和点击数据，方便后续效果复盘。

- **内部知识库引用规范化**：企业知识库中的 FAQ、故障排查案例、以及最佳实践文档经常需要引用 Jira 工单、Confluence 页面、或代码仓库 PR 链接。LinkSphere 通过提供统一的引用前缀和审计日志，确保所有引用均可追溯且符合数据安全政策。

- **开发环境与生产环境链接切换**：在微服务架构中，不同环境的服务发现地址、配置中心端点、以及监控面板 URL 各不相同。LinkSphere 支持根据请求头中的环境标识动态解析目标链接，帮助开发人员在本地、测试、预发布和生产环境之间无缝切换。

## 快速开始

以下步骤帮助您在本地环境中快速启动 LinkSphere 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linksphere/linksphere.git
cd linksphere

# 安装项目依赖
pip install -r requirements.txt

# 初始化本地配置文件
cp .env.example .env
cp config.example.yaml config.yaml

# 导入初始资源清单
python scripts/import_resources.py --source data/resources.csv

# 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8080
```

服务启动后，访问 `http://localhost:8080/admin` 可进入管理后台，默认管理员账号为 `admin`，密码为 `linksphere2024`。

## 项目结构

```
linksphere/
├── app/                                 # 主应用模块
│   ├── api/                             # 路由与接口定义
│   │   ├── redirect.py                  # 核心重定向端点，处理 /x/ 路径解析
│   │   ├── admin.py                     # 管理后台 CRUD 接口
│   │   └── health.py                    # 健康检查状态查询接口
│   ├── core/                            # 核心业务逻辑层
│   │   ├── mapper.py                    # 短链到原始 URL 的映射引擎
│   │   ├── checker.py                   # 链接健康检查器，支持异步并发检测
│   │   └── logger.py                    # 访问日志结构化写入与轮转
│   ├── models/                          # 数据模型与 ORM 定义
│   │   ├── resource.py                  # 资源实体模型，包含 URL、标签、状态字段
│   │   ├── audit.py                     # 审计日志模型，记录操作人、时间、变更内容
│   │   └── session.py                   # 会话与令牌模型，用于权限管理
│   ├── services/                        # 外部服务集成层
│   │   ├── cache.py                     # Redis 缓存操作封装，用于热点链接加速
│   │   ├── exporter.py                  # 静态站点导出器，生成 HTML 资源目录
│   │   └── notifier.py                  # 告警通知服务，异常链接通过邮件或 Webhook 推送
│   └── utils/                           # 通用工具函数
│       ├── validator.py                 # URL 格式校验、域名白名单过滤
```

## 常见问题

**问：如何批量导入大量历史链接资源？**

项目提供了 `scripts/import_resources.py` 导入脚本，支持从 CSV、YAML 和 JSON 三种格式批量导入。CSV 文件需包含 `short_path`、`target_url`、`tags` 和 `description` 四列。导入过程会自动进行 URL 格式校验和短路径唯一性检查，重复条目将跳过并记录警告日志。

**问：健康检查守护进程如何配置检查频率和超时时间？**

健康检查的周期、并发数、以及单次请求超时阈值均在 `config/config.yaml` 文件中的 `checker` 段配置。默认周期为每小时一次，并发数为 10 个连接，超时时间为 5 秒。修改后无需重启主服务，因为健康检查作为独立脚本运行，可通过 systemd timer 或 cron 按新周期重新调度。

**问：生产环境部署时，如何迁移 SQLite 数据到 PostgreSQL？**

LinkSphere 使用 SQLAlchemy 作为 ORM 层，支持多种数据库后端。迁移时，需在 `config/config.yaml` 中将 `database.url` 从 SQLite 连接串更改为 PostgreSQL 连接串（格式为 `postgresql://user:pass@host/dbname`），随后执行 `python scripts/migrate_db.py` 脚本完成表结构同步和数据复制。建议在低峰期操作，并提前做好数据备份。
