# ResourceLink Hub

ResourceLink Hub 是一个面向开发者、技术研究人员与内容创作者的精选技术资源外链汇总系统。该项目不存储任何实际内容，仅提供结构化、可检索的外部资源索引，帮助用户快速定位高质量的在线技术资料、教程、工具与参考文档。

项目定位为技术社区的外链基础设施，适用于个人知识管理、团队技术文档体系建设以及公开技术导航站的资源聚合场景。通过统一的条目标识符体系与分类标签，ResourceLink Hub 将分散的优质外链转化为可复用、可维护的结构化数据集，解决技术从业者在信息过载环境下“找不到、记不住、分享难”的核心痛点。

## 功能概览

**统一条目标识符** 每个资源外链分配唯一编码，支持通过标识符快速检索与引用，便于在文档、Issue 或聊天记录中精准定位。

**多维度分类标签** 资源按技术领域、内容类型、适用水平等维度进行标记，支持按标签过滤与批量导出。

**外链健康检查** 内置自动化脚本，定期探测已收录链接的可达性与状态码变化，辅助维护链接有效性。

**元数据扩展字段** 每条记录包含标题、描述、收录日期、最后验证时间、来源备注等完整元信息，满足知识库建设需求。

**批量导入导出** 支持 CSV 与 JSON 格式的批量数据导入导出，便于与其他知识管理工具（如 Notion、Obsidian）进行数据同步。

**只读 API 接口** 提供基于 RESTful 风格的查询接口，支持按标识符、标签、收录时间范围等条件获取资源数据，方便二次开发。

**命令行交互工具** 内置 CLI 程序，允许用户在终端中完成资源检索、新增、更新与验证操作，提升日常维护效率。

**静态站点生成模式** 支持将资源数据渲染为静态 HTML 导航页面，可直接部署至 GitHub Pages 或任何 Web 服务器，作为团队内部的技术导航站。

## 应用场景

**个人技术知识库构建** 开发者在学习新技术或筹备技术分享时，可使用 ResourceLink Hub 收藏并分类整理散落在各平台的高质量外链，配合标签体系实现个人知识资产的长期积累与快速回溯。

**团队文档体系资源沉淀** 技术团队在编写项目文档、运维手册或入职指南时，往往需要引用大量外部资料。ResourceLink Hub 可作为团队文档体系的外链中间层，统一管理所有外部引用，避免链接散落、失效后难以批量更新。

**技术社区导航站运营** 开源社区或技术媒体可基于 ResourceLink Hub 的静态站点生成模式，快速搭建面向特定技术领域（如云原生、人工智能、前端工程化）的公开导航网站，为社区成员提供便捷的资源入口。

**技术培训与课程辅助** 教育机构或企业培训部门可将课程中涉及的延伸阅读材料统一录入 ResourceLink Hub，学员通过标识符即可获取最新外链，降低教材中 URL 的维护成本。

## 快速开始

以下操作步骤帮助您在本地环境中快速启动 ResourceLink Hub 服务。

```bash
# 克隆代码仓库
git clone https://github.com/resourcelink-hub/resourcelink-hub.git
cd resourcelink-hub

# 安装项目依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate  # Windows 系统请使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化本地数据库并导入示例资源数据
python manage.py initdb
python manage.py import-sample

# 启动本地开发服务器
python manage.py runserver --port 8080
```

启动成功后，访问 http://localhost:8080 可查看本地实例。CLI 工具的使用说明可通过 `python manage.py --help` 获取。

**第二步：选择待解决的 Issue 或提交新建议** 访问项目的 GitHub Issues 页面，查看标记为 help-wanted 或 good-first-issue 的工单。如果您有新的功能建议或缺陷报告，请先搜索现有议题以避免重复，然后按照模板提交新 Issue。

## 常见问题

**问：ResourceLink Hub 是否提供对外公开的在线服务？**

答：项目本身不运营公开的在线服务实例。用户需自行部署本地实例或将其作为数据工具集成至自有系统。项目提供的静态站点生成模式可用于快速搭建个人或团队内部的导航页面，但我们不托管任何用户数据。

**问：收录的外链失效时系统如何处理？**

答：CLI 工具中的 check 子命令会定期对所有已收录链接发起 HEAD 或 GET 请求，并记录状态码。当检测到 4xx 或 5xx 响应时，系统会更新条目的 last_verified 字段并标记 status 为 inactive。用户可通过 export 命令导出失效链接清单，或通过 API 查询异常条目。项目不会自动删除失效链接，仅提供状态标记，由维护者决定是否更新或移除。

**问：是否支持从现有书签或收藏夹批量导入？**

答：当前版本支持 CSV 与 JSON 格式的批量导入。用户可通过 `python manage.py import --format csv --file bookmarks.csv` 命令导入数据，但需要预先将书签文件转换为符合项目数据模型的格式。社区已提供适用于 Chrome 书签导出和 Firefox 书签 JSON 格式的转换脚本，存放于 scripts/converters/ 目录下。未来版本计划增加对 HTML 书签文件的原生解析支持。
