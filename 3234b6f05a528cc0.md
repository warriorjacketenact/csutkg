# LinkVault —— 结构化外链资源管理与导航系统

LinkVault 是一个面向技术团队、内容创作者与科研工作者的外链资源归集与导航平台。项目定位于解决分散在多个浏览器书签、文档碎片、聊天记录中的高价值链接难以统一管理、难以按上下文检索、难以团队共享的问题。LinkVault 不提供爬虫或采集功能，而是以人工精选与目录化标注为核心工作流，帮助用户将零散 URL 转化为可检索、可分类、可版本控制的知识资产。

本项目适配以下场景：技术团队维护常用开发文档与工具链入口、高校实验室整理课题相关数据集与论文链接、独立开发者收集 API 服务与第三方库主页、内容运营团队建立竞品分析与素材来源库。LinkVault 本身不存储链接内容，仅存储元数据与目录结构，所有原始链接严格保留用户提供的访问地址，确保来源可追溯且不引入中间页或重定向。

## 功能概览

- **目录树分级归档**：支持无限层级的资源分类，每个链接可归属至唯一路径，便于按项目、领域或使用频率组织。
- **批量链接导入**：支持从标准 Markdown 列表、纯文本换行列表或 CSV 文件批量导入 URL，自动去重并校验协议格式。
- **标签系统与全文检索**：每条资源可附加多个标签，支持按标签组合过滤；同时提供基于标题、备注、标签与 URL 片段的轻量级全文搜索。
- **版本快照与变更历史**：每次目录调整或元数据修改均生成可追溯的变更记录，支持回滚至任意历史快照，适用于团队协作审计。
- **只读发布模式**：可将当前目录树导出为静态 HTML 或纯 Markdown 文件，供内部文档站点或 README 直接引用，无需依赖数据库。
- **健康检查与失效提醒**：定时对已收录链接进行 HEAD 请求检测，标记响应异常或证书过期的条目，辅助用户清理或更新资源。
- **元数据扩展字段**：每条链接支持记录收录人、收录日期、优先级（1-5）、简要摘要（200 字以内），便于后续筛选与质量评估。
- **RESTful API 接口**：提供 JSON 格式的目录查询、链接搜索、批量更新接口，方便与其他内部系统（如 Wiki、工单系统）集成。

## 应用场景

**技术团队文档站外链治理**：技术团队常在内部 Wiki 中引用大量外部依赖库、规范文档、教程链接。LinkVault 可作为集中式外链管理后端，团队通过 Web 界面维护链接有效性，并利用只读发布模式生成统一的资源目录页，嵌入现有文档站点，避免链接散落与失效。

**个人知识库链接外挂**：个人用户使用 Obsidian、Notion 或 Logseq 等工具管理知识笔记时，可将 LinkVault 作为外部链接仓库，通过 API 或导出的 Markdown 列表嵌入笔记中。如此既保持笔记内容的轻量与可移植性，又复用 LinkVault 的检索与健康检查能力。

**开源项目 README 资源索引生成**：开源项目维护者常在 README 中列出相关工具、教程、生态项目链接。使用 LinkVault 管理这些外链后，可通过 CLI 命令一键生成符合规范格式的 Markdown 列表，直接复制到 README 中，确保格式统一且便于后续更新。

**合规审计与链接溯源**：企业法务或合规部门需要留存对外宣传资料中引用外部数据的原始链接记录。LinkVault 的版本快照与元数据扩展字段可完整记录链接来源、收录时间与审批状态，满足内部审计对引用来源可追溯的要求。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git 与 Node.js（版本 18.x 或以上）。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（使用 npm）
npm install

# 初始化本地配置与数据库文件
npm run init

# 启动开发服务器，默认监听 http://localhost:5173
npm run dev
```

生产环境构建与启动命令请参阅 `docs/deployment.md` 文档。首次启动后，系统会自动创建 `data/` 目录用于存放 SQLite 数据库文件与日志，默认管理员账户需通过 `npm run setup-admin` 交互式创建。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.0.0 及以上 | 运行时环境，推荐使用 LTS 版本 |
| npm | 8.0.0 及以上 | 包管理器，用于安装依赖与执行脚本 |
| SQLite3 | 系统自带或通过 npm 安装 | 嵌入式数据库，无需额外部署服务 |
| Git | 2.25.0 及以上 | 用于克隆仓库与版本管理 |
| 操作系统 | Linux (glibc 2.28+), macOS (11+), Windows 10/11 (WSL2 推荐) | 跨平台支持，Windows 原生也可运行但建议使用 WSL2 以获得更好的文件性能 |
| 内存 | 最低 512 MB，推荐 1 GB | 数据库缓存与构建过程内存占用 |
| 磁盘空间 | 最低 200 MB | 主要用于数据库文件与日志存储，链接数量增长时相应增加 |

## 项目结构

```
linkvault/
├── src/                           # 核心源代码目录
│   ├── main/                      # 主进程与服务器入口
│   │   ├── app.js                # Express 应用初始化与中间件配置
│   │   └── routes/               # API 路由定义（目录、链接、标签、健康检查等）
│   ├── renderer/                  # 前端渲染层（Vue 3 + Vite）
│   │   ├── views/                # 页面级组件（首页、目录树、搜索、设置等）
│   │   ├── components/           # 可复用 UI 组件（链接卡片、标签选择器、面包屑等）
│   │   └── stores/               # Pinia 状态管理（当前目录、筛选条件、用户偏好）
│   ├── database/                  # 数据库层
│   │   ├── migrations/           # SQLite 数据库迁移脚本（按日期命名）
│   │   ├── models/               # 数据模型定义（Link, Category, Tag, Snapshot）
│   │   └── client.js             # 数据库连接池与查询构造器封装
│   ├── services/                  # 业务逻辑服务
│   │   ├── linkService.js        # 链接增删改查、导入导出、健康检查调度
│   │   ├── categoryService.js    # 目录树 CRUD 与路径解析
│   │   └── snapshotService.js    # 版本快照创建、恢复与差异对比
│   └── utils/                     # 工具函数
│       ├── urlValidator.js       # URL 协议、格式、域名黑名单校验
│       ├── markdownExporter.js   # 将目录树与链接列表导出为 Markdown 格式
│       └── logger.js             # 基于 Winston 的日志记录器（按天轮转）
├── data/                          # 运行时数据目录（不纳入版本控制）
│   ├── linkvault.db              # SQLite 主数据库文件
│   ├── logs/                     # 应用日志与健康检查报告
│   └── exports/                  # 导出生成的临时文件
```

## 常见问题

**Q：LinkVault 与普通书签管理器或笔记软件的内置链接功能有何区别？**

A：普通书签管理器以浏览器为中心，缺乏团队协作、版本追溯与健康检查能力；笔记软件内置链接管理通常较为简单，且与笔记内容强耦合，不易独立导出或共享。LinkVault 定位于独立的外链元数据管理系统，将链接作为核心资产进行目录化、标签化与可审计化管理，并提供 API 和导出能力，方便嵌入其他工作流。它不替代书签管理器，而是作为更严肃的团队级链接知识库使用。

**Q：导入大量链接后，系统性能是否会显著下降？**

A：LinkVault 使用 SQLite 嵌入式数据库，单表支持百万级记录且查询性能良好。对于多数用户（链接数量在 5000 条以内），前端页面加载与搜索响应均在 500ms 以内。若链接数量超过 2 万条，建议定期归档历史快照或开启分页加载模式。系统内置的健康检查采用并发控制（默认同时检测 10 个链接），避免对源站造成压力，同时也不会阻塞其他操作。

**Q：如何从 LinkVault 导出数据以迁移到其他平台？**

A：LinkVault 提供了三种导出方式：一是通过 Web 界面选择目录或标签后，点击「导出 Markdown」生成纯文本列表；二是使用 CLI 命令 `npm run export -- --format json` 导出完整 JSON 数据（含元数据）；三是直接备份 `data/linkvault.db` 文件，该文件可在任何 SQLite 兼容工具中读取。未来版本计划支持导出为 HTML 静态站点和 CSV 通用格式。
