# ResourceBridge

ResourceBridge 是一个面向技术团队与个人开发者的外链资源归集与导航系统。本项目不对托管内容做任何修改或二次分发，仅提供结构化索引与访问加速建议，帮助用户在海量分散的技术文档、工具站点、社区论坛与官方镜像之间快速定位有效信息。项目本身不存储任何第三方文件，所有记录均以原始 URL 形式存在，用户通过本系统获得的是经过分类整理和可用性监测的链接集合。

ResourceBridge 适用于需要频繁查阅多源技术资料、维护内部知识库或构建团队开发资源门户的场景。系统内置链接可用性检查、访问计数与失效标记功能，并对外提供只读查询接口，方便与现有监控平台或自动化脚本集成。项目采用纯静态站点生成方案，所有链接数据以 JSON 格式存储，便于版本控制和离线浏览。

## 功能概览

**链接分类标注** 每条外链可按技术领域、文档类型、优先级和标签进行多维度标记，支持自定义标签体系。

**可用性自动检测** 系统每日定时对已收录链接执行 HEAD 请求，记录响应状态码与耗时，自动标记异常或已失效的 URL。

**访问热度统计** 基于日志分析或前端埋点，统计每个链接的点击次数与最近访问时间，辅助识别高频资源。

**按项目批次管理** 链接按导入批次分组展示，当前批次为第 437/1241 批，便于追溯资源来源和更新节奏。

**只读查询接口** 提供 RESTful 风格的查询端点，支持按标签、状态、域名过滤，方便与 CI/CD 或监控告警系统对接。

**静态站点导出** 支持将当前链接集合导出为纯 HTML 目录页，无需动态服务即可在内网或本地浏览器中打开浏览。

**标签与全文检索** 基于前端索引实现轻量级搜索，支持按标题、描述、URL 关键词和标签组合筛选。

**链接变更追踪** 记录每次链接元数据的修改历史，包括标签调整、状态变更和分类移动，便于审计和回溯。

## 应用场景

技术团队构建内部开发者门户时，可使用 ResourceBridge 统一整理常用依赖镜像源、官方文档入口和社区讨论区，避免团队成员在分散的书签中反复查找。通过可用性检测功能，可提前发现因网络策略或服务下线导致的访问异常，减少无效等待时间。

个人开发者可将本系统作为个人技术收藏夹的管理后端，按项目阶段或学习路线对链接进行阶段标记。例如，可将正在研究的多个开源项目文档、API 参考和示例代码库归入同一批次，随学习进度动态调整标签。

社区或开源组织维护外部资源导航页时，ResourceBridge 提供批量导入和导出能力，支持从 CSV 或 JSON 文件批量更新链接列表。结合静态导出功能，可快速生成符合组织风格的导航页面，无需重复编写 HTML 结构。

## 快速开始

以下操作基于 Linux/macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

克隆代码仓库并进入项目目录：

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
```

安装依赖（项目使用 Node.js 和 npm，请确保已安装 Node.js 16.x 或更高版本）：

```bash
npm install
```

执行本地构建与预览（默认监听 8080 端口）：

```bash
npm run build
npm run serve
```

构建完成后，打开浏览器访问 http://localhost:8080 即可查看当前批次的链接列表。如需更新链接数据，请编辑 data/batch_437.json 文件，然后重新执行构建命令。

## 常见问题

**Q：链接可用性检测是否会频繁请求目标站点，造成对方服务器压力？**

A：检测任务默认在每日凌晨 2:00 至 4:00 之间执行，且每个目标 URL 仅发送一次 HEAD 请求，不下载响应体。请求间隔通过随机延迟（500ms 至 2000ms）分散，避免在短时间内对同一域名发起大量请求。用户可在配置文件中调整检测频率和并发数，默认并发为 5 个请求同时进行。

**Q：如何批量导入自有链接列表？**

A：项目支持从 JSON 和 CSV 两种格式导入。JSON 格式需遵循 data/batch_437.json 的结构，包含 links 数组，每个元素需有 url、title、tags 字段。CSV 格式需包含 URL、标题、标签（以竖线分隔）三列。导入命令为 npm run import -- --file=your_file.json --format=json，具体参数说明请参考 docs/admin-guide.md 中的导入章节。

**Q：静态导出模式是否支持搜索功能？**

A：静态导出的 HTML 页面包含前端搜索与过滤逻辑，所有链接数据在构建时嵌入页面 JavaScript 中。因此即使在不提供动态服务的环境下，用户仍然可以使用标签筛选和关键词搜索功能。需要注意的是，静态导出页面不会实时反映数据更新，如需最新数据请重新执行构建命令。
