# LinkPilot

LinkPilot 是一个面向技术内容创作者、资源聚合者和开发者的轻量级外链资源导航与数据管理工具。该项目定位于帮助用户系统化地收集、分类、检索和展示大量外部 URL 资源，尤其适用于需要维护大型影视、文档、教程或资讯链接库的场景。LinkPilot 不提供内容存储或代理服务，仅专注于链接元数据的结构化组织与展示，解决用户在面对海量散乱链接时难以管理、难以追溯、难以分享的核心痛点。

## 功能概览

**批量链接导入与校验**：支持从 CSV、JSON 或纯文本列表中批量导入 URL，自动执行协议一致性检查、重复去重和可访问性初步探测。

**多维度分类与标签系统**：允许用户为每条链接自定义分类、标签和优先级，支持多级标签树结构，便于按主题、来源、格式或质量等级进行精细化管理。

**全文检索与高级过滤**：基于链接标题、描述、标签、备注等元数据构建倒排索引，提供毫秒级检索响应，支持多条件组合过滤与排序。

**数据快照与版本回溯**：每次导入或手动编辑操作均生成时间戳快照，用户可查看历史变更记录并一键回退至任意先前状态，防止误操作导致数据丢失。

**访问状态监控面板**：内置轻量级定时任务，周期性探测链接可达性并记录响应码与延迟，生成可视化健康度报表，帮助用户及时发现失效资源。

**开放数据导出接口**：支持将当前链接库导出为 JSON、YAML 或 Markdown 表格格式，便于嵌入静态站点生成器、文档系统或作为数据源供其他应用消费。

## 应用场景

**个人影视资源收藏整理**：用户可将分散在各影评网站、下载页面的影片详情链接统一收录至 LinkPilot，按类型、地区、年份等标签归类，配合检索功能快速定位特定影片条目。

**技术文档外部引用管理**：开源项目维护者可使用 LinkPilot 管理 README、Wiki 中引用的所有外部参考链接，定期监控链接有效性，避免文档中出现死链影响用户体验。

**教育课程资源包构建**：教师或培训机构可将每节课涉及的扩展阅读、视频案例、工具官网等链接导入系统，生成结构化的资源清单，方便学员按需访问。

**内容聚合站点后台数据源**：静态博客或门户网站运营者可将 LinkPilot 作为外部链接数据中台，通过导出功能生成链接列表页面，无需重复手动编写 HTML 或 Markdown。

## 快速开始

以下命令演示如何获取源码、安装依赖并启动开发服务器。

```bash
git clone https://github.com/linkpilot/linkpilot.git
cd linkpilot
pip install -r requirements.txt
python app.py --init-db --import-sample
```

执行上述命令后，服务默认监听 127.0.0.1:8080，访问该地址即可进入 Web 管理界面。首次运行时 `--import-sample` 参数会导入一组预置示例链接用于演示。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，用于执行应用逻辑与调度任务 |
| SQLite | 3.35 及以上 | 默认嵌入式数据库，用于存储链接元数据及快照 |
| Redis | 6.0 及以上 | 可选，用于提升检索性能和任务队列管理 |
| Node.js | 16 及以上 | 仅用于前端资产构建，生产环境可复用预编译文件 |
| git | 2.25 及以上 | 用于克隆仓库及后续拉取更新 |
| curl | 7.68 及以上 | 用于链接可达性探测任务的底层网络请求 |

## 项目结构

```
linkpilot/
├── app/                             # 核心应用模块
│   ├── __init__.py                  # 应用工厂函数与配置加载
│   ├── models/                      # 数据模型层（SQLAlchemy ORM 定义）
│   │   ├── link.py                  # Link 实体：url、title、description、tags
│   │   ├── snapshot.py              # Snapshot 实体：版本快照元数据与变更记录
│   │   └── category.py              # Category 实体：分类树节点与父子关系
│   ├── services/                    # 业务逻辑服务层
│   │   ├── importer.py              # 批量导入服务：支持 CSV/JSON 解析与去重
│   │   ├── search.py                # 检索引擎：基于 Whoosh 的全文检索实现
│   │   └── monitor.py               # 监控调度器：周期性探测与状态更新
│   ├── web/                         # Web 路由与视图
│   │   ├── routes/                  # Blueprint 路由划分（api、admin、export）
│   │   └── static/                  # 编译后的前端静态资源（CSS、JS）
│   └── utils/                       # 通用工具函数
│       ├── validator.py             # URL 协议校验与规范化
│       └── task_queue.py            # Redis 或内存队列的轻量封装
├── config/                          # 环境配置
│   ├── development.py               # 开发环境参数（DEBUG、日志级别）
│   ├── production.py                # 生产环境参数（数据库连接池、缓存TTL）
│   └── testing.py                   # 单元测试专用配置
├── data/                            # 数据存储目录（默认不提交至版本库）
│   ├── linkpilot.db                 # SQLite 数据库文件
│   ├── snapshots/                   # 历史快照 JSON 文件存档
│   └── index/                       # Whoosh 全文索引文件
├── tests/                           # 单元测试与集成测试
│   ├── test_models.py               # 数据模型层测试
│   ├── test_services.py             # 业务逻辑层测试
│   └── test_api.py                  # API 接口功能测试
├── docs/                            # 完整文档（用户手册、运维指南、开发者文档）
│   ├── user-guide/                  # 面向最终用户的操作说明
│   ├── operations/                  # 面向运维人员的部署与调优指南
│   └── developer/                   # 面向贡献者的模块设计与扩展说明
├── scripts/                         # 辅助运维脚本
│   ├── backup.sh                    # 数据库与快照自动备份脚本
│   └── import_example.py            # 示例数据导入脚本
├── requirements.txt                 # Python 依赖清单（固定版本）

```

## 贡献指南

1. 阅读项目文档中的开发者手册（docs/developer/architecture.md），了解整体设计模式与模块边界，确认您的改动方向与项目定位一致。

## 常见问题

**问：LinkPilot 是否存储链接指向的原始内容或提供代理访问？**

答：不存储任何原始内容，也不提供代理或镜像服务。LinkPilot 仅维护链接本身的 URL 字符串及其用户自定义元数据（标题、描述、标签等）。所有链接的实际访问行为均发生在用户浏览器或探测任务中，项目本身不承担内容分发职责。

**问：批量导入时遇到重复链接如何处理？**

答：导入服务默认基于 URL 字符串做精确去重。若检测到新导入的链接与现有记录完全一致，系统会跳过该条并记录警告日志，同时返回重复列表供用户手动处理。用户也可在配置中启用“覆盖模式”以新数据替换旧记录的元数据。

**问：监控面板显示链接不可达，但浏览器可以正常打开，是什么原因？**

答：监控任务使用 curl 发起 HTTP 请求，其 User-Agent、超时时间及重定向策略可能与浏览器不同。您可在配置文件中调整探测参数（如 timeout、follow_redirects、allowed_status_codes），同时部分站点会拦截非浏览器请求，此时可考虑设置自定义 User-Agent 或关闭严格状态码校验。
