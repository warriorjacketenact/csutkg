# GBDLS Resource Indexer

GBDLS Resource Indexer is a specialized technical resource aggregation and indexing system designed to catalog, organize, and provide rapid access to a vast collection of distributed reference materials. The project serves as a structured gateway to over 250 curated resources, enabling developers, researchers, and system administrators to efficiently navigate through technical documentation, code repositories, and knowledge bases.

The system addresses the fundamental challenge of managing large-scale resource collections by providing a unified indexing layer that maintains referential integrity while supporting flexible query patterns. Unlike conventional bookmark managers or simple link lists, GBDLS Resource Indexer implements a hierarchical classification system that preserves contextual relationships between resources, making it suitable for enterprise knowledge management, academic research catalogs, and open-source project documentation hubs.

## 功能概览

**Multi-Format Resource Indexing** - Supports automatic extraction and normalization of resource metadata from heterogeneous source formats, including HTML documents, PDF technical papers, and structured data exports.

**Hierarchical Category Management** - Implements a tree-based classification system with support for infinite nesting, enabling fine-grained organization of resources across multiple technical domains.

**Full-Text Search Capability** - Provides real-time search across resource titles, descriptions, and associated tags with relevance scoring and partial-match support.

**Bulk Import Pipeline** - Processes large-scale resource lists through a batched ingestion system that validates URL formats, checks for duplicates, and maintains a persistent audit trail of import operations.

**Export and Serialization** - Generates resource catalogs in multiple output formats including Markdown, JSON, and CSV for integration with external documentation systems.

**Integrity Verification** - Periodically validates resource availability through HTTP status checking and reports broken links with detailed error diagnostics.

**Tag-Based Filtering** - Enables dynamic resource filtering through a faceted tag system that supports boolean operations for precise resource discovery.

**Versioned Change Tracking** - Maintains a complete history of all resource additions, modifications, and removals with timestamps and operator identification.

## 应用场景

Technical Documentation Management - Development teams can utilize the indexer to maintain internal documentation catalogs, ensuring that API references, design documents, and operational runbooks remain discoverable across large organizations. The system tracks resource provenance and update cycles.

Academic Research Catalogs - Research institutions can deploy the platform to organize preprints, conference proceedings, and experimental datasets. The hierarchical classification accommodates complex taxonomies required for interdisciplinary projects.

Open-Source Project Resource Hub - Community-driven projects can leverage the indexer to curate external dependency lists, contribution guides, and ecosystem references. The bulk import capability simplifies the migration from ad-hoc resource lists.

Enterprise Knowledge Base Integration - Organizations can embed the indexer within their existing knowledge management infrastructure, providing a standardized interface for accessing internal and external technical resources scattered across multiple storage systems.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/gbdls/resource-indexer.git
cd resource-indexer

# Install dependencies
pip install -r requirements.txt
python -m spacy download en_core_web_sm

# Run the initialization script with the default resource batch
python scripts/import_batch.py --batch 1227 --source data/resources_1227.json

# Start the indexing service
python -m gbdls_indexer serve --port 8080 --workers 4
```

## 常见问题

Q: 系统如何处理资源URL变更或失效情况？

A: 系统提供定期验证机制，通过后台任务检查所有已索引资源的HTTP状态码。对于返回4xx或5xx状态的资源，系统会在管理界面标记为警告状态，并在连续三次验证失败后将其移至隔离区。管理员可通过API或CLI手动更新资源URL，系统会保留历史URI记录以维持引用追踪。

Q: 是否支持分布式部署以处理大规模资源集？

A: 当前版本支持水平扩展部署架构。索引服务采用无状态设计，可部署多个实例并通过反向代理实现负载均衡。存储层可切换至PostgreSQL或Elasticsearch集群以支持多节点部署。缓存层使用Redis Sentinel或Cluster模式提供高可用性。具体配置参见docs/architecture.md中的分布式部署章节。

Q: 如何将现有资源集合迁移至本系统？

A: 系统提供多种导入路径。最简方式为通过CLI的import命令直接导入JSONL或CSV文件，每行包含URL、标题和可选标签。亦可使用内置的爬虫工具从现有HTML页面或RSS源中抽取资源链接。对于大型迁移场景，建议使用分批导入策略，每批控制在1000条以内，配合--dry-run选项预先验证数据格式。详细迁移指南参见docs/getting-started.md。
