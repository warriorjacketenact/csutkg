# StreamLink 资源导航系统

StreamLink 是一个面向开发者和技术研究人员的轻量级资源导航与聚合平台，专注于采集、分类和检索互联网中的高质量技术文档、开源项目镜像、在线工具与数据流接口。本项目不提供内容存储，而是通过稳定的外链索引机制，帮助用户快速定位到特定领域的技术资源，解决信息分散、检索效率低下的问题。

项目定位于中大型技术团队内部的知识管理辅助系统，同时也适用于个人研究者构建专属的资源收藏库。StreamLink 的核心机制围绕“流”的概念展开，每个资源条目被视为一个独立的数据流，通过唯一标识符进行组织与版本追踪，从而在大量外链中维持可维护性和可扩展性。

## 功能概览

**资源流式采集**：支持批量导入和手动添加两种方式，将外部 URL 转换为内部可追踪的资源流对象，每条流自动记录添加时间、来源标签和访问状态。

**多维度分类标签**：每个资源流可绑定多个分类标签，系统预置技术文档、视频教程、代码仓库、在线演示、数据 API 等基础类别，允许用户自定义扩展标签体系。

**状态监控与健康检查**：内置轻量级链接可用性检测模块，定时对已收录的 URL 发起 HEAD 请求，标记异常链接并生成报告，帮助维护资源库的时效性。

**快速检索与过滤**：提供基于资源标识符、标签、添加时间范围的组合筛选功能，支持模糊匹配，能够在数百条资源中快速定位目标条目。

**导入导出接口**：支持 JSON 和 CSV 格式的资源列表导入导出，便于与其他工具或团队成员共享资源集合，也支持从浏览器书签文件转换。

**访问统计与热度排序**：记录每个资源流的点击次数和最后访问时间，提供按热度、新增时间、字母顺序等多种排序方式，辅助用户发现高频使用的资源。

## 应用场景

**技术团队内部文档聚合**：开发团队可将分散在多个平台的技术规范、API 文档、设计稿链接统一收录到 StreamLink 中，通过共享资源池减少成员之间的重复询问，提升信息获取效率。适用于 10 人以上的中等规模研发团队。

**开源项目镜像站索引**：对于需要频繁访问国外开源项目镜像或特定软件包下载站的用户，可利用 StreamLink 集中管理数十个镜像站地址，配合健康检查功能快速剔除失效镜像，保障开发环境的稳定性。

**在线学习资源整理**：技术人员在自学新技术时，常收集大量视频课程、博客文章和实战项目链接。StreamLink 的分类标签和检索功能可以帮助学习者按主题（如 Kubernetes、Python、前端框架）组织这些资源，形成个人知识库。

**数据接口测试工具辅助**：后端开发或测试人员可将各类公开 API 的测试地址、Mock 服务地址、WebSocket 调试端点统一纳入资源流管理，配合访问统计记录调试历史，便于回归测试时快速复用。

## 快速开始

以下步骤指导您在本地环境中快速启动 StreamLink 服务。

```bash
# 克隆代码仓库
git clone https://github.com/streamlink/streamlink-nav.git
cd streamlink-nav

# 安装依赖（使用 npm）
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

启动成功后，访问控制台输出中显示的本地地址（通常为 http://localhost:3000），即可进入 StreamLink 主界面。首次启动会自动创建示例资源流数据，方便您体验各项功能。

## 安装要求

StreamLink 依赖以下运行环境与第三方库，请确保在安装前满足所有必需条件。

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或更高 | 是 | 项目运行时环境，建议使用 LTS 版本 |
| npm 9.x 或更高 | 是 | 包管理器，用于安装项目依赖 |
| SQLite 3 | 是 | 内嵌数据库，用于存储资源流元数据和状态 |
| 现代浏览器（Chrome/Firefox/Edge 最新版） | 是 | 用于访问管理界面，支持 ES6+ 语法 |
| 网络连接（外网访问） | 是 | 用于健康检查模块验证外部链接可达性 |
| git 2.x | 否 | 仅从源码构建时需要，用于克隆仓库 |
| Docker 20.x | 否 | 可选，用于容器化部署 |

## 项目结构

StreamLink 采用模块化的 MVC 架构，核心代码按功能域组织在以下目录中。

```
streamlink-nav/
├── src/
│   ├── core/                # 核心引擎模块
│   │   ├── collector.js     # 资源流采集与解析逻辑
│   │   ├── health.js        # 链接健康检查调度器
│   │   └── registry.js      # 资源流注册与元数据管理
│   ├── routes/              # HTTP 路由层
│   │   ├── api.js           # RESTful API 端点定义
│   │   ├── web.js           # 管理界面路由
│   │   └── export.js        # 导入导出接口
│   ├── models/              # 数据模型层
│   │   ├── stream.js        # 资源流实体模型
│   │   ├── tag.js           # 标签模型
│   │   └── history.js       # 访问历史记录模型
│   ├── services/            # 业务服务层
│   │   ├── search.js        # 检索与过滤服务
│   │   ├── stats.js         # 统计聚合服务
│   │   └── sync.js          # 外部同步服务
│   ├── db/                  # 数据库层
│   │   ├── init.sql         # 数据库初始化脚本
│   │   ├── migration.js     # 版本迁移工具
│   │   └── sqlite.js        # SQLite 适配器
│   └── utils/               # 通用工具函数
│       ├── validator.js     # URL 格式验证
│       ├── logger.js        # 日志输出工具
│       └── config.js        # 配置加载器
├── public/                  # 前端静态资源
│   ├── index.html           # 主页面模板
│   ├── css/                 # 样式文件
│   └── js/                  # 前端交互脚本
├── tests/                   # 单元测试与集成测试
│   ├── unit/                # 单元测试用例
│   └── integration/         # 集成测试用例
├── docs/                    # 项目文档
│   ├── deployment.md        # 部署指南
│   ├── configuration.md     # 配置说明
│   ├── usage.md             # 使用手册
│   └── api.md               # API 参考
├── scripts/                 # 运维辅助脚本
│   ├── backup.sh            # 数据库备份脚本

├── .env.example             # 环境变量示例
├── package.json             # npm 项目清单
├── README.md                # 项目说明文档
└── LICENSE                  # MIT 许可证
```

## 贡献指南

StreamLink 欢迎社区开发者提交贡献，包括但不限于代码优化、文档完善、资源列表扩充和问题反馈。请遵循以下步骤参与项目开发。

## 常见问题

问：StreamLink 启动后访问页面空白，控制台提示“数据库连接失败”，如何解决？

答：该问题通常由 SQLite 数据库文件路径权限不足引起。请检查项目根目录下 data 文件夹是否存在且具有读写权限。若不存在，手动创建该目录并重新启动服务。也可通过修改 .env 文件中的 DB_PATH 变量指定其他可写位置。

问：健康检查模块报告大量链接超时，但浏览器中手动访问这些 URL 是正常的，为什么？

答：健康检查模块默认使用 HEAD 请求且超时时间为 3 秒，部分服务器可能拒绝 HEAD 请求或响应较慢。建议在配置文件中将 CHECK_TIMEOUT 调整为 5000 毫秒，并将 CHECK_METHOD 切换为 GET 以获取更准确的可用性状态。同时检查网络环境是否对出站请求有限制。

问：如何将现有的浏览器书签批量导入 StreamLink？

答：从浏览器导出书签为 HTML 文件，然后使用 StreamLink 提供的导入接口（位于“设置 - 导入导出”页面）上传该文件。系统会自动解析书签中的 URL 和标题，生成对应的资源流条目。若书签数量超过 100 条，建议分批导入以避免请求超时。
