# LinkVault

LinkVault 是一个面向开发者、技术研究人员与内容创作者的轻量级技术资源外链管理与导航系统。该项目定位于将散落在互联网各处的优质技术文章、工具站、文档库与社区讨论进行集中收录、分类索引与快速检索，解决个人或团队在技术调研、方案选型与日常开发中“找不到、记不住、分享散”的痛点。

LinkVault 不提供内容存储服务，仅作为结构化外链索引层。用户可通过标签体系、全文检索与批量导入导出功能，在本地或私有化部署环境中维护自己的技术资源库。项目适用于日均处理 1000 条以内链接规模的个人知识管理场景，亦支持多用户只读共享模式。

## 功能概览

- **批量外链导入**：支持从 CSV、JSON 及纯文本列表中批量导入 URL，自动解析标题与元描述，支持去重校验。

- **多级标签分类**：每个链接可绑定多个自定义标签，支持层级标签结构（如 backend/database/mysql），便于按技术领域或使用场景筛选。

- **全文检索与过滤**：基于 SQLite FTS5 或 Elasticsearch 后端，提供标题、描述、标签及 URL 路径的联合检索，支持布尔表达式与模糊匹配。

- **定期可用性探测**：后台任务周期性检查已收录链接的 HTTP 状态码与响应时间，标记失效或重定向资源，生成健康度报表。

- **导入导出与备份**：支持将整个链接库导出为标准 CSV 或 Markdown 列表格式，亦可从备份文件恢复，便于跨设备迁移或团队共享。

- **只读分享模式**：生成只读访问令牌，允许外部用户通过 API 或 Web 界面查询链接库，无需登录系统。

- **访问统计与热力排行**：记录每个链接的点击次数、最后访问时间与来源 IP 聚合，生成近 7 日、30 日热门资源 Top 50。

- **自定义元数据字段**：允许为链接附加技术栈、难度等级、阅读时长、关联项目等自定义键值对，满足专业领域分类需求。

## 应用场景

**技术团队内部知识库沉淀**  
开发团队可将日常遇到的疑难问题解决方案、性能调优案例、官方文档镜像等链接统一收录至 LinkVault，按项目或业务域打标，新成员入职时可快速浏览团队积累的外部参考脉络。

**开源项目周边资源导航**  
开源项目维护者可使用 LinkVault 整理与项目相关的插件生态、社区驱动示例、第三方集成教程及视频讲解，作为项目官网的“社区资源”子页面，降低新用户的上手门槛。

**技术调研与竞品分析**  
架构师或技术负责人进行中间件选型、云服务商对比时，可将各家官方文档、性能测试报告、用户评价帖及安全公告统一收集，利用标签与检索快速对照不同方案的优劣势。

**个人开发者学习路径管理**  
自学者可将不同技术方向（前端、后端、运维、算法）的优质博文、视频课程、在线实验环境按学习阶段分类，配合访问统计功能发现自己的高频参考站点，持续优化学习资料库。

## 快速开始

以下步骤演示如何在本地环境中从源码启动 LinkVault 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装 Python 依赖（推荐使用虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化 SQLite 数据库并创建默认标签
python manage.py migrate
python manage.py init_tags

# 导入示例数据（可选）
python manage.py import_samples

# 启动开发服务器
python manage.py runserver --host 127.0.0.1 --port 8080
```

访问 http://127.0.0.1:8080 即可进入 Web 管理界面，默认管理员账号 admin / admin123，首次登录后请立即修改密码。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9+ | 是 | 核心运行环境，建议使用 3.10 或 3.11 LTS 版本 |
| SQLite 3.35+ | 是 | 内置数据库，支持 FTS5 全文检索扩展 |
| Redis 6.2+ | 否 | 用于缓存热门查询结果与分布式任务锁，生产环境推荐 |
| Elasticsearch 7.17+ | 否 | 替换 SQLite FTS5 作为高级检索引擎，支持更复杂的分析 |
| Node.js 16+ | 否 | 仅在构建前端静态资源时需要，生产环境无需安装 |
| Docker 20.10+ | 否 | 使用容器化部署方式时必需，提供一键启动脚本 |
| Nginx 1.20+ | 否 | 生产环境反向代理与静态资源缓存加速，提高并发能力 |
| supervisor 4.2+ | 否 | 用于进程守护与自动重启，确保后台任务持续运行 |
| git 2.25+ | 是 | 源码拉取与版本更新必需 |

## 贡献指南

LinkVault 遵循开源社区协作规范，欢迎各类形式的贡献，包括但不限于新增功能、修复缺陷、完善文档或提交链接资源。请按以下步骤操作：

1. 在 GitHub Issues 中查找或新建一个与您改动相关的问题，简要描述您要解决的问题或新增的功能，等待维护者确认需求合理性，避免重复劳动或方向偏差。

2. Fork 项目仓库至个人账户，在本地基于 main 分支创建新的功能分支，分支命名建议采用 feat/xxx 或 fix/xxx 格式，确保分支干净且仅包含一次逻辑完整的改动。

3. 完成代码或文档改动后，请确保通过所有现有单元测试，并为新增代码补充对应的测试用例（覆盖率不低于 80%）。同时运行 make lint 检查代码风格，确保符合项目约定的 Go 或 Vue 规范。

4. 提交 Pull Request 时，请填写 PR 模板中的检查清单，关联对应的 Issue 编号，并详细描述改动的实现思路、影响范围以及测试方式。若涉及 UI 变更，请附带截图或录屏。

5. 等待代码审查。维护者会在 3 个工作日内给出反馈，可能需要您根据评审意见进行修改。合并后您的提交将进入 main 分支，并在下一个版本发布时体现在 CHANGELOG 中。

## 常见问题

**Q: 收录的链接如果失效了怎么办？**  
A: LinkVault 后台探测任务默认每 24 小时检查一次所有链接的状态码。失效链接会在 Web 界面标记为“不可达”，并在统计报表中单独列出。您可以在管理面板中手动触发重新探测，或直接编辑链接将其状态改为“已迁移”并填写新的 URL。系统也支持配置 Webhook，在链接连续三次探测失败时发送邮件或钉钉通知。

**Q: 搜索中文时结果不准确或完全没有结果，如何优化？**  
A: 如果使用 SQLite 作为后端，请确认您的 SQLite 版本已编译 FTS5 扩展并支持中文分词（需加载扩展模块）。我们推荐在生产环境使用 Elasticsearch，并在配置文件中设置 ik 或 smartcn 分词器。同时，请检查您的搜索语句是否使用了双引号进行精确短语匹配，或使用 + / - 符号进行必需/排除条件过滤。更详细的调优参数请参考 docs/operations/search-tuning.md。

**Q: 可以将 LinkVault 与我的个人博客或 Wiki 系统集成吗？**  
A: 可以。LinkVault 提供完整的 RESTful API，支持跨域请求（CORS）和只读令牌鉴权。您可以在博客文章中通过 JavaScript 调用查询接口，动态显示与当前文章标签相关的推荐链接。亦可通过 iframe 嵌入只读视图页面，展示特定标签下的链接列表。API 限流策略默认每分钟 60 次请求，如需更高配额可在配置文件中调整。
