# LinkVault 技术资源索引系统

LinkVault 是一个面向开发者、技术研究人员与内容策展人的高性能外链资源聚合与导航平台。该项目旨在解决技术资料分散、优质外链难以系统化管理、团队协作场景下资源发现效率低下的问题，通过结构化的数据组织与轻量级检索机制，将分散于网络各处的视频教程、技术文档、工具站点与案例库整合为统一可查询的知识图谱。LinkVault 适用于需要长期维护技术知识库的研发团队、教育培训机构以及个人技术博主，其核心设计理念为“链接即资产”，通过标准化元数据标注与分类体系，使每条外链均可被追溯、评分与复用。

## 功能概览

**批量链接导入与自动规范化** 支持从文本文件、CSV 或直接粘贴的原始 URL 列表中批量导入资源，系统自动识别协议头、去除冗余查询参数并校验可访问性，降低人工整理成本。

**多维度标签分类与全文检索** 每条链接可关联多个自定义标签（如“机器学习”、“前端工程化”、“系统设计”），并基于标题、描述与标签内容构建倒排索引，支持模糊搜索与精确过滤。

**链接健康度监控与失效告警** 后台定时任务对已收录的链接进行 HTTP 状态码检查，当检测到 4xx 或 5xx 响应时，通过邮件或 Webhook 通知管理员，保障资源库的持续可用性。

**团队协作与审核工作流** 支持多用户角色划分（管理员、编辑者、访客），编辑者提交的新链接需经过管理员审核方可公开可见，审核日志完整记录操作人、时间与备注。

**个性化收藏集与分享** 用户可将常用链接分组收藏为自定义合集，并生成只读分享链接供团队外部成员访问，无需注册账号即可浏览指定资源集合。

**开放 API 与数据导出** 提供 RESTful API 接口，支持第三方应用以 JSON 格式拉取链接元数据，同时支持将全量数据导出为 CSV 或 Markdown 表格，便于离线备份或迁移。

## 应用场景

研发团队内部知识库建设：技术团队在日常开发中积累大量参考文档、API 手册与调试工具地址，LinkVault 可作为统一入口，通过标签体系按项目或技术栈分类，新成员入职时可快速获取所有必备资源。

在线教育课程辅助材料管理：培训机构或高校教师可将每门课程涉及的延伸阅读视频、在线实验环境与代码仓库地址录入系统，学生通过一个聚合页即可访问全部学习素材，避免链接散落在邮件或聊天记录中。

技术社区内容策展与 Newsletter 生成：独立博主或社区运营者可使用 LinkVault 整理每周优质技术文章与视频，结合健康度监控功能自动剔除失效链接，一键生成每周推荐列表的 Markdown 源文件。

多项目环境下的环境配置参考：运维或架构师可将不同项目依赖的中间件面板、监控仪表盘与日志查询工具地址集中管理，按环境（开发、测试、生产）打标，快速切换上下文。

## 快速开始

以下命令将克隆项目仓库、安装依赖并启动开发服务器，默认监听 3000 端口。

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
npm install
npm run dev
```

生产环境构建与启动请参考以下命令：

```bash
npm run build
npm start
```

若使用 Docker 部署，可通过项目根目录下的 Dockerfile 构建镜像并运行容器：

```bash
docker build -t linkvault:latest .
docker run -p 3000:3000 -d linkvault:latest
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 nvm 管理版本 |
| npm | 9.x 或以上 | 包管理器，用于安装依赖与执行脚本 |
| SQLite | 3.35+ 或 PostgreSQL 14+ | 主数据存储，开发环境默认使用 SQLite，生产环境建议切换至 PostgreSQL |

| Nginx | 1.22+ | 可选的反向代理与静态资源服务组件，用于生产环境负载均衡与 HTTPS 终结 |
| Git | 2.30+ | 版本控制工具，用于克隆仓库与获取更新 |

## 项目结构

```
linkvault-core/
├── src/
│   ├── controllers/               # 请求处理控制器，负责解析输入、调用服务层并返回响应
│   │   ├── linkController.js      # 链接的增删改查与导入导出接口
│   │   ├── userController.js      # 用户注册、登录、权限与个人信息管理
│   │   └── healthController.js    # 健康检查任务调度与状态查询
│   ├── services/                  # 业务逻辑层，封装数据操作与外部调用
│   │   ├── linkService.js         # 链接元数据处理、标签关联与去重逻辑
│   │   ├── monitorService.js      # 定时轮询链接状态、失效通知与重试策略
│   │   └── searchService.js       # 基于 lunr.js 的倒排索引构建与查询解析
│   ├── models/                    # 数据模型定义，使用 Sequelize ORM 映射数据库表
│   │   ├── Link.js                # 链接表字段定义（URL、标题、描述、状态码等）
│   │   ├── Tag.js                 # 标签表与链接-标签多对多关联表
│   │   └── User.js                # 用户表与角色枚举定义
│   ├── middleware/                # Express 中间件栈
│   │   ├── auth.js                # JWT 令牌验证与角色权限守卫
│   │   ├── rateLimiter.js         # 基于 Redis 的请求限流与突发流量控制
│   │   └── logger.js              # 结构化日志记录（输出到文件与 stdout）
│   ├── routes/                    # 路由定义，将 URL 路径映射至控制器方法
│   │   ├── api.v1.js              # REST API 路由聚合，版本 v1 所有端点
│   │   └── webhook.js             # 外部系统回调接收路由（用于监控告警）
│   ├── utils/                     # 通用工具函数
│   │   ├── validator.js           # URL 格式校验、协议规范与安全过滤
│   │   ├── crawler.js             # 轻量级页面抓取工具，用于自动提取标题与描述
│   │   └── exporter.js            # 数据导出为 CSV / JSON / Markdown 的转换器
│   ├── config/                    # 配置管理，支持环境变量覆盖与多环境切换
│   │   ├── index.js               # 配置加载入口，合并默认值与 process.env
│   │   └── database.js            # 数据库连接池配置与迁移文件路径
│   └── app.js                     # Express 应用实例化、中间件注册与启动入口
```

## 常见问题

Q: 系统支持批量导入的 URL 数量上限是多少？单次导入最大文件大小有何限制？

A: 单次通过 CSV 或文本文件导入的链接数量上限为 5000 条，文件大小不超过 2MB。若需导入超过该数量的数据，建议拆分为多个批次或使用 API 接口分页提交。系统后台对单次批量任务采用队列化处理，避免阻塞主线程，任务完成后会通过站内消息通知发起者。

Q: 健康度监控的检测频率如何配置？是否支持自定义告警接收渠道？

A: 默认监控间隔为每 24 小时执行一次全量检测，可在系统配置页面（管理员权限）调整至 12 小时或 6 小时。告警渠道目前支持电子邮件、企业微信 Webhook 和 Slack 三种方式，配置时需提供对应的接收地址或令牌，监控结果会附带失效链接的详细信息及最后一次成功响应的状态码。

Q: 部署后如何迁移已有数据至 PostgreSQL？是否支持从旧版本 SQLite 平滑升级？

A: 项目提供了迁移脚本 scripts/migrate.js，可通过指定 --from sqlite --to postgres 参数执行自动迁移。迁移过程会复制所有链接、标签、用户与收藏集数据，并保持外键关联不变。建议在生产环境迁移前先在预发布环境进行演练，并确保数据库字符集设置为 UTF-8 以支持多语言标题。
