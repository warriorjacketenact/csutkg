# VODLink Collective

VODLink Collective 是一个面向技术内容聚合与数字资源索引的开源外链管理平台。项目定位于为开发者、技术研究者以及内容运营团队提供一套结构清晰、可扩展的链接资源汇集方案，尤其适用于需要批量管理和呈现大量外部视频点播（VOD）相关入口的场景。

本项目不仅是一个静态链接列表，更是一套围绕 URL 资源组织的元数据框架。它解决了分散链接难以归类、缺乏上下文、难以团队协作维护的痛点，通过标准化的目录结构与文档体系，将碎片化的网络资源转化为可复用、可审计、可分享的知识资产。目标用户包括开源文档维护者、技术社区运营、教育内容策划者以及任何需要系统化整理网络书签的个体。

## 功能概览

- **批量链接归集与去重**：支持以文本流方式导入大量 URL，自动进行协议与域名级别的归一化校验，并提供基于路径特征的重复条目检测，确保资源库的整洁性。

- **分层分类目录导航**：依据链接路径模式、文件类型或预期用途，将资源动态映射至多级分类树中，用户可通过目录快速定位至特定前缀或关键词下的相关条目。

- **元数据扩展与注解**：每条链接可关联自定义标签、状态标记（如有效、失效、待审）、维护者备注以及时间戳，便于团队协作时沟通上下文。

- **自动化健康检查**：集成可配置的链接存活探测模块，定期对已收录 URL 执行 HTTP 状态检查，并生成可用性报告，帮助及时发现并清理失效资源。

- **全文与字段级检索**：内置轻量级搜索接口，支持对 URL 本身、分类标签、注解内容等进行关键词匹配，结果按相关度排序输出。

- **多格式数据导出**：支持将资源列表导出为 JSON、CSV 或纯文本格式，满足与其他系统（如监控工具、静态站点生成器、数据分析流水线）的集成需求。

- **变更历史记录**：记录每次新增、删除或修改链接的操作日志，包含操作人、时间及变更摘要，支持回溯与审计。

- **访问权限分级**：提供基于角色的只读、编辑、管理三级权限模型，适用于将资源库开放给公开用户浏览，同时限制写操作仅对授权成员开放。

## 应用场景

- **技术文档外部引用管理**：当项目文档需要引用大量外部视频教程、API 参考或社区讨论帖时，可使用 VODLink Collective 集中维护这些引用链接，并在文档中通过统一标识符动态引用，避免链接散落各处难以更新。

- **在线教育课程资源包整理**：教育机构或独立讲师可将课程中涉及的补充阅读材料、案例演示视频、工具下载地址等通过本系统分类存储，按课时或主题生成专属资源导航页，方便学员按需获取。

- **社区优质内容聚合与分享**：技术社区运营者可将成员推荐的优秀博文、会议录像、开源项目演示等链接统一收录，经过审核后对外发布为精选资源清单，降低信息过载，提升内容发现效率。

- **企业内部知识库外链托管**：企业技术团队可将常用的内部工具面板、监控看板、文档系统入口等集中管理，配合健康检查功能，确保关键业务链接始终可用，减少因链接失效导致的工作中断。

- **个人书签体系结构化迁移**：个人开发者可将浏览器中积累的大量杂乱书签导出后导入本系统，借助分类与注解功能重新组织，形成长期维护的个人知识外网资源索引。

## 快速开始

以下步骤帮助您在本地环境中快速启动 VODLink Collective 服务。

```bash
# 克隆代码仓库
git clone https://github.com/example/vodlink-collective.git
cd vodlink-collective

# 安装项目依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 初始化本地数据库并导入示例资源
python manage.py initdb
python manage.py load --source data/sample_links.txt

# 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8080
```

上述命令执行完毕后，可通过浏览器访问 `http://localhost:8080` 查看资源列表首页。如需导入您自己的链接数据，请参考 `docs/import_guide.md` 中的格式说明。

## 项目结构

项目采用分层模块化设计，核心逻辑、数据模型、工具脚本与用户界面彼此分离，便于独立迭代与测试。

```
vodlink-collective/
├── cli/                                 # 命令行入口与交互逻辑
│   ├── main.py                          # 主调度器，注册所有子命令
│   ├── commands/                        # 各子命令实现（initdb, load, check, export）
│   │   ├── initdb.py                    # 数据库初始化与迁移控制
│   │   ├── load.py                      # 从文本/JSON/CSV 导入链接
│   │   ├── check.py                     # 触发健康检查并输出报告
│   │   └── export.py                    # 导出为指定格式
│   └── utils.py                         # CLI 通用辅助函数（日志、进度条）
├── core/                                # 核心业务逻辑与数据模型
```

## 常见问题

**问：导入链接时提示格式错误，但我的 URL 看起来是有效的，如何解决？**

答：导入器默认对 URL 执行严格校验，要求包含协议头（http:// 或 https://）且域名部分符合标准格式。若使用纯文本文件导入，请确保每行仅包含一个 URL，且不含多余空白字符或尾随斜杠（路径内斜杠除外）。如仍遇到问题，可尝试使用 `--skip-validation` 选项跳过格式检查，但此举可能引入无效条目，建议优先检查原始数据。

**问：健康检查模块报告大量链接超时，但浏览器中可正常访问，是什么原因？**

答：健康检查使用 requests 库的默认超时设置（连接超时 5 秒，读取超时 10 秒）。若目标服务器响应较慢或存在地域性网络延迟，可能触发超时。可通过修改 `core/health.py` 中的 `timeout` 参数值调整等待时间。此外，部分服务器会拦截非浏览器 User-Agent 请求，可考虑在配置中启用自定义 UA 头。

**问：如何将当前 SQLite 数据库迁移至 PostgreSQL 以备生产环境使用？**

答：项目数据层基于 SQLAlchemy ORM，理论上支持多数据库后端。迁移时，首先在目标 PostgreSQL 中创建空数据库，然后使用 `python manage.py db dump --format json > data.json` 导出当前数据，再修改配置文件中 `SQLALCHEMY_DATABASE_URI` 指向 PostgreSQL 连接串，最后执行 `python manage.py db load --from data.json` 导入数据。建议在迁移前完整备份原始 SQLite 文件。
