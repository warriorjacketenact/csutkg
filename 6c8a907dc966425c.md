# ResourceCatalog Pro

ResourceCatalog Pro 是一个面向技术团队、内容策展人与开源项目维护者的结构化外链资源汇总与管理平台。该项目并非简单的书签集合，而是一套具备分类索引、状态标记与快速检索能力的资源导航系统，适用于需要长期维护大量外部参考链接的技术文档库或知识库项目。

项目定位为“技术资源的中转与分发枢纽”，解决的核心问题包括：分散在多个文档中的外链难以统一更新与校验、资源失效后无法快速定位、新成员加入项目时无法迅速了解可用的外部参考体系。ResourceCatalog Pro 通过集中的目录结构与规范化的资源条目编排，使外链管理成为项目基础设施的一部分，而非临时性的文档工作。

## 功能概览

批量资源导入：支持从结构化数据源批量导入 URL 列表，自动解析路径参数并生成初始目录索引。

状态标记系统：为每条资源提供生效、失效、待审核、已迁移四种状态标记，支持手动与定时探测切换。

分类视图生成：按资源路径前缀、域名、内容主题自动划分视图层级，支持多维度筛选与排序。

检索与过滤接口：提供基于路径关键词、状态、导入批次的组合过滤能力，适配 CLI 与 Web 两种调用方式。

变更审计日志：记录每条资源的添加、删除、状态变更与备注修改操作，支持回滚与导出。

导出适配器：支持将资源列表导出为 Markdown 表格、JSON Schema、HTML 目录页与纯文本清单四种格式。

插件化探测引擎：允许用户自定义资源可达性探测策略，包括 HTTP 状态码校验、关键字匹配与响应时间阈值。

## 应用场景

技术文档库的外链治理：当项目包含大量参考链接、API 文档地址或社区教程时，可使用 ResourceCatalog Pro 统一收录并定期校验可用性，避免文档中出现大量死链影响读者体验。

开源项目的贡献者引导：维护者可将常用开发资源、设计规范、代码示例仓库等链接集中管理，新贡献者通过资源目录即可快速了解项目依赖的外部生态，减少重复提问。

企业内部技术周报汇总：技术团队每周会产出大量周报并附带参考链接，使用本项目的批次管理功能可按周次归档，形成可持续检索的历史资源库。

个人知识库的外链索引：个人笔记或博客系统中若存在大量散落的引用链接，可通过本项目的结构化导入功能统一整理，并利用分类视图快速定位特定主题的资源。

## 快速开始

以下操作步骤适用于 Linux、macOS 及 Windows WSL 环境，假设已安装 Git 与 Node.js 18 以上版本。

```bash
git clone https://github.com/resource-catalog-pro/resource-catalog-pro.git
cd resource-catalog-pro
npm install
npm run build
npm start -- --batch 60 --source ./data/import.json --output ./dist/catalog.md
```

如需使用开发模式启动 Web 界面，可执行：

```bash
npm run dev
```

默认监听 127.0.0.1:3400，可通过环境变量 PORT 与 HOST 进行覆盖。

## 项目结构

```
resource-catalog-pro/
├── src/                           # 核心源代码目录
│   ├── core/                      # 核心引擎模块
│   │   ├── catalog.ts             # 目录树构建与节点管理
│   │   ├── batch.ts               # 批次导入与版本控制逻辑
│   │   └── resolver.ts            # URL 解析与规范化工具
│   ├── probes/                    # 探测引擎实现
│   │   ├── http-probe.ts          # HTTP/HTTPS 可达性探测
│   │   ├── keyword-matcher.ts     # 响应内容关键字匹配
│   │   └── probe-scheduler.ts     # 定时探测任务调度器
│   ├── adapters/                  # 导出适配器集合
│   │   ├── markdown-adapter.ts    # Markdown 表格与列表导出
│   │   ├── json-adapter.ts        # JSON Schema 导出
│   │   └── html-adapter.ts        # 静态 HTML 目录页生成
│   ├── web/                       # Web 界面与 API 路由
│   │   ├── routes/                # Express/Koa 风格路由定义
│   │   ├── middleware/            # 鉴权、日志与错误处理中间件
│   │   └── static/                # 前端静态资源分发
│   └── cli/                       # 命令行入口与参数解析
│       ├── commands/              # 各子命令实现（import, export, probe）
│       └── repl.ts                # 交互式 REPL 调试环境
├── config/                        # 配置文件目录
```

## 常见问题

Q: 导入大量链接时如何避免重复条目？

A: 系统在校验阶段会对 URL 进行标准化处理（移除尾部斜杠、统一协议大小写），并基于标准化后的字符串与已有目录进行比对。若需强制覆盖，可传入 --force 参数；若需保留历史记录，可使用 --dry-run 预览变更。

Q: 探测引擎是否支持需要认证或带 Cookie 的站点？

A: 当前版本的 HTTP 探测模块支持自定义请求头与 Cookie 注入，您可以在配置文件的 probes.http 字段中设置 headers 与 cookieJar 选项。对于 OAuth 等复杂认证流程，建议使用插件化机制编写自定义探测策略。

Q: 如何将本项目的资源列表与现有静态站点生成器集成？

A: 您可使用 markdown-adapter 导出 Markdown 格式的资源清单，并将其作为源文件放入 Hugo、VuePress 或 Docusaurus 项目的 content 目录。对于需要动态渲染的场景，推荐使用 json-adapter 导出 JSON 数据，再由前端框架自行消费。
