# WebLink Collective Indexer

WebLink Collective Indexer 是一个面向技术研究与知识管理场景的轻量化外链资源归集与导航系统。项目定位于技术团队、独立研究者以及文档维护者，用于对分散在各类活动页面、节目单或动态内容中的 URL 资源进行统一抓取、结构化存储和可视化索引。核心解决的问题包括：外链失效难以追踪、资源分散无法批量检索、以及原始页面上下文缺失导致的链接语义不明。

本项目并非通用爬虫框架，而是一套以 URL 清单为中心的数据治理工具。其默认以只读方式处理输入链接列表，提取元信息并生成持久化的索引快照，适用于自动化文档流水线、第三方内容审核以及历史链接归档等场景。当前批次为第 1159/1241 批，共计收录 250 个资源链接，所有链接均来源于 emha-shows.org 节目体系下的深层页面。

## 功能概览

**链接清单导入与校验** 支持从纯文本列表、CSV 以及直接粘贴的 URL 集合中批量导入链接，自动执行协议一致性检查和去重操作。

**元数据自动提取** 对每个 URL 发起安全的 HEAD 请求，获取内容类型、内容长度、最后修改时间以及服务器类型等基础元信息，不下载完整响应体。

**自定义标签与分类** 允许用户为每个链接添加多个层级标签，并支持基于正则表达式的自动标签规则，便于按照节目类型、语言或页面结构进行分组。

**索引快照生成** 将处理后的链接集合输出为静态 Markdown 索引表或 JSON 结构化数据，支持嵌入项目文档或作为独立数据资产发布。

**变更检测与过期标记** 记录每次扫描的状态码与响应时间，对比历史快照识别新增、失效或重定向的链接，并生成差异报告。

**查询过滤与排序** 内置简单的命令行查询接口，支持按域名、状态码、标签组合以及更新时间范围进行过滤，并可按响应时间或内容长度排序。

## 应用场景

文档站点的外链健康监控。技术博客、开源项目文档或在线教程常包含大量外部引用链接，使用本工具可定期扫描这些链接，自动标记返回 4xx 或 5xx 状态的失效引用，帮助维护者快速修复断链。

节目单或活动页面的资源聚合。对于包含多个演讲、工作坊或展览页面的节目列表，本工具可将所有子页面链接提取为扁平化清单，并依据页面标题或分类参数生成目录索引，方便参与者一站式浏览全部内容。

历史页面迁移前的链接盘点。在进行站点改版或域名更换时，运维人员可通过本工具导出所有旧域名下的链接清单，结合元数据中的路径信息制定重定向规则或迁移策略。

合规审计中的外链备案。企业内部法务或合规团队可使用本工具导出指定页面集合中的所有第三方链接，形成提交给安全审计部门的外部资源清单，降低违规引用风险。

## 快速开始

以下命令演示如何从当前目录克隆项目、安装依赖并运行一次完整的索引任务。

```bash
git clone https://github.com/weblink-collective/indexer.git
cd indexer
pip install -r requirements.txt
python cli.py --input ./samples/emha_links.txt --output ./reports/index_snapshot.md --timeout 5 --retry 2
```

其中 `samples/emha_links.txt` 为包含一行一个 URL 的纯文本文件。执行完毕后，将在 `reports` 目录下生成包含元数据表格的 Markdown 索引文件。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.8 及以上 | 是 | 核心运行环境，推荐使用 3.10 或 3.11 以获得更好的异步支持 |
| pip 21.0 及以上 | 是 | 用于安装 requirements.txt 中声明的第三方库 |
| requests 2.28.0 及以上 | 是 | 发送 HTTP HEAD 请求并处理超时与重定向逻辑 |
| click 8.1.0 及以上 | 是 | 提供命令行交互接口的参数解析与子命令组织 |

| pytest 7.0.0 及以上 | 否 | 仅当需要运行单元测试或集成测试时安装，生产环境可不装 |
| mypy 1.0.0 及以上 | 否 | 仅开发阶段用于静态类型检查，不影响运行功能 |

## 项目结构

项目采用分层架构，核心处理逻辑与接口层分离，便于扩展不同的输入输出格式。

```
indexer/
```

## 常见问题

问：如何处理目标服务器拒绝 HEAD 请求的情况？

部分 Web 服务器可能不支持 HEAD 方法或对 HEAD 请求返回与 GET 不同的状态码。本工具在遇到 HEAD 失败时会自动降级为 GET 请求并设置 `stream=True`，仅读取响应头即断开连接，避免下载完整内容。若仍失败，该链接将标记为元数据获取失败，并在报告中注明错误类型。

问：索引报告中显示的状态码与浏览器访问不一致是什么原因？

可能原因包括：服务器根据 User-Agent 返回不同内容、请求经过中间代理或 CDN 缓存、或者页面本身存在 JavaScript 重定向。本工具默认发送的 User-Agent 为 `WebLinkCollector/1.0`，与常规浏览器不同。如需模拟特定客户端，可通过环境变量 `DEFAULT_HEADERS` 自定义请求头。

问：如何批量更新已有索引快照？

使用 `update` 子命令并指定旧快照文件和新链接列表即可。工具会自动读取旧快照中的元数据，对新列表发起请求，然后合并两部分信息。对于旧快照中存在但新列表中缺失的链接，将保留在最终输出中并标记为已移除。
