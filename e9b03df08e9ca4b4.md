# RY MFS 外部资源索引与分发系统

RY MFS 是一个面向技术团队与内容研究者的轻量级外部资源索引与结构化分发平台。项目定位于将分散在多个外部路径中的技术文档、配置文件、示例代码、数据集描述等资源进行统一归纳，并通过标准化访问方式提供给内部开发流水线、自动化测试环境以及知识库系统。本仓库不直接托管资源实体，而是维护一套可校验的索引清单与访问契约，便于用户快速定位并获取所需内容。

本项目的目标用户包括：需要批量引用外部资源的持续集成系统维护人员、依赖多版本配置文件的微服务开发团队、以及需要追踪外部数据源变更的研究人员。通过本仓库提供的索引结构与辅助脚本，用户可以在不感知底层存储细节的前提下，实现资源的版本化引用、完整性校验以及访问异常的重试策略配置。

## 功能概览

**统一资源索引表** 维护超过两百条外部资源路径的记录，每条记录包含资源标识符、预期内容类型、最后同步时间戳以及访问优先级标记。

**路径规范校验工具** 提供命令行脚本，可对索引中每条 URL 进行协议一致性、域名合法性以及路径格式检查，确保所有条目符合仓库定义的访问契约。

**批量拉取与缓存机制** 支持基于索引清单的并发下载任务，自动将远端资源缓存至本地工作目录，并生成对应的 SHA-256 校验文件供后续完整性验证。

**变更追踪与差异报告** 每日定时任务对比当前索引与远端资源响应头中的修改时间字段，生成新增、删除或内容更新的差异清单，并以结构化日志输出。

**访问统计与健康度看板** 记录每次资源请求的响应状态码、延迟分布与失败原因，通过内置的统计模块生成可视化的健康度报告，帮助维护者识别不稳定资源。

**多格式导出适配器** 支持将索引数据导出为 JSON、YAML 以及 CSV 格式，便于集成到不同技术栈的下游工具中，例如 Ansible 清单、Prometheus 监控配置或 Terraform 变量文件。

## 应用场景

自动化部署流水线中的依赖预取。在 CI/CD 流程的初始化阶段，运维人员可执行本仓库提供的拉取脚本，一次性将本次构建所需的所有外部配置文件与工具包下载至构建机本地缓存，避免构建过程中因网络波动或源站限流导致的失败。

多环境配置文件的版本同步管理。开发团队使用本仓库索引不同部署环境（开发、测试、预发布、生产）对应的配置资源路径，通过切换分支或标签即可统一调整所有环境引用的外部资源版本，降低人工修改配置项带来的错误风险。

数据科学项目的外部数据集引用。数据工程师可将训练数据集、验证集以及特征映射文件的远端路径注册至本索引系统，配合校验和机制确保每位团队成员拉取到完全一致的数据内容，消除因数据版本不一致导致的实验结果不可复现问题。

离线文档站点的资源预打包。技术文档维护者利用本项目的批量拉取功能，定期将外部图表、API 定义文件以及交互式演示资源同步到内部文档服务器，实现在内网环境中完整呈现原本依赖外网 CDN 的文档页面。

## 快速开始

以下命令演示如何从本仓库克隆索引定义、安装依赖脚本并执行首次资源同步。所有操作均在 POSIX 兼容环境中完成。

```bash
git clone https://github.com/your-org/ry-mfs-index.git
cd ry-mfs-index
pip install -r requirements.txt
python scripts/fetch_all.py --output ./cache --parallel 8
```

执行上述命令后，系统将读取仓库根目录下的 index.yaml 文件，根据其中记录的全部资源路径发起并发请求，并将获取到的内容保存至 ./cache 目录。首次运行可能需要数分钟，具体耗时取决于网络状况与远端资源响应速度。

## 安装要求

本项目的运行依赖以下环境与工具库，建议在 Ubuntu 20.04 LTS 或更高版本的操作系统中部署。所有依赖项均可通过 pip 与系统包管理器完成安装。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | 3.8 及以上 | 所有脚本及核心逻辑的解释器环境 |

| pyyaml | 5.4.1 及以上 | 解析索引清单的 YAML 格式定义 |
| click | 8.0.0 及以上 | 命令行交互接口与参数解析框架 |
| tqdm | 4.62.0 及以上 | 批量下载任务的进度条可视化支持 |
| pytest | 6.2.5 及以上 | 单元测试与集成测试运行器（仅开发环境） |

## 项目结构

以下 ASCII 目录树展示本仓库的组织方式，包括核心脚本、配置目录、测试套件以及文档路径。每行末尾的注释简要说明该目录或文件的职责。

```
ry-mfs-index/
├── index.yaml                      # 主索引清单，记录所有外部资源路径及元数据
├── requirements.txt                # Python 运行时依赖声明
├── setup.py                        # 项目安装与分发配置
├── pytest.ini                      # 单元测试框架配置文件
├── .env.example                    # 环境变量模板，用于配置代理与超时参数
├── scripts/                        # 可执行脚本目录
│   ├── fetch_all.py                # 批量拉取所有索引资源的入口脚本
│   ├── validate_urls.py            # 校验索引中每条 URL 的协议与格式合规性
│   ├── check_updates.py            # 对比远端资源修改时间，生成变更报告
│   └── export_formats.py           # 将索引导出为 JSON、YAML、CSV 格式
├── src/                            # 核心源代码目录
│   ├── __init__.py                 # 包初始化文件
│   ├── loader.py                   # 加载并解析 index.yaml 文件的模块
│   ├── fetcher.py                  # 封装 HTTP 请求逻辑与重试策略
│   ├── cache.py                    # 本地缓存管理，包括目录创建与文件写入
│   ├── validator.py                # 响应内容类型与校验和验证
│   └── stats.py                    # 记录请求统计信息并生成健康度摘要
├── tests/                          # 测试套件目录
│   ├── test_loader.py              # 针对 loader 模块的单元测试
│   ├── test_fetcher.py             # 针对 fetcher 模块的单元测试，包含模拟响应
│   ├── test_cache.py               # 缓存读写与目录结构测试
│   └── fixtures/                   # 测试用的固定样本数据
│       └── sample_index.yaml       # 精简版索引用于快速测试
├── docs/                           # 文档根目录
│   ├── user-guide/                 # 用户手册章节
│   ├── ops/                        # 运维相关文档
│   ├── developer/                  # 开发者指南
│   └── design/                     # 设计决策与架构说明
└── .github/                        # GitHub 自动化工作流配置
    └── workflows/                  # CI 流水线定义
        └── daily_sync.yml          # 每日定时触发资源同步与健康检查
```

## 常见问题

问题：批量拉取过程中部分资源返回 403 或 429 状态码，如何处理？

解答：此类状态码通常表示远端服务器对请求频率或来源 IP 进行了限制。建议首先检查 scripts/fetch_all.py 中的 --delay 参数，适当增加请求间隔（例如设置为 2 秒）。若问题持续，可配置 .env 文件中的代理服务器地址，通过代理池轮换出口 IP。此外，项目内置的指数退避重试机制会自动处理临时性故障，默认重试 3 次，可通过环境变量 RY_RETRY_COUNT 调整重试次数。

问题：如何添加新的资源路径到索引中？

解答：请直接编辑仓库根目录下的 index.yaml 文件。新增条目需包含 resource_id（全局唯一字符串）、url（完整路径）、content_type（如 application/json 或 text/plain）以及 optional_headers（若需要特殊请求头）。编辑完成后，运行 scripts/validate_urls.py 对新条目进行格式校验，确认无误后提交变更。建议在提交信息中注明新增资源的用途与来源，便于其他维护者审阅。

问题：本地缓存目录占用磁盘空间过大，如何清理？

解答：项目默认缓存目录为 ./cache，所有下载的文件均保留原始名称并附带 .sha256 校验文件。您可手动删除该目录下的部分文件，或运行 scripts/cache_cleaner.py --days 30 命令自动清理 30 天前访问过的缓存文件。若需修改缓存保留策略，可在 index.yaml 中为每个资源单独设置 ttl_days 字段，系统将在每日同步任务中自动删除超期缓存。
