# WebResource Navigator

WebResource Navigator 是一个面向技术研究者、内容聚合者与知识管理者的高密度外链资源整合平台。该项目并非传统的爬虫或采集工具，而是一套半自动化的资源索引体系，专注于对特定领域内的优质外链进行结构化整理、分类归档与状态跟踪。项目定位为技术向的资源罗盘，帮助用户在信息过载的环境中快速定位到高价值的外部内容，而非替代搜索引擎或内容分发网络。

项目核心目标用户包括开源社区维护者、技术文档编写者、DevOps 工程师、数据分析师以及各类需要频繁查阅外部参考资料的技术决策者。通过集中式的链接库管理与轻量级的元数据标注机制，WebResource Navigator 将散落在网络各处的碎片化知识串联为可检索、可追溯、可审计的知识链路。

## 功能概览

**批量链接导入与去重** 支持从 CSV、JSON 及纯文本列表批量导入外部 URL，自动进行语法标准化与重复项过滤，确保资源库的整洁性。

**自定义分类标签系统** 用户可为每条链接赋予多级分类标签与自由关键词，支持基于标签组合的快速筛选与统计，便于构建领域知识图谱。

**链接可用性健康检查** 内置异步 HTTP 状态检测模块，定期对已收录链接进行可达性验证，自动标记失效链接并生成告警报告，维持资源库的鲜活度。

**全文元数据提取** 对每条链接自动抓取页面标题、描述描述、内容摘要及最后修改时间，生成可搜索的元数据索引，提升检索精度。

**Markdown 格式文档生成** 根据用户定义的模板与分类规则，一键将链接库导出为符合开源项目规范的 Markdown 文档，便于集成至项目仓库或静态站点。

**访问频率与热度统计** 记录每条链接的点击次数与访问时间分布，提供简单的热度排序功能，辅助用户识别高频参考资源。

**私有备注与协作批注** 支持为每条链接添加内部备注、阅读状态标记及协作者评论，适用于团队共享资源库的场景。

**多格式数据导入导出** 除 Markdown 外，支持导出为 HTML 目录、JSON 结构化数据及 OPML 订阅列表格式，兼容主流知识管理工具。

## 应用场景

**技术文档编写中的参考资料管理** 技术作者在撰写长篇教程或架构设计文档时，需引用大量外部规范、论文或博客。WebResource Navigator 可集中收纳所有引用链接，自动生成符合文档规范的参考附录，避免手工维护引用列表带来的遗漏与格式错误。

**开源项目 README 外链资源沉淀** 开源项目维护者通常在 README 中放置若干外部链接，但随项目迭代，这些链接易失效或过时。利用本项目的健康检查与元数据提取功能，可定期更新外链状态，确保 README 中每个外部引用均准确有效。

**数据分析竞赛中的数据集索引** 数据科学团队在参与竞赛或研究时，需频繁查阅多个数据仓库、API 文档和基准测试报告。本项目的分类标签与热度统计能力可帮助团队快速筛选高频使用的核心资源，降低信息检索成本。

**企业内部知识库外链治理** 大型企业的内部 Wiki 中往往散落大量指向外部供应商、技术社区或标准组织的外链。使用 WebResource Navigator 可对这些外链进行统一审计、合规性检查与访问权限标注，提升知识库的规范性与安全性。

## 快速开始

以下步骤指导您在三分钟内完成 WebResource Navigator 的本地部署并导入第一批链接数据。

```bash
# 克隆项目仓库至本地
git clone https://github.com/webresource-navigator/webresource-navigator.git

# 进入项目根目录并安装依赖
cd webresource-navigator
pip install -r requirements.txt

# 执行初始化数据迁移与配置
python manage.py migrate
python manage.py runserver
```

## 安装要求

WebResource Navigator 基于 Python 3.10 及以上版本开发，依赖主流异步网络库与数据处理库，具体环境要求如下表所示。

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.10 或更高 | 核心运行时环境，建议使用 3.11 以获取更好的异步性能 |
| pip | 22.0 或更高 | Python 包管理工具，用于安装项目依赖 |
| aiohttp | 3.9.0 或更高 | 异步 HTTP 客户端，用于并发链接健康检查与元数据抓取 |
| beautifulsoup4 | 4.12.0 或更高 | HTML 解析库，用于提取页面标题、描述等元数据 |
| sqlite3 | 系统内置 | 轻量级嵌入式数据库，用于存储链接记录与统计信息（生产环境可切换至 PostgreSQL） |
| lxml | 4.9.0 或更高 | 高性能 XML/HTML 解析器，作为 beautifulsoup4 的后端加速解析过程 |
| pandas | 2.0.0 或更高 | 数据分析库，用于处理批量导入导出的结构化数据 |
| pyyaml | 6.0.0 或更高 | YAML 格式解析器，用于读取用户自定义配置模板 |
| tqdm | 4.65.0 或更高 | 进度条工具，用于长任务执行过程中的状态反馈 |

## 项目结构

项目目录遵循 Django 风格的应用布局，同时整合独立工具模块与文档目录，便于功能分离与维护。

```
webresource-navigator/
├── manage.py                     # Django 项目管理入口，用于运行开发服务器与数据库迁移
├── requirements.txt              # 项目依赖清单，固定所有第三方库版本号
```

## 贡献指南

WebResource Navigator 遵循开源社区协作规范，欢迎各类形式的贡献，包括但不限于代码提交、文档完善、问题反馈与功能建议。所有贡献者需遵守行为准则，并在提交前完成以下步骤。

第一步，查阅项目看板与议题列表。访问 GitHub Issues 页面，查找标记为 good-first-issue 或 help-wanted 的待办任务，避免与其他贡献者重复工作。对于新功能提议，请先创建一个议题描述您的想法，获得核心维护者反馈后再着手实现。

第二步，派生项目仓库并创建功能分支。将主仓库 Fork 至个人账号，克隆至本地后，基于 main 分支创建描述性的分支名称，例如 feature/add-rss-exporter 或 fix/duplicate-checker-timeout。所有开发工作均在该分支上进行。

第三步，编写代码并确保测试覆盖。新增功能需附带相应的单元测试，测试用例应覆盖正常路径与异常边界。运行现有测试套件确保未引入回归问题。代码风格遵循 PEP 8 规范，使用 black 与 flake8 进行自动格式化与检查。

第四步，提交变更并签署开发者原产地证书。提交信息采用约定式提交格式，例如 feat: 或 fix: 前缀。所有提交必须签署 DCO，即在提交信息中添加 Signed-off-by 行，表明您有权贡献该代码。

第五步，发起拉取请求并参与代码审查。推送到个人远程仓库后，向主仓库的 main 分支发起 Pull Request。在 PR 描述中清晰说明变更内容、测试结果与相关议题编号。根据审查意见进行修改，直至获得至少一名维护者的批准。

## 常见问题

问：项目是否支持对 HTTPS 与 HTTP 混合链接的统一检查？如何处理自签名证书或内网地址？

答：健康检查模块默认遵循 HTTP 标准重定向链，支持 HTTPS 与 HTTP 协议混合检查。对于自签名证书或内网地址，可通过配置环境变量 CHECKER_SSL_VERIFY=false 关闭 SSL 证书验证，同时设置 CHECKER_ALLOW_PRIVATE=1 允许访问私有 IP 地址段。需要说明的是，关闭验证会降低安全性，仅建议在可信内网环境或开发阶段使用。

问：导入大量链接时，如何避免重复条目以及如何更新已有链接的元数据？

答：去重机制基于 URL 的标准化形式，即移除末尾斜杠、统一协议小写化，并剥离常见的跟踪参数如 utm_source。导入过程中，若检测到已存在的标准化 URL，将触发更新策略：可选覆盖旧元数据、跳过保留旧数据或合并标签集合。您可在导入配置文件中通过 duplicate_policy 参数控制具体行为，默认值为 merge，即合并新旧标签并刷新最后访问时间。

问：导出的 Markdown 文档能否自定义章节结构与头部信息？

答：导出模块支持 Jinja2 模板引擎，用户可在 templates/markdown/ 目录下放置自定义模板文件，覆盖默认的标题层级、分类排序规则及额外注释字段。同时支持在导出命令中通过 --template 参数指定模板路径，实现按项目风格动态生成 README 或附录文档。
