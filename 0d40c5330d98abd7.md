# LinkVault 资源聚合导航系统

LinkVault 是一个面向技术社区、知识管理团队和个人研究者的轻量级外链资源聚合与导航系统。该项目并非传统意义上的爬虫或采集器，而是一个基于人工精选与结构化标签体系的静态资源索引平台，旨在帮助用户从海量碎片化信息中快速定位高价值技术文档、工具站点与社区讨论。

项目定位为“技术外链的精选目录”，适用于需要维护团队知识库、构建技术雷达、或定期跟踪特定领域动态的开发者与架构师。LinkVault 不提供内容存储或代理转发，仅保留原始 URL 的元数据与分类标记，确保引用的准确性与可追溯性。

## 功能概览

- 多维度资源分类：支持按技术领域、文档类型、更新频率、语言版本进行多级标签筛选。

- 批量链接导入与校验：接受 CSV 或 JSON 格式的链接清单，自动去重并校验 URL 可访问性，返回状态码与响应时间。

- 全文元数据提取：对每个收录链接自动提取页面标题、描述、关键词及 Open Graph 信息，用于生成索引卡片。

- 自定义标签系统：允许用户为每个链接添加项目级私有标签，支持模糊搜索与标签云统计。

- 快照与变更追踪：记录每个链接的首次收录时间、最近校验时间以及内容摘要哈希值，用于检测目标页面重大变更。

- 只读 API 接口：提供基于 RESTful 风格的查询接口，支持按标签、域名、收录批次等条件过滤，返回 JSON 格式结果集。

- 静态站点生成器：内置模板引擎，可将当前资源库导出为完整的静态 HTML 导航站，适合部署到 CDN 或内部文档服务器。

## 应用场景

技术团队内部知识库维护：架构组或 DevOps 团队可使用 LinkVault 统一收集团队成员推荐的第三方库、运维手册和故障排查案例，通过标签系统按项目或技术栈分类，减少重复搜索成本。

开源项目 README 外链管理：开源项目维护者常需在文档中引用大量依赖项、学习资料和社区讨论链接。LinkVault 可作为外链的中间管理平台，定期校验链接有效性，避免 README 中出现死链。

技术调研与竞品分析：产品经理或技术调研人员在评估多个解决方案时，可将候选产品文档、性能测试报告、社区评价链接统一录入系统，利用标签记录对比维度，生成调研报告底稿。

个人开发者的书签库升级：替代浏览器自带的书签管理，提供更强大的检索、标注和过期提醒功能，尤其适合订阅技术博客、教程系列和 API 参考文档的重度阅读者。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，要求已安装 Git、Node.js 18+ 和 npm 或 yarn。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# 安装项目依赖
npm install

# 使用示例数据运行开发服务器
npm run dev -- --batch=983 --import=./samples/batch_983.json
```

执行完成后，终端会输出本地预览地址（通常为 http://127.0.0.1:5173），同时会在 ./dist 目录下生成当前批次的静态资源索引页面。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，用于执行索引构建与 API 服务 |
| npm 或 yarn | 最新稳定版 | 包管理器，用于安装项目依赖 |
| Git | 2.25 以上 | 版本控制，用于克隆仓库与提交更新 |
| SQLite | 3.35 以上 | 嵌入式数据库，用于存储链接元数据与标签关系 |
| curl 或 wget | 任意现代版本 | 用于链接校验模块发送 HTTP HEAD 请求 |
| openssl | 1.1.1 以上 | 用于生成链接摘要哈希值 |
| Python 3.9+（可选） | 仅当启用 ML 标签推荐时 | 用于运行辅助机器学习脚本 |
| Docker（可选） | 20.10 以上 | 用于容器化部署测试环境 |

## 常见问题

问：系统提示“链接校验超时”或“连接被拒绝”，如何处理？

答：LinkVault 默认的 HTTP 请求超时为 10 秒，重试次数为 2 次。如果目标服务器响应较慢或存在防火墙限制，您可以在 config/default.json 中调整 httpTimeout 和 retryCount 参数。对于内网或需要代理的环境，请设置 HTTP_PROXY 和 HTTPS_PROXY 环境变量。另外，部分站点会返回 403 状态码，此时需要检查 User-Agent 配置，项目默认使用 LinkVault/1.0 标识，可改为常见浏览器标识绕过简单限制。

问：如何将已收录的链接从一个标签迁移到另一个标签？

答：项目提供了标签迁移脚本 scripts/migrate-tags.ts。您需要准备一个映射配置文件（JSON 格式），指定原标签名与目标标签名的对应关系，然后执行 npm run migrate -- --config=./mapping.json。该操作会批量更新 link_tags 关联表，同时保留每条链接的收录时间和校验历史。建议在迁移前通过 npm run export -- --backup 导出当前数据库快照，以便回滚。

问：静态站点生成的中文页面出现乱码，如何解决？

答：请检查模板文件是否以 UTF-8 without BOM 格式保存，同时在生成器的输出配置中明确指定 charset=utf-8。如果问题仍存在，可能是系统环境变量 LANG 未设置为 zh_CN.UTF-8，请在启动生成命令前执行 export LANG=zh_CN.UTF-8。对于 Windows 环境，建议在 PowerShell 中运行 $OutputEncoding = [System.Text.Encoding]::UTF8 后再执行生成命令。
