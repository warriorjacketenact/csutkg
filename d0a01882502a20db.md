# VODLink Hub

VODLink Hub 是一个面向视频流媒体技术研究、内容分发网络（CDN）性能分析以及在线视频平台架构学习者的外链资源聚合与导航系统。本项目并非一个传统意义上的视频播放器或内容管理系统，而是一个高度结构化的技术资源索引库，旨在解决视频技术开发者在查找特定格式、编码标准、传输协议或性能测试样本时面临的链接分散、难以定位的问题。

本项目通过系统化的分类和目录树组织，将大量原本孤立的技术演示、样本视频、测试流和协议验证链接整合为统一的检索体系。其目标用户包括视频编码工程师、流媒体服务器运维人员、CDN质量评测专家以及相关领域的科研工作者。借助 VODLink Hub，用户能够快速从庞大的链接集合中筛选出符合当前测试或研究需求的精确资源，从而极大地提升技术调研与验证环节的效率。

## 功能概览

**分类导航体系** 提供基于视频编码格式、分辨率、帧率及传输协议的初级筛选结构，使用户能够按技术维度定位资源。

**快速检索入口** 内置基于关键字的链接元数据搜索功能，支持对链接所指向内容的编码参数、容器格式等信息进行查询。

**批量状态检测** 支持对收录的链接进行可达性与响应延迟的批量探测，帮助用户甄别当前有效的测试资源。

**自定义标签系统** 允许用户为链接添加自定义业务标签（如“H.265测试”、“4K高码率”），构建个性化的分类视图。

**结构化目录树展示** 以 ASCII 目录树形式呈现链接的存储层级与分类逻辑，便于理解资源组织架构。

**元数据扩展支持** 预留链接对应的格式、大小、时长等元数据字段接口，便于后续版本的数据补全与展示。

**配置持久化存储** 用户自定义的标签与分类配置可保存于本地存储中，实现跨会话的配置复用。

## 应用场景

在视频编码器的研发迭代过程中，工程师需要使用大量包含不同运动复杂度、纹理特征和亮度变化的测试序列来评估编码效率。VODLink Hub 聚合的链接能够为此类测试提供多样化的样本来源，避免研发团队自行采集与制作测试素材的重复劳动。

CDN 服务商在进行节点性能基准测试时，需要从全球不同区域发起对特定视频资源的并发拉流请求。本项目收录的链接可作为性能拨测的目标资源池，帮助测试团队快速构建覆盖多种码率和封装格式的测试用例集。

在线教育平台的技术选型团队在评估第三方视频转码服务或播放器 SDK 时，需使用标准化的测试视频集来验证兼容性与画质表现。VODLink Hub 提供的结构化链接索引可充当这一标准化测试集的资源目录，简化选型过程中的样本获取流程。

## 快速开始

以下步骤将引导您在本地环境中快速运行 VODLink Hub 的基础索引服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/vodlink-hub/vodlink-hub.git

# 进入项目根目录
cd vodlink-hub

# 安装所需依赖（基于 Node.js 环境）
npm install

# 启动本地索引服务，默认监听端口 3000
npm start
```

## 项目结构

```
vodlink-hub/
├── config/                           # 项目全局配置文件目录
│   └── default.json                  # 包含服务端口、数据库路径等默认设置
├── src/                              # 核心源代码目录
│   ├── index.js                      # 服务入口文件，启动HTTP服务器
│   ├── router/                       # 路由层，处理请求分发
│   │   ├── api.js                    # RESTful API 路由定义
│   │   └── static.js                 # 静态资源路由
│   ├── controller/                   # 控制器层，实现具体业务逻辑
│   │   ├── linkController.js         # 链接查询、筛选、状态更新逻辑
│   │   └── tagController.js          # 标签新增、删除、关联逻辑
│   ├── service/                      # 服务层，封装数据访问与外部调用
│   │   ├── databaseService.js        # SQLite 数据库连接与基础CRUD操作
│   │   └── healthCheckService.js     # 链接可达性检测与响应时间测量
│   └── model/                        # 数据模型定义
│       ├── LinkModel.js              # 链接实体模型，包含URL和元数据字段
│       └── TagModel.js               # 标签实体模型，包含名称与分类
├── data/                             # 数据存储目录
│   └── vodlink.db                    # SQLite 数据库文件，存储链接与标签数据
├── scripts/                          # 辅助脚本目录
│   ├── importLinks.js                # 用于从JSON文件批量导入链接记录
│   └── exportStats.js                # 导出链接状态统计报告
├── docs/                             # 项目文档目录
│   ├── user-guide.md                 # 用户使用手册
│   ├── developer-guide.md            # 开发者贡献指南
│   └── api-reference.md              # API接口详细参考文档
├── tests/                            # 单元测试与集成测试目录
│   ├── unit/                         # 单元测试用例
│   └── integration/                  # 接口集成测试用例
├── .gitignore                        # Git版本忽略规则文件
├── package.json                      # Node.js 项目配置文件，定义依赖与脚本
└── README.md                         # 项目说明文档（本文件）
```

## 贡献指南

我们欢迎并鼓励社区开发者为本项目贡献链接资源、分类建议或代码改进。请遵循以下步骤提交贡献。

首先，在 GitHub 上 Fork 本仓库至您的个人账户，并将 Fork 后的仓库克隆至本地开发环境。请确保您的开发分支基于最新的 main 分支创建。

其次，若您贡献的是新的视频链接资源，请按照 `data/link-template.json` 中定义的元数据格式，提供链接所对应的编码格式、分辨率、容器类型等信息，并通过项目提供的 `scripts/importLinks.js` 脚本进行导入验证。

再次，若您贡献的是代码功能改进或缺陷修复，请确保新增或修改的代码通过了所有现有单元测试，并为新功能补充相应的测试用例。提交前请执行 `npm run lint` 检查代码风格的一致性。

最后，通过 Pull Request 提交您的贡献。请详细描述本次提交的内容、目的以及相关的测试结果，以便项目维护者进行评审与合并。

## 常见问题

问：启动服务时提示 SQLite3 数据库连接失败，应如何处理？

答：请检查 `data/` 目录是否存在且具有写入权限。若该目录不存在，请手动创建。若问题依旧，请尝试删除 `data/vodlink.db` 文件后重新启动服务，系统将自动重建数据库表结构。

问：导入大量链接时脚本执行缓慢，有无优化建议？

答：导入脚本默认每批次提交100条记录。若您的机器内存充足，可修改 `scripts/importLinks.js` 中的 `BATCH_SIZE` 常量值（例如调整为500或1000）以提升批量插入性能。同时，建议在导入时关闭其他占用数据库连接的应用。

问：如何更新已收录链接的元数据信息？

答：您可以通过调用 `PATCH /api/links/{id}` 接口更新指定链接的元数据字段。具体请求体格式请参考 `docs/api-reference.md` 中的相关说明。对于批量更新，建议编写脚本调用接口循环处理。
