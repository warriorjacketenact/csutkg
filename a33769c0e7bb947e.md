# LinkVault Core

LinkVault Core 是一个面向技术团队与独立开发者的外链资产统一管理平台，专注于解决多源链接分散、失效不可追踪、上下文缺失等问题。项目以链接为最小管理单元，提供从采集、打标、监控到归档的完整生命周期能力，适用于文档站点、资源导航页、项目 README 聚合等场景。LinkVault Core 本身不依赖任何前端框架，核心引擎基于 Node.js 实现，可嵌入现有 CLI 工具或 CI 流程，亦可作为静态站点生成器的数据源中间件。

## 功能概览

**批量链接导入与规范化** 支持从 CSV、JSON、Markdown 列表及纯文本中批量抽取 URL，自动完成去重、协议补全检测、末尾斜杠标准化，并提供冲突报告。

**自定义元数据标签体系** 允许为每条链接附加键值对形式的标签，例如 `topic=database`、`status=stable`、`lang=zh`，支持基于标签的快速过滤与分组输出。

**链接可达性健康检查** 内置基于 Node.js `http` 模块的异步探测引擎，可配置超时与重试策略，定期检测链接是否返回 2xx/3xx 状态码，并生成不可达链接清单。

**多格式数据导出** 支持将链接库导出为 Markdown 列表、JSON 映射表、HTML 无序列表或 CSV 报告，便于嵌入不同文档系统或静态站点。

**链接上下文备注系统** 每条链接可附带 280 字符以内的备注说明，用于记录来源、用途、注意事项或内部评审意见，备注内容参与全文检索。

**变更历史审计日志** 所有新增、删除、修改操作均写入本地 SQLite 审计表，记录操作时间、操作类型与旧值，支持回滚至任意历史版本。

**命令行交互界面** 提供完整的 CLI 子命令，包括 add、list、check、tag、export、history，支持脚本化调用与人工运维两种模式。

## 应用场景

**技术文档站点的外链资产维护** 当团队维护一份包含数十个外部参考链接的技术文档时，LinkVault Core 可定期自动检测链接有效性，并在文档构建流程中输出失效链接清单，避免文档引用的外部资源悄然变更或消失。

**个人开发者知识库的资源索引** 开发者在研究新技术栈或阅读开源项目源码时，常积累大量临时收藏的 URL。LinkVault Core 提供轻量级标签与备注功能，帮助将散落的链接组织为可检索、可分类的知识索引，替代传统的浏览器书签文件夹。

**开源项目 README 中的资源导航生成** 对于需要列举大量社区资源、插件列表或案例地址的开源项目，LinkVault Core 支持从结构化数据源一键生成格式统一的 Markdown 链接列表，减少手动维护链接格式的工作量，同时通过健康检查提前发现已失效的推荐资源。

**CI 流程中的链接合规检查** 在持续集成流水线中加入 LinkVault Core 的检查步骤，自动扫描项目仓库中的所有 Markdown 文件，提取外部链接并进行可达性与协议安全性校验，阻断包含不可用或非 HTTPS 外链的构建版本。

## 快速开始

以下命令演示从克隆仓库到运行基础链接检查的完整流程。

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
npm install
node bin/linkvault.js check --source ./sample-links.txt --output ./report.json
```

若需初始化一个新的链接库并添加第一条记录，可使用以下命令：

```bash
node bin/linkvault.js init ./my-vault
node bin/linkvault.js add "https://example.com/docs" --tag "official,guide" --note "Primary reference for v2 API"
node bin/linkvault.js list --tag "official"
```

## 项目结构

项目采用模块化分层设计，核心逻辑与命令行入口分离，便于二次开发与单元测试。

```
linkvault-core/
├── bin/
│   └── linkvault.js                 # CLI 入口，负责解析命令参数并调用对应模块
├── lib/
│   ├── core/
│   │   ├── vault.js                 # 链接仓库核心类，管理增删改查与持久化
│   │   ├── probe.js                 # HTTP 探测引擎，包含超时控制与重试逻辑
│   │   └── tag-index.js             # 标签倒排索引，支持多标签组合查询
│   ├── adapters/
│   │   ├── json-adapter.js          # JSON 格式读写适配器
│   │   ├── markdown-adapter.js      # Markdown 列表生成与解析适配器
│   │   └── sqlite-adapter.js        # SQLite 存储适配器，含表结构初始化
│   ├── cli/
│   │   ├── commander.js             # 基于 commander 的命令注册与参数校验
│   │   └── output-formatter.js      # 终端输出格式化（表格、颜色、分页）
│   └── utils/
│       ├── url-normalizer.js        # URL 去重、协议补全、路径标准化
│       ├── validator.js             # 链接格式与标签命名规范校验
│       └── logger.js                # 分级日志写入器，支持文件与控制台输出
├── test/
│   ├── unit/                        # 单元测试，覆盖核心类与工具函数
│   └── integration/                 # 集成测试，模拟完整 CLI 执行流程
├── docs/                            # 完整文档目录，结构与导航章节对应
├── samples/                         # 示例数据文件，用于快速上手与演示
│   ├── sample-links.txt
│   └── sample-tags.json
├── .github/
│   └── workflows/
│       └── ci.yml                   # GitHub Actions 持续集成配置
├── package.json                     # npm 清单，声明依赖与脚本命令
├── README.md                        # 本文件
└── LICENSE                          # MIT 许可证文本
```

2. 创建新的功能分支，分支命名格式为 `feat/功能简述` 或 `fix/问题编号`。所有代码修改须附带相应的单元测试，确保测试覆盖率达到现有水平以上。

3. 提交代码前运行 `npm run lint` 与 `npm run test`，确保代码风格一致且所有测试通过。提交信息应遵循 Conventional Commits 规范，使用 `feat:`、`fix:`、`docs:`、`refactor:` 等类型前缀。

4. 向主仓库的 `main` 分支发起 Pull Request，在描述中清晰说明改动目的、实现方案及可能的兼容性影响。核心维护者将在 7 个工作日内完成评审。

5. 若涉及新增外部依赖或修改存储结构，需同步更新 `docs/design/` 下的设计文档，并在 PR 中标记对应文档变更。

## 常见问题

**链接健康检查报告显示大量超时，但浏览器可以正常访问**

探测引擎默认超时时间为 3000 毫秒，若目标服务器响应较慢或网络环境受限，可能出现误报。可通过 `--timeout` 参数调整超时阈值，例如 `--timeout 8000`。同时建议检查是否被目标站点防火墙拦截，可尝试添加 `--user-agent` 参数模拟常见浏览器标识。

**标签查询结果不完整，部分链接未被召回**

标签匹配默认采用精确匹配模式，若标签值包含前后空格或特殊字符，可能导致匹配失败。建议使用 `list --tags` 子命令查看当前所有已注册标签的标准化版本。对于多标签组合查询，需注意各标签之间为逻辑与关系，若希望使用逻辑或，请分别查询后合并结果。

**从旧版本迁移至新版本时数据格式不兼容**

项目在 `lib/adapters/` 中提供了版本迁移脚本，运行 `node bin/linkvault.js migrate --from v1 --to v2` 即可自动完成数据转换。执行迁移前请备份 `vault.sqlite` 文件。若自定义了导出模板，需手动检查模板中的字段映射是否与新版数据结构一致。
