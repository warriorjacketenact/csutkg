# WebIndexer

WebIndexer 是一个面向技术研究、内容聚合与数字档案建设的轻量级外链资源索引系统。项目定位于帮助个人开发者、内容策展人及小型研究团队，将散落在网络各处的优质外部链接进行结构化收录、分类存储与快速检索，从而构建可长期维护的外部知识索引库。

WebIndexer 不提供爬虫抓取或自动更新功能，而是强调人工筛选与精细标注。通过预设的分类模板与标签体系，用户可以将任意 URL 纳入本地索引仓库，并按批次、领域、质量等级进行多维度管理。本项目适用于需要长期跟踪特定主题资源变化、建立个人学习库或团队共享知识图谱的使用者。

## 功能概览

**多批次索引管理**：支持按项目批次对资源链接进行分组收录，每批次可独立命名并记录收录时间、收录人及备注信息。

**结构化分类存储**：内置影视资源、学术文献、技术文档、新闻资讯、数据接口等默认分类，用户亦可自定义扩展分类体系。

**字段扩展与标签系统**：每条链接可附加自定义字段（如清晰度、语种、文件格式、访问日期），并支持多标签交叉筛选。

**快速检索与过滤器**：基于链接标题、分类、标签、收录批次的多条件组合检索，结果以列表或卡片视图呈现。

**链接可用性检测**：集成简单的 HTTP 状态检测模块，可批量检查已收录链接的可访问性并标记异常状态。

**Markdown 格式数据导出**：支持将索引数据导出为结构化的 Markdown 表格或列表，便于嵌入技术文档、研究报告或项目 README。

**本地化离线运行**：所有数据存储在本地 SQLite 数据库中，无需外部网络服务即可完整使用全部功能。

## 应用场景

**个人知识库构建**：研究人员或技术爱好者可将 WebIndexer 作为外部资源采集工具，围绕特定课题（如计算机视觉、开源许可证、Web 框架演进）持续收集并标注相关链接，形成可检索的个人知识外脑。

**团队技术周报素材库**：技术团队可使用 WebIndexer 统一收录每周发现的优质博客、工具站点、开源项目，按周批次录入并添加简要评价，在周报撰写或内部分享时快速定位素材。

**影视与媒体资源策展**：内容编辑或媒体从业者可将 WebIndexer 用于影视资料、纪录片、访谈节目的链接收录，按主题、地区、年份等维度组织，配合标签系统实现精细化策展。

**开源项目外链附录维护**：开源项目维护者可在项目仓库中集成 WebIndexer 导出的资源列表，作为文档附录或参考资料的动态更新来源，避免硬编码链接散落在文档各处。

## 快速开始

以下步骤适用于 Linux / macOS / Windows（WSL 环境）的本地部署。

```bash
# 克隆项目仓库
git clone https://github.com/webindexer/webindexer.git
cd webindexer

# 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate   # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化数据库并运行
python scripts/init_db.py
python main.py
```

启动后终端将显示本地服务地址（默认为 http://127.0.0.1:5000），通过浏览器访问即可开始录入与管理链接资源。

## 项目结构

```
webindexer/
├── main.py                     # Flask 应用入口，注册路由与启动服务
├── config.py                   # 全局配置（数据库路径、检测超时、分页大小等）
├── requirements.txt            # Python 依赖清单（Flask, requests, markdown 等）
├── .gitignore                  # Git 忽略规则（venv, __pycache__, *.db 等）
│
├── core/                       # 核心业务逻辑层
│   ├── indexer.py              # 索引记录的增删改查、分页与排序逻辑
│   ├── classifier.py           # 分类与标签管理模块（含自定义字段校验）
│   ├── checker.py              # HTTP 链接可用性检测实现（含重试与超时策略）
│   └── exporter.py             # Markdown / JSON / CSV 格式导出引擎
│
├── web/                        # Web 界面与 API 层
│   ├── routes/                 # 路由蓝图划分
│   │   ├── index.py            # 首页与统计看板路由
│   │   ├── records.py          # 记录管理（列表、详情、编辑、删除）
│   │   ├── batches.py          # 批次管理（新建、合并、删除批次）
│   │   └── export.py           # 导出与下载接口
│   ├── templates/              # Jinja2 模板文件
│   │   ├── layout.html         # 基础页面骨架
│   │   ├── index.html          # 统计总览与最近收录
│   │   ├── records.html        # 记录列表与检索界面
│   │   ├── record_edit.html    # 单条记录的编辑表单
│   │   └── export.html         # 导出选项与预览
│   └── static/                 # CSS / JavaScript / 图标资源
│       ├── style.css           # 全局样式（暗色主题适配）
│       └── app.js              # 前端交互（检索、分页、批量操作）
│
├── scripts/                    # 辅助脚本与工具
│   ├── init_db.py              # 初始化 SQLite 数据库表结构
│   ├── import_batch.py         # 从 CSV / JSON 批量导入链接
│   └── check_all_links.py      # 命令行下全量检测链接状态
│
├── tests/                      # 单元测试与集成测试
```

## 常见问题

**Q：WebIndexer 是否支持在线自动抓取链接标题或元数据？**

A：当前版本不包含自动抓取功能。我们坚持人工标注优先的设计原则，以确保每条链接的分类、标签和备注都经过人为判断，避免自动抓取带来的错误分类或冗余信息。用户可在录入表单中手动填写标题、描述和自定义字段。未来版本可能提供可选的元数据自动补全插件，但默认保持离线可控。

**Q：如果收录的链接失效或域名变更，系统如何处理？**

A：WebIndexer 提供按需执行的链接可用性检测功能，用户可手动触发单条或批量检测。检测结果会更新记录的 status 字段（正常 / 异常 / 超时），但不会自动删除记录。建议用户定期（如每月）运行 scripts/check_all_links.py 检查整体状况，对于异常记录可选择人工复核后标记为失效或移除。系统不提供自动跳转或重写逻辑，以保证数据真实性。

**Q：能否在多台机器之间同步索引数据？**

A：由于数据存储在本地 SQLite 文件中，多机同步需借助外部文件同步工具（如 rsync、Syncthing、网盘同步目录）。WebIndexer 本身不提供云端同步服务。对于团队协作场景，建议将 data/webindexer.db 文件置于版本控制系统忽略的共享存储中，或使用 export 功能定期导出为 Markdown / JSON 格式后通过 Git 仓库共享。未来版本将考虑引入基于 Git 的增量同步机制。
