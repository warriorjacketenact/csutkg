# ResourceBridge

ResourceBridge 是一个面向技术团队与内容运营人员的结构化外链资源管理与导航系统。该项目定位于解决分散在不同平台、缺乏统一维护入口的技术资源链接的集中化管理问题，通过标准化的目录分类与版本化记录方式，帮助团队建立可维护、可追溯的外部知识资产库。

目标用户包括技术文档撰写者、开源项目维护者、技术社区运营人员以及需要长期跟踪特定领域信息源的研究人员。ResourceBridge 不提供爬取或自动更新功能，而是以人工精选与定期审核为核心理念，确保每条资源的可用性与分类准确性。

## 功能概览

批量链接导入与去重校验：支持从文本文件、CSV 或直接粘贴的 URL 列表中批量导入链接，自动进行格式规整与重复检测。

多级分类标签系统：允许用户为每条资源自定义主分类与子分类，并附加业务场景、语言、地域等标签，便于后续筛选。

链接可用性健康检查：内置简单的 HTTP 状态码检测模块，可定期对已入库的链接进行可达性测试，标记异常链接。

资源变更历史记录：每次对链接标题、分类或状态的修改均记录操作人与时间戳，支持回溯变更日志。

Markdown 格式数据导出：将当前资源库导出为符合本 README 结构的 Markdown 文档，直接用于项目文档或站点导航页。

外部引用关系标注：支持为每个链接标注引用来源与推荐优先级，便于团队内部评审时快速定位高价值资源。

权限分级管理：区分管理员、编辑者、只读访客三种角色，控制资源增删改与健康检查触发权限。

全文检索与快速过滤：基于标题、分类标签、原始 URL 关键词的轻量级搜索，配合分类树筛选，提升大型资源库的查找效率。

## 应用场景

技术文档团队维护外部参考链接库：当团队撰写技术方案或故障复盘报告时，需要引用大量外部文档、标准规范或社区讨论帖。ResourceBridge 可作为统一的引用源管理后台，避免每次重新搜索或使用个人书签夹。

开源项目 README 资源汇总页生成：开源项目维护者可以将项目相关的生态工具、学习材料、镜像站点等链接录入系统，定期自动生成格式统一的资源列表章节，减少手动维护成本。

技术社区内容运营的内容素材库管理：社区运营人员可将待翻译的文章、合作方站点、活动报名链接等按主题分类存储，并与编辑团队共享访问权限，提升协作效率。

企业内部技术雷达与工具链导航：中小型技术团队可将常用的开发工具镜像、内部系统入口、云服务控制台链接集中配置，配合健康检查及时发现地址变更或服务下线。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run start
```

执行完上述三步后，服务默认监听 3000 端口。访问 http://localhost:3000 即可进入资源管理面板。首次启动会自动创建示例数据与管理员账户，初始密码在控制台日志中输出，请及时修改。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x LTS 或更高 | 是 | 运行时环境，推荐使用 nvm 管理版本 |
| npm 9.x 或更高 | 是 | 依赖管理工具，随 Node.js 一同安装 |
| SQLite 3 | 是 | 内置轻量级数据库，无需额外安装，用于存储资源元数据 |
| Git 2.30 或更高 | 否 | 仅在使用版本化数据迁移功能时需要 |
| 系统内存 512 MB 以上 | 是 | 生产环境建议 1 GB 以上，确保健康检查并发性能 |
| 可访问外网的网络环境 | 否 | 链接健康检查功能需要出站网络权限，内网部署时可禁用该模块 |

## 项目结构

│   ├── core/                      # 资源管理核心逻辑
│   │   ├── importer.js            # 批量链接导入与去重
│   │   ├── classifier.js          # 分类树维护与标签匹配
│   │   └── health-check.js        # HTTP 健康检查调度器

│   │   ├── routes/                # 按资源类型划分的路由定义

│   │   ├── pages/                 # 资源列表、详情、导入页面
│   │   └── components/            # 分类筛选器、表格、搜索框
│   ├── exporters/                 # 数据导出模块
│   │   └── markdown.js            # 生成 Markdown 格式资源清单
│   └── config/                    # 环境变量与启动配置
├── docs/                          # 完整文档目录
│   ├── user-guide/                # 用户手册章节

## 贡献指南

1. 查阅 docs/developer-guide/ 中的开发环境搭建文档，确保本地 Node.js 版本与依赖满足要求，并运行 npm install 安装所有开发依赖。

2. 在 Issue 列表中选择未被指派的 bug 或 feature 请求，或新建 Issue 描述你希望解决的问题，等待维护者确认后再开始编码。

3. 从 main 分支切出新的功能分支，分支命名遵循 feature/功能简述 或 fix/问题简述 格式，提交信息使用英文祈使句，保持简洁明确。

4. 完成代码或文档修改后，运行 npm test 确保所有现有测试通过，并为新增功能添加对应的单元测试或集成测试用例。

5. 提交 Pull Request 到 main 分支，在 PR 描述中关联对应的 Issue 编号，并简要说明改动范围与测试覆盖情况。PR 需要通过至少一位维护者的 Code Review 后方可合并。

## 常见问题

Q: 导入大量链接时页面响应缓慢，如何优化批量操作性能？

A: 当单次导入超过 500 条链接时，建议使用 scripts/batch-import.js 命令行工具进行后台导入，避免 HTTP 请求超时。同时可调整环境变量中的 BATCH_SIZE 与 IMPORT_TIMEOUT 参数。生产环境可配合消息队列将导入任务异步化。

Q: 健康检查模块报告大量链接超时，但浏览器中可以正常访问，是什么原因？

A: 健康检查模块默认使用 HEAD 请求并设置 5 秒超时。部分服务器对 HEAD 请求响应较慢或直接拒绝，可在配置文件中将检查方法改为 GET，并适当延长超时时间。另外，请确保部署环境的外网访问策略未限制出站请求。

Q: 导出 Markdown 时生成的链接列表包含很多无关 URL，如何控制导出范围？

A: 在导出页面的筛选条件中，可按分类、标签、创建时间范围或最后检查状态进行过滤。同时支持在导出配置中指定排除特定分类或仅导出通过健康检查的链接，以满足不同场景的文档生成需求。
