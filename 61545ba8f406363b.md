# ResourceBridge

ResourceBridge 是一个面向技术内容聚合与外部资源导航的开源工具集，定位为技术团队、独立开发者与内容研究者的外部资源整理与统一浏览入口。项目核心解决以下问题：技术文档、视频教程、参考案例散落在不同平台，难以统一检索与结构化呈现；自建资源站需要频繁处理 URL 抓取、元信息提取与列表渲染，重复劳动成本高。ResourceBridge 提供标准化数据管道与展示模板，将外部链接列表转化为可维护、可扩展、可二次分发的静态站点内容。

## 功能概览

**批量链接导入与标准化**：支持从纯文本列表、CSV 或 JSON 批量导入外部 URL，自动解析域名、路径与参数，生成统一格式的资源条目。

**自动元信息补全**：对导入的每个链接尝试拉取页面标题、摘要描述与最后修改时间，减少手动注释工作量。

**分类标签系统**：允许为每个资源条目标记一个或多个分类标签，支持按标签动态生成分类视图。

**静态站点生成器集成**：内置针对 VuePress、Hugo 与 MkDocs 的内容格式转换器，可将资源列表输出为对应框架的导航配置文件或 Markdown 索引文件。

**资源状态监控**：定期检查已收录链接的可访问性，对返回 4xx 或 5xx 状态的链接发出标记提醒，便于运维人员及时清理或更新失效资源。

**全文检索支持**：基于标题与摘要字段生成简易全文索引，满足站内快速搜索需求。

**版本化导出**：每次全量导出自动生成时间戳版本号，支持回滚至任意历史导出状态。

## 应用场景

**技术团队内部知识库外链管理**：团队在维护内部技术文档时，需要频繁引用外部视频教程、官方规范与社区讨论帖。ResourceBridge 可将这些散落链接统一收录并按主题分类，最终生成一个外链汇总页面，嵌入团队 Wiki 或 Confluence。

**开源项目官方资源导航页建设**：开源项目通常需要维护一个资源导航页，列出相关教程、生态工具与案例演示。ResourceBridge 允许项目维护者以纯文本形式维护链接列表，自动生成结构化的导航 Markdown 文件，直接用于项目 docs 目录或 GitHub Pages。

**技术内容创作者选题素材库构建**：技术博主或视频创作者在调研阶段需要收集大量参考材料。使用 ResourceBridge 可以将调研过程中的临时链接全部收录，后期统一添加备注、分类与优先级标记，形成可复用的选题素材库。

**离线文档镜像站点前置处理**：部分内部网络环境需要定期同步外部技术资源。ResourceBridge 可作为前置处理工具，将待镜像的 URL 清单进行格式清洗与分类整理，再交给下游下载或抓取工具执行具体镜像任务。

## 快速开始

以下步骤在 Linux / macOS / Windows WSL 环境下均可执行。

```bash
# 克隆项目仓库
git clone https://github.com/resource-bridge/resource-bridge.git
cd resource-bridge

# 安装依赖（使用 pip 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行示例导入任务（将 data/sample_links.txt 中的 URL 导入并生成导航页）
python bridge.py --input data/sample_links.txt --output docs/navigation.md --format markdown
```

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.8+ | 是 | 核心运行环境，所有脚本基于 Python 开发 |
| pip 21.0+ | 是 | 用于安装 requirements.txt 中声明的第三方库 |
| requests 2.28+ | 是 | 用于发起 HTTP 请求，拉取链接元信息 |
| beautifulsoup4 4.11+ | 是 | 用于解析 HTML 页面，提取标题与摘要 |
| lxml 4.9+ | 否 | 作为 beautifulsoup4 的备选解析器，可提升解析速度，建议安装 |
| markdown 3.4+ | 否 | 仅在输出 Markdown 格式时需要 |
| pyyaml 6.0+ | 否 | 仅在输出 Hugo / MkDocs 前端配置时需要 |
| pytest 7.0+ | 否 | 仅开发测试阶段需要，生产环境可不安装 |

## 项目结构

项目目录结构及各模块职责说明如下：

```
resource-bridge/
├── bridge.py                     # 主入口脚本，解析命令行参数并调度核心流程
├── requirements.txt              # Python 依赖声明文件
├── pyproject.toml                # 项目元信息与构建配置（PEP 621）
├── README.md                     # 项目说明文档（本文件）
├── LICENSE                       # MIT 许可证文本
│
├── src/                          # 核心源码目录
│   ├── __init__.py               # 包初始化
│   ├── importer/                 # 输入解析模块
│   │   ├── __init__.py
│   │   ├── base.py               # 定义抽象导入器基类
│   │   ├── text_parser.py        # 纯文本列表解析器（每行一个 URL）
│   │   ├── csv_parser.py         # CSV 格式解析器，支持自定义列映射
│   │   └── json_parser.py        # JSON 数组或对象解析器
│   ├── fetcher/                  # 元信息拉取模块
│   │   ├── __init__.py
│   │   ├── http_client.py        # 封装 requests 会话，配置超时与重试
│   │   ├── metadata_extractor.py # 使用 BeautifulSoup 提取 title 与 meta description
│   │   └── cache.py              # 本地缓存已拉取的元信息，避免重复请求
│   ├── exporter/                 # 输出格式转换模块
│   │   ├── __init__.py
│   │   ├── base.py               # 定义抽象导出器基类
│   │   ├── markdown_exporter.py  # 输出为 Markdown 索引表格
│   │   ├── hugo_exporter.py      # 输出为 Hugo 导航页 front matter + 正文
```

## 常见问题

Q: 导入的链接数量很大（数千条），元信息拉取速度很慢怎么办？

A: 元信息拉取模块默认启用缓存机制，第二次运行时相同 URL 不会重复请求。首次运行时建议使用 --delay 参数控制请求间隔，避免被目标站点限流。对于大批量任务，可先关闭元信息拉取（--no-fetch）只做链接整理，后续单独运行 fetch 子命令补充元信息。

Q: 输出的 Markdown 表格中，部分链接标题为空或乱码，如何解决？

A: 标题为空通常因为目标页面未设置 title 标签或页面需要 JavaScript 渲染。ResourceBridge 默认只解析静态 HTML，对于 JS 渲染页面可尝试在 fetcher/http_client.py 中启用 Selenium 或 Playwright 驱动（需额外安装），但会显著降低处理速度。乱码问题通常由页面编码声明不准确导致，可在 metadata_extractor.py 中显式指定 encoding='utf-8' 或 'gbk'，并根据实际页面调整。

Q: 如何定期自动执行链接状态检查并发送报告？

A: 项目未内置定时调度功能，但 monitor/checker.py 可独立调用。推荐使用系统 crontab（Linux）或 Task Scheduler（Windows）定时运行以下命令：python bridge.py --check-only --input data/links.txt --report reports/status_$(date +%Y%m%d).json。该命令只执行检查不触发导出，生成的报告可后续通过 reporter.py 转换为 Markdown 或 HTML 格式，便于邮件或企业微信机器人推送。
