# LinkVault 聚合资源导航系统

LinkVault 是一个面向技术团队、开源社区与个人开发者的轻量级聚合资源导航系统。该项目并非传统意义上的网络爬虫或链接检测工具，而是一个以人工精选与结构化整理为核心的外链资源汇总平台，旨在解决信息过载时代下高质量技术资料与多媒体内容难以被系统化发现、分类与复用的问题。

LinkVault 的目标用户包括技术文档维护者、社区运营人员、RAG 知识库构建者以及需要定期批量管理外部链接资源的开发团队。系统提供清晰的多级分类视图、资源状态标记、访问频率统计与失效链接检测接口，帮助用户从杂乱的链接集合中快速筛选出可用、相关且稳定的信息源。项目本身不存储任何第三方内容，仅保留元数据与结构化索引，符合开源合规要求。

## 功能概览

- **多级分类索引**：支持按主题、来源域名、文件类型与预期用途对链接进行标签化分组，每条资源可归属多个分类维度。

- **批量导入与去重**：提供 CSV 与 JSON 格式的链接批量导入接口，自动识别重复条目并生成合并建议报告。

- **可用性健康检查**：内置异步 HTTP 状态码检测模块，可周期性验证链接可达性，并标记响应超时或 4xx/5xx 状态的记录。

- **元数据自动补全**：通过响应头与内容片段推测资源类型、内容编码与最后修改时间，减少人工录入工作量。

- **全文检索与过滤**：基于 URL 路径关键词、分类标签与自定义备注字段提供快速过滤能力，支持正则表达式高级查询。

- **访问热度统计**：记录每个链接的查询次数与最近访问时间，辅助判断资源的实际使用频率与价值。

- **导出与集成**：支持将筛选结果导出为纯 URL 列表、Markdown 列表或结构化 JSON 文件，便于嵌入其他文档或流水线。

- **扩展钩子机制**：允许用户编写自定义 Python 或 Shell 脚本，在链接状态变更或新增时触发通知、同步或二次处理逻辑。

## 应用场景

**技术文档站点的外链管理**：开源项目维护者可使用 LinkVault 统一管理 README、Wiki 或官网中引用的所有外部参考链接，定期检测失效引用并批量更新，避免文档中出现死链。

**RAG 知识库的数据源编排**：AI 应用开发团队在构建检索增强生成系统时，需从大量候选网页中筛选稳定的语料来源。LinkVault 提供分类与健康检查能力，帮助团队将可信链接推送至向量数据库。

**社区资源周报自动生成**：技术社区运营人员可将每日收集的链接批量导入系统，按周导出带分类标记的 Markdown 列表，直接作为社区周报的内容素材，减少手动排版成本。

**个人开发者的书签库重构**：长期积累的浏览器书签难以维护。LinkVault 提供去重、分类与失效检测功能，帮助开发者将零散书签重构为结构清晰、可检索的技术参考库。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，要求已安装 Git 与 Python 3.10 及以上版本。

```bash
git clone https://github.com/example/linkvault.git
cd linkvault
pip install -r requirements.txt
python linkvault.py --init --import ./samples/links.json
```

执行完毕后，系统将在本地 8080 端口启动 Web 管理界面，同时生成 `data/vault.db` SQLite 数据库文件与 `logs/` 运行日志目录。首次运行会自动创建默认分类标签与管理员账户，初始密码输出在终端日志中。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 ~ 3.12 | 核心运行环境，低于 3.10 将无法使用异步语法 |
| SQLite | 3.35 及以上 | 内嵌数据库，用于存储链接元数据与分类关系 |
| aiohttp | 3.9.0 及以上 | 异步 HTTP 客户端，用于并发健康检查 |
| beautifulsoup4 | 4.12.0 及以上 | HTML 解析器，用于自动补全页面标题与编码信息 |
| lxml | 4.9.0 及以上 | 高性能 XML/HTML 解析后端，bs4 的推荐依赖 |
| click | 8.1.0 及以上 | 命令行交互框架，提供子命令解析与参数校验 |
| jinja2 | 3.1.0 及以上 | Web 管理界面的模板渲染引擎 |
| pytest | 7.4.0 及以上 | 单元测试与集成测试框架（仅开发环境需要） |

## 贡献指南

1. 查阅问题列表与项目看板，优先选择标记为 `good-first-issue` 或 `help-wanted` 的任务，避免与他人并行冲突。建议在开始工作前在对应 issue 下评论说明认领意图。

2. 派生项目仓库至个人账户，基于 `main` 分支创建功能分支，分支命名遵循 `feature/` 或 `fix/` 前缀加简要描述，例如 `feature/add-json-export`。

3. 编写代码时遵循 PEP 8 风格规范，核心模块需包含 docstring 类型注解。新增功能应补充对应单元测试，确保测试覆盖率达到 85% 以上。提交前运行 `pytest tests/` 验证所有用例通过。

4. 提交信息采用约定式提交格式：`<type>(<scope>): <subject>`，其中 type 包含 feat、fix、docs、refactor、test、chore 等类别。提交历史应保持线性，避免无意义的合并提交。

5. 发起 Pull Request 至 `main` 分支，描述中需说明变更动机、实现方式与测试结果。核心维护者将在三个工作日内进行 Review，并根据反馈进行修改。合并后自动触发 CI 流水线完成打包与文档更新。

## 常见问题

**Q：系统能否处理包含中文或特殊字符的 URL？**

A：LinkVault 在存储与输出时保留 URL 原始编码，不进行自动转义或解码。健康检查模块使用 aiohttp 的严格 URL 校验模式，若原始链接包含未转义的特殊字符，会记录校验失败并跳过检测，用户可在导入前使用标准库 urllib.parse.quote 进行预处理。建议在导入前对包含非 ASCII 字符的链接进行百分号编码。

**Q：健康检查模块是否会因请求频率过高而被目标服务器封禁？**

A：系统默认启用分布式延迟策略，每次请求前插入 100 至 500 毫秒随机间隔，且并发数限制为 20。用户可在配置文件中调整 `checker.interval_min` 与 `checker.max_concurrent` 参数以适配不同目标站点的耐受程度。对于已知敏感站点，可通过分类标签单独排除健康检查。

**Q：如何将现有浏览器书签批量导入 LinkVault？**

A：主流浏览器支持将书签导出为 HTML 文件。用户可使用 `scripts/import_from_csv.py` 脚本，首先将 HTML 书签转换为两列（URL, 标题）CSV 格式，再执行导入命令。系统会自动识别重复链接并生成冲突报告。若需保留原始文件夹分类，可开启 `--preserve-category` 参数将文件夹路径映射为标签。
