# LinkVault 资源聚合系统

LinkVault 是一个面向技术团队与内容运营者的轻量级外链资源汇总平台，专注于将分散于各处的优质技术文档、视频教程、开源工具与社区讨论进行集中式索引与分类管理。该项目并非传统意义上的爬虫或采集系统，而是一套人工精选与半自动化标记相结合的链接治理方案，帮助用户以极低维护成本建立私有或团队共享的知识导航体系。

项目定位为技术资源的中转枢纽，解决开发者在日常学习与项目推进中面临的“链接散乱、遗忘丢失、回顾困难”等核心痛点。LinkVault 提供统一的数据结构、标签体系与检索接口，让每一条外链都具备可追溯、可分类、可备注的完整元数据，从而将零散的浏览行为转化为可持续积累的知识资产。

## 功能概览

**链接快照存档** 支持对每个收录的 URL 自动抓取页面标题、摘要与发布时间，形成静态快照记录，避免原始内容下线后无法追溯。

**多维标签体系** 允许用户为每条资源打上多个自定义标签，并支持按标签组合进行布尔检索，实现精细化的内容筛选。

**批量导入导出** 提供 CSV 与 JSON 格式的批量链接导入接口，同时支持按筛选结果导出资源清单，便于团队内部分享或迁移。

**定期可用性检测** 内置定时巡检任务，自动检测已收录链接的 HTTP 状态码与响应时长，对失效链接进行高亮标记并生成异常报告。

**备注与评分系统** 每条记录支持附加个人或团队的阅读笔记、重要程度评分（1-5 星）以及关联资源推荐，形成轻量级的知识评价网络。

**全文检索与模糊匹配** 基于倒排索引实现链接标题、描述与备注字段的快速全文搜索，支持拼音首字母模糊匹配，提升中文资源的查找效率。

**权限与协作控制** 提供基于角色的访问控制（RBAC），区分管理员、编辑者与只读访客三种权限等级，适配团队内部的知识库共建需求。

## 应用场景

技术团队内部文档沉淀：开发团队可将日常调研中遇到的优秀博客、API 手册、架构设计文档统一录入 LinkVault，并在每周技术评审时通过标签筛选快速回顾本周积累的关键资源，减少重复搜索时间。

开源项目学习路线规划：个人开发者围绕某一技术栈（如 Rust 异步编程、Kubernetes 调度机制）搜集相关视频课程、官方示例与社区最佳实践，利用 LinkVault 的备注字段记录学习心得与阅读顺序，形成结构化的学习路径。

运维故障案例库建设：运维团队可将每次线上事故后参考的解决方案链接、官方修复公告、社区讨论帖集中收录，并关联故障时间、影响范围与恢复步骤，逐步构建可检索的故障知识库。

产品竞品动态追踪：产品经理与市场运营人员定期收集竞品发布公告、行业分析报告与用户反馈帖子，通过评分系统标注重要程度，辅助版本迭代决策与市场策略调整。

## 快速开始

以下命令适用于 Linux/macOS 环境，Windows 用户建议通过 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化 SQLite 数据库并导入默认标签模板
python manage.py init_db
python manage.py load_tags

# 启动开发服务器，默认监听 127.0.0.1:5000
python manage.py runserver
```

访问 http://127.0.0.1:5000 即可进入本地部署的 LinkVault 实例，首次使用请按页面指引创建管理员账户。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 - 3.12 | 核心运行环境，推荐使用 3.11 以获得最佳性能 |
| SQLite | 3.35.0 及以上 | 默认嵌入式数据库，无需额外安装，生产环境可切换至 PostgreSQL |
| Flask | 2.3.x | Web 框架，提供路由、请求处理与模板渲染能力 |
| APScheduler | 3.10.x | 定时任务调度器，用于执行链接可用性检测等后台任务 |
| Whoosh | 2.7.x | 纯 Python 实现的全文检索引擎，支撑搜索功能的核心组件 |
| requests | 2.31.x | HTTP 客户端库，用于快照抓取与状态检测 |
| pytz | 2024.1 | 时区处理工具，确保定时任务与日志时间的正确性 |
| markdown | 3.5.x | 将备注字段中的 Markdown 内容渲染为 HTML 展示 |
| gunicorn | 21.2.x | 生产环境推荐的 WSGI 服务器（Linux 部署必需） |

## 项目结构

```
linkvault/
├── app/
│   ├── __init__.py                 # 应用工厂函数，创建 Flask 实例并注册扩展
│   ├── config.py                   # 环境配置（开发/测试/生产）及敏感参数读取
│   ├── models/
│   │   ├── __init__.py             # 模型基类与数据库会话管理
│   │   ├── link.py                 # Link 表定义：url、title、description、status_code
│   │   ├── tag.py                  # Tag 表定义：name、color、group
│   │   ├── snapshot.py             # Snapshot 表定义：content_hash、archive_path、captured_at
│   │   └── user.py                 # User 表定义：username、role、password_hash
│   ├── services/
│   │   ├── fetcher.py              # 快照抓取服务：requests 请求、HTML 解析、摘要生成
│   │   ├── checker.py              # 可用性检测服务：并发 HTTP 请求、超时控制、结果持久化
│   │   ├── indexer.py              # 全文索引服务：Whoosh 索引更新、查询解析、结果排序
│   │   └── exporter.py             # 批量导出服务：CSV/JSON 流式写入、字段映射
│   ├── routes/
│   │   ├── main.py                 # 主页面、仪表盘、全局搜索路由
│   │   ├── links.py                # 链接增删改查、标签关联、详情页路由
│   │   ├── tags.py                 # 标签管理、合并、重命名路由
│   │   └── admin.py                # 用户管理、系统设置、巡检日志路由
│   ├── templates/                  # Jinja2 模板文件（主布局、列表页、表单页）
│   ├── static/                     # CSS 样式表、JavaScript 脚本、图标字体
│   └── utils/
│       ├── validators.py           # URL 格式校验、标签名合法性检查
│       └── helpers.py              # 日期格式化、分页计算、字符串处理工具
├── tests/                          # 单元测试与集成测试（pytest 框架）
├── scripts/
│   ├── init_db.py                  # 数据库初始化与默认标签种子数据
│   └── migrate_sqlite_to_pg.py     # SQLite 到 PostgreSQL 的数据迁移脚本
├── docker/
│   ├── Dockerfile                  # 多阶段构建，基于 python:3.11-slim
│   └── docker-compose.yml          # 含 PostgreSQL、Redis 的可选生产组合
├── docs/                           # 完整文档目录（详见上文文档导航）
```

## 常见问题

Q: LinkVault 是否可以自动抓取需要登录或带有反爬机制的页面？
A: 目前 LinkVault 仅支持公开可访问的静态页面抓取，对于需要 Cookie 认证或 JavaScript 动态渲染的站点，系统会跳过快照生成并记录为“受限资源”。您可以通过备注字段手动补充关键信息，或使用浏览器插件将页面内容导出为静态 HTML 后手动导入。

Q: 巡检任务发现失效链接后，系统会自动删除该记录吗？
A: 不会。系统仅将链接状态标记为“不可用”并在仪表盘生成警告通知，保留原始数据与历史快照。管理员可批量筛选失效链接并决定是否重新检测、手动更新 URL 或归档删除。此设计避免因临时网络波动导致的误删。

Q: 如何将现有书签（如 Chrome 导出的 HTML 书签文件）迁移至 LinkVault？
A: 项目提供了独立的迁移脚本 `scripts/import_bookmarks.py`，可解析 Chrome、Firefox 或 Edge 导出的标准书签 HTML 文件，自动提取 URL、标题与文件夹层级作为标签。执行命令 `python scripts/import_bookmarks.py --file bookmarks.html --user admin` 即可完成批量导入。
