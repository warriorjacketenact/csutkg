# StreamLink 技术资源导航

StreamLink 是一个面向开发者和技术研究人员的轻量级外链资源导航系统，专注于聚合、分类与快速检索各类技术文档、在线工具、API 参考与开发社区资源。项目定位为“技术资源的统一入口”，帮助用户在不记忆大量分散 URL 的前提下，通过统一的索引结构与简明标签系统，高效定位所需外部资源。

StreamLink 不存储任何第三方内容，仅提供结构化链接索引与基础元信息描述。项目适用于个人开发者、小型团队内部知识库建设，以及开源社区文档站点的外链增强模块。通过静态化部署与极简依赖，StreamLink 可在任意标准 Web 服务器或云存储环境下运行，无需数据库或后端运行时支持。

## 功能概览

**多维度标签分类**：支持按技术领域、文档类型、语言版本等维度对每条外链进行标注，支持多标签组合筛选。

**可定制索引表**：用户可通过 YAML 配置文件自定义资源分类、排序权重与显示名称，无需修改核心代码。

**快速模糊检索**：内置基于标题与标签的本地模糊搜索，响应时间低于 200 毫秒，支持中英文混合查询。

**静态站点生成**：项目提供构建脚本，可将配置数据与模板编译为纯静态 HTML 文件，便于 CDN 分发与版本管理。

**响应式展示布局**：前端界面基于 CSS Flexbox 与 Grid 实现，适配桌面、平板与移动设备，保证各类屏幕下的可读性。

**外部资源状态检测**：可选集成 HTTP HEAD 请求检测模块，定时检查链接可用性，并在界面中标记异常状态。

**开放数据导出**：支持将索引数据导出为 JSON 或 CSV 格式，便于导入其他知识管理工具或进行离线分析。

**低维护成本**：项目不依赖数据库、缓存或任务队列，每月仅需少量时间更新配置文件即可保持内容新鲜度。

## 应用场景

**技术团队内部文档导航**：开发团队可将 StreamLink 部署为内部知识库的入口页面，集中存放常用设计文档、API 规范、测试环境地址与运维手册链接，减少团队成员在浏览器书签中翻找的时间。

**开源项目文档站外链整合**：开源项目维护者可在项目文档站点中嵌入 StreamLink 模块，统一列出相关生态工具、社区论坛、视频教程与第三方扩展仓库，提升文档站点的信息密度与实用性。

**技术培训与教育资料索引**：培训机构或高校实验室可使用 StreamLink 快速搭建课程资源导航页，按课程模块组织在线实验环境、视频回放地址、习题集与参考阅读材料，方便学员课后自学。

**个人技术知识库外链管理**：技术博主或独立研究者可利用 StreamLink 管理个人技术阅读列表，按主题分类收藏深度文章、工具站点与论文预印本链接，并借助检索功能快速回顾特定方向的历史资料。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，要求已安装 Git 与 Node.js 18.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/streamlink/streamlink-index.git
cd streamlink-index

# 安装依赖（使用 npm）
npm install

# 运行开发服务器，默认监听端口 3000
npm run dev
```

执行上述命令后，在浏览器中访问 http://localhost:3000 即可查看默认索引页面。如需构建生产环境静态文件，请执行 `npm run build`，构建产物默认输出至 `dist` 目录。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，用于执行构建脚本与开发服务器 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖 |
| Git | 2.30 及以上 | 版本控制工具，用于克隆仓库与管理配置变更 |
| 现代浏览器 | 最近两个主要版本 | 前端运行时，支持 ES2020 与 CSS Grid 特性 |
| 静态 Web 服务器 | 任意（如 Nginx、Apache、Caddy） | 生产环境部署时用于托管静态文件，非开发必需 |
| 内存 | 最低 512 MB | 构建过程内存占用，推荐 1 GB 以上 |
| 磁盘空间 | 最低 100 MB | 包含源码、依赖与构建产物 |
| 操作系统 | Linux / macOS / Windows WSL2 | 主开发与部署环境，Windows 原生 PowerShell 未完全测试 |
| 网络访问 | 外网可访问 | 用于首次安装依赖及检测外部链接状态（可选功能） |

## 项目结构

```
streamlink-index/
├── config/                           # 配置文件目录
│   ├── default.yaml                  # 默认索引配置，包含分类与标签定义
│   └── custom.yaml                   # 用户自定义覆盖配置（不提交至版本库）
├── src/                              # 核心源代码
│   ├── builder/                      # 静态站点构建引擎
│   │   ├── index.js                  # 主构建流程，整合数据与模板
│   │   └── transformers/             # 数据转换管道（YAML -> JSON -> HTML）
│   ├── server/                       # 开发服务器模块
│   │   ├── dev.js                    # 基于 Express 的本地热重载服务
│   │   └── middleware/               # 请求日志与静态缓存中间件
│   ├── frontend/                     # 前端资源（JavaScript / CSS）
│   │   ├── search.js                 # 本地模糊搜索实现
│   │   ├── filter.js                 # 标签筛选与组合逻辑
│   │   └── theme/                    # 默认主题样式（深色/浅色变量）
│   ├── lib/                          # 通用工具函数
│   │   ├── validator.js              # URL 格式与配置字段校验
│   │   ├── detector.js               # 外部链接状态检测（HTTP HEAD）
│   │   └── exporter.js               # JSON / CSV 导出功能
│   └── templates/                    # 页面模板（EJS）
│       ├── index.ejs                 # 主索引页面布局
│       └── partials/                 # 头部、尾部、卡片等可复用片段
├── public/                           # 静态资源（图标、字体、占位图）
│   ├── icons/
│   └── fallback/
├── dist/                             # 构建输出目录（生产环境静态文件）
├── tests/                            # 单元测试与集成测试
│   ├── unit/                         # 工具函数与构建逻辑测试
│   └── integration/                  # 端到端页面渲染测试
├── docs/                             # 项目文档（详见文档导航）
├── scripts/                          # 辅助运维脚本
│   ├── update-links.sh               # 批量更新链接状态
│   └── backup-config.sh              # 配置备份与压缩
├── .github/                          # GitHub 社区配置
│   ├── ISSUE_TEMPLATE/               # 问题报告与功能请求模板
│   └── workflows/                    # CI 自动化（构建与链接检测）
├── package.json                      # npm 清单（依赖与脚本定义）
├── README.md                         # 项目说明（即本文档）
└── LICENSE                           # MIT 许可证
```

## 常见问题

**问：StreamLink 是否支持在线编辑或添加链接？**

StreamLink 定位为静态索引系统，不提供运行时在线编辑功能。所有链接变更需通过修改 `config/default.yaml` 文件后重新构建。如需在线管理能力，建议搭配 Git 仓库的 Web IDE 或使用第三方 CMS 作为上游数据源。

**问：如何自定义界面颜色与 Logo？**

您可以在 `src/frontend/theme/` 目录中修改 CSS 变量文件，覆盖 `--primary-color`、`--bg-color` 等变量。Logo 图片请替换 `public/icons/logo.svg` 文件，并确保尺寸与原始文件一致。修改后重新运行 `npm run build` 即可生效。

**问：链接状态检测显示异常，但实际网站可访问，是什么原因？**

链接检测模块默认使用 HEAD 请求并设置 3 秒超时。部分外部站点可能禁止 HEAD 请求或响应较慢，导致误报。您可以在 `src/lib/detector.js` 中调整超时参数，或切换为 GET 请求方式（需注意额外流量消耗）。建议对检测结果人工复核后再进行批量处理。
