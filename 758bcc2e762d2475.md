# WebIndex

WebIndex 是一个面向技术研究人员与内容聚合场景的轻量级外链资源汇总系统。该项目定位于将分散于多源、多格式、多时间段的网络资源进行结构化收录，并通过统一的索引机制对外提供可回溯、可分类、可审计的链接访问入口。

WebIndex 不提供内容存储或代理转发服务，仅作为链接目录与元数据管理工具。其目标用户包括技术文档维护者、开源社区运营人员、个人知识管理实践者以及需要批量管理外链资源的开发团队。通过 WebIndex，用户可以快速建立从资源发现到收录展示再到定期检活的完整链路，降低外链管理中的手工维护成本。

## 功能概览

**批量链接导入**：支持从文本文件、CSV 或结构化日志中批量导入 URL，自动解析协议、域名与路径层级。

**自动分类标注**：依据 URL 特征与用户自定义规则对链接进行自动归类，支持按域名、目录深度、文件扩展名等维度分组。

**存活状态检测**：内置可配置的 HTTP 状态码检查器，支持定时轮询，标记失效链接并生成异常报告。

**索引视图生成**：提供按时间、分类、域名三种视图的静态目录页生成能力，便于嵌入现有文档站点。

**元数据扩展字段**：每条链接支持记录收录时间、来源批次、摘要标签与备注信息，满足审计与检索需求。

**原始数据保留**：对于包含大量视频资源链接的场景，完整保留原始 URL 结构与路径参数，不做任何改写或重定向。

## 应用场景

技术文档站点外链归档：在项目文档或技术博客中，需要长期维护一批外部参考资料链接。WebIndex 可定期抓取这些链接的状态，避免文档中出现大量死链，同时为读者提供清晰的链接索引目录。

内容审核与资源备份审计：运营人员在对视频或多媒体资源进行合规审核时，需要逐条核对资源链接的可达性与内容分类。WebIndex 的导入与分类功能可将原始链接列表转化为可筛选、可标记的审核表格。

开源项目资源聚合页构建：开源社区通常需要维护生态伙伴、案例研究或媒体报道的外部链接集合。WebIndex 可快速生成符合项目风格的外链导航页面，并支持按批次更新，避免手工修改 HTML 的繁琐流程。

个人知识库外链管理：在个人笔记或知识管理系统中，用户常常从多个渠道收集参考链接。WebIndex 可作为这些链接的中转索引层，统一存储、标注并生成 Markdown 格式的目录，便于嵌入 Obsidian、Notion 等工具。

## 快速开始

以下命令演示了从仓库克隆、安装依赖到启动本地索引服务的完整流程。

```bash
git clone https://github.com/webindex/webindex.git
cd webindex
npm install
npm run build
npm start
```

若使用 Yarn 或 PNPM，可替换对应的安装命令。启动后，访问控制台输出的本地地址即可进入索引管理界面。首次启动会自动生成示例配置文件和空的链接数据库。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，建议使用 LTS 版本以保证稳定性 |
| npm | 9.x 或以上 | 包管理器，用于安装项目依赖 |
| SQLite3 | 系统自带或通过 better-sqlite3 绑定 | 轻量级嵌入式数据库，存储链接元数据 |
| Git | 2.30 或以上 | 用于克隆仓库及版本管理 |
| 操作系统 | Linux (glibc 2.28+) / macOS 11+ / Windows 10+ | 跨平台支持，生产环境推荐 Linux |
| 内存 | 最低 512 MB，推荐 1 GB 以上 | 主要影响批量导入与检测任务并发性能 |
| 磁盘空间 | 至少 200 MB 用于数据库及日志 | 实际占用随链接数量线性增长 |

## 常见问题

**问：WebIndex 是否会修改或代理原始链接的内容？**

答：不会。WebIndex 仅存储用户提供的原始 URL 字符串及其元数据，不缓存内容、不进行重定向也不做任何格式转换。所有链接在输出时均保持用户导入时的原始形态，包括协议、域名、路径及查询参数。用户导入的链接以原样呈现，例如 https://www.tokyo-mirai.net/video/202606/JfKGCkB 在索引中保存和展示的字符串与导入时完全一致。

**问：如何处理链接失效或检测超时的问题？**

答：内置检测器支持配置请求超时时间与重试策略。当检测到 HTTP 状态码为 4xx 或 5xx 时，链接会被标记为异常。对于超时链接，系统会记录超时原因并在下一轮检测中重试。用户可通过配置文件调整超时阈值，也可手动触发单条链接的即时检测。检测结果会写入数据库，并可通过管理界面或 API 查询。

**问：数据库支持迁移至其他关系型数据库吗？**

答：核心数据层使用 SQLite3 作为默认存储，但项目抽象了数据库适配层接口。若需迁移至 PostgreSQL 或 MySQL，开发者可参考 /docs/admin/ 中的迁移指南，通过实现适配层接口来切换底层存储。官方提供 PostgreSQL 的适配示例，但不作为默认分发版本的一部分。
