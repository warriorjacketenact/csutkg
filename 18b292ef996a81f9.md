# VODLink Indexer

VODLink Indexer 是一个面向技术内容聚合与外部资源索引的开源工具集，专为需要系统化整理、检索和引用海量视频详情页（VOD detail pages）的开发者与内容运营团队设计。该项目通过结构化的 URL 管理模式，将分散的外部视频资源条目统一归纳为可索引、可版本控制的链接数据集，适用于构建自定义视频门户、内容推荐系统的数据层，以及进行批量链接有效性检测与元数据抽取。

目标用户包括：从事视频平台二次开发的工程师、负责内容聚合站点的运营人员、研究网络资源索引架构的学术开发者，以及需要定期同步大量外部视频详情页链接的自动化脚本编写者。VODLink Indexer 不提供视频播放或内容解析功能，专注于链接数据的组织、存储与基础校验，确保资源列表的完整性和可追溯性。

## 功能概览

**批量链接导入**：支持从纯文本文件、CSV 或直接粘贴的 URL 列表中批量导入 VOD 详情页链接，自动去重并校验 URL 格式。

**结构化索引存储**：将每条链接与自定义标签、导入批次（第 718/1241 批）、时间戳和校验状态关联，存储于轻量级 SQLite 数据库中，便于后续检索。

**链接状态检测**：内置 HTTP 请求模块，可定时检测每条链接的可访问性，返回状态码与响应时间，标记失效或重定向链接。

**批次管理与回滚**：每一批次导入的链接均作为独立版本记录，支持按批次查看、导出或回滚删除，方便维护大规模链接库。

**元数据扩展字段**：为每条链接预留可选的元数据字段（如标题占位、分类、优先级），支持用户通过外部脚本批量填充。

**导出与集成**：支持将索引数据导出为 JSON、CSV 或纯 URL 列表格式，便于与其他数据处理流水线或前端展示系统集成。

## 应用场景

视频平台内容迁移与备份：当运营团队需要将旧版视频详情页迁移至新系统时，可使用 VODLink Indexer 批量导入原始链接，生成带批次标记的索引文件，再配合脚本逐个抓取或重定向，确保迁移过程中无链接遗漏。

每日链接健康巡检：内容聚合站点可配置定时任务，每日凌晨调用 VODLink Indexer 的状态检测功能，扫描所有索引链接，生成失效链接报告并发送至运维邮箱，大幅减少手动检查时间。

学术研究中的资源列表标准化：研究网络视频资源分布或链接存活率的学者，可将从不同渠道收集的 VOD 链接统一导入本工具，获得格式一致的索引数据集，并利用导出功能生成符合学术论文附录要求的链接清单。

多团队协作的共享链接库：内容编辑、开发与测试团队可通过共享同一个索引数据库，随时查询特定批次或特定域名的所有 VOD 链接，避免重复录入和链接冲突，提升协作效率。

## 快速开始

以下步骤帮助您在本地环境中快速部署并运行 VODLink Indexer 的基础功能。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/vodlink-indexer.git

# 进入项目根目录
cd vodlink-indexer

# 安装所需依赖（使用 pip 安装 Python 依赖包）
pip install -r requirements.txt

# 运行初始化脚本，创建 SQLite 数据库与默认配置
python scripts/init_db.py

# 执行示例导入任务（将示例 URL 列表导入索引）
python cli.py import --batch 718 --file samples/url_list.txt

# 启动本地 Web 查询界面（可选，默认端口 8080）
python app.py --port 8080
```

## 常见问题

**Q：导入链接时提示“URL 格式无效”，但我确认链接可以正常访问，是什么原因？**

A：本工具在导入阶段会对 URL 进行基础格式校验（检查协议头、域名合法性等），但不会主动发起 HTTP 请求。请检查您的链接是否包含多余的空格、换行符或非 ASCII 字符。如果链接包含中文或特殊符号，请先进行 URL 编码后再导入。您也可以使用 `--strict false` 参数关闭严格校验模式，但这样做会跳过格式检查，可能导致后续检测失败。

**Q：状态检测功能返回大量超时或连接错误，如何调整检测策略？**

A：您可以在 `config/default.yaml` 或通过环境变量调整 `REQUEST_TIMEOUT`（默认 10 秒）和 `MAX_RETRIES`（默认 3 次）。如果目标服务器有反爬机制，建议增加检测间隔（`CHECK_INTERVAL`，默认 1 秒），避免请求频率过高被临时封禁。对于已知的失效链接，可以使用 `cli.py check --skip-failed` 跳过已标记失败的链接。

**Q：如何将本工具与现有的视频内容管理系统（CMS）集成？**

A：推荐使用导出功能将索引数据生成为 JSON 或 CSV 格式，再通过 CMS 提供的数据导入接口进行同步。您也可以编写自定义脚本，直接读取 SQLite 数据库中的 `links` 表，获取所有链接及其元数据字段。如果需要在 CMS 中实时查询链接状态，可考虑将 VODLink Indexer 部署为微服务，通过 REST API 对外提供查询接口（参考 `app.py` 中的示例实现）。
