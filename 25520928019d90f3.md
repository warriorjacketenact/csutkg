# VueBest 技术资源导航

VueBest 是一个面向 Vue.js 生态系统的技术资源导航与聚合平台，专注于收集、分类和索引高质量的 Vue 相关开发资源、组件库、工具链及学习材料。项目定位为 Vue 技术栈的一站式资源入口，服务于前端开发工程师、架构师和技术决策者。

项目核心价值在于解决 Vue 生态中资源分散、版本碎片化和质量参差不齐的问题。通过人工审核与社区投票相结合的方式，VueBest 建立了严格的资源准入标准，确保收录的每一个资源都具有实际生产价值。项目同时提供资源变更追踪和版本兼容性标注，帮助开发团队在技术选型时做出更明智的决策。

## 功能概览

**资源分类索引** 按功能领域对 Vue 资源进行系统分类，包括 UI 组件库、状态管理方案、路由工具、构建优化插件、测试框架、服务端渲染方案和移动端适配工具等。

**版本兼容性标注** 为每个资源明确标注其支持的 Vue 2.x 和 Vue 3.x 版本情况，以及相关的 breaking changes 说明。

**社区质量评分** 整合 GitHub stars、npm 下载量、issue 响应速度和 PR 合并周期等多维度数据，生成综合质量评分。

**资源变更追踪** 自动监测收录资源的版本更新、仓库迁移和停止维护状态，通过 RSS 和邮件列表向订阅用户推送变更通知。

**技术雷达视图** 将资源划分为采用、试验、评估和暂缓四个象限，帮助团队理解各项技术在生态中的成熟度定位。

**私有资源库支持** 支持企业用户搭建内部资源索引，纳入自研组件和定制工具，与公开资源统一检索和管理。

**多语言本地化** 资源说明文档支持中文、英文和日文三种语言切换，资源标签支持多语言关联检索。

**API 开放接口** 提供 RESTful API 供第三方工具查询资源数据，支持按分类、标签、评分和版本条件进行组合筛选。

## 应用场景

技术选型与架构评审场景：技术负责人或架构师在启动新项目时，通过 VueBest 快速对比不同 UI 组件库的功能覆盖度、维护活跃度和社区规模，结合版本兼容性信息确定最适合当前技术栈的组合方案。

日常开发效率提升场景：前端开发者在实现特定功能（如数据表格、富文本编辑器、图表可视化）时，通过 VueBest 按分类检索到经过质量验证的组件库，避免重复造轮子并降低试错成本。

团队技术标准化场景：大中型开发团队使用 VueBest 的私有资源库功能建立内部组件索引，将自研的业务组件和定制的第三方库纳入统一管理，配合技术雷达视图推动团队技术栈的规范化和版本统一。

开源项目发现与贡献场景：Vue 生态的贡献者或学习者通过资源变更追踪功能及时了解社区动态，发现新兴工具和值得关注的项目，参与 issue 讨论或提交 PR 以融入生态建设。

## 快速开始

以下命令演示如何将 VueBest 项目克隆到本地并进行开发和预览。

```bash
# 克隆项目仓库
git clone https://github.com/vuebest/vuebest-resource-hub.git

# 进入项目目录
cd vuebest-resource-hub

# 安装项目依赖
npm install

# 启动开发服务器（默认监听 5173 端口）
npm run dev

# 生产环境构建
npm run build

# 预览生产构建结果
npm run preview
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或 20.x LTS | 是 | 项目运行时环境，推荐使用最新的 LTS 版本。低于 18.x 的版本将无法正常构建。 |
| npm 9.x 或 yarn 1.22+ | 是 | 包管理工具，用于安装项目依赖和执行脚本命令。 |
| Git 2.30+ | 是 | 版本控制系统，用于克隆仓库和管理代码变更历史。 |
| SQLite 3.38+（开发环境） | 否 | 本地开发环境使用 SQLite 作为资源索引数据库。生产环境可替换为 PostgreSQL 14+。 |
| Docker 24.0+（容器化部署） | 否 | 如需使用容器化部署方案，需要安装 Docker Engine 和 Docker Compose。 |
| Redis 7.0+（缓存服务） | 否 | 用于 API 查询结果缓存和会话存储，在高并发场景下建议启用。 |
| Nginx 1.22+（生产网关） | 否 | 推荐在生产环境中使用 Nginx 作为反向代理网关，提供负载均衡和静态资源缓存能力。 |

## 项目结构

```
vuebest-resource-hub/
├── src/
│   ├── api/                         # RESTful API 路由与控制器
│   │   ├── routes/                  # 路由定义文件，按资源类型分组
│   │   └── middleware/              # 认证、日志、限流等中间件
│   ├── core/                        # 核心业务逻辑层
│   │   ├── crawler/                 # 资源信息爬取与更新模块
│   │   ├── scorer/                  # 多维度质量评分算法实现
│   │   └── notifier/                # 变更通知与订阅推送服务
│   ├── models/                      # 数据模型定义（SQLAlchemy ORM）
│   │   ├── resource.py              # 资源主表模型
│   │   ├── category.py              # 分类与标签模型
│   │   └── version.py               # 版本兼容性信息模型
│   ├── services/                    # 外部服务集成层
│   │   ├── github/                  # GitHub API 集成（stars、issues）
│   │   ├── npm/                     # npm 注册表集成（下载量、依赖）
│   │   └── cache/                   # Redis 缓存服务封装
│   ├── web/                         # 前端 Web 应用（Vue 3 + Vite）
│   │   ├── components/              # Vue 单文件组件
│   │   ├── views/                   # 页面级视图组件
│   │   └── stores/                  # Pinia 状态管理
│   └── utils/                       # 通用工具函数库
├── docs/                            # 项目文档（见文档导航章节）
├── tests/                           # 单元测试与集成测试用例
├── scripts/                         # 数据库迁移与数据导入脚本
├── docker/                          # Docker 构建文件与 Compose 配置
```

## 常见问题

问：VueBest 收录资源的审核标准是什么？如何确保资源质量？

答：收录标准包括三个维度：技术维度要求资源在 GitHub 上的 stars 不低于 500、npm 月下载量不低于 10000 且最近一次提交在 6 个月内；功能维度要求资源提供完整的 API 文档、使用示例和 MIT 或 Apache-2.0 等宽松开源许可证；社区维度要求 issue 的平均响应时间不超过 72 小时且 PR 合并率不低于 60%。新提交的资源会经过 14 天的社区公示期，由活跃贡献者投票决定是否正式收录。

问：VueBest 支持 Vue 2 和 Vue 3 的版本区分吗？如何处理版本迁移带来的资源变化？

答：项目对每个资源单独维护版本兼容性字段，支持标注具体支持的 Vue 版本范围（如 2.6.14 - 2.7.16 或 3.0.0 - 3.4.0）。当资源发布新版本变更了对 Vue 主版本的兼容性时，爬虫模块会自动检测并更新标注，同时通过变更通知系统向订阅用户发送版本迁移警告。用户可以在资源详情页查看完整的版本支持时间线。

问：如何在本地开发环境中初始化测试数据？

答：执行 npm run seed 命令会触发数据库填充脚本，该脚本从 /scripts/seed-data.json 读取预置的 200 条示例资源数据进行初始化。如需清空数据重新初始化，可先执行 npm run db:reset 重置数据库结构，再执行 npm run seed。测试数据包含虚构的资源名称、分类标签和评分信息，仅供开发和调试使用，不应在生产环境中执行。
