# LinkVault Resource Aggregator

LinkVault 是一个面向技术开发者、系统运维人员与数据分析师的高质量外链资源汇总平台。本项目定位于对特定域名下的动态内容路径进行系统性收集、分类与结构化呈现，帮助用户快速发现并访问高价值的技术直播、会议回放、教程演示及实时数据看板。通过统一的门户索引，用户无需在多个平台间反复搜索，即可获得持续更新的垂直领域资源清单。

本项目当前批次为第 554/1241 批，涵盖 250 个有效资源链接。所有链接均指向 https://www.3h800.com 域名下的实时直播与活动记录页面，内容覆盖技术分享、行业峰会、开源社区会议、产品发布演示及运维故障复盘等多个细分领域。LinkVault 不直接存储或代理任何视频内容，仅提供公开可访问的 URL 索引与元数据描述，适用于需要快速建立技术活动资料库的团队或个人研究者。

## 功能概览

- **批量链接导入与校验**：支持从 CSV、JSON 或纯文本列表批量导入 URL，自动执行可达性检测与状态码验证，过滤无效或重定向链接。

- **分类标签生成**：基于 URL 路径模式、查询参数及页面标题自动生成分类标签，支持人工修正与合并，便于按主题聚合。

- **元数据提取与缓存**：对每个链接执行异步元数据抓取，提取页面标题、描述、关键词及最后修改时间，结果存入本地 SQLite 缓存，减少重复请求。

- **自定义集合管理**：用户可创建命名集合，将不同批次的链接按项目、日期或主题归类，每个集合支持独立备注与版本标记。

- **只读快照导出**：将选定的链接集合导出为静态 HTML 或 Markdown 表格，方便嵌入文档、Wiki 或内部知识库，无需依赖后端服务。

- **定期健康检查**：内置定时任务，可按日/周/月对已收录链接执行可用性探测，生成异常报告并标记失效资源，维持索引质量。

- **RESTful API 接口**：提供基于 JSON 的只读查询接口，支持按 ID、标签、日期范围或关键词模糊搜索，便于集成到自动化工作流中。

- **操作审计日志**：记录所有链接的添加、删除、分类修改及导出操作，包含时间戳与操作者标识，满足团队协作下的追溯需求。

## 应用场景

**技术会议资料归档**  
团队在参加线上技术峰会或社区 meetup 后，可将会议中发布的直播回放链接统一收录至 LinkVault，按年度和会议名称建立集合，方便后续复盘与新人培训时快速检索。

**运维故障案例库建设**  
运维工程师可将历史故障复盘直播链接、应急处理演示及根因分析报告链接集中管理，结合标签标注故障类型与影响范围，形成可检索的案例知识库，提升应急响应效率。

**开源项目生态追踪**  
开源社区维护者可使用 LinkVault 收集与项目相关的各类线上路演、贡献者会议及版本发布直播链接，按版本号或模块分类，帮助社区成员及时获取最新动态。

**数据分析看板订阅**  
数据分析师可将定期更新的数据指标解读直播、BI 工具演示及行业趋势分析链接纳入集合，配合健康检查功能监控链接有效性，确保报告撰写时引用的素材始终可访问。

## 快速开始

以下步骤帮助您在本地环境中快速启动 LinkVault 实例，完成链接导入与基础查询。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault-core.git

# 进入项目目录
cd linkvault-core

# 安装依赖（使用 pip 与 requirements.txt）
pip install -r requirements.txt

# 初始化数据库（创建 SQLite 数据表与索引）
python scripts/init_db.py

# 导入示例链接列表（包含本批次的部分 URL）
python scripts/import_urls.py --input data/sample_urls.txt --batch 554

# 启动本地开发服务器（默认监听 127.0.0.1:8080）
python app.py runserver
```

访问 http://127.0.0.1:8080 即可查看已导入链接的概览面板，并通过 Web 界面执行搜索、分类与导出操作。

## 安装要求

LinkVault 基于 Python 3.9 以上版本开发，依赖轻量级 Web 框架与异步 HTTP 客户端。下表列出了核心依赖项及必要的外部环境。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | >= 3.9, < 3.12 | 运行时环境，建议使用 pyenv 或 conda 管理版本 |
| pip | >= 21.0 | Python 包管理工具，用于安装依赖库 |
| SQLite | >= 3.35 | 嵌入式数据库，用于存储链接元数据与审计日志 |
| aiohttp | >= 3.8.0 | 异步 HTTP 客户端，用于并发抓取页面元数据 |
| beautifulsoup4 | >= 4.11.0 | HTML 解析库，用于提取页面标题与描述 |
| click | >= 8.0.0 | 命令行交互框架，用于实现 CLI 工具命令 |
| python-dotenv | >= 0.19.0 | 环境变量加载器，用于管理敏感配置项 |
| pytest | >= 7.0.0 | 测试框架，仅开发环境需要，生产环境可不安装 |
| flask | >= 2.2.0 | Web 框架，提供管理界面与 REST API 服务 |

## 项目结构

LinkVault 采用模块化设计，核心代码、数据层与辅助脚本分离，便于维护与扩展。以下为项目根目录下的主要目录与文件布局。

```
linkvault-core/
├── app/                                # 核心应用包
│   ├── __init__.py                     # 包初始化与工厂函数
│   ├── routes/                         # 路由层，处理 HTTP 请求
│   │   ├── api.py                      # RESTful API 端点实现
│   │   └── web.py                      # 管理界面页面路由
│   ├── models/                         # 数据模型与 ORM 映射
│   │   ├── link.py                     # Link 实体定义与校验逻辑
│   │   ├── collection.py               # Collection 集合模型
│   │   └── audit.py                    # AuditLog 审计日志模型
│   ├── services/                       # 业务服务层
│   │   ├── fetcher.py                  # 异步元数据抓取服务
│   │   ├── checker.py                  # 链接健康检查调度器
│   │   └── exporter.py                 # 快照导出服务（HTML/Markdown）
│   └── utils/                          # 通用工具函数
│       ├── validators.py               # URL 格式与协议校验
│       └── parsers.py                  # 命令行参数与配置文件解析
├── scripts/                            # 运维与初始化脚本
│   ├── init_db.py                      # 创建 SQLite 数据库表结构
│   ├── import_urls.py                  # 批量导入 URL 列表文件
│   └── run_health_check.py             # 手动触发全量健康检查
├── config/                             # 配置文件目录
│   ├── default.toml                    # 默认配置（端口、超时、并发数）
│   └── production.toml                 # 生产环境覆盖配置
├── tests/                              # 单元测试与集成测试
│   ├── test_models.py                  # 数据模型单元测试
│   ├── test_fetcher.py                 # 抓取服务模拟测试
│   └── conftest.py                     # pytest 共享 fixture
├── docs/                               # 项目文档源码（Markdown）
│   ├── getting-started.md              # 入门指南
│   ├── user-guide.md                   # 用户操作手册
│   ├── api-reference.md                # API 接口详细规范
│   └── faq.md                          # 常见问题汇总
├── data/                               # 数据存储目录
│   ├── linkvault.db                    # SQLite 主数据库文件
│   └── sample_urls.txt                 # 示例链接列表（用于演示）
├── logs/                               # 日志文件目录
│   ├── app.log                         # 应用运行日志
│   └── checker.log                     # 健康检查专用日志
├── requirements.txt                    # 生产环境依赖列表
├── requirements-dev.txt                # 开发环境额外依赖（测试、Lint）
├── Dockerfile                          # 容器构建文件
```

LinkVault 欢迎社区贡献者提交新的资源链接、改进现有元数据或完善项目文档。请遵循以下流程以确保贡献顺利合并。

1.  **复刻仓库并创建分支**：从主仓库复刻至个人账户，然后基于 `main` 分支创建以 `feature/` 或 `fix/` 为前缀的新分支，例如 `feature/add-batch-555`。

2.  **遵循链接收录规范**：新增链接必须来自公开可访问的页面，且内容与现有分类主题一致。提交前请使用脚本 `scripts/check_url.py` 验证链接可达性，并确保不包含重复项。

3.  **更新资源列表与元数据**：若新增链接属于新的批次，请在 `data/` 目录下创建对应的批次文件，并同步更新 `docs/` 中相关文档的示例或引用。对于已有链接，若发现标题或描述过时，可提交修正。

4.  **编写或更新测试用例**：对于涉及代码逻辑的修改（如新增解析规则或 API 字段），请在 `tests/` 目录下补充相应的单元测试，确保测试覆盖率达到现有标准。

5.  **提交拉取请求**：推送分支后，向主仓库的 `main` 分支发起拉取请求。请在描述中清晰说明变更目的、涉及的文件列表以及验证方法。项目维护者将在三个工作日内完成审查。

## 常见问题

**问：导入大量链接时出现超时或连接重置，应如何优化？**

答：元数据抓取服务默认使用 10 个并发请求，超时时间为 15 秒。若目标站点响应较慢，可通过修改 `config/default.toml` 中的 `fetcher.concurrency` 和 `fetcher.timeout` 参数降低并发数或延长超时。同时，建议在低峰时段执行批量导入，并利用 `--retry` 参数启用失败重试机制。

**问：数据库文件 `linkvault.db` 体积增长过快，如何清理历史数据？**

答：审计日志表 `audit_logs` 会随操作次数持续增长。可执行 `scripts/cleanup_logs.py --days 90` 删除 90 天前的日志记录。若需要彻底清理无效链接，可使用 `scripts/prune_links.py --status 404` 移除所有返回 404 状态的链接，该操作会同时级联删除关联的集合映射。

**问：如何将本项目的链接数据迁移到其他数据库如 PostgreSQL？**

答：LinkVault 的模型层使用 SQLAlchemy ORM，理论上支持多种关系型数据库。迁移时需执行以下步骤：1）在 `config/production.toml` 中配置 `database.url` 为 PostgreSQL 连接串；2）运行 `scripts/migrate_db.py --target postgresql` 自动转换表结构与索引；3）使用 `scripts/export_data.py --format json` 导出 SQLite 数据，再通过 `scripts/import_data.py` 导入至新数据库。注意时间戳与布尔类型的字段映射差异。
