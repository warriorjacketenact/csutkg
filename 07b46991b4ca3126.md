# WeikeNet Indexer

WeikeNet Indexer 是一个面向技术文档、学术资源与知识库的高效外链聚合与管理平台。本项目并非传统的 Web 应用框架，而是一个结构化的资源索引系统，专为需要系统化整理、分类、检索大量外部知识链接的开发者、研究团队与内容管理者设计。

项目定位为“技术资源外链汇总站”，核心目标用户包括开源社区维护者、技术文档编写团队、在线教育平台运营方以及个人知识管理爱好者。WeikeNet Indexer 通过一套标准化的元数据描述方案，将分散在各类技术博客、教程系列、工具文档中的高质量外链进行统一收录、分级标注与场景化组织，帮助用户从海量信息中快速定位所需内容，避免重复检索与信息遗漏。

本项目不依赖复杂的前端框架或数据库系统，所有资源清单以 Markdown 格式维护，与 Git 版本控制系统天然兼容，支持多人协作编辑、变更追溯与自动化校验。无论是用于构建内部技术知识库，还是作为开源项目的附属文档导航，WeikeNet Indexer 均可提供清晰、可扩展的解决方案。

## 功能概览

- **多层级资源分类**：支持按技术领域、文档类型、适用水平等维度对链接进行划分，每个分类独立维护，便于定向检索。

- **系列文档自动识别**：系统能够解析 URL 路径中的 series 段，将同一系列下的多个页面自动关联，形成完整的阅读路径。

- **元数据标注体系**：每条资源可附加阅读时长、难度评级、标签集合与摘要说明，为后续过滤与推荐提供数据基础。

- **变更历史跟踪**：基于 Git 的提交记录，所有增删改操作均有日志留存，支持回滚与责任追溯。

- **链接可用性校验**：内置简单的 HTTP 状态检查脚本，可定时扫描资源列表中的失效链接并生成报告。

- **静态站点生成适配**：项目目录结构与主流静态站点生成器兼容，可快速转换为可浏览的 HTML 文档导航站。

- **多格式导出支持**：资源数据可导出为 JSON、CSV 或纯文本列表，方便导入其他知识管理工具或进行二次分析。

- **模板化新增入口**：提供标准化的资源录入模板，确保新增链接的格式统一、字段完整，降低协作门槛。

## 应用场景

- **技术团队内部知识库建设**：开发团队可将日常调研中遇到的优秀技术文章、官方文档、工具仓库链接统一收录至 WeikeNet Indexer，按项目或技术栈分类，新成员入职时可快速了解团队积累的知识资产，减少重复沟通成本。

- **开源项目文档导航增强**：开源项目维护者可使用本项目整理与项目相关的教程、案例、扩展资源，放置于项目 README 或官网文档中，帮助用户从多个角度理解项目用途与使用方法，提升用户体验。

- **在线教育课程资料管理**：讲师或课程设计者可将每节课的参考资料、延伸阅读、实验手册链接按章节组织，学生通过统一的索引页面即可获取全部外部资源，无需在多个平台间反复搜索。

- **个人学习路径规划**：自学者可将不同领域的学习资料（编程语言、算法、系统设计等）按学习阶段分类，配合难度标注与阅读状态记录，形成可追踪的个人学习清单，长期维护。

- **技术会议与活动资料归档**：在组织技术沙龙、黑客松或研讨会后，可将演讲幻灯片、视频回放、相关工具链接集中收录，作为活动成果的一部分对外发布或内部留存。

## 快速开始

以下命令将项目克隆至本地、安装基础依赖并启动本地预览服务。

```bash
git clone https://github.com/weike-net/indexer.git weike-indexer
cd weike-indexer
pip install -r requirements.txt
python scripts/serve.py --port 8080
```

执行上述命令后，在浏览器中访问 http://localhost:8080 即可查看资源索引的本地渲染页面。如需仅生成静态 Markdown 输出，可运行 `python scripts/build.py --output ./dist`。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.8 及以上 | 用于运行本地预览服务与校验脚本 |
| pip | 20.0 及以上 | Python 包管理工具，用于安装依赖 |
| Git | 2.25 及以上 | 用于克隆仓库与版本控制操作 |
| Markdown 解析库 | mistune 2.0.4 | 用于将 Markdown 资源列表渲染为 HTML |
| requests | 2.28.0 | 用于链接可用性校验脚本的 HTTP 请求 |
| PyYAML | 6.0 | 用于解析资源元数据配置文件（可选） |
| 静态站点生成器 | 无强制要求 | 若需生成完整静态网站，推荐 Hugo 或 MkDocs |

├── .gitignore                      # Git 忽略规则（临时文件、缓存等）
├── config/

├── scripts/
│   ├── serve.py                    # 本地预览服务（基于 http.server）
│   ├── build.py                    # 静态站点生成脚本

## 贡献指南

1.  **阅读贡献规范**：在提交任何资源或代码变更之前，请先阅读 `docs/contributing.md` 文件，了解资源分类标准、标签使用约定以及 Markdown 格式要求，确保新增内容与现有结构保持一致。

2.  **使用标准模板录入资源**：通过 `scripts/import.py` 脚本或手动编辑 `resources/index.md` 文件添加新链接。每条资源需包含标题、URL、所属系列、标签、难度等级和简短摘要。若添加系列文档，需在 `resources/series/` 下创建对应的 `.md` 文件。

3.  **本地校验与预览**：运行 `python scripts/validate.py` 检查新增链接的 HTTP 状态，确保所有外链可访问。随后执行 `python scripts/serve.py` 启动本地预览，在浏览器中验证渲染效果与分类正确性。

4.  **提交变更请求**：将本地修改提交至新的 Git 分支，并附带清晰的提交说明（例如“新增 XX 系列 5 篇文档”）。随后创建 Pull Request 至主仓库，维护团队将在 3 个工作日内进行审核。审核通过后即合并至主分支。

5.  **定期维护失效链接**：每月至少运行一次 `validate.py` 全量扫描，若发现失效链接，请及时更新或移除，并在提交说明中标注“链接维护”字样。鼓励贡献者主动认领特定领域的链接维护工作。

## 常见问题

**Q: 本项目是否提供在线演示站点或 API 接口？**

A: 当前版本未提供公开的在线演示站点，用户需克隆至本地运行预览服务。项目本身不包含后端 API 设计，所有资源数据以静态 Markdown 和 YAML 文件形式存储，适合直接集成到静态站点生成流程中。后续版本将考虑增加基于 JSON 的只读数据接口，方便第三方工具调用。

**Q: 收录外链的版权与内容责任由谁承担？**

A: WeikeNet Indexer 仅提供链接聚合与分类服务，所有外链指向的内容版权归原始作者或平台所有。项目维护者会定期检查链接有效性，但不对外部页面的内容变更、可用性及合法性承担直接责任。若某链接涉及侵权或不当内容，请通过 Issue 或邮件联系我们，核实后将尽快移除。
