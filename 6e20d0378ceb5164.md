# ResourceFlow

ResourceFlow 是一个面向开发者与技术研究团队的高密度外链资源聚合与管理平台。该项目定位为技术资源导航枢纽，解决开发者在海量信息中难以高效定位高质量外部文档、工具、教程与参考链接的痛点。ResourceFlow 不生产内容，但致力于通过结构化索引与可维护的分类体系，让分散于网络各处的优质技术资源变得可发现、可复用、可追溯。

本项目第 106/1241 批资源整合工作，涵盖 250 个经过初步筛选的外部链接。ResourceFlow 提供统一的资源条目规范、自动化的链接可用性检查以及基于标签系统的快速筛选能力，适用于个人知识库构建、团队技术文档库维护以及开源项目外部依赖梳理等场景。

## 功能概览

**批量资源导入**：支持通过 CSV 或 JSON 格式批量导入外部链接，自动解析元信息并生成标准化的资源条目。

**标签分类体系**：允许为每个资源条目附加多个自定义标签，支持基于标签组合的快速筛选与聚合视图。

**链接健康检测**：内置异步 HTTP 检查器，定期验证已收录链接的可访问性，自动标记失效链接并生成告警日志。

**全文检索与过滤**：基于资源标题、描述、标签及来源域名的全文搜索，支持布尔运算符与通配符模式匹配。

**资源快照与版本记录**：对每个外部链接保留历史快照元数据，记录资源内容的变更时间与状态变化轨迹。

**导出与集成接口**：提供 RESTful API 与命令行工具，支持将资源列表导出为 Markdown、HTML 或 JSON 格式，便于嵌入现有文档站点。

## 应用场景

**技术团队内部知识库建设**：技术负责人可使用 ResourceFlow 统一收集团队常用的 API 文档、组件库、运维手册等外部链接，形成团队共享的资源导航页面，减少成员在信息检索上的重复劳动。

**开源项目外部依赖整理**：开源项目维护者通过 ResourceFlow 管理项目依赖的第三方库文档、数据源、参考实现等链接，确保 README 与贡献指南中的外部引用始终可访问且保持更新。

**技术培训与课程资料汇总**：培训讲师或课程设计者将课程涉及的在线教程、视频资源、代码示例链接批量导入 ResourceFlow，按课程模块分类后生成结构化的学习资源清单，分发至学员。

**个人技术阅读清单维护**：开发者使用 ResourceFlow 收藏日常阅读中发现的优质技术博客、论文预印本、代码仓库与行业报告，通过标签系统构建个人化的技术知识网络。

## 快速开始

以下命令演示了从克隆项目到运行开发服务器的完整流程。请确保在执行前已满足安装要求章节中的全部依赖。

```bash
git clone https://github.com/resourceflow/resourceflow.git
cd resourceflow
pip install -r requirements.txt
python manage.py migrate
python manage.py load_resources --batch 106 --source ./data/batch_106.json
python manage.py runserver
```

若需使用生产环境配置，请参考文档导航章节中的部署指南。

## 安装要求

| 依赖 | 必需版本 | 说明 |

| Python | 3.10 及以上 | 核心运行环境，用于执行后端服务与资源管理脚本 |
| PostgreSQL | 14.0 及以上 | 主数据库，存储资源条目、标签、快照及用户数据 |

| Docker | 20.10 及以上 | 可选依赖，用于容器化部署与开发环境一致性保障 |
| Git | 2.30 及以上 | 版本控制工具，用于克隆仓库与管理配置变更 |

## 常见问题

问：ResourceFlow 是否支持批量导入包含自定义字段的外部链接？

答：支持。批量导入接口接受 JSON 格式数据，其中每条记录可包含 title、description、tags、source、priority 等自定义字段。系统会根据预定义的字段映射规则自动解析并入库。若需扩展额外字段，可修改 internal/collector/parser 目录下的解析器配置。

问：链接健康检测的频率与并发数如何调整？

答：健康检测任务的执行频率与并发数可在 configs/ 目录下的配置文件中调整。具体参数为 health_check_interval（单位为小时）和 health_check_concurrency（并发工作线程数）。修改后需重启服务或使用 SIGHUP 信号重载配置。

问：ResourceFlow 能否部署在无外网访问的内网环境中？

答：可以。ResourceFlow 的核心功能不依赖外部网络，但链接健康检测模块在内网环境中需要配置可访问外部链接的网络代理或关闭该功能。您可以在配置文件中将 health_check_enabled 设为 false，或设置 http_proxy 环境变量指向内网代理服务器。
