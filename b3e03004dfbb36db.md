# HLS Resource Aggregator

HLS Resource Aggregator 是一个面向多媒体技术开发者与流媒体服务运维人员的高效外链资源汇总工具。本项目系统化收录并分类管理大量与 HLS 协议、视频流处理、播放器集成及 CDN 分发相关的技术参考链接，旨在解决开发者在实际工作中遇到的文档分散、示例缺失、配置方案难以追溯等问题。

本项目适用于以下用户群体：流媒体后端工程师、前端视频播放器开发者、CDN 运维人员、音视频算法研究人员以及技术选型评估团队。通过结构化的数据组织和清晰的资源分类，用户可以在数秒内定位到所需的具体技术参考页面，极大提升开发和排障效率。

## 功能概览

**结构化资源索引** 提供按协议版本、播放器类型、CDN 厂商等多维度分类的资源导航，支持快速筛选。

**实时链接健康检查** 内置周期性链接可用性检测机制，自动标记失效或重定向的资源条目，确保参考链接的有效性。

**自定义标签系统** 允许用户为每个资源条目添加自由标签，便于建立个人化的分类体系和快捷访问列表。

**全文搜索支持** 基于资源标题、描述、标签和 URL 路径关键词的全文检索引擎，支持模糊匹配与精确查询。

**导入导出机制** 支持批量导入外部链接清单和导出当前资源库为 JSON、CSV 格式，便于团队协作与数据迁移。

**访问统计分析** 记录每个资源链接的被访问次数与最后访问时间，为资源热度评估和内容优化提供数据支持。

**响应式管理界面** 提供适配桌面与移动设备的 Web 管理控制台，方便运维人员随时随地进行资源维护。

**开放 API 接口** 提供 RESTful API 用于第三方系统集成，支持资源的增删改查及分类管理操作。

## 应用场景

**新项目技术选型** 团队在启动新的流媒体项目时，可以通过本资源库快速查阅不同 HLS 实现方案、播放器组件和 CDN 配置案例，对比各方案的优劣，缩短调研周期。

**线上问题紧急排查** 当生产环境出现播放卡顿、切片加载失败或码率切换异常时，运维人员可借助资源库中的故障排查手册和调试工具链接，快速定位问题根因并恢复服务。

**技术文档统一管理** 企业可将内部积累的 HLS 最佳实践文档、第三方服务商对接指南和常见问题解决方案集中收录于本系统，形成团队共享的知识库，降低人员流动带来的信息流失风险。

**自动化测试用例参考** 质量保障团队可利用资源库中收集的各类 HLS 流地址和测试页面，构建自动化播放测试用例，覆盖不同编码格式、分辨率和网络条件下的播放稳定性验证。

## 快速开始

以下步骤指导您在本机快速部署 HLS Resource Aggregator 开发环境。

```bash
# 克隆项目仓库
git clone https://github.com/hls-resource-aggregator/hls-ra.git
cd hls-ra

# 安装项目依赖
npm install

# 启动开发服务器
npm run dev
```

执行完上述命令后，访问控制台地址 http://localhost:3000 即可开始使用。生产环境部署请参考 `docs/deployment.md` 中的详细配置说明。

## 常见问题

**问：项目支持哪些数据库后端？**  
默认使用 SQLite 作为嵌入式数据库，适合中小规模部署。生产环境可通过修改 `config/production.json` 中的数据库连接串切换到 PostgreSQL 或 MySQL。切换后需手动运行迁移脚本创建表结构。

**问：链接健康检查的频率和超时时间如何配置？**  
健康检查任务默认每 24 小时执行一次，单次请求超时时间为 10 秒。用户可在管理控制台的系统设置页面调整上述参数，或通过环境变量 `CHECK_INTERVAL` 和 `REQUEST_TIMEOUT` 覆盖默认值。检查结果会记录至访问日志表，并可通过 API 查询。

**问：如何将现有资源库迁移到另一台服务器？**  
执行 `scripts/backup.js` 生成包含完整资源数据和标签体系的 JSON 导出文件。在目标服务器部署完成后，通过管理控制台的导入功能上传该文件，系统会自动重建索引并校验链接有效性。迁移过程中不会影响源服务器的正常运行。
