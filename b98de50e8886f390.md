# LinkVault 资源索引系统

LinkVault 是一个面向技术团队与内容创作者的轻量级外链资源归集与导航工具。该项目定位为技术资源汇总站，专门解决多源、多批次、多类别外部链接的收录、分类、检索与展示问题。目标用户包括开源项目维护者、技术社区运营人员、知识库管理者以及需要批量管理外链资源的开发人员。

LinkVault 提供从链接导入、自动分类、元数据提取到静态站点生成的全链路能力，帮助用户将散落在各处的碎片化 URL 转化为结构清晰、可检索、可分享的资源导航页面。项目本身不依赖数据库，所有数据以 Markdown 和 YAML 形式存储，便于版本控制与协作编辑。

## 功能概览

**批量链接导入** 支持从 CSV、JSON、Markdown 列表及纯文本格式批量导入 URL，自动去重并校验可访问性。

**智能分类引擎** 基于 URL 路径模式、域名特征与可配置规则集，对导入链接进行自动归类与标签标注。

**元数据抓取** 自动抓取每个链接的页面标题、描述、关键词及最后修改时间，生成索引所需的富元数据。

**静态站点生成** 内置模板引擎，将链接数据渲染为响应式 HTML 导航页面，支持按分类、标签、时间多种视图展示。

**全文检索** 基于 MiniSearch 提供轻量级客户端全文检索能力，支持对标题、描述、标签字段的模糊查询与权重排序。

**数据快照与回滚** 每次导入或编辑操作自动生成数据快照，支持一键回滚至任意历史版本，保障数据操作安全。

**访问状态监控** 定时检测所有外链的可访问性，标记失效链接并生成健康报告，便于及时清理或更新。

## 应用场景

**开源项目文档站外链管理** 开源项目维护者可使用 LinkVault 整理项目文档中引用的外部参考资料、依赖项目地址、社区论坛链接，生成独立的资源导航页，避免文档正文被大量外链充斥。

**技术团队内部知识库导航** 技术团队可将日常开发中积累的 API 文档、技术博客、内部系统地址、运维手册等外部分散资源统一收录至 LinkVault，生成团队共享的知识入口页面，减少信息查找时间。

**社区月刊与资源周报生成** 技术社区运营人员可将每周/每月收集的优秀文章、工具、视频教程等链接导入 LinkVault，自动生成带分类索引的静态页面，作为社区内容沉淀与传播的载体。

**个人书签系统替代方案** 个人开发者可使用 LinkVault 替代浏览器书签，将收藏的各类开发资源按项目、语言、领域等维度组织为可公开或私有部署的导航站点，实现跨设备、跨浏览器的统一访问入口。

## 快速开始

以下命令演示如何在本地快速启动 LinkVault 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装依赖（使用 npm）
npm install

# 导入示例数据并启动开发服务器
npm run seed
npm run dev
```

执行完上述命令后，访问控制台输出的本地地址即可查看 LinkVault 实例。生产环境部署请参考 `docs/deployment.md` 文档。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 及以上 | 是 | 运行时环境，推荐使用 LTS 版本 |
| npm 9.x 及以上 | 是 | 包管理器，用于安装项目依赖 |
| Git 2.30 及以上 | 是 | 用于克隆仓库及版本管理 |
| 现代浏览器（Chrome/Firefox/Edge 最新两版） | 是 | 访问管理界面与生成的导航页面 |
| 网络连接 | 是 | 首次启动需下载依赖包，元数据抓取需访问外链 |
| SQLite 3（可选） | 否 | 如需使用持久化缓存层可安装，默认使用内存缓存 |
| Redis 7.x（可选） | 否 | 用于多实例部署时的缓存与任务队列协调 |
| Nginx/Apache（可选） | 否 | 生产环境推荐用于反向代理与静态资源服务 |

## 常见问题

**问：LinkVault 能否处理百万级别的外链数据？**

答：LinkVault 核心设计定位于中小规模资源归集场景，单实例在默认配置下可稳定处理五万条以内的链接数据。若数据量超过此规模，建议结合外部 SQLite 或 PostgreSQL 数据库替代内存缓存，并调整批处理大小参数。对于百万级别数据，推荐使用企业级搜索引擎或专用数据库方案，LinkVault 可作为前端展示层配合使用。

**问：导入外部链接时能否自动抓取页面完整内容而非仅元数据？**

答：默认配置下，LinkVault 仅抓取页面标题、描述、关键词等元数据字段，不存储完整 HTML 内容，以节约存储空间并规避版权风险。如需抓取全文，可在配置文件中开启 `metadata.fullContent` 选项，并设置 `content.maxSize` 限制单页面最大抓取字节数。开启此功能后请务必遵守目标网站的 robots.txt 协议。

**问：如何迁移 LinkVault 数据到另一台服务器？**

答：LinkVault 所有数据以文件形式存储在 `data/` 目录下，迁移时只需打包该目录并复制到目标服务器相同路径即可。恢复后执行 `npm run rebuild` 命令重建索引。如需跨版本迁移，请先查阅对应版本的 `docs/operations/migrate.md` 文档，按步骤执行迁移脚本。
