# LinkVault

LinkVault 是一个面向技术调研、内容聚合与知识管理场景的轻量级外链资源归集与导航系统。该项目定位于个人开发者、技术内容创作者以及小型团队，用于对分散在多个平台或批次中的外部链接进行结构化存储、分类展示与快速检索。LinkVault 不提供爬虫或自动化采集能力，而是强调人工编排与元数据标注，帮助用户从大量原始 URL 中提炼出可复用的知识索引。

LinkVault 的核心使用场景包括：技术周报或月刊的链接素材管理、开源项目 README 中的参考资料整理、企业内部知识库的外链白名单维护，以及教育培训场景下的课程扩展阅读清单管理。通过将原始链接列表与自定义分类标签、状态标记和备注字段结合，LinkVault 能够将无序的 URL 集合转化为具有业务语义的导航目录。

## 功能概览

- **批量链接导入**：支持通过命令行或 Web 表单一次性导入大批量原始 URL，自动解析路径参数并生成基础条目记录。

- **自定义分类体系**：允许用户为每个链接指定多级分类标签，并可创建虚拟分类视图，便于按主题或用途重新组织链接集合。

- **状态与进度跟踪**：为每个链接提供“待审阅”、“已审阅”、“已归档”、“需更新”等状态标记，方便团队协作时的进度管理。

- **全文检索与过滤**：基于链接的 URL、自定义标题、描述文本和标签进行快速检索，并支持按状态、分类、创建时间等多维度过滤。

- **元数据扩展字段**：支持为每个链接添加备注、重要性评分、关联项目编号以及自定义键值对属性，满足不同业务场景的扩展需求。

- **导出与分享**：支持将当前视图下的链接列表导出为 Markdown 表格、JSON 结构化数据或纯文本列表，便于嵌入到其他文档或系统中。

- **审计日志**：记录每个链接条目的创建、修改和状态变更历史，便于追溯与回溯。

## 应用场景

- **技术内容策划**：内容编辑或技术博主在策划系列文章或视频时，可使用 LinkVault 收集和整理大量参考资料，并为每个链接标注适用主题、优先级和阅读状态，从而高效构建内容大纲。

- **项目依赖与参考溯源**：开发团队在技术选型或架构评审阶段，可将各类技术博客、官方文档、社区讨论帖集中收录至 LinkVault，为决策提供可追溯的论据来源。

- **教育培训扩展阅读**：讲师或培训师可在课程准备期间，将大量延伸阅读材料统一入库，然后按课程章节生成分组链接列表，方便学员课后按图索骥。

- **个人知识库外链管理**：知识管理爱好者可将日常浏览中发现的有价值网页存入 LinkVault，配合定期回顾机制，防止有价值的网络资源随时间流逝而被遗忘。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境。请确保已安装 Git 与 Node.js 18+。

```bash
# 克隆仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖
npm install

# 以开发模式启动 Web 界面
npm run dev
```

首次启动后，访问控制台输出的本地地址（通常为 http://127.0.0.1:3000）即可进入 LinkVault 仪表盘。系统默认使用 SQLite 作为存储引擎，无需额外配置数据库连接。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，建议使用官方 LTS 版本 |
| npm | 9.x 或更高 | 包管理器，随 Node.js 一同安装 |
| SQLite3 | 3.35 或更高 | 嵌入式数据库，系统将自动创建数据文件 |
| Git | 2.30 或更高 | 用于克隆仓库及版本控制 |
| 操作系统 | Linux / macOS / Windows 10+ (WSL2) | 生产环境推荐 Linux 或 macOS |

可选依赖（非必需）：
- Redis 7.x：如启用分布式缓存或会话共享，需要额外配置 Redis 服务。
- Nginx 或 Caddy：反向代理与静态资源缓存建议配合使用。

## 常见问题

**Q：LinkVault 是否支持 PostgreSQL 作为生产数据库？**

A：当前稳定版本以 SQLite 为默认存储引擎，适用于单机或小规模团队场景。从 v2.0 开始，我们计划引入对 PostgreSQL 的官方支持，并已在开发分支中提供适配器原型。如果您有迫切需求，可以在配置文件中将 dialect 字段修改为 postgres，并自行配置连接字符串，但请注意此模式尚未经过完整生产测试。

**Q：导入大量链接（例如超过 5000 条）时，界面响应变慢，如何优化？**

A：批量导入时系统会逐条进行 URL 格式校验和元数据解析，对于大规模数据，建议通过命令行脚本（scripts/seed-links.js）进行导入，该脚本会绕过部分前端渲染开销。同时，您可以调整 config/default.yaml 中的 batchSize 参数，控制每次数据库写入的批量大小。在生产环境中，配合 Redis 缓存可显著提升列表加载速度。

**Q：不同用户之间如何共享链接集合？**

A：LinkVault 当前版本以单用户本地模式为主，所有数据存储在同一数据库中。若需多用户协作，我们建议使用同一数据库文件的网络共享存储（如 NFS），或通过导出功能将链接列表导出为 JSON 或 Markdown 文件，再通过版本控制系统（如 Git）进行共享与合并。企业级多租户功能已在 v3.0 规划中。
