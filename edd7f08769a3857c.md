# LinkVault Resource Aggregator

LinkVault 是一个面向开发者和技术研究人员的轻量级外链资源汇总与导航工具。它不直接托管内容，而是通过结构化的索引机制，将分散在互联网各处的技术文档、开源工具、在线服务与学习资料按照功能域进行归类整理，帮助用户快速定位到特定领域的高质量外部资源。

本项目定位于技术团队内部知识库的补充层与个人开发者的书签管理替代方案。LinkVault 不依赖数据库，所有资源索引基于 Markdown 文件与 YAML 头部元数据生成静态站点，适用于 CI/CD 自动化构建流程。目标用户包括需要维护团队技术周报的工程师、搭建个人导航页面的独立开发者，以及需要快速查阅特定技术栈外链参考的科研人员。

## 功能概览

**自动资源抓取与元数据解析**：支持从用户指定的 URL 列表或本地 Markdown 文件中提取链接标题、域名类别与路径特征，生成结构化索引条目。

**多维度分类标签系统**：每个资源条目可关联多个标签（如 database、monitoring、security），支持按标签组合过滤与反向索引查询。

**静态站点生成器集成**：内置适配器支持将资源列表导出为 VuePress、Hugo 或 MkDocs 兼容的目录格式，便于发布为团队内部文档站点。

**链接可用性健康检查**：提供周期性 HTTP 状态码探测功能，自动标记响应超时或返回 4xx/5xx 状态码的链接，生成异常报告。

**全文检索与模糊匹配**：基于 Bleve 或 Zinc 等嵌入式搜索引擎，允许用户通过关键词或路径片段快速定位目标资源，无需记忆完整 URL。

**导入导出互操作性**：支持从浏览器书签 HTML 文件、Raindrop.io JSON 导出包以及 Delicious XML 格式批量导入链接，并可导出为纯文本列表或 CSV 表格。

## 应用场景

**技术团队周报资源汇编**：团队技术负责人可定期将本周阅读的博客文章、开源仓库地址与在线工具链接录入 LinkVault，自动生成带分类的周报附录，减少手动整理时间。

**个人开发环境起始页构建**：开发者将日常使用的 API 文档、CI/CD 控制台、日志查询平台与镜像仓库地址统一托管于 LinkVault，通过本地启动一个静态服务即可获得自定义起始页，无需依赖第三方书签同步服务。

**开源项目外部依赖追溯**：开源项目维护者可在仓库 docs 目录下维护一份 external-resources.md，使用 LinkVault 的命令行工具将其转换为格式统一的资源导航页面，方便贡献者快速了解项目所引用的协议、规范与参考实现。

**离线文档资源索引备份**：在网络受限的内网环境中，运维人员可将常用内部 Wiki、制品库地址与监控面板链接通过 LinkVault 生成离线索引快照，便于在跳板机上无网络访问时快速查阅。

## 快速开始

以下指令演示如何在本地环境克隆仓库、安装依赖并启动开发服务器。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（使用 Go Modules 或 pnpm，根据实际语言调整）
go mod download
# 或者
pnpm install

# 运行本地开发服务器
go run cmd/linkvault/main.go serve --port 8080
# 或者
pnpm run dev
```

启动成功后，访问 http://localhost:8080 即可浏览默认的资源索引页面。如需导入用户提供的 URL 列表，请将链接文件存放于 `data/sources/` 目录下，然后执行 `linkvault import --source data/sources/links.txt`。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Go 语言环境 | 1.21 及以上 | 核心后端服务编译依赖，若使用预编译二进制则无需安装 |
| Node.js 运行时 | 18.x 或 20.x LTS | 仅当启用前端仪表盘或静态站点生成功能时需要 |
| Git 版本控制 | 2.25 及以上 | 用于克隆仓库及获取子模块更新 |
| Make 构建工具 | 3.82 及以上 | 提供统一的构建命令入口，非强制但推荐 |
| curl 或 wget | 任意稳定版本 | 用于健康检查模块中的 HTTP 探测，系统通常预装 |
| SQLite 数据库驱动 | 内置（CGO 可选） | 默认使用纯 Go 实现的 SQLite 驱动，无需外部安装 |

## 项目结构

项目采用标准的 Go 语言工程布局，同时包含前端资源目录。以下为核心目录树结构及其功能说明。

```
linkvault/
├── cmd/                                 # 可执行程序入口目录
│   └── linkvault/                       # 主命令行工具入口
│       └── main.go                      # 解析子命令、初始化配置并启动服务
├── internal/                            # 内部私有包，不对外暴露
│   ├── collector/                       # 资源抓取与链接解析模块
│   │   ├── parser.go                    # 从 Markdown/HTML/YAML 提取 URL 及元数据
│   │   └── dedupe.go                    # 基于域名与路径指纹的去重逻辑
│   ├── health/                          # 链接可用性健康检查模块
│   │   ├── checker.go                   # 并发 HTTP 探测与超时控制
│   │   └── reporter.go                  # 生成异常状态汇总报告
│   ├── index/                           # 内存索引与 Bleve 搜索引擎适配层
│   │   ├── bleve_adapter.go             # 索引创建、更新与查询接口实现
│   │   └── tokenizer.go                 # 自定义路径分词器，优化短路径匹配
│   └── storage/                         # 持久化存储抽象层
```

## 常见问题

**Q: LinkVault 是否支持私有化部署的 GitLab 或 Gitea 作为资源来源？**

A: 目前内置的导入适配器仅支持本地文件与标准 HTTP/HTTPS 公开链接。若需要从私有 Git 仓库或 API 端点抓取资源，您可以在 `internal/collector/` 下自行实现一个适配器，参考现有 `parser.go` 中的接口定义即可。我们欢迎社区贡献此类适配器。

**Q: 健康检查模块会频繁请求外部链接，是否会造成目标服务器压力？**

A: 默认配置下，健康检查以每 24 小时一次的频率执行，且并发请求数限制为 10，避免突发流量。您可以在 `configs/linkvault.yaml` 中调整 `check_interval` 与 `concurrency` 参数。对于大规模资源列表（超过 500 条），建议将检查分散至多台机器或使用随机延迟抖动。

**Q: 如何迁移已有的浏览器书签到 LinkVault？**

A: 您可以从 Chrome 或 Firefox 导出书签为 HTML 文件，然后放置于 `data/sources/` 目录，执行 `linkvault import --from-html bookmarks.html`。该命令会自动解析 `<a>` 标签的 `href` 与文本内容，生成初始索引。对于其他格式，请参考文档导航中的导入适配器说明。
