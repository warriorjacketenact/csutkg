# EON Resource Gateway

EON Resource Gateway 是一个面向技术社区与开发者群体的外链资源整合与导航系统。项目定位于对分散在网络各处的优质技术文档、工具链入口、社区教程及知识库页面进行结构化归集，并提供统一的访问入口与分类索引。项目目标用户包括开源贡献者、DevOps 工程师、技术内容管理者以及需要持续跟踪多源技术信息的研究人员。EON Resource Gateway 通过将大量外部链接以元数据表格与分类标签的形式组织起来，解决了个人书签管理混乱、团队知识共享困难、新成员上手缺少权威入口等问题，同时也为自动化巡检与链接可用性监控提供了数据基础。

## 功能概览

**链接分类索引** 系统依据 URL 路径前缀与来源域自动将资源归入技术文档、社区讨论、工具镜像、教程指南等类别，并生成分类视图。

**元数据提取与标签化** 对每个收录链接自动抓取页面标题、描述与关键词，生成可搜索的标签集合，便于按主题或技术栈筛选。

**周期性可用性探测** 内置轻量级巡检任务，按可配置周期对全部收录链接发起 HTTP 请求，检测状态码与响应时间，标记异常条目。

**团队共享书签池** 支持多用户协作，允许团队成员共享资源列表、添加个人备注与推荐优先级，形成组织内部的集体知识库。

**导入导出与备份** 提供 JSON 与 CSV 格式的链接数据导入导出能力，方便迁移、备份或与其他自动化工具集成。

**访问统计与热度排序** 记录每个链接的点击次数与最近访问时间，支持按热度、新增时间、响应速度等维度排序。

**自定义标签与目录树** 用户可创建多级目录结构，将链接拖拽至自定义分类节点，并批量应用标签以增强检索效率。

**公开页面发布** 支持将选定分类或标签下的链接生成为静态导航页面，便于对外发布或嵌入团队文档站点。

## 应用场景

**技术团队新成员入职引导** 新加入的开发人员往往需要翻阅大量内部文档与外部参考资源。EON Resource Gateway 可将团队常用的编程规范、框架文档、CI/CD 指南、监控面板入口等统一收录，新成员仅需访问网关首页即可获得完整的资源地图。

**开源项目外部资源聚合** 开源项目维护者通常会在 README 或 Wiki 中引用大量第三方依赖、学习资料与社区论坛。借助 EON Resource Gateway，这些链接可以被集中管理并对外暴露为稳定的导航页面，同时通过可用性检测及时下架失效链接，提升文档质量。

**技术博客与教程站点的延伸导航** 技术博主或在线教育平台可为自己的文章或课程配套部署一个资源网关，将每篇文章提到的参考链接、代码仓库、在线工具等统一收录，并按照文章分类或标签进行组织，为读者提供一站式的延伸阅读入口。

**自动化链接巡检与告警** 运维或内容运营团队可部署 EON Resource Gateway 作为内部巡检系统的一部分，定期扫描所有收录的外部链接，当检测到大量 4xx 或 5xx 响应时触发告警，帮助团队及时发现外部依赖的异常状态。

## 快速开始

以下步骤将在本地环境启动 EON Resource Gateway 实例，默认使用 SQLite 数据库存储链接元数据与用户配置。

```bash
git clone https://github.com/eon-resource/gateway.git
cd gateway
pip install -r requirements.txt
python manage.py migrate
python manage.py load_initial_links
python manage.py runserver 0.0.0.0:8000
```

执行上述命令后，服务将在本地 8000 端口启动。访问 http://localhost:8000 即可进入网关首页，页面将加载已初始化的链接目录。如需添加新链接，可使用管理员账号登录后台管理界面，或通过命令行导入工具批量导入。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.9 - 3.11 | 是 | 核心运行时环境，推荐使用 3.10 版本以获得最佳兼容性 |
| Django 4.2 LTS | 是 | Web 框架与 ORM 层，用于提供管理后台与 API 接口 |
| SQLite 3.35+ | 是 | 默认数据库引擎，支持 JSON 字段与全文检索 |
| requests 2.28+ | 是 | 用于执行链接可用性探测与页面元数据抓取 |
| celery 5.3+ | 否 | 若需启用周期性巡检任务，需配置 Redis 或 RabbitMQ 作为消息代理 |
| redis 7.0+ | 否 | 用于缓存与任务队列后端，可选但推荐在生产环境部署 |
| nginx 1.22+ | 否 | 生产环境反向代理与静态文件服务，可选用其他 Web 服务器 |
| docker 24+ | 否 | 若需容器化部署，需安装 Docker Engine 与 Compose 插件 |

## 项目结构

```
gateway/
├── manage.py                      # Django 管理入口，用于启动服务与执行命令
├── requirements.txt               # Python 依赖声明，包含 Django、requests、celery 等核心库
├── config/                        # 项目全局配置目录
│   ├── settings.py                # 基础配置，含数据库、中间件、模板、静态文件设置
│   ├── settings_local.example.py  # 本地开发覆盖配置示例，含密钥、调试模式、数据库连接串
│   └── urls.py                    # 根路由分发，映射后台管理、API 端点与前端视图
├── apps/                          # 功能模块聚合目录
│   ├── links/                     # 链接管理核心模块，处理链接 CRUD、分类、标签与元数据
│   │   ├── models.py              # Link、Category、Tag、AccessLog 等数据模型定义
│   │   ├── views.py               # 列表视图、详情视图、API 视图函数
│   │   ├── services.py            # 元数据抓取、链接可用性检测、热度计算等业务逻辑
│   │   └── admin.py               # Django 后台管理界面定制，支持批量导入导出
│   ├── users/                     # 用户与权限模块，扩展 Django 自带认证系统
```

## 常见问题

**Q: 如何更新已收录链接的元数据？**

A: 系统默认每 72 小时自动执行一次元数据刷新任务。如需手动更新，可登录管理后台，在链接列表页面勾选目标条目后选择「刷新元数据」操作，或通过命令行执行 `python manage.py refresh_metadata --url https://www.eoncn.org/view/xxxx.html` 对单个链接进行更新。

**Q: 周期性巡检任务未按预期执行，应如何排查？**

A: 首先确认 celery worker 进程是否正常运行，可使用 `celery -A config status` 检查 worker 状态。其次查看 `logs/tasks.log` 中是否有连接消息代理失败或超时的错误记录。若使用 Redis 作为 Broker，需确保 Redis 服务已启动且配置的数据库索引与密码正确。

**Q: 导入大量链接时出现超时或内存占用过高的问题？**

A: 建议分批导入，每次不超过 500 条。可使用 `scripts/import_csv.py` 并提供 `--batch-size` 参数控制单次提交数量。若数据量极大，可临时调整数据库连接池大小与 Django 的请求超时配置。
