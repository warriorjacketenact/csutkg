# Tangaloomahouse Film Resource Aggregator

Tangaloomahouse Film Resource Aggregator 是一个面向影视技术研究者、数字内容策展人及媒体资产管理工作者的高效外链资源汇总工具。该项目并非传统意义上的电影播放或下载平台，而是一个高度结构化的索引系统，用于收录、分类和快速检索来自特定源站的大量深度链接。

该项目专为解决影视行业从业者在面对海量分散的后期制作素材、技术文档、参考样片及项目源文件时，缺乏统一入口和高效管理工具的痛点而设计。通过将数千个动态生成的深层链接转化为可读、可维护的静态资源目录，本项目极大地提升了影视项目协作中资源发现的效率，降低了对原始数据库查询的依赖。

## 功能概览

**结构化链接索引**：将原始动态URL按照编码规则和路径层级进行归集，提供清晰的目录树视图，方便用户按项目批次或功能模块浏览资源。

**快速模糊检索**：内置基于文件路径哈希值的轻量级检索算法，允许用户通过记忆中的片段代码或文件名关键字，在本地索引中快速定位目标资源。

**资源状态快照**：定期抓取并记录目标链接的响应状态与基础元数据，生成资源可用性报告，帮助用户识别失效或迁移的链接。

**批量导出与集成**：支持将选定的资源列表批量导出为CSV或JSON格式，便于与其他媒体资产管理软件或自动化脚本进行集成。

**本地缓存代理**：提供可选的本地缓存服务，对频繁访问的静态资源（如技术规格PDF、示例短片）进行临时存储，减轻源站压力并提高访问速度。

**自定义标签系统**：允许用户为收录的链接添加自定义标签和备注，构建符合个人或团队工作流的分类体系。

**变更日志追踪**：记录资源库的每一次更新操作，包括新增链接、移除失效链接和元数据修改，确保团队协作过程中的数据可追溯性。

## 应用场景

影视后期制作公司的技术总监可利用本工具对公司购买或订阅的外部素材库、插件下载页、渲染节点文档进行统一归集，使团队成员无需记忆复杂路径即可快速获取所需技术资源。

独立电影研究者或策展人在分析特定制片厂或导演的数字化作品集时，可通过本项目提供的结构化索引，高效梳理大量散落在不同子目录中的样片片段、访谈文稿和幕后照片。

媒体资产管理系统的运维人员可将本项目的导出功能作为数据迁移或备份的前置步骤，通过标准化的链接列表快速验证外部依赖资源的完整性和可达性，从而制定更稳健的数据备份策略。

## 快速开始

以下步骤将引导您在本机快速部署并运行 Tangaloomahouse Film Resource Aggregator 的核心索引服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/tangaloomahouse/film-resource-aggregator.git

# 进入项目根目录
cd film-resource-aggregator

# 安装项目依赖（使用 npm）
npm install

# 构建索引数据库并启动本地开发服务器
npm run build && npm start
```

## 安装要求

| 依赖 | 必需 | 说明 |
| :--- | :--- | :--- |
| Node.js | 是 | 版本需 >= 18.0.0，用于运行核心索引构建脚本及本地服务 |
| npm | 是 | 版本需 >= 9.0.0，用于管理项目依赖包 |
| SQLite3 | 是 | 用于存储和查询链接索引数据，项目内已包含驱动，无需额外安装数据库服务 |

| 现代浏览器 | 否 | 仅用于查看生成的静态报告页面，无特定版本要求 |

## 常见问题

**问：我是否需要配置额外的数据库服务才能运行此项目？**

答：不需要。本项目默认使用嵌入式的 SQLite3 作为存储引擎，所有数据都存储在项目 `data/` 目录下的单一文件中，无需安装或配置独立的数据库服务进程。这大大降低了部署和测试的复杂度。

**问：如果原始链接（如资源列表中的 URL）返回 404 或连接超时，我应该怎么办？**

答：本项目作为资源索引工具，并不负责托管或修复原始内容。当遇到失效链接时，建议您先确认网络环境是否可正常访问 `www.tangaloomahouse.com` 域名。若确定链接已失效，您可以通过项目的标签系统将其标记为“失效”，或使用 `snapshot` 命令生成的状态报告来批量识别并过滤掉这些链接。对于长期失效且无法恢复的链接，您可以在本地索引中将其移除。

**问：如何更新已收录的链接列表以匹配源站的最新变更？**

答：项目提供了 `npm run build` 命令，该命令会根据配置的源站路径规则重新执行抓取和解析流程。对于本批次（第 469/1241 批）的 250 个链接，您也可以通过 `src/indexer/crawler.js` 中的配置调整抓取深度和过滤规则，然后重新运行构建命令来生成最新的索引快照。
