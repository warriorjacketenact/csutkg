# ResourceBridge

ResourceBridge 是一个面向技术社区的开源外链资源汇总与导航系统。该项目不直接存储任何内容，而是通过人工筛选与社区提交的方式，将分散在互联网各处的优质技术文档、工具站点、教程文章、行业报告及开源组件索引至统一平台，帮助开发者、研究人员与技术决策者快速定位高价值外部信息。

项目定位为轻量级资源路由层，适用于个人知识库补充、团队技术选型参考、企业内网导航镜像站等场景。ResourceBridge 本身不依赖外部数据库，所有资源条目以 Markdown 及 YAML 形式存储在仓库中，支持静态站点生成、本地检索与自定义分类。

## 功能概览

**按需分类索引**：资源按技术领域、适用人群、内容形式、更新活跃度等多维度划分，支持快速过滤。

**批量导入与校验**：提供脚本工具，支持从 CSV、JSON 及原始 URL 列表批量导入资源条目，并自动检查链接可访问性。

**全文检索支持**：内置基于 Lunr.js 的轻量级全文搜索，可对资源标题、描述、标签及分类进行即时检索。

**资源状态监控**：定时任务每日检测已收录资源的 HTTP 状态码与响应时间，标记失效或缓慢资源。

**社区提交工作流**：通过 GitHub Issue 模板与 Pull Request 流程，允许社区成员提交新资源或更新现有条目。

**静态站点生成适配**：提供 Jekyll、Hugo 及 VuePress 三套输出模板，可直接生成可部署的静态导航页面。

**自定义标签体系**：管理员可自定义标签层级与颜色映射，实现个性化资源归类。

**访问统计聚合**：集成简单的事件追踪接口，统计各资源被点击的次数与趋势，辅助热度排序。

## 应用场景

**技术团队内部知识库补充**：技术负责人可将 ResourceBridge 部署为团队内部导航页，收录公司常用依赖库镜像、内部文档入口、运维监控面板地址，避免重复沟通与书签丢失。

**开源项目文档站外链管理**：开源项目维护者可在项目文档中嵌入 ResourceBridge 生成的资源列表，为贡献者提供外部参考链接，涵盖编码规范、设计模式、测试工具等延伸阅读材料。

**技术社区资源聚合页面**：技术社区或论坛运营方可使用 ResourceBridge 构建每周精选资源汇总，将分散在微信群、邮件列表中的优质链接结构化存档，便于新成员回溯历史内容。

**个人开发者学习路线参考**：个人开发者可 Fork 该项目，根据自身学习方向创建私有资源集合，例如前端工程化、Go 语言微服务、机器学习基础等专题链接库，并利用检索功能快速回忆相关知识点。

## 快速开始

以下步骤帮助您在本地环境中快速启动 ResourceBridge 实例，进行资源浏览与开发调试。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resourcebridge/resourcebridge.git

# 进入项目根目录
cd resourcebridge

# 安装项目依赖（基于 Node.js 18+ 与 npm 9+）
npm install

# 执行资源索引构建与本地服务器启动
npm run build && npm run serve
```

执行完毕后，访问控制台输出的本地地址（默认 http://localhost:8080）即可查看资源导航界面。若需自定义资源数据，请编辑 `data/resources.yaml` 文件后重新执行 `npm run build`。

|---|---|---|
| Node.js | 18.x 或更高 | 运行时环境，用于执行构建脚本与本地服务 |
| npm | 9.x 或更高 | 包管理器，用于安装项目依赖 |
| Git | 2.30 或更高 | 版本控制工具，用于克隆仓库及提交变更 |

| 可选：Docker | 20.10 或更高 | 如需容器化部署，可选用 Docker 方式运行 |

## 项目结构

```
resourcebridge/
├── bin/                                 # 可执行脚本与命令行工具
│   ├── import.js                        # 批量导入资源条目（支持 CSV/JSON）
│   ├── validate.js                      # 校验所有资源链接的可访问性
│   └── stats.js                         # 统计资源数量、分类分布与状态
├── config/                              # 项目配置文件目录
│   ├── site.yaml                        # 站点标题、描述、主题色等基础配置
│   ├── taxonomy.yaml                    # 分类与标签的层级定义
│   └── validator.yaml                   # 链接校验的超时、重试策略配置
```

## 贡献指南

我们欢迎所有形式的贡献，包括但不限于新增资源链接、修复错误链接、改进文档、提交代码优化建议。

1. 阅读项目行为准则与贡献规范文档（位于 docs/contributing/ 目录下），确保您的贡献符合社区共识。
2. 如需新增或更新资源，请先在 data/resources.yaml 中按已有格式添加条目，并执行 npm run validate 校验链接可访问性。
3. 提交 Pull Request 前，请运行 npm run test 确保所有单元测试及集成测试通过，并更新对应文档（如涉及功能变更）。
4. 若发现链接失效或内容不准确，请通过 GitHub Issue 提交错误报告，标题注明 [Broken Link] 或 [Inaccurate Content]。
5. 对于较大功能改动或架构调整，建议先创建 Issue 进行设计讨论，避免无效开发。

## 常见问题

**问：ResourceBridge 收录资源的标准是什么？是否接受商业性质链接？**

答：收录标准以技术相关性、内容质量、信息密度为核心考量。开源项目、技术博客、官方文档、工具站点均可提交。商业性质链接若提供实质性的技术参考价值（如云服务官方文档、API 调试工具、性能监控平台），亦可收录，但需在描述中明确标注商业属性。纯粹的广告落地页、无实质内容或频繁弹出广告的网站不予收录。

**问：我提交的资源链接长时间未被合并，可能原因是什么？**

答：常见原因包括：链接校验失败（返回 4xx/5xx 状态）、资源描述模糊或过于简略、分类标签选择不当、链接内容与现有资源高度重复。您可以在 Pull Request 或 Issue 中查看管理员反馈，补充必要信息后重新提交。若校验工具提示链接可访问性不稳定，建议提供备选镜像地址。

**问：能否将 ResourceBridge 部署到内网环境，完全不依赖公网访问？**

答：可以。ResourceBridge 核心功能不依赖外部 API，您只需在部署前执行一次 npm run build 生成完整静态文件，然后将构建产物（位于 dist/ 目录）复制到内网 Web 服务器即可。资源链接校验功能在内网环境中需要额外配置代理或关闭校验，具体操作参考 docs/deployment/offline.md 文档。
