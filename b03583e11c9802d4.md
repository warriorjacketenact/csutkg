# EonCN Resource Aggregator

EonCN Resource Aggregator 是一个面向技术研究者、开源开发者与数据从业者的外链资源汇总工具。项目定位于对分散在各技术站点、文档系统与社区平台中的高质量外链进行集中收录、分类索引与状态监控，帮助用户以极低成本建立个人或团队的外链知识库。

该工具解决的核心问题是技术资源链接散落于收藏夹、即时通讯记录、邮件往来与临时笔记中，检索效率低下且易丢失。通过统一入口管理外部 URL，辅以标签体系、存活检测与访问统计，用户可以快速定位所需资源，同时追踪链接可用性变化，避免因目标站点迁移或内容下架导致的引用失效。项目特别适用于需要长期维护技术选型参考、学习路径规划或开源组件清单的开发者群体。

## 功能概览

**批量外链导入**：支持从 CSV、JSON 及纯文本列表批量导入外部 URL，自动解析标题与元数据，减少手工录入成本。

**链接存活状态监控**：定时发起 HEAD 请求检测目标资源可访问性，标记异常状态码与响应超时，输出健康度报表。

**多维度标签分类**：允许用户为每条链接自定义标签集合，支持按标签组合过滤检索，实现精细化资源管理。

**访问频次统计**：记录每个外链的点击次数与最后访问时间，辅助判断资源实际使用价值，为定期清理提供数据依据。

**全文搜索与快速跳转**：基于标题、描述与标签构建倒排索引，支持模糊搜索，点击条目即时在新标签页打开目标链接。

**数据导入导出**：提供标准格式的数据备份与恢复功能，便于在不同部署实例间迁移资源列表，也支持与第三方工具集成。

**链接变更追踪**：对比历史快照，识别目标页面标题或描述的重大变化，提示用户复核资源内容是否仍符合预期。

## 应用场景

技术团队内部知识库建设：团队可将日常开发中遇到的官方文档、API 参考、博客文章、调试工具等链接统一录入系统，新成员入职时可直接浏览已分类资源列表，快速熟悉技术栈与常用工具链，显著缩短上手周期。

开源项目依赖管理辅助：开源维护者可将项目依赖的第三方库主页、镜像源地址、安全公告页面等外链集中管理，定期检查链接可用性，确保依赖资源始终可访问，降低因外部资源不可用导致的构建失败风险。

个人学习路径规划与回顾：学习者按主题（如机器学习、分布式系统、前端框架）创建资源集合，记录每个阶段参考的教程、论文与视频链接，通过访问统计了解自己的学习活跃度，定期清理已掌握内容的链接，保持资源库精炼有效。

技术选型调研记录：架构师在评估多个候选方案时，将各方案的官网、性能测试报告、社区讨论帖、漏洞披露记录等链接统一归档，配合标签区分维度（性能、生态、许可证），形成结构化的选型决策依据。

## 快速开始

以下命令适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆代码仓库
git clone https://github.com/eoncn/aggregator.git
cd aggregator

# 安装项目依赖（使用 pip 和虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动服务
python scripts/init_db.py
python app.py --host 0.0.0.0 --port 8080
```

服务启动后，访问 http://localhost:8080 即可进入资源管理界面。首次运行将自动创建 SQLite 数据库文件 `data/resources.db` 并生成默认管理员账户，登录信息可在启动日志中查看。

│   │   ├── health.py                   # 存活检测服务（并发请求、超时控制）
│   │   └── search.py                   # 全文检索引擎（倒排索引构建与查询）

├── scripts/                            # 运维与辅助脚本
│   ├── init_db.py                      # 数据库初始化（建表、默认数据）
│   ├── import_batch.py                 # 批量导入命令行工具

├── tests/                              # 单元测试与集成测试

第一步：阅读开发者文档 docs/developer-guide.md 了解代码架构、编码规范与测试要求，确保本地环境已安装所有开发依赖（pytest、black、flake8）。

## 常见问题

Q：导入大量外链时页面响应缓慢或超时，应如何优化？

A：建议采用异步导入模式。在管理界面中选择“后台导入”选项，系统将任务放入 APScheduler 队列中异步执行，导入完成后通过邮件或站内通知告知结果。对于超过 1000 条记录的批量导入，推荐使用命令行脚本 scripts/import_batch.py 并指定 --chunk-size 参数分批提交，避免单次请求负载过高。同时可调整 config/production.py 中的 PRELOAD_MAX_WORKERS 值控制并发解析线程数。

Q：链接存活检测出现误报（目标站点正常但标记为异常），如何处理？

A：检测误报通常由网络策略、防火墙或目标站点的反爬机制引起。可依次检查以下配置：调整 HEALTH_CHECK_TIMEOUT 增加到 10 秒以上；设置 HEALTH_CHECK_HEADERS 中的 User-Agent 为常见浏览器标识；若目标站点位于特定地域，可配置 HEALTH_CHECK_PROXIES 使用代理。此外，可在管理界面手动对该链接执行“重新检测”并查看详细响应头信息，辅助判断实际返回状态。
