# RSXZS Technical Resource Aggregator

RSXZS 是一个面向开发人员、系统架构师和技术研究者的综合性技术资源外链汇总系统。本项目不直接托管任何内容，而是通过严格的来源筛选和分类索引机制，为技术社区提供高质量的外部链接导航服务。

本项目的目标用户包括正在进行技术选型的工程师、需要快速查阅特定领域资料的研究人员，以及希望跟踪技术动态的团队管理者。RSXZS 通过人工审核与自动化校验相结合的方式，确保每一条收录的链接均具备可访问性、内容相关性和技术深度，从而帮助用户节省信息检索时间，聚焦于实际问题解决。

## 功能概览

- **多维度分类索引**：按技术领域、应用场景、难度等级和内容形式对每一条链接进行标签化分类，支持快速筛选。

- **链接存活状态监测**：系统定期对收录的 URL 进行 HTTP 请求校验，自动标记失效链接并在管理后台生成告警日志。

- **全文元数据提取**：对每个目标页面提取标题、描述、关键词、发布时间和正文摘要，生成结构化元数据供前端展示。

- **用户自定义收藏夹**：注册用户可将常用链接加入个人收藏夹，并支持为收藏条目添加自定义备注和标签。

- **链接关系图谱**：基于链接之间的引用关系、主题重叠度和共同标签，自动生成可视化知识图谱，辅助用户发现关联资源。

- **访问统计与热度排序**：记录每个链接的被点击次数、被收藏次数和用户评分，支持按综合热度、最新添加和最高评分三种方式排序。

- **RSS 订阅源生成**：为每个分类和标签生成独立的 RSS 订阅链接，方便用户通过 RSS 阅读器跟踪更新。

- **API 查询接口**：提供 RESTful API 供第三方工具调用，支持按关键词、标签、域名和日期范围进行精确查询。

## 应用场景

- **技术选型阶段的横向对比**：当团队需要从多个候选方案中选择最适合的技术栈时，可通过本系统快速查找各方案的官方文档、社区讨论、性能评测和实战案例，汇总后形成对比分析报告。

- **新员工技术培训路径制定**：技术负责人可为新入职的开发人员定制学习路径，利用本系统按难度层级和主题分类的链接资源，安排从入门到进阶的系统化阅读与实操计划。

- **技术分享会素材收集**：计划进行内部分享或外部技术演讲的工程师，可使用本系统按主题标签检索相关的最新文章、论文和项目案例，快速构建分享大纲和演示文稿的参考资料。

- **遗留系统重构前的调研**：在启动遗留系统重构之前，架构师可通过本系统查阅同类系统的重构经验、常见陷阱和最佳实践，结合链接关系图谱发现潜在的风险点和可复用的模式。

- **技术博客与文档写作参考**：技术写作者在撰写博客或技术文档时，可通过本系统检索权威引用源，确保文中引用的数据、案例和规范有据可查，提升内容的可信度。

## 快速开始

以下步骤帮助您在本地环境中快速启动 RSXZS 聚合器的开发实例。

```bash
# 克隆项目仓库
git clone https://github.com/rsxzs/aggregator.git
cd aggregator

# 安装项目依赖（使用 Python 虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化本地数据库并导入示例链接数据
python manage.py migrate
python manage.py import_links --source data/seed_links.json

# 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8080
```

完成上述步骤后，访问 http://localhost:8080 即可在本地查看聚合器主页。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 或更高 | 核心应用运行环境，推荐使用 3.11 长期支持版本 |
| PostgreSQL | 14.x 或更高 | 主数据库，用于存储链接元数据、用户信息和访问日志 |
| Redis | 7.0 或更高 | 缓存层，用于加速热点链接的访问统计和会话管理 |
| Node.js | 18.x 或更高 | 前端资源构建工具链，用于编译静态资源（仅开发环境） |
| Nginx | 1.24 或更高 | 生产环境反向代理服务器，负责负载均衡和静态资源分发 |
| Docker | 24.x 或更高 | 容器化部署可选依赖，用于快速搭建完整的服务栈 |
| Celery | 5.3 或更高 | 异步任务队列，用于定期执行链接存活检测和元数据更新 |

## 贡献指南

RSXZS 项目欢迎社区贡献者参与链接收录、功能开发和文档完善。请遵循以下步骤提交贡献。

1. 查阅问题跟踪器中的待办事项列表，选择未被分配且符合自身技能范围的 Issue，在评论区留言认领。首次贡献者建议从标记为 good-first-issue 的条目开始。

2. 派生本仓库至个人账号，在派生仓库中创建以 issue-编号 命名的功能分支，遵循项目既定的代码风格（Python 代码遵循 PEP 8，JavaScript 遵循 Airbnb 风格指南）进行开发。

3. 编写或更新对应的单元测试与集成测试用例，确保测试覆盖率达到 80% 以上。本地执行测试套件全部通过后方可提交。

4. 提交 Pull Request 至主仓库的 main 分支，PR 描述中须清晰说明解决的问题、实现方案和测试结果。项目维护者将在 7 个工作日内进行代码审查并提出修改意见。

5. 对于链接收录类贡献，请提交包含目标 URL、推荐分类和简短摘要的收录申请，经审核通过后合并入正式链接库。重复或已失效的链接申请将被退回。

## 常见问题

**问：系统收录链接的筛选标准是什么？是否存在内容类型限制？**

答：系统优先收录内容质量高、更新频率稳定、域名信誉良好的技术类站点。具体标准包括：页面内容与软件开发、系统运维、数据科学、人工智能或信息安全等主题直接相关；无恶意弹窗、强制下载或钓鱼行为；站点在过去 6 个月内持续发布新内容。商业推广类、纯电商页面和个人博客中内容明显重复或采集的链接不予收录。

**问：我发现某个已收录的链接已失效或内容严重过时，应该如何处理？**

答：用户可通过页面下方的 报告问题 按钮提交失效反馈，系统将自动记录并进入人工复核队列。管理员收到反馈后在 3 个工作日内确认链接状态，若确认失效则会从索引中移除并更新统计记录。亦可通过 GitHub Issue 提交链接失效报告，标题请使用 [Link Broken] 前缀以加速处理。

**问：是否支持导入我自己的链接收藏夹（如浏览器书签）到本系统？**

答：目前系统支持从 Netscape Bookmark HTML 格式（绝大多数浏览器支持导出）批量导入。用户登录后在 个人设置 - 导入书签 页面上传导出文件，系统将自动解析并提取有效 URL，根据页面标题和域名生成初步分类建议，用户确认后完成导入。该功能当前处于 Beta 阶段，单次导入上限为 1000 条。
