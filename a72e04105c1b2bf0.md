# Nexus Index

Nexus Index 是一个面向技术研究者、数据工程师与开源项目维护者的结构化外链与媒体资源索引系统。本项目不提供内容托管，而是以高度组织化的目录形式，对分散在多个技术媒体、企业公告与行业报告中的高价值链接进行归类、描述与快速检索。Nexus Index 的目标用户包括需要追踪特定技术动态的开发者、进行竞品分析的产品经理，以及需要引用权威来源的技术写作者。

本项目解决的核心问题是：技术信息碎片化导致的关键资源查找效率低下。通过建立统一的索引层，用户无需在多个站点间反复切换，即可在一个确定的命名空间内定位到所需的原始材料。Nexus Index 本身不存储任何版权内容，仅作为指向原始发布页面的导航锚点。

## 功能概览

**结构化索引目录** 提供按技术领域、发布时间与资源类型三级筛选的链接列表，每个条目均附带原始来源的完整元数据描述。

**批量资源导入** 支持通过 CSV 或 JSON 格式批量新增链接记录，自动校验 URL 可用性并提取页面标题与摘要信息。

**状态监控面板** 实时检测已收录链接的 HTTP 响应状态，对返回 4xx 或 5xx 状态的资源进行高亮标记并生成异常报告。

**标签与分类体系** 内置覆盖编程语言、基础设施、安全合规、人工智能等 12 个一级分类及 64 个二级标签，支持自定义扩展。

**全文检索接口** 基于标题、描述文本与标签组合的模糊搜索功能，返回结果按相关度与时间权重排序。

**版本化快照记录** 对每次索引更新操作生成变更日志，记录新增、移除与修改的条目，支持回溯至任意历史版本。

**外部引用计数** 统计每个链接在公开技术文章、论坛与代码仓库中被引用的近似次数，辅助判断资源影响力。

**数据导出工具** 将当前索引全量或按筛选条件导出为 Markdown、HTML 或纯文本表格格式，便于离线查阅或嵌入文档。

## 应用场景

技术调研阶段的快速资料收集。当研发团队计划引入新的中间件或评估云服务方案时，可通过 Nexus Index 在数分钟内聚合来自官方公告、技术博客与行业测评的相关链接，大幅缩短信息搜集周期。

文档编写过程中的外链引用管理。技术作者在撰写设计文档、API 说明或用户手册时，可使用本索引查找权威的外部参考链接，确保引用来源的准确性与可追溯性，避免链接失效导致的文档质量下降。

开源项目 README 与网站的资源附录维护。项目维护者可将 Nexus Index 作为子模块集成至项目文档体系，自动生成并更新外部依赖、参考文献与致谢列表，保持与上游资源的同步。

技术会议与培训材料的配套资料包。讲师或组织者可以基于特定标签筛选索引内容，生成与课程主题强相关的阅读材料清单，分发给学员作为课后扩展阅读的起点。

## 快速开始

以下命令序列适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆仓库至本地
git clone https://github.com/nexus-index/nexus-index.git
cd nexus-index

# 安装依赖（项目基于 Python 3.10 与 Pipenv）
pip install pipenv
pipenv install --dev

# 启动本地索引服务（默认监听 127.0.0.1:8080）
pipenv run python server.py
```

访问 http://127.0.0.1:8080 即可浏览索引界面。首次启动时会自动执行初始数据导入与索引构建，耗时约 30 秒。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10.x 或 3.11.x | 核心运行时，3.12 版本暂未完成兼容性测试 |
| Pipenv | 2023.x.x 或更新 | 依赖管理与虚拟环境隔离工具 |
| SQLite | 3.35.0 或更高 | 嵌入式数据库，用于存储索引条目与元数据 |
| Git | 2.30.0 或更高 | 版本控制与仓库克隆，仅开发时必需 |
| curl | 7.68.0 或更高 | 用于健康检查脚本与外部 URL 状态探测 |
| tzdata | 系统时区包 | 确保时间戳记录的时区正确性，Linux 下通常预装 |

## 项目结构

```
nexus-index/
├── server.py                 # 主入口，启动 HTTP 服务与调度后台任务
├── Pipfile                   # 项目依赖声明，含生产与开发分组
├── Pipfile.lock              # 锁定所有依赖的精确版本哈希
├── config/
│   ├── settings.yaml         # 环境变量覆盖配置（端口、数据库路径、超时阈值）
│   └── schema.json           # 索引条目的 JSON Schema 验证规则
├── core/
│   ├── __init__.py
│   ├── indexer.py            # 核心索引构建逻辑，含倒排表生成与权重计算
│   ├── crawler.py            # 异步 URL 状态探测与摘要提取
│   └── storage.py            # SQLite 读写封装，含连接池与事务管理
├── api/
│   ├── routes.py             # RESTful 路由定义（检索、导出、监控）
│   └── middleware.py         # 请求日志、错误处理与速率限制中间件
├── web/
│   ├── static/               # CSS、JavaScript 与图标静态资源
│   └── templates/            # Jinja2 模板，含首页、详情页与管理面板
├── scripts/
│   ├── import_batch.py       # 批量导入命令行工具，支持 CSV/JSON
│   ├── health_check.py       # 定时巡检所有链接状态，输出报告
│   └── migrate_db.py         # 数据库版本迁移脚本（使用 Alembic）
├── tests/
│   ├── test_indexer.py       # 单元测试：索引构建与检索准确度
│   └── test_crawler.py       # 模拟 HTTP 响应测试爬虫鲁棒性
├── docs/                     # 完整文档源文件（用户手册、维护指南、API 参考）
└── LICENSE                   # MIT 许可证全文
```

## 常见问题

问：索引中的链接无法访问时应该如何处理？

答：项目内置的健康检查脚本 `scripts/health_check.py` 默认每 24 小时运行一次，检测到失效链接后会将其状态标记为 "unreachable" 并在管理面板中高亮显示。用户也可通过 Web 界面手动触发重新检测或提交 "链接失效" 反馈。对于持续不可达的资源，维护团队会在每月底进行统一清理，并尝试在原始域名下查找替代路径。

问：能否将 Nexus Index 部署到内网环境且不依赖外部网络？

答：可以。Nexus Index 的核心功能（检索、分类、导出）完全基于本地 SQLite 数据库运行，无需任何外部 API 调用。仅在首次部署时的初始数据导入阶段需要访问配置文件中列出的原始 URL 以提取元数据。若完全离线使用，用户可事先将所有索引条目的标题与描述整理为 CSV 文件，通过 `scripts/import_batch.py` 的 `--offline` 模式直接导入，跳过实时抓取步骤。

问：如何备份或迁移我的索引数据？

答：所有索引数据存储在项目根目录下的 `data/nexus.db` 文件中。备份时直接复制该文件即可。迁移至新环境时，需确保新环境的 Python 与 SQLite 版本满足安装要求，将 `nexus.db` 文件放置于相同相对路径，然后正常启动 `server.py`。若需同时迁移自定义标签与分类配置，需额外复制 `config/settings.yaml` 中的 `taxonomy` 字段内容。
