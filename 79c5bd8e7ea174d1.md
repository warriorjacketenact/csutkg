# RSXZS Resource Navigator

RSXZS Resource Navigator 是一个面向技术研究人员、开源开发者与数据分析师的高质量外链资源汇总平台。本项目系统化收录来自 rsxzs.com 社区的深度技术讨论帖与专题文章，覆盖云计算、容器化、数据库运维、编程语言实践、系统架构设计及网络安全等核心领域，旨在解决技术从业者在信息检索过程中面临的海量数据筛选困难与高质量内容分散问题。

本项目并非搜索引擎，而是一个经过人工筛选与分类的结构化技术资源索引。每个链接均对应一条具有明确技术主题的社区讨论或文档，用户可以快速定位到自己关心的技术栈或问题域，大幅减少无效阅读时间。项目本身采用静态站点生成方式，所有资源数据以结构化格式存储，便于二次开发与集成。

## 功能概览

- **主题分类索引**：所有资源按技术领域、应用场景和难度层级进行标签化分类，支持快速筛选。
- **社区精华聚合**：收录来自 rsxzs.com 社区的深度讨论帖与技术解决方案，确保内容质量。
- **全文元数据提取**：每个资源条目包含标题、摘要、发布时间和参与度等元数据，帮助用户判断内容价值。
- **多维度检索支持**：基于关键词、标签组合和时间范围进行复杂条件过滤。
- **资源状态监控**：定期检测链接有效性，标记失效或迁移的页面，保证索引可靠性。
- **用户收藏与笔记**：允许注册用户保存关注资源并添加个人备注，便于后续查阅。
- **开放数据导出**：支持将资源列表导出为 JSON、CSV 和 Markdown 格式，方便嵌入其他工具链。
- **自动化更新流水线**：每周触发增量抓取与分类更新，保持资源库的新鲜度。

## 应用场景

- **技术选型调研**：当团队需要评估新的数据库中间件或消息队列系统时，可通过本项目快速找到相关社区讨论与实际使用案例，了解优缺点和踩坑经验，为决策提供参考依据。
- **故障排查参考**：遇到线上服务异常或编译报错时，开发者可以检索本项目中的错误码或异常堆栈关键词，获取社区已有的解决方案和修复建议，缩短问题定位时间。
- **知识体系构建**：初学者或转岗工程师可通过本项目按主题浏览系列资源，系统性地学习 Docker、Kubernetes、Python 异步编程或微服务治理等方向，形成完整知识图谱。
- **技术文章写作素材收集**：技术博主或文档撰写者需要引用真实案例时，可从此处获取可追溯的社区原始讨论链接，确保引用的准确性和权威性。
- **内部团队知识库建设**：企业技术团队可将本项目作为外部资源补充，定期同步优质内容至内部 Wiki，丰富团队知识沉淀。

## 快速开始

以下步骤帮助您在本地环境中快速部署 RSXZS Resource Navigator 的基础索引服务。

```bash
# 克隆项目仓库
git clone https://github.com/rsxzs/resource-navigator.git
cd resource-navigator

# 安装依赖（基于 Node.js 20 LTS 与 pnpm）
pnpm install

# 构建静态资源索引并启动本地预览服务
pnpm run build
pnpm run preview
```

## 贡献指南

我们欢迎并鼓励社区贡献者参与本项目的数据完善、分类优化与功能开发。请遵循以下流程：

1. 查阅 issues 列表，认领未被分配的任务，或提交新 issue 描述您发现的问题或建议改进点。对于新功能建议，请附带使用场景说明。
2. Fork 本仓库，在您的分支上完成修改。若涉及新增资源链接，请按照 `config/sources.toml` 中规定的格式添加，并确保链接可访问。
3. 提交 pull request 前，请运行 `pnpm run lint` 和 `pnpm run test` 确保代码风格一致且所有测试通过。若新增功能，请同步编写或更新对应单元测试。
4. 在 PR 描述中清晰列明改动内容、影响范围以及是否涉及破坏性变更。对于数据新增，需附上手动验证的链接有效性截图。
5. 等待维护者审核。如需修改，请及时响应反馈。合并后您的提交将出现在下一轮索引更新中。

## 常见问题

**Q: 本项目与搜索引擎或 AI 问答工具的区别是什么？**

A: 本项目不提供全文搜索或自动问答功能。我们专注于对已筛选的社区技术资源进行结构化整理和分类索引，帮助用户从数百条高质量的讨论帖中快速定位到特定主题。所有资源均来自 rsxzs.com 社区，且经过人工复核，排除了低质量或无关内容。

**Q: 链接失效了怎么办？**

A: 每个资源链接每日会由自动化流水线进行可达性检测。若发现失效链接，系统会标记为“待验证”状态并通知维护者。如果您发现某个链接无法访问，可以通过提交 issue 或发送邮件通知我们，我们会尽快核实并更新或移除该条目。用户端可以通过前端界面的“状态”筛选器仅查看已验证有效的资源。

**Q: 如何申请添加新的资源分类或调整现有分类？**

A: 分类体系定义在 `config/categories.yaml` 文件中。如果您有合理的分类调整建议，请提交 issue 详细说明新增分类的名称、描述及其覆盖的资源范围。核心维护者会评估建议的合理性，若采纳将纳入下次分类更新中。我们不接受个人或商业推广性质的分类定制请求。
