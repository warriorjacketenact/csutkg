# TVPlay 资源导航系统

TVPlay 资源导航系统是一个面向影视资源聚合与分发场景的轻量级外链管理平台。本项目定位于为内容创作者、资源站运营者以及影视爱好者提供一套结构清晰、可快速部署的影视条目链接整理方案。通过对特定编号体系的影视播放页面进行集中收录与分类展示，帮助用户高效管理大量外部播放链接，避免链接散落、分类混乱、检索困难等问题。

本项目并非一个完整的视频播放或采集系统，而是一个基于静态链接列表的资源导航工具。其核心价值在于将分散的第三方影视播放地址以规范化的条目形式聚合呈现，适用于个人资源站、导航站或内容聚合类站点作为二级页面模块使用。目标用户包括个人站长、资源整理爱好者、影视内容编辑以及需要维护大量外链的中小型内容团队。

## 功能概览

**批量链接收录**：支持以条目形式批量录入外部影视播放页链接，每条链接独立编号并归入对应分类体系。

**编号化索引管理**：每个影视条目均分配唯一内部编号，便于在系统内进行引用、检索和版本追踪，编号规则支持自定义扩展。

**分类筛选与排序**：提供按发布时间、编号顺序、热度标记等多种维度对链接列表进行排序与筛选，适应不同场景下的浏览需求。

**快速跳转与复制**：每条链接均提供一键跳转及复制地址功能，减少用户手动操作步骤，提升外链访问效率。

**数据导入导出**：支持通过结构化文本或表格格式批量导入链接数据，并支持导出为通用格式用于备份或迁移。

**响应式列表展示**：前端展示层针对桌面与移动设备进行适配，确保在不同屏幕尺寸下链接列表均可清晰阅读与操作。

## 应用场景

个人影视资源整理：影视爱好者可将分散在多个网站、论坛的播放链接通过本系统统一收录，为每部影片建立独立的条目卡片，随时快速访问。

小型内容导航站：运营个人导航站或垂直领域内容聚合页的站长，可使用本系统作为影视频道的子模块，将外部播放源整理为有序列表供访客选用。

团队协作链接库：内容编辑团队在策划影视专题或撰写推荐文章时，可通过本系统共享一套统一的播放链接库，避免成员各自保存链接导致的版本混乱。

教学与演示素材管理：媒体相关专业的教师或学生可使用本系统整理课程中需要引用的影视片段链接，按主题分类后方便课堂展示与课后复习。

## 快速开始

以下步骤适用于在 Linux 或 macOS 开发环境下从源码启动本系统。

```bash
# 克隆代码仓库至本地
git clone https://github.com/example/tvplay-nav.git
cd tvplay-nav

# 安装项目依赖（使用 npm 或 yarn）
npm install

# 启动开发服务器，默认监听端口 3000
npm run dev
```

执行上述命令后，在浏览器中访问 http://localhost:3000 即可查看导航系统首页。生产环境部署请参考文档导航章节中的部署指南。

## 贡献指南

1. 复刻本仓库并创建功能分支。请在 GitHub 上点击 Fork 按钮将项目复制到个人账户，然后使用 git checkout -b feature/your-feature-name 创建专属分支进行开发。

2. 按照项目根目录下的 .env.example 文件创建本地 .env 环境变量，确保所有依赖安装完整且开发服务器可以正常启动。新增或修改链接数据时，请统一编辑 src/data/entries.json 文件并遵循已有条目的编号与字段结构。

3. 提交代码前运行测试套件与代码检查工具。执行 npm run test 验证所有单元测试通过，执行 npm run lint 确保代码风格符合项目 ESLint 配置要求。对于新增功能，请同步补充对应的测试用例。

4. 发起 Pull Request 并描述变更内容。推送分支到远程仓库后，在 GitHub 页面创建 Pull Request，请清晰说明本次修改的目的、影响范围以及是否涉及数据结构的变更。项目维护者将在三个工作日内进行审阅。

5. 对于重大功能更新或架构调整，建议先在 Issues 中创建讨论议题，与维护者及其他贡献者达成一致后再投入开发，以避免重复劳动或方向偏离。

## 常见问题

Q: 如何批量添加新的影视链接而不需要逐个手动编辑 JSON 文件？

A: 项目内置了数据导入工具，位于 src/data/importer.js。你可以准备一个 CSV 文件，列头包含 id、title、url、category 等字段，然后在项目根目录执行 npm run import -- --file=./path/to/your-data.csv 即可将数据合并到 entries.json 中。导入工具会自动校验链接格式并跳过重复条目。

Q: 部署后列表加载缓慢或页面空白，可能是什么原因？

A: 首先检查浏览器控制台是否有网络请求报错，确认 entries.json 文件是否被正确加载。如果数据文件较大，建议开启构建时的代码压缩和分块加载功能。同时检查 .env 文件中的 VITE_API_BASE_URL 变量是否指向了正确的数据文件地址。在开发环境下，可以临时使用 localStorage 缓存数据以减少重复请求。

Q: 编号规则是否可以修改，例如从纯数字改为包含字母的混合编码？

A: 可以。编号规则由 src/data/schema.js 中的 idPattern 正则表达式控制，你可以在该文件中将其修改为所需的格式，例如 /^[A-Z]{2}-[0-9]{6}$/。修改后需要同步更新 entries.json 中所有现有条目的编号以匹配新规则，否则数据加载时会因校验失败而报错。建议在修改前备份原始数据。
