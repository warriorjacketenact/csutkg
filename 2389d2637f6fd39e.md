# LinkArk Navigator

LinkArk Navigator 是一个面向开发者、技术研究者和内容策展人的高密度外链资源导航系统。项目定位于解决个人知识管理中的"链接失焦"问题，通过结构化的分类索引与静态站点生成机制，将散落的优质技术文档、工具站点与社区资源整合为可检索、可分享、可版本控制的导航知识库。本项目不提供爬虫或自动化采集功能，所有收录资源均基于人工筛选与社区提交，确保每条外链的可用性与内容质量。

本项目适用于需要维护个人技术书签库的工程师、搭建团队知识导航页的架构师，以及希望将收藏夹转化为公开知识资产的博客作者。LinkArk Navigator 以纯静态 Markdown 为核心，无需数据库支撑，可部署于任意 Web 服务器或托管平台，同时支持通过 GitHub Actions 实现定时链接有效性检查。

## 功能概览

- **分类索引引擎**：按技术领域、资源类型、适用人群对收录链接进行多维度标签化分类，支持快速筛选与反向查找。

- **链接健康监控**：内置基于 HTTP 状态码的链接存活检测模块，每日自动扫描全部外链并生成失效报告，帮助维护导航库的可用性。

- **静态站点生成**：基于项目根目录的 Markdown 数据文件，一键生成响应式 HTML 导航页面，无需额外构建工具链。

- **社区提交接口**：提供标准化的链接提交模板（YAML 格式），允许外部贡献者通过 Pull Request 提交新资源，项目维护者可审阅后合并。

- **全文模糊检索**：集成客户端全文搜索能力，支持按标题、描述、分类标签对已收录链接进行实时模糊匹配查询。

- **访问统计看板**：通过解析 Web 服务器访问日志，生成链接点击热力图与分类访问趋势，辅助内容优化决策。

- **导入导出适配**：支持将导航数据导出为 HTML 书签文件、JSON 结构化数据及 CSV 表格，便于迁移至其他书签管理工具。

- **标签关联图谱**：基于链接共享标签数量构建标签共现网络，以可视化方式展示资源之间的隐性关联关系。

## 应用场景

- **个人技术书签库的规范化管理**：开发者可将多年积累的零散浏览器书签导入 LinkArk Navigator，按项目、语言、框架等维度重新分类，并利用健康监控功能定期清理失效链接，保持收藏夹的长期可用性。

- **团队内部知识导航页搭建**：技术团队可使用本项目构建统一的技术文档入口页面，集中存放内部 Wiki、设计规范、CI/CD 流水线地址、监控面板等关键资源，新成员入职时可快速获得完整工具链索引。

- **技术社区的资源聚合站**：开源社区或技术论坛的管理员可利用 LinkArk Navigator 创建垂直领域的资源导航站点，例如"Go 语言工具大全""Kubernetes 生态地图"等，社区成员可共同维护和更新。

- **技术博客的外链附录页**：博客作者可为每篇技术文章生成配套的外链参考列表，将文中引用的文档、工具、论文链接集中存放于导航站中，既方便读者延伸阅读，也便于作者后续统一更新失效链接。

- **离线环境下的资源清单分发**：在无法访问互联网的内网开发环境中，团队可将 LinkArk Navigator 导出的 JSON 或 CSV 数据作为资源清单，配合内部镜像站地址使用，确保离线环境下的依赖查找效率。

## 快速开始

以下命令可在本地环境中完成 LinkArk Navigator 的克隆、依赖安装与开发服务器启动。

```bash
# 克隆项目仓库
git clone https://github.com/linkark/navigator.git
cd navigator

# 安装依赖（使用 npm）
npm install

# 运行开发服务器，默认监听 localhost:8080
npm run dev
```

执行完毕后，访问控制台输出的本地地址即可查看导航站预览。如需生成静态站点文件，请使用 `npm run build` 命令，输出目录默认为 `./dist`。

## 项目结构

```
navigator/
├── data/                                 # 核心数据目录，存放所有链接及分类定义
│   ├── links/                            # 链接条目目录，每个分类一个 YAML 文件
│   │   ├── frontend.yaml                 # 前端开发类资源（框架、工具库、UI 组件）
│   │   ├── backend.yaml                  # 后端开发类资源（语言、框架、数据库工具）
│   │   ├── devops.yaml                   # 运维与 DevOps 类资源（CI/CD、监控、容器）
│   │   ├── ai-ml.yaml                    # 人工智能与机器学习类资源（框架、数据集、论文）
│   │   └── community.yaml                # 社区与学习类资源（论坛、博客、播客、视频）
│   ├── tags.yaml                         # 全局标签系统定义，包含标签层级与同义词
│   └── meta.yaml                         # 项目元数据，含站点标题、描述、作者信息
├── scripts/                              # 工具脚本目录，存放构建与维护脚本
│   ├── build.js                          # 静态站点生成主脚本，读取 data/ 输出 dist/
│   ├── health-check.js                   # 链接健康监控脚本，并发检测 HTTP 状态
│   ├── import-bookmark.js                # 浏览器书签导入转换脚本（HTML -> YAML）
│   └── export-csv.js                     # 数据导出脚本，生成 CSV / JSON / HTML 书签
├── src/                                  # 前端源码目录，用于生成导航页面的模板与资源
│   ├── templates/                        # 模板引擎文件，使用 EJS 语法
│   │   ├── index.ejs                     # 导航首页模板，展示分类列表与最新链接
│   │   ├── category.ejs                  # 分类详情页模板，展示该分类下全部链接
│   │   └── search.ejs                    # 搜索结果页模板，展示模糊匹配结果
│   ├── assets/                           # 静态资源文件（样式、脚本、字体）
```

## 贡献指南

1. 复刻项目仓库至个人 GitHub 账号，并在本地克隆复刻后的副本。请确保本地 Git 全局配置中包含正确的用户名与邮箱，以便提交记录可追溯。

2. 在 `data/links/` 目录下找到对应分类的 YAML 文件，按照既定模板格式添加新链接条目。每个链接条目必须包含 `title`、`url`、`description`、`tags` 四个字段，其中 `url` 字段需使用原始链接地址，不得添加任何追踪参数。

3. 若现有分类无法覆盖新链接的主题领域，可在 `data/tags.yaml` 中新增标签定义，并同步更新 `data/meta.yaml` 中的分类索引。新增标签需提供明确的中英文名称及简要说明。

4. 提交变更前，请在本机运行 `npm run build` 命令确保构建通过，并使用 `npm run test` 运行全部测试用例。所有测试必须通过方可提交。提交信息应遵循约定式提交规范，例如 `feat: add new link for Vue 3 official docs`。

5. 发起 Pull Request 至主仓库的 `main` 分支，并在 PR 描述中简要说明新增资源的用途与筛选依据。项目维护者将在 3 个工作日内完成审阅，可能要求补充链接的可用性验证或调整分类标签。

## 常见问题

**Q: 链接健康监控检测到失效链接后，项目会如何处理？**

A: 健康监控脚本每日自动运行，检测到 HTTP 状态码非 200 或超时的链接后，会在 GitHub Actions 的日志中输出警告，并生成一份 `broken-links.json` 文件存放于项目根目录。项目维护者会每两周审阅该文件，对持续失效的链接予以移除，并尝试寻找替代资源。用户亦可在本地手动运行 `npm run health-check` 获取当前全部链接的状态报告。

**Q: 我可以使用 LinkArk Navigator 来管理私有的、不对外公开的链接集合吗？**

A: 可以。项目本身不包含任何认证或权限控制机制，但您可以在本地或私有 Git 仓库中独立使用全部数据管理和静态生成功能。只需将 `data/links/` 目录下的内容替换为您的私有链接数据，运行构建脚本即可生成不包含任何公开资源的导航页面。如需部署至内网环境，建议配合 HTTP 基本认证或 VPN 进行访问控制。

**Q: 导入浏览器书签时，原有的文件夹层级结构会丢失吗？**

A: 不会完全丢失。项目提供的 `import-bookmark.js` 脚本会递归读取书签文件夹结构，并将顶层文件夹映射为分类（Category），二级及以下文件夹则转换为标签（Tag）。脚本同时会生成一个映射日志文件，方便您后续手动调整分类归属。需要注意的是，脚本目前支持 Chrome 和 Firefox 导出的 HTML 书签文件格式。
