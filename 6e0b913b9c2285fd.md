# LinkVault Resource Aggregator

LinkVault is a curated, version-controlled repository of technical resource links, serving as a structured knowledge base for developers, researchers, and system administrators. It aggregates and categorizes external references, documentation, tools, and learning materials from across the web into a single, queryable index.

This project targets technical professionals who need to maintain a personal or team-wide bookmark collection with audit trails, tagging, and dependency tracking. Unlike traditional bookmark managers, LinkVault treats each link as a first-class entity with metadata, status checks, and relational mappings to projects, languages, or frameworks. The repository includes over 250 verified resource entries from the 912th batch of an ongoing curation pipeline, covering energy sector data, infrastructure tooling, and algorithmic references.

## 功能概览

**批量资源导入** - Supports ingestion of URL lists in plain text, CSV, or JSON format with automatic deduplication and schema validation against predefined categories.

**存活状态检测** - Performs asynchronous HEAD requests to verify link availability, with configurable timeouts, retry policies, and failure logging for downstream alerting.

**标签与全文检索** - Implements inverted-index search over titles, descriptions, and custom tags, with fuzzy matching and stopword filtering for rapid resource discovery.

**版本化快照** - Maintains Git-compatible change logs for every addition, update, or removal, enabling rollback to any previous state of the resource tree.

**导出与集成** - Generates static HTML, Markdown TOC, or JSON feeds for integration with static site generators, CI/CD pipelines, or monitoring dashboards.

**元数据增强** - Enriches each entry with optional fields: language, framework, license type, author, publication date, and dependency rank based on internal reference counts.

**访问统计仪表板** - Provides a lightweight terminal UI (via curses) or web endpoint (Flask) showing top domains, category distribution, and stale link ratios.

**自动化分类建议** - Uses rule-based heuristics on URL patterns, path segments, and domain whois data to suggest primary and secondary categories for new entries.

## 应用场景

**技术文档聚合** - A backend team can maintain a central registry of API references, SDK manuals, and operational runbooks, replacing scattered browser bookmarks with a searchable, shareable repository that integrates with their internal wiki.

**学术研究参考** - Researchers tracking papers, datasets, and supplemental materials across multiple domains can use LinkVault to organize references by project phase, with export to BibTeX or Zotero-compatible formats for citation management.

**DevOps 资产清单** - Infrastructure engineers can catalog monitoring dashboards, log aggregators, container registries, and configuration management endpoints, associating each with specific environments (staging, production, DR) and on-call rotation schedules.

**渗透测试工具库** - Security teams can curate a private collection of exploit databases, CVE feeds, reverse-engineering frameworks, and payload generators, with checksums and offline mirrors for air-gapped assessment environments.

**技术栈迁移规划** - Architects evaluating framework alternatives can group links by ecosystem (e.g., Python data stack, Rust systems libraries, JavaScript frontend tooling) and compare version compatibility, community activity, and migration effort through annotated metadata.

## 快速开始

Clone the repository, install dependencies, and run the initial indexing routine. All commands assume a Unix-like environment with Python 3.10+ and Git.

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python linkvault.py --init --source data/912_batch.txt --output index.json
python linkvault.py --serve --port 8080
```

The first command initializes the SQLite backing store and imports the 250-entry batch. The second starts the web inspection interface. For headless environments, use `--report` to generate a static summary.

## 贡献指南

We welcome contributions that expand the resource index, improve metadata quality, or enhance the core tooling. All submissions must pass the pre-commit checks and include relevant test coverage for new logic.

第一步：Fork 本仓库并创建功能分支。分支命名遵循 `feature/`、`fix/` 或 `batch/` 前缀，例如 `batch/913_import`。确保本地开发环境已安装所有开发依赖（`pip install -r requirements-dev.txt`）。

第二步：对于新增资源链接，请将 URL 列表放入 `data/batches/` 目录下的对应批次文件，并同时更新 `data/metadata.yaml` 中的分类映射表。每个链接必须附带至少一个标签（tag）和简短描述（description），描述长度不超过 240 字符。

第三步：运行测试套件并确保通过率 100%。使用 `pytest tests/ --cov=src/ --cov-report=term-missing` 检查覆盖率，新增代码行覆盖率不低于 85%。同时执行 `make lint` 调用 black、isort 和 flake8 进行风格统一。

第四步：提交前执行 `python linkvault.py --check data/batches/912_batch.txt` 进行本地健康预检，确保所有 URL 可访问（允许 3% 的失败率，需在提交说明中注明失败原因）。提交信息（commit message）必须遵循 Conventional Commits 格式，例如 `feat(ingest): add CSV parser for batch 913`。

第五步：推送分支并创建 Pull Request。PR 描述需包含变更摘要、测试结果截图或日志片段，以及是否影响现有数据库 schema。至少一位维护者审核通过后，将合并至 main 分支并自动触发 GitHub Actions 重新生成全量索引。

## 常见问题

**问：LinkVault 如何处理 URL 重定向和临时失效链接？**

答：健康检查默认跟随三次重定向（301, 302, 307），并记录最终目标 URL 至数据库的 `resolved_url` 字段。对于返回 429 或 503 的临时故障，系统会以指数退避策略重试最多三次，并将失败原因分类为“临时不可用”、“永久失效”或“超时”。用户可通过 `--retry-count` 和 `--timeout` 参数调整行为。所有失败记录均写入 `logs/check_errors.jsonl` 供离线分析。

**问：能否将 LinkVault 与现有的团队 Wiki 或 Notion 数据库同步？**

答：可以通过导出功能实现集成。运行 `python linkvault.py --export --format markdown --output wiki.md` 生成包含所有链接及标签的 Markdown 表格，该表格可直接粘贴至 Confluence 或 GitHub Wiki。对于 Notion，我们提供实验性的 `--format notion` 选项，输出 CSV 文件，其列结构匹配 Notion 的“导入 CSV”功能。若需要实时双向同步，建议使用 `--webhook` 模式配合 Zapier 或 n8n 工作流。

**问：数据库体积增长后，全文搜索性能如何？**

答：我们在 SQLite FTS5 基础上启用了 `prefix=3,4` 和 `tokenchars=-._` 配置以优化部分匹配和域名搜索。对于 10 万条记录，典型查询（如 `nginx AND (ssl OR tls) NOT debug`）响应时间在 120ms 以内。若数据量超过 50 万条，建议使用 `--prune-history 90` 清理 90 天前的变更日志，并运行 `VACUUM` 命令回收空间。对于更大规模部署，我们推荐使用 PostgreSQL 替代 SQLite，迁移脚本位于 `scripts/migrate_to_pg.py`。
