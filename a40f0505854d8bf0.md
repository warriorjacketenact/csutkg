# IndexBase

IndexBase 是一个面向技术研究、数据归档与信息检索场景的轻量级外链资源索引系统。该项目并非通用爬虫或全文搜索引擎，而是一套结构化外链元数据管理方案，专为需要长期维护大量外部参考链接的团队或个人研究者设计。IndexBase 以可编辑的文本配置和静态生成机制为核心，帮助用户在无需数据库的前提下，实现数千条外链的分类存储、标签过滤与状态跟踪。

IndexBase 的目标用户包括技术文档维护者、网络安全研究员、开源情报分析师以及各类需要系统化管理外链资源的运维人员。通过 IndexBase，用户可以将零散的参考链接转化为具备可读性、可追溯性和可审计性的结构化资源库，从而降低链接失效、分类混乱和检索困难等常见问题带来的维护成本。

## 功能概览

**外链记录与元数据标注** 支持为每条链接记录来源站点、批次编号、归档时间以及自定义标签，便于后续过滤与统计。

**静态索引生成** 基于用户提供的链接列表，自动生成 HTML 与 Markdown 两种格式的索引页面，无需动态后端服务即可部署至任意 Web 服务器或 CDN。

**分类视图与多级筛选** 根据链接路径中的分类代码自动识别资源类型，并生成按类别、批次或关键词筛选的导航视图。

**链接状态探测** 内置轻量级 HTTP 状态检查器，可定期检测外链可达性，并输出状态报告，帮助识别失效或重定向链接。

**导入导出兼容性** 支持 CSV 与 JSON 格式的链接批量导入导出，便于与其他数据处理工具或脚本进行集成。

**增量更新机制** 支持基于批次号的增量追加操作，新链接不会覆盖已有记录，确保历史数据的完整性与可追溯性。

## 应用场景

**技术文档参考库维护** 技术团队在编写系统设计文档或运维手册时，需要引用大量外部资料。IndexBase 可为这些引用链接提供统一的管理入口，并自动生成可供团队内部共享的索引页面。

**开源情报链接归档** 安全研究人员在追踪威胁情报或漏洞公告时，需要长期保存相关参考链接。IndexBase 的批次归档与状态探测功能可帮助研究人员快速识别已失效的参考源，并及时更新。

**学术文献外链整理** 学术作者在撰写论文或综述时，需整理大量在线参考文献。IndexBase 允许按主题标签分类存储这些链接，并导出为符合学术规范的引用列表格式。

**运维监控外链看板** 运维团队可将常用监控面板、日志查询工具和内部文档系统的入口链接统一纳入 IndexBase 管理，生成团队内部导航页，减少日常查找时间。

## 快速开始

以下操作将在本地环境中完成 IndexBase 的克隆、依赖安装与首次索引生成。

```bash
git clone https://github.com/indexbase/indexbase.git
cd indexbase
pip install -r requirements.txt
python indexbase.py --input ./samples/links.txt --output ./dist --batch 1203
```

执行完毕后，可在 `./dist` 目录下查看生成的 `index.html` 与 `index.md` 文件，其中包含了本次批次 1203 的全部链接索引内容。

## 项目结构

```
indexbase/
├── indexbase.py                # 主入口脚本，解析参数并调度索引生成流程
├── requirements.txt            # Python 依赖声明文件
├── config/
│   ├── default.yaml            # 默认配置，包含分类映射与输出模板路径
│   └── custom.yaml             # 用户自定义配置示例，可覆盖分类规则
├── core/
│   ├── parser.py               # 原始链接列表解析器，支持 CSV 与纯文本格式
│   ├── classifier.py           # 基于路径代码与正则表达式的分类引擎
│   ├── probe.py                # HTTP 状态探测模块，支持并发请求与超时控制
│   └── exporter.py             # 索引输出模块，生成 Markdown 与 HTML 格式
├── templates/
│   ├── index.html.j2           # HTML 索引页的 Jinja2 模板
│   └── index.md.j2             # Markdown 索引页的 Jinja2 模板
├── samples/
│   ├── links.txt               # 示例链接输入文件，展示标准格式
│   └── links_with_tags.csv     # 示例带标签的 CSV 输入文件
├── dist/                       # 默认输出目录，存放生成的索引文件
├── tests/
│   ├── test_parser.py          # 解析器单元测试
│   ├── test_classifier.py      # 分类引擎单元测试
│   └── test_probe.py           # 状态探测模块单元测试
└── docs/
    ├── user-guide.md           # 用户手册
    ├── configuration.md        # 配置说明文档
    ├── development.md          # 开发指南
    └── operations.md           # 运维参考文档
```

## 贡献指南

1. 在 GitHub 上 fork 本仓库，并克隆到本地开发环境。确保本地 Python 版本符合安装要求，且已安装所有开发依赖。
2. 在 `tests/` 目录下编写或更新与修改内容对应的单元测试，确保测试覆盖率达到 90% 以上。所有测试用例需在提交前通过。

## 常见问题

**Q: 索引生成后，部分链接显示状态为 UNREACHABLE，但浏览器可以正常访问，如何解决？**

A: 状态探测模块使用 requests 库发送 HTTP 请求，默认超时时间为 10 秒，且不跟随 JavaScript 重定向。若目标站点依赖客户端渲染或存在较严格的访问频率限制，可能导致探测失败。建议在 `config/default.yaml` 中调整 `probe_timeout` 和 `probe_user_agent` 参数，或增加 `probe_retry` 重试次数。对于需要登录或特殊 Cookie 的链接，可将其加入 `probe_ignore_list` 跳过状态检查。

**Q: 我有一批新链接需要追加到已有索引中，但不想覆盖之前的分类和标签，应该如何处理？**

A: 使用 `--append` 参数运行 indexbase.py，并指定与首次生成相同的输出目录。追加模式下，程序会读取已存在的索引元数据文件，将新链接合并至链接列表尾部，并自动分配新的批次编号。原有链接的分类、标签和状态记录将保持不变。如果新链接包含相同的 URL，程序会依据 `--dedup` 参数决定是否跳过或更新。

**Q: 生成的 HTML 索引页面样式能否自定义？**

A: 可以。HTML 输出基于 `templates/index.html.j2` 模板渲染，用户可直接修改该模板文件中的 HTML 结构和 CSS 样式。若需要保留默认模板以便后续升级，可将自定义模板放置于 `config/custom.yaml` 中 `template_override` 指定的路径，程序会优先使用自定义模板。修改模板后，重新运行 indexbase.py 即可生成带有新样式的索引页面。
