# VODLink 技术资源索引系统

VODLink 是一个面向开发者和技术研究人员的结构化外链资源汇总平台，专注于收录和分类互联网上的技术文档、视频教程、工具站点与开源项目参考链接。本系统定位于解决技术信息分散、优质资源难以追溯的问题，通过统一标识符体系与分类导航，帮助用户快速定位所需的技术参考材料。

项目适用于技术团队内部知识库建设、个人学习路径规划以及开源项目文档链整合等场景。当前批次收录资源共计 250 条，涵盖视频资源、技术博客、API 参考、代码托管镜像与社区讨论等多个类别，所有链接均经过基础可用性校验并赋予唯一内部编码。

## 功能概览

统一资源标识体系：每个收录的链接均分配一个形如 /vod/xxx 的内部短码，便于系统内部引用与去重管理。

多维度分类导航：资源按内容类型、技术栈、适用阶段进行粗粒度划分，支持按类别快速筛选。

批量导入与导出：支持通过 CSV 或 JSON 格式批量导入链接列表，并可导出为标准 Markdown 或 HTML 索引文件。

链接可用性检查：内置定时任务对已收录资源进行 HTTP 状态码检查，标记失效链接并生成报告。

自定义标签系统：用户可为每个资源添加自定义标签，实现个性化分类与检索。

全文检索支持：基于资源标题、描述、标签与内部编码进行关键词检索，返回相关度排序结果。

访问统计与热度排序：记录每个外部链接的点击次数，支持按热度展示常用资源。

## 应用场景

技术团队内部知识库构建：团队可以将日常开发中参考的文档、视频和工具链接统一收录至 VODLink，通过分类和标签功能建立团队共享的知识索引，减少重复查找时间。

个人开发者学习路径管理：学习者可以将不同阶段接触到的教程、案例和参考资料按主题或学习顺序组织起来，配合检索功能快速回溯之前查阅过的内容。

开源项目文档外链整合：开源项目维护者可以使用本系统管理项目 README 或 Wiki 中引用的所有外部链接，确保引用来源清晰可追溯，同时利用可用性检查功能及时发现失效引用。

技术社区资源推荐汇总：技术社区运营方可定期批量导入优质外部资源链接，通过热度排序功能筛选出社区成员关注度最高的内容，为内容推荐提供数据依据。

## 快速开始

以下命令序列演示了从代码仓库克隆、安装依赖到启动开发服务器的完整流程。

```bash
git clone https://github.com/your-org/vodlink-indexer.git
cd vodlink-indexer
pip install -r requirements.txt
python manage.py migrate
python manage.py loaddata initial_resources.json
python manage.py runserver
```

生产环境部署时，请参考 `deploy/production` 目录下的 Ansible 脚本与 Docker Compose 配置文件。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心后端运行环境，使用 Django 框架 |
| PostgreSQL | 13.0 及以上 | 主数据库，存储资源条目、标签与访问日志 |
| Redis | 6.0 及以上 | 缓存与任务队列后端，用于异步链接检查 |
| Celery | 5.2 及以上 | 分布式任务队列，执行定时可用性检查 |
| Nginx | 1.20 及以上 | 生产环境反向代理与静态文件服务 |
| Node.js | 16.0 及以上 | 前端构建工具依赖，用于处理静态资源编译 |

## 贡献指南

贡献者请遵循以下流程以确保代码质量和项目一致性。

提交 Issue 报告缺陷或提议新功能：请在 GitHub Issues 页面选择对应模板，清晰描述问题重现步骤或功能需求，并附上相关环境信息（操作系统、Python 版本、浏览器版本等）。

Fork 仓库并创建特性分支：从主仓库 fork 代码后，在本地创建以 feature/ 或 fix/ 为前缀的分支，例如 feature/add-export-json。分支命名应简洁描述变更内容。

编写代码并确保测试通过：所有新增功能或缺陷修复必须包含对应的单元测试，测试覆盖率不得低于现有基线。运行 tox 或 pytest 确保全部测试用例通过。代码风格需符合 PEP 8 规范，并使用 black 与 flake8 进行格式化与检查。

提交 Pull Request 并描述变更：PR 标题应简明扼要，正文需详细说明变更目的、实现方式及影响范围。关联相关 Issue 编号。至少需要一名项目维护者审阅通过后方可合并。

更新文档与变更日志：若变更涉及用户可见功能、配置项或 API 接口，需同步更新 docs/ 下对应手册，并在 CHANGELOG.md 中记录变更条目，标明版本号与变更类型（新增、修复、废弃等）。

## 常见问题

问：系统支持的最大资源条目数量是多少？当链接数量增长到数万级别时性能如何？

答：系统本身不设硬性上限，单表存储量设计为千万级别。在 PostgreSQL 数据库下，资源主表及关联标签表均建有合适索引，对于十万条以内的资源，全文检索和分类筛选的响应时间可控制在 200 毫秒以内。若条目超过此规模，建议启用 Redis 缓存并配置只读从库以分担查询压力。

问：链接可用性检查是否会对外部站点造成压力？检查频率如何设置？

答：检查任务基于 Celery 定时调度，默认每 7 天对所有资源执行一次 HEAD 请求，仅验证 HTTP 状态码，不下载完整内容。对于单个域名，系统内置了请求间隔控制（默认至少间隔 500 毫秒），避免对目标服务器造成突发流量。用户可在管理后台调整检查频率或临时暂停对特定域名的检查。

问：如何从其他系统迁移已有的外链数据至 VODLink？

答：系统提供了自定义管理命令 python manage.py import_from_csv --file=links.csv，支持导入包含 title、url、category、tags 列的 CSV 文件。同时，项目提供了通用 JSON 导入接口 /api/resources/import/，接受符合指定 Schema 的 JSON 数组。迁移前建议先在小规模数据上测试导入流程，确认字段映射无误后再执行全量迁移。
