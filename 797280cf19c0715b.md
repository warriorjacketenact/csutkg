# ZaidArchive

ZaidArchive 是一个面向技术研究与知识管理场景的轻量级外链资源汇总系统。项目定位于帮助开发者、技术写作者、研究人员以及开源社区维护者，高效收集、分类、存档和检索分散在互联网各处的优质技术文章、工具文档、设计素材与参考链接。

本项目不提供内容存储服务，也不对第三方链接内容进行镜像或备份，而是通过结构化的索引机制，将用户提供的链接资源按照既定规范进行整理与呈现，便于团队协作、公开分享以及长期维护。ZaidArchive 适用于个人知识库构建、项目文档外链附录、技术周报素材管理、开源项目 README 资源导航等场景。

## 功能概览

**批量链接导入**：支持通过文本列表或 CSV 格式批量导入外部链接，自动解析 URL 并提取基础元信息。

**分类标签系统**：每个资源条目可关联多个自定义标签，支持按标签快速筛选与归类。

**项目批次管理**：内置批次编号机制，当前为第 457/1241 批，便于追踪资源采集进度与来源。

**只读存档模式**：资源列表以只读方式呈现，防止误修改，保证存档数据的完整性与可追溯性。

**Markdown 原生渲染**：所有资源列表直接以 Markdown 格式输出，兼容 GitHub、GitLab、Gitee 等主流代码托管平台的渲染引擎。

**轻量化无后端依赖**：项目基于静态 Markdown 文件运行，无需数据库或服务端环境，克隆即用。

**结构化元数据标注**：每个链接附带路径层级标识，便于脚本解析与自动化处理。

## 应用场景

技术团队内部知识库外链管理：团队成员在日常开发中积累的大量技术博客、官方文档、API 参考链接，可通过 ZaidArchive 统一归档，并随项目代码一同托管，避免链接散落在聊天记录或个人书签中无法共享。

开源项目 README 资源附录生成：开源项目维护者在撰写文档时，常常需要附带大量外部参考链接。ZaidArchive 可帮助维护者按照固定格式批量整理这些链接，确保 README 中的资源列表规范、完整且易于后续更新。

技术写作与研究报告参考资料整理：技术作者在进行专题写作或行业调研时，需要收集大量来源链接作为引用依据。ZaidArchive 的批次管理功能可以按主题或时间分段记录引用来源，方便最终成稿时统一生成参考文献附录。

个人开发环境配置脚本资源索引：开发者在初始化新环境时，往往需要下载多种工具、依赖包或配置模板。ZaidArchive 可作为这些外部资源的索引清单，配合自动化脚本实现一键拉取。

## 快速开始

以下操作基于 Linux / macOS / Windows WSL 环境，确保系统已安装 Git 和 Node.js（用于后续脚本扩展）。

```bash
# 克隆仓库
git clone https://github.com/zaidarchive/zaidarchive.git
cd zaidarchive

# 安装依赖（用于本地预览与格式校验）
npm install

# 运行资源列表生成脚本，输出当前批次的 README 片段
npm run build -- --batch=457
```

执行完毕后，`dist/` 目录下将生成当前批次对应的 Markdown 资源列表文件，可直接复制或追加到项目的 README 中。

## 贡献指南

提交新批次链接：在 `src/data/` 目录下创建新的批次文件，按照 `batch-template.json` 格式填写链接列表，确保每个 URL 严格遵循原始格式（不添加协议前缀、不修改大小写、不增减尾部斜杠），然后运行 `npm run validate` 进行格式校验。

修复失效链接：若发现资源列表中的链接已不可访问，请在 `src/data/` 中对应批次文件中标记该条目为 `[deprecated]`，并附上替代链接或删除说明，随后提交 Pull Request。

完善文档：若发现文档导航部分的说明不够清晰或缺少使用示例，可在 `docs/` 目录下补充相应内容，并同步更新 `README.md` 中的文档导航表格。

本地预览：运行 `npm run preview` 可在本地启动一个简单的 HTTP 服务，预览当前批次资源列表的渲染效果，确认 Markdown 格式无误后再提交。

提交 Pull Request 前，请确保所有测试通过（`npm test`），且代码风格符合项目 ESLint 配置（`npm run lint`）。

## 常见问题

问：为什么资源列表中的 URL 不能添加 `https://` 或 `http://` 前缀？

答：项目格式规范要求所有 URL 必须按照用户提供的原始形式原样输出，目的是保证链接来源的可追溯性以及避免因自动补全协议导致的访问异常。用户提供的裸域名或带协议字符串均视为最终输出形式。

问：如果某个链接失效了怎么办？

答：ZaidArchive 不负责外部链接的可用性保证。若发现失效链接，请在贡献时标记为 `[deprecated]` 并注明失效日期。项目维护者会定期审查标记条目，并在下一个大版本中移除长期失效的链接。

问：如何添加自定义标签或分类？

答：当前版本支持在批次文件的 JSON 结构中为每个条目添加 `tags` 字段，类型为字符串数组。后续版本将提供基于标签的过滤渲染功能。如需立即使用，可自行扩展 `renderer.js` 中的渲染逻辑。
