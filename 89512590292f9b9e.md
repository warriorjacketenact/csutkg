# LinkVault Resource Aggregator

LinkVault is a curated resource aggregation and navigation system designed for developers, researchers, and technical professionals who need to manage and discover high-quality external references across distributed web sources. The project addresses the fundamental challenge of link rot, context loss, and disorganized bookmark collections by providing a structured, version-controlled, and queryable repository of categorized external URLs with metadata tagging and health monitoring capabilities.

Targeting system administrators, DevOps engineers, and technical architects, LinkVault transforms raw URL lists into a maintainable knowledge base with automated validation, semantic grouping, and usage analytics. The project serves as both a personal bookmarking enhancement tool and a team-shared resource registry that integrates with CI/CD pipelines for continuous link integrity verification.

## 功能概览

**Automated Link Validation** - Periodically checks all stored URLs for HTTP status changes, certificate expiration, and content-type mismatches, reporting broken links with detailed error logs.

**Semantic Categorization Engine** - Automatically classifies URLs into predefined taxonomies using path analysis, domain fingerprinting, and optional NLP-based content inspection for untagged resources.

**Metadata Enrichment Pipeline** - Fetches and stores Open Graph data, page titles, descriptions, and favicon hashes for each resource, enabling rich search results and quick visual identification.

**Versioned Change Tracking** - Maintains a complete audit history of all URL modifications, additions, and removals with timestamped commit messages and user attribution for team environments.

**Queryable RESTful API** - Exposes GraphQL and REST endpoints for programmatic access, supporting filtering by category, status, domain, and custom tags with pagination and sorting.

**Health Monitoring Dashboard** - Provides a real-time web interface showing aggregate link health statistics, response time distributions, and historical availability trends with exportable reports.

**Bulk Import and Export** - Supports JSON, CSV, and Markdown table formats for batch operations, enabling seamless migration from bookmark managers and spreadsheet-based inventories.

**Tagging and Annotation System** - Allows users to attach custom key-value metadata, priority flags, and personal notes to any resource without modifying the original URL structure.

## 应用场景

**Technical Documentation Management** - Documentation teams can aggregate external reference links from API specifications, SDK guides, and third-party libraries into a single validated registry, ensuring all cited resources remain accessible across product release cycles.

**Academic Research Reference Curation** - Researchers compiling bibliographies or systematic literature reviews can import DOI links, preprint servers, and dataset repositories with automatic metadata extraction, reducing manual entry errors and enabling collaborative annotation.

**DevOps Infrastructure Monitoring** - Site reliability engineers can store and track URLs of internal monitoring dashboards, status pages, and incident response runbooks, with automated alerts when any critical link becomes unreachable during on-call rotations.

**Knowledge Base Construction** - Technical writers and community managers can build categorized link collections for onboarding materials, troubleshooting guides, and tool-specific cheat sheets, with version control ensuring historical accuracy.

**Compliance and Audit Trail Maintenance** - Organizations subject to regulatory requirements can maintain immutable logs of external policy references, vendor documentation, and standards bodies' publications with timestamped validation records for auditors.

## 快速开始

Clone the repository, install dependencies, and run the initial setup script:

```bash
git clone https://github.com/your-org/linkvault.git
cd linkvault
npm install --production
npm run build
npm start
```

For development environment with hot-reload and debugging tools:

```bash
npm install --include=dev
npm run dev
```

To run the initial link validation sweep across all preloaded resources:

```bash
npm run validate:all
```

Generate the static site version for deployment to CDN or GitHub Pages:

```bash
npm run generate:static
```

## 项目结构

```
linkvault/
├── src/
│   ├── core/                           # 核心业务逻辑模块
│   │   ├── validator/                  # 链接验证引擎（HTTP 状态检查、证书校验）
│   │   ├── categorizer/                # 自动分类器（基于规则和机器学习）
│   │   ├── enricher/                   # 元数据增强器（OG 标签、标题提取）
│   │   └── tracker/                    # 变更追踪器（版本差异计算）
```

2.  **分支与提交规范** - 基于 main 分支创建特性分支，命名格式为 `feature/描述`、`fix/描述` 或 `docs/描述`。提交信息遵循 Conventional Commits 规范，即 `type(scope): subject` 格式（例如 `feat(validator): add retry policy for 5xx errors`）。每个提交需保持原子性，即一次提交只解决一个问题。

3.  **代码审查与测试要求** - 所有推送自动触发 GitHub Actions 工作流，运行单元测试、集成测试和代码 lint 检查。新增功能必须附带相应的测试用例，测试覆盖率不得低于 85%。提交 Pull Request 后需至少一名维护者 approve 方可合并。

4.  **文档同步更新** - 任何影响用户界面的变更必须同步更新 `docs/user-guide` 目录下的对应章节。API 变更需更新 OpenAPI 规范文件（`src/api/schemas/openapi.yaml`）。新增配置项需补充 `.env.example` 注释说明。

5.  **本地开发环境准备** - 克隆仓库后运行 `npm install` 安装依赖，使用 `docker-compose up -d` 启动 PostgreSQL 和 Redis 容器，执行 `npm run db:migrate` 初始化数据库结构，最后 `npm run dev` 启动开发服务器。详细的本地调试指南位于 `docs/developer-guide/setup.md`。

## 常见问题

**问：链接验证会消耗大量带宽吗？**

验证任务使用 HTTP HEAD 请求获取响应头，不下载完整页面内容，因此单次验证的带宽消耗极小（通常小于 2KB）。对于大批量资源（如本批次 250 个链接），全量扫描的总体数据量约为 500KB，且通过异步队列控制并发数为 10，避免对目标服务器造成压力。验证间隔默认每 24 小时执行一次，用户可在配置文件中调整并发度和超时时间。

**问：如何处理验证失败或暂时不可用的链接？**

系统对验证失败的链接采用指数退避重试策略，分别在第 1、5、15 分钟后重试三次。若全部失败，则标记为 `unavailable` 状态，并在仪表盘以橙色高亮显示。用户可手动触发重新验证或编辑资源信息。若链接永久失效（HTTP 404 或 410），系统建议移除或替换，并提供最近的存档快照链接（如 Internet Archive Wayback Machine 集成）。

**问：能否在离线环境中部署使用？**

完全支持离线部署。所有依赖项（npm 包、Docker 镜像）可预先镜像至私有仓库。验证功能在没有公网访问时仍可工作，但仅限于检查内部网络中的可访问 URL。外部资源验证会跳过并记录警告。如需完整功能，建议至少允许出站 HTTPS 访问用于证书校验和元数据抓取，但用户可通过配置将验证器限制为仅检查内部域名。

**问：数据存储安全如何保证？**

所有资源链接和用户元数据在数据库中静态加密（使用 AES-256-GCM）。API 通信强制使用 TLS 1.3。用户密码使用 bcrypt 哈希（成本因子 12）。审计日志记录所有敏感操作（删除、权限变更）。系统提供可选的 IP 白名单和 API 密钥轮换策略，详细安全配置参见 `docs/security/hardening.md`。
