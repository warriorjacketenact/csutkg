# Resource Atlas

Resource Atlas 是一个面向技术研究人员、开发者与信息分析人员的结构化外链资源汇总系统。项目定位于对分散于互联网各处的优质技术文档、系列文章与参考材料进行系统性收录、分类与索引，帮助用户以可复用的方式组织外部知识链接，降低信息碎片化带来的检索成本。

本项目不提供资源托管或内容镜像，而是作为资源导航层，通过统一的条目标识与分类体系，将外部 URL 映射为可检索、可追溯、可共享的知识条目。目标用户包括需要维护个人知识库的工程师、搭建技术文档站点的团队负责人，以及希望建立内部技术资源目录的企业开发者。

---

## 功能概览

**批量资源收录** 支持一次性导入大量外链，自动提取 URL 基础信息并生成条目编号，便于后续检索与引用。

**分类标签管理** 为每条资源赋予技术领域、内容类型、适用阶段等标签，支持多维度筛选与组合查询。

**系列关系追踪** 通过 series 路径识别同一系列文章的资源关联，自动构建内容序列视图，方便顺序阅读。

**资源状态标记** 支持为每条资源标记有效、失效、待验证等状态，帮助维护资源可用性。

**条目备注扩展** 每条资源可附加自定义备注字段，用于记录阅读心得、关键摘要或内部评审意见。

**数据导入导出** 支持 CSV 与 JSON 格式的批量导入导出，便于与其他知识管理工具协同使用。

**检索与过滤** 提供基于标题关键词、标签、编号范围的检索接口，支持精确匹配与模糊搜索。

**版本追踪** 记录资源条目的新增、修改与删除历史，支持回溯至任意历史版本。

---

## 应用场景

技术团队内部知识库构建。团队可将日常研发中参考的技术博客、官方文档、API 参考手册通过 Resource Atlas 统一收录，按项目或技术栈分类，新成员入职时可快速获取完整的学习资源列表。

个人技术阅读管理。开发者可将长期积累的待读文章、系列教程、演讲视频链接集中管理，利用标签区分优先级与阅读状态，避免收藏后遗忘。

开源项目文档外链维护。开源项目维护者可将项目依赖的规范标准、上下游工具文档、社区讨论帖等外链纳入 Resource Atlas，作为项目文档站点的补充导航模块。

技术培训课程资料汇总。培训组织者可围绕课程大纲，将每节课涉及的扩展阅读材料、练习环境入口、案例代码仓库链接逐条录入，形成完整的课程资源手册。

## 快速开始

以下操作以 Linux / macOS 环境为例，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆代码仓库
git clone https://github.com/resource-atlas/resource-atlas.git
cd resource-atlas

# 安装项目依赖
pip install -r requirements.txt

# 初始化数据库并启动本地服务
python manage.py migrate
python manage.py runserver --host 0.0.0.0 --port 8080
```

服务启动后，访问 http://127.0.0.1:8080 进入 Web 管理界面。首次使用需创建管理员账户，执行 `python manage.py create_admin` 按提示操作。

## 项目结构

```
resource-atlas/
├── cmd/                                    # 命令行入口程序
│   ├── server/                             # Web 服务启动入口 (main.go)
│   └── worker/                             # 后台任务进程 (异步更新资源状态)
├── internal/                               # 内部核心模块，不对外暴露
│   ├── collector/                          # 资源收录引擎，处理 URL 解析与条目生成
│   ├── classifier/                         # 分类器模块，根据 URL 模式与标签规则归类
│   ├── storage/                            # 数据库访问层，封装 CRUD 与迁移逻辑
│   ├── cache/                              # Redis 缓存操作封装，用于高频查询加速
│   └── validator/                          # 链接有效性验证器，支持 HTTP 状态检查
├── pkg/                                    # 可被外部引用的公共库
│   ├── types/                              # 公共数据结构定义（Resource, Series, Tag）
│   └── utils/                              # 工具函数（时间处理、ID 生成、字符串操作）
├── api/                                    # HTTP 路由与处理器定义
│   ├── handlers/                           # 各端点的业务逻辑处理器
│   └── middleware/                         # 认证、日志、限流等中间件
├── web/                                    # Web 前端资源（Vue 3 单页应用）
│   ├── src/                                # 前端源码目录
│   └── dist/                               # 构建输出目录（部署时使用）
├── configs/                                # 配置文件模板与示例
│   ├── development.yaml                    # 开发环境配置样例
│   └── production.yaml                     # 生产环境配置样例
├── scripts/                                # 辅助脚本（数据迁移、测试数据生成）
├── tests/                                  # 单元测试与集成测试用例
├── docs/                                   # 完整文档目录（参见文档导航章节）
├── go.mod                                  # Go 模块依赖定义
├── go.sum                                  # 依赖校验文件
└── README.md                               # 本文档
```

## 贡献指南

本项目欢迎外部贡献者参与改进。请遵循以下流程提交贡献。

1. 阅读项目文档与贡献规范。在提交代码或资源之前，请先阅读 docs/contributing.md 了解代码风格、测试要求与提交信息格式规范。

2. 在 Issue 列表中认领或创建任务。所有功能添加、缺陷修复与文档改进均应关联至少一个 Issue，避免重复工作。对于新增资源收录规则，请同时提供对应的测试用例。

3. 派生项目仓库并在本地开发分支上工作。分支命名遵循 feat/xxx、fix/xxx 或 docs/xxx 前缀，确保与主分支保持同步。

## 常见问题

**问：项目支持的资源数量上限是多少？单次导入最多可处理多少条 URL？**

答：本系统设计上不设硬性上限，实际承载能力取决于部署环境。SQLite 后端推荐单表条目不超过 10 万条，PostgreSQL 后端可支撑百万级条目。单次导入接口支持最大 10000 条 URL，超过此数量请分批提交。本项目批次为第 626/1241 批，共 250 个资源链接，属于常规规模。

**问：如何验证已收录的链接是否仍然有效？**

答：系统内置了 validator 模块，支持定时任务或手动触发对所有资源进行 HTTP HEAD/GET 请求检查。检查结果会更新资源的 status 字段，并可配置失效通知。默认每 7 天自动执行一次全量校验，也可通过管理界面手动启动。

**问：能否导入其他格式的资源数据？是否支持从浏览器书签导出文件转换？**

答：当前版本支持 CSV 与 JSON 导入，字段映射规则参见 docs/data-format.md。对于浏览器书签 HTML 导出文件，项目提供了一个转换脚本 scripts/convert_bookmarks.py，可将 Chrome/Firefox 书签格式转为系统可识别的 CSV 格式后再导入。Edge 和 Safari 的书签需先转换为通用 HTML 格式后再使用该脚本。
