# GBDLS Resource Aggregator

GBDLS Resource Aggregator 是一个面向技术研究人员、数据分析师和开发者的结构化外链资源汇总系统。该项目将分散于网络各处的技术文档、数据源、工具链入口和知识库链接进行统一采集、分类和索引，旨在解决个人收藏夹混乱、链接失效、检索效率低下等常见问题。系统以纯静态 Markdown 方式交付，无需数据库和后台服务，开箱即用，适合作为团队知识库基础组件或个人技术导航站。

项目核心定位为「链接元数据管理工具」，不存储任何第三方内容，仅保留原始 URL 及人工标注的上下文标签。用户可通过目录树和分类表格快速定位所需资源，配合全文搜索插件可实现秒级检索。当前批次为第 1230/1241 批，包含 250 个经过初步筛选的外链资源。

## 功能概览

**分级目录索引**：按技术领域、数据来源、工具类型等多维度生成层级目录，支持无限级嵌套。

**批量链接导入**：支持从 CSV、JSON、纯文本列表批量导入 URL，自动去重并生成标准化条目。

**元数据标签系统**：每条链接可附加版本号、协议类型、访问频率、最后验证时间等自定义标签。

**离线可用性检查**：内置链接存活检测脚本，定时输出失效列表，便于维护更新。

**多格式导出**：支持将索引导出为 HTML 导航页、JSON API 数据、XML 站点地图三种格式。

**全文检索接口**：提供轻量级 Lunr.js 搜索适配器，可在静态页面中实现关键词定位。

**权限分级视图**：支持配置 public / internal / restricted 三级可见性，适配不同团队场景。

**自动备份机制**：每次更新时生成时间戳快照，保留最近 30 个历史版本用于回滚。

## 应用场景

技术团队内部知识库导航：开发团队可将项目部署为内部技术文档门户的入口页面，新成员通过浏览分类目录即可快速找到编码规范、API 设计文档、运维手册和监控面板地址，大幅降低 onboarding 阶段的信息搜寻成本。

数据采集任务调度辅助：数据工程师可在采集任务开始前，通过本项目的「数据源」分类快速核验所有目标接口的 URL 是否仍处于活跃状态，结合存活检测脚本提前发现失效端点，避免任务批量失败。

个人技术阅读工作流管理：技术人员可将日常积累的博客、教程、论文链接按主题导入系统，配合标签功能标记阅读状态（待读/已读/需精读），替代浏览器收藏夹的无序堆积。

离线文档生成前置处理：在制作技术文档离线包或电子书时，可先用本项目统一整理所有外部引用链接，生成清单后交由爬虫工具批量归档，确保离线内容的外链完整性。

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/gbdls/resource-aggregator.git
cd resource-aggregator

# 安装依赖（仅需 Python 3 标准库 + requests）
pip install requests

# 运行索引生成脚本
python build_index.py --input ./data/batch_1230.txt --output ./dist/index.md

# 启动本地预览服务（Python 3）
python -m http.server 8000 --directory ./dist
```

## 贡献指南

1. 在 GitHub 仓库中 Fork 项目到个人账户，然后克隆到本地开发环境。新建分支时请使用 `feat/` 或 `fix/` 前缀，例如 `feat/add-batch-1232`。

2. 新增或更新链接时，请在 `data/` 目录下对应的批次文件中按行追加 URL，同时更新 `data/tags.json` 中的标签定义。对于失效链接，在 `data/deprecated.txt` 中记录并注明日期。

3. 运行 `python scripts/build_index.py --full` 执行完整构建，确保 `dist/` 目录下的所有输出文件均能正常生成。构建过程中如遇格式错误，脚本会输出具体行号，请据此修正。

4. 提交前运行 `python scripts/check_alive.py --timeout 5` 对新增链接进行存活检查，若存在不可达链接，请确认链接有效性或添加 `--skip-dead` 标志说明原因。单元测试可通过 `pytest tests/` 执行。

5. 发起 Pull Request 时，请在描述中明确说明本次变更涉及的批次编号和链接数量，并附上构建日志中无错误、无警告的截图或文本片段。维护者将在 48 小时内完成审核。

## 常见问题

问：构建脚本报错「Line 23: invalid URL format」，如何排查？
答：该错误通常表示批次文件中某一行不是合法的 HTTP/HTTPS URL。请检查对应行是否包含多余空格、制表符或非 ASCII 字符。确认无误后可使用 `--strict` 参数重新运行，脚本会输出更详细的上下文信息。

问：链接存活检测经常超时，导致构建失败怎么办？
答：部分目标服务器可能屏蔽 HEAD 请求或响应较慢。可通过 `--timeout 10` 参数将等待时间延长至 10 秒。若仍大面积超时，建议检查网络环境或使用 `--skip-check` 跳过检测，在日志中备注手动验证结果。

问：如何将本项目的索引数据迁移到其他静态站点生成器（如 Hugo 或 VuePress）？
答：使用 `python scripts/export_json.py --output ./export/` 可生成标准 JSON 格式数据文件，该文件包含全部链接、标签、分类信息。您可在目标生成器中编写数据读取组件，将 JSON 渲染为对应主题的页面。项目本身不绑定任何特定前端框架。
