# LinkVault Resource Aggregator

LinkVault is a high-performance, open-source resource aggregation and navigation system designed for technical teams, researchers, and content curators who need to manage, organize, and share large collections of external links. Unlike traditional bookmark managers, LinkVault provides a structured classification engine, automated availability checking, and a lightweight web interface for browsing categorized resources.

This project targets system administrators, DevOps engineers, and technical writers who handle hundreds of external references, documentation links, or API endpoint lists. LinkVault reduces the overhead of manual link organization through rule-based categorization, duplicate detection, and batch import/export capabilities. The system operates as a static site generator with optional dynamic backend services, making it suitable for both public documentation hubs and private team knowledge bases.

## 功能概览

- **批量链接导入与解析** - 支持从 CSV、JSON 及纯文本列表批量导入 URL，自动提取域名、路径参数及查询字符串，生成标准化资源记录。

- **规则驱动的自动分类引擎** - 基于可配置的域名白名单、路径前缀匹配和关键词标签系统，将导入链接自动归入预定义类别，减少人工整理成本。

- **实时可用性探测** - 后台异步任务定期对已存储链接发起 HEAD 请求，检测 HTTP 状态码及响应时间，标记失效或慢速资源，提供健康状态仪表板。

- **多维度检索与过滤** - 提供按类别、标签、域名、状态码及最后验证时间等多条件组合过滤，支持全文搜索标题与描述字段。

- **静态站点生成与部署** - 一键生成完整的静态 HTML 目录页面，包含分类索引、标签云和资源卡片，可直接托管至任何 Web 服务器或 CDN。

- **RESTful API 接口** - 提供完整的 JSON API 用于第三方系统集成，支持资源的增删改查、批量更新及分类树获取，便于构建前端应用或自动化脚本。

- **链接关系图谱** - 分析链接之间的引用关系及域名共现模式，生成可视化图谱数据，帮助理解资源间的潜在关联。

- **导入导出互操作性** - 支持将资源库导出为标准 HTML 书签文件、OPML 订阅列表及 Markdown 表格格式，方便迁移至其他工具。

## 应用场景

- **技术文档中心资源索引** - 大型开源项目的文档团队可使用 LinkVault 维护外部参考链接库，将分散在各章节中的 API 文档、教程文章和社区论坛链接统一管理，并在每次文档构建时自动生成最新的外部资源附录。

- **DevOps 监控仪表板辅助** - 运维团队可将所有监控系统、日志查看器、数据库管理界面的访问地址录入 LinkVault，利用可用性探测功能定期检查各内部工具的连通性，快速定位故障面板。

- **学术研究文献参考整理** - 研究人员在文献调研阶段收集的大量预印本、数据集页面和工具仓库地址，可通过 LinkVault 进行分类标注，并生成按主题组织的参考列表，便于协作撰写综述论文。

- **企业内部知识库导航** - 企业技术部门可将各部门的 Confluence 空间、Jira 项目面板、代码仓库和 CI/CD 流水线链接统一纳管，构建一站式内部资源门户，减少新员工寻找基础设施入口的时间。

- **开源项目 README 外链维护** - 开源项目维护者可使用 LinkVault 管理项目 README 中引用的所有外部资源，当某个链接失效时自动收到通知，避免文档中出现死链影响用户体验。

## 快速开始

以下指令适用于 Linux 及 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化 SQLite 数据库并运行开发服务器
python manage.py migrate
python manage.py load_sample_data
python manage.py runserver 0.0.0.0:8000
```

访问 http://localhost:8000 即可浏览示例资源库。生产环境部署请参考 `docs/deployment.md`。

## 项目结构

```

├── cmd/                                # 命令行入口及管理工具

│   │   └── main.go                     # 主程序初始化及路由挂载
│   ├── worker/                         # 后台任务执行器
│   │   └── probe.go                    # 链接可用性探测调度逻辑
│   └── cli/                            # 运维管理命令集
│       ├── import.go                   # 批量导入 CSV/JSON 数据
│       └── export.go                   # 导出为各类格式
│
├── internal/                           # 私有核心库，不对外暴露
│   ├── storage/                        # 数据库抽象层及迁移脚本
│   │   ├── postgres.go                 # PostgreSQL 驱动实现
│   │   ├── sqlite.go                   # SQLite 驱动实现
│   │   └── migrations/                 # 版本化 Schema 变更文件
│   ├── classifier/                     # 规则引擎及自动分类逻辑
│   │   ├── rule.go                     # 正则表达式与标签匹配器
│   │   └── taxonomy.go                 # 分类树构建与持久化
│   ├── probe/                          # HTTP 探测及健康检查
│   │   ├── checker.go                  # 并发请求调度与超时控制
│   │   └── metrics.go                  # 响应时间与状态码统计
│   └── api/                            # RESTful 路由及中间件
│       ├── handler/                    # 资源 CRUD 端点处理器
│       └── auth/                       # API Key 与 JWT 认证
│
├── pkg/                                # 可复用的公共工具包
│   ├── urlutil/                        # URL 解析、标准化及去重
│   ├── htmlgen/                        # 静态站点生成模板渲染引擎
│   └── validator/                      # 链接合规性校验与黑名单过滤
│
├── web/                                # 前端静态资源与模板
```

## 贡献指南

## 常见问题

**Q: 可用性探测任务对目标服务器会造成压力吗？探测频率如何控制？**

A: 每个链接的探测请求为轻量级 HEAD 请求，仅获取响应头信息，不下载主体内容。默认探测间隔为 24 小时，并启用指数退避重试策略。并发探测数可通过配置文件的 `probe.concurrency` 参数调整，默认值为 10，避免对单一域名产生突发流量。用户也可在管理界面手动标记某链接为"不探测"以排除特定资源。

**Q: 静态站点生成模式是否支持动态搜索或用户认证？**

A: 静态生成模式输出的为纯 HTML 页面，不支持服务端动态搜索或用户认证。若需要这些功能，请启用 Web 服务模式（`manage.py runserver` 或生产级部署），该模式提供完整的 REST API 及基于 Session 或 JWT 的认证机制。静态模式更适合公开发布、无交互需求的文档站点。

**Q: 如何从旧版本迁移数据，或在不同数据库之间切换？**

A: 项目内置了数据迁移命令 `manage.py migrate`，支持从 SQLite 导出数据为 JSON 格式，再导入至 PostgreSQL。切换数据库时，修改 `configs/app.yaml` 中的 `database.dsn` 连接字符串，执行 `manage.py migrate` 自动建表，然后运行 `manage.py import --from json --file backup.json` 恢复数据。完整迁移步骤参见 `docs/migration.md`。
