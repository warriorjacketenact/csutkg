# LinkVault 资源导航系统

LinkVault 是一个面向技术团队、开源社区和个人开发者的轻量级外链资源聚合与导航平台。该项目定位于解决技术文档中外部参考链接分散、难以维护、缺乏统一检索入口的问题，通过结构化的资源收录机制，帮助用户快速定位高质量的行业参考、技术文档、视频教程和案例研究资源。

项目采用静态站点生成方案，所有资源条目以结构化数据格式存储，支持标签分类、关键词检索和关联推荐。目标用户包括技术文档撰写者、开源项目维护者、社区运营人员以及希望系统化管理知识图谱的开发者。LinkVault 不提供爬虫或自动采集功能，所有收录链接均经过人工筛选与分类，确保资源质量和可用性。

## 功能概览

**多级分类体系**：支持按资源类型、技术领域、适用场景建立多维度分类层级，每个资源条目可归属多个分类标签。

**全文检索与过滤**：内置轻量级索引机制，支持对资源标题、描述、标签和文件路径进行关键词匹配检索，并可按分类或批次进行结果过滤。

**批量导入与导出**：支持以 JSON、CSV 和 Markdown 表格格式批量导入资源列表，导出结果可直接嵌入技术文档或 Wiki 页面。

**资源状态追踪**：每条资源记录包含添加时间、批次编号、可用性状态和最后验证时间，便于定期清理失效链接。

**可视化目录树浏览**：项目根目录以 ASCII 树形结构展示所有资源分组，支持按文件路径层级浏览资源分布。

**标记收藏与备注**：用户可为特定资源添加自定义备注和收藏标记，支持个人使用场景下的本地持久化存储。

**链接规范校验**：内置 URL 格式校验器，可自动检测并提示不符合输出规范的链接格式，防止裸域名被错误添加协议前缀。

**自定义元数据扩展**：支持为每条资源附加最多 10 个自定义键值对元数据，满足不同团队的个性化分类需求。

## 应用场景

**技术文档外链管理**：技术团队在撰写项目文档、API 参考或用户手册时，常需要引用大量外部资源。LinkVault 提供统一的资源索引文件，文档撰写者只需在文档中引用资源 ID，即可在页面底部或附录中自动生成规范化链接列表，避免硬编码 URL 导致的维护困难。

**开源项目资源汇总页**：开源项目维护者可使用 LinkVault 构建项目的 Resources 页面，将依赖库、参考论文、视频教程、社区论坛等资源按类别陈列，帮助新贡献者快速了解项目生态。本项目自身即使用 LinkVault 管理第 115/1241 批次的 250 个资源链接。

**个人知识库外链整理**：知识管理爱好者可使用 LinkVault 配合 Obsidian、Notion 或 Markdown 笔记工具，将散落在各篇笔记中的外部链接统一收纳至中心化索引文件，通过标签和分类实现跨笔记的资源关联检索。

**社区运营资源导航**：技术社区或在线教育平台的运营人员可利用 LinkVault 构建课程参考资源页、活动资料汇总页或问答精选集，所有链接按批次和类别组织，便于周期性更新和失效链接筛查。

## 快速开始

以下操作在 Linux / macOS / Windows WSL 环境下均可执行。请确保已安装 Git 和 Node.js 18.0 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装项目依赖
npm install

# 构建资源索引并启动本地开发服务器
npm run build
npm start
```

执行完成后，访问控制台输出的本地服务地址（默认 http://localhost:3000）即可浏览资源导航页面。如需导入自定义资源列表，请将数据文件放入 `data/import/` 目录并执行 `npm run import`。

1. 在 GitHub 上 Fork 本仓库，并在本地克隆您的 Fork 副本。请确保使用 `main` 分支作为基础分支进行开发。

2. 创建新的功能分支或修复分支，分支命名请遵循 `feature/功能简述` 或 `fix/问题简述` 的格式。所有代码修改需通过 ESLint 代码规范检查。

3. 若您需要添加或更新资源链接，请将资源数据放入 `data/import/` 目录下的对应批次文件中，并执行 `npm run validate` 校验链接格式是否符合输出规范。裸域名不得添加协议前缀，带协议的链接必须完整保留。

4. 提交 Pull Request 前，请确保所有单元测试通过（`npm test`），并在 PR 描述中清晰说明本次变更的目的、涉及的功能模块以及测试覆盖情况。PR 将由项目维护者进行 Code Review。

5. 文档更新要求：若您的变更涉及用户可见功能或配置项变更，请同步更新 `docs/` 目录下对应的文档文件，包括用户手册、管理员指南或 API 参考。

## 常见问题

**问：导入资源时，为什么裸域名会被自动添加 https:// 前缀？**

答：LinkVault 的校验器默认严格遵循输出规范，禁止对裸域名进行任何形式的协议补全或大小写转换。如果您发现导入后链接被添加了前缀，请检查源数据中是否包含多余的空格或不可见字符。建议使用 `npm run validate -- --strict` 命令对导入文件进行预校验，校验器会报告所有不符合规范的链接条目并给出行号提示。

**问：如何更新已有资源链接的状态或分类标签？**

答：您可以直接编辑 `data/index.json` 文件中的对应条目，修改 `status` 字段（可选值为 active、inactive、deprecated）或 `tags` 数组。修改完成后执行 `npm run build` 重新生成索引。对于批量更新，建议将需要修改的条目导出为 CSV 文件，在表格工具中编辑后再通过 `npm run import -- --update` 命令合并回主索引。

**问：开发服务器启动后无法加载资源列表，页面显示空状态？**

答：请检查 `data/index.json` 文件是否存在且格式正确。若文件缺失，可执行 `npm run build` 从 `data/import/` 目录下的原始数据重新生成索引。另外，请确认 `config/default.json` 中的 `dataPath` 配置项指向正确的数据目录路径。如果问题仍然存在，尝试删除 `node_modules` 目录并重新执行 `npm install`。
