# VueBest 外部资源导航系统

VueBest 是一个面向 Vue.js 生态系统的外部资源导航与聚合系统，致力于为 Vue 开发者提供经过筛选和分类的高质量学习资料、开源项目、工具库以及社区讨论的集中访问入口。该项目本质上是一个技术资源的外链汇总平台，通过人工整理与社区贡献相结合的方式，构建一个可扩展的、主题化的 Vue 相关资源目录。

项目主要面向从初级到高级的 Vue 开发者、技术团队负责人、教育培训机构以及开源贡献者。它帮助用户快速定位与 Vue 核心生态、状态管理、路由、构建工具、UI 组件库、服务端渲染、移动端适配、性能优化、测试策略以及类型系统相关的权威外部链接，有效降低信息检索成本，提升开发效率与学习路径的清晰度。

## 功能概览

**按主题分类的资源索引** 系统将收录的外部 URL 按照 Vue 生态下的功能领域（如状态管理、组件库、工具链）进行逻辑分组，每个条目附带简要的适用场景描述。

**社区驱动的资源提交与审核** 提供标准化的外部资源提交接口，允许社区成员推荐优质链接，经过核心团队审核后即可纳入导航体系。

**全文检索与标签过滤** 基于资源标题、描述、标签和分类字段实现快速的全文搜索，并支持多维度标签过滤，帮助用户精确命中目标资源。

**资源健康度监控** 定期对收录的外部链接进行可用性探测，标记失效或重定向的链接，并在管理后台生成报告，供维护者参考和处理。

**自定义收藏夹与分享** 允许注册用户创建个人收藏夹，将常用资源归类保存，并支持生成收藏夹的只读分享链接，便于团队内部知识传递。

**开放数据导出** 提供 JSON 和 Markdown 格式的完整资源列表导出功能，方便其他工具或文档系统二次集成。

## 应用场景

**新项目技术选型** 技术负责人或架构师在启动新 Vue 项目时，可通过 VueBest 快速对比不同状态管理库（Pinia、Vuex、XState）以及 UI 框架（Element Plus、Ant Design Vue、Naive UI）的官方文档与社区评价，从而做出更明智的技术决策。

**开发者日常学习路径规划** 初级开发者可以根据 VueBest 上的学习资源分类，系统性地从官方教程、实战案例、源码解析到性能调优逐步深入，构建完整的知识体系，避免在海量网络信息中迷失方向。

**团队知识库建设** 技术团队可以将 VueBest 作为内部知识库的外部资源补充，将常用的第三方库文档、设计规范、后端 API 模拟工具等链接统一收藏，并共享给新入职成员，缩短上手周期。

**开源项目生态展示** 开源项目维护者可以在 VueBest 上提交自己的项目链接，借助导航站的流量曝光吸引更多潜在贡献者和使用者，同时通过关联标签让项目被更精准的目标受众发现。

## 快速开始

以下命令演示如何将 VueBest 项目克隆至本地、安装依赖并启动开发服务器。

```bash
# 克隆仓库
git clone https://github.com/vuebest/vuebest-navigator.git

# 进入项目目录
cd vuebest-navigator

# 安装依赖（使用 npm 或 yarn）
npm install

# 启动开发服务器（默认监听 http://localhost:3000）
npm run dev
```

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Node.js | >= 18.0.0 | 项目基于 Node.js 运行时，用于执行构建脚本和开发服务器 |
| npm | >= 9.0.0 或 yarn >= 1.22.0 | 包管理器，用于安装项目所有依赖项 |
| Git | >= 2.30.0 | 版本控制工具，用于克隆仓库和管理提交历史 |
| MongoDB | >= 5.0（推荐 6.0） | 主数据库，用于存储资源条目、用户信息和分类数据 |
| Redis | >= 6.2（推荐 7.0） | 缓存中间件，用于会话存储和接口响应缓存 |
| Docker（可选） | >= 20.10 | 容器化部署支持，用于快速搭建开发环境依赖服务 |

## 项目结构

项目采用模块化分层架构，将前端展示、后端 API、数据持久化、缓存策略和资源健康检查拆分为独立目录，便于维护和扩展。

```
vuebest-navigator/
├── frontend/                           # 前端单页应用源码
│   ├── src/
│   │   ├── pages/                      # 页面级组件（首页、资源详情、提交页、收藏夹）
│   │   ├── components/                 # 可复用 UI 组件（搜索栏、标签过滤、资源卡片）
│   │   ├── stores/                     # Pinia 状态管理模块（用户、资源列表、收藏状态）
│   │   ├── api/                        # 封装后端 API 请求方法
│   │   ├── utils/                      # 工具函数（URL 格式化、日期处理、防抖节流）
│   │   └── styles/                     # 全局样式变量与主题配置
│   ├── public/                         # 静态资源（favicon、站点清单、robots.txt）
│   └── vite.config.js                  # Vite 构建配置
├── backend/                            # Node.js + Express 后端服务
│   ├── src/
│   │   ├── controllers/                # 路由控制器（资源 CRUD、用户认证、分类管理）
│   │   ├── models/                     # MongoDB 数据模型（Resource、User、Category、Tag）
│   │   ├── middleware/                 # 中间件（JWT 鉴权、日志记录、错误处理、限流）
│   │   ├── services/                   # 业务逻辑层（资源审核、链接健康检查、缓存策略）
│   │   ├── routes/                     # RESTful 路由定义（v1 版本前缀）
│   │   ├── validators/                 # 请求参数校验（Joi 或 Zod 模式）
│   │   └── config/                     # 配置加载（环境变量解析、数据库连接参数）
│   ├── tests/                          # 单元测试与集成测试（Jest / Mocha）
│   └── app.js                          # 应用入口
├── docker/                             # Docker 编排与容器配置
│   ├── docker-compose.yml              # 全服务编排（Node、MongoDB、Redis）
│   ├── Dockerfile.backend              # 后端服务镜像构建脚本
│   └── Dockerfile.frontend             # 前端静态服务镜像构建脚本
├── docs/                               # 项目文档（用户手册、API 参考、部署指南）
│   ├── user-guide/                     # 用户使用说明
│   ├── maintainer/                     # 维护者操作手册
│   ├── api/                            # API 接口文档（OpenAPI 规范）
│   └── deployment/                     # 生产环境部署手册
```

## 贡献指南

欢迎社区贡献者参与 VueBest 的建设和完善。请遵循以下步骤提交您的贡献。

1.  **查阅贡献者行为准则** 在开始贡献之前，请阅读项目根目录下的 CODE_OF_CONDUCT.md 文件，确保您的行为符合社区友好的协作精神。

2.  **提交资源推荐或问题反馈** 通过 GitHub Issues 提交新的资源链接建议，或者报告已收录链接的失效情况。请使用提供的 Issue 模板填写资源标题、URL、所属分类及推荐理由。

3.  **本地开发环境准备** 克隆项目仓库，按照安装要求配置 Node.js、MongoDB 和 Redis 环境，并运行 npm install 安装所有依赖。创建 .env 文件并填入正确的数据库连接字符串。

4.  **创建特性分支并提交代码** 对于非资源推荐的代码贡献（如功能增强、Bug 修复），请从 main 分支创建以 feature/ 或 fix/ 为前缀的分支。编写代码时遵循 ESLint 和 Prettier 配置，确保单元测试通过。

5.  **发起 Pull Request** 推送分支至远程仓库后，在 GitHub 页面发起 Pull Request。在 PR 描述中清晰说明改动内容、影响范围以及相关的 Issue 编号。核心维护者会在 3 个工作日内进行 Code Review 并合并。

## 常见问题

**Q: 如何快速区分已收录资源的分类标签？**

A: 每个资源条目在提交和审核阶段会被赋予至少一个一级分类（如「状态管理」「UI 组件」「构建工具」）和若干个二级标签（如「Pinia」「Vuex」「Vite」）。在网站首页和搜索页面，用户可以直接点击分类或标签进行过滤。同时，资源详情页会显示完整的分类路径，便于理解其在整个生态中的定位。

**Q: 发现收录的链接已经失效或内容发生重大变化，应该如何处理？**

A: 您可以通过两种方式通知维护团队：一是直接在 GitHub Issues 中选择「链接失效报告」模板提交，系统会自动生成包含 URL 和检测时间的工单；二是使用网站页面下方的「报告问题」浮动按钮，填写简要说明。维护团队会定期（每周一次）处理反馈，并在 72 小时内更新或移除失效链接。

**Q: 是否可以申请在 VueBest 上置顶或推荐某个特定资源？**

A: 目前 VueBest 不提供付费置顶或商业推广位置。所有资源的排序均基于收录时间、社区评分和点击热度综合计算。如果您希望提升某个资源的可见度，建议鼓励团队或社区成员为该项目在 GitHub 上点 Star，并在 VueBest 上提交详细的资源描述和使用心得，高质量的详细描述会获得更高的展示权重。
