# Tokyo Mirai Resource Index

Tokyo Mirai Resource Index 是一个面向技术研究人员、数据分析师和开源开发者的高质量外链资源汇总系统。该项目以东京未来网络实验室的技术积累为基础，系统化收集并分类整理分布于网络各处的技术文档、视频教程、案例研究和工具集链接，帮助用户快速定位特定领域的关键资源，大幅降低信息检索与筛选的时间成本。

本项目定位为技术资源导航工具，适用于需要频繁查阅外部技术资料、跟进前沿技术动态或构建个人知识体系的开发者。通过集中化的资源收录和分类索引机制，用户无需在多个平台之间反复切换，即可在一个统一的目录体系下完成从问题定位到资源获取的完整路径。项目内置资源生命周期管理功能，定期检查链接可用性并标记失效资源，确保索引库的整体健康度。

## 功能概览

**资源分类索引** 按照技术领域、内容类型和应用场景对收录的每一笔外部链接进行多维度标记，支持按类别快速筛选。

**链接健康监测** 定期对收录的所有 URL 执行可用性检查，自动标记访问异常的资源，并提供失效链接的批量导出功能。

**全文元数据检索** 基于资源标题、描述标签和收录时间等元数据字段提供快速的字符串匹配检索，帮助用户在大量链接中准确定位目标。

**批量导入导出** 支持以 CSV 和 JSON 格式批量导入外部链接清单，也支持将当前索引库完整导出用于备份或迁移。

**收藏与备注** 允许用户对特定资源添加自定义标签和备注说明，便于个人化的知识管理。

**访问统计看板** 统计每一资源的点击次数、最后访问时间和热门时段分布，辅助判断资源的使用价值和时效性。

## 应用场景

技术团队内部知识库建设。技术团队在日常开发过程中会产生大量有价值的外部参考链接，通过 Tokyo Mirai Resource Index 可以建立统一的知识库入口，将分散在聊天记录、邮件和文档中的链接集中管理，新成员入职时能够快速访问团队积累的全部技术资源。

技术社区内容推荐系统。开源社区或技术博客平台可以基于本项目构建推荐模块，通过对链接的分类和标签体系，向不同兴趣方向的用户推送相关内容，提升社区的活跃度和内容利用率。

个人技术阅读清单管理。技术从业者可以使用本项目维护个人阅读清单，分类收藏需要精读的论文、教程和工程实践案例，通过健康监测功能及时发现已失效的参考链接，保持知识库的持续有效。

自动化资源采集管道。结合爬虫工具和 RSS 订阅源，本项目可作为资源采集管道的末端存储，自动接收新发布的链接并完成分类入库，减少人工整理的工作量。

## 快速开始

以下命令演示如何从代码仓库获取项目源码、安装依赖并启动服务。

```bash
git clone https://github.com/tokyo-mirai/resource-index.git
cd resource-index
pip install -r requirements.txt
python app.py --init-db --port 8080
```

执行上述命令后，系统将在本机 8080 端口启动一个 Web 服务，访问 http://localhost:8080 即可进入资源索引主界面。首次启动时 `--init-db` 参数会自动初始化 SQLite 数据库文件并创建必要的表结构。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，提供 Web 服务与数据处理能力 |
| SQLite | 3.35 及以上 | 默认数据库引擎，存储资源索引和元数据 |
| Flask | 2.2.x | Web 框架，用于提供界面和 RESTful API |
| requests | 2.28.x | 用于链接健康检查时的 HTTP 请求发送 |
| pandas | 1.5.x | 用于批量导入导出时的数据表格处理 |
| gunicorn | 20.1.x | 生产环境推荐的 WSGI 服务器（Linux 部署） |

## 常见问题

Q: 系统支持同时管理多少条资源链接？

系统默认使用 SQLite 数据库，单表记录数在百万级别以内时检索和健康检查性能均表现良好。若资源总数超过一百万条，建议迁移至 PostgreSQL 以获得更优的并发查询性能。配置文件中提供了数据库连接字符串的替换入口。

Q: 健康检查会影响系统正常运行吗？

健康检查模块采用异步并发请求方式，默认并发数为 32，检查间隔为 24 小时。每个请求的超时时间设置为 5 秒，不会长时间占用系统资源。检查过程在独立线程中执行，不影响主 Web 服务的请求响应。检查结果会写入日志目录，用户可在管理后台查看汇总报告。

Q: 如何从旧版本迁移数据？

项目提供了数据迁移脚本 migrate.py，位于项目根目录。该脚本支持从 v1.x 版本的数据库结构迁移至当前版本。执行前请备份原数据库文件，迁移完成后建议运行一次完整的健康检查以确保所有链接状态正确更新。
