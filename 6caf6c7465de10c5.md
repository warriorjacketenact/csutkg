# IndexHub

IndexHub 是一个面向技术研究者和信息分析人员的轻量级外链聚合与导航系统。该项目不对任何外部资源进行内容转载或镜像存储，仅提供结构化索引与快速跳转能力，帮助用户在特定主题领域内高效定位分散的高价值信息页面。IndexHub 适用于建立个人知识导航、团队共享书签池以及垂直领域情报看板。

IndexHub 的核心定位是“链接的链接”。它不生产内容，也不替代搜索引擎，而是通过人工或半自动化的方式，将散落于不同域名、不同路径下的高质量外链按照主题、类型或时间维度进行重新组织，最终以清晰的目录树和标签系统呈现给终端用户。本项目尤其适合需要定期追踪特定来源列表、但又不希望被通用搜索引擎噪声干扰的深度阅读者。

## 功能概览

- **批量链接导入与解析**：支持从纯文本、CSV 或简单 Markdown 列表中批量导入原始 URL，自动提取域名、路径层级和文件类型标识，为后续分类做准备。

- **多维度标签分类系统**：允许用户为每条链接添加自定义标签，例如按主题、语言、文件格式或可信度分级，并支持多标签联合筛选。

- **静态站点生成输出**：基于配置的链接库和标签体系，一键生成完整的静态 HTML 目录页，无需数据库或后端服务，可直接托管于任意 Web 服务器或 CDN。

- **链接可用性健康检查**：内置周期性 HTTP 头探测模块，可标记失效链接、重定向链过长或响应超时的条目，并生成异常报告。

- **全文元数据缓存与检索**：对每条链接的目标页面进行标题、描述和关键词的轻量级抓取与本地缓存，支持离线环境下的快速标题搜索和内容预览。

- **外部资源版本快照引用**：对于支持互联网存档服务的外部页面，自动生成指向 archive.org 等平台的永久快照链接，增强长期可追溯性。

- **访问统计与热度排序**：基于本地日志或简单的点击计数机制，记录高频访问链接，并支持按七日热度或总点击量进行动态排序。

## 应用场景

- **技术文档聚合门户**：研发团队可将分散在不同官方文档站、博客平台和 GitHub 仓库的 API 参考、设计文档和运维手册集中收录于 IndexHub，形成统一的内部文档导航入口，减少新成员寻找资料的时间成本。

- **行业资讯每日读本**：研究员或分析师可将关注的行业媒体、智库报告发布页、政府统计数据源和头部企业公告栏整理为固定链接集，每日通过 IndexHub 的健康检查功能快速确认哪些来源发布了新内容。

- **开源项目依赖链追踪**：大型开源项目的维护者可以利用 IndexHub 收录核心依赖库的官网、Issue 追踪器、邮件列表归档和 CI 状态面板，在一处即可俯瞰整个生态的实时状态。

- **个人知识收藏夹替代方案**：重度网络阅读者可将 IndexHub 作为书签管理工具的高级替代，利用标签和全文检索功能管理数百个长尾收藏链接，避免浏览器自带书签系统在数量膨胀后的检索困难。

## 快速开始

以下命令适用于 Linux 和 macOS 环境，Windows 用户可通过 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/indexhub/indexhub.git
cd indexhub

# 安装依赖 (Python 3.9+)
pip install -r requirements.txt

# 使用示例链接库生成初始站点
python indexhub build --input ./data/example_links.txt --output ./dist
```

执行完成后，`./dist` 目录下将生成包含索引页、分类页和详情页的完整静态站点，可直接使用任何静态文件服务器启动查看。

## 安装要求

IndexHub 基于 Python 开发，依赖轻量级 HTTP 客户端和模板引擎。生产环境部署建议使用 Linux 系统，开发环境支持跨平台。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | 3.9 - 3.12 | 核心解释器，低于 3.9 将无法使用类型注解新特性 |
| requests | 2.28.0+ | 用于链接健康检查与元数据抓取的 HTTP 客户端库 |
| jinja2 | 3.1.0+ | 静态站点生成所使用的模板渲染引擎 |
| click | 8.0.0+ | 命令行接口解析器，提供 build、check、stats 等子命令 |
| beautifulsoup4 | 4.11.0+ | 用于解析目标页面 title 和 meta 标签的 HTML 解析库 |
| lxml | 4.9.0+ | beautifulsoup4 的底层解析加速器，显著提升大文档处理速度 |
| pytest | 7.0.0+ | 单元测试框架，仅开发阶段需要，生产环境可不安装 |
| flake8 | 5.0.0+ | 代码风格检查工具，仅贡献者开发时使用 |

## 贡献指南

IndexHub 欢迎任何形式的贡献，包括但不限于新增功能、修复缺陷、改进文档或提供更丰富的示例链接库。请遵循以下流程以确保协作顺畅。

第一步，在 GitHub 上 fork 本项目至个人账户，并将 fork 后的仓库克隆到本地开发环境。建议在 dev 分支上进行所有改动，避免直接操作 main 分支。

第二步，安装开发依赖并激活预提交钩子。执行 `pip install -r requirements-dev.txt` 安装 pytest、flake8 等工具，随后运行 `pre-commit install` 启用代码风格自动检查。

第三步，编写代码或文档时，请遵循 PEP 8 编码规范，并为所有新增函数或类添加清晰的 docstring。单元测试必须覆盖新增逻辑的核心路径，保证测试通过率不下降。

第四步，提交变更时使用约定式提交格式，例如 `feat: add support for json input` 或 `fix: handle redirect chain timeout`。提交信息应简明扼要地说明改动意图和影响范围。

第五步，推送分支至个人远程仓库，并在 GitHub 上向主仓库的 main 分支发起 Pull Request。PR 描述中需说明改动动机、实现方式以及手动测试结果，维护者将在三个工作日内进行审查。

## 常见问题

Q: IndexHub 是否会存储外部页面的完整内容或文本？

A: 不会。IndexHub 仅缓存目标页面的标题、meta description 和 response status，不保存正文文本、图片或任何用户隐私数据。所有缓存均存储在本地 SQLite 文件中，默认不对外公开。

Q: 链接健康检查是否会频繁请求目标服务器，造成对方负担？

A: 健康检查模块默认采用顺序请求而非并发爆发，且每个目标域名在 60 秒内最多被探测一次。用户可通过配置 `check_interval` 参数调整探测频率，生产环境建议设置为 3600 秒（每小时一次）以上。

Q: 如何迁移已有的浏览器书签或 Pocket 收藏到 IndexHub？

A: 主流浏览器支持导出书签为 HTML 或 CSV 文件，Pocket 也支持导出 HTML 格式。用户可使用 `indexhub import --from bookmarks.html` 命令将上述格式转换为 IndexHub 的标准链接库格式。具体转换规则参见 `docs/link-format.md` 中的迁移示例。
