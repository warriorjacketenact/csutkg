# GXHY Resource Indexer

GXHY Resource Indexer 是一个面向技术社区的资源链接汇总与导航系统，专为需要系统化整理、检索和分享外部技术资源（如教程、文档、工具链、社区讨论帖）的开发者与内容策展人设计。该项目并非简单的书签管理工具，而是一个具备元数据提取、分类索引与快速查询能力的轻量级资源门户后端。

本项目主要解决以下问题：技术资源散落在论坛、博客、代码托管平台等多个孤立站点，开发者检索特定主题时需要在多个站点间反复跳转，且无法对资源的时效性、所属技术领域、内容质量进行批量评估与筛选。GXHY Resource Indexer 通过统一的数据入口与标准化的资源记录格式，帮助用户构建可维护、可扩展的技术知识索引体系。

## 功能概览

- **资源链接聚合管理**：支持将分散在各类技术社区、论坛、文档站点的 URL 统一收录，并为每个资源记录添加标题、标签、所属批次与收录时间戳。
- **元数据自动补全**：对收录的链接进行可配置的元数据抓取（如页面标题、摘要文本、内容类型识别），辅助用户快速了解资源概要。
- **多级分类与标签系统**：允许用户为每个资源分配一个主分类及多个自定义标签，支持按分类、标签、批次号进行组合过滤与检索。
- **批量导入与导出**：提供基于文本文件或 CSV 格式的批量 URL 导入接口，并支持将索引结果导出为结构化数据（JSON、CSV）供外部工具使用。
- **关键词全文检索**：基于资源标题、描述、标签与自定义备注构建轻量级倒排索引，支持布尔逻辑与通配符查询。
- **资源状态追踪**：记录每个资源的添加时间、最后访问时间、访问状态码，辅助用户识别失效链接或内容变更。
- **批次管理**：项目内置批次编号机制，每批资源可独立命名、备注，便于团队协作或定期更新。
- **命令行交互界面**：提供完整的 CLI 工具，支持资源添加、查询、更新、删除等日常操作，无需图形界面即可高效管理。

## 应用场景

- **技术团队内部知识库建设**：研发团队可将日常调研中发现的优秀技术博客、API 文档、设计规范链接统一录入系统，通过标签区分前端、后端、运维、安全等不同领域，新成员入职时可快速浏览团队推荐的外部学习资源。
- **开源项目文档外链维护**：开源项目维护者可在项目仓库中集成该索引器，将项目依赖的第三方库文档、社区讨论帖、问题修复参考链接进行系统化管理，避免文档中的外链散落各处且难以更新。
- **技术资讯周报素材收集**：技术编辑或社区运营人员可将一周内浏览到的有价值讨论帖、发布公告、工具发布新闻暂存于索引器，通过批次功能（如 2026-06-23 周报）进行分类汇总，导出为 Markdown 列表后直接用于周报撰写。
- **个人技术阅读工作流管理**：开发者可将待阅读的技术文章、待验证的代码示例链接暂存于待读分类，利用检索功能快速筛选出未读资源，配合状态追踪避免重复收藏或遗漏重要内容。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/your-org/gxhy-resource-indexer.git
cd gxhy-resource-indexer

# 安装依赖（Python 3.9+ 环境）
pip install -r requirements.txt

# 初始化本地数据库与配置文件
python indexer.py init --db-path ./data/resources.db

# 添加一批资源链接（示例）
python indexer.py add --batch 288 --source ./data/urls_288.txt

# 启动本地 Web 查询界面（可选）
python indexer.py serve --port 8080
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9.0 或更高 | 核心运行环境，建议使用 3.11+ 以获得性能优化 |
| SQLite | 3.35.0 或更高 | 本地元数据存储引擎，Python 内置支持，无需额外安装 |
| requests | 2.31.0 或更高 | 用于 HTTP 请求与元数据抓取，支持自定义 User-Agent |
| click | 8.1.0 或更高 | CLI 命令行框架，提供子命令与参数解析能力 |
| beautifulsoup4 | 4.12.0 或更高 | 用于解析 HTML 页面标题与摘要信息（可选，启用元数据补全时需要） |
| lxml | 4.9.0 或更高 | BeautifulSoup 的解析器后端，提供更高效的 HTML 解析性能 |
| pytest | 7.4.0 或更高 | 单元测试框架，仅在开发环境执行测试时需要 |
| black | 23.0.0 或更高 | 代码格式化工具，仅在贡献代码时使用 |

## 常见问题

**Q: 该项目是否支持非技术类资源的索引？**

A: 支持。虽然项目定位偏向技术资源汇总，但其数据模型与检索功能不限定内容领域。用户可通过自定义标签与分类来适应任何类型的 URL 资源（如设计素材、学术论文、新闻资讯）。唯一的技术依赖是元数据补全功能对 HTML 页面的解析，但该功能可通过配置关闭，不影响基础索引能力。

**Q: 元数据抓取失败会影响资源收录吗？**

A: 不会。元数据抓取是独立的后台任务，默认在资源添加时异步执行。若因网络超时、目标站点反爬或内容结构变化导致抓取失败，系统会记录失败状态并仅保存用户输入的原始 URL 与自定义备注，资源记录本身仍然完整可用。用户可通过 update-metadata 命令重试失败资源。

**Q: 如何迁移数据库到其他服务器或从 SQLite 切换到 PostgreSQL？**

A: 项目原生支持 SQLite，适合单机或小规模团队使用。若需迁移至 PostgreSQL，可使用项目自带的 scripts/export_batch.py 导出全部资源为 JSON 格式，再编写简单的导入脚本将数据写入 PostgreSQL。项目未内置 PostgreSQL 适配器，但数据模型与 ORM 解耦，用户可自行扩展 database.py 模块以支持其他关系型数据库。
