# RY MFS External Resource Aggregation System

RY MFS 是一个面向技术团队与独立开发者的外部资源聚合与导航系统。该项目定位于对分散在不同路径下的技术文档、工具站点、参考材料与社区内容进行结构化收录与快速检索，帮助用户在不依赖搜索引擎的情况下，通过统一的本地化索引访问高频使用的外部技术资源。

项目主要服务于需要频繁查阅外部技术资料、维护多项目依赖文档、以及进行技术选型调研的开发者群体。通过将零散的 URL 资源按主题与用途分类，RY MFS 解决了传统书签管理方式中检索效率低、缺乏版本追踪、无法与项目工程目录同步的问题。

## 功能概览

**资源分级索引** 系统按照技术栈与使用场景对收录的外部链接进行一级分类，支持用户自定义标签体系，实现多维度资源定位。

**本地缓存与离线访问** 对于支持静态导出的目标页面，系统提供本地缓存机制，允许在网络受限环境下继续查阅关键文档内容。

**变更监控与更新提醒** 系统定期对收录资源进行可用性检测与内容变更扫描，当目标页面发生结构性调整时向维护者发送通知。

**快捷检索接口** 提供基于关键词、标签、收录时间与访问频率的检索能力，检索结果支持按相关性排序。

**导入导出兼容性** 资源列表支持以 JSON、YAML 及 Markdown 表格格式导入导出，便于与其他文档工具或团队知识库系统对接。

**部署灵活性** 系统可运行于本地开发环境、私有服务器或容器化平台，不强制依赖公有云服务。

## 应用场景

技术团队内部知识库建设
技术团队可将 RY MFS 作为知识库的入口层，将日常开发中涉及的设计文档、API 参考、运维手册等外部资源统一收录，并与团队 Wiki 或项目 README 进行交叉引用，降低新成员的上手成本。

技术选型与方案对比
在进行技术选型时，开发者可使用 RY MFS 收集候选技术栈的官方网站、性能测试报告、社区讨论帖与迁移指南，通过分类标签与备注功能记录对比结论，形成可追溯的选型文档。

离线开发环境搭建
对于需要在无公网环境或内网隔离环境中进行开发的场景，RY MFS 的本地缓存功能可将核心依赖文档与工具下载页面提前同步至本地，确保开发流程不受网络条件限制。

开源项目依赖管理
开源项目维护者可使用 RY MFS 记录上游依赖的项目主页、Issue 追踪地址、版本发布日志与安全公告，当上游出现重要更新或安全漏洞时能够第一时间获知。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议通过 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/rymfs/rymfs-agg.git
cd rymfs-agg

# 安装项目依赖（需确保 Node.js 18+ 与 pnpm 已预置）
pnpm install

# 执行初始化资源索引构建
pnpm run build:index

# 启动本地开发服务器
pnpm run dev
```

访问终端输出的本地地址（默认 http://localhost:5173）即可进入资源导航界面。

## 贡献指南

欢迎社区开发者参与 RY MFS 项目的建设与维护。所有贡献需遵循以下流程：

1. 在 GitHub Issues 中查阅现有任务列表，确认无重复工作后创建新的 Issue 描述拟解决的问题或新增功能，等待维护者标注状态后开始实施。

2. 从主分支 main 或 develop 切出功能分支，分支命名遵循 feat/功能名 或 fix/问题简述 的格式。提交信息需符合 Conventional Commits 规范，便于自动生成变更日志。

3. 在本地环境完成开发与自测后，推送到个人远程仓库并发起 Pull Request 至 develop 分支。PR 描述中需关联对应 Issue 编号，并附上测试结果或截图说明。

4. 提交代码需通过项目 CI 流水线中的类型检查、单元测试与代码风格校验。对于涉及资源索引格式或数据库表结构变更的 PR，需同步更新 data/migration 目录下的迁移脚本。

## 常见问题

Q: 系统启动后无法加载资源列表，控制台提示 SQLite 表不存在。
A: 该问题通常出现在首次运行或数据库文件被误删除时。请执行 pnpm run migrate:latest 命令运行所有待执行的数据库迁移脚本，随后执行 pnpm run build:index 重新构建索引。若问题仍然存在，可删除 data/index.db 文件后再次执行上述两条命令，系统将自动重建完整索引。

Q: 如何批量添加自定义外部资源，而不是逐条通过命令行操作？
A: 系统支持通过 YAML 或 JSON 格式的批量导入文件添加资源。请在 configs/imports/ 目录下创建符合 schema 定义的文件，字段包括 url、title、category、tags 与 description。创建完成后执行 pnpm run import:batch --file=自定义文件名 即可完成批量导入。导入模板可参考 docs/developer-guide/import-schema.md。

Q: 变更监控功能是否会频繁发起对外请求，影响目标站点性能？
A: 系统默认采用增量轮询策略，对每个收录资源的检测间隔不低于 24 小时，且请求头中携带了标准的 User-Agent 与限速标识。对于支持 Last-Modified 或 ETag 的站点，系统优先使用条件请求头进行变更检测，仅在资源确实发生变更时才会下载完整内容。用户可在 monitor.config.json 中调整全局轮询频率或针对特定域名设置白名单与黑名单规则。
