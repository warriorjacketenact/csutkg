# VueBest 技术资源导航

VueBest 是一个专注于前端与全栈开发领域的技术资源聚合与导航平台，旨在为开发者提供高质量、可追溯的外部学习资料、工具链文档以及社区讨论索引。本项目不直接托管教程或库文件，而是通过人工筛选与社区提名机制，持续收录与 Vue.js 生态、现代 JavaScript 工程化、UI/UX 设计体系及性能优化相关的优质外链，帮助开发者快速定位权威信息源，减少在低质量内容上的时间消耗。

本项目面向具有 1 年以上前端开发经验的工程师，以及希望从 React 或 Angular 迁移至 Vue 生态的技术团队。通过结构化的频道分类与标签体系，用户可按技术栈、应用场景或问题域进行检索。项目本身以静态站点形式交付，支持本地克隆后通过简单的 HTTP 服务启动，亦可集成至团队内部文档系统或作为个人开发环境的书签管理替代方案。

## 功能概览

**频道化资源分类** 所有外链按技术主题划分为独立频道，每个频道对应一个二维标识码，便于用户按需聚焦，避免信息过载。

**轻量级本地检索** 项目内置基于关键词的客户端过滤功能，用户可在当前频道内快速筛选包含特定术语的资源条目。

**资源状态标记** 每条链接均附带最近检查日期与响应状态标识，帮助用户识别可能失效或迁移的旧有资源。

**社区提名入口** 用户可通过提交 Issue 的方式推荐新资源，经维护团队审核后纳入后续批次更新。

**版本化快照机制** 每批次资源收录均以 Git Tag 形式固化，确保历史引用链路的可回溯性。

**跨域代理中转** 对于部分因网络策略导致访问受限的境外资源，项目提供可选的代理地址替换规则，用户可按需启用。

**自定义标签扩展** 允许用户在本地配置文件中为任意链接添加私有标签，便于个人化分类与优先级排序。

**导出与分享** 支持将当前筛选结果导出为 Markdown 或 CSV 格式，方便团队内部共享或集成到其他文档工具中。

## 应用场景

**技术选型调研** 当团队需要评估 Vue 3 组合式 API 与 Vue 2 选项式 API 的长期维护成本时，可通过本项目相关频道快速获取官方文档、迁移指南及社区最佳实践案例，缩短调研周期。

**日常开发问题排查** 开发者遇到特定报错信息（如响应式数据更新不触发视图渲染）时，可在对应频道中查找社区讨论帖或已归档的 GitHub Issue 链接，借鉴他人解决方案。

**新人上手培训** 团队新成员可通过本项目收录的入门教程系列、项目脚手架模板及视频讲解资源，系统性地建立 Vue 技术认知，无需自行搜索筛选。

**架构设计参考** 在规划中大型应用的状态管理方案（Pinia vs Vuex）或路由设计时，可借助项目收录的架构分析文章与性能测试报告，辅助决策。

**离线文档镜像准备** 在无法直接访问外网的内网开发环境中，运维人员可依据本项目资源列表批量下载文档静态站点，构建内部镜像源。

## 快速开始

以下命令演示如何将本项目克隆至本地，安装基础依赖并启动开发预览服务。

```bash
# 克隆项目仓库
git clone https://github.com/vuebest/navigator.git

# 进入项目目录
cd navigator

# 安装依赖（使用 npm 或 yarn）
npm install

# 启动本地开发服务器，默认监听 3000 端口
npm run serve
```

启动成功后，在浏览器中访问 http://localhost:3000 即可查看资源导航首页。如需构建生产环境静态文件，可执行 `npm run build`，输出目录为 `dist/`。

## 贡献指南

我们欢迎社区开发者以多种形式参与本项目的维护与改进。所有贡献均需遵守行为准则与资源收录标准。

**提交新资源推荐** 通过 GitHub Issue 模板提交资源推荐，需包含资源标题、原始 URL、所属频道建议及 50 字以内的摘要说明。维护团队将在 5 个工作日内评审并反馈。

**修复链接失效报告** 若发现已收录链接返回 404 或超时，请提交 Issue 并标注 `[broken-link]` 前缀，同时提供替代链接（如有）。经确认后将在下一批次更新中替换或移除。

**改进项目文档** 对 README、用户指南或架构文档的修正性 PR 将被优先合并。需确保文档风格保持一致，使用中文撰写，技术术语首次出现时附英文原文。

**开发新功能组件** 若希望为导航页面增加新交互功能（如批量导出、标签云可视化等），请先通过 Issue 讨论设计思路，避免重复劳动。实现时需提供单元测试与使用示例。

**参与批次审核** 在每次新批次资源导入前，社区成员可参与为期 7 天的公开审核期，对候选链接的质量、安全性与相关性进行投票。审核通过后由核心维护者执行合并。

## 常见问题

**Q：本项目与直接收藏浏览器书签有何区别？**

A：浏览器书签通常缺乏分类维度与可检索性，且难以多人协作维护。本项目提供结构化的频道划分、关键词过滤、状态标记以及版本化快照，便于团队共享与历史回溯。同时，项目数据以 JSON 格式存储，可被其他工具或脚本程序化读取，实现自动化工作流集成。

**Q：链接访问时出现超时或证书警告怎么办？**

A：部分境外资源可能受网络环境影响。我们建议优先尝试更换 DNS 或启用 VPN。项目内置的代理中转规则允许用户在 `src/utils/proxyRules.js` 中配置镜像地址替换逻辑。若链接持续不可用，请按贡献指南中的流程提交失效报告，我们将在后续批次中处理。

**Q：如何将本项目部署到内网环境供团队使用？**

A：执行 `npm run build` 生成静态文件后，将 `dist/` 目录下的所有文件复制到内网 Web 服务器（如 nginx）的根目录即可。由于项目本身不依赖后端 API，所有数据均内嵌于前端 JavaScript 文件中，部署后完全离线可用。如需定期同步外网最新资源，可通过脚本定时拉取更新并重新构建。
