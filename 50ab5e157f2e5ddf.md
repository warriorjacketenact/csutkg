# ResourceBridge

ResourceBridge 是一个面向技术团队与个人开发者的外链资源聚合与规范化管理平台。该项目并非传统的爬虫或书签工具，而是一套用于对海量外部链接进行结构化录入、标签化分类、状态监控与团队共享的轻量级数据治理方案。其核心定位在于帮助运维、研发与知识管理角色，将分散于文档、即时通讯或邮件中的零散参考链接，转化为可检索、可审计、可生命周期跟踪的内部知识资产。

ResourceBridge 专为存在以下痛点的组织设计：技术文档中硬编码的第三方依赖地址频繁失效、团队新人无法快速获取经过验证的参考材料、以及跨项目复用外部资源时缺乏统一的引用台账。通过本平台，用户能够以极低的维护成本建立一套“链接即数据”的元信息库，并基于标准化输出接口与现有监控或自动化流水线集成。

## 功能概览

- **链接入库与元数据抽取** 系统提供标准化表单与批量文本导入入口，支持对每个外链记录标题、来源域名、录入人、所属项目标签及生效状态。入库时自动执行基础可达性检测，标记初始响应码与响应时间。

- **多维度标签与分级分类** 允许管理员自定义标签命名空间，例如按技术栈、按环境类型或按文档层级进行划分。每个链接可附带多个标签，并支持按标签组合进行快速筛选与视图保存。

- **状态巡检与异常告警** 内置轻量级巡检调度器，可按照每日或每周频率对已入库链接执行 HEAD/GET 探活。当链接返回非 2xx 状态、超时或证书异常时，系统生成变更记录并可通过 Webhook 向外部通知渠道发送警报。

- **变更审计与版本回溯** 所有链接的字段修改、状态变更、标签增删均记录操作人及时间戳。管理员可查看任意链接的完整历史快照，并支持回退至特定历史版本。

- **结构化数据导出** 支持将链接库按当前筛选条件导出为 JSON、CSV 或 Markdown 表格格式，便于纳入技术文档、知识库周报或外部合规审查材料。

- **权限与团队隔离** 引入项目组概念，不同团队仅可读写其所属命名空间下的链接数据。支持只读观察员与读写管理员两种角色，满足多团队共用一个平台实例的治理需求。

- **搜索与高级过滤** 基于标题、URL 片段、标签、录入时间段及状态码组合进行全文检索与条件过滤，检索结果支持按相关度或最后巡检时间排序。

## 应用场景

- **技术文档中心外链治理** 当技术团队维护一套包含数百篇文档的静态站点时，文档中引用的外部 SDK 下载地址、API 参考文档或社区解决方案链接容易随外部站点改版而失效。ResourceBridge 可作为文档站点的后端链接注册表，通过定时巡检提前发现断链并批量更新引用映射，减少文档维护的人工检查成本。

- **合规审计与供应链溯源** 企业进行开源软件合规审查时，需提供所使用组件官网、源码仓库及许可证文本的原始链接清单。ResourceBridge 允许安全合规团队按季度导入依赖清单链接，统一标记审核状态，并在年度审计时一键生成包含链接响应状态及最后核查时间的报告。

- **团队新人 onboarding 知识包** 新人入职时需要熟悉内部约定的常用工具、镜像源、代码规范参考及日志查询入口。团队负责人可将这批链接预先录入为“新手推荐”标签组，新人登录平台后直接通过标签视图获取全套可信入口，避免从零散聊天记录中拼凑信息。

- **监控告警规则参考源管理** 运维团队在配置 Prometheus 或 Zabbix 告警规则时，常需要参考外部社区提供的最佳实践表达式或故障排查案例。通过 ResourceBridge 将这类参考链接集中存放并标记适用环境，当规则出现偏差时可快速回溯原讨论帖或官方文档，减少故障处理时的检索耗时。

## 快速开始

以下步骤演示如何在本地环境中启动 ResourceBridge 服务端并完成初始链接导入。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resource-bridge/resource-bridge.git

# 进入项目根目录并安装依赖（使用 pip 与 requirements.txt）
cd resource-bridge
pip install -r requirements.txt

# 初始化 SQLite 数据库表结构并创建管理员账户
python manage.py initdb
python manage.py createadmin --username admin --email admin@example.com

# 以开发模式启动服务，默认监听 8000 端口
python manage.py runserver --host 0.0.0.0 --port 8000
```

启动后访问 http://localhost:8000 使用管理员凭证登录，通过界面左侧导航栏的“链接管理”进入批量录入页面，即可开始添加外链记录。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 运行时核心解释器，低于此版本将无法解析类型注解与异步语法 |
| SQLite | 3.35 及以上 | 默认嵌入式元数据库，用于存储链接信息与审计日志；生产环境可切换为 PostgreSQL |
| pip | 22.0 及以上 | 依赖包管理工具，用于安装 requirements.txt 中声明的第三方库 |
| requests | 2.28.0 | 用于执行外链可达性探测与响应码收集 |
| apscheduler | 3.10.0 | 提供后台定时巡检调度能力，支持 cron 风格表达式配置 |
| Flask | 2.2.0 及以上 | Web 服务框架，处理 HTTP 请求与模板渲染 |
| flask-login | 0.6.2 | 用户会话管理与登录态保持 |
| flask-wtf | 1.1.0 | 表单生成与 CSRF 防护，用于链接录入及编辑界面 |
| pytest | 7.4.0 | 仅开发与测试环境需要，用于执行单元测试与集成测试用例 |

## 常见问题

Q: 巡检功能是否会对目标网站造成过大请求压力？

A: 系统默认巡检并发数限制为 5 个并行连接，且每个目标链接在单次巡检周期内仅发起一次 HEAD 请求，优先复用已建立的连接。对于明确不支持 HEAD 方法的端点，系统会自动降级为 GET 请求并仅读取响应头后断开，不下载响应体。默认调度间隔为每日凌晨 2:00 至 4:00 的低峰时段执行，尽可能降低对第三方站点的影响。

Q: 如何将已有的大量书签或浏览器收藏夹批量迁移至 ResourceBridge？

A: 项目提供的 scripts/import_links.py 工具支持解析 Netscape 格式的 HTML 书签导出文件，同时也接受包含 URL 与标题两列的 CSV 文件。执行时可通过 --tags 参数为这批导入链接统一附加标签，后续再通过界面按标签进行细分编辑。对于更复杂的字段映射需求，可参考 API 参考文档中的批量创建接口进行定制化导入。

Q: 生产环境下如何确保巡检调度任务在多进程或多节点部署时不重复执行？

A: 项目不依赖本地文件锁或分布式锁，建议在生产部署中关闭内置调度器，转而使用外部独立调度系统如 Jenkins 或 Kubernetes CronJob 定时调用管理命令 python manage.py probe --once 执行单次巡检。这种方式下调度任务由外部系统保证单点触发，应用层无需关心并发冲突。
