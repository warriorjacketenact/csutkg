# ResourceBridge

ResourceBridge 是一个面向技术团队与个人开发者的外链资源归集与导航系统。该项目不对资源内容进行二次存储或转发，而是通过结构化的索引机制，将分散于互联网各处的优质技术文档、系列教程、参考手册与知识库条目进行统一编目与分类呈现，帮助用户在特定技术领域内快速定位所需信息。

ResourceBridge 定位为“技术资源的入口工具”，适用于需要频繁查阅外部参考资料的技术阅读场景。项目本身不依赖数据库或后端服务，所有资源索引基于静态配置与约定式路由生成，具备轻量、可移植、易于自托管的特点。用户可以通过该项目建立私有的技术外链知识库，亦可将其作为团队内部文档体系的补充导航层。

## 功能概览

**结构化资源编目** 支持对导入的外链资源按系列标识符、内容主题、技术栈标签等多维度进行自动归类，生成清晰的层级索引。

**静态站点生成** 基于配置的 URL 列表与元数据，构建轻量级静态导航页面，无需运行时数据库或后端服务即可独立部署。

**系列聚合视图** 自动识别同一来源站点下属于同一系列（series）的资源链接，以连续阅读流的形式呈现，降低用户在不同页面间的跳转成本。

**标签过滤与检索** 为每个资源条目附加技术标签与阶段标签（如入门、进阶、实战），支持按标签组合进行快速筛选。

**可定制的元数据扩展** 允许用户为每条外链补充自定义描述、阅读时长预估、难度评级与关联笔记，扩展信息存储于本地配置文件中。

**导入导出兼容性** 支持批量导入 URL 列表（文本或 CSV 格式），并支持将当前索引配置导出为标准格式，便于在不同部署实例间迁移。

## 应用场景

**技术团队内部知识导航** 技术团队可将日常研发中高频参考的外部文档、系列博客、官方手册通过 ResourceBridge 统一归集，形成团队共享的技术资源入口，减少成员在重复检索上的时间消耗。

**个人开发者的阅读队列管理** 个人开发者可利用 ResourceBridge 维护自己的技术阅读清单，将待深入学习的系列教程按主题分类，并通过标签标记学习进度与优先级。

**开源项目文档的补充索引层** 开源项目维护者可将项目依赖的第三方库文档、部署环境参考、社区优秀实践文章等外链资源通过 ResourceBridge 组织为独立的导航站点，作为项目主文档的扩展附录。

**技术培训与教学辅助** 培训讲师或教育内容提供者可将课程中涉及的所有课外阅读材料、实验指导手册、环境配置参考文档通过 ResourceBridge 聚合，为学员提供统一且有序的课外资源访问路径。

## 快速开始

以下步骤将指导您在本地环境中完成 ResourceBridge 的克隆、依赖安装与基础运行。

```bash
# 克隆代码仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装项目依赖（使用 npm）
npm install

# 执行构建流程，生成静态导航页面
npm run build

# 启动本地开发服务器
npm run serve
```

执行完成后，通过浏览器访问控制台输出的本地地址（默认为 http://127.0.0.1:8080 ）即可查看导航首页。如需自定义资源列表，请编辑 `config/sources.json` 文件并按格式添加外链条目。

## 常见问题

**问：ResourceBridge 是否会对引用的外链资源进行内容备份或缓存？**

答：不会。ResourceBridge 仅存储 URL 地址及其元数据描述，所有资源内容均通过原始链接直接访问。项目不涉及任何形式的内容抓取、代理转发或本地缓存，用户访问资源时始终跳转至原始来源站点。

**问：我是否可以自定义导航页面的主题样式？**

答：可以。所有样式定义位于 `public/assets/css/main.css`，您可以直接修改该文件以覆盖颜色、字体、间距等视觉属性。如需更深度的布局定制，可调整 `src/templates/layout.hbs` 中的 HTML 结构，并在构建前执行 `npm run build` 重新生成页面。

**问：如何将现有的书签或收藏夹批量导入 ResourceBridge？**

答：您可以使用项目提供的导入脚本 `scripts/import.js`，该脚本支持解析 CSV 格式的输入文件（列标题需匹配 id、url、title、series、tags）。具体用法请参考 `docs/configuration.md` 中关于导入导出功能的章节说明。
