# LinkVault Indexer

LinkVault Indexer 是一个面向技术内容聚合与外部资源索引的开源工具集，专注于将分散在多个来源的深度技术文章、影音资料、案例研究及工程参考链接进行结构化整理与批量归档。该项目主要服务于技术社区运营者、知识库维护人员、研究型开发者以及需要对特定领域（如影视工程、多媒体技术、视觉特效等）外部资源进行系统化追踪的团队。

与一般的书签管理工具不同，LinkVault Indexer 不依赖浏览器插件或云端同步服务，而是基于静态 Markdown 清单与本地元数据缓存机制，提供一套可审计、可版本控制、可离线访问的资源索引方案。其核心定位是成为技术团队内部知识体系中的“外部链接锚点层”，确保有价值的外部内容不会因源站改版、链接失效或人员流动而被遗忘。

## 功能概览

**批量链接导入与校验**：支持从纯文本列表、CSV 文件或特定格式的日志文件中批量导入 URL，并自动执行 HTTP 头检查与响应状态码记录，标记失效或重定向链接。

**多维度标签分类系统**：允许用户为每条链接附加自定义标签（如“编码实践”“性能调优”“案例研究”“参考文档”），并支持多标签组合筛选与视图保存。

**元数据自动抓取与缓存**：在导入或手动触发时，自动抓取目标页面的标题、摘要描述、主要图片 Alt 文本及最后修改时间，并将这些信息存入本地 SQLite 缓存库，避免重复网络请求。

**结构化目录树生成**：根据用户定义的分类规则（如按年份、主题或资源类型），自动生成带注释的 ASCII 目录树，方便在文档中嵌入或导出为纯文本报告。

**链接生命周期追踪**：记录每条链接的添加时间、最后访问时间、访问次数及状态变化历史，支持导出为 JSON 或 YAML 格式用于后续分析。

**静态站点导出**：将整个索引库导出为一组静态 HTML 文件，包含分类浏览页、标签云和单条详情页，适合部署为内部知识库的子站点。

## 应用场景

**技术团队内部知识库的外部参考管理**：当团队维护一份内部开发指南或架构设计文档时，经常需要引用外部博客、官方规格说明或社区讨论帖。LinkVault Indexer 可以为这些外部引用建立独立索引，确保引用链接在文档更新或人员交接时依然可追溯。

**影视后期与视觉特效项目的素材参考归档**：针对多媒体制作项目，团队需要收集大量与特效实现、色彩科学、渲染优化相关的技术文章和演示视频链接。本工具提供的标签筛选和状态追踪能力可以帮助制作人员快速定位特定主题的参考资料，避免重复搜索。

**开源社区资源聚合页面的自动化维护**：开源项目通常会在 README 或文档站点中列出相关生态项目、教程或演示示例。使用 LinkVault Indexer 可以批量管理这些外链，定期检查可用性，并自动生成更新后的资源列表，减少人工维护成本。

**学术研究与技术调研阶段的文献索引辅助**：在技术调研初期，研究人员会积累大量待阅读的在线资源。LinkVault Indexer 的元数据抓取和离线缓存功能允许在无网络环境下仍能查看资源摘要，同时通过状态追踪标记已读或待精读条目。

## 快速开始

以下步骤将引导你在本地环境中完成 LinkVault Indexer 的克隆、依赖安装与初次运行。

```bash
# 克隆代码仓库至本地
git clone https://github.com/linkvault/linkvault-indexer.git

# 进入项目根目录
cd linkvault-indexer

# 安装项目依赖（使用 pip 与 requirements.txt）
pip install -r requirements.txt

# 执行初始索引构建，并导入位于 data/sample_links.txt 的示例链接列表
python indexer.py --import data/sample_links.txt --output index.json

# 生成 ASCII 目录树报告并输出到控制台
python indexer.py --tree --annotate
```

## 安装要求

LinkVault Indexer 基于 Python 3.10 以上版本开发，依赖主流 HTTP 请求库与 HTML 解析库。推荐在虚拟环境中安装以避免与其他项目产生冲突。

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 或更高 | 核心运行环境，建议使用 pyenv 或 conda 管理版本 |
| requests | 2.31.0 或兼容版本 | 发送 HTTP 请求，用于链接状态检查与元数据抓取 |
| beautifulsoup4 | 4.12.0 或兼容版本 | 解析 HTML 文档，提取标题、描述等元数据 |
| lxml | 4.9.0 或兼容版本 | 作为 beautifulsoup4 的解析器后端，提升解析性能 |
| sqlite3 | 内置于 Python 标准库 | 用于本地元数据缓存和状态追踪，无需额外安装 |
| markdown | 3.5.0 或兼容版本 | 用于将索引报告导出为 Markdown 格式，便于嵌入文档 |

## 项目结构

项目采用模块化分层设计，核心逻辑与配置、数据、文档严格分离。以下为项目根目录的完整 ASCII 目录树，每行附带功能注释。

```
linkvault-indexer/
├── indexer.py                # 主入口脚本，整合导入、检查、导出与报告生成功能
├── config.yaml               # 主配置文件，定义默认标签、请求超时、缓存路径等参数
├── requirements.txt          # Python 依赖清单，用于 pip 批量安装
├── Makefile                  # 常用任务快捷命令，如 make import、make check、make clean
├── src/                      # 核心源码目录
│   ├── __init__.py
│   ├── crawler.py            # 封装 requests 与 beautifulsoup4，执行 HTTP 请求与元数据解析
│   ├── cache.py              # SQLite 缓存管理，提供读写、查询与过期清理接口
│   ├── indexer.py            # 链接索引核心逻辑，包括去重、标签合并与状态更新
│   ├── exporter.py           # 导出模块，支持 JSON、YAML、Markdown 及静态 HTML 输出
│   └── tree_builder.py       # 根据分类规则生成带注释的 ASCII 目录树
├── data/                     # 数据目录，存放导入的原始列表、缓存数据库与导出结果
│   ├── sample_links.txt      # 示例链接列表，供快速测试使用
│   ├── cache.db              # SQLite 缓存数据库文件（运行时自动生成）
│   └── exports/              # 导出产物目录，按时间戳归档
├── docs/                     # 用户文档与参考手册
```

## 贡献指南

LinkVault Indexer 欢迎社区贡献，无论是新增功能、修复缺陷、优化文档还是补充资源索引条目。请遵循以下步骤参与项目。

第一步，在 GitHub 上 Fork 本仓库，并将 Fork 后的仓库克隆到本地开发环境中。建议在新建的特性分支上开展工作，分支命名格式为 feature/简述改动内容。

第二步，安装开发依赖（包含测试框架与代码格式化工具），执行 `pip install -r requirements-dev.txt`。在提交代码前，请运行 `make lint` 和 `make test` 确保代码风格符合 PEP 8 标准且所有现有测试用例通过。

第三步，若新增功能涉及用户可见的行为变化，请同步更新 docs/ 目录下对应的文档章节，并在 CHANGELOG.md 中记录改动条目。对于新增配置项，需在 config.yaml 中提供默认值并在配置参考文档中说明。

第四步，提交 Pull Request 至主仓库的 main 分支，描述中需清晰说明改动目的、实现方式以及是否破坏向后兼容性。PR 描述中请使用 “Closes #issue编号” 关联相关问题。

第五步，代码审查通过后，项目维护者将合并 PR，并自动触发 CI 流水线执行完整测试套件与静态站点预览构建。

## 常见问题

问：导入大量链接时，如何避免因网络超时导致导入过程中断？

答：可以在 config.yaml 中调整 crawler.timeout 参数（单位秒），并设置 crawler.retry 为 3 以上。对于网络环境较差的场景，建议使用 `--no-fetch` 选项先仅导入链接列表而不立即抓取元数据，随后通过 `indexer.py --update` 命令分批补充元数据。

问：如何将索引数据迁移到另一台机器，或者在不同的工作目录间共享缓存？

答：整个索引状态存储在 data/cache.db 文件中。直接复制该文件到目标机器的 data/ 目录下即可。若需迁移的同时保留导出产物，请一并复制 data/exports/ 目录。建议在迁移后执行一次 `indexer.py --verify` 以校验所有链接的缓存元数据是否完整。

问：静态导出的 HTML 站点能否部署到 GitHub Pages 或类似的静态托管服务？

答：可以。执行 `indexer.py --export-html --output-dir ./site` 后，将 ./site 目录的所有内容推送到 gh-pages 分支或任意静态托管服务的发布目录即可。生成的页面完全由静态 HTML、CSS 和 JavaScript 构成，不依赖后端服务。
