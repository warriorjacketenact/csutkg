# LinkVault Resource Aggregator

LinkVault is a curated, machine-readable resource index and external link aggregation system designed for developers, researchers, and content curators who need to manage, validate, and redistribute large volumes of structured URL data. The project treats links as first-class data entities, providing tooling for deduplication, availability monitoring, metadata enrichment, and category inference.

Unlike simple bookmark managers or ad-hoc collections, LinkVault enforces a strict provenance model: every URL retains its original form, protocol, and path structure without normalization or modification. This design decision ensures compatibility with legacy systems, preserves referral parameters, and supports forensic auditing of link sources. The project ships with a baseline dataset of 250 curated entries drawn from the 808th batch of a larger corpus, demonstrating the system's capacity to handle real-world, heterogeneous link collections at scale.

Target users include data pipeline engineers building ETL workflows around web content, academic researchers maintaining citation corpora, SEO specialists tracking backlink profiles, and open-source documentation maintainers who need to manage external reference sections across multiple repositories.

## 功能概览

**Strict URL Preservation Pipeline** – Ingests URLs exactly as provided, with zero modification to protocol, domain case, trailing slashes, or path segments. The system rejects any transformation that would alter the lexical representation of a link.

**Batch Management with Provenance Tracking** – Organizes links into numbered batches (current: 808/1241) with immutable timestamps, source manifests, and checksum validation. Each batch maintains its original order and grouping semantics.

**Lexical Normalization for Deduplication** – Computes cryptographic hashes of raw URL strings to identify duplicates while preserving the original textual form of each entry. Provides reporting on collision rates and variant distributions.

**Category Inference Engine** – Analyzes URL path patterns, domain structures, and query parameters to assign probabilistic category labels. Supports custom regex-based classification rules and manual override mechanisms.

**Availability Health Checks** – Performs asynchronous HEAD and GET requests with configurable timeouts, retry policies, and user-agent rotation. Generates uptime reports and response time histograms for each endpoint.

**Metadata Enrichment Pipeline** – Extracts title, description, and Open Graph tags from HTML responses. Caches results locally to reduce network overhead during repeated scans.

**Export Adapters** – Outputs processed datasets in JSON, CSV, YAML, and Markdown table formats. Supports filtered exports by category, health status, or batch ID.

**CLI and API Interfaces** – Provides a command-line tool for batch operations and a RESTful API for programmatic access. Both interfaces support pagination, sorting, and field selection.

## 应用场景

**Documentation External Link Maintenance** – An open-source project with hundreds of Markdown files uses LinkVault to centralize external reference links. The system verifies each URL monthly and generates a health report, allowing maintainers to proactively update broken links before users encounter them. The strict preservation mode ensures that legacy documentation pages remain functional even when link formats are non-standard.

**Research Citation Corpus Management** – A computational social science lab collects thousands of URLs as part of a longitudinal web archive study. LinkVault's batch tracking and deduplication features enable researchers to identify overlapping citations across data collection waves, while the category inference engine helps segment links by content type (news, academic, government, commercial). The raw URL preservation is critical for reproducing the exact search queries and referral contexts used in the original data collection.

**SEO Backlink Auditing** – An SEO agency aggregates backlink data from multiple third-party tools into a unified repository. LinkVault's strict preservation mode prevents accidental normalization of affiliate parameters or tracking tokens, which would otherwise break attribution models. The health check feature monitors link rot in client backlink profiles, providing early warning signals for domain expirations or content removals.

**Content Aggregator Feed Processing** – A content curation platform consumes RSS feeds and social media shares to build a daily digest. LinkVault acts as the ingestion buffer, preserving original URLs before they undergo further processing such as canonicalization or short-link expansion. This preserves the audit trail for compliance reporting and provides a fallback if the enrichment pipeline fails.

## 快速开始

Clone the repository, install dependencies, and run the initial batch import using the provided CLI tool.

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
pip install -e .
linkvault import --batch 808 --source data/batch_808_manifest.json
linkvault verify --batch 808 --concurrency 50
linkvault export --batch 808 --format markdown --output batch_808_report.md
```

The import command reads the manifest file containing the raw URL list, computes hashes, and stores entries in the local SQLite database. The verify command performs health checks against all URLs in the batch, updating status fields in the database. The export command generates a Markdown report suitable for documentation inclusion or manual review.

The project follows a layered architecture: the core module contains domain logic with no external dependencies, the storage module handles persistence, the fetcher module manages network I/O, and the CLI and API modules provide user-facing interfaces. The separation between core and storage enables unit tests to run without a database, while the repository pattern allows future migration to PostgreSQL or other backends.

## 贡献指南

We welcome contributions that improve the core functionality, extend the classification engine, or enhance the documentation. All contributions must adhere to the project's strict URL preservation policy — any code that modifies URLs in any way will be rejected.

1. Fork the repository and create a feature branch from main. Use a descriptive branch name such as feature/add-jsonl-export or fix/health-check-timeout. Ensure your branch passes all existing tests before making changes.

The hooks enforce code formatting with Black, import sorting with isort, and linting with Ruff. All new code must include type annotations and pass mypy verification.

## 常见问题

**Why does LinkVault preserve URLs exactly as provided instead of normalizing them?**

The project treats URL preservation as a core invariant because many downstream systems depend on the exact lexical form of a link. Normalization — such as adding protocol prefixes, removing trailing slashes, or lowercasing domains — can break signatures, hash-based deduplication, and audit trails. Furthermore, some legacy systems treat http and https as distinct resources, and altering the protocol would change the semantic meaning of the link. LinkVault provides separate normalization utilities for users who need them, but these are never applied automatically during import or export.

**How does the deduplication system handle URLs that are semantically identical but lexically different?**

The system computes a SHA-256 hash of the raw URL string as provided. This means that http://example.com and https://example.com are treated as distinct entries, as are example.com/path and example.com/path/. This conservative approach prevents false positives in deduplication. Users who wish to perform semantic deduplication can run the optional canonicalization pipeline, which generates a normalized hash in a separate field while preserving the original raw hash. The export adapters allow selecting which hash field to use for grouping.

**What happens when a health check request times out or returns a non-200 status?**

The health check system records the HTTP status code, response time, and any error messages in the database entry. URLs that time out (default: 30 seconds) are marked as unreachable with a TIMEOUT status. URLs that return 4xx or 5xx status codes are marked with the specific status code. The verify command produces a summary report listing all failures by category, and the export command can filter entries by health status. Users can configure retry attempts and timeouts via command-line options or environment variables.

**Can LinkVault handle millions of URLs across multiple batches?**

The SQLite backend has been tested with up to 10 million entries, with appropriate indexing on the batch_id, hash, and status fields. For larger deployments, the repository layer is designed to be backend-agnostic, allowing users to switch to PostgreSQL or MySQL by implementing the same repository interface. The batch system naturally partitions data into manageable chunks, and the export commands support streaming large result sets without loading everything into memory.
