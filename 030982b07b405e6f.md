# ResourceBridge

ResourceBridge 是一个面向技术团队与内容创作者的轻量级外部资源索引与导航系统。项目定位为技术资源外链汇总站，用于统一管理、分类展示和快速检索分散在各类第三方平台上的优质视频、文档、工具与教程链接。目标用户包括运维工程师、前端/后端开发人员、技术内容运营者以及企业内部知识库管理者。

该项目解决的核心问题是：技术团队在日常工作中积累了大量的外部资源链接（如视频教程、API 文档、案例演示、工具站点），这些链接分散在聊天记录、邮件、云笔记或浏览器书签中，难以共享、检索和更新。ResourceBridge 通过结构化的资源清单与分类导航，将外链资源转化为可维护、可追溯、可协作的知识资产。

## 功能概览

- **资源清单统一管理**：以 Markdown 或 YAML 格式维护全部外链资源，支持批量导入、分类标签与备注说明。
- **多维度分类导航**：按资源类型（视频、文档、工具、案例）、业务领域（前端、后端、运维、数据）、适用场景（新手入门、进阶学习、故障排查）进行分类筛选。
- **快速检索与过滤**：内置模糊搜索与标签过滤机制，支持按标题关键词、来源域名、更新日期等条件快速定位资源。
- **详情页自动生成**：基于资源元数据（标题、链接、描述、标签）自动渲染详情视图，便于团队内部预览和分享。
- **资源状态监控**：周期性检测外链可用性，对失效链接进行标记和告警，避免团队访问无效资源。
- **访问统计与热度排序**：记录各资源的点击次数和引用频率，支持按热度或更新时间排序，帮助团队发现常用资源。
- **导入导出与备份**：支持 JSON/CSV 格式的资源数据导入导出，便于与其他知识管理工具集成，并提供定期自动备份机制。

## 应用场景

1. **技术团队内部知识库建设**：团队 Leader 或技术文档负责人可使用 ResourceBridge 汇总项目中用到的所有外部依赖文档、视频教程和调试工具链接，形成统一入口，降低新成员上手时的信息搜寻成本。

2. **开源项目 README 外链整理**：开源项目维护者将项目相关的生态资源（如社区视频、第三方集成示例、扩展阅读）集中托管于 ResourceBridge 生成的外链清单中，避免 README 文件过长且难以维护。

3. **技术培训与课程配套资源**：培训机构或企业内训讲师创建课程资源导航页，将每节课涉及的扩展视频、案例代码仓库和参考文献统一列出，学员可通过分类快速访问对应章节资料。

4. **技术社区内容聚合**：技术博客作者或社区运营者利用 ResourceBridge 按季度汇总社区产出的优质视频与文章链接，形成可公开访问的资源月刊，提升社区内容复用率。

5. **个人学习路径管理**：开发者可将个人学习过程中收藏的技术视频、官方文档和实战项目链接按学习阶段（基础、进阶、实战）分类管理，配合状态监控及时清理失效链接。

## 快速开始

以下步骤帮助您在本地环境快速启动 ResourceBridge 服务。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git

# 进入项目目录
cd resourcebridge

# 安装依赖（基于 Node.js 22 LTS 与 npm）
npm install

# 复制示例配置文件并修改
cp .env.example .env

# 初始化资源数据（从示例数据生成）
npm run init-db

# 启动开发服务器（默认监听 3000 端口）
npm run dev
```

启动后，访问 http://localhost:3000 即可查看资源清单首页。若需生产环境构建，执行 `npm run build` 后使用 `npm start` 启动。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 22.x LTS 或更高 | 运行时环境，推荐使用官方二进制或 nvm 安装 |
| npm | 10.x 或更高 | 包管理器，随 Node.js 一起安装 |
| SQLite | 3.40 或更高（内置） | 默认轻量级数据库，无需额外安装，适用于小型部署 |
| PostgreSQL | 15.x 或更高（可选） | 生产环境推荐使用，需单独安装并配置连接字符串 |
| Redis | 7.0 或更高（可选） | 缓存与会话存储，非必须但建议在高并发场景启用 |
| Git | 2.40 或更高 | 用于克隆仓库和版本管理 |

## 常见问题

**Q：项目支持哪些数据库？如何切换？**

A：默认使用 SQLite（适合开发和小规模部署），生产环境推荐使用 PostgreSQL。切换方式为在 `.env` 文件中设置 `DB_TYPE=postgres` 并配置 `DATABASE_URL` 连接字符串，同时调整 `DB_POOL_SIZE` 等参数。系统启动时会根据配置自动选择对应的数据库驱动。

**Q：链接可用性检测是如何实现的？对目标服务器是否有影响？**

A：检测模块使用 HTTP HEAD 请求并设置 3 秒超时，仅检测状态码（200/301/302/404 等），不会下载页面内容。检测频率默认每日一次（可配置），请求头携带 `User-Agent: ResourceBridge-HealthCheck` 以便目标服务器识别。对于频繁检测导致的限流问题，可调整 `CHECK_INTERVAL` 环境变量延长检测周期。

**Q：如何从旧版资源清单迁移到 ResourceBridge？**

A：项目提供了数据迁移脚本 `scripts/import-csv.js`，支持从 CSV 或 JSON 格式导入资源数据。标准字段映射为 `title`（标题）、`url`（链接）、`description`（描述）、`category`（分类）、`tags`（标签，逗号分隔）。若您有自定义字段需求，可修改映射配置文件 `config/import-mapping.yaml`。迁移前建议先在测试环境运行导入预览（`--dry-run` 参数）。
