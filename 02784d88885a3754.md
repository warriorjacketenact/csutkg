# LinkVault Resource Aggregator

LinkVault 是一个面向开发者、技术研究人员与内容策展人的轻量级外链资源汇总与导航系统。该项目定位于将分散于网络各处的优质技术文档、工具站、参考手册与开源组件入口，通过结构化索引与可检索元数据，聚合并呈现在统一的本地可部署门户中。LinkVault 并非搜索引擎，亦非社交媒体书签管理器，而是一个以长期可维护性、可审计性与快速访问为核心设计目标的静态资源索引层。

目标用户包括：需要维护团队技术选型参考库的架构师、希望系统化积累学习资料的后端与前端工程师、从事技术写作与教程聚合的内容创作者，以及任何希望以最低心智负担管理大量外链资源的效率型用户。LinkVault 通过标准化字段、标签体系与版本化数据文件，解决传统浏览器书签缺乏结构化、难以共享、易丢失与无法自动化检查存活状态等痛点。

## 功能概览

**批量导入与去重**：支持从 CSV、JSON 及 Markdown 列表批量导入 URL，自动识别并合并重复条目，保留首次添加时间与最高优先级标签。

**存活状态巡检**：内置基于 HTTP HEAD 方法的异步链接存活检查，可配置超时与重试策略，输出可用性报表并标记失效链接。

**全文检索与过滤**：基于倒排索引的标题、描述与标签全文搜索，同时支持按域名、资源类型（文档/工具/社区/仓库）、添加时间范围等多维度过滤。

**分级标签体系**：提供三级标签命名空间（领域/子领域/特性），支持自定义标签别名与标签组合查询，便于构建细粒度分类视图。

**静态站点生成**：内置模板引擎，可将索引数据渲染为完全静态的 HTML 导航页面，适配 GitHub Pages、Nginx 或任何静态托管服务。

**数据快照与回滚**：每次修改自动保存数据快照至 `.snapshots/` 目录，支持按时间戳回滚至任意历史版本，防止误删或误覆盖。

**开放 API 端点**：提供只读 RESTful API，支持 JSON 格式的资源列表查询、单条目详情获取与标签树导出，便于与其他自动化工具集成。

## 应用场景

个人技术知识库构建：开发者可将日常阅读的博客、官方文档、API 参考与视频教程统一收录至 LinkVault，通过标签体系按语言、框架、难度等级组织，形成个人专属的技术地图。每周巡检功能自动提醒失效链接，保持知识库鲜活。

团队共享资源池维护：技术团队可将内部常用依赖库镜像地址、运维手册、设计规范文档与云服务控制台入口集中管理，通过静态站点生成功能发布到内网门户，确保新成员入职即可获得一致的资源访问起点。

技术教程与课程配套索引：培训讲师或在线课程作者可为每期课程生成专属资源列表，随课程内容动态更新，学员通过单一入口即可获取所有延伸阅读材料与实验环境地址，避免在多个平台间反复跳转。

开源项目文档聚合：开源项目维护者可将相关生态工具、插件列表、社区论坛与贡献指南的链接统一纳入 LinkVault，作为项目 README 或官网的补充资源层，降低贡献者查找上下文信息的成本。

## 快速开始

以下指令适用于 Linux/macOS 及 Windows WSL 环境，确保已安装 Git 与 Node.js 18.x 及以上版本。

```bash
# 克隆仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖
npm install

# 以开发模式运行（默认监听 http://localhost:5173）
npm run dev
```

首次启动后，系统会自动创建 `data/links.json` 与 `data/tags.json` 示例数据文件。访问本地服务地址即可浏览预置的示例资源列表。如需导入用户提供的 URL 列表，请将原始链接按行存入 `data/import.txt` 后执行：

```bash
npm run import
```

生成静态站点产物：

```bash
npm run build
```

产物默认输出至 `dist/` 目录，可直接部署至任意 Web 服务器。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.0.0 或更高 | 运行时环境，用于执行核心服务与构建脚本 |
| npm | 9.0.0 或更高 | 包管理器，用于安装项目依赖与运行脚本命令 |
| Git | 2.30.0 或更高 | 版本控制工具，用于克隆仓库与提交数据变更 |

| 网络连接 | 需能访问目标外链域名 | 存活巡检与资源元数据抓取功能依赖对外部域名的网络可达性 |

## 项目结构

```

├── data/                           # 数据存储目录

│   ├── tags.json                   # 标签定义与层级关系，包含别名与颜色标记
│   └── import.template.json        # 批量导入的格式示例文件
├── src/                            # 核心源代码
│   ├── core/                       # 核心逻辑模块
│   │   ├── indexer.js              # 倒排索引构建与全文检索实现
│   │   ├── checker.js              # 链接存活状态异步巡检引擎
│   │   └── snapshot.js             # 数据快照生成与回滚管理
│   ├── api/                        # HTTP API 层
│   │   ├── routes.js               # 路由定义与请求参数校验
│   │   └── handlers.js             # 各端点的业务逻辑处理函数
│   ├── cli/                        # 命令行工具入口
│   │   ├── import.js               # 导入命令实现
│   │   ├── check.js                # 手动触发巡检命令
│   │   └── build.js                # 静态站点生成命令
│   └── web/                        # 前端静态资源模板
│       ├── index.html              # 列表视图主页面
│       ├── detail.html             # 单条目详情页模板
│       └── assets/                 # CSS 与客户端 JavaScript

├── scripts/                        # 运维辅助脚本

│   └── migrate-v1-to-v2.js         # 数据格式升级迁移工具
├── tests/                          # 单元测试与集成测试
```

## 常见问题

**问：LinkVault 是否支持对私有网络或需要登录认证的链接进行存活巡检？**

答：存活巡检模块默认仅发送标准 HTTP HEAD 请求，不携带任何认证凭据。对于需要 Cookie 或 Token 的内网资源，您可以通过配置 `config/checker.json` 中的 `customHeaders` 字段来添加固定的请求头。但请注意，涉及动态认证（如 OAuth 流程）的链接无法通过本工具自动巡检，建议将此类资源标记为手动确认类型并关闭自动检查。

**问：我拥有超过一万条链接，LinkVault 的性能表现如何？数据文件会不会变得过大？**

答：LinkVault 的数据存储基于纯 JSON 文件，单条链接的平均存储开销约为 300-500 字节。对于一万条链接，总文件大小约为 3-5 MB，Node.js 可流畅处理。全文检索采用内存倒排索引，在 5 万条以内的数据量下，单次搜索响应时间通常低于 50 毫秒。若数据量超出此范围，建议使用 `npm run archive` 命令将历史链接归档至独立文件，保持主索引轻量。

**问：静态站点生成后，部署到 Nginx 时页面刷新出现 404，应该如何解决？**

答：LinkVault 生成的静态站点采用客户端路由模式。部署至 Nginx 时，需要在配置中添加 try_files 规则，将所有页面请求指向 index.html。推荐的配置片段为：`location / { try_files $uri $uri/ /index.html; }`。该配置已在 `docs/operations.md` 中的 Nginx 部署示例中完整提供。
