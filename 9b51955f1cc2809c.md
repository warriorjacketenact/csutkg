# ResourceBridge

ResourceBridge 是一个面向开发者和技术研究人员的开放式外链资源聚合与导航系统。该项目不直接托管任何内容，而是通过结构化方式收录并分类整理分布于互联网各处的优质技术文档、工具链入口、参考手册与社区讨论页，帮助用户在特定技术领域内快速定位所需的外部资源。

项目定位为“技术资源的中转站与索引层”，主要解决以下问题：技术资料分散在多个域名与路径下，记忆成本高；搜索引擎结果被商业内容污染，有效链接难以筛选；团队内部共享资源时缺乏统一的入口与分类标准。ResourceBridge 通过可编辑的索引文件与静态页面生成机制，将零散的外链转化为可维护、可检索、可共享的知识库。

## 功能概览

**外链索引管理**：以纯文本或 YAML 格式维护外部 URL 列表，支持按类别、标签、来源平台进行多维分组，所有链接均保留原始协议与路径，不做自动跳转或重写。

**资源分类导航**：内置三级分类体系（领域/子领域/资源类型），每个外链可归属多个分类，生成静态导航页面时自动聚合为树状结构，便于按主题浏览。

**链接可用性探测**：集成异步 HTTP 头检查模块，定期对已收录链接进行可用性验证，标记失效或重定向链接，并生成报告供维护人员审核。

**自定义元数据扩展**：每条外链允许附加标题、描述、维护人、更新日期、标签列表等自定义字段，所有元数据存储于项目仓库内，支持版本控制与变更追溯。

**静态站点生成**：基于项目根目录下的索引数据，生成纯 HTML 静态资源导航站，无需后端服务，可直接部署至任意 HTTP 服务器或对象存储平台。

**导入导出兼容**：支持从浏览器书签 HTML 文件、Markdown 列表、CSV 表格导入外链数据，也可将当前索引导出为通用格式，便于迁移或与其他工具集成。

## 应用场景

技术团队内部文档中心扩展：团队可在现有内部 Wiki 或文档站点旁部署 ResourceBridge 生成的导航页，将常用的第三方库文档、API 参考、运维手册入口集中管理，减少成员在多个标签页之间频繁切换查找的时间。

个人开发者的知识仓库构建：开发者可将日常阅读的博客、教程视频、在线工具、代码示例库等外链按学习路径分类整理，形成个人专属的技术资源库，并通过 Git 同步至多台设备。

开源项目外部依赖索引：开源项目维护者可在项目仓库中附加 ResourceBridge 索引文件，明确列出项目所依赖的协议文档、构建工具官网、持续集成服务入口等，方便贡献者快速获取环境配置所需的外部信息。

技术社区资源推荐聚合：技术社区或公众号运营者可使用 ResourceBridge 生成每周推荐资源列表，将分散在多个平台的优质内容通过统一入口呈现给读者，同时利用元数据标注推荐理由与适用水平。

## 快速开始

以下命令演示如何在本地环境中获取 ResourceBridge 源码、安装依赖并启动开发服务器。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行完毕后，打开浏览器访问 http://localhost:3000 即可查看生成的导航页面。默认索引文件位于 `data/links.yaml`，您可以直接编辑该文件来增删外链。修改保存后，开发服务器将自动重新生成静态页面并刷新浏览器。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，用于执行构建脚本与开发服务器 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖 |
| Git | 2.40 或更高 | 版本控制工具，用于克隆仓库与提交变更 |
| YAML 解析器 | 项目内置 | 用于解析 `data/` 目录下的索引文件，无需额外安装 |
| HTTP 客户端 | 项目内置 | 用于链接可用性检查，基于 Node.js 原生 `http`/`https` 模块 |
| 静态文件服务器 | 可选 | 生产环境部署时可使用 Nginx、Caddy 或 Apache 托管 `dist/` 目录 |
| 内存 | 512 MB 以上 | 构建大型索引（超过 5000 条链接）时建议 1 GB 以上 |
| 磁盘空间 | 100 MB | 用于存放源码、索引文件及生成的静态页面 |

## 项目结构

```
resourcebridge/
├── bin/                                 # 可执行脚本与命令行入口
│   └── cli.js                           # 主命令行工具，封装构建、校验、导入导出命令
├── src/
│   ├── core/                            # 核心逻辑模块
│   │   ├── indexer.js                   # 索引解析器，读取 YAML 并构建内存数据模型
│   │   ├── validator.js                 # 链接校验器，执行异步 HTTP 检查
│   │   └── generator.js                 # 静态页面生成器，将数据模型渲染为 HTML
│   ├── parsers/                         # 格式解析子模块
│   │   ├── yaml-parser.js               # YAML 格式解析与序列化
│   │   ├── bookmark-importer.js         # 浏览器书签 HTML 导入器
```

## 贡献指南

欢迎社区开发者参与 ResourceBridge 的改进与扩展。请遵循以下步骤提交贡献：

第一，在 GitHub 上 Fork 本仓库至个人账户，并克隆到本地开发环境。确保本地 Node.js 版本符合安装要求中的规定。

第二，创建新的功能分支，分支名称应简要描述所解决的问题或新增的功能，例如 `fix/validator-timeout` 或 `feat/json-importer`。请勿在主分支上直接修改。

第三，完成代码变更后，运行测试套件确保现有功能未被破坏。新增功能应附带对应的单元测试或集成测试，测试文件放置于 `test/` 目录下对应子目录。

第四，提交变更时请遵循语义化提交规范，提交信息格式为 `<类型>: <简短描述>`，类型可选 `feat`、`fix`、`docs`、`style`、`refactor`、`test`、`chore`。提交信息正文可附加详细说明。

第五，推送分支至个人远程仓库，随后在 GitHub 上向本仓库主分支发起 Pull Request。Pull Request 描述中请清晰说明变更目的、实现方式及测试覆盖情况，等待项目维护者审阅。

## 常见问题

**问：ResourceBridge 是否托管或缓存外部链接的内容？**

答：ResourceBridge 不托管、不缓存、不代理任何外部链接的实际内容。项目仅存储 URL 字符串及其元数据（标题、描述、标签等）。所有对外部资源的访问均由用户浏览器直接发起，ResourceBridge 不对第三方内容的可用性、合法性或完整性承担任何责任。

**问：索引文件中的链接数量很大时，构建速度是否会显著下降？**

答：对于链接数量在 5000 条以内的索引，构建时间通常在 3 秒以内。链接可用性校验是异步并发执行的，默认并发数为 20，可通过环境变量 `RB_CHECK_CONCURRENCY` 调整。如果不需要实时校验，可以在构建时跳过校验步骤以加快速度。

**问：如何将现有的浏览器书签批量导入 ResourceBridge？**

答：从浏览器导出书签为 HTML 文件，然后使用命令行工具执行导入命令：`node bin/cli.js import --from bookmarks.html --format html`。导入程序会自动解析书签文件夹结构，将其映射为分类层级，并生成对应的 YAML 条目追加到 `data/links.yaml` 文件中。导入前建议备份原有索引文件。
