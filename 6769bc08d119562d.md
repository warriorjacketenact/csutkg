# CdnResourceIndex

CdnResourceIndex 是一个面向 CDN 服务商、网站运维人员、前端性能优化工程师以及技术研究人员的综合性内容分发网络资源导航与链接索引系统。该项目定位于对特定 CDN 节点下的动态资源路径进行结构化收录、分类管理与快速检索，旨在解决大规模 CDN 部署场景下资源链接分散、路径难以追溯、运维排查效率低下等实际问题。

本项目通过对指定域名下的视频点播（VOD）资源路径进行系统性归集与整理，形成可供人工查阅和自动化工具调用的稳定索引目录。项目本身不存储任何实体媒体文件，仅提供公开可访问的资源定位符的整理与展示功能，适用于需要快速浏览、筛选或批量导出特定 CDN 节点资源列表的工作流。

## 功能概览

**资源路径结构化收录**：按照既定命名规范对 CDN 节点下的 VOD 资源路径进行逐条登记，每条记录对应一个独立的资源定位标识符，保证索引条目与源站资源路径的一一对应关系。

**多维度分类索引**：支持按资源路径前缀、路径长度、字符组合特征等多个维度对收录的链接进行自动分类与分组，便于用户根据不同检索意图快速定位目标资源。

**链接状态标注**：对每条收录的资源链接标注其收录批次、所属项目期次以及全局唯一编号，方便用户在跨文档引用或问题反馈时精确定位特定条目。

**批量导出与复制**：提供一键复制全部链接、按筛选条件批量导出链接列表的功能，支持纯文本和结构化数据两种导出格式，适配不同的下游使用场景。

**项目元数据追踪**：每个收录的资源链接均携带项目批次信息（第 939/1241 批），确保在大规模持续收录过程中每一条目均可追溯其收录时间线与版本归属。

**简洁的命令行交互接口**：项目内置轻量级命令行工具，支持资源列表的快速检索、关键字过滤以及统计信息输出，无需图形界面即可完成日常查询操作。

## 应用场景

**CDN 节点资源巡检与审计**：运维人员可利用本项目的索引列表，定期核对 CDN 节点上实际存在的资源路径与索引记录是否一致，快速发现资源遗漏或异常新增的路径条目，提升节点内容审计的效率与准确性。

**前端性能测试样本构造**：前端工程师在进行资源加载性能测试时，可直接从本项目收录的链接列表中选取不同路径结构、不同长度的资源地址作为测试样本，用于模拟真实环境下的 CDN 资源请求场景，避免手工构造测试数据的繁琐过程。

**技术文档与教程配套资源引用**：技术博主或培训讲师在编写涉及 CDN 配置、资源部署或网络请求优化的教程内容时，可将本项目的索引链接作为示例素材使用，为读者提供真实可访问的参考路径，增强教程内容的实操性。

**自动化数据采集管道配置**：数据工程师在搭建针对特定 CDN 节点的资源采集管道时，可借助本项目提供的结构化链接列表初始化采集任务队列，将收录的路径作为种子 URL 快速构建爬虫起始点，缩短管道配置周期。

## 快速开始

以下步骤指导您在本地环境中快速部署并运行 CdnResourceIndex 项目。

```
git clone https://github.com/cdn-resource-index/cdn-resource-index.git
cd cdn-resource-index
pip install -r requirements.txt
python cli.py list --batch 939
```

上述命令执行完成后，终端将输出第 939 批次收录的全部资源链接列表。您也可以使用 `--output` 参数指定导出文件路径，将结果保存为文本文件以便后续处理。

```
├── cli.py                     # 命令行入口文件，处理用户输入与命令分发

│   └── settings.yaml          # 项目配置文件，包含批次参数、输出格式等选项

│   ├── __init__.py            # core 包初始化文件
│   ├── indexer.py             # 核心索引引擎，负责链接的加载、筛选与导出
│   └── validator.py           # 链接格式校验模块，检查 URL 合法性与规范性
├── data/
│   ├── batch_939.json         # 第 939 批次链接数据的 JSON 存储文件
│   └── manifest.json          # 全量批次清单与元数据索引文件
├── docs/
│   ├── getting-started.md     # 入门指南文档，包含安装与首次使用说明
│   ├── link-management.md     # 链接管理操作手册，涵盖增删改查流程
│   ├── cli-reference.md       # 命令行工具完整参考手册
│   └── contributing.md        # 贡献者指南，说明外部提交规范与流程
├── tests/
│   ├── test_indexer.py        # 索引引擎单元测试用例
│   └── test_validator.py      # 链接校验模块单元测试用例
└── scripts/
    ├── export.sh              # 批量导出辅助脚本，支持多种输出格式
    └── import.sh              # 批量导入辅助脚本，用于加载外部链接数据
```

## 贡献指南

第一，在 GitHub 上 Fork 本项目仓库至您的个人账号，随后将 Fork 后的仓库克隆至本地开发环境中。

第二，在 `data/` 目录下新建或修改对应批次的 JSON 数据文件，按照既定的 JSON Schema 格式添加新的资源链接条目，确保每条链接均包含完整的 URL 路径与必要的元数据字段。

第三，运行项目提供的单元测试套件，执行 `pytest tests/` 命令验证新增或修改的内容未破坏现有功能，确保所有测试用例均通过后方可提交。

第四，编写清晰的提交信息（commit message），说明本次变更的具体内容与目的，然后将变更推送至您的 Fork 仓库，最后通过 GitHub 界面发起 Pull Request 至主仓库的 main 分支。

## 常见问题

问：项目收录的资源链接是否可以保证长期有效？

答：本项目仅提供链接索引服务，不保证任何外部 CDN 节点上资源的持续可用性。收录的链接指向第三方 CDN 服务器，资源的有效性与存活时间由该服务器的运营策略决定。用户在使用前应当自行验证链接的可访问性，并根据实际响应状态决定是否采用。

问：如何提交一批新的资源链接到项目中？

答：请先阅读项目文档中的贡献指南（docs/contributing.md），按照指南要求准备包含新链接的数据文件，通过 Pull Request 提交您的变更。项目维护者会审核新增链接的格式规范性与收录必要性，审核通过后合并至主分支并在下一批次中发布。

问：命令行工具能否输出 JSON 格式的结构化数据以供其他程序使用？

答：可以。cli.py 命令支持 `--format json` 参数，当指定该参数后，所有查询和导出结果将以 JSON 格式输出到标准输出或指定的输出文件中，便于其他自动化工具解析和集成。
