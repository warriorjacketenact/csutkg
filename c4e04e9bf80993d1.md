# LinkWeaver 技术资源导航平台

LinkWeaver 是一个面向开发者、技术研究员与内容创作者的轻量级外链资源聚合与导航工具。项目定位为可自托管的书签管理后端与前端展示层，帮助个人或团队将散落在浏览器、笔记软件、即时通讯中的技术参考链接进行统一归档、分类索引与快速检索。

LinkWeaver 解决的核心问题包括：技术链接分散导致的查找效率低下、收藏夹层级过深难以维护、团队内部共享资源缺乏版本管理与访问控制、以及外部优质系列内容（如教程、专栏、案例分析）缺乏结构化组织方式。项目不依赖第三方云服务，支持 SQLite 与 PostgreSQL 存储，开箱即用，适合部署在个人服务器、内网 NAS 或云主机。

## 功能概览

**批量链接导入**：支持从浏览器书签 HTML 导出文件、纯文本列表及 CSV 格式批量导入链接元数据，自动识别标题与路径信息。

**系列分类体系**：基于 URL 路径前缀（如 /series/）自动聚合为系列专题，支持为每个系列设置独立封面、描述与标签，便于按主题浏览。

**全文检索与过滤**：内置基于 SQLite FTS5 或 PostgreSQL tsvector 的全文搜索，支持按标题、路径、系列名称、添加时间等多维度过滤排序。

**访问状态监控**：定时检测已收录链接的 HTTP 状态码，标记失效链接（404、500、超时等），并生成报表供管理员清理或更新。

**只读公开视图与只写管理后台**：前端展示页面无需登录即可访问，管理后台采用基于 Cookie 的简单密码认证，支持增删改查及批量操作。

**RESTful API 接口**：提供完整的 JSON API 用于第三方客户端集成，支持分页查询、按系列筛选、关键词搜索等操作，便于嵌入其他仪表盘。

**数据导入导出**：支持将全部链接数据导出为 JSON 或 CSV 格式，也支持从外部数据源（如 Airtable 导出、Notion CSV）进行增量合并导入。

**自定义元数据扩展**：允许为每条链接添加键值对形式的自定义字段（如阅读状态、重要等级、关联项目编号），满足个性化管理需求。

## 应用场景

个人技术知识库构建：开发者可将日常阅读的系列技术文章、API 文档、架构设计案例按系列归类，配合全文搜索快速回溯以往研究过的技术方案，减少重复查找时间。

团队内部资源库共享：小型研发团队或开源项目组可将 LinkWeaver 部署在内网，作为统一的技术文档入口，收录第三方依赖库主页、官方示例、社区最佳实践等，新成员入职时可一键获取全部参考链接。

内容策划与选题管理：技术博主或课程设计师可利用系列分类功能将待阅读、待翻译、待引用的外链按选题分组，通过自定义字段标记优先级与进度，形成内容生产管线。

运维监控辅助面板：运维工程师可将常用监控面板、日志查询工具、故障排查手册的链接集中管理，结合状态监控功能定期检查各内部工具的可用性，快速定位故障工具。

教育机构课程资源索引：高校教师或培训讲师可为每门课程创建独立系列，收录课件参考链接、视频地址、在线练习平台，学生通过公开只读页面即可访问全部课程外链资源。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，依赖 Python 3.9 及以上版本与 pip 包管理器。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkweaver.git
cd linkweaver

# 创建并激活 Python 虚拟环境
python3 -m venv venv
source venv/bin/activate      # Windows 请使用 venv\Scripts\activate

# 安装依赖包（生产环境建议使用 --no-cache-dir）
pip install -r requirements.txt

# 初始化数据库（默认使用 SQLite，自动创建表结构与 FTS 索引）
python manage.py init-db

# 导入示例链接数据（可选）
python manage.py import-sample

# 启动开发服务器（默认监听 127.0.0.1:8000）
python manage.py runserver
```

访问 http://127.0.0.1:8000 即可看到公开链接列表页面。管理后台入口为 /admin，默认管理员账号密码在首次启动时由 init-db 命令输出，请及时修改。

生产环境部署建议使用 Gunicorn + Nginx，并将数据库切换为 PostgreSQL，具体配置参考 `deploy/` 目录下的示例文件。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 ~ 3.12 | 核心运行环境，3.13 暂未进行完整兼容测试 |
| SQLite | 3.35.0 及以上 | 默认嵌入式数据库，需启用 FTS5 扩展（编译时默认开启） |
| PostgreSQL | 12.0 及以上 | 可选生产级数据库，需启用 pg_trgm 与 vector 扩展 |
| pip | 21.0 及以上 | Python 包管理器，用于安装依赖 |
| Node.js | 16.0 及以上 | 仅用于前端静态资源构建（可跳过，使用预编译版本） |
| Redis | 6.0 及以上 | 可选缓存组件，用于提升搜索响应速度 |
| Nginx | 1.18 及以上 | 推荐反向代理服务器，处理静态文件与负载均衡 |
| Docker | 20.10 及以上 | 容器化部署方式，提供一键启动环境（非必需） |
| git | 2.25 及以上 | 代码克隆与版本管理（开发必需） |

## 项目结构

```
linkweaver/
├── api/                           # RESTful API 路由与视图函数
│   ├── v1/                        # API v1 版本实现
│   │   ├── endpoints/             # 各资源端点（links, series, tags）
│   │   ├── auth.py                # API Token 校验与权限装饰器
│   │   └── serializers.py         # 请求/响应数据序列化与校验
│   └── middleware/                # 请求日志、限流、CORS 中间件
│
├── core/                          # 核心业务逻辑层
│   ├── models/                    # 数据库模型定义（SQLAlchemy ORM）
```

## 常见问题

问：状态监控检测到大量链接超时或 404，但浏览器手动访问正常，原因是什么？

答：监控服务默认使用 requests 库发送 HTTP HEAD 请求，部分网站可能拒绝 HEAD 方法或对无 User-Agent 的请求返回非标准状态码。可在配置文件中将监控请求方法改为 GET，并设置合理的超时时间（建议 10 秒）。此外，若网站有反爬策略，可尝试启用代理池或降低检测频率。若为内网链接，请确保监控服务与目标主机网络互通。

问：导入包含上千条链接的书签 HTML 文件时页面无响应或超时，如何处理？

答：前端导入接口默认有 30 秒超时限制，大文件应使用命令行工具 `python manage.py import --file bookmarks.html` 进行后台导入，该命令不受 Web 超时限制且会输出详细进度日志。若仍需使用 Web 导入，可在配置中调大 `MAX_CONTENT_LENGTH` 与 `TIMEOUT` 值，并分批次拆分文件。

问：SQLite 数据库在并发写入时出现 "database is locked" 错误，如何解决？

答：SQLite 默认为串行化写入，高并发场景下建议切换至 PostgreSQL。若仍需使用 SQLite，可调整连接池参数：设置 `PRAGMA journal_mode=WAL` 和 `PRAGMA synchronous=NORMAL`，并合理配置连接池大小（建议不超过 5）。生产环境请始终使用 PostgreSQL 以确保数据一致性与写入性能。
