# ResourceLink Hub

ResourceLink Hub 是一个面向开发者、技术研究人员与内容创作者的轻量级外链资源汇总与导航系统。该项目不对资源内容本身进行二次存储或深度加工，而是以高可读性的结构化目录与标签体系，对分布在互联网各处的优质技术文档、交互工具、参考手册与社区帖子进行索引与分类，帮助用户在特定技术领域内快速定位高价值外部链接。

项目定位为“技术外链的元目录”，适用于需要维护大量外部参考链接的个人知识库、团队文档站或开源项目附属导航页。ResourceLink Hub 不依赖数据库，基于静态 Markdown 与 JSON 索引运行，支持快速部署到任意 Web 服务器或静态托管平台，并允许用户通过 Pull Request 方式提交新链接或更新分类规则。

## 功能概览

- **按技术领域分类索引**：资源按编程语言、框架、运维、算法等一级类别归组，每个类别下可挂载任意数量的外链，并支持多级标签过滤。

- **资源卡片元数据展示**：每个外链条目可记录标题、简短描述、推荐指数、更新日期与适用场景，方便用户在点击前评估其价值。

- **全文模糊搜索**：基于前端倒排索引实现资源标题、描述与标签的快速检索，无需后端服务，搜索响应时间低于 200 毫秒。

- **批量导入与校验**：支持从 CSV 或纯文本列表批量导入 URL，自动去重并对可访问性进行 HTTP 状态预检，标记失效链接。

- **链接时效性监控**：内置简单的定时任务接口，可每日检查已收录链接的响应状态，生成失效报告并支持邮件或 Webhook 通知。

- **自定义分类树**：允许用户通过编辑单一 JSON 配置文件来新增、删除或重排分类层级，无需修改代码逻辑。

- **暗色主题与响应式布局**：前端界面同时支持亮色与暗色主题，且针对移动设备、平板与桌面进行自适应优化，确保在不同屏幕尺寸下均有良好的浏览体验。

- **访问统计与热度排行**：基于匿名页面点击日志，统计各链接的周/月访问频次，生成热门资源 Top 20 榜单，辅助用户发现流行内容。

## 应用场景

- **技术团队内部知识库导航**：研发团队可将日常使用的 API 文档、监控面板、日志查询工具、内部设计文档等外链统一收录至 ResourceLink Hub，作为团队首页或浏览器新标签页，减少成员在收藏夹中查找链接的时间成本。

- **开源项目附属资源站**：开源项目维护者可将项目相关的生态工具、插件列表、示例代码仓库、视频教程及社区讨论帖集中整理为外链导航，放在项目 README 或 GitHub Pages 中，帮助贡献者与用户快速了解项目周边生态。

- **技术博客或 newsletter 的参考链接附录**：技术博主在撰写系列文章或发布周报时，可将文中提及的所有外部参考资料、数据来源或工具地址统一汇总为 ResourceLink Hub 的一个子分类，方便读者一键访问原文，避免在文章内堆砌长链接。

- **在线教育或培训课程配套资源目录**：培训机构或大学课程讲师可将每节课涉及的延伸阅读、实验环境地址、在线编译器与习题答案链接整理为按课时划分的导航页面，学生可通过统一入口获取所有外部资源，无需在多个平台间切换。

- **个人知识管理（PKM）外链中台**：知识管理爱好者可将不同工具（如 Notion、Obsidian、Logseq）中的外部链接抽离出来，统一存放在 ResourceLink Hub 中，再通过标签或分类反向同步到各工具，实现链接的单源管理。

## 快速开始

以下命令可在本地快速启动 ResourceLink Hub 开发实例，默认监听端口 8080，并挂载示例资源数据。

```bash
git clone https://github.com/resourcelink-hub/resourcelink-hub.git
cd resourcelink-hub
npm install
npm run build
npm start
```

若需使用开发热重载模式，请执行 `npm run dev`，访问 http://localhost:5173 预览。生产环境部署建议使用 `npm run build` 生成静态文件，然后通过 nginx 或 serve 工具托管 `dist` 目录。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 运行构建工具链与开发服务器，推荐使用 nvm 管理版本 |
| npm | 9.x 或 10.x | 依赖包管理工具，随 Node.js 一同安装 |
| Git | 2.30 及以上 | 用于克隆仓库及后续通过 git hook 执行提交前校验 |
| 现代浏览器 | Chrome 110+ / Firefox 110+ / Edge 110+ | 前端界面基于 ES2022 与 CSS Grid 构建，不支持 IE 及旧版 Safari |
| 硬盘空间 | 至少 200 MB | 包含源码、依赖包及构建产物，若需存储日志或缓存索引则建议预留 1 GB 以上 |

## 项目结构

```
resourcelink-hub/
├── config/                         # 全局配置与分类树定义
│   ├── categories.json             # 一级分类与二级标签的层级树
│   ├── monitoring.json             # 链接时效性检查的阈值、周期与通知方式
│   └── search-index.json           # 预构建的倒排索引快照（启动时加载）
├── data/                           # 资源条目存储目录
│   ├── entries/                    # 按分类存放的 .json 文件，每个文件包含该分类下的所有链接元数据
│   │   ├── architecture.json
│   │   ├── database.json
│   │   ├── devops.json
│   │   ├── frontend.json
│   │   ├── language.json
│   │   └── security.json
│   ├── tags-index.json             # 标签到条目 ID 的映射，用于快速筛选
│   └── hot-rankings.json           # 基于点击日志计算的周/月热度排行缓存
├── src/
│   ├── core/                       # 核心逻辑模块
│   │   ├── loader.js               # 加载配置与数据文件，构建内存数据池
```

## 贡献指南

我们欢迎任何形式的贡献，包括但不限于新增资源链接、改进分类体系、修复前端显示问题以及完善文档。请按照以下步骤提交你的贡献。

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境。确保你的 Node.js 版本符合安装要求，并执行 `npm install` 安装所有依赖。

2. 创建新的功能分支，分支命名格式为 `feature/简短描述` 或 `fix/问题编号`。对于新增资源链接，请在 `data/entries/` 下对应的分类文件中按已有格式追加条目，并确保 `url` 字段为原始链接，`title` 与 `description` 使用中文或英文填写完整。

3. 在提交前运行 `npm run test` 确保所有单元测试通过，并执行 `npm run lint` 进行代码风格检查。若新增或修改了分类树，需同步更新 `config/categories.json` 并执行 `npm run rebuild-index` 重新生成搜索索引。

4. 提交 Pull Request 到主仓库的 `main` 分支，并在 PR 描述中说明本次变更的内容、影响范围以及是否涉及破坏性改动。若新增外部链接，请附上链接的简要说明与推荐理由。

5. 等待维护者审核。审核过程中可能会要求你补充元数据、调整分类或修复链接可达性问题。合并后你的贡献将出现在下一版本的资源列表中，并记录在贡献者名单中。

## 常见问题

**问：ResourceLink Hub 是否会对收录的外链内容进行备份或缓存？**

答：不会。本项目仅存储 URL 及其描述性元数据，不抓取、不缓存、不代理任何外部资源的内容。所有链接点击后均直接跳转至原始目标地址，用户的访问行为与原始站点之间的数据交互不受本项目影响。我们强烈建议用户遵守各目标站点的使用条款与 robots 协议。

**问：如何批量更新已有链接的状态或元数据？**

答：你可以使用 `scripts/check-all-links.js` 脚本对所有已收录链接进行可达性探测，该脚本会生成一份 CSV 格式的状态报告，包含每个链接的 HTTP 状态码、响应时间与最后检查时间。若需要批量修改元数据（如分类、标签或描述），建议导出对应分类的 JSON 文件，在编辑器中批量替换后再重新导入，最后执行 `npm run rebuild-index` 更新搜索索引。

**问：前端搜索为何无法搜到最新添加的链接？**

答：搜索索引在启动时加载 `config/search-index.json` 的快照。如果你在项目运行期间通过 API 或直接修改数据文件新增了条目，需要触发索引重建。在生产环境中，建议在数据变更后手动执行 `npm run rebuild-index` 并重启服务，或配置 Webhook 在数据更新时自动触发构建流程。开发模式下可使用 `npm run dev` 配合热重载，但索引更新仍需要手动触发。
