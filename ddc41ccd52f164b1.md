# LinkTrawl Resource Aggregator

LinkTrawl is a high-performance, crawler-friendly resource indexing and external link aggregation system designed for technical researchers, data analysts, and content curators who need to systematically catalog, validate, and distribute large volumes of structured URL resources. The project addresses the fundamental challenge of managing hundreds or thousands of external links across multiple domains while maintaining metadata integrity, accessibility monitoring, and batch export capabilities.

Target users include open-source intelligence researchers, SEO technical auditors, academic citation managers, and DevOps engineers building automated link pipelines. LinkTrawl provides a lightweight yet extensible framework that transforms raw URL lists into queryable, version-controlled resource catalogs with minimal operational overhead.

## 功能概览

**批量资源导入** - Supports ingestion of plain-text URL lists, CSV exports from browser bookmarks, and scraped link datasets through a unified CLI interface.

**自动元数据抓取** - Fetches HTTP status codes, content-type headers, page titles, and last-modified timestamps for each submitted URL without blocking on failures.

**分类标签系统** - Allows hierarchical tagging of resources by domain category, content language, file type, and trust level using YAML-based taxonomy definitions.

**去重与规范化** - Detects duplicate entries via URL fingerprinting and canonicalization rules, automatically stripping tracking parameters and enforcing protocol consistency.

**健康检查调度** - Runs periodic HEAD requests against all stored URLs, generating availability reports and flagging broken or redirected links.

**多格式导出** - Exports resource lists as JSON, CSV, Markdown tables, and static HTML indices suitable for embedding in documentation sites.

**查询过滤器** - Provides grep-like pattern matching, domain whitelisting/blacklisting, and date-range filters for subset extraction.

**插件钩子系统** - Supports custom Python scripts that execute before import, after export, or on scheduled intervals for advanced data enrichment.

## 应用场景

**技术文档库维护** - Documentation teams can maintain a centralized reference section by feeding LinkTrawl with external specification links, ensuring all cited resources remain accessible across release cycles. The health check feature automatically notifies maintainers when vendor documentation moves or becomes unavailable.

**OSINT 数据聚合** - Researchers collecting public data sources from multiple jurisdictions can import lists of government portals, NGO reports, and news archives, then export filtered subsets by region or topic for downstream analysis pipelines.

**SEO 反向链路审计** - Digital marketing professionals use LinkTrawl to inventory backlink profiles, verify that partner sites are still linking to their content, and identify expired domains that require replacement with alternative authoritative sources.

**学术参考文献校验** - Academic authors and preprint servers can validate reference URLs in manuscripts, flagging broken links before submission and generating machine-readable citation manifests that comply with repository preservation policies.

**内部知识库构建** - Enterprise engineering teams aggregate internal tool dashboards, API documentation endpoints, and runbook references into a single browsable catalog that updates automatically as services are deployed or decommissioned.

## 快速开始

Clone the repository, install dependencies, and run the initial import workflow with the following commands:

```bash
git clone https://github.com/your-organization/linktrawl.git
cd linktrawl
pip install -r requirements.txt
python linktrawl.py import --source resources.txt --output catalog.json
python linktrawl.py serve --port 8080
```

For batch processing the supplied resource list, place the raw URLs in a file named `batch-99.txt` and execute:

```bash
python linktrawl.py batch --input batch-99.txt --validate --tag "technical-reference"
python linktrawl.py export --format markdown --section "resource-list" --output README.md
```

## 贡献指南

欢迎社区开发者提交改进和扩展。请遵循以下流程以确保代码质量和项目一致性：

1. 阅读架构文档和插件开发参考，理解核心数据流和钩子系统的工作方式。在提交功能性变更之前，建议在 GitHub Issues 中开启一个讨论议题，与维护者确认需求方向。

2. 克隆仓库并安装开发依赖（`pip install -r requirements-dev.txt`），启用 pre-commit 钩子以自动运行 black 格式化和 flake8 检查。所有新增代码必须包含对应的单元测试，测试覆盖率不应低于 80%。

3. 对于新增插件或扩展功能，请同时更新 docs 目录下的相应文档页面，并在插件代码头部添加详细的 docstring 说明其用途、配置参数和钩子挂载点。

4. 提交 pull request 时，请确保标题遵循语义化提交规范（如 `feat: 添加批量重试机制` 或 `fix: 修复 HTTPS 重定向处理`），并在描述中引用相关的 issue 编号和测试结果截图。

5. 所有贡献将经过至少两名维护者的代码审查，CI 流水线将自动运行测试套件和静态分析。审查通过后，维护者将负责合并并更新 CHANGELOG 文件。

## 常见问题

**问：导入大量 URL 时内存占用过高怎么办？**

答：LinkTrawl 默认采用流式处理，逐条读取输入文件并立即写入临时索引，不会将全部数据加载至内存。如果仍然遇到内存压力，可以通过 `--chunk-size` 参数调整批处理大小（默认 1000 条/批）。对于超过 10 万条的超大规模数据集，建议启用 SQLite 后端存储模式（在配置文件中将 `storage.backend` 设为 `sqlite`）。

**问：健康检查发现失效链接后，系统如何处理？**

答：系统将失效链接的状态码和错误原因记录至 `failed.log` 文件，并在下一次调度周期（默认每 24 小时）重新尝试最多三次。连续三次失败的链接会被标记为 `dead` 状态，从活跃目录中移除并归档至 `dead_links.json`。管理员可以通过 `linktrawl.py report --dead` 命令生成失效链接报表，或通过 Webhook 接收实时告警。

**问：能否自定义抓取时使用的 User-Agent 和请求头？**

答：可以。在配置文件的 `fetcher.headers` 字段中可指定任意键值对，全局生效。如果需要对特定域名使用不同的请求头，可在 `config/taxonomy.yaml` 中为对应标签设置 `headers_override` 属性。插件系统也允许在 `on_before_fetch` 钩子中动态修改请求参数。
