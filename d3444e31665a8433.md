# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的外链资源聚合与导航系统。该项目并非简单的书签管理工具，而是一个具备分类索引、状态监控与批量导入导出能力的轻量级外链中台。其目标用户包括开源文档维护者、技术社区运营人员以及需要集中管理大量外部参考链接的研发团队。

LinkVault Core 通过结构化的数据组织方式，将散落在各处的 URL 转化为可追溯、可审计的知识资产。用户可以通过该项目快速建立内部技术资源索引，避免链接失效导致的文档空洞，同时借助批量处理能力降低人工维护成本。

## 功能概览

**批量链接导入**：支持通过文本文件或标准输入流一次性导入大量 URL，自动去重并识别域名归属。

**链接状态检测**：内置异步 HTTP 健康检查器，可定期探测每个链接的可访问性，自动标记失效或重定向状态。

**分类标签系统**：允许用户为每个链接添加多个自定义标签，支持基于标签的快速过滤与分组统计。

**全文检索支持**：基于链接标题、描述文本与标签组合的轻量级全文检索引擎，支持模糊匹配与前缀搜索。

**数据导出管道**：可将索引数据导出为 JSON、CSV 或 Markdown 表格格式，便于嵌入技术文档或迁移至其他系统。

**访问统计看板**：记录每个链接的点击次数与最后访问时间，提供简单的热度排序功能。

**黑白名单机制**：支持配置域名级别的访问控制规则，自动拦截可疑或非技术类外链。

**快照备份恢复**：定期生成索引数据的时间点快照，支持按版本回滚至任意历史状态。

## 应用场景

技术文档仓库的外链治理：在大型开源项目的文档库中，维护人员可使用 LinkVault Core 统一管理所有外部引用链接。当某个依赖库更换域名或文档迁移时，系统可快速定位受影响条目，避免读者遇到死链。

社区资源周报生成：技术社区运营者每周汇总社区内分享的优质外链，通过该系统完成分类归档后，一键生成 Markdown 格式的资源列表，直接粘贴至周报邮件或论坛公告中。

内部研发知识库的链接审计：企业研发团队的知识库通常包含大量指向内部 Wiki、Jira 工单、代码评审记录的链接。LinkVault Core 可定期执行全量链接存活扫描，并生成审计报告，帮助团队及时发现权限变更或服务下架导致的链接失效。

开源项目 README 链接同步：开源项目维护者可将项目 README 中引用的所有外链导入系统，当需要批量更新 URL 前缀或替换域名时，利用系统的批量编辑功能完成修改，再导出更新后的 Markdown 内容。

## 快速开始

以下命令演示了从源码克隆到启动服务的完整流程。默认使用 8080 端口提供 Web 界面与 RESTful API。

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
go mod download
go run cmd/linkvault/main.go --port 8080 --config ./configs/default.yaml
```

若需要立即导入一批链接，可通过标准输入传递文件路径或直接传入 URL 列表。

```bash
echo "https://www.photographertoronto.net/movie/wTKN" | go run cmd/linkvault/main.go --import-stdin
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Go 编译器 | 1.21 或更高 | 核心后端服务使用 Go 语言开发，依赖泛型特性 |
| SQLite 数据库 | 3.35 及以上 | 嵌入式关系型数据库，无需独立部署，用于存储链接元数据 |
| Node.js 运行时 | 18.17 LTS 或更高 | 仅用于前端管理界面的构建工具链，生产环境可禁用 |
| make 构建工具 | 3.81 或更高 | 用于执行 Makefile 中定义的编译与打包任务 |
| git 版本控制 | 2.25 或更高 | 克隆仓库及版本标签识别，非强制但推荐 |
| curl 命令行工具 | 7.68 或更高 | 运行状态检测脚本时用于 HTTP 探测，可选替换为 wget |

├── internal/

│   │   └── health.go             # 健康状态检测器：并发探测、超时控制、结果缓存
│   ├── storage/                  # 数据持久化层
│   │   ├── sqlite.go             # SQLite 驱动适配，表结构迁移与事务封装
│   │   └── snapshot.go           # 快照序列化与反序列化，版本差异计算
│   ├── api/                      # HTTP 路由与中间件
│   │   ├── routes.go             # RESTful 端点注册：导入、查询、导出、状态更新
│   │   └── validator.go          # 请求参数校验：URL 格式、标签白名单、分页边界
│   └── worker/                   # 后台异步任务
│       ├── scanner.go            # 周期性扫描任务调度器，基于 cron 表达式

## 常见问题

**问：导入大量链接时，系统如何避免重复记录？**

系统基于 URL 的标准化形式（移除末尾斜杠、统一小写域名、解码百分号编码）计算哈希指纹，在写入数据库前执行去重检查。用户可通过命令行参数 --dedup-mode 控制去重策略，可选 strict（完全匹配）、domain-only（仅域名）、或 disabled（关闭去重）。默认使用 strict 模式。

**问：健康状态检测是否会影响目标服务器的正常访问？**

检测器默认使用 GET 方法并携带 User-Agent 标识为 LinkVault-Core/1.0，同时遵循 robots.txt 的爬取规则。并发请求数可通过配置文件调整，默认不超过 32 个并行连接。超时时间设为 5 秒，避免长时间阻塞。对于内网地址或私有 IP，检测器默认跳过，可显式配置扫描范围以启用。

**问：如何将 LinkVault Core 中的数据迁移至其他系统？**

系统提供两种导出方式：通过 Web 界面的 "导出" 按钮生成 JSON 或 CSV 文件，或使用命令行工具执行 linkvault export --format markdown --output links.md 直接生成 Markdown 表格。导出的 JSON 结构包含链接 URL、标题、标签数组、添加时间、最后检测状态与检测时间戳，便于下游系统解析。
