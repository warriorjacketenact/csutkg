# LinkVault Resource Aggregator

LinkVault 是一个面向开发者、技术研究人员与内容创作者的轻量级外链资源汇总与导航工具。项目定位为“技术资源索引中间件”，通过结构化的数据组织方式，将分散于多业务域的外部链接整合为可检索、可分类、可版本化的内部知识库。目标用户包括运维工程师、全栈开发人员、技术文档撰写者以及需要批量管理外部参考链接的团队。

项目核心解决以下问题：团队协作中外部链接散落于文档、邮件、即时通讯记录，导致复用困难；技术调研过程中积累的大量参考网址缺乏统一管理手段；自动化流程（如 CI/CD 文档生成、日报自动汇总）需要稳定可编程的链接数据源。LinkVault 不直接存储页面内容，仅提供链接元数据（标题、标签、访问状态、入库时间）的组织框架，并内置定时连通性检查，帮助用户维护链接健康度。

## 功能概览

**批量链接导入** 支持从纯文本、CSV 及 JSON 格式批量导入外部 URL，自动解析协议与域名，去重并生成唯一标识符。

**分类标签系统** 允许用户为每条链接添加多级标签（如 backend / database / monitoring），支持基于标签的快速筛选与统计。

**连通性健康检查** 后台定时任务对已入库链接发起 HEAD 请求，标记异常状态（超时、4xx、5xx），并生成可视化报表。

**Markdown 文档生成** 内置模板引擎，可将指定标签或全量链接输出为符合项目风格的 Markdown 资源列表，便于直接粘贴至 README 或 Wiki。

**RESTful API 接口** 提供基于 JSON 的查询接口，支持按标签、入库时间、域名过滤，方便集成至外部仪表盘或自动化脚本。

**访问统计看板** 记录每条链接的点击次数与最后访问时间，辅助判断资源热度与长期价值。

**私有化部署支持** 项目完全离线运行，所有数据存储于本地 SQLite 数据库，无需外部网络依赖（除链接检查任务外）。

## 应用场景

**技术团队内部知识库构建** 团队在开发过程中积累了大量第三方库文档、API 参考、故障排查博客。使用 LinkVault 统一入库并添加业务域标签，新成员可通过标签快速获取领域内关键资源，缩短上手周期。

**自动化运维日报生成** 运维人员将监控面板、日志平台、报警管理页面的链接录入系统，结合连通性检查结果，每日自动生成包含所有关键入口状态的可视化报告，发送至团队钉钉或邮件组。

**开源项目外链管理** 开源项目维护者通常需要在 README 中维护大量社区资源、教程、相关项目链接。LinkVault 提供单命令导出为 Markdown 列表的功能，确保文档中的外链始终与内部库保持一致，避免死链或过时链接流出。

**技术调研归档** 技术选型或竞品分析过程中，调研人员收集数十至数百个参考网址。LinkVault 允许按调研批次（batch）组织链接，并添加备注字段记录每个链接的核心结论，后续回溯时无需重新浏览页面。

## 快速开始

以下操作基于 Linux / macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动服务
python manage.py migrate
python manage.py runserver --host 0.0.0.0 --port 8080
```

访问 http://localhost:8080 即可进入 Web 管理界面。首次启动将自动创建默认管理员账户（admin / admin123），请登录后及时修改密码。

## 项目结构

```
linkvault/
├── src/                                 # 核心源代码目录
│   ├── core/                            # 数据模型与数据库操作层
│   │   ├── models.py                    # SQLAlchemy ORM 模型定义（Link, Tag, CheckRecord）
│   │   ├── database.py                  # 连接池与会话管理
│   │   └── repository.py                # 增删改查的仓储接口实现
│   ├── api/                             # RESTful 路由层
│   │   ├── routes.py                    # 所有 API 端点注册
│   │   ├── schemas.py                   # Pydantic 请求/响应校验模型
│   │   └── middleware.py                # 鉴权与日志中间件
│   ├── checker/                         # 连通性检查异步任务
│   │   ├── worker.py                    # 基于 APScheduler 的定时调度器
│   │   ├── http_client.py               # 带超时与重试策略的 requests 封装
│   │   └── reporter.py                  # 生成检查结果报表（HTML/Markdown）
│   ├── exporter/                        # 导出引擎
│   │   ├── markdown.py                  # Markdown 列表生成器（支持自定义模板）
│   │   ├── csv_writer.py                # CSV 批量导出
```

5. 发起 Pull Request 至 main 分支，描述中写明变更动机、影响范围及测试结果。至少一名维护者审核通过后合并。

## 常见问题

**Q: 导入大量链接（超过 5000 条）时页面卡顿，如何处理？**

A: 推荐使用命令行导入方式：python cli/main.py import --file links.json --batch-size 500。该方式采用分批次提交事务，避免内存溢出。Web 界面单次导入上限为 1000 条，以保障前端响应速度。

**Q: 连通性检查是否会频繁触发目标服务器限流？**

A: 检查任务默认并发数设置为 5，间隔至少 10 秒发起下一个请求，且仅对同一域名每分钟最多检查一次。可通过 config/production.env 中的 CHECKER_RATE_LIMIT 调整阈值。同时支持配置白名单域名跳过检查。

**Q: 如何从外部系统（如 Confluence 或 Notion）自动同步链接？**

A: 项目提供了通用的 HTTP 导入接口，外部系统可调用 POST /api/links 批量提交 JSON 数据。我们也提供了 Notion 官方 Python SDK 的集成示例，位于 contrib/notion_sync.py，可直接参考修改。
