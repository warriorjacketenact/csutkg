# LinkVault 技术资源索引系统

LinkVault 是一个面向开发者、技术研究人员与开源社区维护者的结构化外链资源汇总与导航工具。项目定位为“技术知识库的元索引层”，不直接托管内容，而是通过可维护的链接集合与分类体系，帮助用户从海量分散的在线资源中快速定位高价值信息。目标用户包括需要持续跟踪技术动态的架构师、撰写技术文档的工程师、以及搭建团队知识库的DevOps人员。项目采用纯静态Markdown驱动，配合轻量级脚本实现链接可用性检测与分类统计，解决个人书签杂乱、团队共享困难、资源失效不可知等常见痛点。

## 功能概览

**批量链接导入与去重** 支持从文本文件或标准输入流批量导入URL，自动识别重复条目并生成唯一标识符，避免冗余存储。

**分类标签系统** 每个资源可关联多个自定义标签（如#performance、#security、#frontend），支持按标签组合过滤与反向索引生成。

**可用性健康检查** 内置异步HTTP探测模块，可配置超时与重试策略，定时检测链接状态并生成失效报告，标记302/404/500等异常码。

**结构化导出接口** 支持将索引数据导出为JSON、YAML或纯HTML目录页，便于嵌入静态站点生成器（如Hugo、VuePress）或CI/CD流水线。

**变更日志追踪** 记录每次增删改操作的时间戳与操作者（本地模式记录系统用户），提供diff视图，满足审计与回溯需求。

**全文检索与模糊匹配** 基于倒排索引实现关键词搜索，支持URL路径、标题、描述字段的模糊查询，响应时间控制在200毫秒以内。

**自定义元数据扩展** 每条记录允许附加自定义键值对（如priority、owner、deadline），满足企业级知识管理场景的定制化要求。

**配额与速率限制** 对公开API端点实施令牌桶限流，防止恶意爬取，单IP每分钟最大请求数为60次。

## 应用场景

**技术团队内部知识库构建** 团队Leader可将分散在个人浏览器书签、Slack聊天记录、邮件中的技术文章、工具站点、API文档统一汇总至LinkVault，分配标签与负责人，形成可持续维护的团队知识资产。新成员入职时可直接通过标签筛选学习路径，缩短上手周期。

**开源项目README外链管理** 开源项目维护者通常需要在README中引用大量依赖文档、社区论坛、示例项目。使用LinkVault维护独立的外链清单，通过导出功能生成Markdown格式的列表，一键嵌入项目文档，避免手动更新造成的链接漂移。

**技术雷达与趋势跟踪** 技术决策者可以定期从Hacker News、Reddit、技术周报等来源收集新兴工具与框架，利用LinkVault的标签系统标记“评估中”、“已采用”、“已废弃”等状态，结合健康检查模块自动预警失效或迁移的域名，辅助技术选型决策。

**离线文档镜像源管理** 在受限网络环境中，运维人员可通过LinkVault维护内部镜像源的URL映射表，记录官方源与镜像源的对应关系，配合导出功能生成pacman、apt或pip的源配置文件，实现多环境一致性。

## 快速开始

以下指令适用于Linux/macOS环境，Windows用户建议使用WSL2或Git Bash。

```bash
# 克隆代码仓库至本地
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装Python依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地数据库并启动Web服务（默认监听127.0.0.1:8080）
python linkvault.py init
python linkvault.py serve --port 8080
```

首次启动后，访问 http://127.0.0.1:8080 即可进入仪表板。通过 `--import` 参数可批量导入初始链接列表，示例数据位于 `samples/urls.txt`。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 或更高 | 核心运行环境，建议使用3.11+以获得性能优化 |
| SQLite | 3.35.0 或更高 | 嵌入式数据库，用于存储链接元数据与标签关系 |
| aiohttp | 3.8.0 或更高 | 异步HTTP客户端，用于并发健康检查 |
| click | 8.1.0 或更高 | 命令行交互框架，提供子命令解析能力 |
| jinja2 | 3.1.0 或更高 | 模板引擎，用于生成HTML目录页与报告 |
| pytest | 7.0.0 或更高 | 仅开发测试需要，生产环境可不安装 |
| black | 22.0.0 或更高 | 仅代码格式化需要，非运行时依赖 |
| 磁盘空间 | 至少100 MB | 用于存储数据库文件与日志，实际随链接数量线性增长 |
| 内存 | 最低512 MB，推荐1 GB | 健康检查并发数较高时内存占用相应增加 |

## 项目结构

```
linkvault/
├── linkvault.py                # 主入口文件，整合CLI命令与路由分发
├── requirements.txt            # 生产环境依赖清单（固定版本）
├── dev-requirements.txt        # 开发测试额外依赖（pytest, black, mypy）
├── config/
│   ├── default.yaml            # 默认配置（端口、数据库路径、日志级别）
│   └── schema.json             # 配置项JSON Schema校验文件
├── core/
│   ├── __init__.py
│   ├── database.py             # SQLite连接池、表结构初始化、CRUD操作
│   ├── models.py               # Pydantic数据模型（Link, Tag, CheckResult）
│   ├── checker.py              # 异步健康检查器，含指数退避重试逻辑
│   └── indexer.py              # 倒排索引构建与全文检索实现
├── api/
│   ├── __init__.py
│   ├── routes.py               # aiohttp路由注册与请求参数解析
│   ├── handlers.py             # 各端点具体业务逻辑（增删改查、导入导出）
│   └── middlewares.py          # 限频中间件、异常统一捕获、日志记录
├── templates/
│   ├── base.html               # Jinja2基础模板（含导航栏与页脚）
│   ├── dashboard.html          # 仪表板：统计卡片与最近活动列表
```

## 常见问题

**Q: 健康检查模块是否会因为目标站点响应缓慢而阻塞主进程？**

A: 不会。健康检查采用aiohttp提供的异步非阻塞IO模型，所有探测任务被封装为协程并发执行，主事件循环不会等待单个请求完成。同时支持通过 `--concurrency` 参数控制最大并行数（默认20），避免对目标站点造成压力。超时时间与重试次数可在 `config/default.yaml` 中分别调整（默认超时10秒，重试2次）。

**Q: 如何迁移或备份现有的链接索引数据？**

A: 所有元数据存储在SQLite数据库文件（默认为 `data/linkvault.db`）中。备份时直接复制该文件即可，支持跨平台恢复。如需迁移至其他数据库系统（如PostgreSQL），可使用 `linkvault.py export --format json` 导出全量数据，再通过目标数据库的导入工具完成迁移。项目官方不提供直接连接外部数据库的选项，以保持轻量级与零运维特性。

**Q: 能否与现有的静态站点生成器（如VuePress、Docusaurus）集成？**

A: 可以。LinkVault提供了 `export --format html` 命令，生成独立的目录页HTML文件，该文件包含所有链接的分页列表与标签云，且不依赖外部资源（所有样式内联）。您可以将此HTML文件放置于静态站点的 `public` 或 `docs/.vuepress/dist` 目录下，通过iframe或直接引用作为资源导航子页面。同时支持 `export --format markdown` 生成适用于GitHub README的表格或列表格式。
