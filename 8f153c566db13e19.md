# ResourceLink Hub

ResourceLink Hub 是一个面向开发者、技术研究人员与内容策展人的轻量级外链资源汇总平台。该项目并非传统的爬虫或采集系统，而是一个结构化的外部链接治理与导航工具，用于对分散在互联网各处的优质技术文档、工具站点、社区讨论与参考实现进行集中收录、分类标注与状态监控。

项目定位为“技术资源的中转站与索引层”，主要解决以下问题：团队成员在查阅资料时反复搜索相同链接导致的效率损耗；外部资源失效后无法及时发现并替换；缺乏统一的元数据描述导致跨项目引用混乱。ResourceLink Hub 通过标准化的链接入库流程、自动化的可用性检测以及可视化的分类看板，帮助技术团队构建属于自身的知识外围系统。

## 功能概览

**链接入库与去重校验** 提供命令行与 Web 表单两种录入方式，系统自动根据 URL 指纹与域名规则执行去重，避免重复条目占用存储空间。

**分类标签与多级目录映射** 支持为每条外链赋予技术领域、资源类型、适用人群等自定义标签，并可挂载至多个虚拟目录下，实现同一资源在不同上下文中的灵活呈现。

**可用性健康检查** 内置异步任务调度器，可按照每日或每周频率对已收录链接发起 HEAD 请求，检测返回状态码并记录响应耗时，异常链接自动归入待复核队列。

**全文检索与过滤查询** 基于倒排索引实现按标题、描述、标签、域名和收录时间的多条件组合搜索，结果可按相关度或更新时间排序。

**导入导出与批量操作** 支持以 CSV、JSON 和 Markdown 表格格式导出链接清单，亦可通过标准模板批量导入新链接，适用于迁移或初始化数据场景。

**收藏与备注协作** 授权用户可为特定链接添加个人备忘或团队共享批注，记录使用心得、适配版本或已知限制，便于后续成员快速理解资源价值。

**访问统计与热度排序** 统计每条链接被查看或跳转的次数，生成趋势图表，辅助识别高频使用的核心资源，为目录重组提供数据依据。

## 应用场景

技术团队内部知识库构建：开发组可将日常调研中发现的优秀博客、API 参考手册和开源组件仓库统一录入 ResourceLink Hub，并与内部 Wiki 或项目看板联动，减少重复沟通成本。

技术文档编写素材管理：文档工程师在编写系统设计说明或用户手册时，需频繁引用外部标准规范或第三方工具主页，通过本平台集中存放相关外链，可在版本迭代中快速更新引用集合。

开源项目外部依赖溯源：开源维护者可在项目仓库中嵌入 ResourceLink Hub 的只读视图，向社区展示所引用的数据源、算法实现或测试框架出处，提升透明度与合规性。

技术培训与新人引导：团队培训负责人可创建专题链接集合，覆盖开发环境搭建、编码规范、CI/CD 流水线示例等，新成员通过访问聚合页面即可获得完整入门路径。

个人技术阅读流管理：技术爱好者可用其整理每日订阅的资讯站点、RSS 源或论文预印本仓库，替代散乱的书签夹，配合健康检查功能及时清理失效来源。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git 与 Node.js 18+。

```bash
# 克隆代码仓库
git clone https://github.com/resourcelink-hub/resourcelink-hub.git
cd resourcelink-hub

# 安装项目依赖
npm install

# 以开发模式启动本地服务
npm run dev
```

执行完成后，打开浏览器访问 http://localhost:3000 即可进入 ResourceLink Hub 仪表盘。首次启动将自动执行数据库迁移脚本，生成默认分类与管理员账户。管理员初始密码将在控制台日志中打印，请于首次登录后及时修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用官方二进制或 nvm 安装 |
| npm | 9.x 或 10.x | 包管理器，随 Node.js 一同分发 |
| SQLite | 3.35 以上（内置） | 默认嵌入式数据库，无需额外安装，生产环境可切换至 PostgreSQL |
| Git | 2.30 以上 | 用于克隆仓库与版本控制 |
| 操作系统 | Linux内核4.0+ / macOS 11+ / Windows 10 2004+ | 支持以上平台，Windows 下建议使用 WSL2 |
| 浏览器 | 现代 Chromium / Firefox / WebKit 引擎 | 用于访问管理界面，移动端响应式支持 |
| 网络环境 | 可访问公网 | 用于健康检查任务向外发起请求，内网部署需配置代理 |
| 磁盘空间 | 至少 200 MB | 用于存储代码、数据库文件及日志，随链接数量增长可适当扩容 |
| 内存 | 最低 512 MB，推荐 1 GB | 开发模式内存占用较高，生产模式可降低 |

```

│   │   ├── pages/                 # 页面路由组件
│   │   └── components/            # 可复用 UI 组件
│   └── api/                       # RESTful API 服务（Fastify）
│       ├── routes/                # 路由定义与控制器
│       └── middleware/            # 鉴权、日志与错误处理
├── core/                          # 核心业务逻辑层
│   ├── link-manager/              # 链接增删改查与去重引擎

│   ├── tag-system/                # 标签分类与索引子系统

│   ├── types/                     # TypeScript 类型定义与 Zod 校验
│   ├── constants/                 # 系统常量、错误码与默认配置
```

## 常见问题

Q: 健康检查任务发现大量链接超时或返回 5xx，但手动访问浏览器正常，如何解决？

A: 这是由于健康检查模块默认使用无头 HTTP 客户端，未携带浏览器特有的 Cookie 或 User-Agent 头部。请检查目标站点是否对自动化请求做了限制，可在配置文件中调整健康检查的请求头字段、超时时间（默认为 10 秒）和重试次数。若仍无法解决，可将该链接标记为“需人工验证”，绕过自动检测。

Q: 导入包含 500 条链接的 CSV 文件时，页面提示超时或部分数据丢失，应如何优化？

A: 当前版本的单次批量导入上限为 200 条，建议将大文件拆分为多个批次执行。若需一次性导入大量数据，推荐使用命令行脚本 scripts/bulk-import.js，该脚本支持流式处理与事务回滚，可应对万级数据量。同时请检查 CSV 文件是否包含 BOM 头或特殊分隔符，系统默认以逗号为分隔符并自动修剪空白。

Q: 能否将 ResourceLink Hub 部署到内网环境且完全不访问公网？

A: 可以。在配置文件中将 sync.external 设为 false，即可禁用健康检查中对外部 CDN 的依赖，同时关闭版本更新提示。数据库迁移与本地检索功能均不依赖外部网络。但请注意，收录的链接本身若指向公网资源，在跳转时仍需用户端具备网络访问能力。
