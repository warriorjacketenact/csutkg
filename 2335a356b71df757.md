# ResourceLink Aggregate Service

ResourceLink Aggregate Service 是一个面向技术文档维护者、知识库管理者与内容聚合平台的轻量级外链资源归集与健康度检查工具。该项目定位于解决大规模外链资源在长期运维过程中出现的链接失效、源站迁移、内容变更及分类混乱等问题，通过结构化的数据整理与可重复的自动化校验流程，帮助用户将散落在各类手册、教程或文档中的外部引用纳入统一管理视图。

本项目不提供爬虫或自动采集功能，专注于为已收集的资源集合提供标准化的录入、分类、状态标记与批量导出能力。目标用户包括开源项目文档维护者、企业知识库管理员、技术培训内容策划人员以及需要定期审计外部引用链路的运维工程师。通过使用 ResourceLink Aggregate Service，用户可以显著降低手工整理链接表格的耗时，减少因链接过期导致的文档质量下降，并快速生成符合各类文档规范要求的资源列表章节。

## 功能概览

**结构化资源录入** 提供基于 Markdown 列表的批量链接导入接口，支持按来源域名、内容主题、文件类型或采集批次自动归类，所有原始 URL 保持原样存储，不进行任何自动改写或跳转包装。

**链接状态标记系统** 允许用户为每条资源手动添加状态标签，包括但不限于有效、待复核、已迁移、内容变更、永久失效等，支持自定义标签扩展，便于后续筛选与审计。

**多维度分类视图** 根据 URL 路径模式、查询参数或用户指定的分组规则，将资源列表渲染为按类别分节的 Markdown 表格或嵌套列表，可直接嵌入项目文档或 Wiki 页面。

**批量导出与格式化** 支持将选中的资源集合一次性导出为符合本项目 README 格式要求的资源列表章节，确保所有 URL 严格遵循原始输入格式，不添加协议前缀、不修改大小写、不附加 HTML 标签或 Markdown 链接语法。

**资源变更历史追踪** 记录每次资源列表的更新操作，包括新增链接、删除链接、分类调整与状态变更，支持回溯至任意历史版本，便于团队协作与变更审计。

**链接去重与冲突检测** 自动识别重复提交的 URL，并提示用户确认保留或覆盖策略，避免同一资源在多个分类中重复出现导致统计偏差。

**自定义元数据扩展** 允许为每条资源附加备注字段，用于记录来源说明、收录原因、关联需求编号或内部审核备注，元数据支持全文检索。

**命令行交互与脚本集成** 提供完整的命令行操作界面，支持通过 Shell 脚本或 CI/CD 流水线调用，便于将资源审计任务集成至文档发布流程中。

## 应用场景

技术文档版本发布前的资源审计。在每季度发布产品文档新版本时，文档维护团队需要批量核对手册中引用的全部外部链接是否仍然可访问，以及目标内容是否与文档描述一致。用户可将待审计的链接列表导入本系统，逐条复核并标记状态，最终生成附带状态说明的资源清单随文档一并发布。

开源项目 README 与 Wiki 的链接维护。开源项目维护者通常需要在 README 中列出大量相关资源链接，包括依赖项目、参考资料、社区论坛、扩展工具等。随着项目发展，部分链接可能失效或更换域名。用户可利用本项目的分类导出功能，快速生成格式统一的资源列表章节，替换原有 README 中的陈旧内容。

企业知识库外部引用集中管理。大型企业的内部技术知识库通常包含数千篇文档，其中散布着大量指向外部供应商、标准组织、技术社区或镜像站点的链接。知识库管理员可定期从数据库中提取所有外部 URL，导入本系统进行统一分类与健康度评估，并将评估结果反馈给各文档负责人。

技术培训课程配套资源汇总。培训讲师在准备课程材料时，往往需要为学员提供一份包含参考资料、在线工具、示例代码仓库和视频教程的链接清单。使用本项目可以将散落在幻灯片、讲义和邮件中的链接汇总为结构化的资源列表，并支持按课程模块或学习阶段分组输出。

## 快速开始

以下命令序列演示了从代码仓库克隆项目、安装依赖并执行一次基础资源导入与导出的完整流程。

```bash
git clone https://github.com/resourcelink/aggregate-service.git
cd aggregate-service
pip install -r requirements.txt
python cli.py import --batch 639 --source ./samples/links_639.txt
python cli.py classify --batch 639 --pattern "fzflbg.com/series"
python cli.py export --batch 639 --format readme --output ./output/resource_list_639.md
```

上述命令执行完毕后，用户可在 output 目录下找到生成的资源列表文件，该文件已按照本项目 README 的资源列表格式完成分类与排版。

## 项目结构

```
aggregate-service/
├── cli.py                      # 命令行入口模块，注册所有子命令并调用对应业务逻辑
├── config/
│   ├── default.yaml            # 默认配置文件，包含分类规则模板与导出格式预设
│   └── schema.json             # 配置文件的 JSON Schema 校验定义
├── core/
│   ├── importer.py             # 资源导入模块，支持文本列表、CSV 及 JSON 格式解析
```

## 常见问题

导入资源列表时系统提示链接格式错误，但我的链接可以在浏览器中正常打开。

本项目在导入阶段仅执行基础的 URL 语法校验，例如检查是否包含协议头、是否包含非法字符等。浏览器能够正常打开并不代表 URL 符合 RFC 3986 标准。如果您的链接包含未转义的空格、中文标点或特殊符号，请先进行 URL 编码后再提交。您也可以使用 `--strict=false` 参数关闭严格校验模式，但关闭后可能影响后续分类规则的匹配准确性。

导出生成的资源列表与我在 README 中手动编写的格式不一致。

本项目导出模块严格按照配置文件中定义的模板生成输出。如果您需要自定义列表缩进、是否包含序号、分组标题样式等，请编辑 `config/default.yaml` 中的 `export.template` 段落。修改后使用 `--config` 参数指定自定义配置文件路径即可生效。默认模板采用与本文档资源列表章节完全一致的格式规范。

如何迁移已有的大量链接数据至本系统。

您可以将现有链接整理为每行一个 URL 的纯文本文件，或包含 URL 与备注字段的 CSV 文件。使用 `import` 命令的 `--format` 参数指定输入格式，系统会自动解析并导入。对于 CSV 格式，默认列顺序为 URL、状态、分类、备注，您可以通过 `--column-map` 参数自定义列映射关系。导入完成后使用 `export` 命令即可生成符合本项目规范的资源列表。
