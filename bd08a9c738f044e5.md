# TechLink Archive

TechLink Archive 是一个面向开发者与技术研究人员的结构化外链与技术资源归档系统。本项目不直接托管文章或代码库内容，而是以高可维护性的链接索引方式，对分散于互联网各处的优质技术文档、案例解析与参考资料进行系统性收集、分类与版本管理。项目适用于需要建立个人或团队知识外脑的技术人员，以及对特定技术领域进行长期追踪的研究者。

本项目解决的核心问题包括：技术书签杂乱无章、优质内容随浏览器崩溃丢失、团队内部分享缺乏统一入口、以及技术决策时缺乏可回溯的参考依据。通过将外链资源以扁平化条目结构存储，并配合自动化校验与元数据标注机制，TechLink Archive 使得大规模外链管理具备工程级别的可操作性与可维护性。

## 功能概览

**结构化条目索引**：每条资源以独立条目形式存储，包含标题、来源域名、抓取时间与自定义标签，支持多维度筛选与排序。

**原始链接永久留存**：系统强制保留资源原始 URL 不做任何改写，确保引用链条的完整性与可追溯性，避免因链接二次加工导致的访问失效。

**批次化导入机制**：支持按批次导入大量链接，每批次附带批次编号与数量统计，便于进行增量更新与差异比对。

**域名级聚合视图**：自动提取资源域名并进行聚合统计，可快速查看来自同一来源的全部条目，辅助评估来源权威性。

**标签与全文检索**：基于条目元数据与备注字段提供检索能力，支持模糊匹配与精确匹配两种模式，检索结果按相关度排序。

**快照状态校验**：周期性对已归档链接进行可用性探测，返回 HTTP 状态码并标记异常链接，便于及时清理或更新失效资源。

## 应用场景

**技术团队内部知识库构建**：研发团队可将日常调研中发现的第三方技术博客、官方文档、解决方案贴等统一归档至 TechLink Archive，新成员入职时可快速浏览历史积累的资源清单，缩短业务与技术栈的熟悉周期。

**个人技术阅读工作流管理**：独立开发者或技术爱好者可利用本项目的批次导入功能，将每周阅读的 RSS 订阅、Hacker News 热帖、GitHub Trending 项目等集中录入，形成个人技术阅读的时间轴记录，方便月末进行阅读复盘与重点文章重访。

**技术选型与竞品分析素材收集**：在进行框架选型或中间件对比时，可将各候选方案的官方网站、性能评测报告、社区讨论帖、迁移案例等链接归档至同一批次下，通过标签区分不同对比维度，形成结构化的选型依据文档。

## 快速开始

以下指令演示如何从 GitHub 克隆本项目、安装依赖并启动本地开发服务。

```bash
git clone https://github.com/techlink-archive/techlink-core.git
cd techlink-core
npm install
npm run dev
```

执行完成后，访问控制台输出的本地地址（通常为 http://localhost:3000）即可进入条目管理界面。首次启动会自动创建示例条目与默认标签分类。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 nvm 管理版本 |
| npm | 9.x 或更高 | 包管理器，用于安装项目依赖 |
| SQLite3 | 系统自带或由 better-sqlite3 捆绑 | 默认嵌入式数据库，无需额外安装 |
| Git | 2.30 或更高 | 用于克隆仓库与版本管理 |
| curl / wget | 任意稳定版本 | 用于快照状态校验脚本的 HTTP 探测 |
| 操作系统 | Linux / macOS / Windows WSL2 | 开发与生产环境均以 Unix-like 系统为优先支持 |

## 项目结构

项目采用模块化分层设计，核心代码与配置分离，便于独立维护与扩展。

```
techlink-core/
├── src/                           # 源代码主目录
│   ├── core/                      # 核心业务逻辑模块
│   │   ├── entry-manager.js       # 条目增删改查与校验逻辑
```

## 常见问题

问：项目启动后控制台提示数据库连接失败，应如何处理？

答：请检查 data 目录是否存在且具有写入权限。首次启动时程序会自动创建 techlink.db 文件，若目录不存在或权限不足则会导致连接失败。可手动创建 data 目录并赋予 755 权限。若使用自定义数据库路径，请确认 custom.yaml 中的 path 字段指向正确。

问：快照校验功能显示大量链接为不可用状态，是否意味着这些资源已被删除？

答：快照校验仅返回目标服务器的 HTTP 状态码。状态码为 4xx 或 5xx 时标记为异常。但部分站点可能启用反爬机制或临时限流，导致校验请求被拒绝。建议在非高峰期重新运行校验脚本，并适当增加请求超时时间。若连续多次校验均失败，则可确认链接失效，建议移除或更新对应条目。

问：如何将现有浏览器书签批量导入本系统？

答：本项目未直接集成浏览器书签解析器，但提供了通用的 CSV 与 JSON 导入接口。用户可将浏览器书签导出为 HTML 文件，再使用社区提供的转换工具将 HTML 书签转换为包含 URL 与标题的 CSV 文件，最后通过命令行工具的 import-csv 子命令完成批量导入。具体转换工具与导入命令请参考 /docs/user-guide/batch-import.md 章节。
