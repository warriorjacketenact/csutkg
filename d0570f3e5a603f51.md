# LinkVault Resource Aggregator

LinkVault is a high-performance, stateless resource aggregation and navigation system designed for technical researchers, open-source contributors, and knowledge workers who need to organize, categorize, and rapidly access large volumes of distributed web resources. The project addresses the fundamental challenge of managing hundreds of unstructured external links by providing a lightweight, queryable metadata layer that transforms raw URL collections into browsable, searchable, and maintainable knowledge bases.

Target users include DevOps engineers maintaining internal documentation portals, academic researchers tracking preprint repositories and data sources, open-source maintainers curating community resource lists, and technical writers building comprehensive reference hubs. LinkVault does not host or proxy external content but provides deterministic indexing, tag-based classification, availability monitoring, and structured output generation for static site deployment or integration with existing documentation pipelines.

## 功能概览

**Bulk URL Ingestion** - Import hundreds of resource links from plain text files, CSV exports, or direct clipboard input with automatic deduplication and protocol normalization.

**Hierarchical Tagging System** - Assign multiple tags per resource with support for namespace collision resolution and inheritance-based category propagation across nested tag groups.

**Availability Health Checks** - Schedule configurable HEAD request probes to detect broken links, certificate expiry, or redirect chains, with alert hooks for Slack and email.

**Static Site Generator Integration** - Export categorized resource lists as Markdown tables, HTML definition lists, or JSON feeds compatible with Hugo, Jekyll, and VuePress.

**Search and Filter DSL** - Query resources using a domain-specific language supporting prefix matching, exclusion filters, and boolean combinations against URL components and tag sets.

**CLI and REST API Dual Interface** - Operate the system via interactive terminal commands or programmatic HTTP endpoints with OpenAPI v3 specification for client generation.

**Versioned Snapshot History** - Maintain immutable revisions of the entire resource index with timestamped rollback points and diff visualization between revisions.

## 应用场景

Technical Documentation Portal Enrichment - Documentation teams maintaining large-scale developer portals can use LinkVault to manage external references, API specification links, and community tutorial collections, ensuring all outbound references remain current through automated health checks.

Academic Literature Curation - Research groups compiling systematic literature reviews can aggregate preprint servers, dataset repositories, and supplementary material URLs into a single indexed corpus, with tag-based filtering by publication year, methodology, or research domain.

Open-Source Community Resource Hub - Project maintainers can build and maintain a community-driven resource page that accepts pull requests for new links, validates URL accessibility, and automatically regenerates the public-facing navigation table on each commit.

Internal Enterprise Knowledge Base - Organizations can centralize links to internal tools, dashboards, runbooks, and policy documents, applying access control tags and generating department-specific views without duplicating the underlying URL data.

## 快速开始

```bash
# Clone the repository from the official source
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# Install dependencies using the included setup script
./scripts/setup.sh
# or manually with pip for Python-based deployment
pip install -r requirements.txt

# Run the initial import and generate the default resource index
linkvault import --input resources.txt --format plain
linkvault build --output ./public --template default
linkvault serve --port 8080
```

## 项目结构

```
linkvault-core/
├── cmd/                              # 命令行入口点与 CLI 调度器
│   ├── linkvault/                    # 主二进制程序入口 (main.go)
│   │   ├── main.go                   # 初始化根命令、全局标志和子命令注册
│   │   └── version.go                # 编译时注入的版本号和构建哈希
│   └── internal/                     # CLI 内部命令实现（不对外暴露）
│       ├── import.go                 # 从文件、stdin 或 URL 批量导入资源
│       ├── build.go                  # 触发静态站点生成管道
│       ├── serve.go                  # 启动开发服务器（含热重载）
│       └── health.go                 # 手动触发健康检查并输出报告
├── pkg/                              # 可被外部导入的公共库代码
│   ├── indexer/                      # 资源索引引擎核心
│   │   ├── parser.go                 # 解析不同输入格式（plain, csv, jsonl）
│   │   ├── deduper.go                # 基于规范化 URL 的重复检测与合并
│   │   └── resolver.go               # 处理相对路径、协议补全和片段处理
│   ├── tagger/                       # 标签系统的实现
│   │   ├── namespace.go              # 标签命名空间管理与冲突解决
│   │   ├── inheritance.go            # 父标签到子标签的继承传播逻辑
│   │   └── query.go                  # 标签查询 DSL 的解析与执行
│   ├── probe/                        # 资源可用性探测模块
│   │   ├── checker.go                # 并发 HTTP 探测工作池
│   │   ├── cert.go                   # TLS 证书有效期与信任链检查
│   │   └── reporter.go               # 探测结果的结构化输出（JSON, Prometheus）
│   ├── storage/                      # 数据库抽象层与迁移管理
│   │   ├── sqlite.go                 # SQLite 驱动实现及连接池配置
│   │   ├── schema.go                 # 数据库表定义与版本迁移脚本
│   │   └── snapshot.go               # 快照创建、恢复和差异比较
│   └── export/                       # 导出器集合：支持多种输出格式
│       ├── markdown.go               # 生成 Markdown 表格和列表
│       ├── html.go                   # 生成独立 HTML 页面（含搜索框）
```

## 常见问题

问题：导入大量 URL 时出现内存不足错误，应如何处理？

回答：LinkVault 默认将全部资源加载至内存进行去重和标签解析。对于超过 50000 条记录的数据集，建议使用 --stream 标志启用流式处理模式，该模式使用磁盘临时存储和分批次提交事务。同时可以调整 SQLite 的缓存大小参数（--cache-size-mb 选项）以优化内存使用。如果使用 Docker 部署，请增加容器的内存限制并配置交换空间。

问题：健康检查报告大量超时，但手动访问这些 URL 时可以正常打开，原因是什么？

回答：默认的健康检查超时时间为 3 秒，且 User-Agent 设置为 LinkVault/1.0。部分服务端会对非浏览器 User-Agent 进行限流或延迟响应。解决方法是修改 probes.yaml 配置文件中的 timeout 字段至 10 秒，并设置 custom-user-agent 为常见的浏览器标识符。对于需要 JavaScript 渲染的页面，请启用 --enable-headless 选项（需要安装 Playwright 依赖），该选项使用无头浏览器执行完整页面加载后再判定可用性。

问题：如何将现有的静态 HTML 资源列表迁移到 LinkVault 管理？

回答：LinkVault 提供 legacy-import 子命令，支持解析常见格式的 HTML 列表结构。指定 --selector 参数以匹配链接容器元素（例如 ul.resources li a），并配置 --attribute 提取 URL 值（默认为 href）。对于嵌套结构，可以使用 --depth 控制解析层级。导入后建议运行 linkvault validate 命令检查所有提取的 URL 是否格式合法，再执行 linkvault tag --auto 基于 URL 路径自动生成初步标签建议，最后人工调整标签树。
