# LinkPilot Resource Aggregator

LinkPilot is a curated technical resource index and external link management system designed for developers, researchers, and technical content curators who need to organize, validate, and distribute large volumes of reference materials across distributed knowledge bases. The project addresses the fundamental challenge of link rot, broken references, and disorganized bookmark collections by providing a structured metadata layer over raw URL inventories.

The system operates as a static site generator and API service that consumes plain-text URL lists, enriches each entry with fetch status, content-type detection, and optional semantic tagging, then produces browseable HTML indices, JSON feeds, and Markdown catalogues. LinkPilot is built for maintainers of technical documentation hubs, open-source learning repositories, and internal developer portals who routinely handle link lists exceeding several hundred entries per batch.

## 功能概览

**批量导入与归一化** - Accepts raw URL lists in plain text, CSV, or JSON lines format, normalizes protocol variants, strips tracking parameters, and deduplicates entries with configurable collision resolution.

**存活检测与状态标注** - Performs asynchronous HEAD and GET requests with configurable timeouts and user-agent rotation to verify resource accessibility, caching status codes and response timings.

**内容类型识别** - Inspects Content-Type headers and file extensions to classify resources as HTML documents, PDF files, images, video streams, API endpoints, or generic binary assets.

**元数据提取** - For HTML resources, extracts title, description, and Open Graph tags; for PDFs, extracts page count and author metadata when available.

**模板化输出** - Generates static HTML catalogues, Markdown inventory tables, JSON API responses, and RSS feeds for change notification, with customizable Handlebars or Jinja templates.

**定时巡检与变更报告** - Supports scheduled re-validation of all tracked links with delta reporting, flagging newly dead URLs, changed content types, or redirect chain alterations.

**标签系统与全文检索** - Allows manual or auto-tagging based on domain patterns and keyword heuristics, with integrated search across titles, descriptions, and tags using a lightweight inverted index.

## 应用场景

**技术文档库维护** - Documentation teams managing thousand-page developer portals use LinkPilot to periodically audit external references, ensuring that all cited RFCs, specification documents, and third-party library homepages remain accessible. The system generates a health dashboard that highlights stale or moved resources before they reach end-users.

**开源学习资源汇总** - Community-driven learning repositories aggregate course materials, video lectures, and interactive tutorials from across the web. LinkPilot provides a structured inventory layer that transforms disparate link collections into searchable, filterable resource catalogues with availability indicators, reducing the maintenance burden on volunteer curators.

**企业知识库外链治理** - Internal knowledge bases frequently accumulate broken external references as domain names change hands and hosting services shut down. LinkPilot integrates with existing wiki platforms via webhook notifications, pushing link status updates directly into confluence pages or markdown files, enabling proactive repair workflows.

**学术参考文献验证** - Research groups compiling literature reviews or systematic meta-analyses rely on stable citations to preprint servers, journal landing pages, and data repositories. LinkPilot's batch validation feature produces compliance reports suitable for supplementary materials sections, documenting the last-verified timestamp for every external resource.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/example/linkpilot.git
cd linkpilot

# Install dependencies using pipenv or virtualenv
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt

# Run the ingestion pipeline with a sample URL list
python pilot.py ingest --input ./data/urls_330.txt --output ./catalogue/ --format html
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9+ | 是 | 核心运行时，所有主要功能依赖该版本或更高版本 |
| pip 21.0+ | 是 | Python 包管理工具，用于安装 requirements.txt 中列出的依赖项 |
| aiohttp 3.8+ | 是 | 异步 HTTP 客户端，用于高并发链接状态检测与内容获取 |
| beautifulsoup4 4.11+ | 是 | HTML 解析库，用于提取页面标题、元标签和 Open Graph 数据 |

| redis 6.0+ | 否 | 可选缓存后端，用于存储检测结果以减少重复请求 |
| nginx 1.18+ | 否 | 生产环境静态文件服务推荐，非开发环境必需 |
| docker 20.10+ | 否 | 容器化部署支持，用于快速启动完整工具链 |

## 项目结构

```
linkpilot/
├── pilot.py                         # 主入口，命令分发器，包含 ingest/validate/serve 子命令
├── requirements.txt                 # 生产依赖锁定，aiohttp, bs4, jinja2, pyyaml
├── setup.py                         # 打包配置，定义 console_scripts 入口点
├── .env.example                     # 环境变量模板，配置 REDIS_URL, TIMEOUT, USER_AGENT
├── src/
│   ├── core/                        # 核心业务逻辑
│   │   ├── ingester.py              # 读取与归一化，支持 .txt, .jsonl, .csv
│   │   ├── validator.py             # 异步 HTTP 验证器，重试与超时策略
│   │   ├── metadata.py              # HTML/PDF 元数据提取器，使用 BeautifulSoup
│   │   └── registry.py              # 内存+磁盘链接注册表，支持增量更新
│   ├── output/                      # 输出渲染器
│   │   ├── html_renderer.py         # 生成响应式 HTML 目录，基于 Jinja2
│   │   ├── markdown_renderer.py     # 输出 Markdown 表格与列表，用于文档集成
│   │   └── json_exporter.py         # JSON 流式导出，适用于 API 消费
│   ├── scheduler/                   # 定时任务模块
│   │   ├── cron_parser.py           # 解析 crontab 表达式，计算下次执行时间
│   │   └── runner.py                # 后台巡检调度器，集成 apscheduler
│   ├── search/                      # 轻量级全文检索
│   │   ├── indexer.py               # 构建倒排索引，基于 title/tag/domain
│   │   └── query.py                 # 布尔查询解析器，支持 AND/OR 组合
│   └── utils/                       # 通用工具函数
│       ├── network.py               # 网络辅助，代理检测、DNS 缓存
│       └── logging.py               # 结构化日志配置，JSON 格式输出
├── config/
│   ├── default.yaml                 # 默认配置，超时 10s，并发数 50
│   └── production.yaml              # 生产覆盖，并发数 200，启用 redis 缓存
├── templates/
│   ├── catalogue.html               # HTML 主模板，含状态徽章与搜索框
│   └── report.md.j2                 # Markdown 报告模板，用于批次摘要
├── static/                          # 静态资源，CSS 与 JavaScript 前端
│   ├── style.css                    # 响应式设计，适配桌面与移动端
│   └── app.js                       # 前端交互，排序与过滤逻辑
├── data/
│   ├── batches/                     # 输入批次存储，按批次号命名
```

## 常见问题

**问：验证大量链接时如何避免被目标服务器屏蔽？**

答：LinkPilot 支持配置 user-agent 轮换池、请求间隔延迟（per-host delay）以及随机化抖动。生产环境建议设置 concurrency=10，per_host_delay=0.5 秒，并启用 redis 缓存以避免重复验证同一域名下的多个路径。对于敏感目标，可在配置中排除特定域名或设置最大请求数阈值。

**问：如何处理需要 JavaScript 渲染才能获取真实内容的页面？**

答：LinkPilot 默认只进行静态 HEAD/GET 请求，不执行 JavaScript。对于严重依赖客户端渲染的 SPA 应用，建议在 metadata 提取阶段使用可选的 Playwright 或 Puppeteer 集成，该功能通过 extra 依赖安装。启用后框架会为配置的域名列表自动切换到无头浏览器模式，但会显著增加验证耗时与资源消耗。

**问：输出的 HTML 目录能否嵌入到现有的静态站点中？**

答：可以。html_renderer 支持生成独立的 HTML 文件，所有 CSS 与 JavaScript 均内联或使用相对路径引用。您可以将输出文件放置在任意 Web 服务器的静态目录下。若需要与 Jekyll、Hugo 等静态站点生成器集成，可改用 markdown_renderer 输出 .md 文件，并将其放入站点的 content 目录。
