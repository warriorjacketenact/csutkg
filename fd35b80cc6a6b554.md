# WebResource Navigator

WebResource Navigator 是一个面向技术研究人员、数据分析师和内容聚合者的高效外链资源整理与导航工具。该项目定位于将分散在网络各处的专题文章、技术笔记和系列教程进行统一抓取、分类索引与结构化展示，解决个人收藏夹混乱、跨站资源检索困难以及知识碎片化的问题。

本项目不生产原始内容，而是通过元数据提取与标签化组织，为用户提供一套轻量级、可自部署的资源目录系统。适用于需要长期跟踪特定领域技术动态、维护团队知识库或构建个人学习路线的场景。

## 功能概览

**自动化资源抓取**：基于用户提供的 URL 列表，递归获取页面标题、描述与发布时间等元数据，支持多层级系列文章识别。

**智能分类索引**：根据 URL 路径特征与页面关键词，自动将资源归入预设或自定义分类树，支持多标签重叠。

**全文检索支持**：内置轻量级倒排索引，允许用户按标题、正文片段或标签组合快速定位目标资源。

**系列文章关联**：自动识别同系列链接（如路径包含 /series/），构建前后文导航与阅读进度追踪。

**多格式导出**：支持将整理后的资源目录导出为 Markdown、JSON 或 CSV 格式，便于导入其他知识管理工具。

**静态站点生成**：提供内置模板引擎，可将资源列表渲染为可直接部署的静态 HTML 文档，方便团队内部分享。

**访问频率监控**：记录每个外链的点击次数与最后访问时间，辅助用户识别高价值内容。

**定期更新检查**：通过可配置的定时任务，重新抓取已收录 URL 并检测内容变更，保持资源新鲜度。

## 应用场景

**技术团队内部知识库维护**：开发团队可将项目依赖的第三方文档、技术博客和 API 参考链接统一纳入 WebResource Navigator，新成员入职时可通过该工具快速了解团队常用技术栈与学习路径。

**个人研究主题追踪**：研究人员在开展文献调研时，可将大量候选 URL 导入系统，利用分类和检索功能筛选核心文献，并通过系列关联功能理清专题发展脉络。

**内容运营编辑辅助**：内容聚合网站的编辑可使用本工具对投稿来源、参考链接进行去重和归类，快速生成每周推荐列表或专题合集。

**离线阅读环境准备**：在网络受限的环境中，用户可借助本工具的导出功能，将选定资源列表转换为结构化文档，配合其他离线浏览器进行批量阅读。

## 快速开始

以下命令演示了从克隆仓库到启动本地服务的完整流程。

```bash
git clone https://github.com/example/webresource-navigator.git
cd webresource-navigator
pip install -r requirements.txt
python app.py --init --source urls.txt --output ./dist
```

其中 urls.txt 为包含目标 URL 列表的文本文件，每行一个链接。执行完毕后，可通过浏览器访问 http://127.0.0.1:5000 查看生成的资源导航界面。

## 项目结构

```
webresource-navigator/
├── app.py                  # 命令行入口与 Web 服务启动脚本
├── requirements.txt        # Python 依赖声明文件
├── config.yaml             # 全局配置文件，含分类规则与抓取参数
├── docs/                   # 文档目录，含入门指南与 API 参考
│   ├── getting_started.md
│   ├── configuration.md
│   ├── api_reference.md
│   └── deployment.md
├── core/                   # 核心逻辑模块
│   ├── fetcher.py          # 发送 HTTP 请求，处理重定向与超时重试
│   ├── parser.py           # 基于 beautifulsoup4 的 HTML 元数据提取
│   ├── indexer.py          # 构建倒排索引与标签管理系统
│   └── scheduler.py        # 定时任务调度，执行定期更新检查
├── web/                    # Web 界面相关资源
│   ├── static/             # 静态资源目录，含 CSS 与 JavaScript
│   │   ├── style.css
│   │   └── app.js
│   └── templates/          # Flask 模板文件，用于渲染导航页面
│       └── index.html
├── data/                   # 数据存储目录
│   ├── urls.txt            # 默认初始 URL 列表文件
│   ├── index.db            # SQLite 数据库，存储资源元数据与索引
│   └── cache/              # 页面缓存目录，减少重复抓取
├── scripts/                # 辅助工具脚本
│   ├── import_urls.py      # 从外部文件批量导入 URL
│   └── export_json.py      # 导出当前索引为 JSON 格式
└── tests/                  # 单元测试目录
    ├── test_fetcher.py
    ├── test_parser.py
    └── test_indexer.py
```

## 贡献指南

本项目欢迎各类贡献，包括但不限于新增功能、修复缺陷、完善文档和补充测试用例。

提交代码前请先阅读 docs/ 目录下的开发规范文档，并确保所有现有单元测试通过。新功能需附带对应的测试用例。

使用 GitHub Issues 报告缺陷或提出功能请求，描述时请包含复现步骤、预期行为与实际行为，并附上相关日志或截图。

提交 Pull Request 时，请确保分支基于最新的 main 分支，且提交信息遵循常规提交格式（type: subject），便于自动生成变更日志。

对于文档类贡献，可直接在 docs/ 目录下修改 Markdown 文件并提交 PR，无需额外运行测试。

## 常见问题

**问：项目支持 HTTPS 代理或内网环境访问吗？**

答：支持。可通过 config.yaml 中的 proxy 字段配置 HTTP 或 HTTPS 代理地址。对于内网环境，可在 fetcher.py 中自定义证书验证策略或关闭 SSL 验证（不推荐在生产环境中使用）。

**问：如何处理目标页面内容为动态渲染（JavaScript 依赖）的情况？**

答：本工具默认使用 requests 获取静态 HTML，对于需要 JavaScript 渲染的页面，建议结合 playwright 或 selenium 驱动无头浏览器进行抓取。项目 core/fetcher.py 中预留了可替换的抓取后端接口，用户可自行实现并注册。

**问：索引数据库是否可以迁移至其他关系型数据库？**

答：当前默认使用 SQLite 便于单机部署。若需迁移至 PostgreSQL 或 MySQL，可修改 core/indexer.py 中的数据库连接适配器，并调整对应的表结构定义。项目未对数据库层做强绑定，理论上支持任何符合 DB-API 2.0 规范的数据库驱动。
