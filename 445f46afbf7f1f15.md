# ResourceBridge 资源聚合导航系统

ResourceBridge 是一个面向技术研究、学术资料整理与知识管理场景的高质量外链资源聚合与导航系统。本项目定位为技术资源的中转枢纽，通过对分散在网络各处的专题节目、文档资料和结构化信息进行集中索引与分类呈现，帮助研究人员、内容策展人和技术文档工程师在信息过载的环境中快速定位高价值内容。

本项目解决的核心问题在于：大量有深度的专题页面缺乏有效的发现机制，用户往往需要依赖搜索引擎进行多次尝试性检索，效率低下且容易遗漏关键资料。ResourceBridge 通过建立统一的资源索引层，将特定领域的节目单、议程安排、专题文档等内容进行结构化收录，并提供清晰的项目内导航与分类浏览能力。

## 功能概览

**批量资源索引**：系统提供基于固定根路径的资源映射机制，能够将大量分散的节目页面通过短标识符进行统一管理和快速访问。

**分类目录导航**：所有收录资源按照内容主题和文件类型进行逻辑分组，用户可通过分类浏览快速定位感兴趣的内容板块。

**全文元数据检索**：每个资源条目均附带标题、类型、最后更新时间和内容摘要等元数据字段，支持基于关键词的快速筛选与排序。

**外部链接直出**：项目核心能力在于将外部资源链接原样呈现，不进行任何改写、重定向或中间页跳转，确保访问路径的透明性与可追溯性。

**资源状态监控**：内置链接可用性检查机制，定期扫描已收录资源的访问状态，并在项目状态页面中标注异常链接。

**批量导入与导出**：支持通过结构化数据文件批量新增资源条目，也支持将当前索引库导出为 JSON 或 CSV 格式，方便与其他工具集成。

**自定义标签系统**：用户可为每个资源条目添加自定义标签，实现跨分类的个性化内容组织。

## 应用场景

学术资料整理与文献索引：研究人员在开展文献综述或专题研究时，可将大量分散的在线讲座、研讨会节目单和会议议程通过 ResourceBridge 建立集中索引，配合标签系统实现按主题、时间或演讲者的多维度筛选。

技术团队知识库建设：软件开发团队在维护内部技术文档时，可将项目依赖的第三方参考资料、培训视频链接和架构设计讨论记录通过本系统统一管理，降低新成员的知识获取门槛。

内容策展与 Newsletter 生产：内容编辑和策展人在策划技术周报或专题通讯时，可利用本项目的批量导入导出功能快速汇总候选链接，并在分类导航的辅助下完成内容筛选与编排。

个人书签系统升级：长期在多个技术社区和学术平台积累了大量书签的开发者，可通过 ResourceBridge 将浏览器书签导出文件批量导入，获得更强大的分类检索和状态监控能力。

## 快速开始

以下命令序列演示了从代码仓库克隆、安装项目依赖到启动本地开发服务器的完整过程。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行上述命令后，开发服务器默认监听 3000 端口。用户可通过浏览器访问 http://localhost:3000 查看资源索引首页。生产环境部署请参考 `docs/deployment.md` 文档。

## 安装要求

本项目基于 Node.js 运行时构建，依赖 npm 包管理工具。以下为完整的环境依赖清单：

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Node.js | 18.17.0 或更高 | 项目运行时基础环境，推荐使用 LTS 版本 |
| npm | 9.0.0 或更高 | 包依赖管理与脚本执行工具 |
| PostgreSQL | 14.0 或更高 | 主要数据存储，用于资源条目、标签和用户数据持久化 |
| Redis | 6.0 或更高 | 缓存层与会话存储，提升检索响应速度 |
| Git | 2.30.0 或更高 | 代码版本控制与仓库克隆 |
| PM2 | 5.0.0 或更高 | 生产环境进程守护与负载均衡（生产部署必需） |
| Nginx | 1.20.0 或更高 | 反向代理与静态资源服务（生产部署推荐） |

## 项目结构

```
resourcebridge/
├── src/                                 # 源代码主目录
│   ├── core/                           # 核心模块：资源索引引擎与数据模型
│   │   ├── indexer.js                  # 资源索引器，负责解析和存储资源条目
│   │   ├── resolver.js                 # URL 解析器，处理根路径与短标识符映射
```

## 贡献指南

本社区遵循贡献者公约 2.1 版。所有贡献者需在提交代码前签署个人贡献者许可协议。具体参与流程如下：

1. 查阅 `docs/developer-guide/` 目录下的开发指南，了解项目技术栈、代码风格规范以及本地开发环境的完整配置步骤。建议在开发前运行 `npm run setup:dev` 命令完成环境初始化。

2. 在 Issue 列表中选择标注为 `help-wanted` 或 `good-first-issue` 标签的任务，或在讨论区提出新的功能建议并获得核心维护者的反馈确认后再开始工作，避免无效开发。

3. 从 `main` 分支创建功能分支，遵循 `feature/描述` 或 `fix/描述` 的命名模式。提交信息需符合约定式提交规范，格式为 `<type>(<scope>): <subject>`，其中 type 包括 feat、fix、docs、style、refactor、perf、test、chore 等。

4. 完成代码变更后，确保所有单元测试和集成测试通过，并为新增功能补充对应的测试用例。提交 Pull Request 时需填写完整模板，包含变更摘要、测试覆盖说明以及相关 Issue 编号。

5. Pull Request 需至少获得一位核心维护者的批准，并通过持续集成流水线的所有检查后方可合并。合并后将自动触发文档站点和演示环境的重新部署。

## 常见问题

问：项目启动后无法连接到 PostgreSQL 数据库，应如何排查？

答：首先检查 `.env` 文件中的 `DATABASE_URL` 配置是否正确，确保主机地址、端口、用户名和密码与数据库实际设置一致。其次确认 PostgreSQL 服务是否已启动，可通过 `pg_isready` 命令检测。若使用 Docker Compose 环境，请确保所有容器均已正常运行。最后检查数据库是否已完成迁移，可执行 `npm run migrate:latest` 命令应用所有待执行的迁移脚本。

问：资源列表中的外部链接出现访问超时或返回 404 状态码，系统会如何处理？

答：系统的监控服务会按可配置的时间间隔（默认每 24 小时）对已收录的资源链接进行批量探测。检测到异常状态的链接会在管理面板中标红显示，并记录最后一次成功访问的时间戳。用户可手动触发重新检测，也可通过 API 导出异常链接清单。系统不会自动删除异常链接，但会将其在搜索结果中的排序权重下调。

问：如何将现有浏览器书签批量导入 ResourceBridge？

答：主流浏览器（Chrome、Firefox、Edge）支持将书签导出为 HTML 格式文件。系统 CLI 工具提供了 `npm run import:bookmarks -- --file=bookmarks.html` 命令，可解析该格式并自动提取标题和 URL。导入过程中系统会尝试去重，若检测到已存在的 URL 则会跳过并记录日志。对于大型书签集合，建议分批导入以避免内存压力。
