# ResourceBridge

ResourceBridge 是一个面向开发者和技术研究人员的开源外链资源聚合与导航系统。项目定位为高质量技术资源的统一入口，通过结构化分类与可检索的索引机制，帮助用户从海量分散的网络链接中快速定位具备实际参考价值的文档、工具、教程与参考实现。项目本身不存储任何第三方内容，仅提供链接的整理、标签化描述与可用性监测，适用于个人知识库构建、团队技术选型参照以及开源项目依赖调研等场景。

ResourceBridge 的核心目标用户包括：需要频繁查阅外部技术资料的后端工程师、运维工程师与技术架构师；负责技术栈调研与选型的团队负责人；以及希望系统化积累学习路径的进阶开发者。项目采用静态站点生成方案，所有链接数据以结构化标记存储，支持通过脚本批量更新状态、生成索引页，并可搭配定时任务实现链接存活性检查。

## 功能概览

- 链接分类索引：支持按技术领域、资源类型、适用层级等维度对收录链接进行多级分类，便于按场景浏览。

- 状态监测看板：内置链接可用性检测模块，可标记失效链接，生成健康度报告，辅助维护者及时清理或更新。

- 全文检索支持：基于标题与描述字段提供轻量级关键词检索，帮助用户在大量链接中快速定位相关内容。

- 标签体系与筛选：每条链接可附加多个自定义标签，支持按标签组合筛选，适应复杂查询需求。

- 导入导出扩展：提供 JSON 与 CSV 格式的数据导入导出接口，便于与其他知识管理工具或爬虫系统集成。

- 静态页面生成：无需后端服务即可构建完整导航页面，生成的 HTML 结构适合部署在任意 Web 服务器或对象存储上。

- 自定义元数据扩展：允许为每条链接添加备注、优先级、推荐指数、收录日期等元数据字段，满足个性化管理需要。

## 应用场景

- 技术团队内部知识库建设：技术负责人可将项目部署为团队共享的资源导航页，统一存放常用开发文档、运维手册与内部工具地址，新成员入职时可快速了解团队技术生态。

- 开源项目依赖调研与参考：开发者在引入新依赖或评估技术方案时，可通过项目收录的外部链接快速获取相关技术博客、性能测试报告与社区讨论汇总，缩短调研周期。

- 个人学习路径管理：学习者可将分散在各个平台的教学视频、官方文档、代码示例与问题讨论链接集中归档，形成个人专属的学习资源索引，并按学习阶段或难度进行标签化管理。

- 技术社区资源共建：社区维护者可利用项目的分类与标签机制，组织社区成员共同整理某一技术领域的优质外链，形成公开可访问的垂直资源导航站。

## 快速开始

以下命令适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装依赖
pip install -r requirements.txt

# 初始化数据目录并生成静态站点
python bridge.py init
python bridge.py build --output ./dist
```

完成上述三步后，`./dist` 目录下将生成完整的静态导航页面，您可以使用任意 HTTP 服务器进行预览：

```bash
python -m http.server 8000 --directory ./dist
```

如需进行链接状态检测，可执行：

```bash
python bridge.py check --timeout 3 --report ./reports/health.json
```

## 项目结构

```
resourcebridge/
├── bridge.py                 # 主控制台入口，整合 init、build、check 等命令
├── config.yaml               # 站点全局配置，包括标题、分类映射、检测参数
├── requirements.txt          # Python 依赖列表
├── README.md                 # 项目说明文档（本文件）
├── LICENSE                   # MIT 许可证文件
│
├── data/                     # 链接数据存储目录
│   ├── links/                # 每条链接以独立 YAML 文件存放
│   │   ├── tech-001.yaml     # 单条链接数据：标题、URL、标签、备注、收录日期
│   │   └── ...
│   ├── categories.yaml       # 分类层级定义
│   └── tags.yaml             # 标签库及同义词映射
│
├── src/                      # 核心逻辑模块
│   ├── __init__.py
│   ├── loader.py             # 数据加载与解析，支持 YAML 与 JSON
```

## 贡献指南

欢迎社区贡献者参与 ResourceBridge 的开发与维护。请遵循以下流程提交贡献：

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境。建议在 dev 分支上进行所有修改，保持主分支与上游同步。

## 常见问题

Q: 项目是否存储链接指向的原始内容或快照？

A: 不存储。ResourceBridge 仅保留链接地址、标题、标签和描述等元数据，所有原始内容仍由源站点提供。用户访问链接时直接跳转至第三方页面，项目本身不承担内容审查与版权责任。

Q: 如何批量导入已有链接数据？

A: 支持通过 `python bridge.py import --format csv --path ./data/import.csv` 命令批量导入。CSV 文件需包含 title、url、tags、description 列。导入前建议使用 `--dry-run` 参数预览解析结果，确认无误后再执行正式导入。

Q: 链接健康度检测的访问频率是否有限制？

A: 检测模块默认采用单线程顺序请求，间隔 0.5 秒，以避免对目标站点造成压力。用户可通过 `--concurrency` 参数调整并发数，但建议谨慎使用。检测超时默认为 3 秒，可依据网络环境通过 `--timeout` 调整。
