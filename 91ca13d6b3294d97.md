# Resource Index Project

Resource Index Project 是一个面向技术研究者、开发者和内容策展人的轻量级外链资源汇总系统。该项目定位于构建可维护的、结构化的外部知识索引，通过集中管理分散于网络各处的技术文章、教程、案例与参考资料，帮助用户降低信息检索成本，提升知识发现效率。

本项目并非一个内容生产平台，而是一个严格的资源映射与导航工具。它假设用户已经拥有或正在积累大量分散的 URL 资源，并需要一套标准化的手段对其进行分类、标注、检索与呈现。Resource Index Project 提供开箱即用的目录结构、元数据模板和静态站点生成逻辑，使得 250 条乃至更大规模的外链集合能够被清晰管理，并最终输出为可供团队共享或公开访问的文档站点。

## 功能概览

- **批量资源导入**：支持通过脚本从 CSV、JSON 或纯文本列表批量导入 URL，自动解析并生成对应的 Markdown 条目文件。
- **多维度分类体系**：允许用户为每个资源标记主题类别、技术栈标签、难度等级和阅读时长，形成多维度的检索索引。
- **自动元数据抽取**：对于支持的平台，系统可尝试自动抓取页面标题、描述和发布日期，减少手动录入负担。
- **状态追踪管理**：内置资源状态字段，包括待读、已读、已归档、失效链接等，便于长期维护外链库的健康度。
- **静态站点生成**：项目内置模板引擎，可将所有资源条目渲染为静态 HTML 站点，支持全文搜索、标签过滤和时间排序。
- **周期报告输出**：支持按周或按月生成新增资源列表和失效链接报告，便于团队同步知识库更新动态。
- **标签云与关联推荐**：基于标签共现关系，自动生成标签云和相似资源推荐，提升资源之间的可发现性。
- **多格式导出**：支持将整个资源索引导出为纯 Markdown 文件包、JSON 数据包或 CSV 表格，便于迁移或二次加工。

## 应用场景

- **技术团队内部知识库维护**：研发团队可将日常遇到的优秀技术博客、解决方案文章、调试笔记等外链统一收录至本系统，按项目或技术领域分类，新成员入职时可快速浏览团队积累的优质资料。
- **个人学习路径管理**：自学者可将不同课程、文档、练习项目等外链按学习阶段组织，设置阅读状态和优先级，避免在大量浏览器书签中迷失，形成可追溯的学习轨迹。
- **开源社区文档导航**：开源项目维护者可使用本系统整理生态周边的教程、插件列表、迁移案例和常见问题解答链接，作为项目官方文档的外延补充，减轻主仓库文档的臃肿压力。
- **技术资讯聚合与筛选**：内容策展人可定期批量添加技术新闻、版本发布公告、安全预警等时效性资源，利用状态追踪功能标记已读和重要程度，快速筛选出值得深度阅读的内容。

## 快速开始

以下步骤帮助您在本地环境中快速启动 Resource Index Project。

```bash
# 克隆项目仓库
git clone https://github.com/resource-index-project/resource-index.git
cd resource-index

# 安装依赖（基于 Node.js 环境）
npm install

# 初始化资源数据库并导入示例数据
npm run init-db
npm run import-examples

# 启动本地开发服务器
npm run dev
```

执行完毕后，打开浏览器访问 http://localhost:3000 即可查看资源索引的本地预览。导入示例数据后，您可以在 `data/raw` 目录下看到若干 Markdown 示例文件，参照其格式即可开始添加自己的外链资源。

## 项目结构

```
resource-index/
├── bin/                                 # 命令行工具入口
│   ├── cli.js                           # 主 CLI 脚本，注册所有子命令
│   └── commands/                        # 子命令实现
│       ├── import.js                    # 批量导入命令
│       ├── export.js                    # 多格式导出命令
│       ├── check.js                     # 链接有效性检查命令
│       └── build.js                     # 静态站点构建命令
│
├── src/                                 # 核心源码目录
│   ├── core/                            # 核心逻辑模块
│   │   ├── database.js                  # SQLite 数据库连接与 ORM 映射
│   │   ├── resource.js                  # 资源增删改查业务逻辑
│   │   ├── tag.js                       # 标签管理系统
│   │   └── fetcher.js                   # 自动元数据抓取实现
```

## 常见问题

**问：导入大量 URL 时如何避免重复条目？**

系统在导入过程中会自动检查 URL 的标准化形式（去除末尾斜杠、统一协议大小写等），并与数据库现有记录进行比对。若发现重复，导入脚本会跳过该条目并记录警告日志，您可以在日志文件中查看所有被跳过的重复 URL 列表。此外，您也可以启用 `--strict` 模式，让导入过程在遇到第一个重复条目时立即终止并报错。

**问：自动元数据抓取失败时如何处理？**

自动抓取依赖目标网站的结构化数据支持，部分网站可能因反爬机制、动态渲染或无 meta 标签导致抓取失败。对于这些情况，系统会回退到使用 URL 中的路径片段作为标题占位符，并在资源条目中标记 `meta_status: failed`。您可以通过编辑对应的 Markdown 文件手动补充标题和描述，或使用 `npm run retry` 命令对标记失败的条目重新尝试抓取。对于长期抓取失败且无法手动补全的资源，建议直接删除或标记为待人工处理。

**问：静态站点生成后搜索功能无法正常工作？**

搜索功能依赖前端索引文件 `search-index.json`，该文件在构建过程中由 `src/static/js/build-index.js` 脚本生成。如果搜索无结果，请首先确认构建过程是否完整执行且无报错，然后检查输出目录中是否包含该索引文件。若索引文件存在但内容为空，可能是因为 `data/raw` 目录下没有有效的 Markdown 资源文件，或文件格式不符合解析规范。请参照 `docs/user-guide/format.md` 检查资源文件的格式，并重新执行 `npm run build`。
