# LinkHub Archive

LinkHub Archive 是一个面向技术研究者、开发者和信息分析人员的高质量外部链接聚合与导航系统。该项目并非传统的网页爬虫或书签管理器，而是一套精心编排的、经过人工筛选与分类的互联网资源索引体系。其核心目标在于解决信息过载时代下，高质量、长尾技术内容难以被发现与追溯的痛点。

项目定位为“技术资源的元数据库”，收录范围涵盖编程实践、系统架构、前沿论文解读、历史技术文档归档以及特定领域（如网络协议、数据库内核、编译原理）的深度讨论。LinkHub Archive 不仅提供链接的静态列表，更通过结构化的元数据标注（来源域名、内容摘要、发布时间、关联标签）为每个条目赋予上下文，使其成为可检索、可关联的知识网络节点。本项目适用于需要快速建立领域知识图谱、进行技术史回顾或获取稀有参考资料的专业人士。

## 功能概览

**结构化资源索引** 所有收录链接均按照内容主题、技术领域和文档类型进行多维度分类，支持按标签筛选与全文检索。

**原始来源溯源** 每个条目均保留完整的原始 URL 与来源域名信息，确保引用的可追溯性与完整性。

**内容摘要自动生成** 通过启发式规则提取页面标题、首段文本及关键元数据，生成简洁准确的内容概要。

**批量导入与验证** 支持从 CSV、JSON 及 Markdown 列表批量导入链接，并自动执行可达性与重定向检测。

**标签系统与关联推荐** 基于共享标签和共现关系，自动推荐与当前条目语义相近的其他资源。

**静态站点生成** 项目内置模板引擎，可将索引数据渲染为纯静态 HTML 站点，便于内网部署或 GitHub Pages 托管。

## 应用场景

**技术团队内部知识库建设** 研发团队可利用 LinkHub Archive 整理项目相关的技术文档、第三方库主页、故障排查案例及性能调优笔记，形成统一的知识入口，减少重复查找时间。

**学术研究与文献综述** 研究人员可通过本系统快速定位特定领域（如分布式系统、机器学习框架、编程语言设计）的系列文章与讨论串，辅助完成文献综述与技术选型报告。

**个人开发者学习路线规划** 开发者可根据自身技术栈（如 Rust、Kubernetes、PostgreSQL）筛选相关资源，系统性地阅读进阶材料，构建完整的技术认知体系。

**技术社区内容聚合** 开源社区维护者可利用本工具聚合项目相关的所有外部引用（如设计文档、会议演讲录像、相关 RFC），为贡献者提供一站式导航。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户可通过 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkhub-archive/linkhub.git
cd linkhub

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 导入示例数据并启动本地服务
python manage.py import --source data/sample_links.json
python manage.py build --output ./dist
python manage.py serve --port 8080
```

访问 http://127.0.0.1:8080 即可查看生成的导航站点。如需更新索引，可运行 `python manage.py crawl --check` 执行链接可达性验证。

## 项目结构

```
linkhub/
├── bin/                             # 可执行脚本与入口文件
│   ├── manage.py                   # CLI 主控程序，集成导入、构建、验证等子命令
│   └── cron_jobs.sh                # 周期性任务脚本（链接健康检查与索引更新）
├── src/                             # 核心源代码目录
│   ├── core/                        # 数据模型与业务逻辑
│   │   ├── models.py               # SQLAlchemy ORM 模型定义（Link, Tag, Category）
│   │   ├── parser.py               # 链接解析与元数据提取模块
│   │   └── validator.py            # URL 可达性验证与重定向处理
│   ├── engines/                     # 索引引擎与检索实现
│   │   ├── indexer.py              # 倒排索引构建与增量更新
│   │   └── searcher.py             # 布尔检索与标签过滤查询接口
│   ├── generators/                  # 静态站点生成器
│   │   ├── html_renderer.py        # Jinja2 模板渲染引擎封装
│   │   └── sitemap_builder.py      # XML 站点地图及 RSS 订阅源生成
│   └── utils/                       # 通用工具函数
│       ├── http_client.py           # 带超时与重试机制的 HTTP 请求封装
│       └── logger.py                # 结构化日志记录（JSON 格式输出）
├── data/                            # 数据存储与示例文件
│   ├── sample_links.json            # 示例导入数据（符合项目 schema）
│   ├── taxonomy.yaml                # 分类体系与标签别名定义
│   └── linkhub.db                   # SQLite 数据库文件（默认路径）
├── config/                          # 配置文件目录
│   ├── settings.py                 # 全局配置（数据库连接、缓存策略、超时阈值）
│   └── logging.conf                # 日志级别与输出目标配置
```

## 常见问题

**Q: 项目是否提供在线演示站点或 SaaS 服务？**

A: 目前项目以自托管（self-hosted）模式为主，不提供公共云服务。但您可以通过执行 `python manage.py build` 生成完整的静态 HTML 文件，并将其部署至任何支持静态文件托管的平台（如 GitHub Pages、Cloudflare Pages 或内网 Web 服务器）。示例部署脚本可参考 docs/operations/deployment.md。

**Q: 如何更新已有链接的内容摘要或标签？**

A: 支持两种更新模式。手动模式：通过 CLI 命令 `python manage.py edit --url <原始URL>` 进入交互式编辑，修改后自动更新数据库。自动模式：运行 `python manage.py crawl --refresh`，该命令会重新抓取所有链接并尝试更新元数据，但注意抓取频率需遵守目标网站的 robots.txt 协议，建议设置较长的间隔（如每周一次）。

**Q: 导入大量链接时性能如何？能否支持百万级条目？**

A: 项目默认采用 SQLite 数据库，经测试在 10 万级链接规模下，导入速度约为 1200 条/秒，检索响应时间低于 200ms。若预期数据量超过 50 万条，建议切换至 PostgreSQL 后端（需修改 config/settings.py 中的数据库连接字符串）。项目未对条目数量设置硬性上限，但需注意磁盘空间与内存占用（主要受倒排索引影响）。
