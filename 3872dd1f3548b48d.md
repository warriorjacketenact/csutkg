# LinkVault

LinkVault 是一个面向技术内容策展人与知识管理者的结构化外链资源聚合系统。本项目并非一个传统的爬虫或采集工具，而是一套严格的资源索引规范与验证框架，旨在解决个人书签杂乱、团队技术选型信息孤岛以及项目文档中外部链接缺乏可追溯性的问题。通过定义统一的资源条目格式与元数据标准，LinkVault 帮助开发团队将散落在各处的技术文档、工具官网、社区讨论与视频教程整合为可被 CI 流程校验的单一事实源。

本项目定位为技术基础设施的信息层组件，适用于需要长期维护大量外部引用的文档型仓库、技术周刊自动化生成管线以及企业级架构决策的知识库底座。LinkVault 不存储任何第三方内容，仅提供指向原始信息的确定性路径与状态标记，从而在保证信息鲜活度的同时规避版权与存储合规风险。

## 功能概览

- **确定性资源标识**：为每一条外链生成基于内容特征的不变标识符，支持在链接失效时通过本地缓存或 Wayback Machine 进行回溯。

- **批量状态巡检**：内置链接活性检查引擎，支持按批次（当前批次第 803/1241 批）并发探测资源可达性，自动标记 4xx、5xx 及 DNS 解析异常。

- **元数据扩展模板**：针对不同资源类型（文档、工具、视频、社区）提供预设的标签体系与描述字段模板，确保策展一致性。

- **变更追踪与审计日志**：记录每条资源的添加、移除与属性更新操作，便于团队协作时追溯信息变动原因。

- **结构化导出适配器**：支持将资源库导出为 JSON、YAML 或 CSV 格式，可无缝接入 Docusaurus、VuePress 或 MkDocs 等静态站点生成器。

- **访问频率统计**：基于引用次数与本地构建日志分析资源热度，辅助识别核心依赖与低频冗余链接。

- **合规性校验钩子**：提供预提交钩子脚本，在 Git commit 阶段自动检查新增 URL 的域名资质与协议安全性，阻断混合内容或已知恶意域名的引入。

## 应用场景

- **技术文档中心外链治理**：大型项目的文档站点往往包含数百个指向外部依赖、API 参考或教程的链接。LinkVault 可作为文档 CI 流水线的一环，在每次构建前自动扫描所有外链状态，并生成报告，帮助文档维护者及时发现失效引用并更新或替换。

- **团队技术雷达数据源管理**：技术雷达或周刊的编辑团队需要定期从多个信息源采集候选项目。LinkVault 的资源池化能力允许编辑使用统一界面收录候选链接，附加评估标签（如“试用”、“评估中”、“弃用”），并随时间轴追踪态度变化。

- **开源项目 README 链接聚合**：开源项目的 README 或 Wiki 页面常包含大量生态相关链接。通过 LinkVault 管理这些引用，可避免直接编辑 Markdown 文件时的格式混乱，同时利用自动化检查确保所有示例项目、插件市场或社区论坛的链接始终有效。

- **企业采购选型知识库构建**：企业在进行中间件、云服务或开发工具选型时，需要调研大量厂商官网、性能对比报告与用户案例。LinkVault 的元数据扩展能力支持记录调研日期、评估维度打分与内部评审意见，形成结构化的决策记忆。

## 快速开始

以下命令演示如何初始化 LinkVault 工作区、安装依赖并执行首次资源导入与状态检查。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装 Python 依赖（推荐使用虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 执行批次导入与状态巡检
# --batch 指定批次编号，--source 指向资源列表文件
python linkvault.py import --batch 803 --source ./data/urls_803.txt
python linkvault.py check --batch 803 --concurrency 10
```

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 或更高 | 核心运行时环境，用于执行导入、校验与导出逻辑 |

## 项目结构

```
linkvault/
├── cmd/                                # 命令行入口模块
│   ├── __init__.py                     # 包初始化，导出 CLI 命令映射
│   └── main.py                         # 主入口，解析子命令（import, check, export）
├── internal/                           # 内部核心实现（不对外暴露）
│   ├── fetcher/                        # HTTP 异步获取与重试逻辑
│   │   ├── __init__.py
│   │   ├── client.py                   # 基于 aiohttp 的会话池与连接复用
│   │   └── middleware.py               # 请求拦截器，注入 User-Agent 与超时策略
│   ├── parser/                         # 资源列表解析与校验模块
│   │   ├── __init__.py
│   │   ├── reader.py                   # 读取 txt / csv / json 格式源文件
│   │   └── validator.py                # URL 格式校验、去重与规范化
│   ├── storage/                        # 本地持久化层（基于 sqlite3）
│   │   ├── __init__.py
│   │   ├── repository.py               # 资源 CRUD 与批次管理
│   │   └── migrations/                 # 数据库版本迁移脚本
│   │       └── 001_initial_schema.sql  # 初始建表语句
│   └── reporter/                       # 检查报告生成器
│       ├── __init__.py
│       ├── console.py                  # 终端彩色表格输出
│       └── markdown.py                 # 生成 Markdown 格式的巡检摘要
├── pkg/                                # 可复用的公共库
│   ├── types/                          # 类型定义（Pydantic 模型）
│   │   ├── resource.py                 # Resource 实体定义
│   │   └── batch.py                    # Batch 元数据定义
│   └── utils/                          # 工具函数集
│       ├── timeutil.py                 # 时间戳转换与格式化
│       └── urlutil.py                  # URL 解析、域名提取与路径处理
├── config/                             # 配置目录
```

## 常见问题

**Q: 为什么导入一批 250 条的链接后，检查发现大量超时或 404 错误？**

A: 这通常由目标网站的访问策略、网络环境或资源本身的生命周期导致。建议首先检查本地网络出口是否被目标站点限制，尝试降低检查并发数（通过 `--concurrency` 参数调整）。若大量链接持续不可用，可考虑使用 `--retry` 参数增加重试次数，并检查目标站点的 robots.txt 是否屏蔽了自动化请求。对于确定为永久失效的链接，应从资源库中移除或标记为 `deprecated`。

**Q: LinkVault 支持哪些资源列表格式？能否从浏览器书签导出文件直接导入？**

A: 目前 LinkVault 原生支持纯文本（每行一个 URL）、CSV（需包含 `url` 列）及 JSON（顶层为 URL 字符串数组）三种格式。浏览器书签导出文件（如 Netscape 格式）暂不直接支持，但您可以使用社区提供的转换脚本或手动提取其中的 URL 列表。我们计划在后续版本中增加对通用书签格式的适配器。

**Q: 如何将 LinkVault 的巡检结果通知到团队聊天工具（如飞书、钉钉或 Slack）？**

A: 推荐在 CI 流水线中集成自定义 Webhook 步骤。LinkVault 的 `reporter` 模块支持输出 JSON 格式的详细报告，您可以编写一个轻量级脚本解析该 JSON，并调用对应聊天机器人的消息发送 API。项目 `docs/ci_integration.md` 中提供了一个基于 GitHub Actions 和 Slack 的参考实现。
