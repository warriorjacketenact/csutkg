# RYMPortal

RYMPortal 是一个面向开发者和技术研究人员的轻量级统一资源导航与代理访问工具。该项目通过结构化组织外部技术资源链接，提供稳定、可扩展的短链访问入口，帮助用户快速定位并获取分布于互联网各处的技术文档、工具链镜像、代码仓库镜像、开源社区讨论区以及各类技术资讯。

本项目定位为技术基础设施的辅助层，不存储任何用户数据或受版权保护的内容，仅作为公开资源的索引与访问加速点。目标用户包括运维工程师、架构师、后端开发者、SRE 团队以及需要频繁访问国际技术资源但受网络条件限制的研究人员。

## 功能概览

**统一短链访问入口**：提供基于路径参数的短链解析服务，将冗长或动态变更的外部 URL 映射为固定格式的访问链接，降低记忆成本与传播复杂度。

**资源分类与标签索引**：内置分类映射表，支持按技术领域、资源类型、源站地域等维度对链接进行标记，便于用户按需过滤与检索。

**可用性健康检查**：周期性对配置的全部外部链接执行 HTTP 探活检测，自动标记不可用或响应超时的资源节点，并在管理界面中高亮提示。

**访问统计与热度分析**：记录每个短链的请求频次、首次访问时间、最近访问时间以及来源 IP 地域分布，提供基础的热度排行数据。

**批量导入与导出**：支持通过 YAML 或 JSON 格式的配置文件批量导入链接集合，也可将当前生效的映射表导出为结构化文件用于备份或迁移。

**自定义重定向规则**：允许运维人员为特定短链配置临时重定向、永久重定向或反向代理转发模式，适应不同的资源访问策略。

**访问鉴权与白名单控制**：支持基于 IP 网段、HTTP Referer 或静态 Token 的访问限制，可用于内部系统或封闭测试环境。

**简易 Web 管理界面**：提供只读状态面板与基本管理操作界面，无需命令行即可查看资源状态、触发健康检查或导出日志摘要。

## 应用场景

**技术文档镜像站入口聚合**：团队内部往往需要同时参考多个外部文档站点，如不同版本的语言规范、框架 API 参考或云服务官方指南。RYMPortal 可将这些分散的文档入口聚合为统一的短链列表，新成员加入时仅需记忆一个基础域名即可获取全部文档索引。

**CI/CD 流水线中的依赖缓存代理**：在自动化构建过程中，频繁从海外源拉取依赖包可能因网络波动导致构建失败。运维人员可将国内可用的镜像站地址或代理网关地址配置为短链，在流水线脚本中引用这些短链，当上游地址变更时仅需在 RYMPortal 中更新映射，无需修改所有流水线配置文件。

**跨境技术资讯订阅与分发**：技术团队需要关注国际开源社区动态、安全公告或版本发布信息。通过 RYMPortal 将相关 RSS 源、邮件列表存档或社交媒体账号的链接集中管理，团队可定期通过健康检查功能感知哪些信息源已失效或变更，保持信息渠道的有效性。

**离线环境的外部资源清单维护**：在物理隔离或半隔离网络环境中，运维人员需提前申报并导入允许访问的外部域名列表。RYMPortal 可作为该列表的本地索引层，清晰记录每个外部资源的原始地址、申请用途与审批状态，便于定期审计与更新。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议通过 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/rymportal/rymportal.git
cd rymportal

# 安装依赖（使用 pip 和 npm）
pip install -r requirements.txt
npm install --prefix frontend

# 构建前端静态资源
npm run build --prefix frontend

# 初始化本地配置文件
cp config/example.yaml config/production.yaml

# 启动服务（默认监听 8080 端口）
python main.py --config config/production.yaml --port 8080
```

启动后访问 http://localhost:8080/admin 可查看管理面板。默认管理员账号为 admin，初始密码在首次启动时打印于控制台日志中。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心服务运行时环境，用于 API 服务与调度任务 |
| Node.js | 18.x 及以上 | 前端管理界面的构建工具链运行时 |
| SQLite | 3.35 及以上 | 内置轻量级数据库，用于存储映射关系与统计日志 |
| Redis | 6.2 及以上 | 可选依赖，用于多实例部署时的会话共享与缓存加速 |
| Nginx | 1.20 及以上 | 生产环境推荐前置反向代理，用于 TLS 终结与静态资源缓存 |
| git | 2.30 及以上 | 用于从仓库克隆源码，非运行时必需 |
| make | 3.82 及以上 | 用于执行自动化构建脚本，非运行时必需 |
| curl | 7.68 及以上 | 健康检查模块依赖的命令行工具，需确保 PATH 中可用 |

## 项目结构

```
rymportal/
├── main.py                      # 服务入口，初始化应用与启动调度器
├── config/
│   ├── example.yaml             # 配置模板，包含所有可调参数示例
│   └── production.yaml          # 生产环境配置，由管理员维护
├── core/
│   ├── __init__.py
│   ├── resolver.py              # 短链解析核心逻辑，含哈希查找与重定向处理
│   ├── health.py                # 健康检查模块，管理探活任务与状态存储
│   ├── stats.py                 # 统计收集模块，记录访问日志与聚合计算
│   └── mapper.py                # 映射关系管理，支持 YAML/JSON 导入导出
├── api/
│   ├── __init__.py
│   ├── routes.py                # 所有 API 路由注册与请求预处理
│   └── middleware.py            # 鉴权中间件、限流中间件与日志中间件
├── frontend/
│   ├── src/                     # Vue.js 前端源码目录
│   │   ├── components/          # 可复用 UI 组件，包含表格、图表、状态标签
│   │   ├── views/               # 页面级组件，包括仪表盘、列表页、详情页
│   │   └── store/               # Pinia 状态管理，存储当前映射表与筛选条件
│   ├── dist/                    # 构建输出的静态资源目录，由 nginx 托管
│   └── package.json             # 前端依赖声明与构建脚本
├── scripts/
│   ├── migrate_db.py            # 数据库迁移工具，用于 schema 升级
│   ├── seed_links.py            # 批量导入初始链接数据的命令行脚本
│   └── backup.sh                # 每日定时备份脚本，压缩并归档 SQLite 文件
├── tests/
│   ├── unit/                    # 单元测试，覆盖核心模块的关键函数
│   ├── integration/             # 集成测试，验证 API 与数据库的协同逻辑
│   └── fixtures/                # 测试用静态数据，包含模拟映射配置
├── docs/                        # 完整文档目录，含用户手册、运维手册与 API 参考
├── logs/                        # 运行时日志输出目录，按日期滚动存储
├── data/                        # SQLite 数据库文件与缓存文件存放位置
├── requirements.txt             # Python 生产依赖列表
├── requirements-dev.txt         # Python 开发与测试额外依赖
├── Makefile                     # 常用任务快捷命令，如 build、test、clean
└── LICENSE                      # MIT 许可证文件
```

## 常见问题

**问：短链是否可以自定义，而不使用系统自动生成的随机路径？**

答：可以。在管理界面创建映射时，用户可自行指定路径字符串，只需满足长度不少于 4 个字符且仅包含字母、数字、连字符和下划线的约束。系统会检查路径唯一性，若已存在则提示冲突。自定义路径不会影响原有解析性能。

**问：健康检查模块是否会对目标站点造成过大压力？**

答：健康检查默认采用单线程顺序执行，每个目标间隔至少 5 秒，且仅发送 HEAD 请求获取响应头，不下载响应体。默认超时时间设为 3 秒，避免长时间等待。用户可在配置文件中调整并发数与超时阈值，对于敏感目标可单独配置白名单跳过检查。

**问：如何将当前所有映射数据迁移到另一台服务器？**

答：首先在源服务器执行导出命令 `python scripts/seed_links.py --export links_export.json`，将导出的 JSON 文件复制到目标服务器后，执行导入命令 `python scripts/seed_links.py --import links_export.json --overwrite`。若需保留统计历史，需同时复制 data/ 目录下的 SQLite 数据库文件，并在目标服务器配置中指向该文件路径。
