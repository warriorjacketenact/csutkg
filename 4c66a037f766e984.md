# ResourceLink Collective

ResourceLink Collective 是一个面向开发者、技术研究人员与内容策展人的高性能外链资源聚合系统。该项目并非传统的内容管理系统，而是一个专注于外链元数据整理、分类、校验与批量导出的技术资源汇总站。其核心定位是帮助技术团队在文档建设、知识库维护、SEO 外链策略制定以及技术调研等场景中，高效管理大量分散的 URL 资源。

本项目解决的核心问题包括：手动整理海量链接易出错、链接失效难以追踪、缺乏结构化的资源分类体系、以及资源复用与共享困难。ResourceLink Collective 提供了一套完整的命令行工具与轻量级 Web 界面，支持批量导入、自动状态检查、标签化分类以及多格式导出（JSON、CSV、Markdown），适用于需要长期维护外部链接索引的技术团队与内容运营者。

## 功能概览

- **批量链接导入与解析**：支持从纯文本、CSV 及 Markdown 文件中批量提取 URL，自动解析域名、路径参数及扩展名，并生成基础元数据记录。

- **链接状态健康检查**：内置异步 HTTP 检测模块，可配置超时与重试策略，定期对资源库中的链接进行可达性验证，标记失效、重定向及不安全的协议混合内容。

- **多维度标签分类体系**：允许用户为每条链接自定义标签（如 "official-doc"、"api-reference"、"blog-tutorial"、"video-course"），并支持基于标签的快速筛选与统计。

- **结构化资源导出**：支持将当前资源库导出为 Markdown 列表、HTML 目录树或 JSON 格式的完整元数据包，便于嵌入项目文档、Wiki 或静态站点生成器。

- **重复链接检测与合并**：基于标准化后的 URL 字符串与域名指纹算法，自动识别并合并重复提交的资源记录，保留首次出现的标签与备注信息。

- **资源变更日志追踪**：记录每条链接的添加时间、最后检查时间、状态变化历史及操作者信息（基于系统用户或 API Key），满足审计与回溯需求。

- **命令行交互与脚本集成**：提供完整的 CLI 子命令（add、check、list、export、tag），支持在 CI/CD 流水线或定时任务中无头运行，便于自动化运维。

- **轻量级 Web 预览面板**：内置基于 Flask 的只读预览服务，支持按标签、状态、时间范围筛选查看资源列表，适合团队内部共享与演示。

## 应用场景

**技术文档站点维护**：技术写作团队在维护大型开源项目文档时，需要引用大量外部规范、教程和 API 参考链接。ResourceLink Collective 可用于集中管理这些引用链接，定期自动检查链接有效性，并在文档重新构建前生成最新的可用链接列表，避免文档中出现死链。

**SEO 外链策略执行**：运营人员针对特定关键词制定外链建设计划，需要分批收集和整理合作站点、论坛帖子和行业目录的 URL。本系统支持按批次导入链接，添加策略标签（如 "高权重"、"待交换"、"已收录"），并导出符合站点地图规范的链接清单，提升搜索引擎优化工作效率。

**技术调研与竞品分析**：研究人员在评估技术选型或分析竞品生态时，需收藏大量相关的项目主页、代码仓库、性能测试报告和社区讨论帖。通过本项目的标签分类和备注功能，可以构建结构化的调研知识库，并随时导出为研究简报或对比表格的素材。

**内部知识库资源聚合**：企业或团队内部通常存在多个分散的文档平台、监控面板和运维工具入口。ResourceLink Collective 可作为统一的内外部资源导航索引，将常用链接按团队、项目或环境分类，并提供健康检查功能，及时发现内部服务域名变动或证书过期问题。

## 快速开始

以下命令演示了如何在本地环境中克隆项目、安装依赖并启动基础服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resourcelink/collective.git
cd collective

# 创建并激活 Python 虚拟环境（推荐）
python3 -m venv venv
source venv/bin/activate  # Windows 系统请使用 venv\Scripts\activate

# 安装核心依赖与开发依赖
pip install -r requirements.txt
pip install -e .

# 初始化本地 SQLite 数据库并创建基础表结构
python manage.py init-db

# 导入示例资源列表（包含项目自带的测试数据）
python manage.py import --file samples/example_links.txt --tag "sample"

# 启动内置 Web 预览服务（默认监听 127.0.0.1:5000）
python manage.py runserver
```

│   │   ├── parser.py                   # URL 解析、标准化与重复指纹计算
│   │   └── exporter.py                 # JSON / CSV / Markdown 格式导出器

│   │   ├── main.py                     # Click 主入口与全局上下文
│   │   ├── add.py                      # 导入与添加资源子命令
│   │   ├── check.py                    # 触发健康检查子命令
│   │   ├── list.py                     # 列表查询与筛选子命令
│   │   └── export.py                   # 导出子命令
│   ├── web/                            # Web 预览面板
│   │   ├── app.py                      # Flask 应用工厂
│   │   ├── templates/                  # Jinja2 模板目录
│   │   └── static/                     # CSS 与 JavaScript 静态资源

## 常见问题

**Q: 导入包含大量链接的文件时，为什么有些链接被标记为重复或跳过？**

A: 系统默认基于标准化后的 URL 字符串（移除末尾斜杠、统一协议为小写、解码百分号编码）计算唯一指纹。如果两条链接解析后指向相同的标准化 URL，则后导入的记录会被标记为重复并跳过，以避免冗余。您可以通过 `import --no-dedup` 选项强制导入全部原始记录，但系统仍会在数据库中记录重复关系。

**Q: 健康检查模块如何处理需要登录或带有反爬机制的站点？**

A: 默认的健康检查仅执行基础的 TCP 连接和 HTTP HEAD/GET 请求，状态码为 2xx 或 3xx 视为可达。对于需要登录、验证码或 JavaScript 渲染的页面，系统可能无法获取有效内容，此时会将状态标记为 "unreachable"。您可以在配置文件中调整 `checker.timeout` 和 `checker.user_agent`，或为特定域名设置忽略检查策略。

**Q: Web 预览面板启动后无法访问，提示端口被占用怎么办？**

A: 默认端口为 5000。若该端口已被其他进程占用，您可以在启动命令中指定其他端口，例如 `python manage.py runserver --port 5001`。另外，确保防火墙或安全组策略允许该端口的入站连接，尤其是在远程服务器上部署时。
