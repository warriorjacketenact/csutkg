# WebLink Navigator

WebLink Navigator 是一个面向技术研究者、内容策展人与自动化工作流开发者的高质量外链资源归集与导航系统。本项目并非简单的书签集合，而是一套具备结构化元数据、可检索性与版本追踪能力的资源索引框架，专注于对分布式网络内容进行系统性整理与语义化标注。

项目定位于解决个人与团队在信息过载环境下对特定领域垂直资源的高效查找、分类存储与快速复用问题。通过统一的资源条目模型与可扩展的标签体系，WebLink Navigator 将散落在不同平台与域名下的高价值链接转化为可编程、可共享的结构化数据集，适用于技术文档聚合、行业动态监控、学术参考管理以及自动化数据采集管道中的种子源维护等场景。

本项目面向三类核心用户：需要系统化管理大量外链资源的技术内容运营者；希望构建私有或团队共享知识库的研发工程师；以及依赖稳定外部数据源进行模型训练、信息检索或定时任务编排的自动化系统开发者。

## 功能概览

**结构化资源条目模型** 每条资源支持标题、原始URL、来源域名、抓取时间戳、内容摘要与自定义标签字段，便于后续过滤与排序。

**多级分类与标签系统** 支持为每个链接分配一个主分类与多个功能标签，分类层级可自定义，满足不同业务场景下的组织逻辑。

**链接状态健康检查** 内置HTTP状态码探测与响应时间记录功能，可定期检测已收录资源的可访问性，自动标记异常条目。

**全文检索与高级过滤** 基于标题、摘要、标签与来源域名的组合查询，支持正则表达式与布尔逻辑，提升大型资源集下的查找效率。

**数据导入导出接口** 提供JSON、CSV与Markdown表格三种导出格式，支持批量导入外部链接列表，便于与其他工具链集成。

**版本变更追踪** 每次增删改操作均记录时间与操作类型，支持回滚至任意历史版本，保证资源目录的可审计性。

## 应用场景

技术文档站点聚合管理 技术团队可将分散在官方文档、社区博客与GitHub仓库中的参考链接统一收录，按技术栈分类，并定期检查链接有效性，避免文档中的引用失效。

自动化数据采集种子维护 数据工程师可将本系统作为爬虫入口URL的集中管理后台，通过标签筛选不同领域的种子链接，并按健康检查结果动态调整采集优先级。

行业资讯与竞品动态监控 内容运营人员可订阅特定分类下的新增链接，结合外部RSS或变更通知机制，快速获取行业内的最新发布内容，减少重复搜索时间。

学术参考文献外部链接备份 研究人员可将论文中引用的在线资源、数据集与工具页面统一归档，保留快照元数据，防止因原链接失效导致参考信息丢失。

## 快速开始

以下命令序列将在本地环境完成项目克隆、依赖安装与开发服务器的启动。

```bash
git clone https://github.com/weblink-navigator/weblink-navigator.git
cd weblink-navigator
npm install
npm run dev
```

执行完毕后，访问控制台输出的本地地址即可进入资源管理界面。默认使用SQLite作为存储后端，无需额外配置数据库服务。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.0.0 或更高 | 运行时环境，推荐使用LTS版本 |
| npm | 9.0.0 或更高 | 包管理器，用于安装项目依赖 |
| SQLite3 | 3.31.0 或更高 | 嵌入式数据库，用于存储资源条目与元数据 |
| Git | 2.25.0 或更高 | 版本控制工具，用于克隆仓库与贡献代码 |
| 网络连接 | 稳定访问公网 | 用于链接健康检查与资源抓取功能 |
| 操作系统 | Linux / macOS / Windows | 跨平台支持，Windows需启用WSL2或使用PowerShell |

## 贡献指南

1. 在 GitHub 仓库中提交 Issue 说明您希望修复的问题或新增的功能，等待维护者确认后再开始编码工作，避免重复劳动。

2. Fork 本仓库到您的个人账号下，在本地新建一个功能分支，分支名称应简要描述本次变更内容，例如 `fix-import-csv-encoding` 或 `add-tag-filter-api`。

3. 完成代码修改后，请确保通过所有现有单元测试，并为新增功能补充对应的测试用例。运行 `npm run test` 验证本地测试通过率。

4. 提交 Pull Request 到主仓库的 `main` 分支，在 PR 描述中清楚说明变更目的、实现方式以及影响范围，并关联相关 Issue 编号。

5. 等待代码审查。维护者可能会提出修改意见，请及时响应并更新 PR。合并后您的贡献将出现在下一个版本的更新日志中。

## 常见问题

问：项目是否支持 PostgreSQL 或 MySQL 作为生产数据库？

答：当前版本仅内置 SQLite 适配器，但存储层已抽象为统一接口。您可以通过实现 `IStorageAdapter` 接口来对接其他关系型数据库。社区贡献的 PostgreSQL 适配器正在开发中，预计在下个次要版本中合并。

问：如何批量导入现有书签文件，例如浏览器导出的 HTML 收藏夹？

答：项目提供了 `scripts/import-from-bookmarks.ts` 脚本，支持解析标准 Netscape 格式的书签 HTML 文件。您可以将浏览器导出的文件路径作为参数传入，脚本会自动提取标题与 URL 并生成对应的资源条目。具体用法请参阅 docs/user-guide/batch-import.md。

问：链接健康检查的频率和超时时间是否可以调整？

答：可以。健康检查的调度间隔、并发数、单次请求超时时间以及重试策略均可在 `config/default.json` 中的 `healthChecker` 字段下配置。生产环境建议根据网络状况与服务负载合理设置，避免对目标站点造成过大压力。
