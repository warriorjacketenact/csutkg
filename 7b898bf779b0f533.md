# LinkVault Core

LinkVault Core 是一个面向技术团队与内容创作者的轻量级外链资源汇总与导航系统。项目定位于将散落在各类技术文档、博客、教程中的高质量外部链接进行结构化整理，通过系列化标识与分类索引，帮助开发者快速定位所需的技术参考、设计素材、开发工具与学习资料。目标用户包括前端与后端工程师、技术文档撰写者、开源社区维护者以及教育培训机构的课程设计人员。LinkVault Core 通过统一的条目格式、可扩展的元数据标记以及简洁的静态站点生成逻辑，解决了技术资源链接分散、检索效率低、共享不便的常见问题，可作为团队内部知识库的补充模块独立运行。

## 功能概览

**系列化条目管理**：每个资源链接以系列标识符进行分组，支持按系列编号批量导入、导出与更新，便于维护大规模链接集合。

**静态资源索引生成**：基于本地标记文件自动生成索引页面，支持按系列、标签、发布日期等多维度筛选，无需数据库依赖。

**链接健康状态检查**：内置周期性连通性检测工具，可标记失效链接并生成报告，确保资源库的长期可用性。

**自定义元数据扩展**：每个条目支持标题、描述、分类、优先级、维护状态等自定义字段，满足不同场景下的分类需求。

**批量导入与导出**：支持 CSV 与 JSON 格式的批量链接导入导出，便于与现有工具链（如电子表格、爬虫脚本）集成。

**轻量级搜索接口**：提供基于关键词与正则表达式的本地搜索功能，支持模糊匹配与大小写敏感切换。

**访问统计与热度标记**：记录每个链接的本地访问频次，自动标记高频资源，辅助团队识别核心参考资料。

**响应式导航模板**：内置适配桌面与移动设备的 HTML 模板，可直接用于生成对外公开的资源导航页面。

## 应用场景

技术团队内部知识库的链接聚合：开发团队可将日常使用的 API 文档、设计规范、运维手册等外部链接通过 LinkVault Core 统一管理，按项目或技术栈分系列存储，新成员入职时可快速获取全部必需参考资源。

开源项目文档的外链整理：开源项目维护者可在项目仓库中集成 LinkVault Core，将依赖的第三方库、参考实现、社区讨论帖等整理为外链附录，随项目版本发布，方便用户追溯技术背景。

在线教育课程的配套资源导航：培训机构或独立讲师可使用 LinkVault Core 为每门课程生成配套资源页，将视频、代码仓库、习题答案、延伸阅读等链接按课时系列组织，学员可一站式访问全部外部材料。

技术博客的扩展阅读管理：技术博主可将文章中引用的全部外部链接统一存入 LinkVault Core，自动生成每篇文章的扩展阅读列表，提升内容的可验证性与深度。

## 快速开始

以下命令可在 Ubuntu 22.04 或 macOS Monterey 及以上环境中完成 LinkVault Core 的克隆、安装与首次运行。

```
git clone https://github.com/linkvault/core.git linkvault-core
cd linkvault-core
pip install -r requirements.txt
python linkvault.py --init --series 632/1241
```

执行完成后，系统将在 `data/series/` 目录下生成系列索引文件，并启动本地预览服务，默认监听 `127.0.0.1:8080`。用户可通过浏览器访问该地址查看资源列表。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9+ | 是 | 核心运行环境，用于执行链接管理脚本与生成静态页面 |
| pip 21.0+ | 是 | Python 包管理工具，用于安装 requirements.txt 中列出的依赖库 |
| Git 2.30+ | 是 | 用于克隆仓库及后续版本更新，支持通过 HTTPS 或 SSH 协议拉取 |
| requests 2.28+ | 是 | 用于链接健康检查与远程资源元数据获取，需保持网络连通 |
| markdown 3.4+ | 是 | 将系列条目的描述文本从 Markdown 渲染为 HTML，用于页面展示 |
| pytest 7.0+ | 否 | 仅在开发或运行单元测试时需要，生产环境可不安装 |
| virtualenv 20.0+ | 否 | 推荐用于隔离 Python 环境，避免依赖冲突，非强制但建议使用 |

## 项目结构

```
linkvault-core/
├── linkvault.py               # 主入口脚本，负责命令行解析与子命令路由
├── requirements.txt           # 生产环境与开发环境所需的 Python 依赖列表
├── config.yaml                # 全局配置文件，包含端口、缓存路径、检查间隔等参数
├── data/                      # 数据存储根目录
│   ├── series/                # 系列条目存储目录，每个系列为一个 .json 文件
│   │   ├── 632.json           # 第632批系列索引元数据与链接列表
│   │   └── 1241.json          # 第1241批系列索引元数据与链接列表
│   ├── cache/                 # 链接健康检查结果的本地缓存，减少重复网络请求
│   │   └── status.db          # SQLite 数据库，存储每个 URL 的最后检查时间与状态码
│   └── exports/               # 导出文件存放目录，支持 CSV 与 JSON 格式输出
├── src/                       # 核心功能模块源码目录
│   ├── parser.py              # 链接解析模块，负责校验 URL 格式与提取元数据
│   ├── checker.py             # 健康检查模块，使用 requests 库发送 HEAD 请求
│   ├── indexer.py             # 索引生成模块，构建内存中的搜索与分类索引
│   ├── renderer.py            # 模板渲染模块，将数据与 HTML 模板结合生成页面
│   └── utils.py               # 通用工具函数，包括日期格式化、文件读写、日志配置
├── templates/                 # Jinja2 模板文件目录
│   ├── base.html              # 基础布局模板，包含公共头部与底部
│   ├── index.html             # 资源列表首页模板，展示全部系列摘要
│   └── detail.html            # 单个条目详情页模板，展示完整描述与元数据
├── tests/                     # 单元测试与集成测试用例目录
│   ├── test_parser.py         # 针对 parser.py 的测试用例
│   ├── test_checker.py        # 针对 checker.py 的测试用例
│   └── fixtures/              # 测试用的固定样例数据，包含模拟系列 JSON 文件
├── docs/                      # 用户文档与开发文档目录
│   ├── getting-started.md     # 入门指南
│   ├── entry-specification.md # 条目规范
│   ├── cli-reference.md       # 命令行参考
```

## 常见问题

问：LinkVault Core 是否支持对链接条目进行标签分类和多级目录组织？
答：支持。每个条目在 JSON 文件中可包含 tags 数组字段，用户可自定义任意标签。同时支持通过 config.yaml 中的 category_mapping 配置项将系列映射到多级目录结构，生成导航时按目录层级展示。目录深度不限，但建议不超过三级以保证导航清晰度。

问：如何迁移已有的书签或收藏夹数据到 LinkVault Core？
答：项目提供了 import 子命令，支持从 Chrome 书签导出的 HTML 文件、Firefox 书签 JSON 以及通用 CSV 格式导入。运行 python linkvault.py import --format chrome --path bookmarks.html --series 632 即可将书签导入到指定系列。导入前可使用 dry-run 参数预览解析结果，确认无误后再执行实际导入。

问：链接健康检查是否会频繁触发目标站点的访问限制？
答：健康检查默认使用 HEAD 请求，仅获取响应头而不下载完整页面内容，对目标站点影响极小。检查间隔默认设置为 24 小时，且每个 URL 独立调度，避免集中突发请求。用户可在 config.yaml 中调整 interval_hours 参数增加间隔时间，或设置 user_agent 模拟不同客户端以降低被拦截概率。
