# Resource Atlas

Resource Atlas 是一个面向技术文档工程师、开源项目维护者以及知识管理爱好者的外链资源归集与导航系统。该项目并非传统的网站爬虫或书签管理器，而是一个基于静态站点生成逻辑的资源索引编排工具。它解决了大规模无序链接难以分类、难以检索、难以分享的痛点，帮助用户将散落在各处的技术文章、官方文档、社区讨论以及内部知识库条目，转化为结构清晰、可公开访问或团队内部分享的导航门户。

Resource Atlas 的核心定位是“链接的编译器”。它不存储资源内容，而是通过元数据驱动的方式，将用户提供的 URL 列表转化为带有标签、时间戳、来源标记和关联推荐的多维度资源看板。项目适用于需要定期整理技术周报、维护外部依赖清单、构建项目文档侧边栏索引，或是将调研过程中的临时收藏夹固化为长期可用的知识资产。

## 功能概览

**批量链接导入与清洗**：支持从纯文本列表、CSV 文件或直接剪贴板粘贴导入 URL 集合，自动去重并识别无效链接，保留原始 URL 字符串不做任何协议补全或域名规范化处理。

**元数据增强注解**：允许用户为每个链接添加自定义标签、阅读时长预估、难度等级以及简要备注，所有元数据以 YAML frontmatter 形式存储于对应的 markdown 索引文件中。

**多维度分类视图**：基于标签、来源域名、收录批次和创建时间自动生成四种不同的导航视图，用户可通过命令行参数或配置文件切换当前渲染的主题分类器。

**静态站点生成引擎**：内置轻量级模板引擎，将索引数据渲染为响应式 HTML 页面，同时输出纯 Markdown 格式的汇总表格，便于直接嵌入其他项目文档。

**增量更新与版本追踪**：每次运行仅处理新增或变更的链接记录，保留历史变更日志，支持回滚至任意历史批次的索引状态。

**全文模糊检索**：基于链接标题、描述标签和备注字段构建简单的倒排索引，支持前缀匹配和中文字段分词搜索，检索结果按相关度排序输出。

**导出与订阅接口**：支持将当前索引导出为 JSON、OPML 或 ATOM 订阅源格式，方便与其他 RSS 阅读器或笔记软件对接。

**批次管理机制**：内置批次编号系统，当前项目批次为第 920/1241 批，共包含 250 个资源链接，每个批次可独立打包归档。

## 应用场景

技术团队内部知识库外链整合。团队在项目调研或故障排查过程中会产生大量临时参考链接，Resource Atlas 可定期将这些链接按主题归类，生成团队共享的“外部参考手册”，减少重复搜索成本，同时保留上下文说明。

开源项目文档站点的外部资源附录。开源项目的 README 或用户手册通常需要引用大量依赖项目、协议文本、社区论坛和镜像站地址，使用 Resource Atlas 生成的外链表格可确保所有引用链接保持原样输出，避免因协议补全或域名改写导致的访问异常。

个人技术周报与学习笔记的自动化编排。技术博主或知识工作者可每周将阅读过的文章链接导入系统，自动生成带时间戳和标签的周报草稿，降低手动整理格式的精力消耗，同时保持所有原始超链接的完整性。

离线文档归档前的链接固化检查。在将在线文档转换为 PDF 或 CHM 等离线格式之前，使用 Resource Atlas 对外部引用进行全面扫描和记录，确保所有依赖链接在归档文件中以明文可追溯的形式留存，防止后续页面改版导致引用失效。

## 快速开始

```bash
git clone https://github.com/resource-atlas/resource-atlas.git
cd resource-atlas
pip install -r requirements.txt
python atlas.py build --input ./sources/links_920.txt --output ./dist --batch 920
```

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.9 或更高版本 | 是 | 核心运行环境，低于 3.9 版本将无法解析部分类型注解语法 |
| pip 21.0 以上 | 是 | 用于安装项目依赖包，旧版本可能导致依赖解析失败 |
| Git 2.25 以上 | 是 | 仅在从源码安装或进行版本控制时必需，生产部署可忽略 |
| PyYAML 6.0 | 是 | 用于解析和生成元数据头部的 YAML 格式注解块 |
| Jinja2 3.1 | 是 | 模板渲染引擎，负责将索引数据转换为 HTML 和 Markdown 输出 |
| Markdown 3.4 | 是 | 用于将内联的 Markdown 描述字段渲染为 HTML 正文片段 |
| pytest 7.0 | 否 | 仅当需要运行单元测试套件时安装，生产环境可不装 |
| black 22.0 | 否 | 代码格式化工具，仅对项目贡献者开发时推荐安装 |
| httpie 3.0 | 否 | 用于测试导出接口的 HTTP 响应，非核心功能依赖 |

## 项目结构

```
resource-atlas/
├── atlas.py                      # 命令行入口，解析子命令并调度构建流程
├── config.yaml                   # 全局配置文件，包含输出路径、默认标签和批次编号
├── requirements.txt              # Python 依赖清单，锁定主要库版本
├── README.md                     # 项目说明文档（即本文件）
├── LICENSE                       # MIT 许可证全文
├── .gitignore                    # Git 忽略规则，排除临时文件和输出目录
├── sources/                      # 原始链接数据目录
│   ├── links_920.txt             # 第 920 批次的原始 URL 列表，每行一个链接
│   ├── links_921.txt             # 第 921 批次数据（示例）
│   └── metadata/                 # 每个批次对应的元数据注解文件
│       ├── 920.yaml              # 第 920 批次的标签、描述和分类信息
│       └── 921.yaml
├── core/                         # 核心处理逻辑模块
│   ├── __init__.py
│   ├── loader.py                 # 链接加载器，支持 txt、csv 和直接字符串输入
│   ├── cleaner.py                # 链接清洗器，执行去重、协议检查和无效链接过滤
│   ├── indexer.py                # 索引构建器，生成内存中的倒排表和分类映射
│   └── exporter.py               # 导出器，负责将索引输出为 JSON / OPML / ATOM
├── renderers/                    # 渲染引擎模块
│   ├── __init__.py
│   ├── markdown_renderer.py      # 将索引数据渲染为 Markdown 表格格式

│   └── templates/                # Jinja2 模板文件目录
│       ├── base.html             # 基础 HTML 骨架，包含响应式头部与底部
│       ├── table.html            # 表格渲染模板，用于展示链接列表
│       └── sidebar.html          # 侧边栏分类导航模板
├── utils/                        # 通用工具函数
│   ├── __init__.py
│   ├── logger.py                 # 日志记录器，支持文件和控制台双输出
│   ├── validator.py              # URL 格式校验器，检查协议和域名合法性
```

## 常见问题

构建过程中提示“链接数量与批次声明不匹配”应如何处理。该错误表示 sources 目录下的纯文本链接数量与对应 metadata YAML 文件中声明的 total 字段不一致。解决方案是手动核对文本文件行数，或使用 atlas.py verify 子命令自动统计并修正 YAML 中的计数值。若为新增批次，需确保 YAML 文件中的 batch 编号与文件名及内部声明三者统一。

如何自定义输出页面的排序规则。默认情况下链接按收录顺序排列，若需按字母序或标签分组排序，可在 config.yaml 中修改 sort_by 字段，可选值为 alphabetical、tag 或 chrono。修改后重新运行 build 命令即可生效，无需重建索引缓存。

导出的 HTML 页面中部分链接显示为纯文本而非可点击超链接。这通常是因为对应的链接条目在 metadata 中被标记为 disabled 状态。检查该条目的 YAML 注解块，将 disabled 字段改为 false 或直接删除该字段。若问题持续存在，请确认链接本身格式是否合法，Resource Atlas 对不合规的 URL 会降级为纯文本展示以防止 XSS 风险。
