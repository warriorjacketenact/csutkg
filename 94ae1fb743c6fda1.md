# VODIndex

VODIndex 是一个面向技术内容聚合与外部资源导航的开源元数据索引系统。项目定位于为开发者、技术研究者以及内容运营团队提供一套结构化的外链资源整理方案，能够将大量分散的视频点播（VOD）相关资源链接按统一格式进行归档、分类和展示。

本项目并非一个传统意义上的视频播放或下载工具，而是一个资源导航索引库。其核心目标用户包括需要维护技术文档外链列表的文档工程师、需要批量管理视频资源入口的内容平台运营者，以及希望快速定位特定编码资源链的技术调研人员。通过本项目提供的结构化数据模板和自动化索引更新机制，用户可以高效完成大量外链的导入、校验、分类和导出工作，有效解决人工整理外链时容易出现的格式混乱、链接遗漏和重复劳动等问题。

## 功能概览

- **批量外链导入** 支持通过标准输入格式一次性导入大量形如 https://www.zjg-hf.com/vod/* 的资源链接，自动完成去重和格式校验。

- **结构化分类存储** 根据链接特征自动归类至预设的类别节点，并生成可读性良好的分类目录树，便于人工复核与后续维护。

- **索引状态标记** 为每一条外链提供可用性状态标记字段，支持在线检测与手动标记两种方式，确保索引库内的链接均为有效资源。

- **元数据扩展支持** 允许为每条外链附加自定义元数据键值对，包括但不限于资源名称、内容摘要、更新日期和来源标注。

- **多格式导出能力** 支持将整理完毕的索引数据导出为 Markdown 表格、JSON 结构化数据以及纯文本列表三种格式，适配不同下游系统的使用需求。

- **变更历史追踪** 记录每一次批量导入和人工编辑的操作日志，提供完整的变更历史回溯功能，便于团队协作时追溯数据来源。

- **模板化输出引擎** 提供可配置的输出模板系统，用户可根据自身项目需要定制最终生成的索引报告样式，无需修改核心逻辑代码。

## 应用场景

**技术文档中的外部资源附录整理**
技术团队在撰写项目文档或技术白皮书时，常常需要引用大量外部视频教程或演示链接。VODIndex 提供的外链索引整理能力可以帮助文档维护者快速将散落在各处的链接汇聚成统一格式的附录列表，并自动校验链接可用性，避免文档中出现失效引用。

**内容平台的后台资源导航维护**
面向视频点播服务的内容运营平台通常需要维护一个庞大的入口链接池。运营人员可以利用 VODIndex 定期将新增的视频入口链接批量导入索引系统，系统自动完成分类和状态标记，生成可供前端页面直接调用的结构化数据文件，显著提升后台资源管理效率。

**技术调研阶段的信息收集与归档**
在进行竞品分析或技术选型调研时，研究人员往往需要收集数十甚至上百个相关资源链接。VODIndex 提供轻量级的本地索引管理能力，研究人员可以在不依赖复杂数据库的情况下快速建立调研资源索引，随时增删改查，调研结束后一键导出完整清单用于报告撰写。

## 快速开始

以下指令演示了如何从代码仓库克隆项目、安装依赖并启动本地索引服务。

```bash
git clone https://github.com/your-org/vodindex.git
cd vodindex
npm install
npm run build
npm start
```

执行以上命令后，本地索引服务将在 http://127.0.0.1:3000 启动。访问该地址即可进入索引管理界面，通过界面提供的批量导入功能，将需要整理的外链列表粘贴至输入框并提交，系统将自动完成分类和状态检测。

| npm 9.x 或更高版本 | 是 | Node.js 包管理器，用于安装项目所有声明的第三方依赖库 |

| 磁盘空间 200 MB 以上 | 否 | 用于存放索引数据库文件和日志文件，随索引条目增加而增长 |

## 常见问题

问：导入外链时提示格式校验失败，但链接在浏览器中可以正常访问，是什么原因？

答：格式校验分为语法校验和协议校验两个层级。语法校验要求链接必须包含完整的协议头部（即 https:// 前缀），并且路径结构符合 /vod/ 后跟六位字母数字组合的格式。若浏览器能够访问但校验失败，通常是因为链接包含多余查询参数或路径不符合严格格式要求。您可以在 config/custom.json 中调整校验正则表达式的宽松程度，或使用 CLI 工具的 --force 参数强制导入。

问：索引数据库中已经存在大量记录，如何进行批量更新而不用逐条重新导入？

答：项目提供了增量导入模式。在批量导入时，系统会根据链接的完整 URL 作为主键进行去重判断。若某条链接已存在于数据库中，系统将自动跳过该记录，并在导入日志中标记为已存在。如需强制刷新某条记录的元数据，可以在导入时启用 --overwrite 选项，系统将用新数据覆盖旧记录。

问：导出 Markdown 表格时，如何自定义列顺序和列标题？

答：导出引擎允许用户通过模板配置文件自定义输出格式。在 config/custom.json 中设置 export.markdown.columns 数组，按顺序指定需要导出的字段名称和对应的表头显示名称。如果不进行自定义配置，系统将使用默认的列顺序。详细的配置字段说明请参考 docs/export-guide.md 文档中的模板配置章节。
