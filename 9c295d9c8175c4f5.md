# LinkForge

LinkForge 是一个面向技术团队与内容研究者的结构化外链资源聚合与导航系统。项目定位于解决分散在各类文档、邮件、即时通讯中的高价值链接难以统一归档、无法按上下文检索、缺乏生命周期管理的问题。LinkForge 不提供爬虫或自动化采集功能，而是围绕人工精选与批量导入两条主线，为每个链接赋予分类标签、来源批次、关联项目与摘要备注，最终生成静态导航站点或嵌入现有知识库的 iframe 面板。目标用户包括开源项目维护者、技术文档工程师、开发者社区运营人员以及需要进行大量文献引用管理的科研工作者。

## 功能概览

**批量外链导入与去重**：支持通过 CSV、JSON 或纯文本列表批量提交 URL，系统自动识别协议头与路径结构，剔除重复条目并生成导入报告。

**多级分类与标签体系**：每个资源可关联一个主分类与多个自定义标签，分类支持无限层级嵌套，便于按技术领域、项目阶段或内容类型组织链接。

**链接健康状态巡检**：每日定时检测已收录资源的 HTTP 状态码，标记失效链接（4xx、5xx）并发送通知，支持导出失效清单。

**全文检索与过滤器**：基于标题、描述、标签、来源批次、创建时间等多维度组合过滤，检索结果可排序并支持 RSS 订阅。

**批次管理与回溯**：每个导入批次自动记录时间、操作人、链接数量与来源描述，支持按批次整体导出或批量更新标签。

**嵌入与分享面板**：为每个分类或标签生成独立的嵌入代码（iframe 或 JavaScript 片段），可将链接列表直接嵌入团队内部文档、Confluence 页面或 GitHub README。

## 应用场景

开源项目文档站的外链管理：开源项目通常需要引用大量外部依赖、教程、示例代码和社区讨论帖。LinkForge 可为项目的每个子模块建立独立的链接分类，并在版本发布前批量校验所有引用链接的可用性，避免文档中出现死链。

技术团队周报与知识库沉淀：团队每周会产生大量值得收藏的技术文章、工具网站和内部系统地址。使用 LinkForge 按周批次导入链接并添加标签，可将周报中的散落链接转化为可持续检索的知识资产，同时支持按季度归档。

在线课程与技术培训资源包：讲师在为学员准备学习材料时，需要聚合外部视频、在线练习、官方文档和社区问答。LinkForge 的嵌入面板功能允许将整理好的链接列表直接插入课程管理平台，学员无需在不同页面间切换即可访问全部资源。

产品需求调研与竞品分析：产品经理在调研阶段会收集大量竞品官网、行业报告、用户反馈帖和 API 文档。LinkForge 的自定义标签与备注字段能清晰区分竞品类别、调研时间和优先级，便于后续复盘与决策支持。

## 快速开始

以下命令将在本地启动 LinkForge 开发实例，默认使用 SQLite 数据库，无需额外配置外部数据库服务。

```bash
git clone https://github.com/linkforge/linkforge.git
cd linkforge
pip install -r requirements.txt
python manage.py migrate
python manage.py load_batch_demo
python manage.py runserver
```

访问 http://127.0.0.1:8000 即可进入仪表盘。管理员默认账号 admin，密码 admin123，首次登录后请立即修改。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9+ | 是 | 核心运行环境，推荐使用 3.11 或更高版本以获得性能优化 |
| Django 4.2+ | 是 | Web 框架，提供 ORM、路由、管理后台及模板引擎 |
| SQLite 3.35+ | 是 | 默认数据库，适合单机或小规模部署；生产环境建议切换至 PostgreSQL |
| redis 6.0+ | 否 | 用于缓存链接健康状态检测结果与检索索引，提升大规模数据下的响应速度 |
| celery 5.3+ | 否 | 异步任务队列，用于执行定时巡检与批量导入后台任务 |
| gunicorn 21.0+ | 否 | 生产环境推荐的 WSGI 服务器，多进程模式提高并发能力 |
| nodejs 18+ | 否 | 仅用于前端资源编译，若使用预编译静态文件则可忽略 |

## 贡献指南

1. 在 GitHub 上 fork 本仓库并克隆到本地，创建新分支时使用 `feature/` 或 `fix/` 前缀，分支名需简要描述改动内容，例如 `feature/add-json-export`。

2. 编写代码时请遵循 PEP 8 风格规范，提交前运行 `make lint` 执行 flake8 与 black 检查。新增功能需在 `tests/` 下补充对应测试用例，确保测试覆盖率不低于百分之八十五。

3. 提交 Pull Request 前请将当前主分支的更新合并至你的分支，解决所有冲突。PR 描述需填写改动摘要、影响范围、测试结果以及是否涉及数据库迁移。

4. 若计划引入新的第三方依赖，请先在 issue 中说明必要性并等待维护者回复。外部 API 集成或新增配置项时，需同步更新 `.env.example` 与对应文档。

## 常见问题

Q: 导入包含上千条链接的大文件时页面长时间无响应，应该如何处理？

A: 建议使用异步导入模式。在管理后台选择“异步导入”选项，系统会将任务交给 celery worker 后台执行，导入完成后会向绑定邮箱发送通知。对于超过五千条的文件，建议拆分为多个批次导入以避免内存溢出。

Q: 健康检查检测到失效链接后会自动删除吗？

A: 不会自动删除。系统默认将失效链接标记为 `status = 'dead'` 并保留原有分类与标签，同时记录最后一次检测时间与状态码。你可以在仪表盘的“失效链接”视图中手动确认后批量删除或更新 URL。该设计是为了防止误判（临时维护或网络波动导致的短暂不可用）造成数据丢失。

Q: 能否将 LinkForge 嵌入到现有的 Confluence 或 Notion 页面中？

A: 可以。每个分类或标签详情页均提供了嵌入面板功能，点击“获取嵌入代码”会生成一段 iframe 或 JavaScript 片段。将代码复制到 Confluence 的“插入 HTML”宏或 Notion 的“嵌入”块中即可。嵌入面板支持浅色与深色主题，并会自动适应父容器宽度。
