# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究、数字取证和内容分析场景的轻量级外链资源聚合与索引系统。该项目不直接存储任何第三方内容，而是通过结构化方式整理和分类来自公开网络的主题资源链接，帮助研究人员、开发者和内容审核人员快速定位特定编号的资源条目。LinkVault 适用于需要批量管理外部链接、建立资源映射关系或进行定期链接可用性检查的工作流。

目标用户包括学术研究者、爬虫开发者、数据标注团队以及需要维护外部资源索引的系统管理员。LinkVault 以静态资源索引为核心，提供清晰的分类目录和元数据标注机制，使得大规模外链集合具备可读性和可维护性。

## 功能概览

- **批量链接导入与解析**：支持通过文本文件或标准输入流导入大量 URL 列表，自动解析 URL 结构并提取域名、路径、扩展名及资源编号。

- **分类标签与元数据管理**：允许为每条链接添加自定义标签（如类别、优先级、状态），并支持基于标签的快速筛选与统计。

- **链接健康状态检测**：内置异步 HTTP 检测模块，可周期性检查链接可达性，返回状态码和响应时间，便于识别失效资源。

- **全文检索与过滤**：基于内存索引的轻量级搜索功能，支持按 URL 片段、资源编号或自定义注释进行关键字匹配。

- **多种格式导出**：支持将索引数据导出为 JSON、CSV 或 Markdown 表格格式，便于集成至文档系统或数据分析流水线。

- **资源编号映射表**：自动提取 URL 中的数字编号（如 theater/6070.html），建立编号到完整 URL 的快速映射，支持反向查询。

- **变更日志记录**：记录每次链接集合的增删改操作，保留操作时间戳和操作类型，便于审计与回溯。

## 应用场景

- **数字取证与调查分析**：调查人员可以使用 LinkVault 整理案件相关的外部线索链接，按编号和标签分类，定期检查链接存活状态，确保证据链的完整性。

- **内容聚合与定期监控**：内容运营团队可将分散在多个页面的资源链接统一导入 LinkVault，设置周期性健康检查任务，及时清理失效链接，提升用户体验。

- **学术文献外部引用管理**：研究人员在撰写综述或论文时，可利用 LinkVault 管理大量参考文献的外部链接，快速导出为标准化格式，避免手工整理带来的遗漏和错误。

- **数据标注流水线预处理**：数据标注团队可将待处理的资源链接导入 LinkVault，根据标签分配任务批次，完成后批量导出标注结果，提高标注流程的自动化程度。

## 快速开始

以下步骤指导您在本地环境快速启动 LinkVault 服务。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（使用 pip 和 venv）
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化本地数据库并导入示例链接
python linkvault.py init
python linkvault.py import --file samples/links.txt
python linkvault.py serve --port 8080
```

启动成功后，访问 http://localhost:8080 可查看 Web 界面，或通过 CLI 执行查询、导出等操作。

## 常见问题

**问：LinkVault 是否存储资源内容的副本？**

答：不存储。LinkVault 仅保存 URL 地址、标签、检测状态和用户自定义注释。所有资源内容均保留在原始第三方站点上，用户访问时直接跳转至源地址。建议使用者遵守相关网站的 robots.txt 规则和访问频率限制。

**问：健康检查模块对目标服务器会造成压力吗？**

答：健康检查默认采用异步并发方式，但并发数默认限制为 10 个连接，且每个请求超时时间设置为 10 秒。同时支持配置检查间隔（默认每小时一次），避免对目标服务器产生高频请求。对于大规模链接集合，建议在非高峰时段运行检查任务。

**问：如何迁移已有链接数据到新版本？**

答：LinkVault 提供数据库迁移脚本（位于 src/storage/migration.py）。升级版本后，执行 python linkvault.py migrate 即可自动升级数据库 Schema，无需手动导出导入。建议在执行迁移前备份 data/linkvault.db 文件。
