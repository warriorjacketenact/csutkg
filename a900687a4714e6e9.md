# LinkTheater 资源导航系统

LinkTheater 是一个面向技术研究者和信息分析人员的轻量级资源导航与外链聚合工具。该项目定位于将分散在网络各处的特定主题页面（如技术文档、案例研究、历史存档、演示文稿等）进行集中索引与分类展示，帮助用户快速定位并访问高价值外部资源。LinkTheater 不提供内容存储服务，仅作为结构化导航层存在，适用于个人知识管理、团队协作研究以及公开资源站的入口整合。

项目采用静态页面生成架构，所有资源链接以数据文件形式维护，支持自动化校验、分类标记与访问统计。LinkTheater 的目标用户包括技术文档工程师、数据分析师、学术研究人员以及任何需要系统化管理大量外链的个体或团队。通过统一的导航界面和分类逻辑，用户无需记忆冗长的 URL 即可在数秒内抵达目标页面。

## 功能概览

**批量链接导入**：支持从 CSV、JSON 或纯文本列表批量导入外部 URL，自动去重并生成初始索引记录。

**智能分类标记**：基于 URL 路径模式、域名特征或自定义标签规则，对导入链接进行自动或半自动分类，便于后续检索与筛选。

**链接可用性检测**：内置周期性的 HTTP 状态检查机制，标记失效链接并生成报告，帮助维护资源库的健康度。

**访问热度统计**：记录每个外链的点击次数与最后访问时间，支持按热度排序，识别高频使用资源。

**多视图导航界面**：提供列表视图、卡片视图和分类层级视图三种浏览模式，适应不同使用习惯与屏幕尺寸。

**全文检索与过滤**：支持按标题、描述、标签、域名等字段进行关键词检索，并支持多维度过滤组合。

**数据导入导出**：支持将整个索引库导出为 CSV、JSON 或 HTML 书签文件，便于迁移、备份或与其他工具集成。

**自定义元数据扩展**：允许为每个链接添加自定义字段（如备注、优先级、关联项目编号），满足个性化管理需求。

## 应用场景

**技术文档中心入口**：技术团队可将 LinkTheater 部署为内部文档门户的导航层，聚合 API 参考、架构设计文档、部署手册、故障排查指南等各类外部链接，新成员通过导航树即可快速熟悉项目全貌。

**学术研究文献索引**：研究人员在文献调研阶段收集的大量 PDF 链接、预印本仓库、数据集页面和工具官网，可通过 LinkTheater 统一归档并添加研究主题标签，避免遗忘或重复收集，同时支持多人协作维护共享文献库。

**开源项目资源汇总页**：开源项目维护者可以使用 LinkTheater 构建项目的官方资源导航页，将社区论坛、贡献指南、示例代码仓库、视频教程、相关工具链等链接集中展示，降低贡献者和用户的信息获取成本。

**个人书签管理与同步**：个人用户可将 LinkTheater 作为自托管的书签管理系统，替代浏览器内置书签，实现跨浏览器、跨设备的统一访问入口，并利用分类和检索功能高效管理成千上万条书签。

## 快速开始

以下步骤指导您在本地环境快速启动 LinkTheater 服务。

```bash
# 克隆代码仓库
git clone https://github.com/linktheater/linktheater.git
cd linktheater

# 安装依赖（使用 npm）
npm install

# 运行开发服务器
npm run dev
```

执行上述命令后，开发服务器将在 http://localhost:3000 启动。浏览器打开该地址即可访问导航界面。默认加载示例数据目录中的演示链接集合。如需替换为自己的链接数据，请编辑 `data/links.json` 文件并重启服务。

## 项目结构

项目目录遵循模块化设计原则，核心代码与配置数据分离，便于维护和扩展。

```
linktheater/
├── src/                           # 源代码主目录
│   ├── core/                      # 核心逻辑模块
│   │   ├── indexer.js             # 链接索引引擎，负责解析和存储
│   │   ├── checker.js             # 可用性检测器，执行 HTTP 状态检查
│   │   └── stats.js               # 统计聚合器，处理点击与热度数据
│   ├── routes/                    # 路由层
│   │   ├── api.js                 # RESTful API 路由定义
│   │   └── web.js                 # 前端页面路由（SSR）
│   ├── models/                    # 数据模型层
│   │   ├── link.js                # 链接实体模型（字段、校验）
│   │   └── category.js            # 分类实体模型（层级、别名）
│   ├── services/                  # 业务服务层
│   │   ├── importService.js       # 批量导入服务（CSV/JSON）
│   │   └── exportService.js       # 导出服务（HTML/JSON）
│   ├── utils/                     # 通用工具函数
│   │   ├── urlValidator.js        # URL 格式校验与规范化
│   │   └── logger.js              # 日志记录器（按级别输出）
│   └── app.js                     # 应用入口文件
├── data/                          # 数据存储目录
│   ├── links.json                 # 主链接索引库（JSON 格式）
│   ├── categories.json            # 分类定义文件
│   └── meta.db                    # SQLite 元数据库（统计信息）
├── public/                        # 静态资源目录
```

## 常见问题

**问：LinkTheater 是否存储外部链接对应的页面内容？**

答：不存储。LinkTheater 仅维护 URL 字符串及其元数据（标题、标签、描述、分类等），不抓取、不缓存、不代理任何外部页面的内容。所有对外部资源的访问均直接由用户浏览器发起，遵循标准 HTTP 协议。项目不承担外部链接的内容审核责任，仅供导航便利。

**问：如何迁移现有书签数据到 LinkTheater？**

答：LinkTheater 内置了导入服务，支持 Chrome 书签导出文件（HTML 格式）、通用 CSV 模板以及 JSON 格式的数据文件。具体操作步骤为：首先从浏览器导出书签为 HTML 文件，然后在 LinkTheater 管理界面选择“导入”功能，上传文件并指定分类映射规则。系统将自动解析并生成索引记录。对于大型书签库（超过 5000 条），建议使用命令行导入脚本以获得更好的性能和进度反馈。

**问：链接可用性检测的频率和策略是怎样的？**

答：默认每周日凌晨 2:00 执行全量检测。检测策略包括：超时时间 10 秒，遵循 HTTP 重定向（最多跟随 5 次），记录最终响应状态码。对于返回 4xx 或 5xx 状态的链接标记为“异常”，超时或网络不可达标记为“可疑”。用户可在配置文件中调整检测频率、超时阈值和白名单域名（跳过检测）。检测结果通过管理仪表板展示，并可按需导出为报告。
