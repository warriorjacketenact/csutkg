# LinkVault 技术资源索引系统

LinkVault 是一个面向开发者、技术研究人员与开源爱好者的高质量视频技术资源聚合与导航系统。该项目并非传统意义上的视频播放或下载工具，而是一个结构化的外链资源索引引擎，旨在解决技术学习过程中优质分散、检索效率低下的问题。系统通过标准化的资源描述与分类机制，将大量分布于不同平台的技术讲座、架构分享、语言教程与工程实践视频进行统一收录与快速定位，适用于个人知识管理与团队技术分享场景。

作为第 737/1241 批资源整合计划的一部分，LinkVault 当前版本收录了共计 250 个经过初步筛选的技术视频链接，资源主题覆盖前端开发、后端工程、云原生基础设施、人工智能基础、数据库调优、DevOps 实践等多个方向。项目本身不存储任何视频文件，仅提供元数据索引与访问引导服务，所有视频内容版权归属原始发布方。

## 功能概览

**结构化资源索引**：系统将收录的 URL 按照视频内容所属技术领域进行初步分类，并提供标签化检索入口，帮助用户在 250 个链接中快速定位感兴趣的主题。

**外链跳转中间层**：所有视频链接通过项目提供的页面进行展示与跳转，用户点击后直接访问原始视频平台，项目不干预或修改目标 URL 的任何参数。

**资源状态标记**：对每个收录的链接进行可访问性状态跟踪，定期更新失效链接或内容变更链接的状态，保证索引库的时效性。

**关键词快速过滤**：用户可通过输入技术关键词（如 Kubernetes、Rust、微服务）对当前资源列表进行实时筛选，仅显示标题或描述中包含关键词的视频条目。

**访问统计面板**：提供每个链接的点击次数统计，帮助团队识别热门资源与冷门资源，为后续资源筛选提供数据依据。

**书签与收藏功能**：允许注册用户将特定视频链接加入个人收藏夹，便于后续反复查阅，支持收藏夹的导入与导出。

**多维度排序机制**：支持按收录时间、点击热度、资源状态、标题字母顺序等多种方式对资源列表进行排序，满足不同浏览习惯。

## 应用场景

技术团队内部知识库建设：团队技术负责人可以将 LinkVault 部署为内部知识导航页，将团队推荐的视频教程、技术分享录播、会议演讲等资源统一收录，新成员入职时通过该索引快速了解团队技术栈与学习路径。

个人开发者学习路线管理：独立开发者或自学者面对海量技术视频时，可利用 LinkVault 的标签与过滤功能建立个人学习资源库，将 B站、YouTube、技术会议官网等不同来源的视频链接集中管理，按学习阶段或主题分类，避免资源散落在浏览器书签中难以查找。

技术社区资源共建：开源社区或技术论坛可基于 LinkVault 搭建社区推荐资源墙，由社区成员提交优质视频链接，项目维护者审核收录，形成经过社区背书的质量可控的技术视频导航站，降低新人筛选优质内容的时间成本。

离线资源清单备份：对于网络环境受限或需要长期保存资源关系的场景，LinkVault 的资源列表可作为结构化数据导出为 JSON 或 CSV 格式，便于离线存档或迁移至其他知识管理工具。

## 快速开始

以下命令演示了如何从 GitHub 克隆项目、安装依赖并启动开发服务器。请确保系统已安装 Node.js 18.x 及以上版本和 npm 9.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git

# 进入项目目录
cd linkvault

# 安装项目依赖
npm install

# 启动开发服务器，默认监听端口 3000
npm run dev
```

执行完上述命令后，访问控制台输出的本地地址（通常为 http://localhost:3000）即可看到资源索引首页。生产环境部署请参考 `docs/deployment.md` 文档，使用 `npm run build` 构建静态文件并通过 Nginx 或 Caddy 托管。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 项目运行时环境，建议使用 LTS 版本以保证稳定性 |
| npm | 9.x 或 10.x | 包管理器，用于安装项目依赖与执行脚本命令 |
| SQLite | 3.x（内置） | 默认数据库引擎，用于存储用户收藏与访问日志，无需额外安装 |
| 现代浏览器 | Chrome 110+ / Firefox 110+ / Edge 110+ | 前端界面需要浏览器支持 ES2022 语法和 CSS Grid 布局 |
| 网络环境 | 可访问外网 | 资源跳转功能需要用户端网络能够访问目标视频平台（如 YouTube、Bilibili 等） |
| 磁盘空间 | 至少 200 MB | 用于存放项目源代码、依赖包及 SQLite 数据库文件 |
| 内存 | 建议 512 MB 以上 | 开发模式运行时的内存占用，生产环境可进一步优化 |

## 项目结构

```
linkvault/
├── src/                                # 源代码主目录
│   ├── server/                         # 后端服务代码
│   │   ├── index.ts                    # 服务入口，初始化 Express 应用与数据库连接
│   │   ├── routes/                     # 路由定义层
│   │   │   ├── resources.ts            # 资源列表查询、详情、状态更新接口
│   │   │   ├── favorites.ts            # 用户收藏增删查接口
│   │   │   └── stats.ts                # 点击统计与热度排行接口
│   │   ├── controllers/                # 请求处理控制器
│   │   │   ├── resourceController.ts   # 资源相关业务逻辑
│   │   │   └── userController.ts       # 用户偏好与收藏逻辑
│   │   └── models/                     # 数据模型与 ORM 映射
│   │       ├── Resource.ts             # 资源表结构定义（URL、标题、分类、状态）
│   │       └── ClickLog.ts             # 点击日志表结构
│   ├── client/                         # 前端界面代码
│   │   ├── pages/                      # 页面级组件
│   │   │   ├── Home.tsx                # 首页资源列表与筛选栏
│   │   │   ├── Favorites.tsx           # 个人收藏页面
```

## 贡献指南

我们欢迎并感谢任何形式的贡献，包括但不限于新增资源链接、改进文档、修复 Bug 或提交功能建议。请遵循以下步骤参与项目贡献。

第一，Fork 本仓库并从 `main` 分支创建您的特性分支。请使用描述性的分支名称，例如 `feature/add-resource-tags` 或 `fix/search-filter-error`。

第二，在本地完成您的修改后，请确保所有现有测试用例通过，并为新增功能编写相应的单元测试或集成测试。测试命令为 `npm run test`。

第三，提交代码时请遵循 Conventional Commits 规范，提交信息格式为 `<type>(<scope>): <subject>`，例如 `feat(resource): add batch import support`。这有助于自动生成变更日志。

第四，推送到您的远程仓库并发起 Pull Request 至本仓库的 `main` 分支。请在 PR 描述中清楚说明变更内容、动机以及相关 Issue 编号（如有）。

第五，项目维护者将在收到 PR 后的 3 个工作日内进行 Code Review，可能会提出修改建议。通过 Review 后您的代码将被合并至主分支，并出现在下一个版本发布中。

## 常见问题

Q: LinkVault 是否存储或缓存视频文件本身？

A: 不存储。LinkVault 仅保存视频链接的元数据（标题、分类、来源描述等），所有视频内容仍托管于原始平台。用户点击链接后直接跳转至源站，项目本身不承担视频传输带宽或版权责任。

Q: 如何批量更新资源链接的状态（如失效检测）？

A: 项目内置了 `scripts/check-links.ts` 脚本，可通过 `npm run check-links` 手动执行。该脚本会并发请求所有收录链接，根据 HTTP 状态码判断可用性并更新数据库。建议每周执行一次，也可配置系统定时任务自动运行。

Q: 能否将 LinkVault 部署到无服务器环境（如 Vercel 或 Cloudflare Workers）？

A: 当前版本依赖 SQLite 本地数据库与 Node.js 运行时，不适合直接部署至纯无服务器环境。但您可以使用 `npm run build` 生成静态前端文件，配合后端 API 部署至支持持久化存储的 PaaS 平台（如 Railway 或 Fly.io）。我们计划在后续版本中提供 PostgreSQL 支持以扩展部署选项。
