# YJDLB Resource Aggregator

YJDLB Resource Aggregator 是一个面向技术内容聚合与外部资源导航的开源项目，旨在帮助开发者、内容研究者与技术爱好者系统性地收集、分类和引用来自 yjdlb.com 域名下的综合内容资源。该项目定位为轻量级的外部资源元数据索引工具，通过结构化的 Markdown 文档和自动化脚本，将分散的深层链接转换为可维护、可追溯、可快速检索的本地知识库。

项目主要服务于需要定期跟踪特定内容源更新、进行批量链接可用性检测、或构建自定义导航页面的技术用户。通过本项目提供的资源清单与辅助工具，用户可以显著降低手动整理大量 URL 的时间成本，同时获得一套可扩展的资源管理框架。

## 功能概览

**批量链接导入与解析**：支持从原始数据源批量导入 URL，自动解析路径参数与内容类型标识，生成标准化的资源条目。

**资源分类索引**：根据 URL 路径中的分类标识（如 /zongyi/）自动归类，并提取文件名后缀作为内容类型标记，便于后续过滤与统计。

**可用性健康检查**：集成简单的 HTTP 状态检测脚本，可定期对资源列表中的每个链接进行可达性验证，输出异常报告。

**元数据提取模板**：提供可配置的元数据提取规则，支持从 URL 中解析潜在的内容主题关键词，生成人类可读的摘要信息。

**静态导航页生成**：基于资源列表自动生成静态 HTML 导航页面，支持按字母顺序或分类筛选，方便内网部署或本地浏览。

**增量更新机制**：支持通过差异对比工具识别新增或失效链接，仅处理变更部分，避免全量重建带来的性能开销。

## 应用场景

**技术文档外链管理**：技术团队在维护项目文档时，可将本项目的资源列表作为外部参考资料的统一入口，确保所有引用链接均经过初步校验和分类，减少文档中的死链数量。

**内容聚合站点构建**：内容创作者可利用本项目的静态页面生成功能，快速搭建一个以 yjdlb.com 资源为核心的主题导航站，无需后端开发即可实现资源的分类展示与检索。

**自动化监控报警**：运维人员可结合健康检查脚本与定时任务，对资源列表中的关键链接实施可用性监控，当检测到大量链接异常时触发报警，及时通知相关方处理。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/your-org/yjdlb-resource-aggregator.git

# 进入项目目录
cd yjdlb-resource-aggregator

# 安装依赖（Python 3.8+ 环境）
pip install -r requirements.txt

# 运行资源导入与验证脚本
python scripts/import_resources.py --input resources/raw_urls.txt --output resources/index.json

# 生成静态导航页
python scripts/generate_site.py --config config/site.yaml --output docs/index.html
```

│   ├── processed/                          # 处理后的结构化数据
│   │   ├── index.json                      # 资源主索引（JSON 格式）
│   │   └── metadata.db                     # SQLite 数据库，存储资源元数据与历史检查记录
│   └── reports/                            # 报告输出目录
│       └── health_check_20260624.log       # 可用性检查日志
├── docs/                                   # 文档与站点输出目录
│   ├── index.html                          # 生成的静态导航首页
│   ├── user_guide.md                       # 用户手册
│   └── developer_guide.md                  # 开发者指南

## 常见问题

**问：导入资源时遇到大量连接超时错误，应如何解决？**

答：这通常是因为网络环境或目标服务器限制所致。建议首先检查本地网络连接，确认能够正常访问 yjdlb.com 域名。如果网络正常，可尝试在 import_resources.py 脚本中调整 requests 库的超时参数（timeout 值），或者通过配置代理（设置 HTTP_PROXY 环境变量）来规避地域性访问限制。另外，也可以使用 check_health.py 脚本单独对超时链接进行重试，该脚本支持 --retry 参数指定重试次数。

**问：生成的静态导航页中，某些链接显示为失效，但实际上通过浏览器可以访问，如何解决？**

答：这种情况通常是由于目标站点启用了反爬机制（如 User-Agent 校验或 JavaScript 重定向），而健康检查脚本仅发送简单的 GET 请求。解决方法是在 check_health.py 的请求头中添加常见的浏览器 User-Agent 字符串，或者启用 allow_redirects 参数跟随重定向。如果问题依然存在，建议在配置文件中将检查模式从 HEAD 请求改为 GET 请求，并增加响应内容的关键词校验（如检测页面标题是否存在）。

**问：如何更新资源列表并重新生成站点？**

答：当需要添加、删除或修改资源 URL 时，只需编辑 data/raw/urls_835_batch.txt 文件，然后按顺序运行 import_resources.py 和 generate_site.py 脚本。项目内置的增量更新机制会自动对比新旧索引差异，仅处理变更的条目，因此重新生成站点的速度较快。如果需要完全重建索引，可传递 --force 参数强制执行全量处理。
