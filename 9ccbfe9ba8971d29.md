# LinkTide Resource Aggregator

LinkTide 是一个面向开发者和技术研究人员的结构化外链资源汇总平台，专注于对互联网上的技术文档、工具站点、开源项目镜像及数据资源进行系统化整理与索引。项目定位于为技术团队提供可复用、可扩展的外部资源引用基线，解决在项目调研、技术选型、文档编写和知识库建设过程中外部链接分散、失效风险高、引用格式不统一等问题。LinkTide 本身不存储任何第三方内容，仅作为导航与引用层，通过标准化条目格式和周期性可用性检查，帮助用户建立稳定的外部资源引用体系。本项目适用于需要维护大量外链引用的文档站点、技术博客聚合器、企业内部知识中台以及开源项目依赖链追踪场景。

## 功能概览

**结构化资源索引** 对每个收录的外链资源分配唯一内部标识符，并按照来源域名、资源类型、更新频率等维度进行多级分类，支持快速筛选与定位。

**可用性健康检查** 内置基于 HTTP 状态码和响应时间的周期探测机制，自动标记失效或响应过慢的资源链接，并生成异常报告。

**引用关系图谱** 记录资源之间的关联引用关系，支持展示某个资源被哪些条目引用，以及某个条目依赖了哪些外部资源，便于影响分析。

**元数据模板管理** 为不同类型资源（如技术文档、开源仓库、在线工具、数据API）提供差异化的元数据录入模板，确保每条资源描述信息完整且结构一致。

**批量导入与导出** 支持通过 CSV 和 JSON 格式批量导入外部链接清单，并支持将索引数据导出为 Markdown 表格、JSON 或 YAML 格式，便于集成到静态站点生成器或其他文档系统。

**标签与版本追踪** 允许为资源打上自定义标签，并记录资源内容的重大变更版本（如API版本更新、文档大版本发布），辅助用户判断引用的时效性。

## 应用场景

技术文档站点的外链管理 技术博客、开源项目文档站或 API 参考手册中往往包含大量指向第三方服务的链接。LinkTide 可以作为这些链接的统一管理后台，当外部资源地址变更或服务下线时，维护人员可通过 LinkTide 的检测报告快速定位并更新文档中的引用，避免文档中的链接腐烂。

技术选型调研辅助 在团队进行技术选型时，需要横向对比多个框架、工具或云服务的官方文档、社区活跃度指标、性能测试报告等外部信息。LinkTide 允许调研人员将候选技术的相关资源链接统一收录，并添加对比标签和备注，形成可共享的选型参考基线。

内部知识库的引用规范化 企业内部的 Confluence、Notion 或 GitBook 知识库中经常混杂着大量非标准格式的外链。LinkTide 提供统一的引用入口和标准化元数据，知识库编辑者只需在文档中引用 LinkTide 的资源 ID，即可保证所有外部链接的格式一致且可追溯，同时降低因人员变动导致的链接维护断层风险。

开源项目依赖链追踪 对于依赖大量第三方库和服务的开源项目，维护者可以使用 LinkTide 记录每个依赖项的官方网站、文档地址、Issue 追踪器和版本发布日志。当依赖项出现安全漏洞或重大更新时，能够快速评估影响范围并推动升级。

## 快速开始

以下步骤适用于在本地环境部署 LinkTide 服务并进行基础资源导入操作。

```bash
git clone https://github.com/linktide/linktide-core.git
cd linktide-core
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver --host 0.0.0.0 --port 8080
```

访问 http://localhost:8080 即可进入 Web 管理界面。首次启动时系统将自动创建默认管理员账户，用户名 admin，初始密码在控制台日志中输出。登录后可通过 资源管理 -> 批量导入 功能上传包含外链数据的 CSV 文件进行测试。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 至 3.11 | 核心运行环境，推荐使用 3.10 以获取最佳性能与兼容性 |
| PostgreSQL | 13.0 及以上 | 主数据库引擎，用于存储资源元数据、标签和检测记录 |
| Redis | 6.2 及以上 | 缓存与任务队列后端，用于异步执行健康检查任务 |

| Nginx | 1.20 及以上 | 生产环境推荐的反向代理服务器，用于负载均衡和静态文件服务 |
| Supervisor | 4.0 及以上 | 用于在生产环境中管理 LinkTide 后台任务进程（可选） |

## 项目结构

```
linktide-core/
├── src/                                # 核心源代码目录
│   ├── aggregator/                     # 资源聚合与索引模块
│   │   ├── fetcher.py                  # 基于 aiohttp 的异步资源抓取器
│   │   ├── parser.py                   # 响应内容解析与元数据提取
│   │   └── indexer.py                  # 资源条目入库与去重逻辑
│   ├── checker/                        # 健康检查任务模块
│   │   ├── probe.py                    # HTTP/HTTPS 探测策略实现
│   │   ├── scheduler.py                # 基于 APScheduler 的定时任务调度
│   │   └── reporter.py                 # 检测报告生成与通知输出
│   ├── web/                            # Web 管理界面后端
│   │   ├── routes/                     # Flask 蓝图路由定义
│   │   ├── models/                     # SQLAlchemy 数据模型 (Resource, Tag, CheckRecord)
│   │   └── schemas/                    # Marshmallow 序列化与反序列化模式
│   ├── frontend/                       # 前端静态资源源码
│   │   ├── components/                 # Vue 3 单文件组件
│   │   ├── views/                      # 页面级组件 (仪表盘、资源列表、导入导出)
```

更新文档与资源列表 若本次贡献涉及新增配置项、API 端点或命令行参数，请同步更新 docs 目录下的对应文档。若涉及新增外部资源索引条目，请按照 resources/schema.json 中定义的格式修改并提交对应的 JSON 文件，确保资源分类、标签和备注字段完整。

社区沟通渠道 项目核心开发团队在 Gitter 上设有公开聊天室，供贡献者讨论技术方案和解决阻塞问题。重大架构变更或新功能提案建议先在聊天室中发起讨论，达成初步共识后再着手编码实现，以减少 PR 被拒绝的风险。

## 常见问题

Q: 健康检查任务对目标服务器会造成较大压力吗？
A: LinkTide 的探测策略默认采用指数退避间隔，且每个目标资源的探测并发数限制为 2。对于同一域下的多个资源，系统会自动进行请求合并（若支持 HEAD 方法）以减少重复连接。默认检查频率为每 24 小时一次，每次超时时间为 10 秒。用户可以在配置文件中调整这些参数以适应自己的网络环境。

Q: 如何迁移已有的外链清单到 LinkTide？
A: 系统支持三种迁移路径：第一，通过 Web 界面的批量导入功能上传 CSV 或 JSON 文件，系统会自动校验 URL 格式并进行去重；第二，通过命令行脚本 scripts/import_batch.py 直接指定文件路径，适合大批量导入；第三，通过 REST API 的 /api/resources/batch 端点进行程序化导入。所有导入操作均会生成导入报告，列出成功、跳过和失败的条目明细。

Q: LinkTide 是否可以部署在完全离线内网环境中？
A: 可以。LinkTide 不强制要求外网访问权限，其核心功能（资源索引、元数据管理、引用关系图谱、导入导出）均可在内网中正常运作。仅健康检查模块中的外部探测功能在内网环境中会因无法访问外部目标而失败，但用户可以通过配置将检查策略调整为仅检测内网可达的服务，或者禁用探测任务，仅将 LinkTide 作为静态目录管理工具使用。
