# EON 技术资源导航站

EON 技术资源导航站是一个面向开发者和技术研究人员的结构化外链资源汇总平台，专注于收集、分类和索引互联网上分散的高质量技术文档、工具链入口、知识库文章及社区讨论帖。项目定位为技术领域的轻量级资源中枢，帮助用户快速定位到特定技术栈的官方文档、实践指南、故障排查案例以及架构设计参考，避免在分散的信息源中重复检索。

本项目不提供内容托管或镜像服务，仅作为 URL 索引与分类导航工具运行。所有资源链接均来自公开互联网，EON 项目团队不对链接指向的内容负责，也不对链接的可用性做持续性保证。使用者应自行评估各链接指向内容的安全性与适用性。

## 功能概览

**按技术领域分类索引** 将收录的 URL 按照编程语言、中间件、协议规范、安全审计、运维监控等维度进行初步分类，支持按类别浏览。

**快速关键词检索** 基于资源标题与摘要信息提供简单的关键词匹配检索能力，帮助用户从数百条链接中定位相关内容。

**资源状态标记** 对每个收录链接记录添加时间与可用性探测状态，辅助用户判断资源的时效性与可访问性。

**外部链接跳转中间页** 提供统一的跳转中间页，在离开本站前展示目标 URL 信息，降低误触恶意链接的风险。

**自定义分类标签系统** 用户可为收藏的资源链接添加自定义标签，实现个人化的分类管理。

**批量导入与导出** 支持通过 JSON 或 CSV 格式批量导入外部链接列表，也支持将当前索引导出为结构化数据文件。

**访问统计与热度排序** 统计各链接的被点击次数，按热度排序展示高频访问资源，辅助发现社区关注热点。

## 应用场景

技术团队内部知识库建设：团队技术负责人可将本项目部署为内部导航页，将团队常用的云服务控制台、代码仓库、CI/CD 流水线入口、监控面板等链接统一收录，新成员入职时可快速熟悉基础设施访问路径。

个人开发环境书签管理替代：开发者可将个人常用的技术博客、在线工具、API 参考文档、错误码查询网站等分散在各浏览器的书签统一迁移至 EON 导航站，通过标签分类和检索功能实现跨设备访问。

技术社区资源分享：开源社区维护者可将本项目作为社区资源索引页，将社区积累的教程、视频、幻灯片、示例项目等外链集中展示，降低社区成员的信息获取门槛。

技术培训与教学辅助：培训讲师可将课程所需的所有外部参考资料、实验环境入口、在线评测平台链接集中配置于导航站，学员通过单一入口即可访问全部教学资源，避免在课堂中反复复制粘贴 URL。

技术文档归档辅助：技术文档撰写者可利用本项目的分类和标记功能整理个人收藏的技术规范、白皮书、标准草案等长期参考材料，构建个人化的技术档案库。

## 快速开始

以下命令适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆代码仓库
git clone https://github.com/eon-tech/eon-navigation.git
cd eon-navigation

# 安装项目依赖
npm install

# 启动开发服务器
npm run dev
```

生产环境部署建议使用以下命令构建静态文件并启动生产服务：

```bash
npm run build
npm start
```

Docker 部署方式：

```bash
docker build -t eon-navigation .
docker run -p 3000:3000 eon-navigation
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 20.11.0 以上版本 |
| npm | 9.x 或 10.x | 包管理器，随 Node.js 一同安装 |
| SQLite | 3.35.0 以上 | 内置数据库，用于存储链接索引与标签数据 |
| Git | 2.30.0 以上 | 用于克隆仓库及版本管理 |
| 操作系统 | Linux (Ubuntu 20.04+), macOS 12+, Windows 10/11 (WSL2) | 支持主流操作系统，Windows 下建议使用 WSL2 以获得最佳性能 |
| 内存 | 最低 512MB，推荐 1GB 以上 | 开发模式下内存占用较高，生产环境可降低 |
| 磁盘空间 | 最低 200MB | 用于存放代码、数据库文件及日志 |
| 网络 | 可访问公网 | 用于拉取依赖包及访问收录的外部链接 |
| 浏览器 | Chrome 90+, Firefox 88+, Edge 90+ | 管理界面需要现代浏览器支持 ES6 语法 |

## 项目结构

```
eon-navigation/
├── src/                               # 源代码主目录
│   ├── server/                        # 服务端代码
│   │   ├── index.js                   # 服务入口，初始化 Express 应用与数据库连接
│   │   ├── routes/                    # 路由定义层
│   │   │   ├── api.js                 # RESTful API 路由，处理链接增删改查
│   │   │   ├── page.js                # 页面渲染路由，处理服务端模板输出
│   │   │   └── redirect.js            # 外链跳转中间页路由，记录点击日志
│   │   ├── controllers/               # 控制器层，实现具体业务逻辑
│   │   │   ├── linkController.js      # 链接资源管理逻辑
│   │   │   ├── tagController.js       # 标签系统管理逻辑
│   │   │   └── statsController.js     # 访问统计与热度计算逻辑
│   │   ├── models/                    # 数据模型层
│   │   │   ├── Link.js                # 链接实体模型，定义字段与验证规则
│   │   │   ├── Tag.js                 # 标签实体模型
│   │   │   └── ClickLog.js            # 点击日志模型
│   │   └── utils/                     # 服务端工具函数
│   │       ├── database.js            # SQLite 数据库初始化与连接池管理
│   │       ├── validator.js           # URL 校验与输入清洗函数
```

## 常见问题

Q: 如何批量导入外部链接？

A: 项目支持通过 JSON 格式批量导入。请在管理后台的「导入」页面下载模板文件，按照模板格式整理链接数据（包含 URL、标题、分类、标签等字段），然后上传 JSON 文件进行导入。单次导入上限为 1000 条记录，超出请分批操作。导入过程中系统会自动去重，重复 URL 不会被重复收录。

Q: 链接失效或指向恶意网站如何处理？

A: 项目内置了链接可用性探测功能，每天凌晨自动检测所有收录链接的 HTTP 状态码，状态码非 2xx 或 3xx 的链接将被标记为「异常」。用户也可在浏览时手动点击「报告链接问题」按钮进行反馈。管理员审核确认后会将失效或恶意链接移至隔离区，不再对外展示。对于被标记为异常的链接，系统会在跳转中间页给出安全提示。

Q: 本项目是否可以部署在内网环境，不访问外网？

A: 可以。项目核心功能（链接索引、分类管理、检索、跳转）均不依赖外网服务。仅需在首次部署时确保网络畅通以安装 npm 依赖，安装完成后即可在内网环境中独立运行。所有收录的外部链接均为用户自行添加，项目本身不主动请求外部资源，也不收集任何用户数据。
