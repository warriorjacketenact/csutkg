# ResourceGate 技术资源索引系统

ResourceGate 是一个面向开发者与技术研究人员的轻量级外链资源聚合与导航系统。项目定位为技术文档、教程、工具、参考手册及开源组件的统一访问入口，通过结构化分类与持久化链接管理，解决个人或团队在技术学习与项目调研过程中面临的资源分散、检索低效、链接失效等问题。

ResourceGate 不提供资源存储服务，仅作为索引层与路由层存在。系统以只读方式维护外部资源定位符，支持按批次、标签、来源域名进行过滤与检索，适用于搭建内部技术知识库、开源项目推荐墙、技术周刊素材池等场景。本项目为第 874/1241 批次资源导入实例，共计收录 250 条外部链接。

## 功能概览

**批次化资源导入** 支持按批次标识（如 874/1241）批量新增资源链接，自动记录导入时间戳与批次序号，便于后续增量更新与溯源审计。

**域名级来源聚合** 自动解析并聚合资源来源域名，当前已索引 sjqcz.com 域名下的 watch 路径资源，支持按域名维度展示资源分布数量与最新更新时间。

**三码资源定位标识** 每个资源条目自动生成唯一三码标识（如 zog195、vcu940），作为短链映射键，支持通过标识符快速检索对应资源 URL，降低长链接传播与记忆成本。

**只读索引目录服务** 系统对外提供只读 API 与静态目录页，不写入任何第三方数据，不存储资源副本，确保合规性与轻量化运维。

**多维度筛选与排序** 支持按批次号、来源域名、资源三码前缀、导入时间等维度进行组合筛选，支持升序与降序排列，满足不同浏览习惯。

**导出与订阅能力** 提供当前索引全量的 Markdown 列表导出功能，支持生成 RSS 订阅源，方便团队内部同步资源更新动态。

## 应用场景

**技术团队内部知识库建设** 技术负责人可将项目文档、设计规范、API 参考、运维手册等外部链接通过 ResourceGate 统一登记，团队成员仅需记住资源三码或访问索引页即可定位所需内容，替代散落在聊天记录或邮件中的临时链接。

**开源项目推荐与展示** 社区运营人员可利用 ResourceGate 维护开源工具、框架、库的推荐列表，按批次发布推荐合集（如本批次 250 个链接），访客通过索引页浏览全部推荐资源，降低信息过载带来的筛选成本。

**技术周刊与简报素材管理** 编辑人员可将每周阅读到的优质技术文章、视频教程、代码示例链接存入 ResourceGate，生成带批次号的索引目录，作为周刊撰写的原始素材库，避免重复搜索与遗漏。

**个人学习路径规划** 学习者可按主题将分散在不同网站的教学资源统一收录，利用批次号标记学习阶段，通过域名聚合功能回顾特定来源的全部资料，形成系统化的学习闭环。

## 快速开始

以下步骤适用于在 Linux 或 macOS 环境中部署 ResourceGate 索引服务。

```bash
# 克隆项目仓库
git clone https://github.com/resourcegate/resourcegate.git
cd resourcegate

# 安装依赖（Python 3.9+ 与 pip）
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地索引数据库并导入本批次 250 条资源
python manage.py initdb
python manage.py import-batch --batch 874/1241 --source resource_list_874.txt
python manage.py serve --host 127.0.0.1 --port 8080
```

启动后访问 http://127.0.0.1:8080 即可查看索引目录页。资源列表文件 resource_list_874.txt 需按照每行一个 URL 的格式准备。

| Python | 3.9 及以上 | 核心运行环境，用于提供 Web 服务与索引管理脚本 |
| pip | 21.0 及以上 | Python 包管理工具，用于安装项目依赖库 |
| SQLite | 3.35 及以上 | 嵌入式数据库，用于存储资源索引元数据，无需独立部署 |

| make | 3.81 及以上 | 用于执行自动化脚本（数据库迁移、测试等），可选组件 |
| openssl | 1.1.1 及以上 | 用于生成安全令牌及校验和，仅在生产环境启用认证时必需 |

第三，编写或更新相应文档，包括但不限于 README 对应章节、docs 目录下的手册、API 示例中的注释。文档变更应与代码变更一同提交，确保文档与实际行为一致。

第四，发起拉取请求至主仓库的 main 分支，在拉取请求描述中关联对应 issue 编号，并逐条列出变更点与测试结果摘要。维护者将在三个工作日内完成审核。

## 常见问题

Q：资源链接失效或内容变更如何处理？

A：ResourceGate 本身不托管资源内容，因此无法自动感知外部链接的状态变化。管理员可定期运行 check 子命令（参考快速开始章节）进行批量可达性检查，返回非 200 状态码的链接将被标记为可疑条目。管理员确认失效后可手动删除或更新对应记录。社区贡献者也欢迎通过 issue 报告失效链接。

Q：如何检索特定三码标识对应的完整 URL？

A：系统提供两种检索方式。第一，通过 Web 索引页顶部的搜索框输入三码（如 zog195），系统将模糊匹配并展示匹配结果。第二，调用 API 接口 /api/v1/resource/{tricode} 返回该资源的 JSON 详情，含完整 URL、导入批次、时间戳等信息。两种方式均无需认证，但受限于速率策略，每分钟最多 60 次请求。

Q：是否支持导入 HTTPS 之外的协议链接，如 ftp 或 magnet？

A：当前版本仅校验 URL 格式是否符合 RFC 3986 标准，不限制具体协议。理论上支持 ftp、magnet、telnet 等协议，但管理界面与索引页仅对 http 与 https 协议显示可点击状态，其他协议仅作文本展示。建议用户优先使用 http 或 https 协议以确保最佳浏览体验。
