# GBDLS Resource Index

GBDLS Resource Index 是一个面向技术研究、数据分析和学术检索的轻量级外链资源汇总系统。本项目不对资源内容进行二次加工，而是以结构化索引的方式，将分布于 gbdls.net 域名下的海量数据条目进行集中归类与快速导航，帮助研究人员、开发者和信息分析人员高效定位所需的外部参考链接。

本项目定位于技术资源索引层，不对目标链接的可用性、合法性和内容准确性作出任何明示或暗示的保证。用户应当自行判断目标资源的适用性，并遵守目标网站的使用条款。GBDLS Resource Index 本身不存储、缓存或转发任何外部资源内容，仅提供链接的整理与分类展示功能。

## 功能概览

**批量链接导入**：支持通过文本文件或标准输入流一次性导入大量 URL 条目，自动解析 gbdls.net 短链格式，生成内部记录。

**多维度标签分类**：每个资源条目可赋予多个分类标签，支持按主题、日期、来源类型等维度进行筛选与聚合展示。

**快速检索过滤**：基于内存索引的字符串匹配算法，支持对资源标题、描述和标签进行全文检索，检索响应时间低于 200 毫秒。

**访问状态监测**：定期对已收录的链接进行 HTTP 头探测，标记异常状态码，协助用户筛选有效资源。

**导入导出兼容**：支持 JSON、CSV 和纯文本列表格式的批量导入导出，便于与其他数据处理工具链集成。

**访问统计看板**：记录各资源条目的被查看次数，提供热门资源排行，辅助用户发现高价值链接。

## 应用场景

**技术文献检索**：研究人员在开展文献调研时，可通过本索引快速筛选特定主题域下的历史资源链接，大幅缩短人工逐条翻阅的时间成本。

**数据源追溯**：数据分析师在处理外部数据时，可利用本项目的分类标签体系快速定位相关数据来源链接，辅助数据溯源与交叉验证工作。

**外部链接资产管理**：内容运营团队可将本项目作为外部引用链接的统一登记中心，集中管理对外输出内容中引用的所有参考链接，便于后续维护与更新。

**信息安全分析**：安全分析人员可批量导入可疑链接列表，利用本项目的状态监测功能定期检查目标资源的存活情况，辅助威胁情报分析流程。

## 快速开始

以下命令演示了从代码仓库克隆项目、安装依赖以及启动服务的最小化操作流程。

```bash
git clone https://github.com/gbdls-resource/index.git
cd index
pip install -r requirements.txt
python app.py --init-db --import-links ./data/initial_links.txt
```

执行上述命令后，服务默认监听本地的 8080 端口。用户可通过浏览器访问 http://localhost:8080 查看索引主页。首次启动时，系统会自动创建 SQLite 数据库文件并导入初始链接列表。如需导入用户自定义的链接文件，可将文件路径替换为实际路径后重新执行导入命令。

## 安装要求

本项目的运行依赖以下基础软件与 Python 库，建议在干净的系统环境中进行部署。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | 3.9 及以上 | 核心运行环境，低于此版本将导致语法兼容性问题 |
| SQLite | 3.35 及以上 | 内置数据库引擎，用于存储链接元数据与标签关系 |
| requests | 2.28.0 及以上 | 用于发起 HTTP 请求，执行链接状态探测任务 |
| beautifulsoup4 | 4.11.0 及以上 | 可选依赖，用于解析目标页面的标题与描述信息 |
| lxml | 4.9.0 及以上 | 作为 beautifulsoup4 的解析器后端，提升解析性能 |
| flask | 2.2.0 及以上 | Web 服务框架，提供 RESTful API 和简易管理界面 |
| pytest | 7.2.0 及以上 | 仅开发测试时需要，用于执行单元测试与集成测试 |
| gunicorn | 20.1.0 及以上 | 生产环境推荐使用的 WSGI 服务器，不适用于 Windows 原生环境 |
| virtualenv | 20.16.0 及以上 | 推荐用于创建独立的 Python 虚拟环境，避免依赖冲突 |

## 项目结构

项目采用标准的 Python Web 应用目录布局，各模块职责清晰分离，便于后续扩展与维护。

```
gbdls-resource-index/
├── app/                                # 应用主包
│   ├── __init__.py                     # 包初始化，创建 Flask 应用实例
│   ├── routes.py                       # 路由定义，包含主页、检索、导入导出等端点
│   ├── models.py                       # 数据模型定义，包含 Link、Tag、CheckRecord 等 ORM 类
│   ├── services/                       # 业务逻辑层
│   │   ├── link_parser.py              # 链接解析服务，处理 gbdls.net/x/ 格式的短链扩展
│   │   ├── checker.py                  # 状态监测服务，异步执行 HTTP 探测任务
│   │   └── exporter.py                 # 导出服务，支持 JSON 与 CSV 格式输出
│   ├── templates/                      # Jinja2 模板文件
│   │   ├── base.html                   # 基础布局模板，包含导航栏与页脚
│   │   ├── index.html                  # 主页模板，展示资源列表与检索框
│   │   └── detail.html                 # 详情页模板，展示单个链接的完整元数据
│   └── static/                         # 静态资源文件
│       ├── css/                        # 样式表文件
│       │   └── style.css               # 自定义样式，覆盖 Bootstrap 默认主题
│       └── js/                         # 前端脚本文件
│           └── main.js                 # 检索交互与分页加载逻辑
├── data/                               # 数据存储目录
│   ├── links.db                        # SQLite 主数据库文件
│   └── initial_links.txt               # 初始链接列表文件，用于首次导入
├── tests/                              # 测试代码目录
│   ├── test_parser.py                  # 链接解析服务的单元测试
│   ├── test_checker.py                 # 状态监测服务的单元测试
│   └── test_routes.py                  # 路由端点的集成测试
├── scripts/                            # 运维脚本目录
│   ├── import_batch.py                 # 批量导入脚本，支持从文件读取并入库
│   └── cleanup_duplicates.py           # 重复条目清理脚本，合并相同链接的记录
├── docs/                               # 文档目录
│   ├── quickstart.md                   # 快速入门指南
│   ├── admin.md                        # 系统管理手册
│   ├── api.md                          # API 接口参考文档
│   └── development.md                  # 开发与贡献指引
├── requirements.txt                    # Python 依赖清单
├── app.py                              # 应用启动入口文件
└── README.md                           # 项目说明文件（本文档）
```

## 常见问题

问：导入大量链接时，系统是否会立即对每个链接执行状态探测？

答：不会。导入阶段仅执行链接格式校验和元数据提取，不进行实时的 HTTP 探测。状态探测任务由后台调度器按固定间隔批量执行，默认间隔为 24 小时，避免对目标服务器造成不必要的请求压力。

问：如何导出当前索引中的全部链接列表？

答：您可以通过访问 /api/export 端点并指定 format 参数来导出数据。例如，/api/export?format=csv 将返回 CSV 格式的完整列表，包含链接、标题、标签和最后检查时间等字段。该接口支持分页参数，适用于大规模数据导出场景。

问：项目是否支持私有化部署并离线运行？

答：支持。本项目完全基于本地 SQLite 数据库运行，不依赖任何外部在线服务或第三方 API。您可以在完全离线的内网环境中部署并正常使用所有核心功能，包括链接导入、检索和状态监测。唯一需要外部网络访问的功能是状态监测模块的 HTTP 探测，该模块可选择性关闭。
