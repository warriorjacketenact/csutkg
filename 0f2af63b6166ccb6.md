# ResourceHub

ResourceHub 是一个面向开发者、技术研究人员与内容创作者的精选技术资源导航站。项目定位于对互联网上分散的高质量技术文档、工具链、开源项目、学术论文与工程实践进行系统化收集、分类与索引，帮助用户在信息过载的环境中快速定位到真正有价值的内容。

本项目并非简单的书签集合，而是通过结构化的资源组织方式，配合自动化检测脚本与社区贡献机制，确保每条资源的可用性与时效性。项目面向的典型用户包括：正在进行技术选型的架构师、需要查阅最新规范的前后端工程师、关注基础设施运维的 SRE 团队，以及希望在特定领域构建系统知识图谱的研究者。ResourceHub 通过统一的资源描述格式和分类体系，解决了传统收藏夹难以共享、难以检索、难以维护的痛点。

## 功能概览

- **多维度资源分类**：资源按照技术领域、应用场景、受众层级进行三级标签体系划分，支持快速筛选与定位。

- **自动化可用性检测**：每日定时任务对所有收录资源进行连通性与响应状态检查，自动标记异常条目并发送通知。

- **结构化元数据描述**：每条资源包含标题、描述、标签、收录时间、更新频率、语言等十余个字段，便于二次加工与导出。

- **社区共建机制**：支持用户通过 Issue 与 Pull Request 提交新资源、更新失效链接或补充分类信息，所有变更经过审核后合并。

- **全文检索与高级过滤**：基于资源标题、描述、标签和分类内容构建全文检索引擎，支持按技术栈、难度等级、资源类型等多条件组合过滤。

- **资源关系图谱**：自动分析资源间的引用关系与共现标签，生成可视化知识图谱，辅助用户发现关联内容。

- **版本快照与变更追踪**：每次资源列表更新均生成版本快照，支持回溯任意历史版本的资源集合，便于对比变化。

- **个性化订阅与推送**：用户可根据标签或分类订阅资源更新，系统在新增或变更时通过 Webhook 或邮件推送通知。

## 应用场景

- **新技术选型调研**：技术负责人需要在短时间内了解某个新兴框架或工具的生态成熟度。通过 ResourceHub 的标签过滤与资源关系图谱，可以快速获取该领域的官方文档、社区实践、性能评测与迁移指南，大幅压缩信息搜集时间。

- **团队知识库初始化**：新组建的研发团队需要建立统一的技术规范与参考文档库。团队可以使用 ResourceHub 作为起点，批量导出特定标签下的资源列表，结合内部评审机制形成团队知识库基线。

- **个人系统化学习路径规划**：开发者希望从零开始系统学习某个技术领域（如分布式系统、机器学习工程）。ResourceHub 的资源分级标签与难度标识可以帮助学习者按照由浅入深的顺序组织学习材料，避免盲目跳跃。

- **开源项目文档站外链整合**：开源项目维护者需要在项目 README 或文档站中引用大量外部参考链接。维护者可以直接引用 ResourceHub 中对应分类的资源列表，或通过 API 动态获取最新资源集合，减少手动维护链接的负担。

## 快速开始

以下步骤帮助您在本地环境快速启动 ResourceHub 服务。

```bash
# 克隆代码仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 安装项目依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地数据库并导入示例资源数据
python manage.py migrate
python manage.py loaddata resources/fixtures/initial_data.json

# 启动开发服务器
python manage.py runserver
```

服务启动后，访问 http://127.0.0.1:8000 即可浏览本地资源列表。管理员后台地址为 http://127.0.0.1:8000/admin ，默认管理员账户为 admin / admin123（首次启动后请立即修改密码）。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 核心运行时，用于后端服务与定时任务调度 |
| PostgreSQL | 14.0 及以上 | 主要关系型数据库，存储资源元数据与用户信息 |
| Redis | 6.0 及以上 | 缓存与消息队列后端，用于异步任务与会话存储 |
| Node.js | 18.0 及以上 | 前端静态资源构建工具链运行环境（仅开发需要） |
| Elasticsearch | 7.17 及以上 | 全文检索引擎，可选组件，不安装时回退为 SQL 模糊搜索 |
| Docker | 20.10 及以上 | 容器化部署方式下的必要依赖（生产环境推荐） |

├── backend/                         # Django 后端服务主目录
│   ├── settings/                    # 分环境配置文件
│   │   ├── base.py                  # 基础配置
│   │   ├── development.py           # 开发环境配置
│   │   └── production.py            # 生产环境配置
│   ├── apps/                        # 所有 Django 应用
│   │   ├── resources/               # 资源管理核心应用
│   │   │   ├── models.py            # Resource, Tag, Category 数据模型
│   │   │   ├── views.py             # API 视图与页面渲染
│   │   │   ├── serializers.py       # DRF 序列化器

## 贡献指南

ResourceHub 的发展依赖于社区的共同参与。欢迎任何形式的贡献，包括但不限于提交新资源、更新失效链接、完善分类体系、改进代码实现和优化文档内容。

第一步：浏览项目 Issue 列表，查找标记为 "good first issue" 或 "help wanted" 的待办事项。若无适合的 Issue，可自行创建新 Issue 描述建议或问题。

第二步：Fork 本项目到个人账户，在本地新建分支进行修改。分支命名建议遵循 `feature/功能描述` 或 `fix/问题描述` 的格式。对于资源增删改的提交，需要附带更新资源的元数据字段（标题、描述、标签、分类）。

第三步：提交修改前运行完整的测试套件，确保所有单元测试与集成测试通过。新增资源导入功能时，需要补充相应的测试用例覆盖边界条件。

第四步：提交 Pull Request 到主仓库的 main 分支。PR 描述中需清晰说明变更内容、影响范围以及是否涉及破坏性变更。对于涉及资源列表的大规模修改，建议拆分为多个 PR 分批提交。

第五步：PR 至少需要一名项目维护者审核。审核通过且所有 CI 检查通过后，由维护者合并。合并后资源列表将自动触发重新索引与可用性检测任务。

## 常见问题

**Q：如何提交一条新的资源链接？**

A：推荐方式是通过 GitHub Issue 提交资源请求。请使用 Issue 模板中的 "资源提交" 类型，填写资源标题、原始 URL、所属分类、标签以及简短描述。项目维护者会在 7 个工作日内审核并决定是否收录。您也可以直接 Fork 仓库，在资源数据文件（位于 backend/apps/resources/fixtures/ 目录下）中添加条目，然后提交 Pull Request。对于批量提交（超过 10 条），建议先通过 Issue 与维护者沟通，避免审核流程拥堵。

**Q：某个收录的资源链接已经失效，应该如何处理？**

A：ResourceHub 每日自动检测所有收录资源的可用性，失效链接会被标记为 "异常" 状态并在列表页面置灰显示。如果您发现某个链接在自动检测中未被标记为失效，但实际无法访问，可以通过以下方式通知维护团队：在对应资源详情页点击 "报告失效" 按钮，或在 Issue 中使用 "链接失效" 模板提交报告。维护者将人工复核，若确认失效则会移除该条目或寻找替代链接。

**Q：如何在本地开发环境中重置数据库并重新导入所有资源？**

A：执行以下命令序列：首先运行 `python manage.py flush` 清空当前数据库数据（该操作不可逆，请提前备份）。然后运行 `python manage.py migrate` 重新应用所有数据库迁移。最后运行 `python manage.py loaddata resources/fixtures/initial_data.json` 导入初始资源数据集。如果您需要导入最新版本的外部资源列表，可以在 `scripts/` 目录下运行 `python import_from_csv.py --url https://www.6chuang.com/theater/export`（具体导入地址请查阅脚本帮助信息）。
