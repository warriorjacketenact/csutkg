# EON Resource Index

EON Resource Index 是一个面向技术研究、数据挖掘和知识管理领域的高质量外链资源汇总系统。本项目致力于对分散在互联网各角落的深层技术文档、数据看板、交互式工具及研究型页面进行系统化的索引与分类管理，帮助研究人员、开发者以及数据科学家快速定位所需信息，减少在无效链接和信息噪音中的时间损耗。

本项目服务于月活超过五万的技术从业者群体，涵盖机器学习工程、前端架构、系统运维、数据可视化等多个方向。EON Resource Index 本身不生产内容，而是通过严格的人工审核与自动化健康检查机制，维护一份高可用、高相关度的外部资源清单，并以清晰的项目结构提供给社区使用。通过本索引站，用户可以在一处汇聚分散于数十个独立域名的优质技术资料，实现跨站点、跨主题的一站式资源导航。

## 功能概览

**批量资源导入与解析**：支持从标准输入、文件导入或 API 接口批量录入 URL 资源，系统自动完成协议检查、重复去重及初始元数据抓取，便于快速建立私有资源池。

**自动化链接健康巡检**：内置基于异步 HTTP 请求的链接可用性检查模块，支持自定义超时时间与重试策略，每日定时输出失效链接报表，确保索引库长期稳定可用。

**多维度分类与标签系统**：每一笔资源均可标记多个自定义标签，支持按技术领域、内容类型、适用经验等级等多维度进行筛选与聚合，方便不同角色的用户按需浏览。

**全文搜索与模糊匹配**：基于倒排索引的轻量级检索引擎，支持对资源标题、描述关键词以及标签组合的快速检索，并提供拼写容错与同义词联想功能。

**个性化收藏与阅读列表**：用户可创建多个命名收藏夹，将常用或待读的资源归类保存，支持收藏夹的导入导出，便于团队内部分享研究素材。

**开放 API 与第三方集成**：提供 RESTful API 接口，允许开发者将本索引系统集成至内部知识管理平台、自动化运维脚本或自定义数据管道中，支持 JSON 与 XML 两种数据交换格式。

**访问统计与热度排行**：记录各资源链接的点击次数与最近访问时间，生成周榜与月榜，帮助社区发现当前热点技术文章与工具，促进优质内容的二次传播。

## 应用场景

**技术团队内部知识库建设**：技术团队可将 EON Resource Index 作为内部知识库的链接中台，将散落在团队 Wiki、Slack 历史记录和邮件列表中的外部参考资料统一入库，并结合标签系统按项目或技术栈分类，新成员入职时可快速了解团队常用的工具链与学习资料。

**数据科学研究项目的文献管理**：数据科学项目通常涉及大量论文链接、数据集地址和开源模型仓库。研究人员可利用本项目的资源列表功能，按阶段（数据清洗、特征工程、模型训练、结果评估）组织外部链接，配合健康巡检功能确保关键数据集在实验周期内始终可访问。

**技术博客与社区内容策展**：技术博主或社区运营人员可将 EON Resource Index 作为内容策展的后台，围绕特定技术主题（如 Rust 异步编程、Kubernetes 调度策略）收集并分类外部优质文章，通过本项目导出的资源清单可直接嵌入博客文章或社区置顶帖中。

**自动化数据采集管道的种子管理**：对于需要定期从特定来源采集数据的爬虫系统，本项目可作为种子 URL 管理端，运维人员通过 API 动态增删种子链接，系统自动检测链接有效性并记录响应时间变化，为调度策略调整提供数据依据。

## 快速开始

以下指令适用于 Linux / macOS / WSL 环境，要求系统已安装 Git 与 Python 3.9 及以上版本。

```bash
# 克隆项目仓库至本地
git clone https://github.com/eon-resource-index/eon-index.git
cd eon-index

# 安装项目依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地数据库并启动索引服务
python manage.py initdb
python manage.py runserver --host 0.0.0.0 --port 8080
```

启动成功后，访问 http://localhost:8080 即可进入本项目的 Web 管理界面。首次启动将自动创建默认管理员账户，用户名与密码将在控制台输出，请及时记录并修改。

## 项目结构

```
eon-index/
├── app/                                # 核心应用主目录
│   ├── __init__.py                     # 应用工厂与配置加载入口
│   ├── models/                         # 数据模型层（ORM 定义）
│   │   ├── resource.py                 # 资源实体模型，含 URL、标题、描述、标签
│   │   ├── user.py                     # 用户模型，含认证与收藏关系
│   │   └── check_log.py                # 健康检查日志模型，记录每次巡检结果
│   ├── services/                       # 业务逻辑层
│   │   ├── fetcher.py                  # 资源元数据抓取服务，含超时与重试逻辑
│   │   ├── checker.py                  # 链接健康检查服务，支持并发批量检测
│   │   └── indexer.py                  # 标签索引与全文检索服务
│   ├── api/                            # RESTful API 路由层
│   │   ├── v1/                         # API 版本 v1
│   │   │   ├── resources.py            # 资源的 CRUD 与检索接口
│   │   │   └── health.py               # 健康检查状态查询接口
│   └── web/                            # Web 管理界面路由
│       ├── dashboard.py                # 总览面板，显示统计与最近失效链接
│       └── collection.py               # 个人收藏夹管理界面
├── scripts/                            # 运维与工具脚本
│   ├── initdb.py                       # 初始化数据库表结构与默认数据
```

## 常见问题

Q: 健康检查模块判定链接失效的标准是什么？能否自定义阈值？

A: 健康检查模块默认采用三阶段判定：TCP 连接超时（默认 5 秒）、HTTP 响应超时（默认 10 秒）以及 HTTP 状态码不在 200-399 范围内。任一阶段失败即标记为失效。您可以在 config/production.py 中调整 CHECK_CONNECT_TIMEOUT、CHECK_READ_TIMEOUT 以及 CHECK_STATUS_WHITELIST 三个配置项，以适配不同的网络环境或业务需求。

Q: 本项目支持从其他书签管理工具迁移数据吗？

A: 目前支持从 Netscape HTML 书签导出格式（常见于 Chrome、Firefox 及 Edge 浏览器）以及 Delicious 的 JSON 导出格式进行迁移。您可在 Web 管理界面的「设置 - 导入」板块上传对应格式的文件，系统将自动解析并批量创建资源记录。其他格式的迁移工具正在开发中，欢迎贡献解析模板。
