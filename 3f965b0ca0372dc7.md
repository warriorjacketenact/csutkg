# VODLink Hub

VODLink Hub 是一个面向视频点播(VOD)服务提供商、内容分发网络(CDN)运维人员以及流媒体技术研究者的外链资源聚合与路由管理工具。该项目定位于构建一个结构化的视频内容标识符解析与转发枢纽，帮助用户统一管理分散在多平台、多区域的内容寻址标识，解决因资源标识符散乱导致的内容分发链路断裂、播放路由失效以及跨域调度困难等问题。

通过将大量异构的内容标识符（即各视频源的路径散列值）纳入标准化的路由表中，VODLink Hub 提供了可预测的、可审计的外部资源引用映射能力。其核心价值在于将非结构化的 URL 路径集合转化为具备业务语义的管理视图，降低运维复杂度与人工记忆成本，适用于需要对成百上千个外部视频资源进行集中引用的内容聚合平台。

## 功能概览

批量资源导入与标准化：支持一次性导入数以千计的原始视频资源外链，并自动执行协议一致性校验与路径规整，确保所有引用的资源定位符符合统一格式规范，降低因书写错误导致的访问异常。

智能路由映射表生成：基于导入的 URL 列表自动生成可配置的哈希路由映射表，为每个外链分配内部别名与访问策略，支持按业务线、地域或内容类型对资源进行逻辑分组。

定时健康检查与可用性探测：对已收录的所有外部资源端点执行周期性 TCP/HTTP 存活探测，自动标记不可用或响应超时的链接，并提供可视化的健康状态仪表盘。

外链变更追踪与版本记录：监控已收录 URL 的响应状态码与内容摘要变化，当目标资源发生迁移、删除或内容替换时记录差异日志，便于运维人员快速定位上游变更。

细粒度访问控制与白名单管理：支持按 IP 段、Referer 来源或用户代理特征设置外链资源的可访问策略，防止资源被非授权站点盗用或遭受恶意请求洪泛。

RESTful 管理 API 与批量操作接口：提供完整的 JSON 风格 API 用于增删改查已收录的外链资源，支持基于标签筛选、状态过滤以及批量启用/停用操作，可无缝融入现有运维流水线。

结构化日志与审计追踪：记录所有对外部资源的访问请求、路由决策以及管理操作行为，输出符合 ELK 或 Splunk 格式的日志流，满足安全审计与故障排查需求。

## 应用场景

内容聚合平台的外链资产整理：当运营中的视频门户或新闻媒体网站需要引用来自不同视频源提供商的数百个节目链接时，通过 VODLink Hub 统一录入各来源的播放页地址，生成内部路由编号，避免直接在页面模板中硬编码分散的外部 URL，从而便于后续批量替换或下线失效链接。

跨区域 CDN 调度的路由基准：对于部署了多套边缘节点的流媒体服务，每个节点可能对应不同的回源地址。运维团队将各节点的资源标识符录入 VODLink Hub，形成地域与 URL 的映射字典，调度层可根据请求来源地域快速检索对应的最优回源链接，提升内容加载速度。

开发测试环境的外链模拟：前端或移动端开发人员在构建播放器时，经常需要切换不同的测试视频源。通过预先在 VODLink Hub 中导入一组涵盖不同分辨率、码率和封装格式的测试外链，开发环境可按需引用这些标识符，避免频繁修改配置文件或重新编译代码。

自动化监控系统的探测目标配置：运维监控系统往往需要对外部依赖的资源可用性进行持续拨测。将全部第三方视频源地址录入 VODLink Hub 后，监控平台可直接从该项目的导出接口获取最新的探测目标列表，确保监控覆盖范围与资源台账同步。

## 快速开始

以下操作指导您在本机环境中克隆代码仓库、安装项目依赖并启动 VODLink Hub 的核心服务进程。

```bash
# 克隆代码仓库至本地工作目录
git clone https://github.com/your-org/vodlink-hub.git

# 进入项目根目录并安装所有必需的 Python 依赖包
cd vodlink-hub
pip install -r requirements.txt

# 初始化数据库表结构并导入示例外链数据（包含预置的 250 条资源记录）
python manage.py migrate
python manage.py load_initial_links

# 启动开发调试服务器，默认监听 127.0.0.1:8000
python manage.py runserver
```

│   │   └── health_prober.py                # 可用性探测调度引擎
│   ├── api/                                # RESTful API 接口层
│   │   ├── v1/                             # API 版本 v1 端点定义
│   │   │   ├── endpoints.py                # 资源增删改查路由
│   │   │   └── serializers.py              # 请求与响应序列化器

## 常见问题

Q: 健康检查探测模块为何需要系统安装 curl 命令，而不是使用 Python 的 requests 库？

A: 生产环境中的部分老旧视频源服务器仅支持 HTTP/0.9 或具有特殊的 SSL 握手行为，Python requests 库在处理这类边缘情况时可能报错或超时。curl 作为系统级工具具备更完善的协议兼容性与代理支持能力，且通过子进程调用可以精确控制超时和重试策略，避免 GIL 阻塞事件循环。

Q: 如何批量更新已收录的 250 条外链信息，例如统一替换域名或修改路径前缀？

A: 您可以使用项目提供的管理命令 python manage.py bulk_update_links --old-domain old.cdjinniu.com --new-domain www.cdjinniu.com，该命令会遍历所有记录并执行字符串替换。若需要更复杂的规则，可通过 REST API 的 /api/v1/links/bulk_patch 端点提交 JSON 格式的更新指令集。

Q: 导入的 URL 数量达到成千上万条时，路由解析性能是否会下降？

A: 路由解析模块采用前缀树（Trie）结构存储所有路径散列，查找时间复杂度为 O(L)，其中 L 为路径长度，与总记录数无关。同时 Redis 缓存层会缓存高频解析结果，对于 QPS 超过 1000 的场景，建议部署多个 API 工作节点并共享 Redis 后端以水平扩展。
