# EMHA Programme Resource Index

EMHA Programme Resource Index 是一个面向技术研究者、内容策展人与开源数据挖掘者的结构化外链资源汇总系统。本项目不对链接内容做二次分发，而是提供一套标准化的索引框架与元数据组织方式，帮助用户在大量分散的节目资源中快速定位、分类与筛选目标条目。

本项目的目标用户包括：需要批量处理外链资源的爬虫开发者、维护技术文档或知识库的文档工程师、以及对特定主题链接进行人工审核的内容运营人员。EMHA Programme Resource Index 不替代任何上游站点的功能，仅作为导航层存在，所有资源链接均指向原始来源。

## 功能概览

批量导入与解析 支持一次性导入大批量 URL 条目，自动识别路径层级与文件扩展名模式，便于后续批量操作。

分类标签生成 基于 URL 路径中的关键词与文件名哈希特征，为每个条目生成可配置的初步分类标签，降低人工归类成本。

去重与冲突检测 对导入的链接列表执行基于完整 URL 的精确去重，并检测路径冲突或命名不规范条目，输出警告日志。

自定义元数据附加 允许用户为每个链接附加自定义备注、优先级标记或状态标签，元数据以独立配置文件存储，不污染原始数据。

多格式导出 支持将索引结果导出为 JSON、CSV 或纯文本列表格式，适配不同下游工具链的输入需求。

只读代理校验 提供可选的只读 HTTP 头检查功能，验证目标链接的可访问性状态，但不发起完整内容下载，避免对上游造成压力。

增量更新机制 支持基于时间戳或内容哈希的增量更新策略，仅处理新增或变更的链接条目，减少重复计算。

## 应用场景

技术文档中的参考链接管理 技术团队在编写系统设计文档或运维手册时，往往需要引用大量外部资源。本项目可作为内部索引层，将所有参考链接统一录入并分类，文档中仅需引用索引 ID，方便后续统一更新链接状态。

数据爬虫的种子库维护 数据采集工程师通常需要维护一个大型种子 URL 列表。使用本项目的批量导入与校验功能，可以快速验证种子列表的有效性，过滤无效条目，并生成结构化的种子配置文件。

知识库周期性外链审计 知识库管理员需要定期检查外部链接是否失效或内容变更。本项目提供的只读校验与增量更新能力，可帮助管理员高效完成周期性审计任务，仅输出状态变更的条目，减少人工逐条点击的工作量。

离线资源映射表构建 在网络隔离或弱网环境下，技术人员可事先使用本项目导出完整的链接映射表，包含 URL 与本地缓存路径的对应关系，便于离线环境下的资源导航。

## 快速开始

以下步骤将指导您在本地环境中快速启动 EMHA Programme Resource Index 的基础功能。

```bash
# 克隆项目仓库
git clone https://github.com/emha-programme/resource-index.git
cd resource-index

# 安装依赖（使用 Python 3.9 及以上版本）
pip install -r requirements.txt

# 运行索引构建脚本，导入示例链接列表
python index_builder.py --input ./data/raw_links.txt --output ./data/index_output.json
```

上述命令会读取 `raw_links.txt` 中的链接列表，执行去重与基础分类，并将结果输出为 JSON 格式的索引文件。若需自定义元数据，可参考 `config/schema.yaml` 中的配置模板。

## 项目结构

```
emha-resource-index/
├── index_builder.py          # 主入口脚本，负责协调整个索引构建流程
├── requirements.txt          # Python 依赖列表，用于快速安装所需库
├── config/
│   ├── schema.yaml           # 定义输入输出数据结构规范与分类映射规则
│   └── validator_rules.json  # JSON Schema 格式的校验规则，用于严格检查链接格式
├── core/
```

## 常见问题

执行 index_builder.py 时提示 "ModuleNotFoundError: No module named 'yaml'"
该错误表示 pyyaml 依赖未正确安装。请先执行 pip install -r requirements.txt 安装全部依赖。若网络环境受限，可手动安装 pyyaml 与 requests 两个核心包。

去重后输出的条目数量少于预期
请检查输入文件是否为 UTF-8 编码，且每行仅包含一个 URL。若 URL 中包含查询参数或片段标识符，去重逻辑会基于完整 URL 进行匹配，因此不同查询参数会视为不同条目。如需忽略查询参数，可在 config/schema.yaml 中调整 normalize_query 配置项。

HTTP 校验功能返回大量超时错误
超时错误通常由网络环境或上游服务器响应缓慢导致。您可以在 utils/http_checker.py 中调整 timeout 参数（默认 5 秒）。若不需要校验可访问性，可在 index_builder.py 中通过 --skip-check 参数跳过该步骤。
