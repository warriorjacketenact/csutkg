# Tangalooma Resource Index

Tangalooma Resource Index 是一个面向技术研究者和内容聚合者的外部资源索引系统，专注于对分散在网络各处的技术文档、影视资料、学术论文链接进行结构化整理与快速检索。该项目不存储任何实际内容，仅提供索引与引用能力，适用于需要建立个人知识库、团队技术周报素材库或自动化采集管道原型的场景。

项目定位为轻量级元数据索引中间件，目标用户包括技术写作者、开源社区维护者、数据采集工程师以及信息管理研究人员。通过统一的条目编码体系与可扩展的分类结构，帮助用户在大量原始链接中建立可维护的引用关系，降低信息流失与重复查找的成本。

## 功能概览

自动条目编码生成：每个收录的链接均分配唯一内部标识符，支持按批次、来源、类型进行筛选与回溯。

多维度标签分类：支持按内容主题、文件类型、来源域名等维度对链接进行标记与分组。

原始链接直出模式：所有引用链接保持原始格式输出，不进行任何协议补全、域名规范化或路径改写，确保引用精确性。

静态索引构建：基于目录树与表格文档生成完整的资源导航页面，无需数据库依赖，适合静态托管。

批量导入与去重：支持一次性导入大批量链接，自动检测重复条目并提供合并建议。

快速检索接口：提供基于条目编号、关键词、分类标签的命令行与 HTTP 查询接口。

审计日志记录：记录每次索引更新操作，支持版本差异对比与回滚。

## 应用场景

技术周报素材整理：技术社区编辑可使用本索引系统收集当周值得关注的技术文章、工具发布与会议录像链接，按主题分类后生成周报草稿。

自动化采集管道原型：数据采集工程师可将本索引作为采集任务的输入清单，记录每个数据源的访问状态与更新时间，便于调试采集逻辑。

个人知识库外链管理：研究员或开发者可将散落在浏览器书签、笔记软件中的大量外链统一导入本索引，建立可检索、可归档的本地知识引用体系。

团队文档引用规范：技术团队可在内部文档中引用本索引生成的条目编码，替代直接暴露原始链接，便于后续链接迁移或更新时统一修改。

## 快速开始

以下命令演示如何从 GitHub 克隆项目、安装依赖并启动本地索引服务。

git clone https://github.com/tangalooma/resource-index.git
cd resource-index
pip install -r requirements.txt
python index.py --batch 477 --input links.txt --output output/

上述命令将处理 links.txt 中的链接列表，生成索引文件存放于 output/ 目录下。批次号 477/1241 表示当前处理的是全部批次中的第 477 批。

## 常见问题

问：如何处理链接失效或内容变更的情况？

答：本索引系统只存储链接本身，不存储内容。建议用户定期运行 index.py --check 命令对已有链接进行可达性检查，系统会输出失效链接列表供手动处理。对于失效链接，可更新原始输入文件后重新运行索引生成。

问：是否支持多批次增量更新？

答：支持。使用 --batch 参数指定不同批次号，系统会为每个批次生成独立的索引文件。如需合并多个批次，可使用 --merge 参数配合 --output unified.json 输出合并后的结果。

问：输出的 Markdown 文档中链接为何不显示为可点击形式？

答：本系统设计上遵循原始链接直出原则，确保引用精确性，避免因 Markdown 渲染差异导致链接被篡改。用户如需可点击链接，可通过后处理脚本将裸 URL 转换为 HTML 标签，但此操作不在本项目功能范围之内。
