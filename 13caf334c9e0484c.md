# WebIndex 资源导航系统

WebIndex 是一个面向技术研究者、开源社区贡献者和信息分析人员的结构化外链资源归集与导航系统。本项目定位于将分散在各类技术博客、官方文档、社区讨论与知识库中的高质量外链资源，按照分类体系进行系统化整理、版本追踪与快速检索，帮助用户从海量信息中精准定位所需技术资料。

本项目适用于需要频繁查阅外部技术文档的开发者、需要建立团队知识库的架构师，以及希望追踪特定技术领域动态的研究人员。通过标准化的资源描述格式与层级化目录结构，WebIndex 解决了外链资源易失效、分类混乱、检索效率低等常见问题，提供一套可本地部署、可定制扩展的资源管理方案。

## 功能概览

**层级化分类体系**：基于技术领域与资源类型建立多级分类目录，支持从顶层领域到具体文章页面的逐级下钻，便于按主题浏览资源。

**原始链接精确归集**：每条资源均以原始 URL 形式存储，保留完整的协议与路径信息，确保链接可追溯、可验证，避免二次跳转导致的失效风险。

**资源状态标记系统**：为每条资源记录添加状态标记，包括有效、待验证、已迁移等状态，方便维护团队定期巡检与更新。

**多维度检索接口**：提供按资源分类、按关键词、按更新时间等多种检索路径，支持命令行与 Web 界面两种查询方式。

**版本化资源快照**：每次资源列表更新时自动生成快照记录，支持回溯历史版本的资源构成，便于对比变化与恢复误删条目。

**批量导入导出机制**：支持从 CSV、JSON 等格式批量导入外部资源列表，也可将当前资源库导出为标准化格式用于备份或迁移。

**可扩展插件体系**：预留插件接口，允许开发者编写自定义解析器以支持新的资源来源格式，或增加自动化验证、自动分类等扩展功能。

## 应用场景

技术团队内部知识库建设：团队可将日常开发中参考的技术博客、API 文档、最佳实践文章等外链资源统一收录至 WebIndex，建立团队共享的技术资源池。新成员入职时可快速获取经过团队验证的优质资源列表，缩短学习曲线。

技术社区资源聚合与分享：开源社区维护者可使用 WebIndex 整理项目相关的生态资源，包括官方文档、社区教程、视频讲解、相关工具等。社区成员可通过统一的资源导航页获取完整的项目周边信息，降低信息分散带来的困扰。

个人技术阅读与学习管理：技术爱好者可利用 WebIndex 构建个人学习资源库，按技术领域分类收藏阅读材料，配合状态标记系统追踪阅读进度与资源有效性，形成可持续积累的个人知识资产。

技术调研与竞品分析辅助：在进行技术选型或竞品分析时，WebIndex 可辅助建立对比资源集，集中收录各竞品的官方站点、技术评测、用户反馈等外链，便于横向对比与综合分析。

## 快速开始

以下命令序列适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
git clone https://github.com/webindex/webindex.git
cd webindex
pip install -r requirements.txt
python scripts/init_db.py --config config/default.yaml
python app.py --port 8080
```

执行完毕后，在浏览器中访问 http://localhost:8080 即可进入 WebIndex 导航系统界面。默认管理员账户为 admin，初始密码在首次启动时输出于终端日志中，请及时修改。

## 常见问题

问：如何批量添加大量外链资源？

答：使用 scripts/import_batch.py 脚本，接受 CSV 或 JSON 格式的输入文件。输入格式要求包含 url、title、category 三个字段。具体格式样例参见 docs/user-guide/batch-import.md。导入前脚本会进行格式校验与链接可达性预检，失败条目会记录于 logs/import_errors.log 中供排查。

问：资源链接失效如何处理？

答：系统内置了链接验证服务 scripts/verify_links.py，可定期执行以检测所有已收录资源的 HTTP 状态码。发现失效链接时，系统会将该资源状态标记为 invalid，并记录失效时间。维护人员可定期查看状态为 invalid 的资源列表，尝试更新链接或将其移除。验证周期建议设置为每周一次，可通过 cron 或 systemd timer 自动调度。

问：如何迁移数据库至其他系统？

答：使用 scripts/export_db.py 将当前数据库导出为 SQL 转储文件或 JSON 格式的数据包。在目标系统上执行 scripts/import_db.py 导入即可。若目标系统使用不同数据库引擎（如 PostgreSQL），需在 config/production.yaml 中修改 database 配置段，系统启动时会自动根据配置生成对应的数据库 schema。
