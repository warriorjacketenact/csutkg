# ResourceHub

ResourceHub 是一个面向开发者、技术研究人员与内容创作者的轻量级技术资源导航与外链聚合工具。该项目定位于解决信息碎片化时代下高质量技术资源分散、检索路径长、收藏管理成本高的问题，帮助用户将分散于多个域名的技术文档、学习资料、示例项目、视频教程与工具站点统一纳入本地可索引、可分类、可快速访问的资源管理体系中。

ResourceHub 本身不存储任何实体资源内容，仅提供结构化的外链元数据组织能力与本地运行环境。目标用户包括需要维护团队技术文档索引的架构师、希望建立个人学习知识库的研发工程师、以及需要批量整理项目参考链接的产品与运营人员。通过本项目提供的标准化目录树与导航机制，用户可将任意数量外部链接纳入统一管理，并配合本地静态服务实现秒级检索与跳转。

## 功能概览

- **多源链接批量导入与管理**：支持通过文本配置或脚本批量录入外部链接，系统自动解析 URL 结构并归类至预设分类节点。

- **分层级目录导航系统**：基于三级目录结构（项目根目录/分类目录/资源条目）组织资源，支持按技术领域、内容类型、来源域名等维度快速过滤。

- **本地轻量级静态服务**：内置基于 Node.js 或 Python 的静态服务器，一键启动后可在本地网络内通过浏览器访问资源面板，无需数据库依赖。

- **资源元数据缓存与健康检查**：对每条外链记录标题、添加时间、访问次数与最后检测状态，支持定期批量检测链接可用性并标记失效条目。

- **可定制化分类模板**：提供预设分类模板（文档、视频、工具、示例、社区、博客），用户可根据实际链接特征调整分类名称与层级关系。

- **全文检索与标签过滤**：基于链接标题、域名、分类路径与用户自定义标签建立简易倒排索引，支持关键词模糊匹配与多标签组合筛选。

- **纯静态导出与部署**：支持将当前资源列表与导航页面整体导出为纯静态 HTML/CSS/JS 文件，便于托管至任意 Web 服务器或对象存储服务。

## 应用场景

- **团队技术文档集中入口**：研发团队可将内部 Wiki、API 文档、设计稿原型、测试报告等散落于不同系统的链接统一收录至 ResourceHub，作为团队知识入口首页，新成员入职时可快速了解项目全貌。

- **个人学习路径整理**：开发者可将自己在学习 Rust、Kubernetes、机器学习等方向时收藏的官方文档、优质博客、视频课程、开源示例库按阶段分类，形成可复用的学习路线图，避免重复检索。

- **技术资讯与周报素材库**：技术编辑或社区运营人员可将每日浏览到的行业动态、发布公告、技术提案等链接存入 ResourceHub，并按日期或主题打标，为撰写周报或技术资讯汇总提供结构化素材源。

- **离线环境下的资源索引**：在无法访问公网的内网开发环境中，ResourceHub 可预先导入所有依赖的项目镜像地址、私有仓库链接、内部工具站点，配合本地静态服务实现内网资源导航。

## 快速开始

以下步骤帮助您在五分钟内完成 ResourceHub 的克隆、依赖安装与本地运行。

```bash
# 1. 克隆项目仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 2. 安装依赖（使用 npm 或 yarn）
npm install
# 或
yarn install

# 3. 启动本地开发服务
npm run dev
# 或
yarn dev

# 4. 在浏览器中访问
# 默认地址：http://localhost:3000
```

若您偏好使用 Python 环境，也可使用内置的轻量服务器脚本：

```bash
# 使用 Python 3 启动静态服务
python -m http.server 8080 --directory public
# 访问 http://localhost:8080
```

## 安装要求

| 依赖项 | 必需版本 | 说明 |
|---|---|---|
| Node.js | >= 16.0.0 | 用于运行构建脚本与开发服务器，推荐使用 LTS 版本 |
| npm | >= 8.0.0 或 yarn >= 1.22.0 | 包管理器，用于安装项目依赖 |
| Python 3 | >= 3.8 （可选） | 若使用 Python 静态服务器模块，需确保安装 |
| Git | >= 2.30.0 | 用于克隆仓库与版本管理 |
| 现代浏览器 | 最新版 Chrome / Firefox / Edge | 用于访问管理界面，需支持 ES6 与 Fetch API |
| 磁盘空间 | >= 50 MB | 存放项目源码、依赖包及缓存索引文件 |
| 网络连接 | 外网访问（首次导入链接时） | 用于解析链接标题与元数据，之后可离线运行 |

├── src/                            # 核心源代码目录

## 常见问题

**问：ResourceHub 是否支持导入带查询参数或哈希片段的复杂 URL？**

答：支持。系统在导入时会对 URL 进行完整记录，不会主动去除 `?` 后的查询字符串或 `#` 后的锚点片段。检索和跳转时均保留原始输入。但健康检查模块在执行连通性测试时会自动忽略片段部分，仅针对基础地址发送请求，以避免服务端因锚点问题返回非预期状态码。

**问：批量导入大量链接（超过 1000 条）时页面会出现卡顿，如何优化？**

答：当链接总数超过 500 条时，建议启用分页模式或虚拟滚动。您可以在 `custom.json` 中设置 `pagination.enabled` 为 `true` 并指定 `pageSize`。同时，初始加载时系统默认只解析前 200 条链接的元数据，其余链接仅在滚动或翻页时按需解析，以降低首屏渲染压力。若需全量解析，可运行 `scripts/health-report.js` 生成离线报告。

**问：如何在多台机器之间同步我的链接收藏与分类配置？**

答：ResourceHub 的所有资源数据与配置均存储于 `public/data/` 目录下的 `links.json` 和 `config.json` 文件中。您可以将这两个文件纳入版本控制（如 Git）或通过云存储同步。项目本身不提供云端存储服务，但支持通过导出功能生成单一 JSON 备份文件，并在其他机器上使用导入脚本恢复。
