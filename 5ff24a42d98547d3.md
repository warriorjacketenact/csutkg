# LinkVault 企业级外链资源聚合系统

LinkVault 是一个面向技术团队、内容运营者和研究人员的企业级外链资源聚合与导航系统。该项目定位于解决海量外部链接分散管理、重复采集、访问失效以及缺乏结构化分类等痛点问题，提供一套完整的资源采集、清洗、存储与展示解决方案。

LinkVault 的目标用户包括运维工程师、数据爬虫开发者、SEO 策略分析师以及需要批量维护外部链接资源的知识库管理员。通过统一的管理控制台，用户可以按业务维度对资源进行标记、版本追踪和可用性监控，显著提升链接资源的复用率与维护效率。当前版本为第 523/1241 批次，累计管理资源链接共计 250 条。

## 功能概览

**资源批量入库** 支持通过 CSV 或 JSON 格式批量导入链接清单，系统自动进行去重校验和协议规范性检查。

**分类标签引擎** 基于链接路径特征和页面 Meta 信息，自动为资源打上业务属性标签，支持人工二次修正。

**定时健康巡检** 内置异步任务调度器，可配置周期性 HEAD 请求检测，自动标记失效或响应超时的链接。

**多维度检索过滤** 按域名、路径前缀、状态码、标签、入库时间等条件组合筛选，快速定位目标资源。

**访问统计看板** 记录每个资源的点击频次、最后访问时间和引用来源，为运营决策提供数据支撑。

**权限分级管理** 支持管理员、编辑员、访客三级角色划分，控制资源的增删改查与导出权限。

**数据快照导出** 支持将当前筛选结果导出为 Markdown 表格或结构化 JSON，便于嵌入文档或二次开发。

## 应用场景

**技术文档站外链管理** 技术团队在维护项目文档时，往往需要引用大量外部规范、SDK 下载页或社区讨论帖。LinkVault 可集中收纳这些链接，并在文档构建流水线中通过 API 调用获取最新有效地址，避免文档内出现死链。

**爬虫任务链接队列维护** 数据采集工程师可为不同的爬虫任务配置独立的链接队列，通过 LinkVault 的状态标记控制爬取进度，同时利用巡检功能提前剔除已失效的入口 URL，减少无效请求对采集效率的影响。

**企业知识库资源归档** 企业内部培训资料、政策文件或行业报告常分散于多个共享目录或邮件附件中。LinkVault 允许运营人员将这些分散的外部参考链接按主题分类归档，形成可检索的外部知识索引层。

**SEO 外链建设效果追踪** SEO 运营人员可将已发布的外链资源录入系统，定期监测目标页面的可访问性和页面标题变化，辅助评估外链的存续状态和内容相关性变更情况。

## 快速开始

以下指令适用于 Linux / macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkvault/linkvault-core.git

# 进入项目根目录
cd linkvault-core

# 安装 Python 依赖（推荐使用虚拟环境）
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 初始化 SQLite 数据库表结构
python manage.py migrate

# 导入示例资源数据（包含本批次 250 条链接）
python manage.py loaddata seed/batch_523_1241.json

# 启动开发服务器，默认监听 8000 端口
python manage.py runserver
```

访问 http://127.0.0.1:8000 即可进入控制台首页，默认管理员账号为 admin，密码请在首次启动后查看控制台输出日志获取临时令牌。

## 贡献指南

**问题反馈与建议** 使用 GitHub Issues 提交缺陷报告或功能请求，请附带清晰的重现步骤、系统环境版本以及相关日志片段。建议使用项目提供的 Issue 模板，标注严重级别和影响范围。

**代码提交规范** Fork 项目后创建特性分支，遵循 PEP 8 编码规范，并确保所有单元测试通过（执行 pytest 即可）。提交信息请采用 type(scope): subject 格式，例如 fix(probes): 修复超时任务未释放连接池的问题。

**文档补充完善** 欢迎修订用户手册、API 示例或故障排查章节。文档位于 docs/ 目录下，使用 reStructuredText 格式，修改后需执行 make html 验证渲染效果，避免出现语法警告。

**三方集成扩展** 如需为 LinkVault 增加新的巡检协议（如 TCP 端口探测、HTTP 状态码校验）或告警通道（如 Slack、企业微信），请参考 dev-guide/plugin-architecture.rst 中的接口规范编写独立模块，并附带单元测试用例。

**审核与合并** 所有 Pull Request 需至少两名项目维护者 Approve 后方可合并。合并前会自动执行 CI 流水线，包含代码风格检查、安全漏洞扫描和覆盖率门禁（要求不低于 85%）。

## 常见问题

**Q: 健康巡检任务触发后，部分链接始终显示超时，但浏览器访问正常，如何排查？**

A: 该情况通常由服务器网络环境或防火墙策略导致。请按以下步骤排查：首先检查运行 LinkVault 的主机是否能够通过 curl 命令直接访问目标链接（注意携带相同的 User-Agent 头）；其次，查看 probes 模块的日志文件，确认是否启用了代理配置（检查 .env 中的 HTTP_PROXY 变量）；最后，可以在巡检配置中调整超时阈值（默认 5 秒）和重试次数（默认 2 次）。若目标站点存在反爬机制，建议在资源详情页手动将状态修正为「已验证」。

**Q: 从旧版 SQLite 迁移到 PostgreSQL 后，部分中文字段出现乱码，如何修复？**

A: 该问题通常源于字符集不一致。请在迁移前确保 PostgreSQL 数据库使用 UTF8 编码（执行 SHOW SERVER_ENCODING 确认）。若已迁移完成，可执行 ALTER DATABASE your_db SET client_encoding = 'UTF8'; 并重启服务。对于已存储的乱码数据，建议重新执行数据导入操作，而非直接转换。详细迁移步骤可参考 docs/ops-guide/migration-to-postgres.rst 章节。

**Q: 资源列表导出为 Markdown 时，链接过长导致表格排版混乱，是否有优化方案？**

A: 在导出界面勾选「缩短显示」选项，系统将自动使用链接的路径末尾标识符替代完整 URL 显示，并在表格末尾追加「完整地址」列以供复制。同时，您也可以在导出前对列表进行域名过滤，分批导出不同来源的链接，以控制单表宽度。该功能在 v2.1.0 及以上版本中默认启用。
