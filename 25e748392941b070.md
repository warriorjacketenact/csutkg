# LinkVault Core

LinkVault Core 是一个面向技术团队与个人开发者的外链资源归集与结构化索引系统。项目定位于解决分散在各类技术文档、社交媒体与内部 Wiki 中的高价值链接难以统一管理、检索与共享的问题。LinkVault Core 不提供爬虫或自动采集功能，而是通过标准化的 Markdown 条目模板、标签体系与目录规范，帮助用户将零散的 URL 资源转化为可维护、可审计、可版本控制的知识资产。目标用户包括开源项目维护者、技术内容运营团队、研发效能组以及需要长期跟踪特定领域信息源的研究人员。

## 功能概览

**多级分类索引** 支持用户为每条链接分配至少两级分类标签，并内置常用技术领域分类词典，便于按主题聚合。

**元数据模板系统** 提供预定义的 Markdown frontmatter 模板，包含标题、来源作者、收录日期、失效检测状态、简要摘要等字段，确保链接录入格式统一。

**批量导入与校验** 支持从 CSV 或结构化 Markdown 列表批量导入 URL，自动校验 URL 格式、去重，并标记疑似失效链接。

**静态站点生成集成** 内置轻量级脚本，可将链接库一键渲染为静态 HTML 导航页面，适合部署为团队内部知识门户或项目官网子站点。

**失效链接巡检报告** 周期性对已收录链接进行 HTTP 状态检查，生成失效链接清单，并支持导出为 CSV 以便人工复核。

**全文检索与过滤** 基于链接标题、摘要、分类标签和备注字段提供简单的关键词过滤与正则匹配检索，不依赖外部搜索引擎。

## 应用场景

开源项目文档站点的外部参考链接管理。开源项目的 README 或用户指南中常引用大量外部依赖、教程或 API 文档链接。LinkVault Core 可作为子模块集成至项目仓库，将散落链接集中存放于 `references/` 目录，并通过脚本自动生成链接状态徽章，帮助用户快速识别文档中失效的外部引用。

技术团队内部周报与技术雷达的资料归档。研发团队在周报或技术分享中会提及大量文章、工具仓库与视频链接。LinkVault Core 提供统一的条目模板，团队成员可按照既定规范将链接提交至共享仓库，由维护者审核合并，形成可持续积累的技术情报池。

个人知识库中的外链去重与备份。开发者在使用 Obsidian、Foam 或 Logseq 等工具构建个人知识体系时，可通过 LinkVault Core 提供的 CLI 工具对外链进行统一格式化、去重和离线摘要备份，防止个人笔记中的链接因原始页面变更而丢失上下文。

技术课程或培训材料的配套链接索引。讲师或课程设计者可利用 LinkVault Core 为每期课程建立独立的链接集，按课时或主题分类，并生成可打印的 Markdown 清单，方便学员课后查阅。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，需预先安装 Git 与 Node.js 18 LTS 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# 安装项目依赖
npm install

# 执行初始化配置，并启动本地开发服务
npm run init
npm run dev
```

执行 `npm run dev` 后，控制台会输出本地预览地址，默认为 `http://127.0.0.1:4173`。首次运行时会自动生成示例链接数据与默认分类配置。

## 贡献指南

1. 阅读项目 `docs/specs/template.md` 文档，理解链接条目标注规范与分类树结构，确保新增或修改的条目符合统一格式要求。

2. 从 `data/links/` 目录下选取尚未分配或标注为 `unreviewed` 的链接文件，按照模板补充完整元数据，包括标题、摘要、标签和来源日期。

3. 在本地通过 `npm run test` 执行全部单元测试与格式校验，确保新增条目未引入格式错误或重复记录。若测试失败，请根据控制台输出修正对应条目。

4. 提交 Pull Request 时，请使用项目提供的 PR 模板，并在描述中说明链接来源、分类依据以及是否经过有效性检查。若为批量导入，需附加导入脚本的执行日志。

5. 项目维护者会在 48 小时内进行 Review，可能要求补充来源说明或调整分类。合并后，链接将进入下一轮周期性巡检队列。

## 常见问题

问：如何批量导入大量链接而不逐一录入？
答：使用 `bin/lv-add.js --batch` 参数并指向一个 CSV 文件，CSV 表头需包含 `url`、`title`、`category` 三列。导入前建议使用 `--dry-run` 进行预检，查看可能的格式错误或重复项。

问：链接失效检查是否会访问外部网站，是否影响隐私？
答：检查器仅发送标准 HTTP HEAD 请求，不抓取页面内容，不记录任何 Cookie 或请求头中的额外信息。检查结果仅保存在本地 `data/audit/` 目录中，不会自动上传。用户可在配置文件中关闭自动检查。

问：能否将 LinkVault Core 部署为多人协作的在线服务？
答：项目本身定位为本地优先的静态工具，不包含用户认证或数据库组件。若需团队协作，建议将 `data/` 目录置于 Git 仓库中，通过分支和 PR 流程进行协同。对于需要 Web 界面的场景，可自行封装 API 层并调用核心库函数。
