# LinkVault

LinkVault 是一个面向技术研究者和内容聚合场景的轻量级外链资源管理与导航系统。本项目定位于为开发团队、技术社区及个人知识库提供结构化的外部链接收录、分类索引与快速检索能力，解决海量分散链接难以整理、难以共享、难以追踪变更的痛点。LinkVault 不依赖复杂的前端框架，以静态站点生成方式输出可部署的导航页面，同时提供命令行工具用于链接可达性检测与元数据抓取，适用于中小规模技术资源库的日常运维。

## 功能概览

**批量链接导入**：支持从 CSV、JSON 及纯文本列表批量导入 URL，自动去重并识别失效链接。

**分类标签体系**：允许为每条链接绑定多个自定义标签，支持基于标签的快速筛选与聚合视图。

**可达性健康检查**：内置异步 HTTP 探测模块，可定期检查链接可用性，输出状态报告并标记异常条目。

**元数据自动补全**：尝试从目标页面提取标题、描述及图标，减少手动录入工作量。

**静态站点生成**：将链接数据渲染为响应式 HTML 导航页，无需数据库，可直接托管于任何静态服务器。

**全文检索**：基于标题、描述、标签及 URL 路径提供轻量级关键词搜索，支持中英文分词。

**导入导出兼容性**：数据存储为 JSON 格式，可导出为 Markdown 表格或 HTML 列表，便于嵌入技术文档或 Wiki。

## 应用场景

技术团队内部知识库管理：开发团队可将常用 API 文档、运维手册、内部工具地址纳入 LinkVault 统一管理，替代浏览器书签的分散保存方式，新成员入职时可直接通过导航页访问所有必需资源。

开源项目文档站外链整合：开源项目维护者可在项目文档中嵌入 LinkVault 生成的资源列表页，集中陈列依赖库、参考文章、社区论坛及镜像站地址，提升文档的完整性与可维护性。

技术社区资源聚合：技术社区运营方可利用 LinkVault 按月或按主题聚合用户提交的优质外链，生成每周资源精选列表，通过静态页面对外发布，降低内容管理成本。

个人研究素材库：研究人员可将论文、专利、数据集及工具站按课题分类存档，配合全文检索快速定位历史收藏的参考资料，避免重复检索与遗忘。

## 快速开始

以下命令演示了从克隆代码到启动本地服务的完整流程：

```bash
git clone https://github.com/example/linkvault.git
cd linkvault
pip install -r requirements.txt
python linkvault.py --serve --port 8080
```

执行上述命令后，LinkVault 将在本地 8080 端口启动开发服务器，默认加载 `data/links.json` 中的示例数据。用户可通过 `--import` 参数导入自定义链接列表，例如 `python linkvault.py --import ./my_links.csv`。

## 常见问题

**问：导入大量链接时出现超时或内存占用过高，应如何优化？**

答：LinkVault 默认使用并发度为 10 的异步探测池，若链接数量超过 1000 条，建议将 `config/default.yaml` 中的 `checker.concurrency` 调整为 5 以降低系统负载。此外，可启用 `checker.cache_ttl` 参数，将可达性结果缓存 24 小时，避免重复探测同一域名。对于内存优化，可分批导入，每次处理 200 条，使用 `--batch-size` 参数控制单次提交量。

**问：生成的静态页面中的链接无法正常访问，如何排查？**

答：首先确认链接本身是否可访问，可使用 `python linkvault.py --check` 命令单独运行健康检查，查看输出日志中的状态码与错误信息。若链接返回 4xx 或 5xx 状态码，检查目标服务器是否屏蔽了 LinkVault 的 User-Agent（默认设置为 `LinkVault/1.0`），可在配置中修改 `checker.user_agent` 为常见浏览器标识。若为网络代理环境，需在 `config/default.yaml` 中配置 `checker.proxy` 字段。

**问：如何将 LinkVault 与其他工具（如 Obsidian 或 Notion）联动？**

答：LinkVault 支持导出为 JSON 与 Markdown 表格两种格式。对于 Obsidian，可使用 `scripts/export_markdown.py` 生成表格，直接粘贴至 Obsidian 笔记中。对于 Notion，建议先导出为 CSV，再通过 Notion 的导入功能批量创建数据库条目。后续版本将考虑增加 Notion API 直接同步能力，目前可通过自定义脚本调用 `core/importer.py` 中的转换函数实现中间格式适配。
