# LinkVault Resource Aggregator

LinkVault is a curated, machine-readable resource index and external link aggregation system designed for developers, researchers, and technical content curators who need to organize, validate, and distribute large volumes of URL-based references. Unlike traditional bookmarking tools, LinkVault treats links as first-class data entities with metadata, freshness scoring, and categorical tagging.

This project addresses the fundamental problem of link rot, context loss, and poor discoverability in sprawling reference collections. By providing a structured repository with automated health checks, semantic grouping, and exportable catalogs, LinkVault enables teams to maintain high-quality external resource libraries that remain useful over time. It is particularly suited for documentation hubs, academic reference managers, and open-source project wikis that depend on third-party content.

## 功能概览

**Bulk URL Ingestion** – Import hundreds of links from plaintext lists, CSV files, or JSON feeds with automatic deduplication and normalization.

**Metadata Enrichment** – Automatically fetch page titles, descriptions, Open Graph tags, and content-type headers for each submitted URL using a non-blocking worker queue.

**Health Monitoring** – Schedule periodic HEAD and GET requests to detect broken links, TLS certificate expiry, and unexpected status code changes, with configurable retry policies.

**Categorical Tagging** – Apply hierarchical tags (e.g., "documentation/api", "blog/tutorial", "video/recorded") based on URL patterns, content analysis, or manual overrides.

**Search and Filter** – Full-text search over titles, descriptions, and tags, combined with facet filters for status, category, and last-checked timestamp.

**Export Pipelines** – Generate static markdown catalogs, JSON dumps, sitemap XML, or CSV reports for integration with static site generators and data analysis tools.

**Access Control** – Support for read-only public views and write-protected administrative interfaces, with API key authentication for programmatic updates.

## 应用场景

**Documentation Maintenance** – A technical writing team maintains a large "awesome list" style resource page. Using LinkVault, they ingest candidate URLs, automatically fetch titles and descriptions, and periodically scan for broken links. The system generates an updated markdown table that they commit to their docs repository each week.

**Academic Reference Curation** – Research groups collect hundreds of external datasets, preprints, and tool repositories across multiple projects. LinkVault provides a shared index with tag-based filtering (e.g., "dataset/geospatial", "paper/nlp"), enabling lab members to quickly locate relevant resources without duplicating efforts.

**Open-Source Project Wiki** – A large open-source framework maintains a community-driven ecosystem page listing plugins, tutorials, and case studies. LinkVault's health monitoring alerts maintainers when third-party sites go offline, and the export pipeline generates a clean markdown list that stays in sync with the project's GitHub wiki.

**Internal Developer Portal** – An enterprise platform team curates internal tool links, API references, and runbooks. LinkVault's access control separates public-facing URLs from internal-only resources, while the search and filter interface helps new engineers onboard faster.

## 快速开始

```bash
git clone https://github.com/your-org/linkvault.git
cd linkvault

# Install dependencies using pipenv or virtualenv
pip install -r requirements.txt

# Configure environment variables
cp .env.example .env
# Edit .env with your database URL and worker settings

# Initialize database schema
python manage.py migrate

# Start the development server
python manage.py runserver --host 0.0.0.0 --port 8080

# In a separate terminal, start the health check worker
python worker.py --queue health-check
```

## 常见问题

**问：LinkVault 是否支持导入非公开的或需要认证的 URL？**

答：目前 LinkVault 的健康检查和元数据抓取仅针对公开可访问的资源执行。对于需要 HTTP 基本认证、API 密钥或 Cookie 的 URL，系统会将其标记为 "restricted" 状态并跳过自动探测。您可以通过配置环境变量 FETCHER_ALLOWED_AUTH_TYPES 来启用对特定认证类型的实验性支持，但官方不建议在生产环境中抓取需要凭证的资源。

**问：如何自定义健康检查的频率和超时阈值？**

答：所有健康检查参数均通过环境变量或 configs/.env 文件控制。主要变量包括 HEALTH_CHECK_INTERVAL (默认 86400 秒，即每天一次)、HEALTH_REQUEST_TIMEOUT (默认 10 秒) 和 HEALTH_RETRY_COUNT (默认 3 次)。修改这些值后需要重启 worker 进程。对于单个 URL 的覆盖，可以通过 API 更新记录中的 check_override 字段。

**问：批量导入 250 个以上 URL 时，系统如何处理速率限制？**

答：导入 API 支持批量提交，但后端 worker 会使用令牌桶算法限制每秒发出的 HTTP 请求数量，默认值为 10 请求/秒，以避免对目标服务器造成压力。您可以通过环境变量 FETCHER_RATE_LIMIT 调整该值。对于大型批次，建议使用异步导入模式，即提交后立即返回任务 ID，通过任务状态端点查询进度，避免 HTTP 超时。
