# LinkVault Resource Aggregator

LinkVault 是一个面向技术社区和内容研究者的结构化外链资源汇总平台，专注于从特定数据源中提取、分类和呈现高质量的视频内容链接。本项目并非视频托管站点，也不提供任何视频文件存储或流媒体服务，而是作为一个纯粹的导航与索引层，将分散于同一域名下的不同内容入口进行系统化收集与展示。

项目主要服务于以下四类用户：需要批量获取特定主题视频页面地址的数据采集工程师；研究内容平台 URL 结构与分类体系的信息架构师；希望通过固定入口快速访问大量垂直领域视频资源的普通浏览者；以及需要将此类外链集合嵌入自身应用或文档系统的开发者。

LinkVault 的核心价值在于将原本零散的、仅通过路径后缀区分的视频详情页聚合为可读、可检索、可监控的资源清单，并提供稳定的文档化输出格式，便于自动化工具链集成。

## 功能概览

**批量外链提取**：从指定基础域名下递归扫描 /vodshow/ 路径，提取全部视频详情页 URL，并以结构化列表形式输出。

**路径语义解析**：对每个 URL 的路径后缀（如 xjqxmr.html）进行可配置的映射处理，支持将无意义字符串转换为可读标题或分类标签。

**分类标签生成**：根据路径特征与预设规则库，为每个资源自动生成至少一个主题分类，便于按领域检索。

**去重与健康检查**：内置链接去重机制，并支持对每个 URL 进行基础的可访问性探测，标记异常条目。

**多格式导出**：支持 Markdown 表格、JSON 数组、纯文本列表三种导出格式，适配不同使用场景。

**增量更新支持**：提供差异比对功能，当源站新增或移除视频页面时，仅输出变化部分，减少全量同步开销。

**自定义过滤规则**：允许用户通过正则表达式或关键词列表过滤特定后缀或路径模式，仅保留目标资源。

**元数据扩展接口**：每个 URL 条目可附带自定义备注字段，用于记录抓取时间、状态码、响应大小等调试信息。

## 应用场景

**内容聚合站点的导航页构建**：个人站长或小型内容团队可利用 LinkVault 快速生成某个领域（如影视、教育、娱乐）的外部资源导航，将本项目的输出列表直接嵌入网站侧边栏或专属页面，提升用户跳转效率。

**数据分析与 URL 模式研究**：数据科学家或爬虫开发者可通过本项目输出的完整链接清单，分析目标站点的路径命名规律、参数传递方式以及目录层级深度，为后续编写定制化采集脚本提供样本依据。

**自动化监控系统的输入源**：运维或测试工程师可将 LinkVault 生成的链接列表作为定时健康检查任务的输入，批量探测每个页面是否可访问、响应时间是否正常，从而及时发现站点故障或内容下架。

**文档系统的参考资料附录**：技术文档撰写者或知识库维护人员可将本项目输出的资源清单作为附录材料，附加在教程、研究报告或项目手册末尾，为读者提供延伸阅读入口。

## 快速开始

以下步骤将指导您在本地环境中快速搭建 LinkVault 并生成第一份资源列表。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/linkvault.git

# 进入项目根目录
cd linkvault

# 安装依赖（使用 pip 安装 Python 3.8+ 所需包）
pip install -r requirements.txt

# 运行核心采集脚本，生成资源列表（默认输出 Markdown 格式）
python crawl.py --base-domain https://www.365myfl.com --output-format md --output-file resources.md
```

执行上述命令后，程序将自动扫描 https://www.365myfl.com/vodshow/ 路径下的所有页面，提取并去重后生成 resources.md 文件。您可使用任意 Markdown 阅读器打开该文件查看结果。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.8 或更高 | 核心运行环境，提供异步 I/O 与 HTTP 请求支持 |
| requests | 2.28.0 或更高 | 发送 HTTP 请求获取页面内容，处理重定向与超时 |
| beautifulsoup4 | 4.11.0 或更高 | 解析 HTML 文档，提取页面中的链接与元数据 |

| urllib3 | 1.26.0 或更高 | requests 的底层依赖，管理连接池与 SSL 验证 |
| certifi | 2022.06.15 或更高 | 提供根证书包，确保 HTTPS 请求的合法性验证 |

## 常见问题

**问：采集过程中遇到 HTTPS 证书验证错误如何处理？**

答：若目标站点使用自签名证书或证书链不完整，可在 config.yaml 中将 verify_ssl 参数设置为 false，但请注意此操作会降低安全性，仅建议在测试环境中使用。生产环境请优先更新 certifi 包或手动导入正确的根证书。

**问：输出列表中包含大量重复 URL，如何解决？**

答：LinkVault 默认启用基于完整 URL 字符串的去重机制。若仍出现重复，请检查源站是否在 /vodshow/ 路径下存在软链接或别名映射。您也可以启用 strict_dedupe 模式（在 config.yaml 中设置），该模式会同时比对 URL 的解析路径和查询参数。

**问：如何仅采集特定时间之后新增的链接？**

答：使用 scripts/incremental.py 辅助脚本，配合 --since 参数传入起始日期（格式 YYYY-MM-DD），程序会将当前采集结果与历史记录进行比对，仅输出新增条目。首次运行时需要先生成一份完整的基准列表。
