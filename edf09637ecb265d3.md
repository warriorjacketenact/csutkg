# WebIndex Resource Aggregator

WebIndex Resource Aggregator 是一个面向技术调研、内容聚合与知识管理场景的轻量化外链资源汇总系统。该项目定位于帮助开发者、技术内容创作者以及运维人员高效收集、分类、展示和管理来自多个数据源的结构化链接资源，解决信息碎片化导致的检索效率低下与资源复用困难问题。

本项目提供一套标准化模板与自动化工具链，能够将原始链接列表转换为带分类索引、可读性良好的静态文档站点或 Markdown 目录输出，适用于构建内部技术文库、开源导航站、论文参考文献库以及项目文档外链附录等场景。所有资源条目均保留原始 URL 完整性，支持批量校验、去重与元数据扩展。

## 功能概览

**批量链接导入与清洗**：支持从 CSV、JSON 纯文本列表批量导入 URL，自动去除空白字符、重复条目并校验协议格式，输出标准化资源清单。

**多层级分类索引**：基于正则表达式与关键词匹配规则，为每条链接自动生成分类标签，支持用户自定义分类映射表，实现按主题、来源或批次分组展示。

**静态站点生成器集成**：内置 Handlebars 模板引擎，可将资源数据渲染为响应式 HTML 页面，包含搜索框、分类筛选器和分页导航，便于部署为独立导航站。

**Markdown 目录导出**：一键生成符合开源项目 README 风格的资源列表章节，支持按预设小节（如官方文档、社区讨论、示例项目）组织输出，直接用于项目文档。

**链接可用性监控**：集成异步 HTTP 探针，定期检测资源链接的响应状态码与访问延迟，标记失效链接并生成健康报告，保障资源库质量。

**元数据扩展字段支持**：每条资源可附加标题、描述、维护状态、收录日期等自定义属性，扩展数据以 YAML Front Matter 形式存储，便于后续二次处理。

**权限与版本管理**：基于 Git 进行资源变更追踪，支持分支管理不同版本资源集，提供简单的审核流程，确保新增或修改链接经过确认。

## 应用场景

**技术团队内部知识库构建**：技术团队在日常开发中积累大量外部参考链接（API 文档、博客教程、调试工具），使用 WebIndex 可以将这些分散的浏览器书签统一入库，生成带分类索引的团队知识导航页，新成员入职时可快速了解常用资源分布。

**开源项目外链附录维护**：开源项目 README 或官方文档通常需要引用大量第三方依赖、学习资料或社区讨论帖。WebIndex 的资源列表导出功能可帮助维护者批量更新外链章节，确保所有 URL 格式统一且经过可用性校验，减少手动编辑错误。

**学术文献数据管理**：研究者在文献调研阶段需要收集数十至数百篇论文链接、数据集地址或工具仓库地址。WebIndex 支持为每条链接添加研究主题标签、阅读状态和笔记备注，输出为结构化的文献索引表，便于后期撰写文献综述时快速回溯。

**运维监控面板关联资源整理**：运维团队负责管理多个监控系统、日志平台和报警工具的管理后台地址。通过 WebIndex 分类索引功能，可按环境（生产/预发布/测试）或服务类型（计算/存储/网络）分组整理，生成内部运维导航页面，减少紧急情况下查找入口的时间。

## 快速开始

以下命令演示如何克隆项目、安装依赖并运行本地开发服务器，将示例资源列表渲染为静态站点。

```bash
# 克隆项目仓库
git clone https://github.com/webindex/webindex-aggregator.git
cd webindex-aggregator

# 安装依赖（使用 npm）
npm install

# 运行开发服务器（默认监听 3000 端口）
npm run dev
```

执行上述命令后，访问 `http://localhost:3000` 即可查看示例资源导航页面。若要导入自定义资源列表，请将包含 URL 的文本文件放入 `./data/raw/` 目录，然后运行 `npm run build` 重新生成站点。

## 常见问题

**Q：导入的链接列表中包含非 HTTP/HTTPS 协议（如 ftp、mailto），解析器如何处理？**

A：默认校验规则仅保留 HTTP 与 HTTPS 协议条目，其余协议会被记录到跳过日志中，并在生成报告时以警告形式提示。若需支持其他协议，可在 `src/config/default.yaml` 中修改 `allowedProtocols` 列表，扩展允许的协议类型。

**Q：构建生成的静态页面包含大量资源条目，首次加载速度较慢，如何优化？**

A：项目支持分页与懒加载配置。在 `src/config/default.yaml` 中设置 `pagination.pageSize` 控制每页显示条目数，建议值为 50 至 100。同时可启用 `lazyLoad` 选项，使图片与外部资源按需加载。对于超大规模资源集（超过 1000 条），建议启用增量构建模式，仅重新生成发生变动的页面。

**Q：链接可用性监控是否会对外部站点造成压力？探针频率如何控制？**

A：监控模块采用单线程顺序请求，默认超时时间为 5000 毫秒，并发数限制为 3。在 `src/config/default.yaml` 中可调整 `monitor.interval`（检测间隔，默认 24 小时）和 `monitor.timeout` 参数。生产环境建议将监控任务部署为独立的定时任务（如每日凌晨执行），避免频繁检测对第三方站点产生不必要流量。
