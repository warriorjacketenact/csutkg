# WebVOD Resource Aggregator

WebVOD Resource Aggregator 是一个面向多媒体技术研究人员、网络内容分析学者以及数字资源管理者的高密度外链聚合系统。本项目并非一个传统意义上的内容提供平台，而是一个经过精心编目的技术资源导航工具，专注于收集和整理与网络视频点播（VOD）技术架构、流媒体传输协议、内容分发网络策略及多媒体数据封装格式相关的深层链接。

项目定位为技术研究辅助设施，目标用户包括流媒体工程师、CDN运维人员、数字版权保护技术研究者以及网络协议分析爱好者。系统通过自动化校验机制对收录的资源端点进行可用性探测与元数据抽取，帮助用户快速定位特定技术栈下的参考实现、规范文档或实验环境入口，有效降低在复杂网络环境中进行技术调研时的时间成本与信息过滤负担。

## 功能概览

**自动化链接健康度检查**：系统每日执行异步任务对收录的每一个资源端点进行TCP连接测试与HTTP状态码验证，自动标记异常链接并生成可用性报告，确保资源列表的实时有效性。

**多维度资源分类体系**：依据资源端点的路径特征、响应头信息及页面元数据，将收录链接自动归入视频编码、传输协议、播放器内核、DRM方案、性能优化等十余个技术类别，支持按分类筛选与批量导出。

**元数据智能抽取引擎**：对于可访问的资源页面，系统自动抽取页面标题、关键描述、技术栈标签、最后更新日期等元数据，构建结构化的资源卡片，便于用户快速评估资源价值。

**自定义标签与注释系统**：用户可为任意资源链接添加自定义标签、技术备注和使用心得，形成个人化的知识注解层，所有注释数据支持云端同步与团队共享。

**RESTful API 开放接口**：提供完整的只读API接口，允许第三方工具以编程方式查询资源列表、获取元数据详情及拉取健康状态快照，便于与其他技术栈集成。

**数据快照与版本回溯**：每日自动生成资源列表的完整数据快照，支持按时间点回溯任意历史状态，为长期跟踪资源演变趋势提供数据基础。

**批量导入与导出机制**：支持以JSON、CSV和Markdown表格格式批量导入外部链接清单，同时支持将当前资源列表导出为多种结构化格式，满足离线分析与归档需求。

**访问统计与热度分析**：记录每个资源链接的被访问次数与访问趋势，生成热度排行榜，辅助识别社区关注度较高的技术资源与新兴热点方向。

## 应用场景

**流媒体技术栈选型调研**：技术决策者在评估不同流媒体解决方案时，可通过本项目快速获取涵盖HLS、DASH、RTMP等主流协议的实际部署案例与参考实现链接，横向对比各方案的实现复杂度与社区活跃度，为选型决策提供翔实的素材支撑。

**CDN节点性能对照实验**：CDN运维工程师可利用项目中收录的分布于不同地理位置的边缘节点资源，进行实际的拉流测试与首屏时间比对，评估各厂商的调度策略与缓存命中率，据此优化自有CDN架构的路由规则。

**学术研究中的语料采集**：从事网络多媒体传播研究的学者可将本项目作为数据源索引，批量获取特定时间范围内活跃的VOD服务端点，用于构建大规模流媒体流量模型或分析自适应码率算法的实际部署比例。

**自动化监控系统的目标源配置**：运维开发人员可将项目API输出的健康资源列表直接对接至自建监控系统，作为拨测任务的初始目标池，避免因手工维护监控目标列表而产生的遗漏与滞后问题。

## 快速开始

```bash
# 克隆项目代码仓库
git clone https://github.com/webvod/aggregator.git

# 进入项目根目录并安装依赖
cd aggregator
pip install -r requirements.txt

# 初始化本地配置文件并启动服务
cp config.example.yaml config.yaml
python main.py --init-db --sync-resources
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，用于执行资源抓取、元数据处理及API服务 |
| SQLite | 3.35 及以上 | 默认元数据存储引擎，支持并发读取与每日快照归档 |
| Redis | 6.2 及以上 | 可选组件，用于缓存高频访问的资源元数据以提升API响应速度 |

| beautifulsoup4 | 4.11.0 及以上 | HTML解析库，用于从资源页面抽取标题、描述等元数据 |
| pyyaml | 6.0 及以上 | 配置文件解析器，用于加载应用配置与环境变量 |
| apscheduler | 3.10.0 及以上 | 任务调度库，驱动每日健康检查与数据快照生成 |
| uvicorn | 0.21.0 及以上 | ASGI服务器，用于生产环境部署API服务进程 |
| pytest | 7.2.0 及以上 | 开发环境依赖，用于执行单元测试与集成测试套件 |

│   │   ├── checker.py                   # 异步健康检查器，实现TCP/HTTP探测
│   │   ├── crawler.py                   # 元数据抓取与解析协调器
│   │   ├── scheduler.py                 # 每日定时任务编排器

│   │   ├── routes.py                    # 路由注册与请求分发
│   │   ├── schemas.py                   # Pydantic数据校验模型
│   │   └── middleware.py                # 速率限制与认证中间件
│   ├── models/                          # 数据模型与ORM映射
│   │   ├── resource.py                  # 资源链接实体模型
│   │   ├── metadata.py                  # 元数据实体模型
│   │   └── tag.py                       # 标签与注释实体模型
│   ├── utils/                           # 通用工具函数集
│   │   ├── http.py                      # httpx客户端封装与重试策略

## 常见问题

**问：资源列表中的链接无法访问怎么办？**

系统每日自动执行健康检查，检测到不可用链接后会在API响应中标记其状态为inactive。用户亦可手动触发单链接即时检查，具体方法请参考用户手册中的相关章节。对于持续不可用的资源，系统会在连续七次检查失败后自动将其移至归档区并从活跃列表中移除。

**问：如何将本项目部署为长期运行的后台服务？**

生产环境推荐使用systemd或supervisor管理进程，配合uvicorn启动API服务。详细部署步骤、环境变量配置说明以及Nginx反向代理示例均收录于运维手册之中。对于高并发场景，建议增加Redis缓存层并配置多worker进程模式。

**问：是否支持导入自定义的私有资源列表而不公开共享？**

支持。用户可在本地配置文件中指定私有数据源路径，系统会将其与公共资源列表合并后一并提供检索与检查服务，私有数据仅存储在本地，不会同步至公共仓库或API接口。
