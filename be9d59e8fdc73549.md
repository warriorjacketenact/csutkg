# TangalooHouse Film Resource Aggregator

TangalooHouse Film Resource Aggregator 是一个面向影视技术研究者、数据归档工程师与媒体资源管理者的高密度外链索引系统。本项目不对任何外部资源的内容进行二次托管、转储或代理分发，仅以结构化索引方式对目标路径进行可验证的收集与分类。项目定位为技术资源外链汇总站，适用于需要批量维护、校验与回溯特定编码格式影片元数据链接的工作流。

本项目并不提供直接的播放、下载或内容预览功能，其核心交付物是一组可被脚本化调用的稳定 URL 列表及其附属的分类标记，便于下游系统进行可用性检测、元数据抓取或访问频率控制。目标用户包括影视数据管道开发者、归档合规审计人员以及从事媒体指纹分析的研究者。

---

## 功能概览

**批量链接状态巡检** 提供对全部收录链接的 HTTP 响应码与重定向链路的批量检测能力，支持导出检测报告。

**按影片编码分组索引** 所有链接依据 URL 路径中的影片编码段（如 nst-3347、smi-9185）自动生成分组视图，便于按制作公司或批次筛选。

**路径哈希指纹校验** 每个链接末尾的随机路径段（如 thhy.html、dfsmlk.html）可作为内容指纹的组成部分，支持与外部元数据库进行交叉比对。

**只读型静态数据集** 本项目维护的链接列表为只读快照，不对目标资源进行任何写入操作，确保引用层的稳定与可复现。

**Markdown 原生表格输出** 所有索引数据以 Markdown 表格形式呈现，兼容主流代码托管平台与文档渲染引擎，无需额外解析器。

**低依赖运行环境** 仅依赖标准 Python 3 环境与 requests 库，无需数据库或容器化支持即可完成基础巡检任务。

---

## 应用场景

**影视数据管道构建** 数据工程师可将本项目收录的链接作为输入源，编写分布式爬虫或异步下载任务，批量获取目标页面内容用于后续的元数据提取与结构化存储。

**归档链接有效性审计** 归档管理员定期运行链接检测脚本，检查各资源路径是否仍可访问，并记录 404、301、超时等状态变化，生成合规性报告。

**媒体指纹交叉验证** 研究者提取每个链接末尾的路径哈希，与第三方影视数据库的指纹库进行匹配，用于确认影片版本、剪辑时长或地区发行差异。

## 快速开始

以下指令演示如何克隆本仓库、安装依赖并执行基础链接巡检脚本。

```bash
git clone https://github.com/tangaloohouse/film-resource-aggregator.git
cd film-resource-aggregator
pip install -r requirements.txt
python check_links.py --input RESOURCES.md --output report.json
```

执行上述命令后，系统将读取 RESOURCES.md 文件中的所有链接，逐条发起 HEAD 请求，并将状态码、响应时间、重定向链记录至 report.json 文件中。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.8 或更高版本 | 是 | 运行时基础解释器，用于执行链接巡检与数据导出脚本 |
| requests 库 | 是 | 发送 HTTP 请求并处理响应状态码与重定向 |
| markdown 库 | 否 | 仅在需要将 Markdown 表格转换为其他格式时使用 |
| pytest 库 | 否 | 运行单元测试以验证链接解析函数的正确性 |
| flake8 库 | 否 | 代码风格检查，用于贡献者提交前的本地校验 |
| pre-commit 钩子 | 否 | 可选安装，用于自动执行提交前的格式与语法检查 |

## 文档导航

| 层面 | 目录 | 回答的问题 |
|---|---|---|
| 索引层 | RESOURCES.md | 当前收录的全部外链地址及其原始路径是什么 |
| 工具层 | scripts/check_links.py | 如何对全部链接进行批量状态检测并输出结构化报告 |
| 配置层 | config/settings.yaml | 巡检超时时间、重试次数、并发数等参数如何调整 |
| 元数据层 | metadata/fingerprints.csv | 每个链接对应的路径哈希与影片编码如何映射 |
| 变更层 | CHANGELOG.md | 每次版本迭代新增、移除或修正了哪些链接 |
