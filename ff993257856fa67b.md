# VueBest 资源导航系统

VueBest 是一个面向 Vue.js 生态系统的技术资源聚合与导航平台，专为前端开发者、架构师和技术团队设计，用于集中管理、分类检索和快速访问高质量的 Vue 相关学习资料、组件库、工具链及社区项目。本项目并非一个传统的框架或库，而是一个结构化的外链资源索引系统，通过人工精选与社区贡献机制，持续维护一份高信噪比的 Vue 技术地图。

本项目解决的核心问题包括：Vue 生态中资源分散、质量参差不齐导致的发现成本过高；官方文档之外的中文实践案例与深度解读内容缺乏统一入口；以及团队内部技术选型时缺乏可参考的权威外链依据。VueBest 通过频道化分类、标签化检索和版本兼容性标注，帮助开发者从海量信息中快速定位所需资源，显著提升学习与开发效率。

## 功能概览

**频道化资源分类** 所有外链按技术领域划分为多个频道，每个频道聚焦一个特定主题，如组件库、状态管理、性能优化、测试工具等，便于用户按需浏览。

**全文检索与过滤** 内置基于关键词的搜索功能，支持按资源类型、Vue 版本兼容性、维护状态等维度进行过滤，快速缩小结果范围。

**资源质量评级** 每一条收录的外链均经过社区投票与维护者初审，附带质量星级评分与简短推荐语，帮助用户判断资源价值。

**版本兼容性标注** 针对每个资源，明确标注其兼容的 Vue 2.x / 3.x 版本信息，避免引入不兼容依赖。

**社区贡献工作流** 提供标准化的资源提交模板与审核流程，鼓励开发者推荐优质外链，形成良性增长的资源池。

**个性化收藏集** 用户可创建自定义资源收藏集，用于项目技术选型备选清单或个人学习路径规划。

**更新日志追踪** 自动监测已收录资源的更新频率，对长期未维护或已废弃的链接进行标记提醒。

## 应用场景

技术团队内部知识库构建 团队技术负责人可使用 VueBest 整理团队推荐的技术栈资源列表，统一新成员的学习路径与项目依赖选型参考，减少因信息不对称导致的重复调研成本。

个人开发者系统化学习 自学者可按频道分类从基础教程、实战案例到高级原理剖析的顺序进行系统性学习，避免在零散资料中迷失方向。

开源项目作者推广与反馈 开源组件或工具的作者可提交自己的项目至相关频道，获取早期用户反馈与社区曝光，同时通过质量评级了解用户真实评价。

技术选型调研 架构师在进行技术选型时，可通过本平台快速对比不同解决方案的维护活跃度、社区热度与兼容性信息，辅助决策。

## 快速开始

以下步骤将指导您在本地环境中快速启动 VueBest 资源导航系统的开发实例。

```bash
# 克隆代码仓库至本地
git clone https://github.com/vuebest/vuebest-navigator.git

# 进入项目根目录
cd vuebest-navigator

# 安装项目依赖（使用 npm 或 yarn）
npm install

# 启动本地开发服务器，默认访问地址为 http://localhost:3000
npm run dev
```

## 项目结构

```
vuebest-navigator/
├── src/
│   ├── api/                      # 后端 API 路由与控制器，处理资源增删改查、用户认证等逻辑
│   ├── components/               # Vue 3 可复用组件库，包含资源卡片、搜索栏、分页器等
│   │   ├── common/               # 通用基础组件，如按钮、输入框、模态框
│   │   ├── resource/             # 资源相关专用组件，如资源列表、详情视图、评级组件
│   │   └── user/                 # 用户相关组件，如登录表单、收藏夹管理界面
│   ├── composables/              # Vue Composition API 组合式函数，封装数据获取、状态管理逻辑
│   ├── layouts/                  # 页面布局模板，包含头部导航、侧边栏、底部版权信息
│   ├── pages/                    # 路由对应的页面视图组件，每个频道对应一个动态路由页面
│   ├── router/                   # Vue Router 路由配置，定义资源列表、详情、贡献提交等路由
│   ├── stores/                   # Pinia 状态管理模块，管理资源缓存、用户会话、偏好设置
│   ├── styles/                   # 全局样式表与主题变量，基于 CSS 自定义属性实现暗色模式
│   ├── types/                    # TypeScript 类型声明文件，定义资源条目、用户、频道等数据结构
│   └── utils/                    # 工具函数集合，包括日期格式化、URL 验证、文本截断等
├── server/                       # 服务端代码目录（基于 Node.js + Express）
```

## 贡献指南

我们欢迎并鼓励社区开发者向 VueBest 提交资源推荐或改进建议。请遵循以下标准化流程以确保贡献质量。

第一步：复刻与克隆 在 GitHub 上复刻本项目仓库至个人账号，然后克隆至本地开发环境，并安装所有依赖。

第二步：创建功能分支 从 main 分支切出一个新的功能分支，分支命名应遵循 feat/资源名称-频道 或 fix/问题简述 的格式。

第三步：提交资源数据 按照 /docs/contributing/resource-template.md 中的模板格式，在对应的频道数据文件中新增资源条目，包含标题、URL、描述、标签和版本兼容性信息。提交前请运行 npm run validate 验证数据格式。

第四步：发起拉取请求 推送本地分支至远程复刻仓库，然后通过 GitHub 界面发起拉取请求至主仓库的 main 分支。请在请求描述中简要说明新增资源的价值与来源。

第五步：等待审核与合并 维护者将在 3 个工作日内审核拉取请求，可能会要求补充信息或修改格式。审核通过后即合并入主分支，新资源将在下一次部署后上线。

## 常见问题

问：如何报告收录资源中的失效链接或错误信息？

答：您可以在该资源详情页下方点击“报告问题”按钮，填写具体问题描述后提交。维护者会定期处理反馈并更新数据。您也可以直接在 GitHub Issues 中提交问题，并附上相关 URL 与错误说明。

问：为什么某些资源显示为“待审核”状态？

答：新提交的资源会经过人工初审与社区投票两个阶段。初审主要检查链接可用性、内容相关性以及是否违反社区准则。通过初审的资源进入为期 7 天的社区公示期，期间获得足够支持票后即转为正式收录状态。

问：能否申请新增一个资源频道？

答：可以。请通过 GitHub Issues 提交新增频道提案，说明该频道的主题范围、目标用户群以及至少 5 个拟收录的候选资源链接。维护者将评估该主题的覆盖面与社区需求度，决定是否开设新频道。
