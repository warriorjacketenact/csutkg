# ResourceBridge

ResourceBridge 是一个面向技术研究者、内容创作者和知识工作者的结构化外链资源导航与聚合系统。该项目并不生产原始内容，而是通过人工筛选与机器学习辅助分类，将分散在网络各处的高质量信息页面进行索引、标签化与版本管理，帮助用户从海量链接中快速定位具备长期参考价值的资料。

ResourceBridge 主要服务于两类用户：一类是需要持续跟踪特定领域动态但缺乏高效工具的技术从业者，另一类是希望建立个人知识库但受限于信息过载的学习者。项目通过统一的链接入库规范、周期性健康检查（链接有效性、内容变更检测）以及多维度的元数据标注，显著降低外链资源的管理成本。

## 功能概览

**批量链接导入与标准化** 支持从文本文件、CSV 或浏览器书签导出文件批量导入 URL，自动去除冗余参数、识别重复条目，并依据域名与路径特征生成初步分类建议。

**多维度标签与分类体系** 每个资源条目可绑定多个自定义标签，同时系统依据 URL 结构和页面标题进行自动补全标签推荐。分类体系支持无限层级嵌套，适应不同用户的知识组织习惯。

**链接健康状态监控** 定期对已收录的 URL 发起 HEAD 请求，检测返回状态码、响应时间以及页面标题变化。对失效链接（4xx、5xx）、重定向链过长或内容重大变更的条目进行标记并生成报告。

**全文检索与高级筛选** 基于 URL、页面标题、自定义描述、标签和分类路径进行布尔检索。支持按入库时间、最后访问时间、响应状态、域名等字段组合筛选，结果可导出为 JSON 或 CSV 格式。

**资源快照与变更追踪** 对关键链接保存 HTML 快照（可选），并记录每次快照的哈希值。当页面内容发生显著变化时，系统发送通知并生成差异对比视图，便于用户追溯信息来源的演变。

**多用户协作与权限管理** 支持团队工作区模式，不同成员可共享资源池但各自维护私有标签与注释。管理员可设定读写权限、审核新链接的入库申请，以及查看操作审计日志。

**开放 API 与数据导出** 提供 RESTful API 接口，允许第三方工具查询资源库、批量新增链接或获取健康状态报告。所有数据可通过计划任务导出为标准格式，便于与其他知识管理工具集成。

## 应用场景

技术文档团队维护外部参考库 技术文档编写过程中需要引用大量外部规范、标准文档和社区讨论。ResourceBridge 允许团队将分散的参考链接集中管理，并为每条链接添加适用产品版本、审核状态和备注，确保文档中的外部引用始终可追溯且保持更新。

学术研究者整理文献与数据源 人文社科或计算科学领域的研究者经常需要跟踪多个数据发布平台、预印本服务器和项目主页。通过 ResourceBridge 的标签与分类功能，可以按照研究主题、数据来源机构或时间范围对链接进行组织，同时利用健康监控及时发现失效的数据源链接。

技术博客作者选题与素材管理 独立博主在撰写深度技术文章时，需要积累大量案例、对比数据和官方公告。使用 ResourceBridge 可以按写作计划创建专属分类，为每个链接标注阅读状态、重要程度和关联文章草稿，显著提高素材检索效率。

开源项目维护外部依赖与参考实现 开源项目通常需要引用上游依赖的文档、参考实现仓库以及社区最佳实践。ResourceBridge 可帮助维护者建立一份结构化的外部资源清单，并配合健康监控功能在依赖文档发生变动时提前做出调整。

## 快速开始

以下操作在 Linux / macOS 环境下验证通过，Windows 用户可使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化 SQLite 数据库与默认配置
python scripts/init_db.py --config config/default.yaml
python scripts/import_seed_links.py --source data/seed_urls.txt

# 启动开发服务器（默认监听 127.0.0.1:5000）
python app.py
```

访问 http://127.0.0.1:5000 即可进入 Web 管理界面。首次登录需使用初始化时生成的 admin 账户凭证，该凭证将在控制台日志中输出。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 - 3.11 | 核心运行环境，低于 3.9 将无法使用类型注解与异步特性 |
| SQLite | 3.35 及以上 | 内置数据库，用于存储链接元数据、标签和用户信息。支持 JSON 函数以提升查询灵活性 |
| Redis | 6.0 及以上 | 可选依赖，用于缓存 API 响应和会话存储。未安装时降级为内存缓存 |
| node.js | 16.x 及以上 | 仅用于前端静态资源构建（Web UI），后端运行不依赖 |
| npm | 8.x 及以上 | 配合 node.js 安装前端依赖，若使用预编译静态文件则无需安装 |
| git | 2.25 及以上 | 用于版本管理和后续通过 pull 获取资源索引更新 |
| curl / wget | 任意稳定版本 | 健康检查模块依赖命令行工具发送 HTTP 请求，需确保已安装 |
| 系统时区数据 | tzdata 或等效包 | 用于正确记录链接检查时间与用户操作时间戳 |

│   │   └── auth.py                   # 登录、会话管理与 OAuth 回调
│   ├── models/                       # 数据模型与 ORM 映射

## 贡献指南

ResourceBridge 欢迎社区贡献，所有贡献需遵守行为准则。请按以下步骤提交变更。

1. 查阅 issues 列表，确认当前待解决的问题或功能请求。若计划新增较大特性，建议先创建讨论 issue 以明确设计方向，避免重复劳动或设计偏差。

2. 从 main 分支派生个人开发分支，命名格式为 feature/简短描述 或 fix/问题编号。本地开发时请运行 pre-commit 钩子（已配置在 .pre-commit-config.yaml）以确保代码风格与主仓库一致。

3. 编写或更新单元测试，确保新增代码的测试覆盖率达到 85% 以上。所有测试用例需通过 pytest 验证，且不得破坏现有 API 的向后兼容性。若涉及数据库迁移，请同时提供升迁与回滚脚本。

4. 提交 pull request 前，请同步主仓库最新代码并解决冲突。PR 描述中需明确说明变更动机、实现方式以及测试结果。若修复了已报告的 issue，请使用 Fixes #编号 自动关联。

## 常见问题

Q: 导入大量链接时页面响应缓慢，如何优化？

A: 建议使用命令行导入脚本（scripts/bulk_import.py）而非 Web 界面上传。该脚本支持分批提交和事务回滚，可避免 HTTP 超时。同时可调整 config/default.yaml 中的 batch_size 参数（默认 200）以适应不同服务器性能。若导入源为 CSV 文件，请确保字段顺序与模板一致，否则需通过 --mapping 参数指定列映射。

Q: 健康检查报告大量链接返回 403 或 429，如何处理？

A: 部分站点会限制自动化请求。可在配置中启用 respect_robots 选项（默认关闭），系统将尝试解析目标站点的 robots.txt 并遵守 Crawl-delay 指令。同时可调整 check_interval 增加检查间隔，或通过 user_agent_pool 随机切换 User-Agent 字符串。对于持续返回 403 的链接，建议手动验证后将其加入 ignore_list 以停止重复检查。

Q: 能否将 ResourceBridge 的数据迁移至 PostgreSQL？

A: 可以。项目使用 SQLAlchemy ORM，支持 PostgreSQL、MySQL 等主流关系型数据库。需在配置文件中将 driver 从 sqlite 切换为 postgresql，并提供完整的连接字符串（包含用户名、密码、主机、端口和数据库名）。迁移前请运行 scripts/export_backup.py 导出 SQLite 数据为 JSON 格式，再通过 scripts/import_from_json.py 导入至新数据库。注意 PostgreSQL 需启用 uuid-ossp 扩展以兼容默认的 UUID 主键生成策略。
