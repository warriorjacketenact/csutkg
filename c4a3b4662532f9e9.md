# WebLink Navigator

WebLink Navigator 是一个面向技术研究者、内容策展人和开发者的结构化外链资源管理与导航系统。该项目并非一个传统的爬虫或书签工具，而是一个基于静态 Markdown 构建的知识路由中枢，旨在解决信息过载时代下高质量外链资源难以被系统化组织和持久化引用的痛点。项目核心定位为“技术资源的外链汇总站”，通过严格的目录规范与批次管理，为特定主题（如影视技术、数字媒体处理、跨领域数据标注等）提供可审计、可追溯的链接索引服务。

目标用户包括需要维护大量外部参考链接的技术文档撰写者、从事数据标注与样本采集的 AI 训练工程师、以及需要对外部依赖资源进行版本化管理的开源项目维护者。项目本身不存储任何外部资源实体，仅提供标准化的引用清单与快速访问路由，确保资源引用的透明度与长期可用性。

## 功能概览

- **批量链接导入与校验** 支持通过标准输入或文件流导入大批量 URL 列表，自动执行协议一致性检查与重复项去重，并生成导入报告。
- **结构化目录树生成** 根据 URL 路径特征自动推断资源类别，在项目根目录下生成符合业务语义的文件夹层级，并为每个链接附加采集时间戳与来源批次标签。
- **外链存活状态检测** 集成异步 HTTP 健康检查模块，定期对已收录的链接执行 HEAD 请求，标记不可达资源并触发管理员告警，降低死链率。
- **元数据标签系统** 允许用户为每个链接自定义键值对元数据（如技术栈、优先级、关联 Issue 编号），支持基于标签的快速筛选与视图聚合。
- **多格式导出服务** 支持将当前索引库导出为 JSON、CSV 或纯文本站点地图格式，便于与其他自动化流水线或静态站点生成器集成。
- **变更审计日志** 记录每一次链接增删改操作的操作人、时间戳与变更摘要，满足团队协作场景下的可追溯性要求。
- **全文模糊检索** 基于 URL 片段和自定义备注构建轻量级倒排索引，支持前缀匹配与通配符查询，帮助用户在海量链接中快速定位目标项。
- **批次快照管理** 为每一批导入的链接创建独立的快照目录，保留原始数据清单，支持批次回滚与版本对比，避免误操作导致的数据污染。

## 应用场景

- **技术文档的外部参考附录管理** 技术博客作者或 API 文档维护者可将文章中引用的所有第三方规范、工具主页或社区讨论链接统一托管于 WebLink Navigator，并在文档发布前执行一次死链检查，确保所有引用有效且可访问。
- **AI 数据集构建中的源文件追溯** 数据采集工程师在构建多模态数据集时，通常需要记录原始视频、图像或文本素材的来源 URL。通过本项目的元数据标签功能，可清晰标记每个链接对应的数据子集编号、版权协议及采集日期，满足数据治理合规要求。
- **开源项目的依赖资源镜像索引** 当开源项目依赖外部预训练模型或大型二进制文件时，维护者可使用本系统记录官方下载地址、备用镜像站及对应的 MD5 校验值链接，避免因官方源变动导致构建流程中断。
- **学术文献的补充材料路由** 研究团队在发表论文时，可将实验数据、代码仓库、附录视频等补充材料的公开链接集中整理为一份由 WebLink Navigator 管理的清单，并在论文中引用该清单的固定版本号，提升成果的可复现性。
- **社区知识库的外部精选推荐** 技术社区运营者可将高频问答中涉及的外部教程、工具评测或业界标准整理成精选链接列表，利用批处理快照功能按周或按月发布推荐合集，降低社区成员的信息筛选成本。

## 快速开始

以下命令演示了如何从 GitHub 克隆项目、安装依赖并启动本地开发服务。请确保你的环境已满足安装要求章节中列出的所有必需组件。

```bash
git clone https://github.com/weblink-navigator/weblink-navigator.git
cd weblink-navigator
pip install -r requirements.txt
python manage.py runserver --host=0.0.0.0 --port=8080
```

首次启动时，系统会在 `./data/` 目录下自动生成默认的索引模板与配置文件。若需导入示例链接清单，可执行：

```bash
python cli.py import --batch 1127 --source ./samples/links_1127.txt
```

导入完成后，可通过 Web 界面或 REST API 访问索引内容。

## 项目结构

```
weblink-navigator/
├── cli.py                         # 命令行入口，处理导入、导出与校验逻辑
├── manage.py                      # 开发服务器与数据库迁移控制脚本
├── requirements.txt               # Python 依赖清单（锁定了 aiohttp 与 click 版本）
├── config/
│   ├── settings.yaml              # 主配置文件：端口、超时阈值、日志级别
│   └── schema.json                # 链接元数据的 JSON Schema 校验定义
├── data/
│   ├── index.db                   # SQLite 数据库文件（元数据与标签存储）
│   └── snapshots/                 # 批次快照根目录
│       └── 1127/                  # 当前批次（1127）的原始导入清单
│           └── manifest.json      # 包含导入时间、记录数及原始 URL 列表的摘要
├── src/
```

## 贡献指南

我们欢迎并鼓励社区提交改进建议与功能补丁。为确保协作流程的顺畅，请遵循以下步骤：

1. 查阅问题追踪器中的“待认领”列表，或提交新 Issue 详细描述你希望解决的功能缺口或缺陷。在开始编码前，请至少等待一位维护者的确认回复，以避免与其他贡献者的工作重复。
2. 派生本项目至你的个人账户，并在本地新建一个描述性名称的功能分支（例如 `feature/add-batch-rollback`）。所有开发工作应在该分支上进行，禁止直接在主分支提交。
3. 编写代码时请遵循 PEP 8 编码规范，并为所有新增的函数或类添加 docstring 注释。对于涉及外部 HTTP 请求的逻辑，必须编写对应的单元测试用例，确保测试覆盖率达到 80% 以上。
4. 提交代码前，请运行完整的测试套件（`pytest tests/`）并确保所有现有测试通过。若你的改动涉及数据库 Schema 变更，需同时提供迁移脚本。

## 常见问题

**Q: 项目是否会定期自动检查所有链接的可用性？如果发现死链，系统会如何处理？**

A: 系统默认每天凌晨 2:00 执行一次全量存活检查，但该功能默认处于关闭状态以避免对目标服务器造成不必要的请求压力。管理员可通过修改 `config/settings.yaml` 中的 `scheduler.enabled` 参数开启。检测到不可达链接时，系统会在日志中记录警告，并通过 Web 控制台的“告警面板”进行视觉高亮。目前版本不会自动删除或修改链接条目，所有操作需由管理员手动确认。

**Q: 我能否将已有的浏览器书签 HTML 文件直接导入到该项目中？**

A: 项目原生支持标准的 Netscape 书签格式（即浏览器导出的 .html 文件）导入。你可以使用 `cli.py import --type bookmarks --source bookmarks.html` 命令进行转换导入。系统会自动解析书签的标题、添加时间及文件夹层级，并将其映射为本项目的标签与备注字段。对于非标准格式的书签文件，建议先转换为 CSV 格式再执行导入。

**Q: 分布式部署或多实例运行时，如何保证数据库与快照文件的一致性？**

A: 当前版本的设计目标为单机部署，并未内置分布式锁机制。若需在多实例环境下运行，建议将 SQLite 数据库替换为 PostgreSQL，并使用共享存储（如 NFS 或 S3）存放快照目录。项目文档的运维手册章节已提供基于 Docker Compose 的参考部署方案，该方案通过卷挂载和单主数据库解决了并发写入冲突问题。
