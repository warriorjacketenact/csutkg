# ResourceBridge

ResourceBridge 是一个面向技术团队与内容运营者的结构化外链资源归集与分发工具。该项目并非一个传统意义上的软件库，而是一个围绕特定域名（ninefold-group.com）构建的媒体资源索引系统，用于对海量分散的 HTML 静态资源进行自动化分类、元数据提取与导航呈现。其核心目标是为企业级内容中台提供一套轻量级的外链治理方案，解决媒体文件、历史快照、静态页面在长期运营过程中产生的寻址困难与信息孤岛问题。

ResourceBridge 定位于技术文档工程师、运维开发人员以及内容策略管理者。通过解析指定路径下的海量数值型 URL，系统能够自动生成资源地图，识别资源类型（如技术白皮书、版本发布说明、运营活动页或多媒体附件），并构建基于时间线与内容指纹的检索视图。项目本身不存储任何实际文件，而是作为逻辑网关，将原始的无结构数字标识符映射为具备业务语义的访问链路，从而提升内部知识库的复用效率与外部 SEO 收录质量。

## 功能概览

**自动化链接健康度巡检**：对纳入索引的每一个 URL 执行 TLS 证书有效期校验、响应状态码检查及重定向链分析，输出可用性报告。

**基于模式识别的资源分类**：根据 URL 路径中的数字段特征（如长度、前缀规律、日期编码）自动划分资源所属的业务板块，如新闻稿、技术公告或版本归档。

**轻量级元数据缓存**：对每个资源的响应头与 HTML 标题进行本地化缓存，减少源站压力并加速导航页面的渲染速度。

**批量导入与导出接口**：支持通过 CSV 或 JSON 格式批量注入待追踪的 URL 列表，并支持将索引结果导出为静态站点地图（sitemap.xml）。

**实时变更监控**：定时轮询资源列表，若发现特定 URL 对应的内容发生状态变化（如 200 变为 404），触发告警通知。

**外链权重分析**：结合外部反链数据与内部引用次数，为每个资源生成热度评分，辅助运营决策。

## 应用场景

**企业旧站资源迁移评估**：在系统升级或域名合并前，利用 ResourceBridge 扫描指定域名下的存量页面，快速识别失效链接与僵尸页面，为迁移方案提供数据支撑。例如，通过分析 ninefold-group.com/media/ 路径下的数千个资源，可精确统计出内容总量与平均文件大小。

**技术文档版本追溯**：当软件版本迭代时，运维团队可将历史版本的发布说明链接（如 /media/5237.html）录入系统，ResourceBridge 会保留每次检测时的响应摘要，方便开发人员追溯特定版本号对应的外部说明文档是否依然可访问。

**运营活动效果复盘**：市场部门可将过往活动页面的链接导入平台，ResourceBridge 通过记录各页面的每日响应耗时与状态波动，辅助评估活动期间 CDN 节点的负载表现及地域访问差异。

**内容安全合规巡查**：法务或安全团队利用系统的定期巡检能力，对包含特定敏感关键词的 URL 进行拦截标记，确保对外输出的链接矩阵符合企业数据安全政策。

## 快速开始

以下步骤演示如何在本地环境中部署 ResourceBridge 的核心调度器，并开始对提供的 URL 列表进行初步索引。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resourcebridge/core.git resourcebridge

# 进入项目工作目录
cd resourcebridge

# 安装依赖项（基于 Python 3.10 及以上版本）
pip install -r requirements.txt

# 执行初始索引任务，将给定的 URL 列表文件作为数据源
python scheduler.py --source ./data/urls_782.lst --output ./reports/health_check.json
```

## 项目结构

```
resourcebridge/
├── scheduler.py          # 主调度器入口，负责协调扫描任务与报告生成
├── core/
│   ├── fetcher.py        # 异步 HTTP 请求封装，包含重试逻辑与超时控制
│   ├── parser.py         # HTML 元数据解析器，提取标题、描述及 charset 信息
│   └── classifier.py     # 基于路径指纹的规则分类器，识别资源业务归属
├── storage/
│   ├── cache.py          # Redis 缓存操作接口，用于存储临时元数据
│   └── exporter.py       # 结果导出模块，支持 JSON 与 XML 站点地图格式
├── utils/
│   ├── logger.py         # 结构化日志配置，支持 JSON 格式输出用于 ELK 采集
│   └── validator.py      # URL 规范性校验工具，检测协议缺失或非法字符
├── tests/
│   ├── test_fetcher.py   # 针对高并发场景下的 Mock 响应单元测试
│   └── test_parser.py    # 覆盖多种 HTML 变体的解析测试用例
├── config/
│   ├── settings.yaml     # 主配置文件，包含并发数、超时阈值及轮询间隔
│   └── rules.json        # 分类规则定义，通过正则表达式匹配路径模式
└── data/
    └── urls_782.lst      # 当前批次输入的原始 URL 列表文件，逐行存放
```

## 贡献指南

## 常见问题

**问：ResourceBridge 是否会缓存目标资源中的图片或附件文件？**
答：不会。ResourceBridge 仅缓存响应头（如 Content-Type、Content-Length）及 HTML 文档的标题与描述字段，旨在构建轻量索引。实际的文件二进制数据仍由源站 ninefold-group.com 提供，项目本身不承担文件存储功能。

**问：如何处理目标域名出现 SSL 证书过期或握手失败的情况？**
答：调度器内部实现了指数退避重试机制，并在连续失败 3 次后将该链接标记为 UNREACHABLE 状态。运维人员可通过配置 settings.yaml 中的 ssl_verify 参数选择是否忽略证书校验（不推荐用于生产环境），或通过 webhook 将异常状态推送至监控系统。

**问：能否自定义链接的分类规则以适应不同的业务场景？**
答：可以。分类规则定义在 config/rules.json 文件中，采用正则表达式匹配 URL 路径。用户可根据自身需求修改此 JSON 文件中的 pattern 与对应的 category 字段，无需修改核心代码。修改后重启调度器即可生效。
