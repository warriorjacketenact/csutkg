# ResourceWatch 技术资源外链聚合平台

ResourceWatch 是一个面向开发者、技术研究人员与开源项目维护者的外链资源聚合与导航系统。本项目定位于对特定域名下的深度技术内容链接进行结构化收集、分类展示与快速检索，帮助用户从大量分散的 URL 中高效定位到与开发运维、系统监控、性能调优、安全审计等方向相关的具体页面。

本项目不提供爬虫或自动化采集功能，而是以人工精选与社区贡献的方式，围绕 sjqcz.com 域名下的 watch 路径资源建立索引目录，并提供标签过滤、最近更新排序和访问热度统计等辅助功能。目标用户包括需要持续跟踪特定技术站点更新内容的工程师、希望建立私有技术资源导航的团队，以及希望复用结构化外链数据进行二次开发的 SDK 集成者。

## 功能概览

**结构化资源索引**：对收录的全部 URL 按内容主题、文件类型和更新时间三个维度建立索引，支持多条件组合筛选。

**实时可用性检测**：每隔六小时对已收录链接进行 HTTP 状态码检查，自动标记失效链接并生成通知队列。

**标签分类体系**：为每个资源链接分配一到三个功能标签，涵盖监控、日志、配置、部署、安全、网络、存储等十余个技术领域。

**访问统计看板**：记录每个链接的点击次数、最近访问时间和来源 Referer，提供按 PV 排序的热门资源排行。

**Markdown 格式导入导出**：支持将整个资源列表以标准化 Markdown 表格形式导出，便于嵌入技术文档或 Wiki 系统。

**RESTful API 接口**：提供 JSON 格式的查询接口，支持按标签、关键字和时间范围获取资源子集，方便第三方工具集成。

**用户自定义收藏夹**：注册用户可将常用链接加入个人收藏夹，并添加私人备注和评分。

**更新订阅机制**：支持 RSS 和邮件两种订阅方式，当资源列表发生新增或变更时自动推送更新摘要。

## 应用场景

**技术团队内部知识库建设**：开发团队可将本项目部署为内部导航页，将与项目相关的监控面板、日志查询入口、CI/CD 流水线地址和运维手册统一收录，新成员入职时通过浏览资源列表即可快速熟悉整套技术栈的访问入口。

**开源文档中的参考链接管理**：开源项目维护者在 README 或官方文档中引用大量外部参考资料时，可使用 ResourceWatch 维护一个独立的链接索引页面，避免在主文档中堆砌冗长 URL，同时利用可用性检测功能及时发现失效的参考链接。

**技术博客的延伸阅读聚合**：技术博主可将过往文章中提及的所有工具官网、论文原文、代码仓库和视频教程链接汇总至本项目，为读者提供一站式延伸阅读入口，提升博客内容的实用价值。

**DevOps 监控工具链整合**：运维工程师可将 Prometheus、Grafana、ELK、Zabbix 等监控组件的访问地址，以及各告警管理后台的链接统一纳入 ResourceWatch，配合标签分类实现生产环境入口的集中管控。

## 快速开始

以下命令将项目克隆至本地、安装依赖并启动开发服务器。

```bash
git clone https://github.com/resourcewatch/resourcewatch.git
cd resourcewatch
npm install
npm run dev
```

执行完成后，访问本地 3000 端口即可查看资源列表首页。如需构建生产版本，请运行 `npm run build` 后使用 `npm run start` 启动。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或更高版本 | 是 | 项目运行时环境，推荐使用 LTS 版本 |
| npm 9.x 或更高版本 | 是 | 依赖包管理工具，随 Node.js 一同安装 |
| SQLite 3.x | 是 | 内置数据库引擎，用于存储资源索引和访问统计 |
| Git 2.x | 是 | 用于克隆仓库和版本管理 |
| PM2 5.x | 否 | 生产环境进程守护工具，非开发环境必需 |
| Nginx 1.20+ | 否 | 推荐用于生产环境反向代理与静态资源缓存 |
| Redis 7.x | 否 | 启用缓存加速和会话存储时可选用 |

## 贡献指南

欢迎社区开发者参与 ResourceWatch 的改进与扩展。请遵循以下流程提交贡献。

第二步：派生仓库并创建功能分支。将主仓库 Fork 至个人账号，使用 `git checkout -b feature/your-feature-name` 创建新分支，分支名称应简要描述本次变更的功能或修复内容。

## 常见问题

Q：如何批量导入自定义的外部链接列表？

A：本项目支持通过 CSV 和 JSON 两种格式批量导入链接。管理员可在后台管理界面的「导入」选项卡中上传文件，文件须包含 url、title、tags 和 category 四个字段。同时，项目根目录下的 `scripts/import-from-csv.js` 脚本也提供了命令行导入方式，适用于自动化流水线场景。

Q：链接可用性检测会产生误报吗？

A：检测引擎默认使用 HEAD 请求方法，超时时间设为 5 秒，并自动跟随重定向。对于返回 429 或 503 状态码的站点，引擎会进行三次重试，间隔分别为 1 秒、3 秒和 5 秒。若三次重试后仍返回非 2xx 状态，则标记为失效。用户也可在管理后台手动触发单项重新检测，以覆盖自动检测结果。

Q：本项目是否支持多用户权限管理？

A：ResourceWatch 内置了基于角色的访问控制，支持管理员、编辑者和只读访客三种角色。管理员拥有全部操作权限，包括用户管理和系统配置；编辑者可新增、修改和删除链接，但无法管理用户；只读访客仅能浏览和搜索资源列表。默认安装后初始管理员账号需通过首次启动时的命令行引导设置。
