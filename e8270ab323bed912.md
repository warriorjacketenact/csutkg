# LinkVault Core

LinkVault Core 是一个面向技术内容聚合与知识库管理的高性能外链资源归集系统。该项目定位于解决开发者在文档维护、技术调研和项目学习过程中面临的信息碎片化问题，提供从原始 URL 采集、结构化存储、分类索引到快速检索的完整工作流支持。

本项目主要服务于技术文档工程师、开源项目维护者、技术社区运营人员以及需要长期跟踪大量外部技术资源的研究者。通过将分散于不同来源的链接纳入统一的目录体系，LinkVault Core 帮助用户建立可维护、可共享、可追溯的技术外链知识图谱，显著降低信息遗忘与重复检索的认知成本。

## 功能概览

**链接归集与去重**：支持批量导入原始 URL，自动识别重复条目并基于内容指纹进行智能合并。

**多级分类索引**：允许用户为每条链接分配自定义标签、项目阶段及领域分类，生成多维度导航视图。

**元数据自动提取**：对于符合通用页面结构的外链，系统自动抓取页面标题、发布时间与摘要信息。

**状态监控与健康检查**：定期对已收录链接进行可访问性探测，标记失效或响应超时的资源。

**全文检索与过滤**：基于标题、标签、描述及关联笔记内容的轻量级全文搜索，支持布尔表达式与日期范围过滤。

**导入导出标准化**：支持 JSON、CSV 及 Markdown 列表格式的批量导入导出，便于与其他知识管理工具交换数据。

**静态站点生成**：将链接集合渲染为静态 HTML 目录页，适用于团队内部文档站点或项目维基。

## 应用场景

技术文档团队可将 LinkVault Core 作为官方文档的外部参考链接管理中心，统一维护所有引用资料的来源与版本，避免文档中的外链随项目迭代而大量失效。

开源项目维护者可以利用该项目整理社区中与本项目相关的教程、插件、衍生作品及讨论帖，形成活跃的生态资源墙，降低新贡献者的信息获取门槛。

个人技术研究者可将其作为个人阅读清单与书签管理系统的替代方案，通过本地部署实现对收藏链接的全文检索、失效告警和笔记标注，构建长期可用的私人知识数据库。

技术社区运营方可利用 LinkVault Core 生成每周资源精选列表，通过静态站点导出功能快速发布到社区博客或邮件简报中，实现内容资产的二次利用。

## 快速开始

```bash
git clone https://github.com/your-org/linkvault-core.git
cd linkvault-core
pip install -r requirements.txt
python main.py --init
python main.py --import ./samples/links.csv
python main.py --serve --port 8080
```

首次运行 `--init` 会在当前目录生成默认的配置文件 `config.yaml` 和 SQLite 数据库文件 `data/vault.db`。导入示例链接集后，可通过内置的轻量级 Web 界面或 RESTful API 进行浏览与管理。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.9+ | 是 | 核心运行时环境，建议使用 3.10 及以上版本以获得更好性能 |
| SQLite 3.35+ | 是 | 内置数据库引擎，用于存储链接元数据与索引 |
| requests 2.31+ | 是 | 用于外链状态检查与元数据抓取 |
| beautifulsoup4 4.12+ | 是 | HTML 解析与页面标题、摘要提取 |
| lxml 4.9+ | 推荐 | 作为 beautifulsoup4 的解析器后端，提供更快的解析速度 |
| markdown 3.5+ | 否 | 仅在导出为 Markdown 格式或生成静态站点时需要 |
| jinja2 3.1+ | 否 | 仅在静态站点生成功能中用于模板渲染 |
| pytest 7.4+ | 否 | 单元测试框架，仅开发者需要 |

## 项目结构

```
linkvault-core/
├── main.py                     # 程序入口，负责命令行参数解析与主流程调度
├── config.yaml                 # 用户配置文件，包含数据库路径、检查间隔、服务端口等
├── requirements.txt            # Python 依赖清单，锁定主要库版本号
├── data/
│   └── vault.db                # SQLite 主数据库，存储链接、标签、检查记录表
├── src/
│   ├── core/                   # 核心功能模块
│   │   ├── importer.py         # 导入器，支持 CSV/JSON/Markdown 格式解析
│   │   ├── indexer.py          # 索引器，负责标签与分类的关联构建
│   │   ├── checker.py          # 健康检查器，异步并发探测链接可达性
│   │   └── exporter.py         # 导出器，支持多格式输出与静态站点生成
│   ├── extractors/             # 元数据提取器插件目录
│   │   ├── base.py             # 提取器基类，定义接口规范
│   │   ├── html_parser.py      # 通用 HTML 页面标题与摘要提取
│   │   └── github_repo.py      # GitHub 仓库专用提取器，获取星标与更新时间
│   ├── api/                    # RESTful API 服务模块
│   │   ├── app.py              # Flask 应用实例与路由注册
│   │   ├── schemas.py          # Pydantic 请求与响应数据模型
│   │   └── handlers.py         # 各端点业务逻辑实现
│   └── utils/
│       ├── fingerprint.py      # URL 去重指纹计算工具
│       ├── logger.py           # 统一日志配置与输出格式
│       └── validators.py       # URL 格式校验与规范化辅助函数
├── tests/
│   ├── unit/                   # 单元测试用例，覆盖核心算法与工具函数
│   ├── integration/            # 集成测试，验证数据库读写与 API 交互
│   └── fixtures/               # 测试固定数据集，含示例链接与预期结果
├── docs/                       # 完整文档目录，包含入门、功能、运维与 API 文档
└── scripts/
    ├── migrate_db.py           # 数据库迁移脚本，用于版本升级时变更表结构
    └── seed_sample.py          # 种子数据填充脚本，用于快速构建演示环境
```

## 贡献指南

## 常见问题

**问：导入大量链接时性能如何？是否有数量限制？**

答：系统在 SQLite 数据库中使用了批量插入与索引优化，单次导入一万条链接的耗时约在三秒至五秒之间。理论上没有硬性的数量上限，但建议单个数据库文件中的链接总数不超过五十万条以保证查询响应速度。对于超大规模场景，可配置外部 PostgreSQL 作为后端存储。

**问：元数据自动提取的准确率如何？是否支持自定义规则？**

答：内置的 HTML 提取器对于标准博客、技术文档和 GitHub 仓库页面的标题与摘要抓取准确率较高，但对于动态渲染页面或非标准结构站点可能存在缺失。项目提供了插件机制，用户可通过继承 `base.py` 中的 `BaseExtractor` 类并实现 `extract` 方法，针对特定域名或页面类型定制提取逻辑。

**问：如何迁移或备份已归集的链接数据？**

答：数据库文件 `data/vault.db` 可直接复制迁移。同时，系统支持通过 `--export` 命令将全量数据导出为 JSON 或 CSV 格式，便于跨版本或跨平台迁移。建议定期执行 `--export` 并将导出文件存储于外部备份系统中。
