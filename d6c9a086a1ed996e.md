# EonCN Resource Navigator

EonCN Resource Navigator 是一个面向技术研究、信息检索与知识管理领域的开源外链资源汇总系统。该项目定位于为开发者、数据分析师、技术研究人员以及信息爱好者提供高质量、结构化、可检索的外部资源导航服务，帮助用户在纷繁复杂的网络信息中快速定位有价值的参考资料与技术文档。

该项目通过人工筛选与社区贡献相结合的方式，收集并整理了超过一千两百批次的网络资源链接，涵盖技术博客、学术论文、开源项目文档、行业报告、数据可视化案例、架构设计解析、运维实践记录、编程语言特性分析、前端工程化方案、后端服务治理经验、云计算基础设施配置、安全审计方法论等多个技术细分领域。EonCN Resource Navigator 不生产原始内容，而是作为信息入口与导航中枢，降低用户的信息获取成本，提升技术研究效率。

## 功能概览

**分层资源索引**：按照技术领域、内容类型、适用层级等多个维度对资源链接进行分类组织，支持快速筛选与定向检索。

**批量导入与解析**：支持批次化资源链接导入，自动提取链接特征并进行基础元数据标注，便于后续检索与归类。

**结构化展示与输出**：将资源链接以 Markdown 表格、分类列表、目录树等多种形式进行结构化呈现，适配不同使用场景的阅读习惯。

**社区贡献机制**：内置资源推荐与审核流程，允许社区成员提交新的资源链接，经审核后并入主库，实现资源的持续更新与扩展。

**静态站点生成支持**：项目资源库可无缝对接静态站点生成工具，一键生成完整的导航站点 HTML 页面，便于部署与分享。

**全文检索与标签过滤**：基于资源标题、描述、分类标签等元数据提供全文检索能力，支持多标签组合过滤，精准定位所需内容。

**版本化资源快照**：记录每一批次资源的导入时间与变更历史，支持回溯查看特定时间点的资源集合状态。

**多格式导出兼容**：支持将资源列表导出为 JSON、CSV、Markdown 等多种通用数据格式，便于与其他工具链集成。

## 应用场景

**技术团队内部知识库建设**：技术团队可使用 EonCN Resource Navigator 整理团队内部积累的参考文档、技术博客、开源项目地址等外链资源，形成统一的知识入口，降低新成员上手成本，提升团队信息共享效率。团队可根据自身技术栈定制分类体系，实现知识资产的沉淀与传承。

**个人技术阅读与学习管理**：开发者可利用该项目构建个人技术阅读清单，按照前端、后端、运维、算法等方向分类收藏优质文章与教程，配合检索功能快速回顾以往阅读过的内容，避免重复搜索，形成可持续积累的个人技术资料库。

**开源项目文档聚合导航**：开源项目维护者可基于 EonCN Resource Navigator 搭建项目周边的生态资源导航页，聚合相关的社区教程、插件列表、示例项目、视频讲解等外链资源，帮助用户更全面地了解和使用项目，降低社区参与门槛。

**技术会议与培训材料配套**：技术会议组织者或培训讲师可将该资源库作为会议或课程的配套资料导航，按日程或主题整理相关的延伸阅读链接、视频回放地址、代码仓库地址等，方便参会者课后查阅与复习，提升知识传播的延续性。

## 快速开始

以下指令帮助您在本地环境中快速启动 EonCN Resource Navigator 项目。

```
git clone https://github.com/eoncn/eoncn-resource-navigator.git
cd eoncn-resource-navigator
pip install -r requirements.txt
python build.py --batch 1110 --output ./dist
```

执行上述命令后，系统将自动拉取项目代码，安装 Python 依赖（需 Python 3.8 及以上版本），并针对第 1110 批次资源生成静态导航页面至 `./dist` 目录。您可以使用任意静态 Web 服务器（如 `python -m http.server 8000`）在本地预览生成的站点。

├── config.yml               # 全局配置文件，含分类定义、输出路径、批次参数等
├── requirements.txt         # Python 依赖清单
├── README.md                # 项目说明文档

├── core/                    # 核心逻辑模块
│   ├── parser.py            # 资源链接解析器，提取元数据与校验可用性

│   ├── exporter.py          # 多格式导出器，支持 JSON / CSV / Markdown 输出

│   ├── category.html        # 分类视图页面模板
│   └── detail.html          # 单条资源详情页模板
├── static/                  # 静态资源目录
│   ├── css/
│   │   └── style.css        # 自定义样式表
│   ├── js/
│   │   └── search.js        # 前端检索交互脚本

## 贡献指南

我们欢迎社区成员为 EonCN Resource Navigator 贡献新的资源链接、改进分类体系、修复缺陷或完善文档。请遵循以下步骤参与贡献。

第一步，查阅现有资源库与分类体系。在提交新资源之前，请通过检索功能确认该资源尚未被收录，并了解当前的分类定义，以确保新资源能够被准确归类。

第二步，提交资源推荐或问题反馈。请通过 GitHub Issues 提交资源推荐请求，按照 Issue 模板填写资源标题、原始 URL、建议分类、简要描述及推荐理由。对于缺陷报告或功能建议，同样请使用 Issues 进行登记。

第三步，发起 Pull Request 进行代码或文档修改。对于分类调整、模板优化、文档更新或缺陷修复，请在本地完成修改并充分测试后，提交 Pull Request 至主分支。Pull Request 描述中应清晰说明变更内容、影响范围及测试情况。

第四步，参与审核与迭代。提交的 Issue 或 Pull Request 将由项目维护者进行审核，审核过程中可能需要补充信息或调整修改方向，请保持关注并积极配合，直至变更被合并或关闭。

第五步，成为长期贡献者。累计提交高质量资源链接或代码贡献的社区成员，将被邀请加入项目组织，获得直接推送权限，并参与项目路线图的讨论与制定。

## 常见问题

Q：如何确认某个资源链接是否已经被收录？

A：您可以通过项目生成的导航页面中的检索框进行关键词搜索，或直接查看对应分类列表。若您使用本地命令行环境，也可使用 `grep` 命令在 `data/batches/` 目录下的 JSON 文件中进行字符串匹配检索。项目未提供独立的在线 API 查询接口，但所有数据均以结构化形式存储，便于本地脚本化查询。

Q：我提交的资源链接多久会被审核通过？

A：项目维护者会在每批次整合周期内集中处理社区提交的资源推荐。通常每两周进行一次批次整合，审核周期不超过 15 个工作日。若资源链接需要额外验证（如可用性、内容相关性等），审核时间可能会适当延长。您可以在对应的 Issue 中查看审核进度。

Q：能否在项目生成的静态站点中添加自定义的页面或模块？

A：可以。项目使用 Jinja2 模板引擎，您可以通过修改 `templates/` 目录下的模板文件来调整页面布局与结构。同时，您可以在 `config.yml` 中扩展自定义配置项，并在模板中引用这些配置。如需新增页面类型，建议参考 `core/` 模块中的生成逻辑进行扩展，并保持与现有导出格式的兼容性。
