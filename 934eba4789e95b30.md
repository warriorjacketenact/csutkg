# ResourceBridge

ResourceBridge 是一个面向技术团队与内容运营者的结构化外链资源聚合与导航系统。该项目并非传统的爬虫或简单书签管理工具，而是一套以主题剧场（Theater）为组织单元的资源索引框架，专门用于对高频访问的外部参考链接进行语义化归类、状态监控与团队共享。项目定位为内部技术决策的辅助信息枢纽，适用于需要长期跟踪特定领域（如云原生运行时、数据库选型、性能基准测试）动态的研发小组。

项目通过将原始 URL 抽象为可标记、可过滤、可版本化的条目，解决了分散链接易失效、上下文丢失、协作检索成本高的问题。ResourceBridge 不存储外部页面内容，仅维护资源元数据与健康状态，可独立部署于内网或公网服务器，作为团队知识库的补充层。

## 功能概览

**批量资源注册** 支持通过命令行或 Web 界面一次性导入大批量 URL，自动解析域名与路径结构，提取资源类型与潜在分类标签。

**主题剧院分组** 所有资源按 theater 路径逻辑归入不同主题空间，每个剧院可独立设置可见性、维护人以及刷新周期，实现多项目并行管理。

**链接健康检查** 内置异步 HTTP 状态监控器，定时探测每个资源的可访问性，在仪表盘显著标注异常链接并生成变更日志。

**元数据扩展字段** 每条资源可附加自定义键值对，用于记录关注原因、关联内部文档编号、预期更新频率或备选访问方式。

**全文检索与过滤** 基于资源路径、域名、自定义标签以及健康状态提供多条件组合筛选，支持正则表达式匹配，便于快速定位特定来源或类型的外链。

**审计追踪** 记录资源的添加、删除、状态变更以及元数据修改操作，操作日志关联用户身份，满足团队协作的可追溯需求。

## 应用场景

**技术选型参考库** 团队在评估消息队列或数据库产品时，可将各个厂商的官方性能报告、第三方基准测试以及社区对比文章统一收录至同一个剧院，并为每条链接标注测试环境与测试日期，后续复盘选型决策时可快速回溯原始数据来源。

**项目文档外链治理** 大型项目的技术文档中往往散落大量外部引用，ResourceBridge 可作为这些引用的注册中心，文档仅保留内部资源 ID，由系统统一维护外部 URL 的变更与重定向，避免文档频繁修正失效链接。

**运维故障处理知识索引** 运维团队可将常见错误码对应的官方修复指南、社区讨论帖以及内部既往处理记录以链接形式聚合，配合健康检查功能，在依赖的外部资源出现站点维护时提前预警，减少故障排查中的信息中断。

**月度技术简报素材管理** 负责撰写团队技术简报的成员可利用 ResourceBridge 收集本月值得关注的技术博客、版本发布公告以及行业分析文章，通过标签标记优先级与阅读状态，简报撰写时直接从筛选结果中提取素材，无需重复翻阅浏览器历史记录。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户请使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resourcebridge/resourcebridge.git

# 进入项目根目录
cd resourcebridge

# 安装依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化 SQLite 数据库并创建默认管理员账户
python manage.py initdb --admin admin@resourcebridge.local

# 启动开发服务器，默认监听 8000 端口
python manage.py runserver --host 0.0.0.0 --port 8000
```

访问 http://localhost:8000 进入仪表盘，使用初始化输出的临时密码登录后，可通过界面右上角的批量导入功能一次性添加本项目的资源清单。

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.9 或更高版本 | 是 | 核心运行环境，建议使用 3.11 长期支持版 |
| SQLite 3.35 或更高版本 | 是 | 默认元数据存储引擎，支持 JSON 字段操作 |
| pip 21.0 或更高版本 | 是 | Python 包管理工具，用于安装依赖库 |
| requests 2.28 或更高版本 | 是 | 处理异步健康检查中的 HTTP 请求与超时控制 |
| aiohttp 3.8 或更高版本 | 否 | 启用并发检查模式时显著提升扫描效率，建议安装 |
| Redis 7.0 或更高版本 | 否 | 用于分布式部署下的任务队列缓存与会话共享 |
| Docker Engine 20.10 或更高版本 | 否 | 容器化部署方案所需，生产环境推荐使用 |
| Node.js 18 或更高版本 | 否 | 仅当需要重新编译前端静态资源时依赖 |

## 项目结构

```
resourcebridge/
├── cmd/                                # 命令行入口与启动脚本
│   ├── server/                         # Web 服务启动器 (main.go)
│   └── admin/                          # 管理命令：批量导入、数据库迁移
├── internal/                           # 内部核心包，不对外暴露
│   ├── registry/                       # 资源注册与唯一性校验模块
│   ├── health/                         # 异步健康检查调度器与探测器
│   ├── theater/                        # 剧院空间管理逻辑 (CRUD + 分组)
│   ├── audit/                          # 操作日志记录与查询接口
│   └── storage/                        # 存储层抽象 (SQLite / PostgreSQL 适配)
├── pkg/                                # 可被外部引用的公共库
│   ├── models/                         # 数据实体定义 (Resource, Theater, CheckRecord)
│   ├── filters/                        # 过滤条件构造器 (标签、状态、路径模式)
│   └── utils/                          # 通用工具：URL 解析、时间处理、哈希计算
├── web/                                # 前端资源与模板
│   ├── static/                         # CSS、JavaScript、图标文件
│   ├── templates/                      # Jinja2 模板：仪表盘、剧院视图、表单页
```

## 常见问题

**Q: 项目中的 theater 路径与外部 URL 路径存在重名冲突，系统如何处理？**

A: ResourceBridge 内部的 theater 标识符独立于外部 URL 的路径部分。系统在存储时以完整 URL 字符串作为唯一键的一部分，同时单独提取 theater 字段用于分类。即便两个资源的 URL 路径完全相同，只要域名或协议不同，系统仍将其视为不同条目。内部 theater 命名空间与外部路径不存在耦合关系，因此不会产生冲突。

**Q: 健康检查功能是否会影响被探测站点的性能或产生大量无效流量？**

A: 健康检查模块默认采用低并发、长超时策略，每个资源的探测间隔默认设置为 24 小时，且仅发送 HEAD 请求，不下载响应体。系统支持配置白名单与探测窗口，可指定仅在特定时间段执行检查。对于内部网络或对请求敏感的服务，管理员可在配置文件中完全禁用该功能或调整至手动触发模式。

**Q: 资源清单中的链接数量庞大，批量导入时有无性能限制？**

A: 系统批量导入接口采用流式处理与事务分批提交，单次导入 5000 条以内均可正常完成。当资源总数超过 10000 条时，建议使用命令行导入模式并开启批量提交参数。数据库索引已针对路径字段与状态字段优化，检索与过滤操作在 100000 条级别下仍可维持毫秒级响应。
