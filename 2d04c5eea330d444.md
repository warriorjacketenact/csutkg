# ResourceBridge 技术资源导航站

ResourceBridge 是一个面向开发者与技术研究人员的结构化外链资源聚合平台，专注于收集、分类与索引互联网中的高质量技术文档、交互式学习工具、开源项目演示环境以及各类技术决策参考材料。项目本身不托管实际内容，而是以目录站的形式提供清晰、可维护的资源发现路径，帮助用户在特定技术领域内快速定位所需的外部信息源。

项目定位于技术团队的知识管理辅助工具，适用于架构选型调研、新技术预研、团队培训素材收集以及个人系统性学习规划等场景。通过统一的条目格式、标签化分类与版本追踪能力，ResourceBridge 能够有效降低资源散落带来的信息损耗，提升技术决策的信息完整性。

## 功能概览

**结构化资源收录**：每个资源条目包含标题、外部链接、所属类别、适用技能层级与收录理由，支持用户按需筛选与排序。

**多维度分类体系**：资源按编程语言、基础设施、前端工程、算法与数据结构、运维监控、技术管理六大维度进行划分，满足不同角色的信息获取习惯。

**自动化链接健康检查**：定期对已收录的外链进行可达性检测，自动标记失效链接并生成报告，确保资源列表的长期可用性。

**全文检索与标签过滤**：支持对资源标题、描述、标签进行快速全文搜索，并提供多选标签组合过滤，提升在大规模资源列表中的查找效率。

**导入导出与批量操作**：支持以 JSON、CSV 格式导入导出资源条目，便于团队内部共享资源池或迁移至其他知识管理工具。

**访问统计与热度排序**：记录每个外链的点击次数与最近访问时间，支持按热度、收录时间、名称排序，辅助识别高频使用的核心资源。

**自定义资源集合**：允许用户创建私有的资源分组（如“微服务调研清单”、“Go 语言学习路径”），在不修改全局目录的前提下实现个性化组织。

## 应用场景

技术团队进行新项目技术选型时，团队成员通常需要查阅大量框架文档、性能对比报告与社区活跃度数据。ResourceBridge 可集中收录此类参考链接，并标注适用场景与评估维度，减少重复搜索时间。

开发者个人规划系统性学习路径时，可利用本项目的分类索引快速发现某一领域（如分布式系统、云原生）的入门教程、最佳实践与案例解析，形成递进式学习顺序。

技术培训讲师准备课程材料时，可通过本项目的标签过滤与集合功能快速组织外部阅读清单、在线 Playground 链接与视频资源，分发给学员作为课后扩展资料。

开源项目维护者可将本项目作为项目 README 中“相关资源”章节的数据源，通过自动同步机制保持外部链接的及时更新，降低维护成本。

## 快速开始

以下命令演示如何在本地环境部署 ResourceBridge 实例，包含代码克隆、依赖安装与开发服务启动三个步骤。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

完成上述操作后，访问控制台输出的本地地址（默认 http://localhost:5173）即可使用开发版本。生产环境部署请参考 `docs/deployment.md` 中的容器化配置说明。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Node.js | 18.x LTS 或更高 | 运行时环境，推荐使用官方二进制或 nvm 管理 |
| npm | 9.x 或更高 | 依赖管理工具，随 Node.js 一并安装 |
| Git | 2.30 或更高 | 用于克隆仓库与版本控制 |
| SQLite | 3.40 或更高（内置） | 默认数据库引擎，无需额外安装，生产环境可切换至 PostgreSQL |
| 浏览器 | 现代浏览器（Chrome 110+ / Firefox 110+） | 仅开发调试与最终用户访问需要，服务器端无图形界面要求 |
| 磁盘空间 | 至少 200 MB | 用于存放源代码、依赖包及 SQLite 数据库文件 |
| 内存 | 最低 512 MB，推荐 1 GB | 开发服务器运行与构建过程的内存消耗 |
| 网络 | 可访问公网 | 用于首次启动时下载 npm 依赖以及执行外链健康检查 |

## 贡献指南

本项目的资源收录与分类维护依赖社区贡献，所有外链均需经过人工审核以确保质量与相关性。欢迎通过以下方式参与。

提交新资源建议：在 `data/suggestions/` 目录下创建 JSON 文件，按照模板填写资源标题、链接、类别及收录理由，并发起 Pull Request。模板格式见 `docs/developer/submission-template.md`。

参与链接健康检查维护：定期查看 `DashboardPage` 中的失效链接报告，若确认目标站点已迁移或永久关闭，在对应资源条目中更新链接地址或标记为失效。

完善分类体系与标签定义：若发现现有分类无法覆盖新的技术领域，可在 `backend/config/categories.js` 中提交扩展提案，需附带至少三个已有资源作为分类合理性的佐证。

改进文档与翻译：修订 `docs/` 下的用户指南或开发文档中的语法错误、过时描述，或将部分文档翻译为其他语言以扩大项目受众。

代码缺陷修复与性能优化：从 Issues 中选取带有 `good-first-issue` 标签的任务，遵循 `docs/developer/coding-standards.md` 中的代码规范提交修复。

## 常见问题

**项目是否托管实际的技术文档或代码示例？**

否。ResourceBridge 仅存储外部资源的链接、标题、描述与分类元数据，不缓存或代理任何第三方内容。用户点击链接后将直接跳转至原始站点，所有内容的版权与可用性均属于原始提供方。

**外链健康检查的频率和方式如何？**

系统默认每 72 小时对全部已收录链接执行一次 HEAD 请求检测，超时阈值设为 5 秒。检测结果记录于 `CheckLog` 表中，连续三次检测失败的链接将在前端界面标红提示。管理员可手动触发单条链接的即时检查。

**如何提交一个未收录的站点或文章？**

普通用户可通过前端的“提交建议”表单填写资源信息，系统自动生成 JSON 文件并存放于 `data/suggestions/pending/` 目录。项目维护者会定期审核建议，符合收录标准的资源将合并至主数据库。审核周期通常为 7 个工作日。

**生产环境是否必须使用 PostgreSQL？**

非必需。默认配置使用 SQLite 文件数据库，适合小型部署与个人使用。若预期资源条目超过 5000 条或需要多实例读写，建议按照 `docs/deployment/postgres-migration.md` 的指引切换至 PostgreSQL。
