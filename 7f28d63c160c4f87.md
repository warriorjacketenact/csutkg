# ResourceBridge

ResourceBridge 是一个面向开发者与技术研究人员的开源外链资源聚合与导航系统。该项目不直接托管任何内容，而是通过结构化方式收集、分类并索引来自全球互联网的高质量技术文档、工具站点、学术资料与行业动态链接，帮助用户在高密度信息环境中快速定位所需资源。

项目定位为技术资源的中转枢纽与个人知识管理的辅助工具，适用于需要频繁查阅外部参考资料、跟踪技术演进或建立个人学习路径的工程师、研究员与学生群体。ResourceBridge 本身不依赖外部数据库，所有资源以纯文本形式维护，支持静态部署与本地离线使用。

## 功能概览

- **资源分类索引** 按技术领域、内容类型与适用场景对链接进行多维度标记，支持快速筛选与定位。

- **批量导入与去重** 支持从标准格式文本批量导入链接，自动检测重复条目并生成冲突报告。

- **链接健康检查** 内置周期性链接可达性检测功能，输出失效链接清单并记录状态变更历史。

- **全文检索与标签过滤** 基于标题、描述与自定义标签的轻量级全文检索，支持多标签组合过滤。

- **自定义分类体系** 允许用户建立多级目录结构与标签系统，适应不同使用习惯与组织需求。

- **静态站点生成** 提供将资源数据导出为静态 HTML 目录页面的命令行工具，便于内网部署或文档站点集成。

- **元数据扩展支持** 每条资源可附加备注、优先级、阅读状态与更新日期等元字段，支持个人知识管理场景。

## 应用场景

- **技术团队内部知识库构建** 团队可将常用开发文档、API 参考、设计规范与运维手册链接统一收录，通过 ResourceBridge 生成内部导航页面，减少重复查找时间。

- **个人学习路线规划与资料整理** 学习者可按主题建立学习资源集合，例如前端框架、机器学习、系统设计等，通过标签与备注跟踪学习进度和资料质量。

- **技术资讯聚合与行业动态追踪** 整合技术博客、会议演讲录像、白皮书与发布公告等外部信息源，定期执行健康检查以维持资源有效性，避免链接失效造成的资讯断层。

## 快速开始

以下命令将项目克隆至本地、安装依赖并启动开发服务。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行完毕后，访问控制台输出的本地地址即可开始使用资源管理界面。生产环境部署请参考文档导航章节中的部署指南。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境与包管理基础 |
| npm | 9.x 或以上 | 依赖安装与脚本执行 |
| Git | 2.30 或以上 | 版本控制与克隆操作 |
| 现代浏览器 | Chrome 110+ / Firefox 110+ / Edge 110+ | 管理界面访问 |
| 磁盘空间 | 至少 50 MB | 存储资源索引与本地缓存 |
| 网络访问 | 外网连通（推荐） | 用于链接健康检查与资源访问 |
| 操作系统 | Linux / macOS / Windows（WSL2 推荐） | 跨平台支持 |

│   ├── core/                     # 核心数据模型与索引引擎
│   │   ├── resource.ts           # 资源条目数据结构定义
│   │   ├── indexer.ts            # 索引构建与查询接口
│   │   └── validator.ts          # 链接格式与可达性校验
│   ├── cli/                      # 命令行工具入口
│   │   ├── commands/             # 子命令实现（add, check, export）
│   │   └── main.ts               # CLI 主入口与参数解析
│   ├── web/                      # Web 管理界面
│   │   ├── components/           # UI 组件（列表、筛选、详情）
│   │   ├── pages/                # 路由页面（首页、分类、设置）
│   │   └── hooks/                # 状态管理与数据请求钩子
│   ├── storage/                  # 持久化适配器
│   │   ├── file.ts               # 本地文件系统读写
│   │   └── memory.ts             # 内存缓存与批量操作
│   └── utils/                    # 通用工具函数
│       ├── url.ts                # URL 解析与规范化
│       └── logger.ts             # 结构化日志输出
├── docs/                         # 完整文档目录
│   ├── getting-started.md
│   ├── features/
│   ├── deployment/
│   └── development/

## 常见问题

**问：ResourceBridge 是否存储外部资源的副本或缓存内容？**

答：ResourceBridge 仅存储链接地址、标题、描述与标签等元数据，不下载或缓存任何外部资源的内容。链接健康检查仅发送 HEAD 请求验证可达性，不读取响应体。所有数据以纯文本形式保存在用户本地，项目本身不提供云存储或数据同步服务。

**问：如何迁移或备份已整理的资源数据？**

答：所有资源数据存储于项目目录下的 `data/` 文件夹中，格式为 JSON Lines。直接复制该文件夹或打包压缩即可完成备份。迁移至新环境时，将数据文件夹放置于相同相对路径，项目启动时会自动加载。

**问：链接健康检查是否会对外部站点造成压力？**

答：健康检查模块采用并发控制与请求间隔策略，默认每秒最多发送 5 个请求，且仅针对用户主动触发的检查操作或定时任务（默认每周一次）运行。检查结果本地记录，不对外公开。
