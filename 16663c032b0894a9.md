# LinkVault

LinkVault 是一个面向技术内容策展人与开发者的外链资源归集与导航工具。项目定位为轻量级资源链接管理中间件，帮助团队或个人将散落在各处的外部技术文章、文档、教程与参考链接进行结构化存储、分类索引与快速检索。目标用户包括开源社区维护者、技术博客作者、知识库管理员以及任何需要系统化管理大量外链的开发者。

LinkVault 不试图替代书签管理器或笔记软件，而是提供一套可编程的链接元数据管理方案，支持标签体系、来源追踪、访问状态监控与批量导入导出，使外链从零散的收藏夹条目转化为可被检索、分析与共享的结构化数据资产。

## 功能概览

- **结构化链接存储**：每条链接支持标题、描述、标签、来源站点、收录时间、最后访问状态等十余个元数据字段，数据存储基于 JSON 文件或 SQLite，无需外部数据库依赖。

- **批量导入与去重检测**：支持从 CSV、Markdown 列表、纯文本 URL 清单批量导入链接，导入过程中自动执行基于 URL 标准化的去重检测，避免重复收录。

- **标签与分类树**：支持无限层级的标签体系与分类树，可为链接分配多个标签，支持按标签组合筛选与树形结构浏览。

- **链接可用性检查**：内置异步链接健康检查模块，可配置定时任务对已收录链接进行 HTTP 状态码检测，自动标记失效链接并记录响应时间变化。

- **多格式导出**：支持将链接列表导出为 Markdown 表格、JSON 结构化数据、HTML 导航页或 CSV 格式，便于嵌入文档、博客或迁移至其他系统。

- **只读 API 接口**：提供基于 REST 风格的只读查询接口，支持按标签、关键词、时间范围检索链接，便于与其他工具或脚本集成。

- **全文搜索**：基于标题与描述字段的轻量级全文搜索，支持中文分词与模糊匹配，检索结果按相关性排序。

## 应用场景

- **技术文档站外链管理**：技术文档中常引用大量外部参考资料，LinkVault 可为文档团队提供统一的外链索引库，在撰写文档时快速查询可用参考链接，并在链接失效时提前预警，避免文档中出现死链。

- **开源项目资源汇总页构建**：开源项目 README 或官网需要列出生态相关资源，LinkVault 可将分散的链接分类存储并生成 Markdown 格式资源列表，直接复制粘贴至项目文档，大幅减少手动整理工作。

- **个人技术博客参考文献归档**：技术博主撰写文章时收集的参考链接可通过 LinkVault 统一归档，为每篇文章生成独立的链接集合，方便日后复查或公开分享参考资料库。

- **社区知识库外链整合**：技术社区或论坛的知识库板块往往包含大量外部链接，LinkVault 可作为后台管理工具对链接进行审核、分类与状态监控，提升知识库的可维护性。

- **团队内部技术周报素材收集**：团队成员可随时将阅读到的优质技术文章链接存入 LinkVault，周报负责人通过标签筛选本周新增链接，一键导出为周报素材列表。

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/your-org/linkvault.git

# 进入项目目录
cd linkvault

# 安装依赖（使用 pip 与虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化数据库与配置
python linkvault.py init

# 启动 Web 界面（默认监听 127.0.0.1:8080）
python linkvault.py serve
```

## 常见问题

**Q：LinkVault 是否支持从浏览器书签直接导入？**

目前不支持直接读取浏览器书签文件（如 Chrome 的 Bookmarks 文件或 Firefox 的 places.sqlite）。建议先将书签导出为 HTML 格式，再通过 LinkVault 的批量导入功能将 HTML 中的链接提取并收录。后续版本计划增加对 Netscape 书签格式的直接解析支持。

**Q：链接健康检查是否会频繁访问目标站点，导致被屏蔽？**

LinkVault 的检查模块默认使用 5 秒超时与 2 秒请求间隔，并发数限制为 10 个连接。对于大规模链接库，建议将检查任务安排在非高峰时段运行，或通过配置文件调整并发数与间隔时间，以降低对目标站点的访问压力。

**Q：如何迁移数据到另一台机器？**

直接复制 data/links.db 文件至新机器的相同相对路径下即可。如果需迁移至其他数据库系统，可使用导出功能先生成 JSON 格式的数据备份，再通过导入功能还原，但需注意不同数据库间的字段类型差异。
