# LinkVault Core

LinkVault Core 是一个面向技术内容创作者、开源项目维护者以及数字资源管理者的高可靠性外链资源汇总与导航系统。该项目定位于解决技术文档、项目 README 及知识库中分散链接难以统一维护、失效检测与分类管理困难的问题，通过结构化的数据组织与自动化工具链，帮助用户构建长期稳定的外部资源引用体系。

目标用户包括需要维护大量外部引用链接的开源项目作者、技术博客写手、在线课程讲师以及企业知识库管理员。LinkVault Core 不提供链接内容的生产或代理服务，而是作为链接资源的元数据管理中间层，提供校验、分类、版本记录与批量导出功能，确保项目文档中的每一个外部引用均可被追溯与验证。

## 功能概览

批量链接导入解析：支持从纯文本、CSV 及常见书签导出格式中批量提取 URL 与锚文本，自动去重并识别顶级域名分布。

链接状态健康检查：内置异步 HTTP 头探测与响应超时检测机制，可配置重试策略与状态码白名单，定期输出失效链接报表。

自定义分类标签系统：允许用户为每条链接附加多级标签与备注说明，支持按项目批次、资源类型或优先级进行多维筛选。

结构化目录树导出：根据用户定义的分类规则，自动生成适用于 README、Wiki 或静态站点生成器的 Markdown 嵌套列表与表格代码。

链接变更追踪日志：记录每条链接的添加时间、最后检查时间与响应状态变化历史，便于审计与问题回溯。

权限与协作支持：提供基于项目组的读写权限划分，允许多位协作者同时维护不同分类下的链接集合，操作日志完整可溯。

开放 API 与钩子机制：提供 RESTful 查询接口与 Webhook 触发器，可与 CI/CD 流水线或即时通讯机器人集成，实现链接状态的自动告警。

## 应用场景

技术文档 README 外链规范化维护：开源项目维护者可使用 LinkVault Core 批量管理项目文档中引用的教程、工具站与 API 参考链接，定期自动检测死链并生成更新报告，有效提升文档质量与用户体验。

在线课程资源包整理：讲师或教育机构在准备课程材料时，可将大量推荐阅读、视频地址与代码仓库链接导入系统，按课时或主题分类，并导出为整洁的资源列表页，方便学员一站式访问。

企业知识库外部引用审计：企业知识管理团队可利用本系统的变更日志与健康检查功能，对公司内部 Wiki 中引用的外部技术博客、行业报告与数据源链接进行周期性合规审查，降低信息失效风险。

个人技术博客友情链接管理：技术博主可通过 LinkVault Core 维护交换友情链接的站点列表，自动监控对方站点可访问性，并在博客侧边栏或独立页面中动态生成经过筛选的活跃链接展示区。

## 快速开始

以下命令演示了从 GitHub 克隆项目、安装依赖并启动本地开发服务的完整过程。

```bash
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core
pip install -r requirements.txt
python manage.py runserver
```

执行上述命令后，开发服务器将在本地 8000 端口启动，访问 http://127.0.0.1:8000 即可进入仪表盘界面。首次启动将自动创建 SQLite 数据库文件与默认管理员账户，管理员账户密码将在控制台输出中显示，请及时记录并修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 至 3.12 | 核心运行环境，低于 3.9 版本将导致异步 IO 兼容性问题 |
| Pip | 22.0 或更高 | 包依赖管理工具，用于安装 requirements.txt 中列出的所有库 |
| SQLite | 3.35 或更高 | 默认内置数据库，用于存储链接元数据与检查日志，生产环境可切换至 PostgreSQL |
| Redis | 6.2 或更高 | 可选组件，用于分布式任务队列与缓存加速，非强制安装 |
| Node.js | 16.0 或更高 | 仅用于前端静态资源构建，后端运行无需此环境 |
| Nginx | 1.18 或更高 | 生产环境反向代理与静态文件服务推荐方案，开发环境可跳过 |
| Docker | 20.10 或更高 | 容器化部署支持，可使用 Docker Compose 一键启动全套服务栈 |
| Git | 2.25 或更高 | 版本控制工具，用于克隆仓库与管理配置变更 |

## 项目结构

```
linkvault-core/
├── cmd/                                # 命令行入口与运维工具脚本
│   ├── server/                         # Web 服务器启动入口
│   │   └── main.go                     # 主程序入口，负责加载配置与路由初始化
│   └── checker/                        # 独立运行的链接状态检查器
│       └── main.go                     # 定时任务执行器，可独立编译为二进制
├── internal/                           # 内部核心逻辑模块，不对外公开
│   ├── model/                          # 数据模型与 ORM 映射定义
│   │   ├── link.go                     # 链接记录结构体与状态常量
│   │   └── tag.go                      # 标签分类树结构体
│   ├── service/                        # 业务服务层实现
│   │   ├── import.go                   # 批量导入解析服务
│   │   └── export.go                   # Markdown/JSON 导出服务
│   └── handler/                        # HTTP 请求处理器与路由绑定
│       ├── api.go                      # RESTful API 接口实现
│       └── web.go                      # 后台管理界面路由
```

## 常见问题

Q: 系统提示数据库被锁定，无法执行迁移操作，应如何解决？

A: 此问题通常发生在 SQLite 作为后端数据库且有多个进程同时写入时。解决方法为：首先检查是否有其他终端或后台任务正在运行 manage.py 或 checker 进程，使用 pkill -f linkvault 终止所有相关进程。若问题依旧，请删除 db.sqlite3 文件并重新执行迁移脚本，该操作将重置所有数据，请提前备份重要记录。对于生产环境，建议切换至 PostgreSQL 以规避此类并发锁问题。

Q: 链接健康检查任务总是超时失败，但手动访问目标 URL 是正常的，是什么原因？

A: 默认的 HTTP 超时时间为 3 秒，若目标服务器响应较慢或存在网络延迟抖动，可能导致探测超时。您可以在 configs/app.yaml 中调整 checker.timeout_seconds 参数，建议逐步增加至 10 秒或 15 秒进行测试。同时，请检查部署环境的出口网络防火墙策略，确保允许出站 HTTPS 连接。如果大量链接集中在特定域名下，也可能是该站点启用了反爬机制，您可以在配置中启用浏览器 User-Agent 伪装。

Q: 如何将已有的大量书签文件一次性导入系统，而不需要逐条手动添加？

A: LinkVault Core 支持 Netscape 书签导出格式（即浏览器书签管理器导出的 .html 文件）以及纯文本每行一个 URL 的 .txt 文件。您可以在 Web 界面的导入页面选择对应格式并上传文件，系统将自动递归解析目录结构并转换为标签分类。若需处理 JSON 格式的第三方数据，请参考 API 文档中的 /api/v1/import/json 接口规范，通过 POST 请求批量提交。
