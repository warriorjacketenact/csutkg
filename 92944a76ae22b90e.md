# WebLink Navigator

WebLink Navigator 是一个面向开发者、技术研究人员与信息分析师的轻量级外链资源归集与导航系统。该项目并非传统的网址收藏夹或书签管理器，而是一套结构化的外链数据治理工具，专注于对海量分散链接进行批量导入、分类标注、健康状态监测与访问路径优化。

项目目标用户包括需要维护大量外部参考链接的技术文档编写者、从事互联网信息采集与数据分析的工程师、以及需要系统化管理项目依赖文档链接的架构师。WebLink Navigator 解决的核心问题是：当外链数量超过数百条时，如何保持链接的可访问性、分类可读性与更新可追踪性，避免链接失效导致的文档空洞或信息断层。

本项目的设计理念强调对原始链接的高度保真存储与展示，不对链接内容做摘要改写或重定向代理，确保用户始终直达原始资源。项目提供命令行界面与轻量级 Web 仪表盘两种交互方式，适配服务器端批处理与本地桌面快捷访问两种工作流。

## 功能概览

**批量链接导入** 支持从纯文本列表、CSV 文件或标准输入流中批量读取 URL，自动去重并生成唯一内部标识符，每条链接保留原始地址、导入时间戳与初始状态标记。

**链接健康检查** 内置异步 HTTP 探活模块，可配置超时与重试策略，定期检测每条外链的可访问性，返回状态码与响应时长，标记异常链接供人工复核。

**分类标签系统** 支持为链接添加多级标签与自定义分类目录，标签可基于域名、路径模式或用户自定义规则自动生成，便于后续按主题或来源筛选检索。

**全文检索与过滤** 基于链接的 URL 字符串、导入备注、标签集合与状态字段提供全文搜索能力，支持正则表达式过滤与多条件组合查询。

**数据导入导出** 支持将链接数据集导出为 JSON、YAML 或纯文本列表格式，便于与其他工具链集成或进行版本控制管理。

**访问统计看板** 记录每条链接的被访问次数与最后访问时间，生成简单的热度统计，帮助识别高频使用的核心资源。

**冗余链接检测** 分析链接集合中的重复目标地址或高度相似的路径模式，提示可能存在的内容重复或镜像站点。

**快照备注附件** 允许用户为每条链接附加简短备注或摘要文本，记录个人对该资源的评价、摘要或使用心得，备注内容参与全文检索。

## 应用场景

技术文档维护与参考链接管理 在编写开源项目文档、技术博客或内部技术方案时，需要引用大量外部规范、论文或项目主页。WebLink Navigator 可用于统一管理这些引用链接，定期检查链接有效性，避免文档中出现死链，同时通过标签分类快速定位特定领域的参考资料。

数据采集管道中的链接队列管理 在互联网数据采集或爬虫项目中，中间环节会产生大量的待抓取 URL 队列。本项目可作为轻量级链接暂存与调度工具，对采集到的链接进行去重、分类和状态标记，协助开发者追踪哪些链接已处理、哪些链接响应异常需要重试。

团队知识库外链资产盘点 企业或开源社区的团队知识库（如 Wiki、Confluence 或 Markdown 文档集合）中散落着大量外部链接。通过批量导入这些链接，团队可以获得一份完整的外链资产清单，识别关键依赖，并在外部服务迁移或下线时提前预警。

个人书签库的定期维护与清理 个人浏览器书签数量积累过多后往往缺乏维护。本项目提供命令行下的链接健康检查和分类整理能力，帮助用户快速筛选出已失效的旧书签，并按域名或主题重新组织有效链接，恢复书签库的可用性。

## 快速开始

以下命令展示从克隆仓库到启动服务的完整流程。

```bash
git clone https://github.com/example/weblink-navigator.git
cd weblink-navigator
pip install -r requirements.txt
python cli.py import --input links.txt
python cli.py health-check --timeout 5 --concurrency 10
python web.py --port 8080
```

上述操作依次执行：克隆项目仓库至本地，进入项目目录，安装 Python 依赖，从 links.txt 文件批量导入链接，对已导入链接进行健康状态检查，最后启动 Web 仪表盘服务监听本地 8080 端口。

1. 查阅 issue 列表与项目看板，选择未被指派的待办任务或提议新增功能。对于较大规模的功能改动，建议先在 issue 中描述设计思路与影响范围，获得核心维护者反馈后再着手实现。

2. 派生项目仓库至个人账号，基于 main 分支创建以 feature/ 或 fix/ 为前缀的特性分支。提交信息采用约定式提交格式，即 type(scope): description 的形式，其中 type 包括 feat、fix、docs、style、refactor、test、chore 等类别。

3. 编写或更新对应的单元测试与集成测试，确保新代码的测试覆盖率达到 80% 以上。所有测试用例应能在本地通过 pytest 运行，且不依赖外部网络环境，外部 HTTP 请求需使用 aioresponses 等库进行模拟。

4. 更新文档目录下与变更相关的用户文档或 API 文档，保持文档与实际代码行为一致。若新增命令行参数，需同步更新 docs/commands.md 中的参数表格与示例。

## 常见问题

Q: 导入链接时遇到 SSL 证书验证错误，应如何解决？
A: 健康检查模块默认启用 SSL 证书验证。如果目标站点使用自签名证书或内部 CA 签发的证书，可在配置文件中将 ssl_verify 设置为 false，或在命令行中传入 --no-ssl-verify 选项。请注意，关闭验证会降低安全性，仅应在受信任的内部网络环境中使用。

Q: Web 仪表盘无法显示任何链接，但命令行导入显示成功，可能是什么原因？
A: 首先确认 Web 服务读取的数据库路径与命令行导入时使用的路径一致，默认路径为项目根目录下的 data/links.json。其次检查数据文件是否具有可读权限，以及 Web 服务是否以正确的用户身份运行。如果问题仍然存在，可查看 Web 服务输出的日志信息，确认是否存在 JSON 解析异常或字段不匹配的情况。

Q: 健康检查占用大量网络带宽或 CPU 资源，如何进行调整？
A: 健康检查模块的并发度与超时时间均可通过配置文件或命令行参数调整。建议将并发度设置为不超过 50，超时时间设置为 5 至 10 秒，并启用 --rate-limit 选项限制每秒请求数。对于大规模链接集合，可分批执行健康检查，每次仅检查部分链接，避免对目标服务器造成过大压力。
