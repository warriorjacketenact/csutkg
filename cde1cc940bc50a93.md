# VideoIndex

VideoIndex 是一个面向技术内容创作者与开发者的视频资源索引与管理平台。项目定位为外链资源的中枢管理系统，通过标准化的 URL 映射与分类体系，将分散的视频资源整合为结构化知识库。目标用户包括开源文档维护者、技术教育平台运营方以及需要大量外链支撑的学习资源站点管理员。

VideoIndex 解决的核心问题是视频链接缺乏有效组织与快速检索通道。大量视频资源分散于不同项目、不同文档中，难以统一维护与更新。本项目提供一套轻量级索引框架，支持按主题、批次、用途对链接进行标记与归类，并内置链接可用性检测与状态监控机制，确保资源列表始终处于可访问状态。


## 功能概览

**批量链接导入** 支持通过 CSV、JSON 或纯文本列表形式批量导入视频 URL，自动解析域名与路径参数，提取资源唯一标识符。

**分类标签系统** 允许用户为每条链接添加自定义标签（如 tutorial、demo、interview、workshop），并依据标签进行快速筛选与分组展示。

**可用性监控** 内置异步 HTTP 健康检查模块，定期探测链接响应状态，异常链接自动标记并生成报告，便于运维人员及时处理失效资源。

**索引快照** 支持生成当前资源列表的只读快照，包含时间戳与哈希校验值，确保历史引用记录的完整性与可追溯性。

**导出适配器** 提供多种导出格式，包括 Markdown 表格、JSON API 响应结构、HTML 目录页以及 RSS 订阅源，适配不同下游系统的数据消费需求。

**搜索与过滤** 基于路径关键词与自定义元数据的全文检索引擎，支持模糊匹配与正则表达式查询，提升大型资源库的定位效率。


## 应用场景

开源项目文档站的外链管理。开源项目常在 README 或 Wiki 中引用大量外部视频教程与演示链接，手动维护容易出现死链或版本过时问题。VideoIndex 提供集中式链接注册与校验能力，文档维护者可定期同步索引状态，确保引用资源长期有效。

技术教育平台的课程资源聚合。在线教育平台需要将课程视频、实验演示、讲师访谈等素材统一归档并对外展示。VideoIndex 的分类标签与快照功能能够按课程编号、讲师姓名、主题领域等多维度组织资源，简化内容运营工作流。

企业内部知识库的视频引用治理。企业内网知识库中嵌入的视频链接往往来自不同供应商或内部录制系统，链接格式不统一且缺少生命周期管理。VideoIndex 的批量导入与导出适配器可与企业内部 CMS 对接，实现视频资源的集中登记与变更通知。


## 快速开始

以下命令演示如何在本地环境中获取源码、安装依赖并启动开发服务。

```bash
git clone https://github.com/videoindex/videoindex-core.git
cd videoindex-core
pip install -r requirements.txt
python app.py --init --port 8080
```

```
├── app.py                         # 应用程序入口，初始化索引引擎与路由注册

│   ├── default.yaml               # 默认配置项，包含数据库路径与探测间隔参数
│   └── schema.json                # 链接元数据 JSON Schema 定义
├── indexer/
│   ├── __init__.py                # 索引模块初始化
│   ├── loader.py                  # 批量链接导入与格式解析器
│   ├── classifier.py              # 标签分类引擎，支持规则匹配与贝叶斯分类
│   └── snapshot.py                # 快照生成与哈希校验
├── monitor/
│   ├── health.py                  # 异步 HTTP 健康检查实现

├── exporters/
│   ├── markdown.py                # Markdown 表格导出器

├── web/
│   ├── static/                    # CSS 与 JavaScript 静态资源

├── tests/

└── README.md                      # 项目说明文档
```


## 贡献指南

1. 在 GitHub Issues 中查阅当前未分配的任务列表，选择一项与自身技能匹配的工作项，并在评论中表明认领意向，等待维护者确认。

2. 从 main 分支创建功能分支，分支命名格式为 feature/功能简述或 fix/问题编号，确保分支名称具有描述性。

3. 编写代码时遵循项目根目录下的 .editorconfig 与 pylintrc 规范要求，新增功能需附带对应的单元测试用例，测试覆盖率不低于百分之八十。

4. 提交前执行 tox 或 pytest 命令确保全部测试通过，并运行 pre-commit 钩子进行代码风格自动修正。

5. 发起 Pull Request 至 main 分支，在描述中关联相关 Issue 编号，并附上变更摘要与测试结果截图或日志片段。


## 常见问题

问：导入超过一千条链接时出现内存占用过高现象，应如何优化？

答：建议启用批量流式导入模式，在 loader.py 中设置 chunk_size 参数为 100，同时使用 SQLite 的 WAL 日志模式提升写入并发性能。若仍存在瓶颈，可考虑迁移至 PostgreSQL 后端。

问：健康检查模块误报大量链接为不可用状态，如何调整敏感度？

答：检查 monitor/health.py 中的 timeout 与 retry 参数。默认超时为 5 秒，重试次数为 2 次。对于响应较慢的源站，可适当将 timeout 提升至 15 秒，并将重试次数调整为 3 次。同时确认网络环境是否存在防火墙或代理限制。

问：如何将现有链接数据从旧版 VideoIndex 迁移至当前版本？

答：执行 scripts/migrate_v1_to_v2.py 脚本，该脚本支持读取旧版 SQLite 数据库或 JSON 导出文件，自动映射字段并生成新版兼容的索引记录。迁移前建议对原数据库执行完整备份。
