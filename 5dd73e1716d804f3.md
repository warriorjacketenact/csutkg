# Hyperlink Synchronized Catalog Generator (HSCG)

Hyperlink Synchronized Catalog Generator (HSCG) 是一个面向技术文档团队、知识库维护者和内容聚合平台的开源外链资源汇总与结构化管理系统。HSCG 旨在解决大规模 URL 资源在分散存储、缺乏元数据标注、难以检索和版本追踪等方面的痛点问题，为技术写作者和信息架构师提供一套标准化的链接治理工具链。

本项目的目标用户包括开源项目文档维护者、企业技术博客运营团队、在线教育平台课程资源整理人员以及需要批量管理外部引用链接的软件开发团队。HSCG 不仅仅是一个链接清单，它通过自动化的资源分类、状态检测和结构化输出，将原始的 URL 列表转化为可维护、可追溯、可集成的知识资产。

## 功能概览

批量链接标准化导入 支持从纯文本、CSV 及 JSON 格式批量导入原始 URL 列表，自动识别协议头和域名结构，对畸形 URL 进行清洗和归一化处理。

递归资源状态探测 对每个收录链接执行周期性的可达性检查，返回 HTTP 状态码、响应时长和重定向链信息，标记失效或迁移的资源。

自定义标签与分类引擎 允许用户为每个链接分配多级标签，支持基于 URL 路径模式、域名特征或自定义正则规则的自动打标，实现资源的精细化管理。

结构化目录树生成 根据链接的域名、子目录层级和预设的分类规则，自动生成符合项目结构的 ASCII 目录树，便于版本控制系统的变更追踪。

多格式文档导出 内置 Markdown、reStructuredText 和 HTML 模板引擎，可将链接库导出为技术文档中所需的资源列表、脚注索引或站点地图。

链接变更审计日志 记录每条链接的添加时间、修改历史、状态变更和访问元数据，提供完整的操作审计轨迹，满足企业内部合规要求。

API 服务接口 提供基于 RESTful 架构的 HTTP API，支持第三方工具通过 JSON 格式查询链接库、提交新资源或触发状态批量检测。

## 应用场景

技术文档版本迁移辅助 当技术文档团队将旧版文档站点迁移至新的域名或静态站点生成器时，HSCG 可批量扫描文档源文件中的所有外链，通过状态检测功能定位已失效的引用，并生成重定向映射表，大幅降低迁移后链接破碎的风险。

在线课程资源中心维护 教育平台运营者可将每门课程涉及的外部阅读材料、参考文献和视频源链接统一录入 HSCG，按课程编号和章节进行分类。定期的状态检测能够主动通知维护者移除或替换已下架的外部资源，保证学习体验的连贯性。

开源项目 README 及 Wiki 协同管理 大型开源项目的文档贡献者分布在不同时区，通过 HSCG 维护的中央链接仓库，可以避免多人编辑 Wiki 时产生的重复引用和死链接。每次代码发版前，发布工程师可运行 HSCG 检查工具，确保所有引用链接处于有效状态。

企业内部知识库合规审计 金融或医疗行业的企业内部知识库需要定期审计外部引用是否符合数据安全政策。HSCG 的标签系统和审计日志能够帮助合规官快速筛选出特定域名的链接，追溯链接的引入时间和引入者，并生成符合监管要求的报告。

## 快速开始

以下命令演示了如何在本地环境中克隆 HSCG 仓库、安装依赖并启动开发服务器。

```bash
# 克隆项目仓库至本地
git clone https://github.com/hscg/hscg-core.git
cd hscg-core

# 安装 Python 依赖包（推荐使用虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 系统请使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化 SQLite 数据库并导入示例链接数据
python manage.py initdb
python manage.py load-fixture fixtures/sample_links.json

# 启动开发模式 API 服务，默认监听 127.0.0.1:8000
python manage.py runserver
```

## 安装要求

HSCG 采用 Python 3.10+ 作为核心运行环境，以下表格列出了所有必需的依赖项及其版本约束。

| 依赖名称 | 必需性 | 说明 |
|---|---|---|
| Python 3.10 至 3.12 | 必需 | 核心解释器，低于 3.10 版本将无法解析类型注解语法 |
| SQLite 3.35.0+ | 必需 | 默认嵌入式数据库，用于存储链接元数据和审计日志 |
| aiohttp 3.9.0+ | 必需 | 异步 HTTP 客户端，用于并发执行链接状态探测 |
| lxml 4.9.0+ | 可选 | 若启用 HTML 元数据抓取功能（如页面标题提取），需安装此库 |
| PyYAML 6.0+ | 可选 | 用于支持 YAML 格式的分类规则配置文件导入 |
| pytest 8.0+ | 开发依赖 | 单元测试框架，运行测试套件时必需 |
| black 24.0+ | 开发依赖 | 代码格式化工具，保证提交代码风格一致性 |
| mkdocs 1.6.0+ | 可选 | 用于生成项目文档站点，若需本地预览文档则安装 |
| docker-compose 2.20.0+ | 可选 | 若使用容器化部署方案，需配合 Docker Engine 使用 |

## 项目结构

HSCG 代码仓库遵循分层架构设计，核心模块按功能垂直切割，以下为项目根目录的 ASCII 目录树及注释说明。

```
hscg-core/
├── hscg/                                    # 核心 Python 包目录
│   ├── __init__.py                          # 包版本号及导出符号声明
│   ├── cli/                                 # 命令行接口子模块
│   │   ├── __init__.py
│   │   ├── main.py                          # argparse 入口，命令路由
│   │   └── commands/                        # 各子命令实现（initdb、check、export）
│   ├── core/                                # 核心数据模型与业务逻辑
│   │   ├── __init__.py
│   │   ├── link.py                          # Link 数据类定义，含 URL 解析方法
│   │   ├── catalog.py                       # 分类规则引擎，标签匹配逻辑
│   │   └── status.py                        # 状态检测异步任务编排
│   ├── storage/                             # 持久化层抽象与实现
│   │   ├── __init__.py
│   │   ├── base.py                          # 抽象基类定义（Repository 接口）
│   │   ├── sqlite_repo.py                   # SQLite 具体实现，含迁移脚本
│   │   └── postgres_repo.py                 # PostgreSQL 实现（生产环境）
```

## 贡献指南

HSCG 项目欢迎技术文档工程师、Python 开发者和开源爱好者提交贡献。所有贡献者需遵循以下步骤完成代码或文档的提交。

第一步：阅读行为准则 请仔细阅读项目 CODE_OF_CONDUCT.md 文件，确保认同并遵守开源社区的基本礼仪与协作规范。

第二步：选择待处理任务 访问 GitHub Issues 页面，查找带有 help-wanted 或 good-first-issue 标签的任务。如希望实现新功能，请先创建一个 Issue 描述设计方案，与维护者达成一致后再开始编码。

第三步：本地开发环境准备 克隆仓库后，运行 make dev-setup 命令自动创建虚拟环境并安装所有开发依赖。执行 pre-commit install 注册 Git 钩子，确保每次提交代码时自动执行格式化和基础语法检查。

第四步：编写测试与代码 所有新增功能必须包含对应的单元测试，测试用例放置在 tests/ 目录下对应子目录中。代码风格需符合 Black 和 Flake8 的检查规则，提交前请运行 make lint 和 make test 确保全部通过。

第五步：提交 Pull Request 提交 PR 时请填写提供的模板，清晰说明变更动机、实现方式和测试覆盖情况。PR 标题需遵循 Conventional Commits 格式（如 feat: 或 fix: 前缀）。至少一位项目维护者审核通过后，代码将被合并至主分支。

## 常见问题

问题：HSCG 能否处理需要登录认证或带有反爬机制的链接状态检测？

回答：当前版本仅支持对公开可访问的链接进行基础的 HTTP 状态码检测。对于需要携带 Cookie 或 User-Agent 定制的情况，用户可在配置文件中为特定域名设置请求头覆盖策略。但对于验证码、JavaScript 动态渲染或 IP 封锁等复杂场景，建议配合专业的可用性监控服务使用，HSCG 本身不提供绕过访问限制的能力。

问题：导入的链接数量达到数万条时，状态检测任务如何控制并发以避免目标服务器过载？

回答：HSCG 的状态检测引擎支持用户配置两个关键参数：并发连接数上限和请求间隔延迟。默认并发数为 50，间隔延迟为 200 毫秒，该组合适合大多数公共站点的访问频率限制。用户可在 settings.toml 文件中调整 concurrency_limit 和 rate_delay 字段，对于对稳定性要求高的企业内部资源，建议设置更保守的数值。

问题：如何将 HSCG 与现有的 MkDocs 或 Sphinx 文档生成流程集成？

回答：HSCG 提供了命令行导出能力。用户可在文档构建脚本中添加 hscg export markdown --output docs/links.md 命令，将当前链接库导出为 Markdown 文件。对于 Sphinx 项目，可进一步使用 m2r2 库将导出的 Markdown 转换为 reStructuredText 格式，并通过 .. include:: 指令嵌入至最终页面中，实现链接资源的集中维护和自动更新。
