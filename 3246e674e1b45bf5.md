# LinkVault 聚合资源索引系统

LinkVault 是一个面向技术社区与内容创作者的轻量级聚合资源索引平台，专为批量管理、分类导航与快速检索外部链接而设计。项目定位于解决信息碎片化场景下的链接归集与分发问题，帮助运维人员、文档维护者与站点管理员将分散于多个来源的 URL 资源统一纳入结构化目录体系，并提供清晰的分类视图与静态页面生成能力。项目本身不存储资源实体，仅维护指向原始内容的索引记录，适用于建立个人书签库、团队知识导航或开源项目的外部依赖清单。

## 功能概览

- **批量链接导入与解析**：支持通过纯文本列表、CSV 或简单 Markdown 格式批量录入 URL，自动识别协议头与路径层级，并去重校验。

- **多级分类目录管理**：内置分类标签系统，允许为每个链接分配一个或多个主题标签（如技术文档、视频教程、工具站），并支持自定义分类树。

- **静态索引页生成**：基于配置的目录结构与链接数据，一键生成静态 HTML 页面，可直接部署至 Nginx、Apache 或对象存储服务，无需数据库依赖。

- **链接可用性检查**：提供可选的连通性探测功能，支持 HTTP/HTTPS 头请求验证，标记失效链接并生成健康报告。

- **全文检索与过滤**：为已索引的链接标题、描述与标签建立简单倒排索引，支持关键字模糊匹配与分类过滤。

- **访问统计埋点**：内置轻量级点击计数模块，记录每个链接的引用次数，便于评估资源热度（数据存储于本地 SQLite）。

- **RESTful 管理接口**：提供基于 JSON 的管理 API，支持链接的增删改查与分类调整，便于集成至自动化运维脚本。

- **数据导入导出**：支持将全量索引数据导出为 JSON 或 YAML 格式，亦支持从相同格式文件恢复数据，便于迁移与备份。

## 应用场景

- **技术文档站点外链管理**：技术博客或开源项目文档中常需引用大量外部参考资料，使用 LinkVault 可统一维护这些链接，避免散落于各处造成死链或版本混乱。

- **团队内部知识导航**：研发团队可将常用开发工具地址、内部系统入口、运维手册链接等集中收录，通过分类目录快速定位，减少沟通成本。

- **个人书签库云端化**：个人用户可将浏览器书签导出后导入 LinkVault，配合静态页面生成能力，获得一个自托管的、无商业干扰的书签门户。

- **开源项目依赖清单索引**：大型开源项目可将第三方依赖库的官网、文档、issue 跟踪器链接统一整理，作为项目 README 或 Wiki 的补充资源页。

## 快速开始

以下命令演示了从源码克隆、安装依赖到启动开发服务的完整流程。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装 Python 依赖（项目基于 Python 3.10+）
pip install -r requirements.txt

# 初始化 SQLite 数据库与默认分类
python manage.py initdb

# 导入示例链接数据（含用户提供的资源列表）
python manage.py import --file samples/urls.txt

# 生成静态索引页到 output/ 目录
python manage.py build --output ./output

# 启动本地预览服务（默认端口 8080）
python manage.py serve --port 8080
```

## 项目结构

项目采用分层设计，核心模块与辅助工具分离，便于维护与扩展。

```text
linkvault/
├── src/                           # 核心源码目录
│   ├── core/                      # 索引引擎与数据模型
│   │   ├── models.py              # SQLite 表定义（链接、分类、标签）
│   │   ├── indexer.py             # 链接导入、去重与解析逻辑
│   │   └── query.py               # 检索与过滤接口
```

5. 提交 Pull Request：描述变更目的、实现方式与测试结果，并关联相关 Issue 编号。PR 至少需一名维护者审核后方可合并。

## 常见问题

Q：项目是否必须联网运行？
A：LinkVault 核心功能（导入、索引、生成页面）完全离线运行，不依赖外部网络。唯一需要网络的是链接可用性检查模块，该功能默认关闭，可通过配置项开启。用户可根据实际环境决定是否启用。

Q：静态页面生成后，如何部署到生产服务器？
A：构建命令 python manage.py build 会在 output/ 目录下生成完整的静态站点。您可以将该目录下的所有文件复制至任意 HTTP 服务器（如 Nginx、Apache、Caddy）的根目录下，或上传至阿里云 OSS、AWS S3 等对象存储并开启静态网站托管功能。

Q：项目是否支持 HTTPS 链接的检查？
A：支持。链接检查器基于 aiohttp 库，默认遵循 HTTPS 协议，并支持自签名证书的跳过验证选项（配置中可关闭 ssl_verify）。检查结果会记录响应状态码与响应时间，便于分析。
