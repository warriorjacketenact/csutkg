# ResourceBridge

ResourceBridge 是一个面向技术团队与独立开发者的外链资源归集与导航系统。项目定位于解决多源技术资料分散、书签管理混乱、团队共享困难等实际问题，通过结构化的资源索引机制，将散落在各处的文档、工具、教程与参考链接统一收纳并提供快速检索入口。目标用户包括运维工程师、全栈开发人员、技术管理层以及需要维护大量外部依赖信息的项目维护者。

ResourceBridge 不提供资源存储功能，而是建立一套轻量级的资源映射表，配合元数据标签与分类体系，使团队能够以最低成本维护外部知识图谱。项目本身可作为静态站点部署，也可集成至现有后台系统作为资源模块。

## 功能概览

**批量链接导入** 支持从文本文件、CSV 表格及浏览器书签导出文件批量导入 URL 列表，自动解析标题与元信息。

**分类标签系统** 每个资源可绑定多个自定义标签，支持按标签组合过滤检索，满足多维度筛选需求。

**链接可用性监控** 定时检测已收录链接的 HTTP 状态码，标记失效或重定向资源，生成异常报告。

**团队共享空间** 支持多用户协同维护资源库，记录资源添加人、添加时间及修改历史，便于追溯。

**全文检索** 基于标题、描述、标签及 URL 关键词的快速搜索，支持模糊匹配与精确查询。

**数据导出与备份** 提供 JSON、CSV、Markdown 三种格式的数据导出功能，便于本地备份或迁移至其他平台。

## 应用场景

技术团队内部文档中心通常需要引用大量外部依赖库、API 参考与运维手册，ResourceBridge 可作为统一入口，避免每个成员独立维护书签，减少因人员流动导致的知识断层。团队 Leader 可通过资源访问日志了解成员常用工具分布。

开源项目维护者需要在 README 或官网中展示生态相关资源，ResourceBridge 生成的结构化数据可直接渲染为资源列表页，省去手动维护链接表格的重复工作，同时自动校验链接有效性。

个人开发者积累了大量技术博客、在线工具与代码片段链接，使用 ResourceBridge 的分类与检索功能可快速定位特定领域资源，相较浏览器书签的扁平化管理方式，标签组合筛选效率更高。

企业培训或技术调研阶段，需要将大量参考材料分发给参与者，ResourceBridge 的共享链接功能可生成临时访问凭证，限定有效期与访问次数，兼顾便捷性与安全管控。

## 快速开始

以下命令演示在 Linux 或 macOS 环境下从源码部署 ResourceBridge 服务。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
pip install -r requirements.txt
python app.py --init-db --port 8080
```

执行上述命令后，服务默认监听 8080 端口。首次运行时 --init-db 参数会自动创建 SQLite 数据表及默认管理员账户。管理员初始密码将在控制台输出，请及时修改。

若使用 Docker 部署，可执行以下替代方案：

```bash
docker pull resourcebridge/resourcebridge:latest
docker run -d -p 8080:8080 -v ./data:/app/data resourcebridge/resourcebridge:latest
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9 或更高版本 | 是 | 核心运行环境，低于此版本将导致异步语法错误 |
| SQLite 3.35 或更高版本 | 是 | 默认元数据存储引擎，支持 JSON 函数以处理标签字段 |
| pip 22.0 或更高版本 | 是 | 用于安装 requirements.txt 中列出的 Python 依赖包 |
| Node.js 18 或更高版本 | 否 | 仅当需要重新编译前端静态资源时必需，生产环境可跳过 |
| Redis 7.0 或更高版本 | 否 | 启用缓存与任务队列时使用，不配置则降级为内存缓存 |
| Nginx 1.24 或更高版本 | 否 | 生产环境反向代理推荐，非必须但可提升静态资源性能 |

## 项目结构

```
resourcebridge/
├── app/                           # 主应用目录
│   ├── api/                       # RESTful API 路由层
│   │   ├── v1/                    # API 版本 v1 端点实现
│   │   └── middleware/            # 认证、限流、日志中间件
│   ├── core/                      # 核心业务逻辑
│   │   ├── collector/             # 链接采集与解析引擎
│   │   ├── monitor/               # 可用性监控调度器
│   │   └── exporter/              # 数据导出格式化模块
│   ├── models/                    # 数据模型与 ORM 映射
│   │   ├── resource.py            # 资源实体模型
│   │   ├── tag.py                 # 标签模型与关联表
│   │   └── user.py                # 用户与权限模型
```

## 常见问题

问：导入大量链接时页面卡顿或超时，应如何处理？

答：批量导入接口默认单次限制 500 条记录，超出部分请分批提交。若需一次导入数千条链接，建议使用命令行导入工具 scripts/bulk_import.py，该工具通过后台任务方式处理，不占用 Web 请求超时时间。同时可调整配置文件中的 IMPORT_BATCH_SIZE 与 TASK_TIMEOUT 参数以适应更大规模操作。

问：链接可用性监控是否会影响被检测站点的访问统计？

答：监控模块默认采用 HEAD 请求检测，不下载响应体，且请求间隔不低于 300 秒，User-Agent 标识为 ResourceBridge-Monitor/1.0。若需排除特定域名，可在配置文件的 MONITOR_BLACKLIST 中添加白名单条目。对于内网或私有地址段，监控模块自动跳过检测以避免网络穿透风险。

问：能否将 ResourceBridge 部署为子目录而非独立域名？

答：支持通过配置 APPLICATION_ROOT 参数调整路由前缀，例如设置为 /resources 后，所有路由均以此前缀开头。但需注意前端静态资源路径需同步调整，且在反向代理层需正确重写 Location 头。建议使用独立子域名部署以避免路径重写带来的复杂问题。
