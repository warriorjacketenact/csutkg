# ResourceBridge

ResourceBridge 是一个面向技术团队与内容研究者的结构化外链资源归集与导航系统。项目定位于解决分布式信息环境下优质资源链接的散落、失活与难以检索问题，通过标准化元数据描述与分类索引机制，为开发者、数据分析师及技术决策者提供可复用、可扩展的外部资源挂载框架。本项目不生产原始内容，而是作为资源定位与组织层，帮助用户快速从海量外部站点中定位特定主题的视频素材、数据展示页与专题聚合入口。

ResourceBridge 适用于需要定期同步外部参考链接、构建垂直领域知识图谱或维护技术调研资产库的团队。项目本身以轻量级静态站点形式交付，可独立部署于任何支持 HTTP 服务的环境，也可作为子模块嵌入现有文档系统或数据中台。

## 功能概览

**多源链接挂载**：支持将来自不同域名、不同路径格式的外部链接统一纳入本地索引体系，保留原始 URL 完整语义。

**分类标签生成**：基于 URL 路径模式与域名特征自动生成内容分类建议，辅助人工进行主题标注。

**资源状态探测**：提供可配置的链接可达性检查脚本，定期检测资源是否仍可访问，输出状态报告。

**分层目录映射**：按照项目内置的目录树结构将外部链接映射到本地虚拟路径，方便与文档站点或 API 文档联动。

**元数据扩展接口**：预留 YAML 前置数据块，允许为每个链接附加描述、优先级、更新周期等自定义字段。

**静态站点生成**：内置简单的模板引擎，可将链接列表与元数据渲染为 HTML 导航页面，适合内网发布或 GitHub Pages 托管。

**批量导入导出**：支持 CSV 与 JSON 格式的链接批量导入导出，便于与其他数据工具（如 Excel、AirTable、ETL 流程）集成。

**权限参考模型**：提供基于路径前缀的访问控制示例配置，适用于需要区分内部与外部访问者的场景。

## 应用场景

技术调研资产库维护：技术预研团队在评估新框架或中间件时，通常需要收集大量官方文档、社区案例与性能测试报告。ResourceBridge 可为每个调研课题建立独立的链接集合，并保留来源站点的原始路径，避免调研成果因链接散落而失效。

数据分析管道参考源管理：数据工程师在构建 ETL 流程时，常参考多个数据源的接口文档与样例数据。通过 ResourceBridge 统一记录这些外部数据源入口，可降低因人员变动导致的知识流失风险。

内容聚合站点的种子链接池：内容运营或 SEO 团队需要定期从多个垂直站点采集更新动态。ResourceBridge 可作为种子链接池的基础设施，配合外部爬虫或监控工具使用。

开源项目文档的外部依赖索引：开源项目在 README 或贡献指南中常引用外部规范、标准或参考实现。使用 ResourceBridge 统一管理这些引用，可保证文档中的外链格式一致且便于批量更新。

## 快速开始

以下步骤适用于在 Linux 或 macOS 环境下完成项目克隆、依赖安装与本地运行。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
pip install -r requirements.txt
python main.py --serve --port 8080
```

执行上述命令后，静态导航站点默认在本地 8080 端口启动。访问 http://localhost:8080 可查看已挂载的资源列表。若需仅生成静态文件而不启动服务，可执行 `python main.py --build --output ./dist`。

## 项目结构

```
resourcebridge/
├── main.py                         # 命令行入口，负责解析参数并调度构建流程
├── requirements.txt                # Python 依赖声明，包含 requests、PyYAML、markdown 等
├── config/
│   ├── default.yaml                # 默认配置：端口、输出目录、探测超时阈值
│   └── schema.json                 # 元数据配置的 JSON Schema 校验文件
├── core/
│   ├── __init__.py
│   ├── loader.py                   # 从 YAML/CSV 加载链接列表与元数据
```

## 常见问题

Q：添加大量链接后，静态站点生成速度变慢，如何优化？

A：可调整 `config/default.yaml` 中的 `render.batch_size` 参数，将单次渲染的链接数量控制在 200 条以内；同时检查 `checker.py` 中的并发数设置，过高的并发可能导致 DNS 解析阻塞，建议保持默认的 5 个并发。

Q：部分外部链接包含中文或特殊字符，导致 URL 在页面中显示异常怎么办？

A：确保 `loader.py` 在读取 CSV 时指定 `encoding='utf-8-sig'`，并在 `renderer.py` 中使用 `urllib.parse.quote` 对路径部分进行安全编码。项目内置的 `default.yaml` 中已开启 `url_quote_safe` 选项，默认对非 ASCII 字符进行转义。

Q：如何让 ResourceBridge 同时管理多套独立的链接集合（例如区分测试环境与生产环境）？

A：利用 `--config` 参数指定不同的 YAML 配置文件启动实例，或通过环境变量 `RESOURCEBRIDGE_ENV` 切换配置片段。项目支持配置继承机制，可在 `config/` 目录下维护 `dev.yaml` 与 `prod.yaml` 分别覆盖 `default.yaml` 中的输出目录与探测策略。
