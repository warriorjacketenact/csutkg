# Tokyo Mirai Video Resource Indexer

Tokyo Mirai Video Resource Indexer 是一个面向技术内容创作者、教育工作者及多媒体归档从业者的开源视频资源外链汇总与元数据管理系统。该项目不对视频文件本身进行存储或分发，而是提供一套结构化的索引框架，用于整理、分类和检索来自东京未来网络服务（tokyo-mirai.net）的视频资源链接，帮助用户快速定位特定批次、主题或格式的外部视频内容。

项目定位为轻量级外链资源目录，适用于需要批量管理视频引用、构建自定义播放列表或进行视频源可用性监控的场景。通过统一的索引格式和自动化校验脚本，用户可降低链接散落带来的管理成本，并利用内置的标签系统与时间戳字段实现精细化筛选。目标用户包括独立视频博主、线上课程运营团队、研究机构多媒体资料管理员以及开源社区文档维护者。

## 功能概览

**结构化外链索引** 提供统一的表格化链接记录格式，每个条目包含视频标识符、原始URL、批次编号和收录时间戳，便于人工审阅与程序化处理。

**自动化可用性校验** 内置Python检测脚本，可定期对索引中的全部链接发起HEAD请求，标记失效或重定向的资源，并生成可用性报告。

**分类标签体系** 支持为每条视频链接附加多个自定义标签（如类别、语言、清晰度），并通过标签筛选接口快速提取子集。

**批次管理视图** 按项目批次（如第31/1241批）组织链接分组，展示每批的资源数量、收录日期和校验状态，方便分阶段导入。

**导出与集成接口** 提供JSON、CSV和Markdown三种导出格式，便于与其他媒体管理工具或静态站点生成器集成。

**变更日志追踪** 自动记录每次链接增删改操作的时间、操作者及变更摘要，满足归档审计需求。

## 应用场景

**在线课程视频素材整理** 教育机构可使用本项目索引外部教学视频源，按课程单元建立链接分组，并定期校验视频可访问性，避免课堂播放时出现失效链接。

**技术会议演讲归档** 开发者社区可将历次线上技术分享会的录播视频链接统一收录，附加演讲主题、讲师和日期标签，构建可检索的会议资源库。

**多媒体内容监控看板** 媒体监控团队可利用校验脚本定时扫描索引链接，生成可用性趋势图表，及时发现被下架或迁移的视频资源。

**开源文档外部引用管理** 开源项目维护者可将文档中引用的所有外部视频演示链接纳入索引，统一管理引用状态，在版本发布前执行全面链接检查。

## 快速开始

以下步骤指导您在本地环境中完成项目的克隆、依赖安装和基础运行。

```bash
# 克隆项目仓库
git clone https://github.com/example/tokyo-mirai-indexer.git
cd tokyo-mirai-indexer

# 安装所需依赖（建议使用虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 运行链接校验脚本，检查当前批次所有资源
python scripts/check_links.py --batch 31 --output report.html
```

## 常见问题

Q: 校验脚本报告链接超时，但手动访问浏览器中可正常打开，如何解决？

A: 此情况通常由服务器对User-Agent或请求头进行过滤导致。您可在config.yaml中调整校验器的请求头配置，将`User-Agent`模拟为常见浏览器标识，并适当增加`timeout`参数值（建议从3秒调整为10秒）。若仍失败，可使用`--skip-ssl`选项忽略证书验证。

Q: 如何将一个外部批次的所有链接合并到当前索引中？

A: 使用`scripts/merge_batch.py`脚本（需单独下载），指定外部批次JSON文件路径和目标批次号。合并前系统会自动检查链接去重和格式一致性，并生成合并预览报告，确认无误后执行正式合并。合并操作会保留原始批次的标签和备注字段。

Q: 标签体系是否有层级结构，能否支持多级分类？

A: 当前标签体系为扁平设计，不支持嵌套层级。但您可以使用分隔符（如下划线或连字符）在标签名称中模拟层级，例如`subject_math`、`subject_physics`，并通过标签前缀筛选实现类似分组效果。后续版本将考虑引入标签命名空间支持。
