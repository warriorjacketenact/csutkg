# LinkVault Resource Aggregator

LinkVault is a curated, machine-readable resource index and external link aggregation system designed for developers, researchers, and content curators who need to organize, validate, and distribute large volumes of structured URL data. Unlike traditional bookmarking tools, LinkVault treats links as first-class data entities with metadata, provenance tracking, and automated health checking.

This project targets technical teams that manage external reference collections, dependency mirror lists, documentation portals, or any system where link rot and resource discoverability are operational concerns. LinkVault provides the tooling to ingest, normalize, tag, and export link collections in multiple formats (JSON, YAML, CSV, and static HTML), enabling both human browsing and programmatic consumption. The current release includes the 679th batch of an ongoing 1,241-batch indexing pipeline, covering 250 thematic resources from the i-fights.net theater domain.

## 功能概览

**批量链接摄取** – Ingest large link lists from plain text, CSV, or JSON sources with automatic protocol normalization and duplicate detection.

**元数据富化管道** – Enrich each URL with HTTP status codes, content-type detection, response time metrics, and SSL certificate validity data via configurable middleware.

**标签与分类引擎** – Apply hierarchical tags, custom taxonomies, and free-text annotations to links, with support for regex-based auto-tagging rules.

**健康检查调度** – Run scheduled or on-demand link validity checks with configurable retry policies, timeout thresholds, and webhook notifications for broken links.

**多格式导出** – Export indexed collections as static HTML table, JSON Lines, YAML manifest, or Markdown reference list suitable for embedding in documentation sites.

**查询与过滤 API** – Expose a RESTful query interface supporting filter by tag, status code, domain, or last-checked timestamp, with pagination and field selection.

**导入/导出互操作性** – Import browser bookmarks (Netscape HTML), Pocket CSV exports, or Pinboard JSON; export to same formats for seamless migration.

**审计日志** – Track all link additions, edits, deletions, and check results with timestamped audit records for compliance and debugging.

## 应用场景

**技术文档库维护** – Documentation teams can use LinkVault to manage the external reference sections of API guides, tutorials, and SDK manuals. The health check feature automatically flags dead links in published docs, enabling proactive updates before users encounter 404 errors.

**学术研究参考管理** – Researchers compiling literature reviews or dataset catalogs can import large URL lists, annotate each with research notes, tag by topic or methodology, and export formatted reference sections for papers or data management plans.

**DevOps 监控面板集成** – SRE teams can feed LinkVault with the URLs of internal dashboards, log viewers, and monitoring tools. The health check API integrates with alerting systems to notify on-call engineers when a critical internal tool becomes unreachable.

**开源项目资源页生成** – Open-source maintainers can maintain a single source-of-truth link inventory and render it into the project's README, website, or wiki automatically via CI pipelines, ensuring consistency across all documentation surfaces.

**合规性资产登记** – Security and compliance officers can catalog all external services, APIs, and vendor portals used by an organization, with audit logs tracking when each link was last verified and by whom.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/your-org/linkvault.git
cd linkvault

# Install dependencies using Poetry (recommended) or pip
poetry install --no-dev
# Alternatively: pip install -e .

# Run the initial ingestion pipeline with the sample batch data
python -m linkvault ingest --source data/batch_679.csv --output data/ingested.json
python -m linkvault check --input data/ingested.json --output data/checked.json
python -m linkvault export --input data/checked.json --format markdown --output docs/links.md
```

For development setup with hot-reload and debug logging:

```bash
poetry install --with dev
poetry run pytest tests/
poetry run linkvault serve --host 127.0.0.1 --port 8080 --reload
```

| Poetry 1.4+ | 是 | Dependency management and packaging. Used for lockfile consistency. |
| aiohttp 3.9+ | 是 | Async HTTP client for concurrent link checking and fetching. |
| pydantic 2.5+ | 是 | Data validation and settings management for link schemas. |

| redis 5.0+ | 否 | Optional caching backend for check results to reduce duplicate requests. |

## 贡献指南

1.  Fork the repository and create a feature branch from `main` with a descriptive name such as `feature/add-yaml-exporter` or `fix/health-check-timeout`. Use conventional commit prefixes in your branch name.

2.  Install the development dependencies and pre-commit hooks by running `poetry install --with dev` and `pre-commit install`. This ensures code formatting (Black), linting (Ruff), and type checking (Mypy) run automatically before each commit.

3.  Write or adapt unit tests under `tests/unit/` for any new functionality or bug fixes. Ensure test coverage does not decrease by running `pytest --cov=src/linkvault/` and review the coverage report. Integration tests that make external HTTP requests should use the `@pytest.mark.integration` marker.

4.  Update the relevant documentation in the `docs/` folder. If you add a new CLI command, document its usage in `docs/user-guide/cli.md`. If you modify the API, regenerate the OpenAPI spec by running `python -m linkvault api generate-openapi > src/linkvault/api/openapi.yaml`.

5.  Submit a pull request against the `main` branch with a clear description of the changes, including the motivation, affected components, and any manual testing performed. Reference any related issues using the `#issue-number` syntax. All CI checks must pass before a maintainer will review the PR.

## 常见问题

**Q: 如何处理大量链接的健康检查而不被目标服务器封禁？**

A: LinkVault implements a configurable concurrency limiter (default: 10 concurrent workers) and a rate limiter that respects `Retry-After` headers. You can also set per-domain rate limits via the `--domain-rate` flag in the `check` command. For large batches, the system uses a randomized jitter between requests to avoid synchronized traffic spikes. Additionally, the Redis cache backend stores successful check results for a configurable TTL (default 24 hours), reducing repeated checks for unchanged URLs.

**Q: 如何从现有的浏览器书签或 Pocket 收藏迁移到 LinkVault？**

A: The `ingest` command supports multiple source formats. For browser bookmarks, use the "Export Bookmarks" feature in Chrome or Firefox to generate an HTML file, then run `linkvault ingest --source bookmarks.html --format netscape`. For Pocket, export your list as CSV from the Pocket settings page and use `--format pocket-csv`. LinkVault automatically maps the title, tags, and URL fields to its internal schema. After ingestion, run the `check` command to validate all imported links and tag any broken ones with `status:broken`.

**Q: 导出的 Markdown 列表能否直接用于项目的 README 或 Wiki？**

A: Yes. The `export` command with `--format markdown` produces a clean, ready-to-copy Markdown bullet list. By default, it includes the link URL and a status badge (active, broken, or untested) in brackets. You can customize the template by providing a Jinja2 template file via `--template custom.md.j2`. For CI/CD integration, use the `--no-metadata` flag to export only the bare URLs, which is suitable for automated README updates. The exported output is standard CommonMark and renders correctly on GitHub, GitLab, and most static site generators.
