# Resource Atlas

Resource Atlas 是一个面向技术研究与开发人员的结构化外链资源汇总系统。该项目不存储任何实际内容，而是通过对分散于互联网各处的优质技术文档、工具站点、数据源与交互式演示进行人工筛选与分类索引，帮助用户在高信息密度场景下快速定位所需的外部资源。

该项目适用于需要频繁查阅外部技术资料的研究者、架构师、运维工程师以及开源贡献者。通过统一的索引视图与标准化的外链呈现格式，Resource Atlas 将无序的浏览行为转化为可复用的知识检索路径，有效降低信息过载带来的认知负担。

## 功能概览

**多源资源聚合**：对来自不同域名与路径结构的外部链接进行统一收录，保留原始 URL 完整信息，确保资源可追溯性与访问准确性。

**分类索引体系**：根据资源所属领域、内容类型与目标受众建立多级分类目录，支持按技术栈、应用场景与成熟度进行筛选。

**快速启动模板**：提供标准化的项目初始化流程，包含克隆、依赖安装与本地运行三个步骤，便于用户快速部署私有化实例。

**可扩展数据结构**：基于 JSON Schema 定义的资源条目格式，允许用户自行添加新链接或更新已有条目，无需修改核心代码。

**轻量级静态生成**：采用静态站点生成方案，构建输出为纯 HTML 与 CSS 文件，可部署于任何支持静态托管的服务环境。

**全文检索支持**：内置基于标题与标签的简单检索功能，帮助用户在本地镜像中快速定位特定资源条目。

## 应用场景

**技术选型调研**：当团队需要评估不同中间件、数据库或前端框架时，可通过 Resource Atlas 快速访问官方文档、性能测试报告与社区讨论帖，集中获取多维度参考信息。

**离线文档镜像**：在网络受限的内网开发环境中，用户可将 Resource Atlas 部署为内部知识门户，统一索引已下载或允许访问的外部技术文档地址。

**开源项目维护**：开源项目的维护者可以使用 Resource Atlas 管理项目依赖的相关资源链接，包括规范文档、API 参考与示例代码仓库，便于贡献者查阅。

**学习路径规划**：初学者可按照分类索引顺序浏览资源，从基础教程逐步过渡到高级实践指南，形成系统化的知识获取路径。

## 快速开始

以下步骤适用于 Linux 与 macOS 开发环境。Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resource-atlas/resource-atlas.git
cd resource-atlas

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

执行完上述命令后，访问控制台输出的本地地址（默认为 http://localhost:3000）即可查看资源索引页面。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js | 是 | 版本 18.0 或更高，推荐使用 LTS 版本 |
| npm | 是 | 版本 9.0 或更高，随 Node.js 一并安装 |
| Git | 是 | 用于克隆仓库与版本管理 |
| 现代浏览器 | 是 | 支持 ES2020 与 CSS Grid 的 Chrome、Firefox、Edge 或 Safari |
| 网络连接 | 是 | 初次构建需访问外部 CDN 资源，后续可离线运行 |
| 磁盘空间 | 否 | 建议至少 200 MB 用于存储索引文件与构建产物 |
| 内存 | 否 | 建议 4 GB 以上，用于支持较大规模资源列表的构建操作 |

## 贡献指南

1. 复刻项目仓库至个人账户，并创建新的功能分支。分支命名建议采用 `feature/描述` 或 `fix/描述` 格式。

2. 在 `src/data/sources.json` 中按既定格式添加新资源条目，确保每个条目包含完整的原始 URL、标题、分类标签与简短描述。所有新增条目须通过 `npm run validate` 命令的格式校验。

3. 提交代码前执行 `npm run test` 确保现有测试用例全部通过。若新增功能涉及逻辑变更，需同步编写对应的单元测试。

4. 提交合并请求时，在描述中明确说明本次变更的目的、涉及的文件范围以及是否影响现有功能。至少一位项目维护者审查通过后即可合并。

5. 若涉及对外部资源内容的分类调整或标签重命名，需同步更新 `categories.json` 与 `tags.json`，并确保所有已有条目的引用保持一致。

## 常见问题

Q: 项目是否会自动抓取外部资源的内容或建立本地缓存？

A: 不会。Resource Atlas 仅存储外部资源的 URL 与元数据描述，不抓取、不缓存、不代理任何外部内容。用户点击链接后直接访问原始站点，所有内容的可用性与合法性由原始提供方负责。

Q: 如何批量导入大量外部链接？

A: 支持通过 JSON 或 CSV 格式的批量导入脚本。将待导入链接按 `url`、`title`、`category`、`tags` 字段组织后放入 `data/import/` 目录，执行 `npm run import` 命令即可自动合并至主索引。导入前建议先使用 `--dry-run` 参数进行预览。

Q: 构建后的静态页面是否支持完全离线访问？

A: 构建产物（`dist/` 目录）中的所有 HTML、CSS 与 JavaScript 文件均可离线打开，不依赖外部 CDN 或网络请求。但资源列表中的外部链接本身需要网络环境才能访问，这属于原始资源的访问限制，与项目本身无关。
