# LinkVault Resource Aggregator

LinkVault 是一个面向技术内容创作者、知识库维护者和数字资源管理者的外链聚合与分类导航系统。该项目旨在解决分散在多个平台的技术文章、视频教程、文档站点和工具资源难以统一收录、检索和共享的问题，提供一套标准化的外链收录、分类标注和快速检索方案。

目标用户包括开源社区维护者、技术博客作者、在线教育平台运营者以及企业知识管理团队。LinkVault 不生产内容，而是提供内容入口的结构化管理能力，帮助用户从海量信息中快速定位高价值技术资源。

## 功能概览

- **多层级分类目录**：支持按技术领域、内容类型、适用人群等维度建立多级分类树，每条外链可归属多个分类节点。

- **批量导入与去重检测**：支持从 CSV、JSON 和 Markdown 列表批量导入 URL，自动检测重复条目并合并标签与备注。

- **全文检索与标签过滤**：基于倒排索引实现标题、描述、标签和备注的全文搜索，支持多标签组合过滤与排序。

- **外链健康检查**：定时对已收录的 URL 发起 HEAD 请求，检测响应状态码和重定向链，标记失效或变更的链接。

- **访问统计与热度排行**：记录每条外链的被点击次数、最近访问时间和来源页面，生成日/周/月热度排行。

- **自定义元数据扩展**：允许为每条外链添加键值对形式的自定义字段，如难度等级、所需前置知识、时长估算等。

- **RESTful API 与 Webhook**：提供完整的 JSON API 用于增删改查操作，支持通过 Webhook 与其他系统集成，实现自动同步。

- **数据导出与备份**：支持将全量外链数据导出为 JSON、CSV 或静态 HTML 页面，便于离线查阅或迁移。

## 应用场景

**技术博客的参考链接库**：技术博主在撰写文章时常常引用大量外部资料，使用 LinkVault 可集中管理这些参考资料，为每篇文章生成独立的引用清单，并在文章底部自动展示相关外链卡片。

**开源项目的资源导航页**：开源项目维护者可使用 LinkVault 搭建项目官网的「生态系统」或「相关资源」页面，分类展示社区教程、视频讲解、第三方工具和插件，降低新贡献者的上手门槛。

**企业内部技术周报汇总**：企业技术团队每周会产出大量内部文档、设计稿和会议录屏，LinkVault 可作为内部知识库的入口聚合层，将分散在 Confluence、语雀、飞书文档和网盘中的链接按项目和阶段归集。

**在线课程的外部阅读清单**：在线教育平台可以为每门课程配置扩展阅读清单，将 YouTube 视频、技术文档、学术论文和练习平台的链接统一收录，学员可在课程页面一键访问所有外部材料。

**社区活动资源归档**：技术社区举办黑客松、工作坊或线上分享会后，可将活动相关的报名表、直播回放、代码仓库和幻灯片链接通过 LinkVault 进行归档，生成可公开访问的活动资源合集。

## 快速开始

以下步骤适用于 Linux / macOS 系统，Windows 用户建议使用 WSL2 或 Git Bash 环境。

```bash
# 1. 克隆代码仓库
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# 2. 安装依赖（使用 Python 3.10+ 和 pip）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. 初始化数据库并导入示例数据
python manage.py migrate
python manage.py loaddata fixtures/initial_links.json

# 4. 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8080
```

访问 http://localhost:8080 即可进入管理控制台。默认管理员账户为 admin / admin123，首次登录后请立即修改密码。

生产环境部署请参考 `docs/deployment.md`，推荐使用 Gunicorn + Nginx 组合。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.10 ～ 3.12 | 核心运行环境，低于 3.10 将无法使用 match 语法和类型注解新特性 |
| PostgreSQL | 14.x ～ 16.x | 主数据库，用于存储外链元数据、分类树和访问日志，建议开启 pg_trgm 扩展以支持模糊检索 |
| Redis | 7.x | 缓存与任务队列后端，用于外链健康检查任务的异步调度和热点数据缓存 |
| Node.js | 18.x ～ 20.x | 仅用于前端静态资源构建（Tailwind CSS + Alpine.js），后端运行时无需 Node |
| Nginx | 1.24+ | 生产环境推荐的反向代理服务器，用于负载均衡、静态文件服务和 HTTPS 终止 |
| Docker Compose | 2.20+ | 可选依赖，用于一键启动全套开发环境（含数据库和缓存容器） |

## 项目结构

```

├── src/
│   ├── core/                         # 核心业务逻辑模块
│   │   ├── aggregator.py             # 外链聚合引擎，处理批量导入和去重
│   │   ├── classifier.py             # 分类树管理，支持多级标签体系
│   │   ├── health.py                 # 健康检查调度器，异步检测URL状态
│   │   └── stats.py                  # 访问统计与热度计算
│   ├── api/                          # RESTful API 路由和序列化器
│   │   ├── v1/                       # API v1 版本端点
│   │   │   ├── links.py              # 外链CRUD接口
```

## 常见问题

**Q: LinkVault 是否支持在线预览或代理被收录的外部链接内容？**

A: 不支持。LinkVault 仅存储外链的元数据（标题、描述、标签、分类），不会缓存或代理任何第三方内容。用户点击外链时直接跳转至原始 URL，所有内容的版权和可用性由源站点负责。建议在收录链接时添加访问提示，告知用户即将离开当前站点。

**Q: 健康检查功能如何处理需要登录或带有访问限制的链接？**

A: 健康检查默认仅发送基础 HEAD 请求，不携带 Cookie 或认证头。对于需要登录才能访问的链接，系统会将其标记为「受限」状态并跳过主动检查，同时记录上次手动验证的时间戳。管理员可在后台手动更新受限链接的状态，或配置自定义请求头白名单以覆盖特定域名。

**Q: 如何将 LinkVault 与现有的企业 SSO 系统集成？**

A: LinkVault 提供可插拔的认证后端，默认支持 LDAP 和 OAuth2 协议。管理员可在 `config/production.toml` 中配置 `auth.backend` 参数，并按照 `docs/admin/sso-integration.md` 中的指引填写 issuer URL、client_id 和 client_secret。目前已验证兼容 Keycloak、Okta 和 Azure AD。
