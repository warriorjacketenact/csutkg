# ResourceBridge

ResourceBridge 是一个面向技术团队与个人开发者的外链资源归集与分发平台，专注于解决分散在多个来源的技术文档、工具链、依赖镜像、API 参考、社区讨论帖等外部链接的统一管理问题。项目本身不存储任何实际内容，而是通过结构化的链接索引与分类机制，帮助用户快速定位所需资源，减少在书签、聊天记录、邮件与临时笔记中反复搜索的损耗。

项目定位为轻量级静态资源导航系统，适用于内部技术中台的知识库补充、开源项目的外部依赖清单托管、以及个人学习路线的链接收藏与管理。ResourceBridge 不依赖数据库，所有链接数据以纯文本与 YAML Front Matter 形式存储在仓库中，支持 CI/CD 自动化校验链接可用性，并可无缝集成到现有的文档站点生成工具链中。

## 功能概览

- **多级分类索引**：支持按技术领域、项目阶段、资源类型建立三层分类索引，每一条链接可归属多个标签体系，满足不同视角的检索需求。

- **链接可用性自动化检查**：集成基于 GitHub Actions 的定时任务，每日扫描全部已收录链接，自动标记状态码异常或响应超时的条目，并生成报告。

- **快速全文检索**：基于静态索引文件实现毫秒级的关键词检索，支持标题、描述、标签与路径片段的模糊匹配，无需额外搜索引擎服务。

- **版本化快照记录**：每次链接增删改操作均通过 Pull Request 合并，保留完整的变更历史与审核记录，支持回溯至任意历史版本的资源列表。

- **个性化视图切换**：提供按热度、更新时间、字母序等排序方式，并支持仅显示已验证可用链接的过滤模式，便于快速筛选高价值资源。

- **结构化元数据扩展**：每条链接可附加负责人、所属项目、维护状态、替代链接等自定义字段，满足企业级资源治理需求。

- **OpenAPI 兼容的只读接口**：提供符合 OpenAPI 3.0 规范的 JSON 数据导出端点，允许其他工具直接拉取完整链接列表进行二次开发或集成。

## 应用场景

- **技术团队内部知识库导航**：将团队常用的依赖镜像源、内部文档入口、部署监控面板、日志查询工具等链接统一收归至 ResourceBridge，新成员入职时可直接通过导航页快速熟悉基础设施入口。

- **开源项目的外部资源清单**：开源项目的 README 中常需引用大量外部参考链接，使用 ResourceBridge 可独立维护该清单，主项目仅需引用一个稳定入口，避免多处分发导致链接不一致。

- **技术学习路径规划与分享**：学习者可将自己收集的教程、API 手册、最佳实践文章、视频课程等链接按主题整理为多个 ResourceBridge 实例，通过仓库 Fork 机制与他人共享学习路线。

- **离线文档站点的补充索引**：对于部署在内部网络且无法访问外网的文档站点，ResourceBridge 可作为前置导航层，将所有可访问的外部参考链接集中展示，避免用户在多个窗口间切换。

## 快速开始

以下步骤可在 5 分钟内完成本地环境的搭建与运行。

```bash
# 克隆仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装依赖（项目基于 Node.js 18+ 与 pnpm）
pnpm install

# 启动开发服务器，默认监听 3000 端口
pnpm run dev
```

访问 `http://localhost:3000` 即可查看本地导航页面。如需构建生产环境静态文件，执行 `pnpm run build`，产物输出至 `dist` 目录。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 Active LTS 版本 |
| pnpm | 8.x 或 9.x | 包管理器，用于依赖安装与脚本执行 |
| Git | 2.25 以上 | 版本控制工具，用于克隆与提交变更 |
| YAML 解析器 | 项目内置 | 用于解析链接元数据文件，无需额外安装 |
| HTTP 客户端 | 项目内置 | 用于链接可用性检查，基于 undici 实现 |
| 静态站点生成器 | 项目内置 | 用于构建最终 HTML 页面，基于 Astro 框架 |
| 代码编辑器 | 可选 | 推荐 VS Code 配合 YAML 插件以获得更好的编辑体验 |

## 项目结构

项目采用模块化组织方式，核心逻辑与资源数据分离，便于维护与扩展。

```
resourcebridge/
├── src/                           # 源代码主目录
│   ├── core/                      # 核心逻辑模块，包括链接解析、校验与缓存
│   │   ├── parser/                # YAML 与 Markdown 前置数据解析器
│   │   └── validator/             # HTTP 状态码与响应时间校验器
│   ├── routes/                    # 路由层，定义接口与页面路由映射
│   │   ├── api/                   # OpenAPI 只读接口实现
│   │   └── pages/                 # 静态页面生成路由
│   ├── components/                # UI 组件库，包括搜索框、分类树、链接卡片
│   │   ├── filters/               # 排序与过滤组件
│   │   └── layout/                # 页面框架与导航组件
│   ├── utils/                     # 工具函数集，包括日志、配置读取与错误处理
│   └── styles/                    # 全局样式与主题变量，基于 CSS 自定义属性
├── data/                          # 资源数据存储目录（全部为 YAML 文件）
│   ├── links/                     # 按分类存放链接定义文件
│   │   ├── infrastructure/        # 基础设施类链接定义
│   │   ├── development/           # 开发工具与库类链接定义
│   │   └── operations/            # 运维监控类链接定义
│   └── taxonomy/                  # 分类体系定义文件
├── tests/                         # 单元测试与集成测试用例
│   ├── unit/                      # 单模块测试
│   └── integration/               # 端到端链接检查测试
├── scripts/                       # 辅助脚本，包括链接批量导入与导出工具
```

## 贡献指南

ResourceBridge 欢迎并鼓励外部贡献，无论是新增链接、修复文档错误还是改进核心功能，均按照以下流程进行。

1. 在 GitHub 上 Fork 本仓库，并在本地克隆 Fork 后的副本。创建新分支时请以 `feature/` 或 `fix/` 作为前缀，并简要描述变更内容。

2. 若为新增或修改链接，请在 `data/links/` 下对应的分类目录中编辑 YAML 文件，每条链接需包含 `title`、`url`、`description`、`tags` 四个必填字段。提交前请运行 `pnpm run validate` 校验数据格式。

3. 若为代码变更，请确保所有现有测试用例通过，并为新增功能添加对应的单元测试。测试覆盖率不应低于当前主干分支的水平。

4. 提交 Pull Request 时请填写提供的模板，清晰描述变更目的、影响范围以及是否涉及破坏性改动。PR 标题请遵循 Conventional Commits 规范。

5. 项目维护者将在 3 个工作日内完成 Review。如有修改意见，请及时响应并在原分支上追加提交，无需关闭 PR 后重新开启。

## 常见问题

**问：ResourceBridge 是否必须部署在公开网络才能使用？**

答：不是。ResourceBridge 完全基于静态文件生成，可以部署在任何 HTTP 服务器上，包括本地文件协议、内网服务器或对象存储服务。所有链接检查功能仅在 CI 阶段执行，运行时无需外网访问权限。

**问：链接可用性检查会误报吗？**

答：检查器采用三次重试机制，每次重试间隔 2 秒，仅当三次请求均返回非 2xx/3xx 状态码或超时才会标记为异常。对于已知会进行反爬或限流的站点，可在 `config/validator.config.js` 中单独配置忽略规则或自定义请求头。

**问：能否批量导入现有书签文件？**

答：项目 `scripts/` 目录下提供了 `import-from-html.js` 脚本，支持将 Chrome 或 Firefox 导出的书签 HTML 文件转换为 ResourceBridge 的 YAML 格式。执行 `pnpm run import -- --file=bookmarks.html` 即可生成对应的数据文件，后续可手动调整分类归属。
