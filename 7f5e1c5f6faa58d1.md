# VODLink Hub

VODLink Hub 是一个面向技术内容聚合与视频点播资源管理的开源导航工具。项目定位于帮助开发者、内容研究者以及数据归档人员高效整理、检索和引用分布式的视频详情页资源。该项目不提供视频播放或下载功能，仅作为 URL 资源的结构化组织与展示层，解决大量无序链接的归类、版本追踪与快速访问问题。目标用户包括运维工程师、数据标注团队、爬虫开发者以及需要批量管理外链资源的站点管理员。

## 功能概览

**批量链接导入**：支持从 CSV、JSON 或纯文本列表批量导入形如 hakujutakayama.com 域下的 VOD 详情链接，自动解析资源 ID 与类型标签。

**多级分类与标签**：允许用户为每个链接自定义分类（如纪录片、教程、影视）和搜索标签，支持层级标签体系，便于多维度筛选。

**链接状态监控**：内置轻量级 HTTP 状态检查器，定期探测链接可达性，标记失效或重定向链接，并生成可用性报告。

**全文检索与过滤**：基于资源 ID、URL 路径数字段、导入时间、状态码等元数据构建倒排索引，支持毫秒级检索与组合条件过滤。

**数据导出与分享**：可将当前筛选结果导出为 Markdown 表格、HTML 目录或纯文本列表，便于嵌入技术文档或邮件分享。

**自定义视图模板**：提供三种展示视图（简洁列表、卡片网格、层级树），用户可根据使用习惯切换，并保存为个人偏好。

**RESTful API 接口**：提供完整的 JSON API，支持第三方工具通过 Token 认证进行链接增删改查、状态批量更新及元数据同步。

## 应用场景

**技术文档外链管理**：技术团队在编写项目文档或知识库时，需要引用大量外部参考资料。VODLink Hub 可作为中间层，统一存储、分类和校验这些外链，避免文档中出现死链或重复链接。

**数据标注与清洗**：数据标注团队在整理视频数据集时，需要频繁访问不同 VOD 详情页获取元数据。项目提供批量 URL 导入和快速跳转功能，显著提升标注流程中的信息检索效率。

**爬虫任务调度辅助**：爬虫开发者可利用项目的链接状态监控模块，定期验证目标 VOD 页面的可访问性，辅助判断反爬策略或站点维护周期，减少无效请求。

**个人书签库构建**：个人用户可将分散在浏览器收藏夹中的技术视频链接迁移至 VODLink Hub，通过标签和全文检索实现跨设备的统一访问与管理。

## 快速开始

以下步骤帮助您在本地环境快速启动 VODLink Hub 服务。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/vodlink-hub.git
cd vodlink-hub

# 安装项目依赖（使用 pip 与 requirements.txt）
pip install -r requirements.txt

# 初始化 SQLite 数据库并导入示例链接数据
python manage.py initdb
python manage.py load-sample-links

# 启动开发服务器，默认监听 127.0.0.1:5000
python manage.py runserver
```

启动后，在浏览器中访问 `http://127.0.0.1:5000` 即可进入 VODLink Hub 主界面。管理员默认账号为 `admin`，密码为 `admin123`，首次登录后请及时修改。

## 贡献指南

我们欢迎社区贡献者以多种形式参与项目改进。请遵循以下步骤提交您的贡献。

1. 查阅问题追踪列表：访问 GitHub Issues 页面，查找未被认领的 bug 报告或功能请求。对于较大的功能改动，建议先创建一个讨论议题，与维护者沟通设计思路。

2. 派生仓库并创建特性分支：将主仓库派生至个人账户，然后克隆本地。请基于 `main` 分支创建新的特性分支，分支命名建议采用 `feature/描述` 或 `fix/描述` 格式。

3. 编写代码与测试：所有新功能或修复必须附带对应的单元测试或集成测试，确保测试通过（`pytest tests/`）。代码风格应严格遵循 PEP 8，并使用 `flake8` 或 `black` 进行格式检查。

4. 更新相关文档：若您的改动涉及用户可见功能、API 变更或新增配置项，请同步更新 `docs/` 目录下的相应文档，并在 `README.md` 的「功能概览」或「安装要求」中补充说明。

5. 提交拉取请求：推送分支至派生仓库后，向主仓库的 `main` 分支提交 Pull Request。PR 标题应简明扼要，内容需描述改动背景、实现方案及测试结果。维护者将在 3 个工作日内进行审阅。

## 常见问题

**Q：项目是否支持导入 HTTPS 与 HTTP 混合列表？**

A：支持。VODLink Hub 对链接协议不做限制，无论是 http:// 还是 https:// 开头的 URL 均可正常导入。但健康检查模块会严格遵循重定向链，若最终目标协议与记录不一致，将在状态字段中标记为 "redirect"。本项目对外部站点的内容可用性不承担责任，建议用户定期运行健康检查以更新链接状态。

**Q：如何将现有的浏览器书签批量迁移到 VODLink Hub？**

A：主流浏览器（Chrome、Firefox、Edge）支持将书签导出为 HTML 文件。您可以先使用浏览器导出书签，然后通过项目提供的 `scripts/import_bookmarks.py` 辅助脚本（需自行适配书签 HTML 结构）转换成项目可识别的 JSON 格式，再调用 `manage.py import --format json --file bookmarks.json` 完成导入。未来版本计划内置书签 HTML 的直接解析器。

**Q：项目遇到性能瓶颈时如何横向扩展？**

A：VODLink Hub 默认使用 SQLite 作为数据库，适合小型部署（链接数 < 5 万）。对于更大规模的数据集，建议将 `SQLALCHEMY_DATABASE_URI` 切换至 PostgreSQL 或 MySQL。此外，健康检查模块默认采用单线程顺序探测，可调整 `app/services/health_checker.py` 中的 `MAX_WORKERS` 参数启用多线程并发检查，显著提升大规模链接的扫描速度。
