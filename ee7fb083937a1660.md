# ResourceLink Hub

ResourceLink Hub 是一个面向开发者、技术研究人员与内容创作者的轻量级技术资源聚合与导航平台。该项目并非传统意义上的爬虫或采集系统，而是一套高度结构化的外链资源组织框架，旨在帮助用户从海量分散的网络文档、学习素材与工具站点中，快速定位高价值技术参考信息。

项目核心定位为“技术外链的标准化收录与场景化索引”。它不生产内容，而是通过严谨的分类体系与清晰的文档导航，将原始 URL 资源转化为可检索、可追溯、可理解的知识条目。目标用户包括需要持续跟踪技术动态的软件工程师、开展竞品分析的产品经理、撰写技术博客的开发者，以及进行在线教育素材编排的培训人员。ResourceLink Hub 通过统一的条目元数据描述、状态标记与关联分类，解决了个人收藏夹杂乱无章、团队协作时资源共享效率低下、以及外部链接失效后难以快速定位替代来源等实际问题。

## 功能概览

**资源条目标准化录入**：每个外链资源均需经过标题提取、来源域校验、内容类型预判与主题标签生成四个步骤，确保条目具备完整的元数据上下文，而非单纯存储 URL 字符串。

**多维度分类索引**：支持按技术领域、资源形态、适用场景与内容成熟度四个维度对资源进行交叉分类，同一资源可归属于多个分类视图，满足不同用户的检索习惯。

**链接状态健康检查**：内置周期性链接可达性检测模块，可标记失效链接、临时重定向链接与内容变更链接，辅助管理员及时维护资源库的有效性。

**场景化资源包导出**：允许用户根据特定技术栈或学习路径，筛选相关资源条目并导出为结构化文档、Markdown 清单或 JSON 格式数据，便于嵌入其他工作流。

**文档导航与搜索增强**：提供基于标题关键词、来源域名与分类标签的模糊搜索能力，同时支持按收录时间、访问频次与质量评分进行排序。

**外部资源关联推荐**：基于条目之间的域名关联性、主题重合度与用户协同筛选行为，自动生成“关联资源”推荐列表，降低用户在多个资源间跳转的认知负担。

**访问统计与热度分析**：记录每个外链条目的被查阅次数与导出频次，生成简单的热度趋势数据，帮助团队识别当前关注度较高的技术方向。

## 应用场景

**技术团队内部知识库建设**：开发团队可将日常调研中发现的优质技术博客、开源项目文档、在线 API 参考与性能测试工具统一收录至 ResourceLink Hub，并按照业务模块或技术栈分类。新成员入职时，可通过查阅分类索引快速了解团队常用的外部技术资源分布，缩短上手周期。

**技术博客与教程写作辅助**：技术内容创作者在撰写系列教程或深度解析文章时，需要引用大量外部资料作为论据支撑或延伸阅读材料。使用 ResourceLink Hub 可预先将候选参考资料全部录入系统，通过标签与分类功能进行分组整理，写作过程中直接从平台复制标准化链接，避免临时搜索导致的上下文中断。

**在线教育课程资源编排**：培训讲师或 MOOC 课程设计者为学员准备补充阅读清单时，可将推荐的文档、视频链接、交互式演示站点与练习平台统一纳入 ResourceLink Hub 的课程专属分类下。学员通过平台获得一致的资源访问入口，讲师亦可定期检查链接有效性，及时替换失效内容，保障教学体验的连续性。

**技术调研与竞品分析**：产品经理与技术负责人进行市场调研时，需要同时关注多个竞品的技术文档、社区讨论、版本发布日志与性能基准报告。ResourceLink Hub 支持为每个调研项目创建临时分类，集中存放相关外链，并结合热度统计功能识别近期讨论较为活跃的技术方向，为决策提供参考依据。

## 快速开始

以下步骤指导用户在本地环境中快速部署 ResourceLink Hub 基础服务。

```
git clone https://github.com/resourcelink-hub/resourcelink-hub.git
cd resourcelink-hub
pip install -r requirements.txt
python manage.py runserver --host=0.0.0.0 --port=8080
```

上述命令执行完毕后，访问本地 8080 端口即可进入 ResourceLink Hub 的 Web 管理界面。首次启动将自动创建 SQLite 数据库文件并生成管理员账户，初始用户名与密码请查阅部署目录下的 README 内附说明。

| Python | 3.9 及以上 | 核心运行环境，用于后端服务与数据管理脚本 |
| SQLite | 3.35.0 及以上 | 默认嵌入式数据库，存储资源条目元数据与分类关系 |

| Git | 2.30 及以上 | 用于版本管理与插件扩展安装 |
| curl | 7.68 及以上 | 系统工具，用于外部链接可达性检测脚本的网络请求 |
| jq | 1.6 及以上 | 命令行 JSON 处理工具，用于解析检测结果 |

## 项目结构

```
resourcelink-hub/
├── app/                                 # 主应用核心模块
│   ├── api/                             # RESTful API 路由与视图函数
│   │   ├── endpoints/                   # 按资源类型划分的接口端点
│   │   └── serializers/                 # 数据序列化与反序列化逻辑
│   ├── core/                            # 核心业务逻辑层
│   │   ├── classification/              # 分类索引引擎实现
│   │   ├── health_check/                # 链接健康检查调度器与执行器
│   │   └── statistics/                  # 访问统计与热度计算模块
│   ├── models/                          # 数据模型定义
│   │   ├── resource.py                  # 资源条目主模型
│   │   ├── category.py                  # 分类与标签模型
│   │   └── audit_log.py                 # 操作审计日志模型
│   └── utils/                           # 通用工具函数库
│       ├── http_client.py               # 自定义 HTTP 请求客户端
│       ├── url_parser.py                # URL 解析与规范化辅助
│       └── validators.py                # 输入数据校验器
├── config/                              # 项目配置文件目录
│   ├── settings.py                      # 主配置项（数据库、缓存、任务队列）
│   ├── logging.conf                     # 日志级别与输出格式配置
│   └── resource_mappings.yaml           # 外部资源域名到分类的映射规则
├── frontend/                            # 前端静态资源与模板
```

## 常见问题

问：ResourceLink Hub 是否会自动抓取外部链接的实际内容或存储副本？

答：不会。ResourceLink Hub 仅存储用户提交的 URL 地址及其元数据描述，不抓取、不缓存、不代理外部链接的实际内容。链接健康检查模块仅发送 HTTP HEAD 请求验证可达性，不下载响应体。所有外部资源的访问均需用户通过本地网络环境直接发起，项目本身不涉及任何内容搬运或存储行为。

问：添加大量外部链接后，链接健康检查任务是否会影响主服务性能？

答：健康检查任务默认采用异步队列方式执行，主服务进程不会因检查任务而阻塞。Redis 队列配合独立的工作进程处理检查请求，单个链接的超时时间设置为 5 秒，且检查结果写入独立的状态表，不影响资源检索主查询的性能。建议在生产环境中将检查任务调度至非高峰时段执行，例如每日凌晨 2 点至 5 点。

问：如何批量更新已有资源条目的分类标签？

答：项目提供了数据导入导出脚本，推荐使用 scripts/export_bundle.py 将需要更新的条目导出为 CSV 格式，在本地使用电子表格软件编辑分类字段后，通过 scripts/import_links.py 并指定 --update-existing 参数执行覆盖更新。注意该操作会依据条目 ID 进行匹配，请确保导出时保留原始 ID 列。若需完全重新分类，亦可直接操作 SQLite 数据库中的 category_mapping 表，但建议先备份数据库文件。
