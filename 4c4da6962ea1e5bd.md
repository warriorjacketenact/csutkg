# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的轻量级外链资源聚合与分发系统。该项目定位为技术资源导航站的后端数据核心，专注于解决大规模外链 URL 的批量收录、分类存储、状态监控与结构化输出问题。目标用户包括开源文档维护者、技术社区运营人员、DevOps 工程师以及需要管理大量外部参考链接的研发团队。

LinkVault Core 不提供前端界面，而是通过 RESTful API 与命令行工具暴露数据操作能力，使得用户能够以编程方式对数百乃至数千个外部链接执行批量导入、标签管理、可用性检测与 Markdown 格式导出。本项目特别适用于构建技术手册的附录链接库、项目文档的外部引用索引，以及自动化周报中的资源汇总模块。

## 功能概览

- **批量链接收录**：支持从 CSV、JSON 或纯文本列表中一次性导入大量 URL，自动解析域名与路径参数。
- **结构化分类存储**：基于链接路径模式（如 /voddetail/）与数字 ID 区间，自动归类至不同资源分组。
- **去重与规范化校验**：内置 URL 标准化流程，自动去除冗余查询参数，检测并标记重复条目。
- **Markdown 清单生成**：将存储的链接按用户指定格式（有序列表、无序列表、分类标题）输出为纯 Markdown 文本。
- **可用性状态检查**：支持配置超时阈值，对收录链接发起 HEAD 请求，标记异常状态码或超时记录。
- **标签与备注关联**：允许为每条链接附加一个或多个标签（如 "video"、"backend"、"tutorial"）以及简短备注。
- **增量更新与变更日志**：记录每次批量操作的新增、删除与修改动作，支持回滚至历史版本。

## 应用场景

- **技术文档附录管理**：开源项目维护者在编写用户手册或 API 参考文档时，可使用 LinkVault Core 统一管理所有外部参考链接，并一键生成符合规范的附录章节，避免手动维护带来的遗漏或格式错误。
- **社区资源周报自动化**：社区运营人员每周收集数十个与项目相关的讨论帖、视频教程或第三方工具链接，通过 LinkVault Core 的批量导入与分类功能，快速生成结构清晰的周报资源清单，提升内容发布效率。
- **内部知识库外链审计**：企业技术团队可利用本项目的可用性检查功能，定期扫描内部 Wiki 或 Confluence 中引用的外部链接，及时发现失效资源并生成审计报告，保障知识库质量。
- **课程材料汇编**：教育机构讲师在准备编程课程讲义时，可将大量课后阅读材料、在线演示或环境配置参考链接集中托管于 LinkVault Core，按课时或章节分类导出，便于学生查阅。
- **爬虫数据初步清洗**：数据采集工程师在完成一轮链接抓取后，可将原始 URL 列表输入 LinkVault Core，利用其去重与规范化功能进行预处理，再输出干净的数据集用于后续分析。

## 快速开始

以下操作演示如何在本地环境中获取 LinkVault Core 源码、安装依赖并启动核心服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/core.git linkvault-core
cd linkvault-core

# 安装依赖（使用 Python 虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 系统请使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化本地数据库
python manage.py initdb --config config/dev.yaml

# 启动命令行交互式导入工具（示例：导入当前目录下的 links.txt）
python cli.py import --file ./examples/links.txt --group default

# 生成 Markdown 资源列表（输出至控制台）
python cli.py export --format markdown --output ./output/resource_list.md
```

## 项目结构

```

├── manage.py                  # 项目管理脚本（初始化、迁移、维护）
├── requirements.txt           # 生产环境依赖列表
├── config/
│   ├── dev.yaml              # 开发环境配置（调试级别日志、本地 SQLite）
│   ├── prod.yaml             # 生产环境配置（日志级别、数据库连接池）
│   └── schema.yaml           # 链接元数据字段映射定义
├── core/
│   ├── __init__.py
│   ├── importer.py           # 批量导入模块（CSV/JSON/TXT 解析器）
│   ├── exporter.py           # 导出模块（Markdown/JSON/CSV 生成器）
│   ├── validator.py          # URL 规范化与去重校验
│   ├── health_check.py       # 可用性检测（并发 HEAD 请求）
│   └── tag_engine.py         # 标签管理与自动分类规则
├── storage/
│   ├── __init__.py
│   ├── database.py           # SQLite 连接池与 ORM 基础映射
│   ├── migrations/           # 数据库迁移脚本（版本化 DDL）
│   │   ├── 001_initial.sql
│   │   └── 002_add_indexes.sql
│   └── repositories/         # 数据访问层（链接库、标签库、分组库）
│       ├── link_repo.py
│       └── tag_repo.py
├── api/
│   ├── __init__.py
│   ├── app.py                # Flask 应用工厂
│   ├── routes/               # RESTful 端点路由
│   │   ├── links.py
│   │   └── groups.py
│   └── middleware/           # 请求日志、异常处理、认证中间件
├── tests/
│   ├── unit/                 # 单元测试（针对各模块）
│   ├── integration/          # 集成测试（API 与数据库交互）
```

## 常见问题

**Q：LinkVault Core 是否支持 MySQL 或 PostgreSQL 作为后端数据库？**

当前稳定版本仅内置 SQLite 支持，但项目已在 storage/database.py 中抽象了数据库连接接口。若需使用外部数据库，可参考 docs/deployment.md 中的扩展指南，通过修改配置文件与引入对应驱动来实现。社区贡献的 PostgreSQL 适配器正在开发中。

**Q：导入大量 URL 时内存占用过高怎么办？**

对于超过 10000 条记录的批量导入，建议使用 `--chunk-size` 参数控制单次处理条数（例如 `--chunk-size 500`）。同时，可用性检查模块默认开启并发限制（最大 50 个并发请求），可通过 `--max-workers` 调整并发数以平衡资源消耗。

**Q：如何定期自动更新链接的可用性状态？**

项目本身不包含定时任务调度器，但提供了 `health_check.py` 模块。用户可以借助系统级 cron 任务或 CI 流水线（如 GitHub Actions）周期性调用 `python cli.py check --timeout 5` 命令，并将结果输出至日志文件，实现无人值守的链接状态巡检。
