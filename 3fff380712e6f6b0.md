# ResourceBridge

ResourceBridge 是一个面向开发者、技术研究人员与内容策展人的轻量级外链资源聚合与导航工具。该项目并非传统的网址导航站，而是一套基于静态数据结构的资源引用框架，旨在帮助技术团队系统性地收集、分类、校验和呈现分布在各类平台上的高价值外部链接。ResourceBridge 解决了大型项目文档中外部链接分散、失效难以追踪、引用格式不统一等长期痛点，适用于需要维护大量外链引用的技术文档库、开源项目资源页、内部知识库及学术参考资料集。

ResourceBridge 以纯静态方式运行，不依赖后端数据库，所有资源记录以结构化 Markdown 或 YAML 形式存储，支持 CI 集成进行链接可用性检测。项目本身提供一套标准化的资源录入与展示规范，而非一个已填充内容的成品站。用户可将此框架应用于自己的文档站点生成器（如 VuePress、VitePress、Docusaurus）或静态站点生成工具中，快速构建出带有完整分类、标签和状态标记的资源导航体系。

## 功能概览

**结构化资源录入**：提供标准化的资源条目模板，包括标题、URL、来源平台、内容摘要、标签、录入时间、状态（有效/失效/待审核）等字段，所有数据以 YAML 或 Front Matter 格式存储在对应目录下，便于版本管理和批量处理。

**自动分类与标签聚合**：基于资源元数据中的分类标签和来源域名自动生成索引页，支持多级分类展示，用户无需手动维护分类页面，框架会在构建时自动聚合相同标签或同源域名下的资源条目。

**链接可用性检测**：集成命令行检测工具，可定期对已收录的 URL 发起 HEAD 请求，检查响应状态码，生成失效链接报告，帮助维护者快速定位需要更新或移除的条目，保证资源列表的健康度。

**多视图呈现模式**：支持列表视图、卡片视图和紧凑表格视图三种展示方式，满足不同场景下的浏览需求。列表视图适合快速扫读，卡片视图适合展示资源摘要信息，表格视图适合批量导出或打印。

**全文检索与过滤**：内置基于前端索引的轻量级搜索功能，支持按标题关键词、标签、来源域名、录入日期范围进行组合过滤，搜索结果实时高亮匹配内容，提升用户在大量资源中的查找效率。

**导入与导出兼容性**：支持从 CSV 和 JSON 格式批量导入资源条目，也可将当前资源库导出为通用格式，便于与其他工具（如 Notion、Airtable、Zotero）进行数据交换，降低迁移成本。

**自定义主题与布局**：提供可配置的 CSS 变量和布局选项，用户可以调整页面主色、卡片圆角、最大宽度等视觉参数，使资源导航页与现有站点风格保持一致，无需修改核心组件代码。

## 应用场景

技术文档库外链管理：当一个开源项目或技术产品拥有超过数百篇文档时，其中引用的外部参考链接数量往往十分庞大。ResourceBridge 可作为文档站点的子模块独立运行，集中管理所有外链，并在文档中通过短代码或组件引用资源 ID，实现链接的统一维护和更新，避免在正文中散落裸 URL。

内部团队知识库资源聚合：企业或研究机构内部常存在大量分散的收藏链接，散落在聊天记录、邮件或个人浏览器收藏夹中。使用 ResourceBridge 可以建立一个团队共享的资源导航页，按项目、技术领域或用途分类，新成员入职时可快速了解团队常用的工具、文档和参考网站。

学术参考资料整理：研究人员在撰写论文或技术报告时需要管理大量参考文献和在线资料。ResourceBridge 提供类似文献管理工具的资源条目结构，支持标注来源类型、访问日期和摘要，方便后续生成引用列表或进行文献回顾。

开源项目 README 外链补充页：部分开源项目的 README 因篇幅限制无法详细列出所有相关资源。ResourceBridge 可以作为独立的 Resources 页面部署在项目 Pages 服务上，与主仓库 README 形成互补，读者可通过导航页获取社区推荐的学习资料、工具和案例。

## 快速开始

以下命令演示如何从 GitHub 克隆 ResourceBridge 框架、安装依赖并启动本地开发服务器。请确保已安装 Node.js 16.0 及以上版本和 npm 或 yarn 包管理器。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git

# 进入项目目录
cd resourcebridge

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

执行上述命令后，本地开发服务器默认运行在 http://localhost:3000。打开浏览器访问该地址即可看到示例资源导航页面。如需构建生产环境静态文件，请使用 `npm run build`，输出目录默认为 `dist`。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Node.js 16.x 或更高版本 | 是 | 运行时环境，用于执行构建脚本和开发服务器 |
| npm 8.x 或 yarn 1.22+ | 是 | 包管理器，用于安装项目依赖 |
| Git 2.25+ | 是 | 版本控制工具，用于克隆仓库和管理资源变更历史 |
| 现代浏览器（Chrome 90+ / Firefox 88+ / Edge 90+） | 否 | 仅用于预览和浏览生成的导航页面，服务端构建无需浏览器 |
| markdownlint-cli 或类似工具 | 否 | 可选依赖，用于校验资源条目中的 Markdown 格式是否符合规范 |
| curl 或 wget | 否 | 可选工具，用于在 CI 脚本中执行链接可用性检测 |

## 常见问题

**问：ResourceBridge 是否必须依赖 Node.js 环境？能否在 Python 或 Go 项目中集成？**

答：ResourceBridge 的核心实现基于 Node.js，因此构建过程需要 Node.js 环境。但输出的 `dist` 目录是纯静态文件（HTML、CSS、JavaScript），可以部署在任何 Web 服务器或静态托管服务上。如果您的技术栈不使用 Node.js，仍然可以使用该框架——只需在 CI 流程中单独安装 Node.js 用于构建，然后将构建产物复制到您的服务端目录即可。未来版本计划提供 Docker 镜像，以进一步降低环境依赖。

**问：资源条目的数据格式是否支持自定义字段？如何扩展？**

答：ResourceBridge 的核心数据模型允许在 YAML 或 Front Matter 中添加自定义字段。解析器默认会提取所有顶层键值对，并在渲染模板时暴露给视图层。您可以通过修改模板文件来展示自定义字段内容，无需改动核心解析逻辑。但需注意，自定义字段不会影响内置的分类聚合和搜索索引，如需将自定义字段纳入索引，需扩展 `indexer.js` 中的索引构建逻辑。

**问：链接检测模块如何处理需要登录或具有反爬机制的站点？**

答：链接检测模块默认使用简单的 HEAD 请求，超时时间为 10 秒。对于需要登录或具有反爬机制的站点，HEAD 请求可能返回 403 或 405 状态码，这种情况下检测结果可能不准确。您可以在配置文件中为特定域名设置自定义检测策略，例如改用 GET 请求并携带特定的 User-Agent 头，或跳过某些域名的检测。对于内网或私有域名，建议在检测配置中将其加入白名单，避免因网络不可达而产生误报。
