# LinkVault Resource Aggregator

LinkVault is a lightweight, extensible resource aggregation and navigation system designed for technical teams, researchers, and content curators who need to manage, categorize, and share large volumes of external reference links. The project provides a structured indexing framework that transforms raw URL collections into navigable resource catalogs with metadata tagging, full-text search, and batch import/export capabilities.

Target users include developer documentation maintainers, academic research assistants, technical content editors, and open-source project maintainers who routinely handle link-rich reference materials. LinkVault addresses the common problem of link sprawl by offering a local-first, static-site generation approach that eliminates dependency on third-party bookmarking services while providing version-controlled, auditable resource inventories.

## 功能概览

**Bulk URL Ingestion Pipeline** - Parse and import large URL lists from plain text, CSV, JSON, or markdown sources with automatic deduplication and validity checking.

**Hierarchical Category Tagging** - Assign multiple taxonomy tags to each resource and organize links into nested topic trees for logical browsing and filtering.

**Full-Text Search Engine** - Built-in search over URL titles, descriptions, tags, and source metadata using inverted index with fuzzy matching and relevance ranking.

**Static Site Generator** - Export the entire resource catalog as a self-contained HTML documentation site with responsive design, no server-side runtime required.

**Metadata Enrichment Module** - Automatically fetch page titles, descriptions, and Open Graph data from live URLs during import to reduce manual annotation effort.

**Import and Export Adapters** - Support YAML, JSON, Markdown table, and CSV formats for seamless integration with external documentation pipelines and data processing tools.

## 应用场景

**Technical Documentation Maintenance** - Documentation teams managing large reference sections can use LinkVault to maintain curated lists of external API references, tutorial links, and community resources, ensuring all outbound links remain organized and verifiable during documentation release cycles.

**Academic Literature Indexing** - Research groups compiling bibliographies, preprint repositories, and supplementary material collections can leverage the tagging system to cross-reference papers by topic, author, or publication venue, facilitating collaborative literature reviews.

**Open-Source Project Resource Pages** - Project maintainers can generate and maintain "awesome list" style resource pages directly from their repository, allowing contributors to propose new links via pull requests while keeping the project website automatically synchronized.

**Internal Knowledge Base Curation** - Enterprise technical writers and knowledge managers can build internal developer portals by aggregating distributed documentation links, internal tools, and training materials into a single searchable catalog accessible to entire engineering organizations.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/your-org/linkvault.git
cd linkvault

# Install dependencies using pip
pip install -r requirements.txt

# Run the initial import and generate the static site
python linkvault.py import --input links.txt --output catalog/
python linkvault.py build --source catalog/ --destination site/
```

## 项目结构

```
linkvault/
├── linkvault.py               # 主命令行入口，整合所有子命令
├── requirements.txt           # Python 依赖清单，固定版本号
├── config/
│   ├── default.yaml           # 默认配置项：输出路径、缓存策略、并发数
│   ├── schema.json            # 资源条目 JSON Schema 定义
│   └── taxonomy.yaml          # 默认分类标签体系与层级映射
├── core/
│   ├── __init__.py
│   ├── importer.py            # 导入器核心：解析源文件、去重、预处理
│   ├── indexer.py             # 索引引擎：构建倒排索引、维护词频统计
│   ├── metadata.py            # 元数据抓取：HTTP 请求、HTML 解析、超时重试
│   └── exporter.py            # 导出适配器：YAML/JSON/Markdown 序列化
├── generators/
│   ├── __init__.py
│   ├── static.py              # 静态站点生成器：模板渲染、资源复制
│   ├── sitemap.py             # XML 站点地图生成，用于搜索引擎收录
│   └── rss.py                 # RSS feed 输出，支持订阅更新
├── utils/
│   ├── __init__.py
│   ├── validator.py           # URL 校验：格式检查、协议白名单、黑名单过滤
│   ├── cache.py               # 缓存管理：LRU 实现、过期清理、持久化
│   └── logger.py              # 日志系统：多级别输出、文件轮转、结构化日志
├── tests/
│   ├── unit/                  # 单元测试：每个核心模块对应独立的测试文件
│   ├── integration/           # 集成测试：端到端导入导出流程验证
│   └── fixtures/              # 测试数据集：示例链接列表、预期输出结果
├── templates/
│   ├── base.html              # 站点 HTML 基础模板，包含导航和页脚
│   ├── index.html             # 首页模板：搜索框、分类面板、最近更新
│   └── detail.html            # 详情页模板：完整元数据、标签、相关链接
```

## 常见问题

**问：导入大量 URL 时出现超时或连接错误，如何处理？**

答：元数据抓取模块默认设置了每个请求 10 秒超时和最多 3 次重试。对于批量导入，建议使用 --concurrency 参数控制并发数量，默认值为 5，可根据网络环境适当调低。同时，缓存机制会存储已抓取的元数据，重复导入时不会重新请求，大幅提升后续处理速度。

**问：如何自定义静态站点的外观和布局？**

答：所有模板文件位于 templates/ 目录下，使用 Jinja2 模板引擎渲染。您可以修改 base.html 调整全局布局，修改 index.html 和 detail.html 调整列表页和详情页样式。CSS 样式表位于 templates/static/css/ 目录，支持完全自定义。执行 build 命令时可通过 --theme 参数指定自定义模板目录。

**问：LinkVault 是否支持增量更新而非全量重建？**

答：支持。导入器会维护一个本地索引文件 records.json，记录每条资源的导入时间和内容哈希。执行 import 命令时使用 --update 标志会仅处理新增或变更的条目，已有记录保持不变。执行 build 命令时使用 --incremental 标志会仅重新生成受影响页面，极大缩短大型站点的构建时间。
