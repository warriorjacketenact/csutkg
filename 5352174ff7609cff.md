# Hyperlink Index Catalog System

Hyperlink Index Catalog System (HICS) is a lightweight, community-driven technical resource aggregation platform designed to organize, categorize, and preserve high-value external reference links for developers, researchers, and technical writers. The system addresses the common challenge of fragmented bookmark collections and ephemeral reference storage by providing a structured, version-controlled, and queryable catalog of curated URLs across multiple technical domains.

HICS targets users who regularly work with technical documentation, API references, specification drafts, and community forum discussions. Instead of maintaining scattered browser bookmarks or ad-hoc text files, users leverage HICS to maintain a centralized index with contextual metadata, topic tagging, and cross-referencing capabilities. The project operates as a static site generator with a CLI tool for link validation, metadata extraction, and catalog formatting, ensuring that every referenced resource remains discoverable and well-annotated over time.

## 功能概览

**URL 规范化与去重** - Automatically normalizes incoming URLs to their canonical form, detects duplicate submissions, and maintains a single authoritative entry per resource regardless of how it was originally submitted.

**自动元数据抓取** - Fetches page titles, description meta tags, and Open Graph properties from each submitted URL to populate catalog entries with minimal manual intervention.

**分类标签系统** - Supports hierarchical tagging with support for nested categories, alias resolution, and tag inheritance across related resources.

**链接健康监控** - Periodically checks all stored URLs for HTTP status changes, certificate expiry, and domain resolution issues, flagging broken or redirected links in the catalog dashboard.

**全文检索与过滤** - Provides full-text search across URL slugs, page titles, extracted descriptions, and user-added notes with support for boolean operators and field-specific filters.

**批量导入与导出** - Allows ingestion of URL lists via plain text, CSV, or JSON formats, and exports catalog subsets as Markdown tables, HTML indices, or structured JSON feeds for integration with other tools.

**版本化变更追踪** - Maintains a complete audit log of all additions, deletions, and metadata updates, enabling rollback to previous catalog states and facilitating collaborative curation workflows.

## 应用场景

**技术文档库索引** - A development team maintaining internal documentation portals can use HICS to index external API references, framework guides, and Stack Overflow discussions relevant to their tech stack, ensuring new team members can quickly locate authoritative external resources.

**学术研究参考文献收集** - Graduate students and researchers aggregating supplementary materials for literature reviews can organize preprint servers, dataset repositories, and institutional publication pages into HICS, attaching custom notes about relevance and retrieval dates for each entry.

**开源项目周边生态导航** - An open-source project maintainer can curate a list of community blogs, video tutorials, plugin registries, and migration guides, embedding the generated catalog directly into the project's documentation site to reduce duplicate queries in issue trackers.

**合规审计资源跟踪** - Security or compliance officers needing to monitor regulatory publications, CVE databases, and vendor security advisories can rely on HICS's link health monitoring to be alerted when critical reference pages change or become unavailable.

**个人知识库外链管理** - Independent researchers and technical writers building personal knowledge bases can augment their local notes with a HICS instance that stores all external citations, ensuring that referenced URLs remain stable and correctly attributed even when original pages are restructured.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/hics-org/hyperlink-index-catalog.git
cd hyperlink-index-catalog

# Install dependencies
pip install -r requirements.txt

# Initialize the catalog database and run the ingestion pipeline
python -m hics.cli init
python -m hics.cli import --source urls.txt --format plain
python -m hics.cli validate --parallel 4
python -m hics.cli build --output docs/index.html
```

## 常见问题

**问：系统如何处理被删除或移动的页面？**

系统在每次验证周期中记录 HTTP 状态码的变化。如果某个 URL 返回 404 或 410，该条目会被标记为已失效但不会自动删除，保留历史记录以供人工审查。对于 301 或 302 重定向，系统会更新存储的 URL 为目标地址并记录重定向链。用户可以通过 `hics validate --fix-redirects` 命令批量应用重定向修正。

**问：能否在无网络环境中使用 HICS？**

可以。首次导入 URL 后，所有元数据和验证结果都存储在本地 SQLite 数据库中。在无网络环境下，系统跳过健康检查并使用缓存的元数据生成静态输出。若要强制使用缓存而不发出任何 HTTP 请求，可设置环境变量 `HICS_OFFLINE_MODE=1`。所有导出和构建功能完全离线可用。

**问：如何导入来自浏览器书签导出的 HTML 文件？**

系统支持通过 `hics import --format bookmarks --source bookmarks.html` 命令直接解析 Netscape 书签格式的 HTML 文件。该导入器会提取每个书签的 URL、标题、添加日期和文件夹路径，并将其映射为标签和元数据字段。对于其他格式，可先转换为纯文本 URL 列表（每行一个 URL），然后使用 `--format plain` 导入。
