# ResourceBridge

ResourceBridge 是一个面向技术研究、数据分析与学术写作的轻量级外链资源汇总与导航系统。该项目定位于帮助开发者、研究人员与技术内容创作者快速定位高质量外部技术文档、在线工具、数据集、学术论文与工程实践参考来源，通过统一入口降低信息检索成本，提升技术调研与学习效率。

项目本身不存储任何实体资源文件，仅提供结构化链接管理与分类导航能力。ResourceBridge 适用于需要长期跟踪多个外部信息源、维护个人知识索引或构建团队共享技术情报面板的场景。通过 Markdown 驱动的清单管理与静态站点生成机制，用户可以零依赖、低门槛地维护属于自己的技术资源图谱。

## 功能概览

- **结构化资源清单管理**：支持按主题、来源、文件类型等多维度标签对链接进行分类与筛选，便于快速定位特定领域的外部资源。

- **全文元数据提取**：对每条链接自动提取或手动补充标题、摘要、发布日期与关键词，生成可检索的资源卡片。

- **链接可用性监测**：内置周期性连通性检查模块，标记失效链接并生成报告，保障资源库的长期可用性。

- **多格式数据导出**：支持将资源列表导出为 JSON、CSV 及 HTML 摘要页，便于嵌入笔记系统、文档站点或数据看板。

- **本地化搜索与过滤**：基于关键词、标签或时间范围的即时搜索功能，无需外部搜索引擎即可在本地资源库中完成初步筛选。

- **版本化变更记录**：每次资源增删改操作均生成变更日志，支持回溯历史状态与协作审核。

- **模板化快速添加**：提供命令行交互与 Web 表单两种添加方式，自动校验 URL 格式并引导补全必要元数据。

## 应用场景

- **技术调研与竞品分析**：技术负责人可使用 ResourceBridge 收集竞品官方文档、技术博客、API 参考与社区讨论链接，形成结构化的调研资源包，分发给团队成员进行同步研读。

- **学术论文写作与文献管理**：研究生或科研人员在撰写文献综述时，可将预印本仓库、数据集发布页、工具官方代码库与相关项目主页统一收录，配合元数据字段管理阅读进度与重要程度。

- **开源项目维护与生态追踪**：开源项目维护者可使用本系统跟踪依赖库的更新公告、安全漏洞通告、相关工具链与示例项目，及时掌握上下游生态变动。

- **个人知识库外链增强**：配合 Obsidian、Logseq 或 Notion 等知识管理工具，将 ResourceBridge 作为外部链接的专用索引层，与内部笔记解耦，降低笔记体积并提升引用复用率。

## 快速开始

以下命令将在本地克隆项目仓库、安装依赖并启动开发服务。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行完毕后，访问控制台输出的本地地址（默认为 http://127.0.0.1:5173）即可进入资源管理界面。首次启动将自动生成示例数据文件，位于 `data/sources.json`。用户可直接编辑该文件或通过界面操作添加自定义链接。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | >= 18.0.0 | 运行时环境，用于执行构建脚本与开发服务器 |
| npm | >= 9.0.0 | 包管理器，用于安装项目依赖 |
| Git | >= 2.30.0 | 版本控制工具，用于克隆仓库与提交变更 |
| 现代浏览器 | 最新两个版本 | 用于访问管理界面，支持 Chrome、Firefox、Edge、Safari |
| 磁盘空间 | >= 100 MB | 用于存放项目代码、依赖包与生成的元数据缓存 |
| 网络连接 | 稳定 | 用于首次启动时提取链接标题与摘要信息，以及后续链接可用性检测 |
| 操作系统 | Linux / macOS / Windows | 跨平台支持，Windows 下建议使用 WSL2 或 PowerShell 7+ |

```

│   ├── core/                          # 核心业务逻辑模块
│   │   ├── linker.js                  # 链接解析与元数据提取引擎
│   │   ├── checker.js                 # 链接可用性检测与状态管理
│   │   └── exporter.js                # 多格式数据导出处理器
│   ├── ui/                            # 用户界面组件
│   │   ├── App.vue                    # 主界面根组件
│   │   ├── ResourceList.vue           # 资源列表渲染与分页控制
│   │   └── FilterPanel.vue            # 多维度筛选与搜索面板
│   ├── server/                        # 本地开发服务器与API模拟层
│   │   ├── routes.js                  # 路由定义与请求分发
│   │   └── middleware.js              # 日志、跨域与错误处理中间件
│   └── utils/                         # 通用工具函数集
│       ├── validator.js               # URL格式校验与标准化
│       ├── cache.js                   # 内存缓存与持久化策略
│       └── logger.js                  # 分级日志输出与文件记录
├── data/                              # 数据存储目录（用户可读写）
│   ├── sources.json                   # 主资源清单（核心数据文件）
│   ├── metadata/                      # 自动抓取的补充元数据缓存
```

## 贡献指南

欢迎并感谢任何形式的贡献。请遵循以下流程以确保代码质量和协作效率。

1. 查阅现有 Issues 与 Pull Requests，确认尚未有人认领或解决您计划处理的问题。若无相关记录，请新建一个 Issue 简要描述变更内容、动机与预期影响。

2. 从主分支 `main` 创建新的功能分支，分支命名建议采用 `feature/功能简述` 或 `fix/问题简述` 格式。确保分支基于最新的主分支代码。

3. 编写或修改代码时，请遵循项目内预设的 ESLint 与 Prettier 格式化规则。所有新增功能须包含对应的单元测试用例，测试覆盖率不应低于现有水平。提交信息采用约定式提交格式（如 `feat: 添加批量导入功能` 或 `fix: 修复链接检测超时问题`）。

4. 完成本地验证后，将分支推送到远程仓库并创建 Pull Request。PR 描述中需关联对应的 Issue 编号，并详细列出变更点与测试结果。至少一位项目维护者审核通过后方可合并。

5. 文档更新与代码变更同等重要。若新增或修改了功能，请同步更新 `docs/` 目录下的相关文档，并在 README 的「功能概览」或「安装要求」中反映重大变动。

## 常见问题

**Q：链接可用性检测报告显示大量超时或拒绝连接，是否意味着这些资源已经失效？**

A：不一定。部分外部站点可能对频繁请求有访问限制或需要特定的 User-Agent 标头。建议首先调整检测间隔时间与并发数，避免触发目标服务器的反爬策略。其次，可尝试在配置文件中设置自定义请求头以模拟浏览器访问。若问题持续，请手动访问链接确认实际状态。

**Q：如何将现有书签或收藏夹数据迁移到 ResourceBridge 中？**

A：项目提供了 `scripts/migrate.js` 工具，当前支持从 Chrome 书签导出 HTML 文件和 Firefox 的 JSON 备份格式进行转换。具体操作为：将书签导出文件放置于 `data/import/` 目录，然后执行 `npm run migrate -- --source=chrome --file=bookmarks.html`。转换后的条目将自动合并到 `sources.json` 中，重复链接会依据 URL 进行去重。

**Q：ResourceBridge 是否支持多用户或团队协作场景？**

A：当前版本定位为单机个人工具，未内置用户系统与权限控制。若需团队共享，建议将 `data/sources.json` 文件置于 Git 仓库中，通过版本控制协调变更。未来版本将考虑增加基于文件锁的协作标记与冲突合并向导。
