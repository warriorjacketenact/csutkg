# WebIndex Gateway

WebIndex Gateway 是一个面向技术研究、数据采集与自动化运维场景的轻量级外链资源聚合与导航系统。该项目不提供具体的内容服务，而是作为结构化索引层，将分散于多个数据源的技术文档、API 接口说明、运维手册、日志分析示例及配置模板等外链资源进行统一收敛与分类挂载。

项目定位为技术基础设施的辅助导航组件，适用于需要频繁查阅外部技术资料但缺乏统一入口的研发团队、运维工程师及安全分析人员。通过本地化的资源映射与分类管理，WebIndex Gateway 帮助用户降低跨站点检索的时间成本，提升信息获取的确定性。

## 功能概览

- **外链分类挂载**：支持按技术领域、文档类型或业务模块对导入的外链进行多级分类与标签管理。
- **资源状态检测**：定时检测已收录资源的可访问性与响应状态，辅助识别失效或迁移的链接。
- **快速检索过滤**：基于关键词与分类标签对已收录资源进行即时筛选，支持模糊匹配与精确查找。
- **导入导出接口**：提供标准化的 JSON 与 CSV 格式资源清单导入导出能力，便于批量迁移或备份。
- **访问统计看板**：记录资源被查阅或跳转的次数与时间分布，辅助识别高频使用的核心资料。
- **自定义分类视图**：允许用户根据当前项目或运维阶段创建临时分类视图，不影响全局分类结构。
- **只读镜像模式**：支持生成当前资源清单的只读快照，用于对外公示或归档审计。
- **元数据扩展字段**：每条资源支持自定义键值对元数据，用于记录负责人、更新时间、所属版本等附加信息。

## 应用场景

- **企业内部技术文档导航**：技术团队可将分散于不同代码仓库、知识库或外部供应商站点的文档链接统一收录，通过 WebIndex Gateway 构建团队内部的技术资料门户。
- **运维监控工具链整合**：运维工程师可将各类监控系统、日志平台、告警管理后台的入口地址集中管理，配合状态检测功能快速发现控制台访问异常。
- **安全分析信息聚合**：安全研究人员可将威胁情报源、漏洞数据库、CVE 公告页面、厂商安全通告等外链资源按区域或威胁类型分组，提升应急响应时的信息检索效率。
- **项目交付资源清单交付**：在项目交付阶段，可将涉及的所有第三方依赖文档、API 参考、部署手册等链接导出为只读清单，随项目文档一并移交。

## 快速开始

以下操作以 Linux / macOS 环境为例，假定系统已安装 Git、Node.js 与 npm。

```bash
git clone https://github.com/webindex-gateway/webindex-gateway.git
cd webindex-gateway
npm install
npm run build
npm start
```

服务启动后默认监听 3000 端口，可通过浏览器访问 http://localhost:3000 查看导航界面。如需修改端口或调整数据存储路径，可编辑 config/default.yaml 文件。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用官方预编译版本或 nvm 管理 |
| npm | 9.x 或以上 | 依赖包管理工具，随 Node.js 一并安装 |
| SQLite3 | 3.39.x 或以上 | 嵌入式数据库，用于存储资源索引与分类信息，无需额外部署服务 |
| Git | 2.30.x 或以上 | 用于克隆项目仓库及版本管理 |
| 可用磁盘空间 | 至少 200 MB | 用于存放项目源码、依赖包及数据库文件 |
| 内存 | 至少 512 MB | 生产环境建议 1 GB 以上 |

## 项目结构

```
webindex-gateway/
├── src/
│   ├── core/                       # 核心模块：资源索引管理、分类树维护
│   │   ├── indexer.js              # 资源索引构建与增量更新逻辑
│   │   ├── classifier.js           # 分类树数据结构与遍历算法
│   │   └── validator.js            # 外链格式校验与去重
│   ├── server/                     # HTTP 服务层：路由、中间件、静态资源
│   │   ├── app.js                  # Express 应用初始化与中间件注册
│   │   ├── router.js               # 统一路由定义（前端 API 与健康检查）
│   │   └── static/                 # 前端界面静态资源文件
│   ├── storage/                    # 持久化层：数据库适配器与迁移脚本
│   │   ├── sqlite.js               # SQLite3 连接池与 CRUD 操作封装
│   │   ├── migrations/             # 数据库结构版本管理脚本
│   │   └── seed/                   # 初始分类与示例数据导入
│   ├── scheduler/                  # 后台任务调度：状态检测、统计聚合
│   │   ├── health-check.js         # 外链可访问性定时检测任务
│   │   └── stats-aggregator.js     # 访问统计定时聚合任务
│   └── utils/                      # 通用工具函数：日志、配置、加密
│       ├── logger.js               # 结构化日志输出（JSON 格式）
│       ├── config-loader.js        # 多环境配置文件加载
│       └── crypto.js               # 简单哈希与签名工具
```

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并在本地 clone 已 fork 的副本。新建功能分支时应使用 `feature/` 或 `fix/` 前缀，例如 `feature/add-import-export`。
2. 本地开发时请确保所有单元测试与集成测试通过，新增功能或修复缺陷需附带对应的测试用例。测试命令为 `npm test`。
3. 提交代码前运行 `npm run lint` 执行代码风格检查，并确保无新增的 ESLint 警告或错误。
4. 提交信息应遵循 Conventional Commits 规范，使用 `feat:`、`fix:`、`docs:`、`chore:` 等类型前缀，并附上简明扼要的变更描述。
5. 提交 Pull Request 至主仓库的 `main` 分支，PR 描述中需说明变更目的、实现方式及测试覆盖情况。PR 合并前至少需要一名项目维护者的审核与批准。

## 常见问题

**问：启动后访问页面提示无法加载资源列表，应如何排查？**

答：首先检查 SQLite 数据库文件是否存在于配置指定的路径下，且当前运行用户是否拥有读写权限。其次确认 config/default.yaml 中 database.path 配置项与实际路径一致。若数据库文件缺失，可尝试执行 `npm run seed` 初始化示例数据。若仍无法解决，查看日志文件中的错误堆栈信息，定位具体失败原因。

**问：导入包含大量外链的 CSV 文件时出现超时或内存占用过高，该如何优化？**

答：CSV 导入功能默认采用批量提交方式，但单次导入记录数不宜超过 5000 条。建议将大型文件分割为多个子文件分批导入。若内存仍不足，可在 config/production.yaml 中调整 storage.batchSize 参数为更小值，并适当降低 scheduler.interval 以避免导入与检测任务争抢资源。

**问：资源状态检测任务误判某些可访问站点为不可达，如何处理？**

答：状态检测默认使用 HEAD 请求并设置 5 秒超时。部分站点可能限制 HEAD 请求或响应较慢，可在配置文件中调整 health-check.method 为 GET，并适当增加 health-check.timeout 值。对于自签名证书或内部站点，可在 health-check.ignoreCertErrors 中配置忽略证书校验。
