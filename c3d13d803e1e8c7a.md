# Tokyo Mirai Resource Index

Tokyo Mirai Resource Index 是一个面向技术研究、数据分析与数字档案管理的高密度外链资源汇总系统。本项目定位为技术文献、视频资料与实验数据的统一导航门户，主要服务于研究人员、数据分析师、开源社区维护者以及需要系统化整理大规模外链资源的技术团队。通过结构化的资源分类、可扩展的索引机制以及轻量化的部署方式，该项目帮助用户从海量分散的链接中快速定位高价值内容，降低信息检索与管理的成本。

本项目并非传统的爬虫或采集系统，而是一个基于人工精选与标签化组织的外链元数据库。它通过静态化的资源列表与标准化的字段描述，使得每一类资源都可被追溯、可被分类、可被复用。项目设计上强调可维护性与透明性，所有资源链接均以明文形式记录，便于版本控制与协作更新。

## 功能概览

**多层级资源分类体系**：项目将所有外链资源按照内容主题、来源机构、文件格式等多个维度进行标签化分类，支持通过目录结构快速过滤特定类型的资源。

**静态化高速索引**：基于纯静态 Markdown 与 JSON 结构生成索引页面，无需数据库驱动，可在任何标准 Web 服务器或本地环境中直接浏览，响应速度极快。

**资源状态自动检测**：内置链接可达性检查脚本，可定期对已收录的 URL 进行可用性验证，标记失效链接并生成报告，保证资源列表的健康度。

**标签与关键词检索**：每一类资源均附带描述性标签和关键词元数据，用户可通过命令行工具或 Web 前端对资源进行模糊检索和精确匹配。

**批量导入与去重管理**：支持从 CSV、JSON 及纯文本列表批量导入链接，并自动检测重复条目，避免同一资源的多次冗余收录。

**版本化更新记录**：每次资源列表的增删改操作均会生成变更日志，便于团队协作时追溯修改历史，支持回滚至任意历史版本。

**扩展性钩子机制**：提供预留的钩子函数接口，允许高级用户自定义资源预处理规则、分类映射规则及输出格式模板。

## 应用场景

学术文献与研究报告的快速索引：研究机构可将本项目作为内部知识库的前端导航层，用于整理近期发布的会议论文、技术白皮书及实验视频记录。通过统一入口，团队成员无需记忆繁杂的 URL，直接通过分类标签即可获取相关资料。

开源项目文档与视频教程归档：开源社区维护者可将项目相关的教学视频、版本发布说明及 API 文档链接纳入索引，配合版本化更新记录功能，确保社区成员始终访问到最新或指定版本的资源。

数据密集型实验的资源管理：数据科学团队在进行模型训练或数据分析时，通常需要引用大量外部数据集和工具手册。本项目可作为资源清单的集中管理工具，确保实验记录的可复现性，同时方便新成员快速搭建相同的依赖环境。

企业内部技术周报与知识沉淀：技术部门可利用本项目汇总每周团队内部分享的会议录像、外部技术博客推荐及工具链更新公告，形成持续积累的知识索引，降低信息孤岛风险。

## 快速开始

以下命令演示了如何从 GitHub 克隆本项目、安装必要依赖并启动本地索引服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/tokyo-mirai/resource-index.git

# 进入项目根目录
cd resource-index

# 安装 Python 依赖（用于链接检测与索引生成）
pip install -r requirements.txt

# 运行本地索引生成脚本，输出静态页面至 dist 目录
python build_index.py --input ./data/links.json --output ./dist

# 若需启动本地预览服务器，可执行
python -m http.server 8000 --directory ./dist
```

完成上述步骤后，打开浏览器访问 `http://localhost:8000` 即可查看资源索引首页。

## 安装要求

本项目核心脚本基于 Python 3.9+ 开发，依赖标准库及少量第三方包。推荐在 Linux 或 macOS 环境下运行，Windows 用户可通过 WSL 或 Git Bash 执行。具体依赖如下表所示。

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Python | 3.9 或更高 | 核心脚本运行环境，用于索引生成、链接检测与数据校验 |
| pip | 21.0 或更高 | Python 包管理工具，用于安装 requirements.txt 中列出的依赖 |
| Git | 2.25 或更高 | 用于克隆仓库及版本管理，非运行时必需但建议安装 |
| requests | 2.28.0 或更高 | 用于发送 HTTP 请求以检测链接可用性，若禁用检测功能可忽略 |
| markdown | 3.4.0 或更高 | 用于将 Markdown 格式的资源描述渲染为 HTML，仅在生成 Web 页面时需要 |
| json5 | 0.9.14 或更高 | 用于解析带注释的 JSON 配置文件，便于人工编辑和注释维护 |
| pytest | 7.2.0 或更高 | 仅开发测试时需要，用于运行单元测试套件验证索引逻辑正确性 |

## 项目结构

项目采用标准的 Python 包布局，兼顾脚本执行与模块导入。核心源代码位于 src 目录下，配置与数据文件分别存放于 config 与 data 目录，测试用例独立维护在 tests 目录中。

```
resource-index/
├── src/                                  # 核心源代码目录
│   ├── core/                             # 索引引擎核心模块
│   │   ├── indexer.py                    # 索引生成主逻辑，负责读取链接列表并构建倒排表
│   │   ├── validator.py                  # 链接校验器，包含 HTTP 状态检测与超时重试机制
│   │   └── classifier.py                 # 自动分类器，基于关键词规则对链接进行标签匹配
│   ├── parsers/                          # 输入解析子模块
│   │   ├── json_parser.py                # 解析 JSON 格式的链接清单，支持 json5 注释
│   │   └── csv_parser.py                 # 解析 CSV 格式的导入文件，自动检测分隔符
│   ├── exporters/                        # 输出导出子模块
│   │   ├── markdown_exporter.py          # 将索引结果导出为 Markdown 表格格式
│   │   └── html_exporter.py              # 渲染带样式和检索框的静态 HTML 页面
│   └── cli/                              # 命令行接口模块
│       └── main.py                       # 统一 CLI 入口，支持 build、check、export 子命令
├── config/                               # 全局配置目录
│   ├── default.yaml                      # 默认配置，包含超时时间、分类规则路径、输出目录等
│   └── custom_rules.json5                # 用户自定义分类规则，支持添加新的关键词映射
├── data/                                 # 数据文件目录
│   ├── links.json                        # 主链接数据库，包含本批次全部 250 条记录及元数据
│   └── tags.json                         # 标签体系定义文件，维护所有可用标签及其层级关系
├── tests/                                # 单元测试与集成测试目录
│   ├── test_validator.py                 # 链接校验模块的单元测试，包含模拟 HTTP 响应
│   └── test_classifier.py                # 自动分类模块的测试用例，覆盖边界情况
├── docs/                                 # 文档目录，包含用户手册、管理员指南与开发文档
│   ├── user-guide.md
│   ├── admin-guide.md
│   └── developer-guide.md
├── requirements.txt                      # Python 依赖清单
├── build_index.py                        # 项目根目录下的构建入口脚本，调用 src/cli/main.py
└── README.md                             # 项目说明文档（本文件）
```

## 贡献指南

本项目欢迎外部贡献者参与资源补充、分类优化以及脚本功能增强。所有贡献需遵循既定的数据格式规范与代码风格要求。

第一，提交资源新增或修改请求时，请先在 data/links.json 文件中按照既有字段结构添加新条目，确保 URL 字段与原格式完全一致，并填写描述标签和分类字段。提交前需运行 validator 模块进行链接可达性检测，确保新增链接有效。

第二，若需修改核心脚本逻辑（如 indexer.py 或 classifier.py），请先编写对应的单元测试用例并放置在 tests 目录下，保证原有功能不受影响。所有 Python 代码需遵循 PEP 8 风格规范，并包含必要的函数注释与类型标注。

第三，提交 Pull Request 之前，请确保在本地环境完整执行 `pytest` 测试套件并通过所有用例，同时运行 `build_index.py` 验证索引生成过程无报错。PR 描述中应清晰说明变更内容、影响范围以及测试结果摘要。

第四，对于新增分类规则或标签体系调整，需同步更新 config/custom_rules.json5 以及 docs/admin-guide.md 中对应的描述部分，保持文档与实际逻辑一致。

## 常见问题

问：资源列表中部分链接无法访问，应该如何标记或处理？

答：项目提供了内置的链接校验脚本 `python build_index.py check`，该脚本会遍历 data/links.json 中的所有 URL 并记录 HTTP 状态码。对于返回 4xx 或 5xx 的链接，系统会自动生成一份失效报告并存储在 reports 目录下。维护者可定期审阅该报告，手动确认链接是否永久失效或临时波动，若确认失效则从列表中移除或替换为有效镜像地址。在移除时请同步更新变更日志。

问：索引生成脚本运行时报错 “ModuleNotFoundError: No module named 'json5'”，如何解决？

答：该错误表示 Python 环境中缺少 json5 依赖包。请执行 `pip install -r requirements.txt` 重新安装所有依赖。若仍存在问题，请检查当前激活的 Python 虚拟环境是否正确，或者手动执行 `pip install json5==0.9.14` 安装指定版本。建议使用 Python 3.9 以上的干净虚拟环境进行部署。

问：如何自定义资源分类标签，使得新增链接能够被自动归入特定类别？

答：所有自动分类规则均在 config/custom_rules.json5 文件中定义。该文件使用 JSON5 格式（支持注释），用户可按照 “关键词” : “标签名称” 的键值对形式添加规则。例如，若希望将所有包含 “testbed” 字样的链接归入 “实验平台” 类别，则增加 `"testbed": "实验平台"` 条目。修改后需重新运行 `python build_index.py --rebuild` 以重新生成索引并应用新规则。
