# LinkVault Resource Aggregator

LinkVault is a curated, machine-readable resource index and external link aggregation system designed for technical documentation teams, dependency trackers, and security auditing workflows. It addresses the problem of scattered, unmaintained external references across large codebases by providing a single source of truth for URI-based assets, versioned via a structured identifier scheme.

This project targets DevOps engineers, documentation maintainers, and compliance officers who require deterministic access to a large corpus of external links with known integrity patterns. The repository contains a verified collection of 250 resource locators, each tagged with a unique compound key for traceability and automated validation.

## 功能概览

**Deterministic Resource Indexing** – Each entry is assigned a composite identifier composed of a document serial and a resource ordinal, enabling collision-free referencing across distributed systems.

**Bulk URI Validation Pipeline** – A built-in validation suite performs HEAD and GET requests against all indexed URLs, generating a health report with response status codes and content-type headers.

**Metadata Extraction Layer** – Parses HTML title and meta description tags for each resource, storing extracted metadata in a structured JSON cache to support offline search and classification.

**Tagging and Categorization Engine** – Automatically assigns category labels (e.g., documentation, asset, reference, vendor) based on URL path patterns and content analysis, with manual override support via YAML configuration.

**Versioned Snapshot Export** – Exports the entire index as a timestamped CSV or JSON file, facilitating integration with external dependency management tools and change-log generation.

**Diff and Change Tracking** – Compares two index snapshots to identify added, removed, or modified URLs, producing a concise delta report useful for audit trails.

**Webhook Notification System** – Sends alerts to configured endpoints (e.g., Slack, email) when a resource becomes unreachable or when its metadata changes beyond a defined threshold.

**RESTful Query API** – Provides a lightweight HTTP interface for querying the index by identifier, category, or regex pattern, with response formats including JSON and plain text.

## 应用场景

**Documentation Link Rot Prevention** – Technical writing teams integrate LinkVault into their CI pipeline to automatically test every external link in their documentation suite. The system flags broken or redirected URLs before they reach production, reducing user frustration and support tickets.

**Dependency Asset Verification** – Build systems use LinkVault to maintain a manifest of all external assets (scripts, stylesheets, binaries) required during compilation. The index serves as a golden record, allowing builds to fail early when an external dependency changes location or becomes unavailable.

**Compliance and Regulatory Auditing** – Organizations subject to data retention or content sourcing regulations leverage LinkVault to produce auditable reports of all external references used in their products. The snapshot and diff features provide clear evidence of when and how resource URLs changed over time.

**Research Data Curation** – Academic and research teams managing large collections of referenced datasets, papers, or tools utilize LinkVault to maintain persistent, citable indices. The metadata extraction layer enriches each entry with contextual information, improving discoverability and citation accuracy.

## 快速开始

```bash
# Clone the repository with full history
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# Install dependencies using the provided requirements file
pip install -r requirements.txt

# Run the initial index build and validation
python linkvault.py --build --validate --output ./output/index.json
```

## 项目结构

```
linkvault-core/
├── src/                               # 核心源代码目录
│   ├── indexer/                       # 索引构建与维护模块
│   │   ├── builder.py                 # 从原始链接列表生成索引结构
│   │   ├── validator.py               # 执行并发HTTP验证和健康检查
│   │   └── cache.py                   # 管理本地元数据缓存和过期策略
│   ├── parser/                        # HTML内容解析与元数据提取模块
│   │   ├── extractor.py               # 封装lxml解析逻辑，提取标题和描述
│   │   └── classifier.py              # 基于路径和内容进行自动分类
│   ├── api/                           # RESTful API 服务模块
│   │   ├── app.py                     # Flask应用入口和路由定义
│   │   └── schemas.py                 # 请求和响应的数据验证模型
│   ├── exporters/                     # 数据导出与报告生成模块
│   │   ├── json_exporter.py           # 输出JSON格式索引快照
│   │   ├── csv_exporter.py            # 输出CSV格式，兼容电子表格软件
│   │   └── report_generator.py        # 生成健康状态和变更统计报告
│   └── utils/                         # 通用工具函数和辅助类
│       ├── config_loader.py           # 加载和合并YAML配置文件
│       ├── logger.py                  # 统一的日志记录和格式化
```

## 常见问题

**问：验证过程中出现大量超时错误，如何优化？**

答：超时错误通常由网络延迟或目标服务器限制引起。可以在 `config/default.yaml` 中调整 `timeout` 参数（默认 10 秒）和 `concurrency` 参数（默认 20 个并发请求）。建议根据网络环境逐步调低并发数并增加超时时间。此外，验证工具支持重试机制，可通过 `retry_count` 参数配置重试次数。

**问：如何自定义资源分类规则？**

答：分类规则定义在 `config/categories.yaml` 文件中。每个分类包含一个名称和一个正则表达式列表，用于匹配 URL 路径。您可以添加新的分类或修改现有分类的正则表达式。修改后需要运行 `python linkvault.py --reclassify` 重新应用分类规则到现有索引。

**问：RESTful API 服务的默认访问地址是什么？如何启用认证？**

答：默认情况下，API 服务监听在 `127.0.0.1:5000`。您可以通过 `--host` 和 `--port` 参数修改绑定地址。为启用认证，需要在 `config/default.yaml` 中设置 `api.auth_enabled` 为 `true`，并配置 `api.auth_token` 字段。客户端在请求时需要携带 `Authorization: Bearer <token>` 头。
