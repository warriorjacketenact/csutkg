# LinkPilot Resource Gateway

LinkPilot 是一个面向技术团队、内容创作者与研究者设计的轻量级外链资源汇总与导航系统。项目定位于将分散、易流失的优质外部链接纳入结构化目录，提供可检索、可分类、可审计的统一访问入口。

LinkPilot 主要解决以下问题：团队内部收藏的参考链接散落于文档、即时通讯与邮件中，难以共享与维护；重要外部资源因缺少索引而无法被有效复用；链接失效或内容迁移后缺乏追溯手段。LinkPilot 不对链接内容做二次存储或代理，仅作为元数据组织与跳转枢纽，适用于中大型企业文档中心、开源项目附属资源站、个人知识库外链模块等场景。

## 功能概览

- **多级分类目录**：支持按主题、场景、文件类型、来源域名等维度建立自定义分类树，每个链接可归属于多个标签体系。

- **批量导入与解析**：支持从 CSV、JSON、Markdown 列表批量导入 URL，自动识别协议头、域名与路径层级，提取目录深度与扩展名特征。

- **链接状态巡检**：内置 HTTP 头检测模块，可定期对已收录链接执行可达性检查，标记异常状态码、超时或证书错误，输出巡检报告。

- **元数据扩展字段**：每条链接可记录收录人、收录时间、最后访问时间、内容摘要、适用版本、所属项目阶段等自定义元数据。

- **只读镜像发布模式**：支持生成纯静态 HTML 目录页，将链接列表以只读方式发布到内部站点或 CDN，降低主服务访问压力。

- **检索与过滤接口**：提供基于路径前缀、域名、标签、更新时间范围的组合查询 API，便于与其他自动化工具集成。

- **操作日志审计**：记录链接的增删改、巡检触发、导出操作等行为，支持回溯操作人与时间，满足内部合规要求。

## 应用场景

**技术文档中心外链附录**  
技术团队在编写系统设计文档、API 参考或运维手册时，需要引用大量外部规范、标准、第三方库主页或社区讨论。LinkPilot 可作为文档平台的补充服务，统一管理这些外链，避免在每个文档中重复维护相同 URL。

**开源项目社区资源导航**  
开源项目通常会在 README 或 Wiki 中列出周边生态工具、学习材料、视频教程和插件列表。这些链接随社区发展不断更新，LinkPilot 提供结构化编辑界面，允许维护者快速增删条目，并通过自动化巡检发现失效链接，提升社区体验。

**研究资料收集与共享**  
研究团队在文献调研或竞品分析阶段，会积累大量网页、论文预印本、数据集的链接。LinkPilot 支持按研究方向、时间线、重要程度分类，团队成员可共同维护，减少重复查找时间。

**内部运营活动素材库**  
市场或运营部门在策划线上活动时，需要反复使用往期活动页面、设计参考、文案模板、数据看板等外部链接。LinkPilot 可作为一个轻量级资产索引，按活动代号或日期快速检索。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL2 或 Git Bash 执行。

```bash
git clone https://github.com/linkpilot/linkpilot-gateway.git
cd linkpilot-gateway

# 安装依赖（使用 Python 3.10+ 与 pip）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 使用示例数据初始化本地存储
cp config/example.yaml config/local.yaml
python scripts/init_db.py --config config/local.yaml

# 启动开发服务，默认监听 127.0.0.1:8080
python app.py --port 8080 --debug
```

启动成功后，访问 http://127.0.0.1:8080 可查看示例目录界面。如需导入用户提供的初始链接列表，可使用 `scripts/import_urls.py` 并指定数据文件路径。

## 项目结构

```
linkpilot-gateway/
├── app.py                     # 主入口，初始化应用与路由注册
├── config/
│   ├── default.yaml           # 默认配置（日志级别、巡检间隔、静态输出目录）
│   ├── example.yaml           # 示例完整配置（含分类预设与标签映射）
│   └── local.yaml             # 本地覆盖配置（gitignore，用于开发调试）
├── core/
│   ├── __init__.py
│   ├── models.py              # 数据模型定义（Link, Tag, CheckRecord, AuditLog）
│   ├── storage.py             # SQLite 存储层接口（CRUD、批量操作、查询构造）
│   ├── checker.py             # 异步链接检查器（aiohttp 会话管理、重试策略）
│   └── exporter.py            # 静态镜像导出器（jinja2 渲染、目录生成）
├── api/
│   ├── __init__.py
│   ├── routes_v1.py           # RESTful API 路由（列表、详情、批量导入、巡检触发）
```

## 常见问题

**Q：LinkPilot 是否会缓存或代理外部链接的内容？**  
A：不会。LinkPilot 仅存储 URL 及其元数据，所有访问跳转由客户端直接请求原始地址。巡检功能仅发送 HEAD 请求检查状态码，不下载响应体内容。用户应确保所收录的链接遵守目标网站的使用条款。

**Q：如何迁移已有的大量链接数据到 LinkPilot？**  
A：项目提供了 `scripts/import_urls.py` 脚本，支持纯文本每行一个 URL 的文件格式，也支持包含标题、标签、备注列的 CSV 文件。迁移前建议先使用 `--dry-run` 参数预览解析结果。若需要从浏览器书签或 Notion 导出，可先转换为 CSV 格式再执行导入。

**Q：静态镜像导出与主服务的关系是什么？**  
A：静态镜像导出功能会将当前数据库中的链接列表渲染为独立的 HTML 文件集合，可部署到任何静态托管服务。导出为只读快照，不包含动态检索或巡检功能，适合作为备份或对外公开的简化版本。主服务停止时静态镜像不受影响。
