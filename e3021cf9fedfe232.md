# ResourceHub

ResourceHub 是一个面向开发者与技术研究者的高质量技术资源导航与聚合平台。本项目并非一个传统的软件框架或工具库，而是一个精心编排的、可快速检索的外部技术资源索引系统。它旨在解决技术从业者在信息过载时代面临的核心痛点：无法高效定位到与当前研发任务高度相关的权威文档、实战案例与社区讨论。

ResourceHub 的目标用户包括全栈工程师、运维工程师、架构师以及计算机科学领域的研究者。通过结构化的分类体系与轻量级的本地检索服务，本项目将分散在互联网各处的优质内容（涵盖编程语言规范、云原生实践、数据库调优、前端工程化等数十个细分领域）整合为统一的访问入口。用户既可以通过本项目提供的静态站点进行浏览，也可以基于我们开源的爬虫与索引框架，构建属于自己的私有技术知识库。

## 功能概览

**多维度资源分类体系** 所有收录资源均按照技术栈、应用场景与知识层级进行三级标签标注，支持按语言（Go、Rust、Python、Java）、领域（存储、网络、可观测性）与成熟度（孵化、稳定、已归档）快速筛选。

**轻量级全文检索引擎** 基于 Bleve 实现的离线索引构建工具，可为所有资源标题与摘要生成倒排索引，支持布尔查询与短语匹配，检索响应时间控制在 200 毫秒以内。

**资源健康度监控** 内置周期性链接可用性检查模块，自动标记响应超时或状态码异常的条目，并生成可视化报告，确保索引库的长期有效性。

**自定义标签与注解系统** 允许用户为每个资源条目添加私有备注、自定义标签以及阅读优先级标记，所有元数据以 YAML 格式存储于本地，便于版本控制与团队共享。

**批量导入与导出接口** 提供 JSON 与 CSV 格式的批量数据导入导出功能，支持从浏览器书签、Pinboard、Instapaper 等第三方服务迁移数据。

**静态站点生成器** 集成 Hugo 主题引擎，可一键将资源索引渲染为响应式静态 HTML 页面，适合部署至 GitHub Pages、Cloudflare Pages 或任意对象存储服务。

## 应用场景

个人技术知识库的构建与维护 开发者可以使用 ResourceHub 作为每日阅读清单的管理工具，将分散在个人浏览器收藏夹、笔记软件中的数百个技术链接统一导入，并通过标签系统构建属于自己的技术知识体系，定期回顾与整理。

团队技术文档中心的外延补充 技术团队管理者可以利用本项目的分类与注解功能，将团队内部 Wiki 中缺失的外部参考文档（如特定中间件的性能调优指南、安全漏洞修复提案）进行结构化收录，形成内部开发者门户的附属资源层。

技术选型与调研阶段的信息收集 在进行中间件选型或框架升级时，架构师可通过 ResourceHub 的标签检索快速定位到多个候选方案的真实生产环境案例、压测报告以及社区活跃度数据，大幅缩短调研周期。

离线环境下的文档镜像索引 对于网络受限的内网开发环境，运维人员可借助本项目的导出接口生成资源清单，配合外部下载工具批量获取文档镜像，并在内网搭建只读镜像站点，保障研发环境文档可得性。

## 快速开始

以下步骤将指导您在本地环境中克隆代码仓库、安装依赖并启动 ResourceHub 的索引服务与 Web 界面。

```bash
# 克隆项目仓库至本地
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 安装项目依赖（使用 Go Modules 管理）
go mod download

# 构建二进制文件并初始化索引库
make build
./bin/resourcehub init --config ./configs/default.yaml

# 启动 Web 服务，默认监听 8080 端口
./bin/resourcehub serve --port 8080
```

执行上述命令后，打开浏览器访问 http://127.0.0.1:8080 即可进入 ResourceHub 的检索界面。首次启动时，系统会自动根据 `./data/sources` 目录下的样例数据构建索引。

## 安装要求

本项目基于 Go 语言开发，推荐在 Linux 或 macOS 环境下运行。生产环境部署时建议配合 PostgreSQL 作为元数据存储后端，开发测试阶段可使用内置的 BoltDB 文件数据库。

| 依赖组件 | 必需性 | 说明 |
|---|---|---|
| Go 1.21 及以上版本 | 必需 | 项目编译与运行的基础运行时环境，需配置正确的 GOPATH 与 GOROOT |
| Git 2.25 及以上版本 | 必需 | 用于克隆仓库及在开发模式下拉取依赖的私有模块 |
| Make 构建工具 | 必需 | 用于执行项目定义的编译、测试与打包任务脚本 |
| PostgreSQL 14 及以上 | 可选 | 生产环境推荐使用，提供事务支持与更优的并发查询性能 |
| Redis 6.2 及以上 | 可选 | 用于缓存热点资源元数据，降低数据库查询压力 |
| Node.js 18 及以上 | 可选 | 仅当需要二次开发前端静态站点主题时才需安装 |
| Docker 20.10 及以上 | 可选 | 用于容器化部署，可通过 Compose 一键启动全套依赖服务 |
| 网络连接 | 必需 | 首次启动时需访问外部资源以校验链接有效性，内网环境需配置代理 |

## 常见问题

**问：项目启动时提示索引目录不存在或权限不足，应如何处理？**

答：首次运行 `resourcehub serve` 命令时，系统会在当前工作目录下创建 `.index` 隐藏文件夹用于存放倒排索引文件。请确保运行进程拥有当前目录的读写权限。若需更改索引存储位置，可在配置文件中修改 `index.path` 字段。此外，可以手动执行 `resourcehub init --force` 强制重建索引目录结构。

**问：如何批量导入我自己的浏览器书签或 Pinboard 数据？**

答：ResourceHub 支持通过 JSON 格式导入外部数据。您可以将浏览器导出的书签 HTML 文件使用社区提供的转换脚本（位于 `scripts/convert_bookmarks.py`）转为标准 JSON 格式，然后执行 `resourcehub import --file ./bookmarks.json` 命令完成导入。对于 Pinboard 用户，可直接使用其官方 JSON 导出 API 获取数据，无需额外转换。

**问：内网部署时无法访问外部资源链接，是否会影响本地服务启动？**

答：ResourceHub 的健康度监控功能默认采用异步非阻塞模式执行，即使在网络隔离环境下，Web 服务依然可以正常启动并响应检索请求。您可以在配置文件中将 `crawler.enabled` 字段设置为 `false` 以完全关闭外网探测，或设置 `crawler.timeout` 为更短的值以减少启动等待时间。所有资源链接仅作为元数据存储，不会在检索时强制校验可达性。
