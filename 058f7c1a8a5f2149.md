# EON Resource Aggregator

EON Resource Aggregator 是一个面向技术研究者、开源贡献者与信息分析人员的轻量级外链资源归集与导航系统。该项目不对资源内容进行二次编辑或快照存储，仅提供结构化的访问入口与基础元数据描述，旨在帮助用户快速定位分散于网络各处的技术文档、数据工具、社区讨论与参考实现。

本项目适用于需要维护大量外部链接的个人知识库管理员、技术团队内部文档维护者，以及希望以公开方式共享资源导航的开源项目维护者。通过提供统一的访问视图和分类索引，EON Resource Aggregator 简化了外链管理流程，降低了资源失效率，并使得大规模链接集合的维护工作变得可追溯、可扩展。

## 功能概览

**结构化链接归集**：支持以树形目录和标签系统对大量外链进行归类，并提供批量导入与导出接口。

**原始 URL 透传**：所有资源链接均按用户提供的原始形式原样输出，不进行协议补全、大小写修改或路径规范化，确保访问路径的确定性。

**元数据标注**：每条链接可附加人工维护的简短说明、关键词标签与状态标记（如有效、失效、需审核）。

**依赖环境检测**：启动时自动检查运行环境中的必需依赖版本，并以表格形式输出检测报告。

**静态站点生成**：内置模板引擎可将链接集合渲染为静态 HTML 站点，便于通过任何 Web 服务器托管。

**自动化健康检查**：支持定时对已收录链接进行 HTTP 状态检查，标记异常链接并生成报告。

**访问统计接口**：提供基于日志的访问次数统计，帮助维护者了解高频使用的资源。

## 应用场景

**技术团队内部文档中心**：开发团队可使用 EON Resource Aggregator 统一管理项目依赖的第三方库文档、API 参考、设计规范等外链，确保团队成员快速获取权威资料。

**开源项目外部资源导航**：开源项目维护者可利用本项目整理社区贡献的教程、视频、博客文章链接，作为项目官方文档的补充扩展。

**个人研究知识库**：研究人员可将日常积累的论文链接、数据集地址、工具站收录至系统，通过分类和标签进行高效检索与回溯。

**教育机构课程参考**：教师可汇总课程相关的在线阅读材料、编程练习平台、视频公开课链接，为学生提供集中入口。

## 快速开始

以下命令演示如何从源码获取并启动 EON Resource Aggregator 实例。

```bash
# 克隆仓库
git clone https://github.com/eon-resource/eon-aggregator.git

# 进入项目目录
cd eon-aggregator

# 安装依赖
pip install -r requirements.txt

# 使用默认配置启动服务
python run.py --port 8080
```

启动成功后，可通过浏览器访问 http://localhost:8080 查看资源列表首页。如需导入用户提供的原始链接数据，请将链接列表保存为 CSV 文件并放置于 `data/imports/` 目录，系统将在下次启动时自动加载。

## 项目结构

```
eon-aggregator/
├── run.py                       # 服务入口，解析命令行参数并启动应用
├── requirements.txt             # Python 依赖列表，包含 Flask、requests 等
├── config/
│   ├── default.yaml             # 默认配置，含端口、日志级别、数据库路径
│   └── production.yaml          # 生产环境覆盖配置
├── app/
│   ├── __init__.py              # 应用工厂函数，注册蓝图和扩展
│   ├── routes/
│   │   ├── index.py             # 首页路由，展示资源分类概览
│   │   ├── view.py              # 单个资源详情页路由
│   │   └── admin.py             # 管理后台路由，用于增删改资源
│   ├── models/
│   │   ├── resource.py          # 资源条目数据模型，定义字段与验证规则
│   │   └── category.py          # 分类模型，支持层级标签
│   ├── services/
│   │   ├── importer.py          # CSV/JSON 导入服务，处理批量链接添加

│   └── templates/
│       ├── base.html            # 基础模板，包含公共样式和脚本
│       ├── index.html           # 首页视图，展示分类和最近添加
│       └── detail.html          # 详情页视图，展示单条资源信息
├── data/
│   ├── resources.db             # SQLite 主数据库文件
│   ├── imports/                 # 存放待导入的 CSV 文件
│   └── exports/                 # 导出数据或静态站点的输出目录
├── tests/
│   ├── test_models.py           # 数据模型单元测试

│   └── test_services.py         # 服务层功能测试

└── docs/                        # 文档目录，包含用户手册和开发指南
    ├── user-guide/
    ├── admin-guide/
    └── developer-guide/
```

## 常见问题

**问：系统如何处理外部链接失效问题？**

系统内置了定时健康检查服务，默认每 24 小时对所有已收录链接发起 HEAD 请求。检查结果会记录在数据库中，并通过管理后台的异常链接列表展示。维护者可手动标记失效链接为不可用状态，或通过导出功能生成失效报告。健康检查服务不跟随重定向，仅记录初始 URL 的状态码，以便区分临时跳转与永久失效。

**问：如何迁移已有链接数据至新环境？**

迁移数据需执行以下步骤：首先在源环境中使用导出命令生成 JSON 格式的数据快照，命令为 `python run.py --export data/exports/snapshot.json`。然后将快照文件复制至目标环境的 `data/imports/` 目录，启动目标服务时系统会自动检测并导入。导入过程会检查 URL 的唯一性，重复记录将跳过并记录日志。若数据库结构存在版本差异，请先运行 `scripts/migrate.py` 升级至当前版本。

**问：静态站点生成模式与动态服务模式有何区别？**

动态服务模式（默认）通过 Flask 提供实时 Web 界面，支持在线管理、搜索和统计功能，适合日常维护和使用。静态站点生成模式通过 `python run.py --build` 命令触发，系统会将当前所有资源数据渲染为纯 HTML 文件输出至 `data/exports/` 目录，该模式适合将资源列表部署至无后端环境的托管平台（如 GitHub Pages 或对象存储服务）。两种模式共享相同的数据源和模板，但静态模式下管理功能不可用。
