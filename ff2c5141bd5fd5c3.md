# EON Resource Navigator

EON Resource Navigator 是一个面向技术研究人员、开源贡献者以及信息分析人员的结构化外链资源汇总系统。该项目不直接存储任何内容实体，而是通过人工筛选与自动化校验相结合的方式，对特定领域的高价值信息节点进行索引、分类与维护。项目定位为垂直领域的“信息路由表”，旨在帮助用户从海量噪声中快速定位具备参考意义的公开页面，降低信息获取的边际成本。

本项目服务于两类核心用户：一是需要持续跟踪特定主题动态的深度阅读者，二是希望基于现有外链网络构建自身知识图谱的开发人员。通过提供高度规整的 URL 集合与元数据描述，EON Resource Navigator 能够作为数据源被集成至 RSS 阅读器、内容聚合工具或自动化调研脚本中。项目本身不依赖外部数据库，所有资源索引均以纯文本形式维护于仓库内，确保最大程度的可移植性与可审计性。

## 功能概览

- **按主题分类的外链索引**：所有收录的 URL 均按照技术领域、应用场景或内容形态进行逻辑分组，用户可在不同类别间快速切换，避免在无序的超链接海洋中漫游。

- **基于视图路径的资源定位**：每个资源条目通过唯一的 view 路径进行标识，路径命名遵循可读性规范，便于用户从字符串本身推测资源的大致内容方向。

- **静态外链集合的版本化管理**：资源列表随项目迭代进行增删改，每次变更均通过 Git 提交记录留痕，用户可追溯任一历史版本的资源构成。

- **多维度元数据标注**：除 URL 本身外，项目维护了与每个资源相关的标签、简要说明以及关联场景标注，为二次过滤提供数据基础。

- **低依赖的运行环境**：项目核心逻辑仅依赖标准 Python 环境与常见文本处理工具，无需额外安装大型框架，可在各类轻量级服务器或本地设备上运行。

- **自动化链接可用性检查**：内置基于 HTTP 状态码的存活检测脚本，可定期输出失效链接报告，辅助维护者及时更新或剔除无效条目。

- **可嵌入的外部数据源支持**：资源列表以结构化格式（JSON / YAML）输出，可被外部应用直接消费，适用于构建自定义导航页、浏览器起始页或垂直搜索引擎。

- **社区驱动的更新机制**：用户可通过 issue 或 pull request 提交新增资源推荐或失效反馈，项目维护者定期审核并合并高价值贡献。

## 应用场景

- **技术文献的快速参照**：研究人员在撰写技术方案或学术论文时，可将本项目作为参考文献的外部链接池，快速获取与当前主题相关的背景资料或案例页面，缩短文献检索周期。

- **自动化信息采集的任务配置**：数据采集工程师可将本项目的资源列表作为爬虫种子 URL 的来源，通过过滤特定视图路径下的链接，构建聚焦于特定领域的采集任务，避免爬虫遍历无关域名。

- **个人知识库的补充素材**：知识管理爱好者可使用本项目的 URL 集合作为个人 Wiki 或数字花园的外部引用来源，在整理笔记时直接从索引中选取可信链接，提升知识库的引用质量。

- **开源项目文档的友情链接池**：开源项目的维护者可将本项目收录的同类资源作为“相关项目”或“延伸阅读”章节的素材，丰富自身项目文档的向外引用层次，帮助用户获得更完整的生态视角。

- **企业竞品信息的动态监测**：产品经理或市场分析人员可基于本项目的特定分类子集，配置周期性访问脚本，监测竞品官方页面或行业动态页面的更新频率与内容变化趋势。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，确保系统已安装 Git 与 Python 3.8 及以上版本。

```bash
# 克隆项目仓库至本地
git clone https://github.com/eon-resource/eon-navigator.git
cd eon-navigator

# 安装项目依赖（使用 pip 安装解析与校验工具）
pip install -r requirements.txt

# 运行资源列表校验脚本，检查所有 URL 的格式合规性与基础可访问性
python scripts/validate_links.py --source data/links.json --timeout 3
```

执行完毕后，控制台将输出校验报告，包括总链接数、通过数、失败数以及失败链接的详细状态码。若需生成当前资源的 HTML 导航页面，可执行 `python scripts/build_nav.py --output dist/index.html`。

## 项目结构

```
eon-navigator/
├── data/
│   ├── links.json               # 主资源索引，包含所有 URL 与元数据
│   ├── categories.yaml          # 分类体系定义，映射视图路径至主题标签
│   └── schema/
│       └── link-schema.json     # JSON Schema 用于校验 links.json 结构
├── scripts/
│   ├── validate_links.py        # 校验 URL 格式与 HTTP 存活状态
│   ├── build_nav.py             # 从 links.json 生成静态 HTML 导航页
│   ├── export_csv.py            # 将资源列表导出为 CSV 格式，便于外部处理
│   └── utils/
│       ├── fetcher.py           # 封装 requests 的并发请求工具
│       └── logger.py            # 统一日志输出格式与级别控制
├── docs/
│   ├── user-guide.md            # 面向最终用户的浏览与检索说明
│   ├── maintainer-guide.md      # 面向维护者的分类原则与更新流程
│   ├── api-reference.md         # 脚本接口的函数签名与返回值说明
│   └── contributing.md          # 贡献者行为准则与提交规范
├── tests/
│   ├── test_validate.py         # 单元测试：校验函数的边界条件
│   ├── test_build.py            # 单元测试：构建输出的 HTML 结构完整性
│   └── fixtures/
```

## 贡献指南

1. 提交资源推荐：在 GitHub Issues 中使用“资源推荐”模板，填写目标 URL、推荐分类以及 50 字以内的内容摘要。提交前请确认该 URL 未被本项目收录，且页面内容符合项目所涵盖的技术或信息分析范畴。

2. 报告失效链接：若在浏览或使用过程中发现某链接返回 4xx 或 5xx 状态码，请在 Issues 中选择“链接失效”模板，附上视图路径、检测时间以及复现步骤。维护团队将在 5 个工作日内核实并处理。

3. 参与分类体系优化：若认为现有分类标签无法准确描述某组资源，可 fork 本仓库，修改 data/categories.yaml 文件中的分类定义，并提交 pull request。提交时请确保新增或修改的分类具有明确的适用范围说明，避免标签之间的语义重叠。

4. 改进脚本功能：欢迎针对 validate_links.py、build_nav.py 等核心脚本提交性能优化或功能增强的 pull request。代码需通过项目已有的单元测试，并在 tests/ 目录下为新增逻辑补充对应的测试用例。

5. 完善文档：文档章节若存在表述不清或信息缺失，可直接在 docs/ 目录下修改对应的 Markdown 文件并提交 pull request。文档变更应保持与技术实现的一致性，避免引入过时或错误的操作说明。

## 常见问题

**问：项目是否会对收录的 URL 进行内容审核或安全过滤？**

答：项目维护者在收录资源时会进行初步的人工访问，确认页面内容与所标注分类大致相符，且不包含明显的恶意代码或违规信息。但由于本项目仅为外链索引，不代理或缓存任何外部内容，因此无法对目标页面的实时安全性提供绝对保证。建议用户在访问外部链接时启用浏览器安全插件，并保持系统防火墙处于开启状态。

**问：如果我希望定期同步最新的资源列表，有哪些自动化方式？**

答：您可以通过 Git 定期拉取仓库更新来获取最新的 links.json 文件。对于需要程序化消费的场景，建议直接使用 GitHub 的 raw 文件访问链接，例如 https://raw.githubusercontent.com/eon-resource/eon-navigator/main/data/links.json。此外，项目发布的每个版本均会附带一个稳定的资源快照，您可在 Releases 页面下载对应版本的 JSON 存档。

**问：项目是否接受来自非技术领域的资源链接，例如人文或艺术类内容？**

答：当前项目的主要定位为技术信息与数据分析相关资源。若您推荐的内容在方法论、工具链或数据可视化等方面与技术研究存在交叉，可提交至“跨界参考”分类供维护团队评估。纯粹的人文或艺术类页面暂时不在收录范围内，建议您将其提交至更匹配的垂直导航项目。
