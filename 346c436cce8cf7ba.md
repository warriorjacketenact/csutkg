# VueBest 技术资源导航

VueBest 是一个面向 Vue.js 生态系统的技术资源聚合与导航平台，专注于收录高质量的开源项目、组件库、工具链以及学习资料。本项目旨在解决开发者在海量信息中筛选优质资源的时间成本问题，通过人工筛选与社区投票机制，构建一个高信噪比的技术资源索引系统。

本项目适用于 Vue 技术栈的架构师、前端工程师、技术决策者以及开源项目维护者。无论您是在寻找生产级 UI 组件库、状态管理方案、性能优化工具，还是希望了解社区最佳实践，VueBest 均可提供结构化、可检索的资源导航服务。项目采用静态站点生成技术，确保页面加载性能与 SEO 友好度，同时支持资源标签过滤、版本追踪与维护状态标注。

## 功能概览

**资源分类索引**：按照 UI 框架、工具库、插件、学习资源等维度对每个收录项目进行标签化分类，支持多维度筛选与排序。

**维护状态监控**：自动追踪每个资源的 GitHub Star 数量、最近提交时间、Issue 响应速度，以可视化徽章形式展示项目健康度。

**社区评分与评论**：注册用户可为资源打分并提交使用体验，评分加权计算后影响资源在列表中的默认排序位置。

**个性化收藏夹**：用户可创建自定义收藏列表，便于团队内部共享推荐技术栈，支持收藏夹的导入与导出为 JSON 格式。

**版本兼容性矩阵**：记录每个资源对 Vue 2.x / 3.x 以及主流构建工具（Vite、Webpack）的兼容性信息，帮助开发者快速评估集成风险。

**RSS 订阅源**：提供按分类或标签的 RSS 订阅功能，当收录资源更新版本或新增同类资源时，订阅者将收到通知。

**暗色主题与阅读模式**：内置浅色与暗色两套主题，阅读模式可去除侧边栏与广告，聚焦于资源详情内容。

**批量导入与 API 接口**：支持通过 CSV 或 JSON 文件批量导入资源链接，同时提供 RESTful API 供第三方工具查询资源数据。

## 应用场景

技术选型调研：技术负责人或架构师在进行技术选型时，可通过 VueBest 快速对比同类 UI 组件库或状态管理方案，查看各项目的维护活跃度、社区反馈与版本兼容性信息，辅助决策。

日常开发速查：一线开发者在实现特定功能（如表格编辑、文件上传、图表可视化）时，可通过分类筛选快速找到经过验证的第三方库，并直接访问其文档站点与示例代码。

开源项目发现：开源贡献者或技术爱好者可通过浏览 VueBest 的新增资源列表，发现 Vue 生态中新兴的、有潜力的项目，并参与贡献或提供反馈。

团队知识沉淀：技术团队可基于 VueBest 的收藏夹功能，建立内部推荐技术库清单，统一团队成员的技术选型偏好，减少重复评估成本。

## 快速开始

以下命令可在本地环境完成 VueBest 项目的克隆、依赖安装与开发服务器启动。

```bash
git clone https://github.com/vuebest/vuebest-resource-hub.git
cd vuebest-resource-hub
npm install
npm run dev
```

执行完毕后，访问控制台输出的本地地址（通常为 http://localhost:5173 ）即可浏览资源导航站点。生产环境构建使用 npm run build 命令，输出目录为 dist。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.0.0 或更高 | 是 | 运行构建工具与开发服务器，推荐使用 LTS 版本 |
| npm 9.0.0 或更高 | 是 | 包管理器，用于安装项目依赖与执行脚本 |
| Git 2.30.0 或更高 | 是 | 克隆仓库与版本控制，部分资源更新依赖 Git 操作 |
| SQLite 3 | 否 | 本地开发环境下用于存储用户评分与收藏数据，生产环境可替换为 PostgreSQL |
| Redis 7.0 | 否 | 可选缓存层，用于提升 API 响应速度与 Session 存储 |
| Docker 24.0 | 否 | 用于容器化部署，项目根目录提供 Dockerfile 与 compose.yaml |

## 贡献指南

我们欢迎社区贡献者参与 VueBest 项目的建设。请按照以下步骤提交您的贡献。

第一，Fork 本仓库并在本地克隆您的 Fork 版本，创建以 feature/ 或 fix/ 为前缀的分支进行开发。确保分支名称简洁描述所解决的问题或新增的功能。

第二，在提交 Pull Request 之前，请运行 npm run lint 与 npm run test 确保代码风格符合 ESLint 配置且所有单元测试通过。新增功能需附带对应的测试用例。

第三，若您希望新增资源收录，请勿直接修改资源数据文件，而是通过站点前台的提交表单提交，管理员会在审核后合并。若希望修复文档错误或完善翻译，请直接在 /docs 目录下修改对应的 Markdown 文件。

第四，提交 Pull Request 时请使用项目提供的 PR 模板，清晰描述变更内容、测试覆盖情况以及是否影响现有 API 或数据结构。所有 PR 需要至少两名维护者审核后方可合并。

## 常见问题

问：如何报告收录资源中的错误信息（如过时的 GitHub 链接或错误的版本兼容性标注）？

答：您可以通过两种方式报告。第一，在资源详情页下方点击"报告问题"按钮，填写具体错误描述，系统会自动创建 Issue 并通知维护者。第二，直接在本项目的 GitHub Issues 页面提交问题，并使用 "resource-error" 标签进行分类。我们会在 48 小时内核实并修正。

问：我提交的资源收录请求为什么没有被通过？

答：资源收录请求被拒绝通常有以下原因。资源未提供明确的开源许可证或源码仓库地址。资源与已有收录项目功能高度重复且无显著差异。资源最后一次更新超过 24 个月且未维护。资源文档严重缺失或主要语言非英文且无英文翻译。您可以在提交记录中查看具体拒绝理由，并根据建议修改后重新提交。

问：VueBest 的资源排序规则是什么？如何影响默认列表顺序？

答：默认列表排序采用综合评分算法，权重分配如下。社区评分（1-5 星）占总权重的 40%。GitHub Star 数量经对数归一化后占总权重的 30%。资源最近更新时间占总权重的 20%。收录时长占总权重的 10%。用户可以通过顶部的排序下拉菜单切换为按最新收录、按评分最高或按名称字母顺序排列。
