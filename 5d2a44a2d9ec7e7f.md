# Resource Atlas

Resource Atlas is a community-driven technical resource aggregation and navigation platform designed to organize, catalog, and index high-quality external technical references, articles, tools, and documentation across multiple domains. It serves developers, researchers, and technical writers who need a centralized, well-structured entry point to discover relevant content without wading through search engine noise.

Unlike traditional bookmark managers or simple link lists, Resource Atlas provides a curation layer with metadata extraction, tag-based classification, and full-text search capabilities. The platform automatically pulls title, description, and content type from submitted URLs, enabling users to filter and locate specific resources through a unified query interface. The project targets individuals and teams who maintain large external reference collections and require systematic organization rather than ad-hoc bookmarking.

## 功能概览

**Automated Metadata Harvesting** – Each submitted URL undergoes automatic extraction of page title, meta description, approximate word count, and content category (tutorial, documentation, blog post, specification, etc.) using rule-based parsers and fallback heuristics.

**Tag-Based Classification System** – Resources are tagged with both automatic and user-assigned labels covering technology stack (e.g., Go, React, Kubernetes), content type, difficulty level, and domain area. Tags are normalized to maintain consistency.

**Full-Text Search Engine** – All indexed resources are searchable by title, description, tags, and extracted plain-text content. Search supports boolean operators, phrase matching, and field-specific filters.

**Batch Import and Export** – Resources can be imported in bulk via CSV, JSON, or plain URL lists. Export functionality supports Markdown, JSON, and HTML bookmark formats for integration with other tools.

**Curation Quality Scoring** – Each resource receives a quality score based on link age, domain authority heuristics, content depth signals (length, structure), and community feedback, allowing users to prioritize high-value references.

**Watchlist and Change Detection** – Users can add resources to watchlists; the system periodically checks for content updates and notifies subscribers when significant changes are detected.

## 应用场景

**Technical Documentation Aggregation** – A team maintaining internal developer documentation can use Resource Atlas to aggregate and organize external references, API specifications, and third-party tutorials relevant to their technology stack. Instead of scattering links across wikis or README files, all resources are centralized, searchable, and version-aware.

**Research Literature Indexing** – Graduate students and academic researchers working across multiple disciplines can catalog papers, technical reports, and code repositories related to their field. The tag system supports cross-domain discovery, while quality scoring helps filter preliminary sources from peer-reviewed materials.

**Open Source Project Resource Pages** – Open source maintainers can generate dynamic resource pages for their project documentation, linking to related tools, deployment guides, troubleshooting articles, and community tutorials. The export feature enables static site generation for projects that prefer minimal runtime dependencies.

**Competitive Intelligence Monitoring** – Product and engineering teams tracking industry trends can maintain watchlists of competitor blogs, technology announcements, and specification drafts. Change detection alerts keep the team informed without manual re-visits.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/resource-atlas/resource-atlas.git
cd resource-atlas

# Install dependencies using Poetry (Python 3.10+)
poetry install --no-dev

# Set up environment variables
cp .env.example .env
# Edit .env with your database and search engine configuration

# Initialize database schema and run migrations
poetry run atlas migrate

# Start the development server
poetry run atlas serve --host 0.0.0.0 --port 8080
```

The server will be available at http://localhost:8080. For production deployment, refer to the deployment guide in the docs/ directory.

## 常见问题

**Q: How does Resource Atlas handle broken or unavailable URLs during harvesting?**

The harvest subsystem implements exponential backoff retry with configurable attempts (default 3 retries). If a URL remains unavailable after retries, the resource is marked with a "broken" status and flagged for manual review. The system does not automatically delete broken resources; instead, administrators receive periodic reports to review and remove or update stale entries.

**Q: Can I run Resource Atlas without Elasticsearch for small-scale deployments?**

Yes, a SQLite-backed fallback search mode is available for development and very small deployments (under 1000 resources). To enable this, set SEARCH_BACKEND=sqlite in the environment configuration. Note that full-text search capabilities are reduced; phrase matching and relevance scoring are not supported in the fallback mode. For production use cases with more than 1000 resources, Elasticsearch is strongly recommended.

**Q: How are duplicate URLs handled during import?**

Resource Atlas performs a normalized URL check (removing trailing slashes, converting to lowercase, and stripping fragments) before adding any new resource. If a duplicate is detected, the import process skips the entry and logs a warning. Users can force overwrite with the --overwrite flag during batch import, which updates the existing record's title, description, and tags rather than creating a new entry.
