# LinkPilot Resource Aggregator

LinkPilot 是一个面向技术研究、数据分析与数字取证领域的结构化外部资源聚合平台。项目本身不托管任何实际内容，而是以高度组织化的索引方式，收集并分类来自公开网络的技术文档、案例数据、分析报告与实验环境参考链接，帮助安全研究员、渗透测试工程师、威胁情报分析师以及网络运维人员快速定位所需的外部信息资产。

项目定位为“技术外链的导航中枢”，解决研究人员在大量零散书签中低效查找、重复筛选以及链接失效难以及时发现等问题。通过集中的索引管理与分类标注，LinkPilot 使得团队或个人能够以可维护、可扩展的方式管理其外部知识体系，同时为后续的自动化链路可用性检测与分类模型训练提供基础数据支撑。

## 功能概览

**结构化索引目录**：基于资源类型、应用领域与使用频次，对收录的上千条外部链接进行多级分类，支持快速筛选与批量导出。

**链接生命周期追踪**：记录每条资源的添加时间、最后验证时间与状态标记，便于定期清理失效链接，维持索引库的健康度。

**标签与全文检索**：为每条记录附加多维标签（如 #threat-intel、#malware-sample、#PCAP、#OSINT），支持基于标签组合的精确检索与基于描述字段的模糊匹配。

**批量导入与导出**：支持从常见书签格式（HTML、JSON）批量导入现有链接库，并支持按分类或标签导出为 Markdown、CSV 或结构化 JSON 格式，用于外部工具集成。

**自定义元数据扩展**：允许用户为每条链接添加自定义键值对元数据，例如关联的 CVE 编号、APT 组织名称、地理区域或时间窗口，以满足专业领域的高阶管理需求。

**协作与审核工作流**：内置简单的多用户权限模型，支持管理员、编辑者与只读观察员三种角色，对新增或修改的资源链接进行审核发布，保证索引库质量。

**自动化可用性检查**：后台调度任务可周期性对收录的 URL 发送 HEAD 请求，检测 HTTP 状态码变化，并生成可用性报告，辅助用户及时剔除或更新失效资源。

**开放 API 接口**：提供 RESTful API 用于查询、添加与更新链接记录，便于与其他安全自动化平台（如 SIEM、SOAR）或爬虫框架进行数据互通。

## 应用场景

**威胁情报分析团队的日常情报源管理**：分析师需要频繁访问多个威胁情报共享平台、恶意软件沙箱报告与 IoC 数据源。LinkPilot 可将这些分散的入口统一索引，并按照威胁类型、地域或时间进行标记，分析师无需记忆繁杂的 URL，即可在单一界面快速跳转至所需情报源。

**安全竞赛与培训环境搭建**：在组织红蓝对抗演练或安全技能培训时，教练需要为学员准备大量参考资料，包括漏洞利用代码库、协议分析文档、公开 PCAP 样本以及 CTF 题目环境。LinkPilot 可作为这些资源的统一分发入口，学员通过分类导航即可获得全部所需的外部参考链接，大幅降低环境准备阶段的信息传递成本。

**数字取证与事件响应（DFIR）的标准化工具链索引**：事件响应人员在不同案件中需要反复使用多种取证工具下载地址、日志分析指南以及时间线生成脚本。LinkPilot 允许团队预先建立一套标准化的 DFIR 资源清单，在应急响应启动时，团队成员可立即从索引中获取所有必需的外部资源，减少在紧急情况下的搜寻时间。

**个人研究者的知识体系归档**：独立安全研究员长期跟踪特定领域（如物联网漏洞、云安全配置）时，会积累大量散落的博客文章、演讲幻灯片、代码仓库与在线测试环境。LinkPilot 帮助研究员以结构化的方式整理这些个人收藏，并可通过元数据标注每个资源的研究阶段（如待读、已读、已复现），形成持续积累的个人知识库。

## 快速开始

以下步骤指导您在本机快速启动 LinkPilot 开发或试用环境。

```bash
# 克隆项目仓库
git clone https://github.com/linkpilot/linkpilot.git
cd linkpilot

# 安装项目依赖（基于 Python 3.10+ 与 Poetry）
poetry install --no-dev

# 复制环境变量配置文件并修改数据库连接
cp .env.example .env
# 编辑 .env 文件，设置 DATABASE_URL 与 SECRET_KEY

# 执行数据库迁移，创建初始表结构
poetry run python manage.py migrate

# 导入示例索引数据（可选）
poetry run python manage.py loaddata fixtures/initial_links.json

# 启动开发服务器
poetry run python manage.py runserver 0.0.0.0:8000
```

访问 http://localhost:8000 即可进入 LinkPilot 主界面。默认管理员账号为 admin，密码在首次启动时由初始化脚本生成并输出至控制台日志。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.10 或更高 | 核心运行时环境，类型提示与异步特性依赖 |
| PostgreSQL | 13.0 或更高 | 主要数据存储，用于链接元数据、用户与审计日志 |
| Redis | 6.0 或更高 | 缓存后端与后台任务队列 Broker（Celery 使用） |

| Node.js | 18.0 或更高 | 仅前端静态资源构建时需要（生产环境可预编译） |
| Nginx | 1.20 或更高 | 生产环境推荐反向代理服务器（非强制，可选用其他） |

## 项目结构

以下为 LinkPilot 核心代码仓库的目录组织与各模块职责说明。

```
linkpilot/
├── backend/                          # 后端服务根目录（Django 应用）
│   ├── api/                          # RESTful API 模块，提供外部查询与写入接口
│   │   ├── views/                    # 视图集，按资源、标签、用户拆分
│   │   ├── serializers/              # 数据序列化与反序列化定义
│   │   └── routers/                  # 自动路由注册
│   ├── core/                         # 核心业务逻辑，包含链接索引、元数据管理
│   │   ├── models/                   # 数据模型定义（Link, Tag, Metadata, UserProfile）
│   │   ├── services/                 # 业务服务层，处理链接生命周期与检索
│   │   └── validators/               # URL 格式、元数据合规性校验器
│   ├── scheduler/                    # 后台任务调度（Celery Beat 配置）
│   │   ├── tasks/                    # 周期性任务，如可用性检查、统计报表
│   │   └── signals/                  # 事件触发信号，如新增链接后自动解析元数据
│   ├── settings/                     # 多环境配置（开发、测试、生产）
│   │   ├── base.py
│   │   ├── development.py
│   │   └── production.py
│   └── manage.py                     # Django 管理入口
```

## 贡献指南

LinkPilot 欢迎来自社区的各种形式贡献，包括但不限于新增资源链接索引、改进前端交互体验、优化后端检索性能以及完善文档。请遵循以下步骤参与贡献。

1. 在 GitHub 上 fork 本项目仓库，并 clone 至本地开发环境。确保本机已安装 Python 3.10+、Node.js 18+ 与 PostgreSQL 13+。

2. 创建新的功能分支，分支名称应简要描述本次变更内容，例如 `feature/add-batch-import` 或 `fix/search-filter-bug`。请勿在主分支上直接修改。

3. 进行代码变更或文档更新时，请遵循项目现有的代码风格（Python 使用 Black 格式化，TypeScript 使用 ESLint + Prettier）。所有新增功能必须包含对应的单元测试或集成测试，测试覆盖率不应低于现有基线。

4. 提交前运行完整的测试套件（`poetry run pytest`）与前端构建检查（`npm run build`），确保无回归问题。提交信息请使用语义化格式，以动词开头，简明描述变更内容。

5. 向主仓库的 `develop` 分支发起 Pull Request，并在描述中关联相关 Issue（若有）。项目维护者将在 3 个工作日内进行 Review，提出修改意见或合并变更。

## 常见问题

**问：LinkPilot 是否存储外部链接指向的实际文件内容？**

答：不存储。LinkPilot 仅索引 URL 及其元数据（标题、描述、标签、添加时间等），并不代理、缓存或保存任何外部资源的内容。用户通过 LinkPilot 访问外部链接时，请求直接由浏览器向目标服务器发起，不经过 LinkPilot 服务中转。因此，各外部资源的可用性与内容合法性由其原始提供方负责。

**问：如何批量更新已有链接的分类或标签信息？**

答：LinkPilot 提供了 CSV 与 JSON 格式的批量导入更新功能。用户可在管理后台的“批量操作”模块中下载当前链接列表的模板文件，在本地编辑分类、标签或描述字段后重新上传。系统将根据链接的唯一标识（URL 或自定义 ID）进行匹配并更新对应记录。对于超过 1000 条记录的批量操作，建议使用后台异步任务模式，避免浏览器超时。

**问：部署 LinkPilot 生产环境的最低资源要求是什么？**

答：建议最低配置为 2 核 CPU、4 GB 内存、20 GB 可用磁盘空间。数据库服务（PostgreSQL）与缓存服务（Redis）可部署在同一节点或分离节点。若索引链接数量超过 5 万条，建议将 PostgreSQL 独立部署并配置至少 8 GB 内存以优化查询性能。前端静态资源可通过 Nginx 单独提供，以减轻应用服务压力。
