# LinkHub

LinkHub 是一个面向开发者和技术研究人员的结构化外链资源聚合平台。本项目不生产内容，而是通过人工筛选与自动化校验相结合的方式，对互联网上分散的高质量技术文章、工具文档、社区讨论与深度教程进行系统性归集与索引。项目定位为技术决策支持与知识发现工具，帮助用户在信息过载的环境中快速定位到与自身工作相关的可靠外部资源。

本项目服务于架构师、运维工程师、后端开发者、技术经理以及任何需要查阅技术参考材料或探索解决方案的从业者。LinkHub 通过统一的条目格式、标签体系与定期健康检查，解决书签冗杂、链接失效、检索成本高等常见痛点。项目本身不存储外部内容，仅保留元数据与跳转地址，所有资源版权归原作者所有。

## 功能概览

**自动链接健康检查** 对收录的每一条外部链接进行周期性的可达性检测，标记失效或重定向条目，并提供状态仪表板供维护者查阅。

**多维度标签分类** 为每个资源条目附加技术栈、难度等级、内容类型等标签，支持组合筛选与快速过滤。

**全文元数据检索** 基于标题、描述、标签与来源域名的轻量级检索能力，无需打开外部页面即可判断资源相关性。

**导入与导出兼容** 支持将资源列表导出为 JSON、CSV 及标准浏览器书签 HTML 格式，便于迁移至其他工具或离线备份。

**社区反馈机制** 提供资源推荐、错误报告与评分接口，允许用户参与资源库的完善与质量把控。

**访问统计与热度排序** 记录外部链接的点击次数与最近访问时间，支持按热度或更新时间对资源列表进行排序。

## 应用场景

**技术选型调研** 当团队需要评估不同的开源数据库或消息队列时，可通过 LinkHub 快速检索已收录的对比评测文章、官方文档入口及社区讨论帖，汇总各方观点辅助决策。

**故障排查与应急参考** 运维人员在处理线上异常时，可通过标签筛选快速定位到特定错误码或组件相关的故障排查指南与修复案例，缩短平均修复时间。

**新人入职学习路径规划** 团队为新成员制定成长计划时，可依据 LinkHub 中按难度与领域分类的资源列表，整理出从入门到进阶的阅读清单，减少知识碎片化问题。

**技术方案文档撰写辅助** 技术负责人在编写系统设计文档或架构评审材料时，可引用 LinkHub 中收录的权威参考资料作为论据支撑，提升文档的专业性与可追溯性。

## 快速开始

以下步骤帮助您在本地环境中快速启动 LinkHub 服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkhub/linkhub.git

# 进入项目根目录并安装依赖
cd linkhub
pip install -r requirements.txt

# 初始化本地数据库并启动开发服务器
python manage.py migrate
python manage.py runserver
```

启动成功后，访问本地 8000 端口即可进入 LinkHub 仪表板。默认管理员账户可通过 `python manage.py createsuperuser` 命令创建。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，用于后端服务与数据处理 |
| Django | 4.2 LTS | Web 框架，提供 ORM、路由与管理后台 |
| PostgreSQL | 14 及以上 | 主数据库，存储资源元数据与用户反馈 |
| Redis | 7.0 及以上 | 缓存与任务队列，用于链接健康检查的异步调度 |
| Celery | 5.3 及以上 | 分布式任务队列，配合 Redis 执行周期性资源校验 |
| Node.js | 18 LTS | 仅用于前端静态资源构建，非运行时必需 |
| Nginx | 1.24 及以上 | 生产环境反向代理与静态文件服务，开发环境可选 |

## 常见问题

**Q: LinkHub 是否存储外部资源的内容副本？**

A: 不存储。LinkHub 仅保存外部资源的标题、描述、标签与 URL 地址。所有内容版权归原始作者或站点所有。项目本身不承担因外部链接内容变更或不可用带来的责任，但会通过健康检查机制及时标记失效链接。

**Q: 如何批量导入现有的浏览器书签？**

A: 项目提供了 `scripts/import_bookmarks.py` 脚本，支持解析 Chrome 或 Firefox 导出的 HTML 书签文件。具体使用方法可参考 `docs/user-guide/import-export.md`。导入后，可手动为条目补充标签与分类信息。

**Q: 链接健康检查的频率是多少？是否会影响外部站点？**

A: 默认每 7 天执行一次全量检查，检查间隔可配置。检查过程仅发送单次 HEAD 请求，不下载页面内容，对目标服务器负载影响极小。若目标站点有严格的频率限制，可在环境变量中调整检查并发数与超时时间。
