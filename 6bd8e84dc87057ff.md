# LinkVault

LinkVault 是一个面向技术团队与内容研究者的结构化外链资源聚合平台。本项目并非简单的书签管理器，而是一套围绕高质量外部链接构建的元数据索引系统，旨在解决个人与团队在信息检索过程中面临的链接散落、上下文丢失与协作困难等问题。LinkVault 通过引入分类标签、抓取时间戳与快照摘要机制，将原始 URL 转化为可检索、可分享、可审计的知识资产。

本项目适用于需要长期维护大量外链引用的技术文档编写者、数据分析师、学术研究人员以及开源社区维护者。LinkVault 不提供链接内容本身的存储或代理服务，而是专注于链接的发现、组织、描述与状态监控，确保每一枚外链在其生命周期内均可被准确追溯与评估。

## 功能概览

**批量链接导入** 支持从纯文本、CSV 及常见书签导出格式中批量导入 URL 列表，自动去重并识别失效链接。

**元数据自动补全** 对导入的每一枚链接自动发起标题抓取、描述提取与关键词生成，降低手动标注成本。

**自定义分类体系** 允许用户根据业务场景创建多级分类目录，并将链接灵活归类至一个或多个分类节点下。

**链接健康检查** 定时对已收录链接发起可达性检测与状态码记录，标记重定向、超时与 404 等异常状态。

**快照摘要存储** 存储链接首次收录与每次健康检查时的页面摘要信息，包括标题变更、正文关键词变化，辅助内容变迁分析。

**协作注释系统** 支持团队成员在同一链接下添加评论、标签与重要性评分，构建集体智慧下的链接价值网络。

**开放 API 接口** 提供 RESTful 风格的查询与管理接口，允许第三方工具集成链接数据，实现自动化工作流。

**全文检索与过滤** 基于链接标题、描述、分类与注释内容构建倒排索引，支持多条件组合筛选与排序。

## 应用场景

技术文档团队维护外部参考索引
技术文档编写过程中需引用大量外部规范、标准与第三方库文档。LinkVault 可帮助团队统一管理这些引用链接，并在文档发布前自动校验链接有效性，避免文档中出现死链，提升文档专业度与用户体验。

数据分析师追踪数据源变更
数据分析项目通常依赖多个外部数据接口与公开数据集。分析师可使用 LinkVault 记录每个数据源的访问地址、更新时间与字段说明，当数据源发生变更时，系统通过健康检查与摘要对比及时发现差异，辅助分析师快速定位影响范围。

开源项目维护外部依赖与参考文献
开源项目 README 与贡献指南中常包含大量外部依赖地址、学习资料与社区讨论帖。项目维护者可将这些链接纳入 LinkVault 集中管理，在版本发布前统一审查链接状态，确保用户与贡献者能够顺利访问所有参考资料。

学术研究团队管理实验数据与文献来源
科研人员在实验过程中需记录大量数据来源、工具地址与参考文献。LinkVault 的分类与注释系统支持按照实验阶段、数据类型或研究主题对链接进行细致标注，并可导出为引用格式，简化论文撰写与数据归档流程。

## 快速开始

以下命令演示如何获取 LinkVault 源代码、安装依赖并启动开发服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装 Python 依赖与前端资源
pip install -r requirements.txt
npm install --prefix frontend

# 初始化数据库并运行开发服务器
python scripts/init_db.py
python scripts/import_sample_links.py
python app.py --env development
```

启动成功后，访问控制台地址 `http://127.0.0.1:5000` 即可进入管理界面。默认管理员账号为 `admin`，密码为 `changeme`，首次登录后请立即修改。

## 项目结构

项目采用前后端分离的经典布局，后端基于 Flask 框架组织为 MVC 分层结构，前端以独立目录存放构建产物。

```
linkvault/
├── app/                                 # 后端应用主目录
│   ├── api/                             # RESTful API 路由定义
│   │   ├── v1/                          # API 版本 1 端点
│   │   │   ├── links.py                 # 链接资源的 CRUD 操作
│   │   │   ├── checks.py                # 健康检查任务触发与查询
│   │   │   └── categories.py            # 分类管理接口
│   │   └── __init__.py                  # API 蓝图注册与版本路由
│   ├── models/                          # 数据模型与 ORM 映射
│   │   ├── link.py                      # 链接主表模型，含 URL、标题、描述、状态
│   │   ├── snapshot.py                  # 快照表模型，存储每次检查的摘要数据
│   │   └── user.py                      # 用户与权限模型
│   ├── services/                        # 业务逻辑层
│   │   ├── fetcher.py                   # 页面抓取与元数据解析服务
│   │   ├── checker.py                   # 链接健康检查调度与执行引擎
│   │   └── indexer.py                   # 全文检索索引构建与查询服务
│   ├── tasks/                           # 异步任务定义（Celery 或 RQ）
│   │   ├── check_tasks.py               # 周期性检查任务
│   │   └── import_tasks.py              # 批量导入与元数据处理任务
│   └── utils/                           # 通用工具函数
│       ├── validators.py                # URL 格式校验与规范化辅助
│       └── parsers.py                   # HTML 元标签与正文摘要解析器
├── frontend/                            # 前端单页应用源码
│   ├── src/                             # Vue/React 组件与页面
│   │   ├── components/                  # 可复用 UI 组件（链接卡片、分类树）
```

## 常见问题

**问：LinkVault 是否会存储外部链接的页面内容副本？**
答：不会。LinkVault 仅存储链接的元数据（标题、描述、关键词）以及每次健康检查时记录的页面摘要信息（如标题变更、状态码）。完整页面内容不会被抓取或存储，以尊重内容提供方的版权与服务器资源。

**问：健康检查的周期与并发策略如何配置？**
答：健康检查周期可在系统配置文件中通过 `CHECK_INTERVAL_HOURS` 变量调整，默认值为 24 小时。并发检查任务数量由 `CHECK_CONCURRENCY` 控制，默认同时执行 5 个检查任务。生产环境建议根据服务器负载与目标网站承受能力适当调整该参数。

**问：能否将 LinkVault 部署在无外网访问的内网环境中？**
答：可以。LinkVault 的核心功能（链接管理、分类、注释与检索）均可在内网环境中正常运行。但健康检查与元数据自动补全功能需要能够访问目标链接所在的公网或内网地址。若目标链接全部为内网资源，请确保部署节点具备相应的内网访问权限。

**问：导入大量链接时是否会阻塞主进程？**
答：批量导入任务会通过任务队列（Redis + RQ）异步执行，不会阻塞 Web 控制台的正常响应。用户提交导入请求后即可关闭页面，系统完成后将通过站内通知或邮件（需配置）告知导入结果。
