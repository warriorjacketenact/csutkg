# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究人员、内容运营团队和数据分析师的开源外链资源汇总与管理工具。该项目不生产内容，而是提供结构化的外链索引体系，帮助用户从海量分散的 URL 中快速定位高价值信息节点。LinkVault 定位于轻量级资源网关，通过标签化分类、状态监控和元数据提取，将原始链接列表转化为可检索、可追踪、可分析的结构化数据资产。

本项目适用于需要定期处理大批量外链资源的场景，例如技术文档归档、行业动态追踪、竞品内容分析或学术参考资料整理。LinkVault 提供开箱即用的命令行工具与 Python API，支持自定义解析规则与输出格式，能够无缝接入现有的数据流水线或静态站点生成工作流。

## 功能概览

批量导入与去重：支持从纯文本、CSV 或 JSON 格式导入 URL 列表，自动识别并移除重复条目，保留原始顺序，输出去重统计报告。

存活状态检测：内置异步 HTTP 客户端，并发探测每个链接的可访问性（200/3xx/4xx/5xx），支持自定义超时与重试策略，生成状态摘要表。

元数据自动提取：从目标页面提取标题、描述、关键词及 Open Graph 信息，生成 JSON 格式的元数据缓存，减少重复请求开销。

标签分类引擎：基于正则表达式与关键词库对链接进行自动打标，支持用户自定义分类规则，输出分类统计与未分类列表供人工复核。

自定义输出模板：支持 Markdown 表格、HTML 目录、JSON 结构化数据及 RSS 订阅源四种输出格式，适配静态站点、知识库或监控看板。

增量更新机制：维护本地 SQLite 缓存数据库，仅对新增或失效链接执行请求，大幅提升二次运行效率。

过滤与排序规则：按状态码、响应时间、域名或标签组合进行过滤与排序，支持输出 Top-N 最快响应或最近更新列表。

交互式管理界面：提供轻量级 Web UI（可选），基于 Flask 实现，支持可视化管理资源库、编辑标签、查看详情与导出数据。

## 应用场景

技术博客运营：技术团队可使用 LinkVault 维护“每周优质外链”栏目。编辑将候选 URL 导入系统，自动提取标题与摘要，按标签分类后生成 Markdown 格式的周报草稿，直接粘贴至博客后台，减少手动整理耗时。

学术文献索引：研究人员收集大量论文预印本、数据集仓库和工具主页链接。LinkVault 的存活检测功能定期扫描链接有效性，标记失效资源，同时提取页面描述生成注释，辅助构建个人文献数据库。

产品竞品监控：产品经理收集竞品官网、更新日志、招聘页面和用户反馈论坛链接。通过 LinkVault 的定时任务与 RSS 输出功能，生成聚合订阅源，配合 RSS 阅读器实现每日动态追踪。

数据中台预处理：数据工程师在 ETL 流程中嵌入 LinkVault 作为数据源清洗环节，对上游爬虫输出的原始 URL 列表进行去重、状态验证与元数据补全，生成高质量下游输入。

## 快速开始

以下命令演示如何克隆仓库、安装依赖并运行首次扫描。

```bash
git clone https://github.com/yourorg/linkvault.git
cd linkvault
pip install -r requirements.txt
python linkvault.py --input urls.txt --output report.md --format markdown
```

其中 urls.txt 为每行一个 URL 的纯文本文件。首次运行将自动创建缓存数据库 `cache.db` 与配置文件 `config.yaml`。如需启用 Web UI，请执行：

```bash
python webui.py --port 8080
```

## 项目结构

```
linkvault/
├── linkvault.py                # 命令行入口，解析参数并调度核心流程
├── webui.py                    # Flask Web 界面入口，可选启动
├── config.yaml                 # 用户配置文件，定义标签规则、请求头与输出模板
├── cache.db                    # SQLite 缓存数据库，存储元数据与状态历史（自动生成）
├── requirements.txt            # Python 依赖清单，用于 pip 批量安装
├── core/                       # 核心逻辑模块
│   ├── __init__.py
│   ├── fetcher.py              # 异步 HTTP 请求器，含连接池与重试策略
│   ├── parser.py               # HTML 解析器，提取标题、描述、OG 标签
│   ├── classifier.py           # 标签分类引擎，基于正则与关键词库
│   ├── cache.py                # SQLite 缓存读写接口，支持增量更新
│   └── exporter.py             # 输出格式化器，支持 Markdown / JSON / HTML / RSS
```

## 常见问题

问：存活状态检测时遇到 SSL 证书验证错误如何处理？

答：部分目标站点使用自签名证书或过期证书。您可以在 config.yaml 中设置 verify_ssl: false 跳过验证。请注意，这会降低安全性，仅建议在内网或测试环境中使用。若需保留验证但忽略特定域名，可在 config.yaml 的 ssl_whitelist 字段中添加域名列表。

问：如何自定义标签分类规则？

答：在 config.yaml 中的 classifier 节下定义 rules 列表。每条规则包含 pattern（正则表达式）和 tags（字符串数组）。例如，规则 - pattern: "github\\.com" tags: ["代码托管", "开源"] 将匹配包含 github.com 的链接并打上对应标签。规则按顺序匹配，命中后即停止。未匹配的链接将归入 uncategorized 类别。

问：增量更新机制如何工作？能否手动强制全量刷新？

答：LinkVault 为每个 URL 维护最近一次请求时间戳与响应状态。当再次运行时，默认仅对状态码非 200 或距离上次请求超过 7 天的链接重新探测。您可以通过命令行参数 --force 强制忽略缓存，对所有链接执行全量刷新。缓存数据库 cache.db 位于项目根目录，删除该文件也可达到重置效果。
