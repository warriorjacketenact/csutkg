# LinkRanger

LinkRanger 是一个面向技术研究者、内容策展人与开源社区维护者的轻量级外链资源归集与导航系统。该项目并非搜索引擎，也非通用书签管理器，而是一套专注于“按批次、按主题、按来源”对散落于各处的深度链接（Deep Links）进行结构化归档的静态站点生成工具。LinkRanger 的目标用户包括需要定期整理技术周报的编辑、需要跟踪特定域名下长尾内容的舆情分析人员，以及希望将大量参考链接以低技术债务方式嵌入项目文档的开发者。

LinkRanger 解决的核心问题是：当外部资源链接数量庞大、来源域名分散、且每条链接均携带独特查询参数或路径片段时，如何通过一套本地优先、无需后端数据库、完全基于 Markdown 与 YAML Frontmatter 的流水线，将这些链接转化为可浏览、可检索、可版本控制的静态导航页面。项目本身不存储任何外部内容副本，仅提供链接元数据编排能力，完全遵守 robots.txt 与外部站点的访问策略。

## 功能概览

**批量链接摄入**：支持通过 CSV 或纯文本列表一次性导入数百条外部 URL，自动解析协议、域名、路径与查询参数，并按来源域名与路径深度进行初步分类。

**来源域名聚合视图**：自动统计所有收录链接的根域名与子域名分布，生成按域名分组的索引页，便于用户快速定位特定站点下的所有已收录深度链接。

**自定义元数据标签**：允许为每条链接附加自由格式的标签（如 "tutorial", "reference", "video", "discussion"），并基于标签生成交叉索引，提升长尾资源的可发现性。

**只读静态站点生成**：基于 Python 的静态站点生成器核心，输出纯 HTML/CSS 文件，无需数据库或运行时环境，可直接部署至任何支持静态托管的 Web 服务器或 CDN。

**增量更新与冲突检测**：提供增量导入模式，自动检测新链接与已有链接的重复项，并生成冲突报告，避免人工去重的繁琐操作。

**Markdown 目录树导出**：除 HTML 站点外，支持将整个链接库导出为带注释的 Markdown 目录树文件，便于直接嵌入项目文档或作为技术决策附录。

## 应用场景

**技术周报与月刊素材管理**：内容编辑在收集每周技术动态时，通常需要从多个技术社区、官方博客与论坛中积累数十至上百个待参考链接。LinkRanger 允许编辑按周或按月创建独立批次，将原始链接批量摄入后统一打上 "weekly-2026-w25" 类标签，再通过生成的静态页面进行预览与审核，最终将精选链接以干净格式发布至周报正文。

**开源项目外部参考附录维护**：大型开源项目的设计文档或决策记录（ADR）中常需引用大量外部讨论链接、竞争对手实现方案或性能对比数据。这些链接数量众多且路径冗长，直接堆叠于文档正文会严重影响可读性。项目维护者可利用 LinkRanger 建立独立的 "references" 子项目，按模块或议题组织链接，最后通过导出功能生成结构化的附录章节，直接粘贴至项目根目录的 REFERENCES.md 中。

**学术文献补充材料整理**：研究人员在撰写论文或技术报告时，需整理数据集来源、工具仓库、在线演示及补充阅读列表。LinkRanger 的批次概念可对应论文版本号，确保修订过程中的链接增删均有据可查。生成的静态页面可作为补充材料的在线附件，与论文一同发布。

**域名级别的舆情与动态追踪**：对于需要长期观察特定域名（如新闻站点、政府公告栏或产品更新日志）下内容更新频率与主题变化的用户，LinkRanger 支持按域名筛选与导出链接清单，配合外部 diff 工具可快速识别新增或失效链接，辅助判断内容发布节奏。

## 快速开始

以下命令序列适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-organization/linkranger.git
cd linkranger

# 安装 Python 依赖（建议使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt

# 执行批量导入与静态站点生成
# 默认读取 data/raw/links_batch_261.csv 文件
python -m linkranger ingest --batch 261 --source data/raw/links_batch_261.txt
python -m linkranger build --output dist/
```

执行上述命令后，所有静态页面将被生成于 `dist/` 目录下，可通过任意 HTTP 服务器（如 `python -m http.server 8000 --directory dist/`）进行本地预览。

## 贡献指南

1. 查阅 `docs/developer/architecture.md` 文件，了解核心流水线的数据流向与抽象接口设计。所有新增功能应尽量通过扩展 `parser.py` 或 `exporter.py` 中的策略类实现，避免修改主调度逻辑。

2. 在 `src/tests/` 目录下为新增代码编写对应的单元测试。测试应覆盖正常路径与边界条件（如畸形URL、空列表、超大批次）。运行 `pytest` 确保全部测试通过后再提交。

3. 提交拉取请求（Pull Request）前，请执行 `python -m linkranger build --output dist/` 进行全量构建，验证生成的HTML页面在至少两种主流浏览器（Chromium/Firefox）中渲染正常。

4. 若需增加新的输出格式（如JSON API或RSS feed），请在 `exporter.py` 中注册新的导出器类，并在 `cli.py` 中添加对应命令行选项。导出器的命名应遵循 `*Exporter` 模式。

5. 文档更新与代码变更需同步。若修改了命令行接口或配置文件格式，必须同步更新 `docs/user-guide/` 下的相关章节。文档应使用美式英文撰写，但项目README及面向中文用户的指南可保留中文。

## 常见问题

**Q: 导入的链接数量很大（如超过1000条），生成速度会明显变慢吗？**

A: LinkRanger 的核心操作均为本地文件读写与内存映射，不涉及网络请求或数据库事务。在中等配置的笔记本上（16GB内存，NVMe SSD），处理1000条链接的摄入与构建总耗时通常在2秒以内。若链接数量超过5000条，建议使用 `--no-head-check` 选项禁用可达性探测，以进一步缩短处理时间。

**Q: 如何处理源站点链接失效或域名过期的情况？**

A: LinkRanger 本身不提供自动死链检测功能，但您可以在摄入时通过 `--check` 启用可选的HEAD请求验证。对于已入库的链接，项目提供了 `verify` 子命令，可定期运行并生成失效链接报告。需要注意的是，部分站点可能拒绝HEAD请求或返回429限流状态码，此时建议人工复核。

**Q: 生成的静态页面能否部署到 GitHub Pages 或 Cloudflare Pages？**

A: 可以。`dist/` 目录中全部为静态资源，无服务端依赖。您可以将该目录直接作为 Pages 项目的根目录推送。对于GitHub Pages，建议在仓库设置中指定 `gh-pages` 分支或 `docs/` 文件夹。对于Cloudflare Pages，连接仓库后直接将构建输出目录设置为 `dist` 即可。
