# CDJ Resource Aggregator

CDJ Resource Aggregator 是一个面向技术内容聚合与外部资源导航的开源工具集，定位于帮助开发者、运维人员与技术研究者高效地收集、分类、检索和展示来自指定源站点的海量资源链接。该项目的核心目标是解决多源异构资源链接的可维护性问题，通过结构化存储与标准化输出，将零散的外部链接转化为可被二次开发与自动化处理的数据资产。

本项目适用于需要定期同步外部资源列表并对外提供统一访问入口的场景。项目内置了完整的资源抓取、链接校验、元数据补全和静态页面生成流程，能够将原始的资源标识符（如 `/vod/` 路径下的短码）转换为可供人工浏览或机器调用的结构化数据格式。与常规收藏夹或书签管理工具不同，CDJ Resource Aggregator 强调批次管理、变更追踪与可复现的构建流程，使得大规模资源链接的维护工作具备工程化的基础。

## 功能概览

**批量链接导入与解析**：支持从外部数据源批量导入形如 /vod/xxxxx 格式的资源标识符，自动解析路径参数并生成完整的访问 URL。系统内置去重与格式校验逻辑，确保每条记录符合目标站点的路径规范。

**资源元数据自动补全**：对导入的链接执行 HTTP 头探测与响应状态检查，自动获取资源标题、内容类型、最后修改时间等信息，将原始链接升级为包含丰富上下文的结构化条目。

**分类标签与多级目录管理**：允许用户为每个资源链接打上自定义标签，支持按主题、格式、来源批次等维度建立分类树。分类规则可通过 YAML 配置文件灵活定义，无需修改核心代码。

**静态站点生成引擎**：基于链接数据库与分类配置，一键生成完整的静态 HTML 站点。生成的站点包含索引页、分类浏览页、详情页与全文检索功能，可直接部署到任意 Web 服务器或对象存储服务。

**批次状态追踪与变更审计**：记录每一批次资源导入的时间、来源、操作人与变更摘要，支持按批次回滚或对比不同批次之间的差异。内置的审计日志功能便于团队协作时的责任追溯。

**开放数据导出接口**：提供 JSON、CSV、RSS 三种数据导出格式，方便与其他自动化工具或下游系统集成。导出的数据保留完整的元数据字段，支持自定义字段映射。

## 应用场景

**技术文档站的外链资产整理**：大型技术文档项目通常需要引用大量外部参考链接，这些链接分散在多个 Markdown 文件中，难以统一维护。CDJ Resource Aggregator 可以作为外链资产的集中管理后端，定期扫描文档源文件中的外链并汇总至统一资源库，在链接失效时发出告警并批量更新。

**内部知识库的资源导航门户**：企业内部的 Wiki 或知识库系统往往积累了成百上千的参考链接，但缺乏有效的导航和检索机制。使用本项目的静态站点生成功能，可以快速将这批链接转化为分类清晰、支持搜索的内部资源门户网站，提升知识复用效率。

**爬虫数据管道的链接清洗环节**：在数据采集管道中，原始爬虫输出的链接列表通常包含大量噪声和重复项。CDJ Resource Aggregator 可作为管道的中间处理环节，接收爬虫产出的原始链接列表，执行去重、格式规范化、状态验证等清洗操作，输出高质量的结构化链接数据集供下游使用。

**开源项目的资源推荐页构建**：开源项目在 README 或官方文档中推荐第三方资源时，手动维护列表容易遗漏或过时。通过本项目的批次管理功能，可以为每个版本发布生成对应的资源推荐快照，确保文档中的资源列表与项目版本保持同步。

## 快速开始

以下命令演示了如何从源码启动 CDJ Resource Aggregator 并导入资源链接。

```bash
# 克隆项目仓库至本地
git clone https://github.com/cdj-resource/cdj-aggregator.git
cd cdj-aggregator

# 安装项目依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 执行资源导入与站点构建（以示例数据运行）
python cli.py import --source data/sample_links.json --batch 931
python cli.py build --output ./dist
```

执行完成后，生成的静态站点文件将输出至 `./dist` 目录，可使用任意 HTTP 服务器进行预览，例如 `python -m http.server 8000 --directory ./dist`。

│   ├── __init__.py
│   ├── importer/              # 导入模块：处理外部数据解析与入库

│   │   ├── validator.py       # 校验链接格式、去重与路径规范化
│   │   └── loader.py          # 将校验后的数据写入 SQLite 数据库

## 常见问题

**问：导入大量链接时出现超时或内存不足的错误，应该如何优化？**

答：对于超过 5000 条链接的大规模导入场景，建议分批次执行导入命令，每次处理 1000 条左右。可以通过 `--limit` 与 `--offset` 参数控制每批处理的数量。同时，在 `config/default.yaml` 中调整 `fetcher.timeout` 和 `fetcher.concurrent` 参数，降低并发数以减少内存占用。如果使用默认的 SQLite 数据库，执行 `PRAGMA journal_mode=WAL;` 命令可以提升写入并发性能。

**问：生成的静态站点中部分链接无法访问，如何排查问题？**

答：静态站点中的链接均为原始 URL 的直接引用，无法访问通常由目标站点自身的可用性决定。建议在导入阶段启用 `--verify` 选项，该选项会在导入时对所有链接执行 HEAD 请求，并将状态码记录到数据库的 `last_status` 字段。生成的站点详情页会显示每个链接的最后检查状态，便于快速定位失效链接。此外，可以运行 `python cli.py audit --dead-only` 命令生成一份仅包含失效链接的审计报告。

**问：项目支持增量更新吗，即只处理新增或变更的链接？**

答：支持。导入器默认启用增量模式，会根据链接的路径标识符判断该记录是否已存在于数据库中。对于已存在的记录，系统会比对源数据的 `last_modified` 字段（如果提供）或更新时间戳，仅当内容发生变更时才会更新元数据。如果希望强制全量覆盖，可以在导入命令中添加 `--force` 参数。增量更新的详细信息会记录在批次审计日志中。
