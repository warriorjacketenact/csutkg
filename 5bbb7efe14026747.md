# EON Resource Navigator

EON Resource Navigator 是一个面向技术研究、知识工程与信息检索场景的轻量级外链资源汇总平台。项目定位于为开发者、数据分析师、科研工作者以及信息爱好者提供结构化、可扩展的URL资源导航服务，将散落在网络各处的优质外链按照主题与用途进行系统化归集，降低信息发现与筛选的时间成本。

本项目不生产内容，而是致力于成为内容发现层的基础设施，通过清晰的分类体系与元数据标注，帮助用户在特定领域内快速定位高价值外部资源。EON Resource Navigator 当前收录第 1111/1241 批次，共计 250 个经过人工筛选的链接，覆盖技术文档、开源社区、学术论文库、行业资讯、标准化组织、开发者工具等多个维度。

## 功能概览

- **批量外链导入与解析** 支持从纯文本列表、CSV或JSON格式批量导入URL，自动完成协议识别与去重校验，内置链接可用性预检机制。

- **多层级分类标签系统** 每个资源可绑定多个主题标签，支持按领域（如机器学习、前端工程、数据库）、按资源类型（文档、工具、社区、标准）及按来源机构进行多维度筛选。

- **全文元数据提取** 自动抓取目标页面的标题、描述、关键词及结构化元数据，生成资源摘要快照，便于离线检索与快速预览。

- **自定义收藏夹与列表导出** 用户可创建私有或公开的资源收藏列表，支持按批次、按日期或按标签导出为Markdown、HTML或结构化JSON格式。

- **资源变更监控** 对已收录的URL提供定期可达性检查与内容变更感知，自动标记失效链接或重大内容更新，确保资源库的鲜活度。

- **API查询接口** 提供RESTful风格的查询API，支持按关键词、标签、域名前缀等条件进行精确或模糊匹配，方便集成至下游工作流。

- **静态站点生成能力** 内置模板引擎可将资源数据渲染为全静态导航站点，适用于内网部署或个人知识库发布。

## 应用场景

- **技术团队内部知识库建设** 技术负责人可使用本项目的分类与元数据提取功能，将团队常用的开发文档、API参考、设计规范等外部链接统一归档，生成团队共享的导航页面，减少新成员上手时的信息找寻时间。

- **学术文献与数据源管理** 研究人员在开展文献综述或数据收集工作时，可通过本项目的标签系统对预印本服务器、公开数据集、标准化组织官网等资源进行精细标注，结合导出功能生成参考文献附录或数据来源说明。

- **开源项目依赖与生态追踪** 开源维护者可利用资源变更监控功能跟踪上游依赖、相关工具链或竞品项目的文档站点更新，及时掌握生态变化，降低维护滞后风险。

- **个人兴趣聚合阅读** 技术爱好者可自建私有资源列表，将分散在技术博客、播客节目、电子杂志等不同媒介的入口集中管理，结合定期可达性检查维持阅读源的持续可用。

## 快速开始

以下命令演示了如何从GitHub克隆本项目、安装依赖并启动开发服务。

```bash
# 克隆项目仓库
git clone https://github.com/eon-resource/eon-navigator.git

# 进入项目目录
cd eon-navigator

# 安装依赖（使用npm）
npm install

# 复制环境变量模板并配置
cp .env.example .env

# 初始化本地资源数据库（SQLite）
npm run init-db

# 导入示例批次资源（包含本批次250个链接）
npm run import -- --batch 1111 --file ./data/batch_1111.txt

# 启动开发服务器（默认监听3000端口）
npm run dev
```

访问 http://localhost:3000 即可查看资源导航面板。如需构建生产环境静态站点，请执行 `npm run build`，输出目录为 `./dist`。

## 项目结构

```
eon-navigator/
├── src/
│   ├── core/                     # 核心模块：数据库连接、配置管理、日志系统
│   │   ├── database.js           # SQLite3 连接池与迁移管理
│   │   ├── config.js             # 环境变量加载与默认配置合并
│   │   └── logger.js             # 结构化日志输出（支持JSON与彩色终端）
│   ├── crawler/                  # 资源抓取与元数据提取模块
│   │   ├── fetcher.js            # 基于curl的HTTP请求封装，支持重试与超时控制
│   │   ├── parser.js             # HTML元数据解析（title、description、keywords）
│   │   └── checker.js            # 链接可达性批量检查与状态记录
│   ├── service/                  # 业务逻辑层
│   │   ├── resource.js           # 资源CRUD、标签关联、批次管理
│   │   ├── export.js             # 导出为Markdown、JSON、HTML格式
│   │   └── monitor.js            # 定时监控任务调度与变更通知
│   ├── api/                      # RESTful API 路由层
│   │   ├── v1/
```

## 贡献指南

本项目遵循开源社区协作规范，欢迎各类形式的贡献。请按照以下步骤参与。

1. 查阅问题追踪列表 访问 GitHub Issues 页面查看待解决的问题或功能请求，选择未被认领且与自身技能匹配的任务。对于重大功能提议，建议先创建讨论议题与维护者沟通设计思路。

2. 派生仓库并创建特性分支 将主仓库派生至个人账户，克隆至本地后使用 `git checkout -b feature/your-feature-name` 创建新分支。分支命名建议采用 `feature/`、`fix/`、`docs/` 前缀以区分类型。

3. 编写代码并确保测试通过 遵循项目已配置的 ESLint 与 Prettier 代码风格规则。新增功能需附带对应的单元测试或集成测试，确保 `npm test` 全部通过。若涉及数据库变更，需在 `data/db/migrations` 目录下编写迁移脚本。

4. 提交变更并签署开发者原产地证书 提交信息请采用 Conventional Commits 格式（如 `feat: add batch export to JSON`）。提交前需签署 DCO，即每个提交需包含 `Signed-off-by: Your Name <email>` 行，表示您有权并同意将代码贡献至本项目。

5. 发起拉取请求 推送分支至派生仓库后，在主仓库发起 Pull Request。请在描述中清晰说明变更目的、测试覆盖情况及相关Issue编号。维护者将在三个工作日内进行审阅，可能需要您根据反馈进行修改。

## 常见问题

**问：项目是否需要外部数据库服务？是否支持MySQL或PostgreSQL？**

答：默认使用 SQLite3 作为嵌入式数据库，无需额外安装数据库服务进程，适合个人或小团队快速部署。如需在生产环境使用 MySQL 或 PostgreSQL，可通过修改 `src/core/database.js` 中的适配器配置切换，但需自行安装对应驱动并调整连接参数。社区版默认仅官方支持 SQLite3。

**问：如何更新已导入的资源链接或修改其分类标签？**

答：已导入的资源支持通过 API 或命令行进行更新。使用 `npm run resource:update -- --id <resource_id> --tags "newtag1,newtag2"` 可修改标签。若链接内容变更但URL不变，可通过 `npm run resource:refresh -- --id <resource_id>` 重新抓取元数据。如需批量更新，建议先导出为JSON格式，修改后再调用批量导入接口（使用 `--mode update` 参数）。

**问：资源可达性检查的频率与通知方式如何配置？**

答：检查频率通过环境变量 `MONITOR_INTERVAL` 配置，单位分钟，默认值为 1440（每日一次）。通知方式目前支持控制台日志与Webhook（可配置 `MONITOR_WEBHOOK_URL` 指向企业微信、飞书或Slack机器人）。检查结果会存储在 `monitor_log` 表中，可通过API查询历史记录。对于失效链接，系统会自动标记为 `unreachable` 状态，并保留最后可达时的元数据快照。
