# Tangent Resource Aggregator

Tangent Resource Aggregator 是一个面向技术研究人员、内容策展人与信息分析团队的外链资源归集与结构化整理工具。该项目并非传统意义上的爬虫或采集器，而是一套用于对分散于互联网各处的深度内容进行条目化、标签化与版本化管理的参考实现。其核心定位是对特定域名下的批量内容页进行标识符提取、元数据归类与可追溯链接归档，适用于需要长期维护外部信息源索引的各类工作流。

本项目解决的核心问题在于：当面对大量结构相似但语义独立的资源页面时，如何通过统一的入口对其进行稳定引用、快速检索与状态追踪。通过将每个资源条目映射为独立记录并配合外部元数据服务，使用者可以构建出轻量级但可扩展的知识关联网络，而无需依赖特定厂商的内容管理平台。

## 功能概览

**批量链接归一化处理**：对给定域名下的路径型资源进行自动去重与标识符规范化，确保每个条目具备全局唯一引用键。

**元数据抽象与映射**：为每条外部链接提供可自定义的元数据扩展字段，支持标题、摘要、类型标签及关联批次号的附加存储。

**状态快照与变更追踪**：通过批次编号与时间戳机制记录每次资源导入的增量变化，便于后续比对链接有效性或内容迁移情况。

**结构化列表生成**：将归集后的资源按指定维度（如字母序、批次、类型）输出为 Markdown 原生列表，可直接嵌入文档或站点内容。

**外部资源校验接口**：提供简单的链接可达性检查工具，辅助识别失效或重定向的外部页面，降低维护成本。

**多格式导出支持**：除 Markdown 列表外，支持将归集结果导出为 JSON 与 CSV 格式，方便与其他数据处理工具（如电子表格、数据库）对接。

**轻量级本地部署**：无需外部数据库或复杂运行时环境，仅依赖标准 Python 工具链即可完成安装与使用，适合个人开发者与小团队场景。

**可扩展的过滤器链**：允许用户编写简单的过滤规则（如按路径前缀、文件类型或关键词）对导入的资源进行二次筛选，提升整理效率。

## 应用场景

内容策展团队在整理特定主题的外部参考资料时，可使用本项目快速将一批分散的深度文章链接归入统一的清单，并为每条链接添加策展备注与重要性评级，形成可共享的精选资源目录。

技术文档撰写者在收集 API 参考或实现案例时，可通过本项目将不同来源的示例页面集中归档，并结合元数据字段标注其适用的编程语言、框架版本或依赖条件，便于后期检索与引用。

学术研究人员在文献调研阶段，可利用本项目对预印本、技术报告或项目主页进行批量录入与分类，通过批次号管理不同检索策略下的结果集，提升文献综述的条理性。

运维与安全审计人员在对外部第三方服务页面进行合规性审查时，可借助本项目的链接追踪能力记录每次审查时间与状态变化，形成可追溯的审计线索。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议通过 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/example/tangent-resource-aggregator.git

# 进入项目目录
cd tangent-resource-aggregator

# 安装所需依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行资源归集示例（使用项目内置的测试条目）
python aggregate.py --input samples/links.txt --output resources.md
```

执行完成后，在当前目录下生成的 resources.md 文件即包含归集后的资源列表。如需导入自定义链接，请将每条链接置于文本文件中（每行一条），并通过 --input 参数指定该文件路径。

## 常见问题

Q: 项目对输入链接的格式有何限制，能否处理非 HTTP/HTTPS 协议？

A: 解析模块默认接受 http 与 https 协议的链接，其他协议（如 ftp、file）会被记录为警告并跳过归集。如需支持自定义协议，可继承 core.parser.BaseParser 并重写 validate_scheme 方法后注册至工厂。

Q: 如何对归集结果进行增量更新，避免重复录入相同链接？

A: 项目默认以完整 URL 字符串作为去重键。若需更细致的去重策略（如忽略查询参数或路径尾部斜杠），可在 metadata.py 中调整 normalize_url 函数的实现逻辑，将其替换为自定义归一化函数即可。

Q: 校验模块的超时和重试机制如何配置，是否支持代理？

A: 超时值（默认 5 秒）与重试次数（默认 2 次）可通过 validator.py 中的 DEFAULT_TIMEOUT 与 DEFAULT_RETRIES 常量调整，也支持从环境变量 VALIDATOR_TIMEOUT 与 VALIDATOR_RETRIES 读取。代理支持依赖于 requests 库的 proxies 参数，可在调用 validate_url 时传入。
