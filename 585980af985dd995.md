# ResourceBridge

ResourceBridge 是一个面向开发者、研究人员与技术写作人员的轻量化技术资源外链聚合与导航工具。项目定位为「技术决策的信息前置层」，通过人工筛选与结构化分类，将分散于不同领域的高价值外部链接整理为可检索、可订阅、可嵌入工作流的资源索引体系。

项目不提供内容的存储或镜像，专注于解决信息过载环境下的高质量信源发现效率问题。目标用户包括需要持续跟踪技术动态的架构师、进行竞品分析的产品经理、撰写技术博客的编辑人员，以及希望降低信息筛选成本的个人开发者。

## 功能概览

**结构化分类导航** 按技术领域、内容类型、更新频率与可信度等级对每条外链进行多维度标签标记，支持快速过滤与批量导出。

**关键词与语义检索** 内置轻量级倒排索引，支持基于标题、描述、标签及摘要全文的检索，检索结果按相关性权重排序。

**动态资源清单同步** 通过脚本定期抓取源站变更，对失效链接进行自动标记与通知，维持资源库的健康度。

**自定义订阅源生成** 支持将任意分类或标签组合输出为 Atom/RSS 订阅源，便于集成至阅读器或自动化流程。

**外链元数据增强** 自动补全每条链接的页面标题、最后修改时间、内容类型（文档/博客/工具/视频）及语言代码。

**批量导入与导出** 支持 CSV、JSON 与 OPML 格式的资源列表导入导出，便于与其他知识管理工具交换数据。

**嵌入组件支持** 提供可嵌入任意网页的侧边栏小组件与命令行查询接口，方便团队内部共享资源索引。

## 应用场景

技术团队内部知识库建设：技术负责人可以使用 ResourceBridge 将团队成员分散收藏的优质外链统一汇总，按语言、框架或业务领域分类，形成团队共享的技术雷达，降低重复搜索成本。

技术博客与资讯聚合：技术编辑可以将 ResourceBridge 作为信息采集后台，定期导出特定标签下的最新链接列表，快速发现可撰写的技术主题或案例素材。

个人开发者的学习路径管理：个人开发者可以为自己的学习计划建立独立的资源集合，例如按「Go 语言」「分布式系统」「API 设计」等主题分组，跟踪学习进度与参考资料更新。

开源项目文档站的外链附录：开源项目维护者可以将 ResourceBridge 生成的资源清单嵌入项目 docs 目录，为使用者提供官方推荐的外部学习材料与相关工具列表。

## 快速开始

以下命令将在本地启动 ResourceBridge 实例，默认监听 8787 端口。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run build && npm start
```

首次启动完成后，访问 http://localhost:8787 进入初始化向导，按照提示创建管理员账户并导入示例资源数据。

| 依赖 | 必需 | 说明 |

| Node.js 20.x LTS | 是 | 运行时环境，推荐使用 nvm 管理版本 |
| npm 10.x | 是 | 包管理器，随 Node.js 一同安装 |
| SQLite 3 | 是 | 嵌入式数据库，用于存储资源索引与元数据 |
| Redis 7.x | 否 | 启用缓存与会话持久化时需要，生产环境推荐 |

| Docker 24+ | 否 | 容器化部署支持，提供官方镜像 |
| Git 2.40+ | 是 | 用于版本管理与插件更新 |
| 系统内存 512MB | 是 | 最低运行内存，实际使用建议 1GB 以上 |
| 磁盘空间 1GB | 是 | 用于存储数据库与日志文件 |

## 常见问题

问：ResourceBridge 是否会存储外部链接的内容副本或缓存页面全文？

答：不会。ResourceBridge 仅存储链接的元数据，包括标题、描述、标签、最后访问状态与时间戳。项目不缓存页面正文内容，所有资源访问均重定向至原始 URL。用户需自行承担访问外部链接的合规责任。

问：如何处理外部链接失效或内容变更的情况？

答：系统后台任务会按照可配置的周期（默认为每 7 天）对已收录链接进行 HEAD 请求检查，记录 HTTP 状态码变化。连续两次检查返回 4xx 或 5xx 状态的链接会被标记为「失效」，并在管理界面中以高亮提示。失效链接不会自动删除，用户可手动确认后移除或更新。

问：能否将 ResourceBridge 部署在无公网访问的内网环境？

答：可以。ResourceBridge 完全可以在离线环境运行，但需要提前将所有依赖包下载至本地镜像仓库。此外，资源抓取与更新功能在内网环境下仅能解析内网可访问的域名，外部链接的健康检查将会失败。建议在内网部署时关闭自动同步功能，手动维护资源列表。
