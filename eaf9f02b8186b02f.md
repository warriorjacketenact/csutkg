# LinkSphere 技术资源导航系统

LinkSphere 是一个面向开发者与技术研究人员的结构化外链资源汇总平台，专注于收集、分类与索引互联网中高质量的技术文章、工具文档与工程实践参考。项目本身不存储任何直接内容，而是以目录化索引方式提供可追溯、可版本化的外部资源引用体系。

项目定位为“技术资源的元仓库”，解决个人开发者与团队在信息筛选、重复查找、知识碎片化方面的痛点。通过统一的入口与标准化的引用格式，用户能够快速定位到特定主题的深度阅读材料，同时利用项目的分类与注释机制建立个人或团队的知识图谱。LinkSphere 适合作为技术博客的补充源、团队 onboarding 的知识索引，以及日常技术调研的起点。

## 功能概览

**结构化资源索引**：所有外链按主题与用途划分为多个一级类别，每个类别下附带简要的范围说明，便于用户按领域浏览。

**URL 永久化存档**：每个收录的链接均保留原始 URL 与收录时间戳，确保引用可追溯，避免因源站改版导致参考信息丢失。

**多维度标签系统**：每条资源支持多个功能标签（如“tutorial”、“reference”、“case-study”），支持快速过滤与相关性排序。

**目录树可视化**：项目仓库内提供完整的 ASCII 目录树结构，展示分类层级与各模块职责，降低新贡献者的理解成本。

**自动分类建议**：基于 URL 路径特征与文档名规则，提交新链接时可获得初步分类建议，减少维护者的手工归类工作。

**全文检索支持**：集成轻量级本地检索工具，允许用户在已索引的资源标题与注释中进行关键词搜索，提升查找效率。

**变更历史审计**：所有增删改操作均通过 Git 变更记录追踪，支持回溯任意版本的外链集合状态，便于审查与回滚。

## 应用场景

技术团队内部知识库建设：团队 Leader 或技术文档维护者可以使用 LinkSphere 作为团队技术周报、内部培训材料的补充外链索引，统一存放会议中提及的参考文章、视频教程或官方文档链接，避免知识碎片化散落在聊天记录或邮件中。

个人技术阅读清单管理：开发者可将平时浏览过程中发现的优质技术资源通过提交方式纳入 LinkSphere，构建个人长期跟踪的阅读清单，并利用分类与标签机制按主题（如 Rust 异步编程、Kubernetes 调度、数据库 MVCC 实现）聚合内容，便于周期性回顾。

开源项目文档参考集成：开源项目维护者可以在项目 README 或文档中引用 LinkSphere 中收录的相关外部资源，以减轻自身文档中直接维护大量外链的负担，同时利用 LinkSphere 的审计历史确保引用链接的长期有效性。

技术调研前置信息收集：在进行新技术选型或竞品分析时，研究人员可以快速浏览 LinkSphere 的相应分类，获取已有社区讨论、性能测试报告或替代方案对比文章，作为调研报告的初始参考基线。

## 快速开始

以下操作基于 Unix-like 系统（Linux / macOS），Windows 用户可使用 Git Bash 或 WSL 环境执行。

```bash
# 克隆项目仓库
git clone https://github.com/example/linksphere.git
cd linksphere

# 安装依赖（基于 Python 3.10+ 环境）
pip install -r requirements.txt

# 运行本地索引服务（默认监听 127.0.0.1:8000）
python run.py --mode server --port 8000
```

完成上述三步后，可通过浏览器访问 http://127.0.0.1:8000 查看索引界面，或使用 CLI 工具进行资源查询与新增操作。

## 贡献指南

新增外部资源链接：通过 CLI 命令 `python run.py add --url <URL> --category <分类名>` 提交新链接，系统自动尝试提取标题并建议标签。提交者需在注释中简要说明该资源的核心价值或适用场景。

更新已有记录：若发现链接失效、标题变更或分类不准确，使用 `python run.py edit --id <记录ID>` 进入交互式编辑模式，修改后系统会记录变更日志。

分类体系优化：若现有分类无法覆盖新提交的资源类型，可在 `config/categories.yaml` 中提议新增分类，并附带该分类的范围描述与示例链接。提议需在 Pull Request 中说明理由。

测试覆盖：所有核心解析、验证、检索逻辑变更必须包含对应的单元测试，测试文件位于 `tests/` 目录下，使用 pytest 框架运行。提交前执行 `pytest tests/` 确保全部通过。

文档同步：任何影响用户使用方式的功能变更（CLI 参数、配置项、API 响应结构）须同步更新 `docs/` 下对应的用户或开发者指南，确保文档与代码一致。

## 常见问题

**问：项目是否会定期检查收录链接的可用性？**

答：是的，项目内置了链接有效性检查模块 `core/validator.py`，可通过 `python run.py validate --check-all` 命令触发全量检查。检查结果会生成报告，标记出状态码异常或响应超时的链接，供维护者参考处理。推荐每季度执行一次全量检查，确保资源索引的健康度。

**问：我可以只使用项目的分类和标签体系，而不运行本地服务吗？**

答：可以。分类和标签定义存储在 `config/categories.yaml` 和 SQLite 数据库中，你可以直接解析这些文件来获得分类结构。此外，CLI 提供了 `python run.py export --format json` 命令，将所有资源数据连同分类标签导出为 JSON 格式，便于在其他工具或静态站点生成器中使用。

**问：如果源站内容被删除或迁移，项目如何处理？**

答：项目本身不缓存或备份源站内容，仅保留 URL 与元数据。当 validator 检测到链接持续不可用时，维护者会将该记录标记为“失效”状态，并记录最后验证时间。用户可通过 `python run.py list --status stale` 查看所有失效链接。若源站提供了重定向目标，validator 会记录新 URL 并提示维护者更新。
