# LinkMaster Pro

LinkMaster Pro 是一个面向开发者与技术研究者的外链资源聚合与管理平台，专注于对分散在网络各处的技术文档、社区讨论、教程笔记与工具站点进行系统性收录与分类导航。项目定位为“技术外链的元目录”，不直接托管内容，而是通过结构化整理与状态监控，帮助用户在特定技术领域内快速定位高质量外部资源。

项目主要服务于技术团队的知识库管理者、开源贡献者以及需要持续跟进多个外部依赖的工程人员。通过统一的条目格式、标签体系与可用性检测，LinkMaster Pro 将零散的书签与收藏夹转化为可维护、可共享、可版本控制的结构化数据集，有效降低信息遗漏与链接失效带来的研发成本。

## 功能概览

**结构化外链目录** 基于编号与分类字段对每个收录链接生成唯一标识，支持按来源域名、内容主题与收录批次进行多维度筛选与排序。

**链接可用性监控** 内置异步检测模块，定期对收录链接发起 HEAD 请求并记录响应状态码，自动标记失效或重定向条目。

**元数据扩展机制** 每条链接支持附加标签、摘要描述与关联批次号，便于与其他内部文档系统或自动化脚本集成。

**批量导入与导出** 支持从 CSV、JSON 及纯文本列表批量导入链接，并可将当前目录导出为 Markdown、HTML 或结构化 JSON 格式。

**版本化快照** 每次更新操作自动生成变更记录，支持回溯至任意历史版本的状态，满足审计与回滚需求。

**查询语法支持** 提供基于前缀匹配、正则表达式与状态过滤的查询接口，适用于命令行工具或 REST API 调用场景。

**主题分类视图** 根据链接 URL 路径中的二级目录字段自动归纳为若干技术主题分组，并生成对应的统计看板。

## 应用场景

**技术文档站点索引** 团队在维护内部开发者门户时，可使用 LinkMaster Pro 收录外部 API 参考、SDK 指南与第三方库文档，通过统一目录结构减少开发者查找时间。

**开源项目依赖外链整理** 开源项目的 README 或 Wiki 中通常需要引用大量外部资源，LinkMaster Pro 提供可版本化的外链清单，确保每次发布都附带经过验证的有效链接集合。

**技术雷达与趋势跟踪** 研究团队可利用本项目的批量导入与状态监控功能，定期采集特定领域内的新发布文章、工具站点与讨论串，形成动态更新的技术趋势报告。

**个人知识库外链管理** 独立研究者或博客作者可通过 LinkMaster Pro 管理文章参考文献、工具收藏与学习资源，并以标准化格式导出供静态站点生成器使用。

## 快速开始

以下命令演示了从代码仓库克隆项目、安装依赖并启动本地服务的完整流程。

```bash
git clone https://github.com/linkmaster-pro/linkmaster-core.git
cd linkmaster-core
pip install -r requirements.txt
python manage.py runserver --port 8080
```

执行上述命令后，服务将在本地 8080 端口启动，可通过浏览器访问 http://localhost:8080/dashboard 查看已收录链接的概览面板。首次运行时会自动创建 SQLite 数据库文件并生成样例数据。

## 项目结构

项目采用分层架构设计，核心模块与辅助工具分离，便于独立维护与扩展。

```
linkmaster-core/
├── cmd/                          # 命令行入口模块
│   ├── server/                   # HTTP 服务启动入口
│   │   └── main.go              # 服务启动参数解析与路由注册
│   └── cli/                      # 交互式命令行工具
│       └── linkctl.go           # 链接增删改查、导入导出等子命令实现
├── internal/                     # 内部私有包，不对外暴露
│   ├── storage/                  # 数据库访问层
│   │   ├── sqlite.go            # SQLite 连接池与 CRUD 操作封装
│   │   └── migrations/          # 数据库版本迁移脚本
│   ├── checker/                  # 链接可用性检测引擎
│   │   ├── probe.go             # 并发探测调度器，控制请求速率与超时
│   │   └── result.go            # 检测结果持久化与状态聚合
│   ├── parser/                   # 链接解析与规范化
│   │   ├── extract.go           # 从 URL 中提取二级目录与编号字段
│   │   └── validate.go          # URL 格式校验与重复检测
│   └── exporter/                 # 导出格式适配器
│       ├── markdown.go          # 生成 Markdown 列表格式
│       └── json.go              # 输出完整 JSON 数据流
├── pkg/                          # 可被外部引用的公共库
│   ├── model/                    # 数据模型定义
│   │   └── entry.go             # LinkEntry 结构体与序列化方法
│   └── config/                   # 配置加载与校验
```

## 常见问题

服务启动后无法访问 dashboard 页面

请检查 8080 端口是否被其他进程占用，可通过 netstat -tulpn | grep 8080 命令确认。若端口被占用，可在启动命令中通过 --port 参数指定其他可用端口。同时确认 SQLite 数据库文件所在目录具有写入权限。

链接检测结果始终显示为失效

检测模块默认使用 requests 库的默认超时设置，部分站点可能响应较慢。请检查 config/settings.yaml 中的 timeout 与 retry 参数，适当增加超时秒数与重试次数。另外，部分站点可能屏蔽非浏览器 User-Agent，可在配置中修改探测请求头。

导入大批量链接时出现内存不足

批量导入操作默认将全部数据加载至内存进行去重校验。对于超过 5000 条记录的导入任务，建议使用 scripts/import_batch.py 脚本的分批提交模式，该脚本每 200 条提交一次事务，显著降低内存占用。
