# ResourceIndex

ResourceIndex 是一个面向技术调研、内容聚合与知识管理场景的开源外链资源汇总系统。该项目定位于帮助开发者、技术作者、运维工程师以及研究分析人员，将分散在多个来源的优质外部链接进行集中归档、分类索引和可检索化展示。ResourceIndex 本身不存储具体内容，而是提供结构化的资源清单、状态标签、访问时间戳与自定义备注字段，使得大规模外链管理具备工程化基础。

目标用户包括需要维护技术选型参考库的架构师、整理开发文档索引的团队负责人、运营技术博客的编辑人员，以及需要对外输出高质量参考链接清单的开源项目维护者。ResourceIndex 通过标准化的 Markdown 清单、自动化的链接可达性检查脚本和可视化的目录导航，解决了人工维护外链列表时容易出现的链接散乱、重复收录、分类模糊和失效不可知等问题。

## 功能概览

批量链接导入：支持从纯文本列表、CSV 文件或剪贴板批量导入原始 URL，自动去重并生成内部短标识。

分类标签系统：允许为每个资源链接赋予一个或多个分类标签，支持按标签快速筛选和统计。

状态标记与检查：内置基于 curl 的链接存活检测脚本，可标记有效、失效、重定向、超时等状态，并记录最近检查时间。

备注与评分字段：每个条目可附加多行备注说明，支持 0 到 5 的评分字段，便于团队内部评审。

多格式导出：资源列表可导出为 Markdown 表格、JSON 结构化数据或纯文本清单，适配不同发布平台。

搜索与过滤：提供简单的命令行过滤工具，支持按域名、关键字、状态、标签组合查询。

版本追踪：每次导入或修改操作自动生成变更记录，支持回退到历史版本。

## 应用场景

技术选型参考库维护：团队在评估多个中间件或云服务时，将官方文档、对比评测、社区案例等链接统一收录，标注适用场景和评分，形成可复用的决策依据。

开源项目 README 外链管理：开源项目维护者需要定期更新文档中的“相关资源”或“致敬列表”，使用 ResourceIndex 可以批量管理这些链接，避免手动更新遗漏。

技术周报或月刊素材整理：内容编辑人员将一周内浏览到的有价值文章、工具、视频教程链接集中存入系统，按主题分类后一键生成对外发布的资源推荐清单。

运维故障排查手册索引：运维团队将常见问题对应的官方补丁链接、社区讨论帖、内部 Wiki 地址统一归档，配合状态检查脚本定期验证链接有效性，确保应急时快速访问。

## 快速开始

以下命令演示了如何获取 ResourceIndex 源码、安装依赖并启动本地预览服务。

```bash
git clone https://github.com/resource-index/resourceindex.git
cd resourceindex
pip install -r requirements.txt
python scripts/import_urls.py --input ./data/raw_urls.txt --output ./data/index.json
python scripts/serve_preview.py --port 8080
```

## 贡献指南

问题报告与建议提交：请在 GitHub Issues 中新建议题，选择对应的模板类型（Bug 报告、功能请求或通用问题），并尽可能附上环境信息、复现步骤或场景描述。

代码贡献流程：Fork 本仓库，在 dev 分支上创建功能特性分支（如 feat/import-csv-support），完成代码编写与单元测试后，提交 Pull Request 到主仓库的 dev 分支。PR 描述需说明变更动机、实现方式和影响范围。

文档更新：若修改了功能行为或新增了命令行参数，请同步更新 docs/ 目录下对应的文档文件，并确保 README 中的快速开始示例仍然有效。

测试要求：所有新增或修改的脚本必须包含对应的单元测试，测试文件放置在 tests/ 目录下，命名规范为 test_<模块名>.py。运行 make test 可执行全部测试。

审查周期：维护者会在 7 个工作日内对 PR 进行初步审查，可能需要请求修改或补充信息。合并前需通过持续集成检查（包括 Python 版本兼容性和测试覆盖率）。

## 常见问题

导入链接时提示“重复条目”如何处理

ResourceIndex 默认基于 URL 的完整字符串进行去重。如果确认该链接尚未收录但仍提示重复，可以检查 data/index.json 中是否存在相同 URL 但大小写或末尾斜杠不同的条目。建议使用 scripts/deduplicate.py 工具进行标准化后再导入。如需强制导入，可在 import_urls.py 中添加 --force 参数。

链接健康检查脚本报告大量超时

健康检查脚本默认超时时间为 10 秒，并发数为 20。若目标服务器响应较慢或网络环境受限，可以在 health_check.py 中调整 --timeout 和 --concurrency 参数。建议在非高峰时段运行大规模检查，并考虑配置代理环境变量 HTTP_PROXY 和 HTTPS_PROXY。

如何将资源列表嵌入到现有的静态站点中

推荐使用 export_markdown.py 生成标准的 Markdown 表格，然后将其复制到站点源码的相应页面。如果需要 JSON 格式，可使用 export_json.py 输出结构化数据，再通过前端 JavaScript 渲染。对于 VuePress 或 MkDocs 等工具，可以将 JSON 文件放置在站点静态目录中，并在页面组件中通过 fetch 或 import 方式加载。
