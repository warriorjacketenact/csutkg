# ResourceBridge

ResourceBridge 是一个面向技术社区与开发者的外链资源归集与导航系统。该项目定位于解决多源、多批次、大规模外部链接的收录、分类、检索与状态监控问题，帮助个人开发者、技术团队、内容运营者以及研究机构高效管理分散在各类内容平台上的参考资源链接。ResourceBridge 本身不存储视频、文档或任何第三方内容，仅提供链接的索引、元数据提取与健康度检查能力。

本项目适用于需要批量维护大量外链资源的场景，尤其适合作为技术博客的参考链接后台、开源项目的外部依赖索引、在线课程的资料库管理，或是企业内部知识库的外部参考链接治理工具。ResourceBridge 以静态站点生成方式输出结构化的链接目录，支持通过标签、来源域名、收录批次、状态码等多维度筛选，并提供链接有效性自动巡检机制。

## 功能概览

**批量链接导入** 支持从 CSV、JSON 或纯文本列表批量导入外部 URL，自动解析协议、域名、路径参数，并生成唯一资源 ID。

**自动元数据提取** 对每个收录的 URL 执行轻量级 HEAD 请求与页面标题抓取，提取资源类型、最后修改时间、内容长度等基础信息，作为后续分类与检索的依据。

**多维度标签分类** 允许为每个链接绑定多个自定义标签（如 tutorial、video、api-reference、blog、paper），并支持按标签组合进行快速筛选。

**链接健康状态监控** 周期性（可配置间隔）检测所有已收录链接的可访问性，记录 HTTP 状态码、响应时间、重定向链，对失效链接发出告警并标记状态。

**批次管理与追溯** 每个收录批次单独记录，包含批次编号、导入时间、链接数量、通过率等元数据，便于追踪不同来源或不同时间段添加的资源质量。

**静态站点导出** 将链接目录、标签聚合、状态报告导出为纯静态 HTML 页面，可直接部署到任何 Web 服务器或 CDN 上，无需动态后端即可访问。

**开放 API 接口** 提供 RESTful API 用于链接查询、状态更新、标签管理等操作，支持与其他内部系统或自动化脚本集成。

## 应用场景

**技术博客外部链接治理** 技术作者在撰写文章时引用了大量外部参考链接，使用 ResourceBridge 统一收录并定期检查链接有效性，避免因外链失效导致文章参考价值下降。

**开源项目依赖索引管理** 开源项目的 README、文档站点中往往包含大量指向第三方库、工具、教程的外部链接，通过 ResourceBridge 集中管理这些链接，并为每个链接标记用途分类与版本兼容性说明，降低文档维护成本。

**在线课程资料库后台** 在线教育平台或培训机构在课程中提供大量延伸阅读材料与视频资源链接，利用 ResourceBridge 按课程、章节、知识点维度组织链接，并自动监控外部视频源的可访问性，及时替换失效资源。

**企业知识库外链整合** 企业内部 Wiki 或知识管理系统中分散着大量外部参考链接，通过 ResourceBridge 统一归集并建立链接台账，定期生成链接健康度报告，减少员工访问无效链接的时间损耗。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git、Node.js 18.x 及以上版本与 npm。

```bash
# 克隆仓库
git clone https://github.com/your-org/resource-bridge.git
cd resource-bridge

# 安装项目依赖
npm install

# 初始化配置并启动开发服务器
cp .env.example .env
npm run dev
```

启动成功后，访问 http://localhost:3000 即可进入 ResourceBridge 管理面板。首次启动会自动创建 SQLite 数据库并生成示例链接数据，供体验与测试使用。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 项目运行时环境，推荐使用 nvm 管理多版本 |
| npm | 9.x 或更高 | 依赖包管理器，随 Node.js 一同安装 |
| SQLite3 | 3.35.0 或更高 | 嵌入式数据库，用于存储链接元数据与状态历史，无需额外安装服务 |
| Git | 2.30.0 或更高 | 用于克隆仓库及版本控制操作 |
| 可用磁盘空间 | 至少 500 MB | 用于存放 SQLite 数据库文件、日志及静态导出产物，大规模部署建议预留 2 GB 以上 |

## 项目结构

```
resource-bridge/
├── src/
│   ├── core/                         # 核心业务逻辑模块
│   │   ├── link-manager.ts           # 链接增删改查、状态管理核心类
│   │   ├── batch-processor.ts        # 批次导入、批次状态追踪
│   │   └── health-checker.ts         # 链接健康度巡检引擎
│   ├── api/                          # RESTful API 路由定义
│   │   ├── routes/                   # 按资源类型拆分路由文件
│   │   └── middleware/               # 认证、日志、限流中间件
│   ├── extractors/                   # 元数据提取器集合
│   │   ├── title-extractor.ts        # 从 HTML 中提取页面标题
│   │   ├── type-detector.ts          # 根据 Content-Type 推断资源类型
│   │   └── custom-parsers/           # 支持针对特定域名定制解析逻辑
│   ├── exporters/                    # 静态站点生成与导出模块
│   │   ├── static-generator.ts       # 将链接目录渲染为 HTML 页面
│   │   └── report-builder.ts         # 生成健康度报告与统计图表
```

## 常见问题

**Q: ResourceBridge 是否会缓存或存储第三方链接的内容本身？**

A: 不会。ResourceBridge 仅存储链接的 URL、元数据（标题、类型、状态码）以及批次归属信息，不下载或缓存任何第三方内容。所有对外部链接的请求仅用于状态检查与标题提取，请求头模拟普通浏览器行为，不发送任何敏感信息。

**Q: 健康检查功能对目标服务器是否会产生过大压力？**

A: 健康检查默认采用单线程顺序执行，每个请求超时时间为 5 秒，并发连接数限制为 10。对于大规模链接集（超过 5000 条），建议将巡检间隔配置为 24 小时以上，并安排在业务低峰时段运行。检查过程中仅发送 HEAD 请求，不下载响应体，对目标服务器的资源消耗极低。

**Q: 如何迁移 ResourceBridge 的数据到另一台服务器？**

A: 直接复制 data/database.sqlite 文件即可完成迁移。若需同时迁移日志与静态导出产物，可将整个 data/ 和 public/ 目录打包。停止运行中的进程后再复制文件可避免数据库锁定问题。不同操作系统之间迁移 SQLite 文件无需额外转换，但需注意 Node.js 版本一致。
