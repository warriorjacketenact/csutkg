# Tangalooma Resource Indexer

Tangalooma Resource Indexer (TRI) 是一个面向技术内容聚合与外部资源导航的开源工具集。项目定位为半自动化外链资源整理平台，帮助开发者、技术写作者与内容运营人员从大量无序的 URL 中提取元信息、建立分类索引，并生成结构化的资源清单。

项目主要解决以下问题：当团队或个人积累了数百个外部链接（如文档、教程、案例、工具站），缺乏统一管理方式；手动整理 markdown 列表耗时且易出错；无法快速对链接进行主题归类、重复检测与可用性筛查。TRI 提供可执行的命令行工作流，将原始 URL 列表转化为带有分类前缀、批次标记与基础校验结果的资源目录。

本批次为第 479/1241 批，共处理 250 个资源链接，全部来源于 tangaloomahouse.com 域下的 film 路径。TRI 默认将其视为外部内容引用池，并依据路径中的分类代码（如 opx、rbs、guj）自动生成标签候选。

## 功能概览

**批量 URL 解析器**：支持从纯文本文件或标准输入读取大量 URL，自动识别协议、域名、路径层级与查询参数，提取分类代码与资源标识符。

**分类代码映射表**：内置可扩展的分类字典，将 URL 路径中的三位字母代码（如 opx、rbs、guj）映射为预设内容主题（如运营、后端、前端、算法等），便于后续分组统计。

**资源清单生成器**：根据用户指定模板（如 markdown 表格、json、yaml）输出结构化的资源列表，保留原始 URL 原样，同时附加解析后的元数据字段（分类、文件名、批次号）。

**可用性探测模块**：对每个 URL 发送轻量级 HEAD 请求，记录状态码与响应时间，标记异常链接（超时、4xx、5xx），输出健康度报告。

**去重与变更检测**：基于 URL 完整字符串与路径文件名计算哈希指纹，对比历史批次记录，识别新增、删除或重命名的资源链接。

**增量更新机制**：支持通过配置文件指定上一批次的输出目录，仅处理变化部分，避免全量重新解析。

**多格式导出**：支持 markdown 列表、HTML 目录、JSON API 风格数据、CSV 表格四种输出格式，适配不同下游系统（静态站点、数据看板、CMS）。

## 应用场景

技术文档站点维护：开源项目文档站通常需要引用大量外部参考资料（规范、论文、博客）。维护者每月使用 TRI 扫描一次外部链接库，自动生成“扩展阅读”章节的 markdown 源码，并标记失效链接，确保文档质量。

内容聚合周刊编辑：技术社区运营人员每周收集 50-100 个新链接，通过 TRI 解析分类代码，按主题分组后直接插入周刊模板。分类映射表可自定义为“前端/后端/DevOps/AI”等栏目，节省手动归类时间。

内部知识库巡检：企业技术中台团队使用 TRI 定期巡检 Confluence 或 GitLab Wiki 中的外链，检测是否仍可访问。结合 CI 计划任务，每周生成一份外链健康报表发送至团队邮箱。

迁移辅助工具：当站点域名更换或路径重构时，TRI 提供批量 URL 前缀替换功能，并自动校验新地址的可达性，减少人工逐条修改的遗漏风险。

## 快速开始

以下步骤演示如何从原始 URL 文件生成第一份资源目录。

```bash
# 克隆项目仓库
git clone https://github.com/tangalooma/tri-indexer.git
cd tri-indexer

# 安装依赖（Python 3.9+ 环境）
pip install -r requirements.txt

# 将原始 URL 列表保存为 raw_links.txt（每行一个 URL）
# 运行解析器，输出 markdown 格式的资源列表
python tri_parser.py --input raw_links.txt --output resources.md --format markdown
```

执行后，resources.md 将包含所有 URL 的原始链接、分类标签与基础状态信息。若需启用可用性探测，增加 `--check` 参数。
