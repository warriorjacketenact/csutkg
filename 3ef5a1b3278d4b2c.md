# ResourceBridge 开源外链资源聚合平台

ResourceBridge 是一个面向技术社区与内容创作者的轻量化外链资源汇总与导航系统。项目定位于将分散在互联网各处的优质视频、文档、工具与学习材料，通过结构化方式组织为可检索、可维护的资源索引库，帮助开发者、研究人员与教育工作者快速定位特定主题的参考内容。

项目本身不存储任何实体媒体文件，仅提供元数据记录与跳转链接。所有资源条目均以纯文本形式维护于项目仓库中，支持版本控制、协作编辑与自动化校验。ResourceBridge 适用于个人知识管理、团队技术文档沉淀、公开课配套资料整理以及社区共建内容库等使用场景。

## 功能概览

自动抓取与解析资源元信息：支持从目标页面提取标题、描述、封面图与发布日期，自动填充资源卡片字段，减少人工录入工作量。

多级分类与标签系统：每个资源条目可归属至多个分类节点，并附加自定义标签，支持按主题、难度、语种、资源类型等多维度筛选。

全文检索与模糊匹配：基于轻量级倒排索引实现标题与描述的快速检索，支持拼音首字母与同义词扩展，提升查找效率。

资源状态监控与失效检测：定期对已收录链接执行可用性检查，标记失效或重定向条目，辅助维护者及时更新或下架。

用户收藏与自定义列表：允许注册用户将感兴趣的资源加入个人收藏夹，并创建主题列表（如“机器学习入门路线”“Go 语言微服务实践”）进行二次组织。

导入导出与批量操作：支持 CSV、JSON、Markdown 表格格式的批量导入导出，便于与其他工具链（如 Notion、Airtable、Hugo）集成。

访问统计与热度排序：记录资源点击次数与最近访问时间，提供按热度、更新时间、收录时间排序的浏览视图。

团队协作与审核流程：内置简单的 PR 审核辅助工具，可标记待审条目、添加审核备注，并记录操作日志，适合多维护者场景。

## 应用场景

技术团队内部文档中心：研发团队可将常用的 API 参考、架构设计文档、故障排查案例等外链统一收录至 ResourceBridge，替代散落在聊天记录或浏览器书签中的零散链接，新成员入职时可快速浏览团队推荐的学习材料。

开源项目配套资料站：开源项目维护者可在项目 README 中引用 ResourceBridge 生成的资源列表，为贡献者提供编码规范、提交指南、设计文档的外部参考链接，降低沟通成本。

在线课程与培训辅助：教育机构或独立讲师可将每节课涉及的拓展阅读、视频讲解、在线实验环境链接整理为资源集合，学员可按章节或主题检索，课后自主延伸学习。

技术社区内容聚合：技术论坛或内容平台可基于 ResourceBridge 构建“每周精选”或“热门资源榜”，由社区成员共同提交并投票推荐优质内容，形成自组织的知识筛选机制。

个人知识库外链管理：知识工作者可将浏览器中积累的大量书签迁移至 ResourceBridge，配合标签与备注功能实现分类管理，避免书签栏无序膨胀，同时支持多设备同步访问。

## 快速开始

以下命令演示如何从 GitHub 克隆项目、安装依赖并启动开发服务器。请确保已安装 Node.js 18.x 及以上版本与 pnpm 8.x。

```bash
# 克隆仓库
git clone https://github.com/your-org/resource-bridge.git
cd resource-bridge

# 安装依赖
pnpm install

# 复制环境变量模板并填写必要配置
cp .env.example .env.local

# 初始化数据库（SQLite）
pnpm run db:init

# 导入示例资源数据
pnpm run seed

# 启动开发服务器（默认端口 3000）
pnpm run dev
```

启动成功后，访问 http://localhost:3000 即可浏览本地资源列表。管理员后台路径为 /admin，默认账号密码请参阅 .env.local 中的初始化配置。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或 20.x LTS | 是 | 运行时环境，推荐使用最新的 LTS 版本 |
| pnpm 8.x 或 9.x | 是 | 包管理器，本项目使用 pnpm workspace 管理 monorepo |
| SQLite 3 | 是 | 默认内置数据库，无需额外安装，生产环境可切换至 PostgreSQL |
| Redis 7.x | 否 | 可选缓存层，用于提升检索与热度统计性能，不配置则使用内存缓存 |
| Docker 与 Docker Compose | 否 | 用于一键启动完整依赖栈（含 Redis 与 PostgreSQL） |
| Git 2.30 以上 | 是 | 版本控制，用于克隆仓库与贡献提交 |
| 现代浏览器（Chrome/Firefox/Edge 最新两版） | 否 | 仅前端开发调试需要，生产部署无浏览器依赖 |

## 贡献指南

我们欢迎并鼓励社区贡献，无论是新增资源条目、改进文档、修复缺陷还是开发新功能。请遵循以下流程以确保协作顺畅。

提交新资源条目：在 data/resources/ 目录下创建新的 YAML 文件，按照模板填写标题、原始链接、分类标签、简要描述及来源站点。运行 pnpm run validate 校验格式，通过后发起 Pull Request 至 main 分支。

报告问题或建议功能：使用 GitHub Issues 模板提交，请详细描述复现步骤、预期行为与实际表现，并附上环境信息（浏览器版本、Node.js 版本、操作系统）。功能建议请说明使用场景与预期收益。

代码变更流程：Fork 本仓库，在 feat/ 或 fix/ 前缀的分支上进行开发。确保所有单元测试通过（pnpm test），并新增或更新相关测试用例。提交信息请遵循 Conventional Commits 规范。发起 PR 后，维护者将在 2 个工作日内进行 Review。

文档完善：docs/ 目录下的文档采用 Markdown 编写，欢迎修正错别字、补充示例、完善 API 说明或翻译为其他语言。文档变更无需通过完整的测试流水线，但需确保格式一致且链接有效。

本地开发环境设置：参考快速开始章节完成依赖安装与数据库初始化。如需调试爬虫或解析器，可运行 pnpm run dev:crawler 启动单独的任务进程，并查看 logs/ 目录下的调试输出。

## 常见问题

部署后访问页面提示“数据库连接超时”如何处理？

此问题通常由数据库连接字符串配置错误或数据库服务未启动导致。请检查 .env.local 中 DATABASE_URL 的值是否正确，确保包含主机、端口、用户名、密码与数据库名称。若使用 SQLite，请确认文件路径可写且父目录存在。可运行 pnpm run db:status 查看连接状态。若使用 PostgreSQL 或 MySQL，请确认网络防火墙已放行对应端口。

导入大量资源时界面卡顿或超时怎么办？

批量导入操作建议通过命令行工具执行，而非通过 Web 界面。使用 pnpm run import:csv --path=./data/bulk.csv 可异步处理大批量数据，进度会写入 logs/import.log。若仍需在 Web 端操作，请分批次（每批 50-100 条）导入，并确保服务器内存不低于 512MB。对于超过 1000 条的导入任务，推荐使用 Redis 队列配合后台 Worker 异步处理，详见 docs/operations/batch-import.md。

如何自定义资源条目的展示样式或排序逻辑？

前端展示逻辑位于 apps/web/src/components/resource/ 目录下，您可修改 ResourceCard 组件调整卡片布局，或修改 ResourceList 中的排序函数。排序策略支持按热度、时间、标题字母序，可于 apps/web/src/lib/sorting.ts 中扩展新策略。样式定制使用 Tailwind CSS，全局主题变量定义于 apps/web/src/styles/globals.css，支持深色模式切换。
