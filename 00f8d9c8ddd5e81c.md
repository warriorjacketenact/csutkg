# Resource Atlas

Resource Atlas is a high-performance, community-driven technical resource aggregation and navigation system. It is designed for developers, researchers, and IT professionals who need to organize, catalog, and rapidly access a vast, distributed collection of external technical references, documentation, tools, and media assets. The project addresses the critical challenge of link rot, resource fragmentation, and inefficient discovery by providing a structured, version-controlled, and queryable index over a large corpus of URLs, transforming a flat list of bookmarks into a navigable knowledge base.

Target users include solution architects building reference architectures, DevOps engineers curating deployment playbooks, data scientists maintaining experiment tracking dashboards, and open-source contributors managing project documentation stacks. Resource Atlas does not host the underlying content but acts as a robust cataloging layer, offering metadata extraction, health monitoring, and semantic categorization for each indexed resource. By treating each URL as a first-class entity with tags, status, and contextual notes, the platform enables teams to maintain a living inventory of their external technical dependencies and learning materials.

## 功能概览

**批量资源注入** 支持通过命令行、API 或批量文件导入的方式将大量 URL 纳入系统，自动进行去重和格式校验。

**自动元数据抓取** 对每个提交的 URL 执行异步 HTTP 探查，尝试提取页面标题、描述、内容类型和最后修改时间，丰富索引信息。

**标签与分类引擎** 基于 URL 路径模式、域名和内容特征自动推荐标签，同时允许用户自定义分层分类体系，支持多标签交叉筛选。

**健康状态监控** 定期对已索引的 URL 执行可用性检查，标记失效链接、证书过期或内容变更，生成健康报告仪表板。

**全文与字段检索** 提供对 URL、标题、描述、标签和分类的布尔与模糊检索能力，支持高级查询语法如前缀匹配和排除词。

**结构化导出与集成** 支持将索引数据导出为 JSON、CSV 或 Markdown 表格格式，方便嵌入到其他文档系统或静态站点生成器中。

## 应用场景

**技术文档中心化索引** 大型研发团队可将分散在多个 Wiki、代码仓库和外部博客中的技术设计文档、API 规范和部署指南的链接统一收录，通过 Resource Atlas 建立可检索的文档目录，新成员能快速定位到权威参考源。

**开源项目外部依赖清单管理** 开源维护者可以使用 Resource Atlas 记录项目所依赖的所有外部库、工具链和基准测试数据集的官网或镜像地址，当上游地址变更或失效时，监控功能可及时发出告警，避免构建流程中断。

**研究实验资源备份与共享** 数据科学团队在进行模型训练或数据分析实验时，可将使用的数据集下载链接、预训练模型权重地址和相关论文补充材料链接全部纳入一个实验批次索引，确保实验结果可复现，并方便团队内部共享上下文。

**个人技术知识库构建** 技术爱好者可将长期积累的编程教程、在线课程、技术播客和工具推荐链接通过 Resource Atlas 进行统一管理和标签化，利用检索和分类功能快速找回特定主题的资料，替代传统的浏览器书签夹。

## 快速开始

以下步骤指导您在一台运行主流 Linux 发行版或 macOS 的机器上完成 Resource Atlas 的源码克隆、依赖安装和服务启动。

```bash
# 克隆项目仓库
git clone https://github.com/resource-atlas/resource-atlas.git
cd resource-atlas

# 安装项目依赖（使用 pip 和虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 执行数据库迁移并启动开发服务器
export FLASK_APP=app.py
export FLASK_ENV=development
flask db upgrade
flask run --host=0.0.0.0 --port=8080
```

服务启动后，访问 http://localhost:8080 即可进入 Resource Atlas 的 Web 管理界面。首次启动将自动创建默认管理员账户，密码输出在控制台日志中。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.9 - 3.11 | 是 | 核心应用运行时，低于 3.9 版本将无法解析类型注解语法 |
| PostgreSQL 13+ | 是 | 主数据库，用于存储资源索引、标签和用户数据；不支持 SQLite 作为生产环境替代 |
| Redis 6.2+ | 是 | 用于缓存元数据探查结果和消息队列代理，支持异步任务 |
| Node.js 16+ | 否 | 仅当需要构建前端静态资源或进行 UI 开发时才需要安装 |
| Nginx 或 Apache | 否 | 生产部署时推荐作为反向代理服务器，处理静态文件和负载均衡 |
| 网络连通性 | 是 | 服务器需要能够访问外网以执行 URL 健康检查和元数据抓取 |
| 磁盘空间 >= 10GB | 是 | 用于存储日志、缓存和数据库文件；实际需求随索引规模线性增长 |

## 项目结构

项目采用分层架构设计，核心模块、服务层、数据访问层和基础设施层清晰分离。

```
resource-atlas/
├── app/                                 # 核心应用包
│   ├── api/                             # RESTful API 路由与控制器
│   │   ├── v1/                          # API 版本 1 实现
│   │   │   ├── resources.py             # 资源 CRUD 操作端点
│   │   │   └── health.py                # 健康检查与状态汇总端点
│   │   └── middleware/                  # 认证、日志、限流中间件
│   ├── core/                            # 核心业务逻辑层
│   │   ├── crawler/                     # 异步元数据抓取与解析器
│   │   ├── indexer/                     # 资源索引与标签引擎
│   │   └── monitor/                     # 定期健康检查调度器
│   ├── models/                          # 数据模型与 ORM 映射（SQLAlchemy）
│   │   ├── resource.py                  # Resource 实体定义
│   │   ├── tag.py                       # 标签与分类模型
│   │   └── user.py                      # 用户与权限模型
│   ├── services/                        # 服务层，编排核心模块
│   │   ├── import_service.py            # 批量导入和去重服务
│   │   ├── export_service.py            # JSON/CSV/Markdown 导出服务
│   │   └── query_service.py             # 高级检索与过滤服务
│   └── utils/                           # 通用工具函数
│       ├── http_client.py               # 带重试和超时控制的 HTTP 客户端
│       └── validators.py                # URL 格式与协议校验器
├── migrations/                          # 数据库迁移脚本（Alembic）
│   └── versions/                        # 按时间命名的迁移版本文件
├── tests/                               # 单元测试与集成测试
│   ├── unit/                            # 各模块独立单元测试
│   └── integration/                     # API 与数据库集成测试
├── scripts/                             # 运维与辅助脚本
│   ├── batch_import.py                  # 命令行批量导入工具
│   └── health_check_runner.py           # 手动触发全量健康检查
├── docs/                                # 完整文档源文件
├── requirements.txt                     # 生产环境 Python 依赖清单
├── requirements-dev.txt                 # 开发与测试环境额外依赖
├── Dockerfile                           # 容器化构建定义
├── docker-compose.yml                   # 本地开发环境编排（PostgreSQL + Redis）
├── app.py                               # Flask 应用工厂入口
├── config.py                            # 环境配置管理（开发、测试、生产）
└── README.md                            # 项目主文档
```

## 常见问题

问：Resource Atlas 是否会存储或缓存外部 URL 指向的实际文件内容？

答：不会。Resource Atlas 只存储元数据（如 URL、标题、描述、标签、最后检查时间和状态码），不缓存或代理外部资源的内容。所有对外部资源的访问仍然直接由用户的客户端发起。元数据抓取仅提取 HTML 头部信息，不会下载或存储页面正文、图片或附件。

问：索引的 URL 数量上限是多少？性能会随着规模增长而下降吗？

答：项目在设计上未设定硬性上限。在官方测试环境中，单实例 PostgreSQL 数据库配合 Redis 缓存可稳定支持超过 50 万条资源记录的索引，检索响应时间维持在 200 毫秒以内。当索引量超过百万级别时，建议启用数据库分区和只读副本进行横向扩展。监控任务的执行频率和并发度可通过环境变量进行调优，以平衡检查时效性与系统负载。

问：如何迁移已有的浏览器书签或其他格式的链接列表到 Resource Atlas？

答：项目提供了批量导入脚本 scripts/batch_import.py，支持读取 CSV、JSON Lines 和 Netscape HTML 书签导出格式。您可以将书签文件转换为上述格式之一，然后执行导入命令。脚本会自动进行去重，并将不规范的 URL 尝试修复（例如补全协议头）。具体用法请参考文档用户手册中的导入章节。
