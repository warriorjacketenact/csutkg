# IndexLink Hub

IndexLink Hub 是一个面向技术研究者和信息分析人员的结构化外链资源归集系统。该项目不生产原始内容，而是通过人工筛选与机器辅助分类，将分散于互联网各处的技术文档、工具站、数据面板及社区讨论入口进行统一编录，形成可复用、可追溯的外部知识索引。IndexLink Hub 服务于需要频繁查阅多源技术资料、进行竞品信息对比或搭建个人知识库的开发者与分析师，帮助其降低链接散落带来的管理成本，提升信息检索的确定性。

本项目当前批次为第 1215/1241 批，共计收录 250 个外部资源链接。所有链接均保留原始 URL 结构与协议头，确保引用路径与源站发布时完全一致，避免因自动补全或协议升级导致的访问偏差。IndexLink Hub 本身不存储任何第三方页面内容，仅提供元数据级别的定位信息。

## 功能概览

**批量链接导入** 支持通过 CSV 或 JSON 格式批量导入外部 URL 列表，系统自动解析域名、路径参数及查询字符串，并按预设规则生成内部记录标识。

**多维度标签分类** 每个资源条目可附加技术栈、内容类型、更新频率、语言等自定义标签，便于后续按主题快速筛选。

**原始 URL 保真输出** 系统严格保留用户提交的每一个 URL 原始形态，包括协议头（http/https）、三级域名、端口号及路径大小写，杜绝任何形式的自动改写或规范化。

**链接状态巡检** 内置轻量级 HTTP 状态检查器，可定期对已收录链接进行可达性探测，标记异常条目并生成报告，辅助维护索引健康度。

**结构化文档生成** 根据内部元数据自动生成 Markdown 格式的资源列表文档，支持按批次、标签或收录时间导出，直接用于项目文档或技术博客附录。

**全文检索与过滤** 提供基于关键词的站内检索能力，支持对 URL 片段、标签组合及描述字段进行精确匹配与模糊查询。

**版本差异对比** 针对同一 URL 在不同批次中的收录状态或标签变更进行对比，便于追踪外部资源的内容迁移或失效过程。

## 应用场景

技术调研阶段的素材归集：在进行新框架选型或竞品分析时，研究人员往往需要同时打开数十个相关项目的官网、文档仓库、性能测试报告和社区讨论帖。IndexLink Hub 允许将这批链接以批次为单位统一录入，并附加调研日期、优先级和初步评价，避免在浏览器书签中零散存放。

项目文档中的参考资料附录：开源项目或企业内部技术方案在撰写设计文档时，需要引用大量外部标准、协议规范和参考实现。IndexLink Hub 导出的结构化链接列表可直接嵌入 Markdown 文档的附录章节，且原始 URL 保真特性确保了引用路径的长期有效性。

知识库自动化构建的数据源层：个人或团队在搭建二次元知识库（如基于 Docusaurus 或 VuePress 的技术 wiki）时，可将 IndexLink Hub 作为外部链接数据源层，通过 API 或导出功能将整理好的链接列表注入到知识库的对应页面中，实现内容与引用的分离管理。

链接资产定期审计：对于运营超过一年以上的技术博客或导航站点，大量外链可能因源站改版、域名过期或协议变更而失效。IndexLink Hub 的状态巡检功能可每周自动扫描全部收录链接，输出失效列表，帮助维护者及时更新或移除问题条目。

## 快速开始

以下命令演示了如何从 GitHub 克隆 IndexLink Hub 项目、安装依赖并在本地开发环境中启动服务。

```bash
git clone https://github.com/indexlink-hub/indexlink-hub.git
cd indexlink-hub
npm install
npm run start
```

执行上述命令后，服务默认运行于 http://localhost:5173 。访问该地址即可进入链接管理面板，通过界面上的“新建批次”按钮导入包含 URL 列表的 CSV 文件，或使用命令行工具通过 API 接口提交数据。

## 项目结构

```
indexlink-hub/
├── apps/
│   ├── web/                           # 前端管理控制台 (React + Vite)
│   │   ├── src/
│   │   │   ├── pages/                 # 批次管理、链接列表、巡检报告页面
│   │   │   ├── components/            # 可复用的表格、标签输入、状态指示器
│   │   │   └── api/                   # 前端 API 客户端封装
│   │   └── package.json
│   └── worker/                        # 后台任务执行器 (Node.js + BullMQ)
```

## 常见问题

**Q: IndexLink Hub 会对收录的 URL 进行内容抓取或缓存吗？**

A: 不会。IndexLink Hub 仅存储用户提交的 URL 字符串及其元数据（标签、批次号、收录时间）。系统不会主动抓取页面内容，也不会对任何外部资源进行本地缓存或代理。状态巡检功能仅发送 HTTP HEAD 或 GET 请求以检查响应状态码，不解析或存储响应体。

**Q: 为什么我必须按照原始 URL 原样输出，不能自动将 http 升级为 https？**

A: 部分资源站可能未配置 HTTPS 证书，或 HTTPS 服务与 HTTP 服务返回的内容不同（例如某些旧版管理后台仅监听 80 端口）。自动协议升级会导致用户无法访问目标页面，甚至触发证书警告。IndexLink Hub 的设计原则是绝对忠实于用户提交的原始数据，由用户或资源提供方自行决定协议迁移策略，系统不做任何预判性改写。

**Q: 状态巡检报告显示某个链接不可达，但我在浏览器中可以正常访问，是什么原因？**

A: 可能存在以下几种情况：巡检服务器与您的本地网络环境不同（如地理区域、DNS 解析差异）；目标站点启用了反爬机制，拦截了非浏览器 UA 的请求；或巡检超时阈值设置过短（默认 5 秒），而目标页面响应较慢。您可以在 `worker` 的配置文件中调整 `CHECK_TIMEOUT` 和 `USER_AGENT` 参数，或针对特定域名配置白名单绕过规则。
