# ResourceBridge

ResourceBridge 是一个面向技术团队与内容研究者的外链资产整理与健康度监控工具。该项目并非传统的爬虫或采集系统，而是一套围绕外链资源进行收录、分类、可用性巡检与关联分析的辅助框架。其核心定位是帮助运维人员、技术文档工程师以及知识库管理者，将分散在各类运营系统、第三方服务平台或历史遗留项目中的外部链接，转化为可查询、可追踪、可审计的结构化数据资产。

ResourceBridge 本身不存储具体业务数据，而是通过统一的条目模型对每个外链进行规范化封装，支持自定义标签、归属项目、添加时间、最后检测状态等元信息。项目附带完整的命令行交互界面与计划任务示例，便于集成到现有的 CI/CD 流水线或监控告警体系中。对于需要长期维护大量外部引用链接的团队而言，ResourceBridge 能够有效降低链接失效、域名过期、内容变更等风险带来的维护成本。

## 功能概览

**条目化外链登记** 每条链接以独立条目形式入库，自动解析 URL 基础信息并生成唯一标识，支持批量导入与单条追加。

**自定义分类标签体系** 用户可对每条外链赋予多个自定义标签，支持按项目、按领域、按优先级等多维度组织资源视图。

**健康状态周期性巡检** 内置轻量级 HTTP 状态检查器，可配置巡检周期与超时策略，自动记录每次巡检结果与响应时间。

**变更历史追溯** 每次修改条目标签、备注或状态均记录操作日志，便于审计追踪与回溯分析。

**结构化数据导出** 支持将条目列表导出为 JSON、CSV 以及 Markdown 表格格式，满足文档生成与数据交换需求。

**命令行交互工具集** 提供完整的 CLI 子命令，涵盖添加、查询、更新、删除、巡检、导出等全生命周期操作。

**可插拔存储适配层** 默认使用 SQLite 本地存储，同时提供 PostgreSQL 适配接口，方便生产环境迁移。

## 应用场景

**技术文档中心的外链资产整理** 大型项目的技术文档往往引用大量外部规范、SDK 下载页、API 参考文档等链接。ResourceBridge 可将这些分散在文档仓库中的外链统一收录，定期巡检可用性，并在链接失效时快速定位影响面。

**开源项目 README 与官网的资源清单维护** 许多开源项目在 README 或官网中维护了合作伙伴、相关项目、学习资源等外链列表。随着时间推移这些链接容易过期或被篡改。通过 ResourceBridge 可以建立版本化的资源台账，每次发布前执行一次全量巡检，确保对外引用的可靠性。

**内部知识库的外部引用监控** 企业内部 Wiki 或知识管理系统中常包含指向供应商门户、行业标准组织、技术社区等外部链接。ResourceBridge 能够作为旁路监控工具，定期生成健康度报告，帮助知识库管理员及时发现并替换异常链接。

## 快速开始

以下操作基于 Linux/macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装项目依赖
pip install -r requirements.txt

# 初始化本地数据库并执行示例数据加载
python cli.py init
python cli.py import --source examples/sample_links.json
python cli.py check --all
```

## 贡献指南

**提交 Issue 报告问题** 在提交代码变更之前，请先在 GitHub Issues 中搜索是否已有类似问题。若为新问题，请提供清晰的重现步骤、运行环境信息以及相关的日志片段。

**遵循代码风格规范** 项目使用 Black 作为 Python 代码格式化工具，行宽度设置为 88。提交前请运行 `black .` 进行格式化，并确保通过 flake8 静态检查。

**补充单元测试** 所有新增功能或修复缺陷均应附带对应的单元测试用例，测试代码放置在 `tests/` 目录下，命名遵循 `test_*.py` 规范。运行 `pytest` 确保全部测试通过。

**更新文档与示例** 若变更涉及用户可见的功能或配置项，请同步更新 `docs/user/` 下的对应文档，并在 `examples/` 中补充或调整示例数据。

**提交 Pull Request** 分支命名采用 `feature/` 或 `fix/` 前缀，PR 描述中需说明变更动机、实现方式以及测试覆盖情况。PR 至少需要一名项目维护者审核通过后方可合并。

## 常见问题

**Q: ResourceBridge 是否支持对外链内容进行全文抓取或分析？**

A: 不支持。ResourceBridge 仅执行轻量级 HTTP HEAD 或 GET 请求以检测链接的可达性与响应状态码，不会解析或存储响应体内容。对于需要内容分析的需求，建议结合其他专门的数据处理工具。

**Q: 巡检任务如何与现有监控系统集成？**

A: 项目提供的 CLI 子命令 `check` 支持以 JSON 格式输出巡检结果，可通过标准输出重定向或管道传递给下游系统。同时，`examples/cron_example.sh` 中给出了通过 cron 定时执行并将结果写入日志的示例，便于与 Prometheus 或 ELK 等监控体系对接。

**Q: 能否同时管理多个独立项目的资源列表？**

A: 可以。通过标签体系中的 `project` 预留字段实现多项目隔离，每条条目可归属至不同项目标识。查询时支持按项目标签过滤，导出时也可按项目维度分别生成报告。
