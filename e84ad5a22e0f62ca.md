# LinkVault Resource Aggregator

LinkVault is a high-performance, metadata-driven external resource aggregation and navigation system designed for technical documentation teams, open-source project maintainers, and research-oriented developers. The project addresses the fundamental challenge of managing large-scale, heterogeneous URL collections across multiple domains by providing structured ingestion, semantic classification, and a lightweight static generation pipeline that produces human-readable indices from raw link inventories.

Target users include DevOps engineers maintaining dependency mirrors, technical writers curating reference libraries, and solution architects assembling technology-specific reading lists. LinkVault does not host content; it transforms unstructured URI lists into navigable, queryable catalogs with minimal runtime overhead.

## 功能概览

**Bulk URI Ingestion Engine** - Accepts plain-text URL lists, CSV exports, and JSON feeds; performs deduplication, protocol normalization validation, and malformed entry quarantine.

**Metadata Enrichment Pipeline** - Resolves HTTP response headers, extracts content-type, last-modified timestamps, and server fingerprints; optionally performs HEAD requests in parallel with configurable concurrency limits.

**Hierarchical Classification System** - Applies rule-based tagging using path prefix analysis, domain authority scoring, and MIME type inference; supports user-defined category overrides via YAML configuration.

**Static Site Generator** - Renders categorized link collections as responsive HTML tables, Markdown indices, or JSON API endpoints; includes search filtering by tag, domain, or last-updated date.

**Health Monitoring Subsystem** - Periodically rechecks stored URLs for availability, updates status flags, and generates expiry warnings for resources that return 4xx or 5xx responses over three consecutive checks.

**Access Control Layer** - Supports IP-based allowlisting, basic authentication, and read-only API tokens for private team deployments without external identity providers.

**Export Adapters** - Provides plugins to export filtered subsets as BibTeX, CSV, or plain Markdown lists compatible with static site generators like Hugo and MkDocs.

## 应用场景

Technical documentation teams managing multi-product reference libraries can use LinkVault to maintain a single source of truth for all external references, automatically flagging broken links during weekly build pipelines. The classification engine groups resources by technology domain, reducing duplicate manual curation efforts.

Open-source project maintainers publishing community resource pages can integrate LinkVault into their CI/CD workflows. Every pull request that modifies the URL manifest triggers a regeneration of the public resource index, ensuring the published page always reflects the approved link set.

Research groups conducting systematic literature reviews benefit from the metadata enrichment pipeline. The system captures content-type and server fingerprints for each resource, enabling reproducible environment snapshots and rapid validation of reference availability before paper submission.

Internal platform engineering teams can deploy LinkVault as an internal developer portal component. The health monitoring subsystem provides real-time visibility into the status of external dependencies, allowing teams to proactively replace failing endpoints before they impact build or deployment processes.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# Install dependencies using Poetry
poetry install --no-dev

# Run the ingestion pipeline with the sample manifest
python -m linkvault ingest --input samples/urls.txt --output dist/index.html
```

For production deployments, copy the `config/linkvault.yaml.example` to `config/linkvault.yaml` and adjust the `concurrency`, `timeout`, and `health_check_interval` parameters. The static output directory defaults to `dist/` and can be served by any HTTP server.

## 贡献指南

We welcome contributions from the community. Please follow the steps below to ensure a smooth review process.

1.  Fork the repository and create a feature branch from `main`. Use a descriptive branch name such as `feature/enhanced-csv-exporter` or `fix/classification-rule-typo`.

2.  Install development dependencies using `poetry install --with dev` and set up the pre-commit hooks by running `pre-commit install`. This ensures that code formatting and static checks are applied automatically before each commit.

3.  Write unit tests for any new functionality or bug fixes using pytest. Place tests in the appropriate `tests/unit/` or `tests/integration/` directory. Ensure that the test suite passes locally with `poetry run pytest` before submitting a pull request.

4.  Update the relevant documentation under the `docs/` folder. For user-facing changes, provide a clear description of the new behavior and include code examples where applicable.

5.  Submit a pull request with a detailed description of the changes, the motivation behind them, and any relevant issue numbers. Pull requests must pass all CI checks, including linting, type checking, and the full test matrix, before they can be merged.

## 常见问题

**Q: How does LinkVault handle URL changes or permanent redirects?**

A: The enricher follows HTTP 301 and 302 redirects up to a configurable limit (default 5 hops). The final resolved URL is stored in the `canonical_url` field, and the original input URL is preserved in the `source_url` field for audit purposes. If a redirect target changes frequently, the health monitor will flag the resource as unstable after three consecutive redirect chain variations.

**Q: Can I run LinkVault without an internet connection for the enricher step?**

A: Yes. The ingestion and classification steps do not require network access if you provide a pre-populated metadata cache. You can generate a cache file from a previous online run using the `--cache-export` flag, then reuse it in offline environments with `--cache-import`. The exporter will use the cached metadata for all resources, skipping HEAD requests entirely.

**Q: What is the maximum number of URLs that LinkVault can process in a single batch?**

A: The system is designed for batches of up to 50,000 URLs per manifest. Processing time depends on the concurrency setting and network latency. With the default concurrency of 20 workers, a batch of 10,000 URLs typically completes within 15 to 20 minutes. For larger inventories, we recommend splitting the manifest into multiple files and using the `--merge` flag to combine the output indices.
