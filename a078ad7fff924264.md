# TangalooHouse Film Resource Aggregator

TangalooHouse Film Resource Aggregator 是一个面向影视行业从业者、学术研究者与内容创作者的综合性外链资源汇总平台。该项目并非一个传统的电影数据库或流媒体前端，而是一个高度结构化的技术资源导航站，专注于收集、分类与索引来自 TangalooHouse 域下的深度影片分析页面、幕后技术文档与制作素材参考链接。

本项目定位为专业级的技术外链枢纽，目标用户包括影视特效总监、后期制作协调人、独立制片人、电影学研究员以及自动化爬虫开发者。通过本项目提供的规范化 URL 结构与分类索引，用户可以快速定位到特定编码批次（如 paz-1564、eek-5293）对应的技术拆解文档或拍摄日志，从而大幅提升资料检索效率，避免在庞杂的原始站点中盲目搜索。

作为一个处于第 480/1241 批维护周期的活跃项目，TangalooHouse Film Resource Aggregator 持续收录并校验来自上游源站的最新外链资源，确保所有条目保持可访问性与内容完整性。本项目不存储任何实体媒体文件，仅提供元数据索引与外部链接跳转服务，严格遵循来源站点的 robots 协议与访问频率限制。

## 功能概览

**批次化资源索引** 项目以 250 条链接为一个索引批次，当前批次包含完整的 film 目录下各子分类链接，每个链接均附带唯一的影片编码与资源缩写标识，便于进行增量更新与差异比对。

**结构化 URL 解析** 自动解析 TangalooHouse 域下的三级路径结构（/film/{编码}/{资源缩写}.html），将 URL 拆解为可查询的元数据字段，支持按编码前缀、资源类型、文件后缀进行筛选。

**本地缓存校验** 提供本地元数据缓存机制，在用户执行链接批量检查时，自动记录响应状态码与内容哈希值，辅助识别失效链接或内容变更。

**静态站点生成支持** 项目输出格式兼容主流静态站点生成器（如 Hugo、Jekyll），用户可将索引结果编译为静态 HTML 导航页面，部署于内网或公网服务器。

**命令行交互接口** 内置轻量级命令行工具，支持导入链接列表、执行存活检测、导出分类报告等操作，适合集成至自动化运维脚本。

**多维度分类视图** 支持按影片类型、编码范围、更新时间、资源格式等多个维度对链接进行分组展示，满足不同角色的检索习惯。

**导入导出兼容性** 支持 CSV、JSON、Markdown 表格三种数据交换格式，便于与外部数据库或电子表格软件协同工作。

## 应用场景

**影视后期制作资料归档** 后期制作团队在完成项目后，可将涉及的所有参考链接、技术文档地址通过本项目的索引格式进行统一归档，确保未来需要回溯特定镜头处理方案时能够快速定位。例如，将 paz-1564 批次的特效合成笔记链接与项目编号关联存储。

**学术研究文献引用管理** 电影学研究人员在分析特定导演的视觉风格或特定时期的技术演进时，可将 TangalooHouse 中的深度分析页面作为引用来源，通过本项目的结构化索引生成参考文献列表，避免手动整理 URL 时的格式错误与遗漏。

**自动化内容监控系统构建** 运维开发人员可利用本项目的链接批次列表，构建定时监控脚本，每日检测指定 URL 的可用性与内容变更情况，当源站更新技术文档时自动发送通知，确保团队成员第一时间获取最新资料。

**内部知识库种子数据** 企业或机构在搭建内部影视技术知识库时，可将本项目收录的 250 条链接作为初始种子数据，导入知识库系统后逐步添加内部批注、标签与关联文档，形成定制化的学习资源库。

## 快速开始

执行以下命令完成项目克隆、依赖安装与初始索引构建：

```bash
git clone https://github.com/your-org/tangaloohouse-aggregator.git
cd tangaloohouse-aggregator
pip install -r requirements.txt
python indexer.py --batch 480 --input urls_batch_480.txt --output index.html
```

其中 urls_batch_480.txt 文件包含本批次全部 250 条 URL 链接，每行一条。执行完毕后，将在 output 目录生成分类索引页面 index.html 及对应的元数据缓存文件 cache.db。
