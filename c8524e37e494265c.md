# LinkVault Resource Aggregator

LinkVault 是一个面向开发者、研究人员和技术内容创作者的高质量外链资源汇总与导航系统。该项目并非简单的书签管理器，而是一个具备元数据提取、链接可用性检测、分类标签生成以及全文检索能力的结构化资源仓库。项目核心定位为“技术资源的入口层”，通过对分散在网络各处的优质文档、教程、工具站点、API 参考和社区讨论进行系统化收录与组织，帮助用户从信息过载中解脱出来，快速定位到真正有价值的外部内容。

本项目适用于需要频繁查阅多源技术资料的前后端工程师、运维人员、算法研究员以及技术写作人员。LinkVault 解决了传统浏览器书签缺乏协作性、无法批量验证有效性、难以按上下文关联检索的痛点，同时为团队内部知识库的构建提供了开放的数据导入与导出接口。

## 功能概览

**批量链接导入与解析** 支持从纯文本列表、Markdown 表格或 CSV 文件批量导入 URL，自动解析页面标题、描述字符集及最后修改时间，并剔除重复或无效条目。

**链接可用性监控** 内置定时任务，可按照每日或每周频率对已收录链接进行 HTTP 状态码检查，标记异常链接并生成失效报告，方便管理员及时清理或更新。

**多维度标签分类** 允许用户为每个资源链接打上多个自定义标签，支持按标签层级筛选和组合查询，实现从泛领域到具体技术栈的逐级过滤。

**全文检索与快速跳转** 基于倒排索引的本地搜索引擎，支持对标题、描述以及用户自定义备注的模糊匹配与精确匹配，搜索结果可直接点击跳转至目标页面。

**资源快照与存档** 对关键链接自动生成网页存档快照，在原始页面不可访问时提供备用的文本内容浏览，降低单点失效风险。

**数据导入导出** 提供 JSON、YAML 和 Markdown 格式的完整数据导出，同时支持从浏览器书签 HTML 文件和主流 RSS 订阅源导入数据，方便与其他工具链集成。

**用户协作与评论** 支持多用户环境下的资源备注与评论功能，便于团队内部共享使用心得、版本兼容性警告或替代方案建议。

**RESTful API 接口** 所有核心功能均通过 JSON API 暴露，允许第三方脚本或自动化流水线对资源库进行增删改查操作，满足 CI/CD 集成场景。

## 应用场景

**技术团队内部文档中心** 开发团队可将 LinkVault 部署为内部知识导航页，集中存放项目依赖的官方文档地址、内部设计提案链接、代码评审记录以及运维手册入口，配合标签分类快速区分前端、后端、数据工程等不同领域。

**开源项目 README 外链托管** 开源项目维护者可使用 LinkVault 管理项目 README 中列举的所有参考链接、教程文章和社区论坛，通过可用性监控确保文档中的每一个外部引用长期有效，避免读者遇到死链。

**个人学习路径组织** 自学者可按技术栈或学习阶段建立资源集合，例如将 Go 语言基础教程、标准库文档、性能优化博客和开源项目源码链接统一收纳，借助全文检索在复习时快速找回特定知识点对应的文章。

**技术资讯聚合与筛选** 内容创作者或技术编辑可利用 LinkVault 订阅多个技术博客和新闻站点的 RSS 源，将新发布的文章自动导入为待审链接，经人工筛选后生成每周优质内容推荐列表，发布到团队内部或对外输出。

## 快速开始

以下命令演示了如何在本地环境中克隆代码仓库、安装依赖并启动 LinkVault 服务。

```bash
git clone https://github.com/your-org/linkvault.git
cd linkvault
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver --host=0.0.0.0 --port=8080
```

若使用 Docker 进行部署，可执行以下替代方案：

```bash
docker build -t linkvault:latest .
docker run -d -p 8080:8080 --name linkvault linkvault:latest
```

首次启动后，系统会在 `data/` 目录下生成默认的 SQLite 数据库文件，并自动加载内置的示例资源分类。管理员账户的初始密码将在启动日志中输出，请及时修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 核心运行环境，需包含 pip 和 venv 模块 |
| SQLite | 3.35 及以上 | 默认元数据存储引擎，支持 JSON 字段类型 |
| Redis | 7.0 及以上 | 用于缓存查询结果和存储定时任务队列，可选但推荐 |
| Node.js | 18.x LTS | 仅用于前端静态资源构建，生产环境可忽略 |
| Nginx | 1.24 及以上 | 生产环境反向代理与静态资源服务，非必需但建议 |
| 系统内存 | 1 GB 以上 | 实际占用量取决于资源条目总数与快照缓存策略 |
| 磁盘空间 | 10 GB 以上 | 用于存放 SQLite 数据库、快照存档和日志文件 |

## 常见问题

**问：LinkVault 能否处理需要登录或带有反爬机制的网页链接？**

答：LinkVault 默认仅对公开可访问的 URL 进行标题解析和快照生成。对于需要 Cookie 或 Session 认证的页面，系统会记录 HTTP 401/403 状态并跳过内容提取，但链接本身仍会被保存。用户可在配置文件中为特定域名设置自定义请求头或代理，但不建议用于绕过网站访问限制，请遵守目标网站的 robots.txt 协议。

**问：数据库从 SQLite 迁移到 PostgreSQL 的步骤是什么？**

答：项目使用 SQLAlchemy 作为 ORM，理论上支持多种关系型数据库。迁移时需在 `config/production.toml` 中修改 `database.url` 为 PostgreSQL 连接字符串，然后执行 `python manage.py migrate` 自动建表。现有 SQLite 数据可通过 `scripts/export_json.py` 导出，再通过 API 导入或使用 `psql` 的 COPY 命令批量迁移。建议在低峰期进行，并提前做好完整备份。

**问：如何配置定时链接检查的邮件通知？**

答：在 `config/scheduler.toml` 中设置 `[notifier]` 段落，填写 SMTP 服务器地址、端口、发件人邮箱和授权密码，并将 `alert_emails` 列表设置为接收通知的地址。重启服务后，检查任务会按照 `check_interval_hours` 参数执行，若发现连续两次检查均失败的链接，系统会生成摘要报告并发送至指定邮箱。用户也可在管理面板中按标签或资源分组订阅不同级别的告警。
