# ResourceBridge

ResourceBridge 是一个面向技术研究者、开源开发者与内容策展人的外链资源汇总与管理平台。项目定位为“技术资源的中转枢纽”，通过结构化梳理分散于网络各处的优质外链，帮助用户快速定位特定领域的学习材料、工具文档与参考实现。

项目面向三类核心用户：需要持续追踪技术资料的个人开发者、需要为团队建立知识库的技术管理者、以及需要将大量外链资源进行规范化整理与展示的内容运营者。ResourceBridge 不生产内容，但提供一套标准化的外链收录、分类与展示框架，使原始 URL 集合能够以清晰、可维护、可扩展的形式呈现给最终读者。

## 功能概览

- **外链批量收录**：支持一次性导入大量原始 URL，自动识别链接形态并保持原样输出，杜绝协议补全或域名改写。
- **分类索引生成**：根据 URL 路径特征与资源主题，将链接自动归入对应类别小节，便于读者按需查阅。
- **项目结构可视化**：内置 ASCII 目录树生成器，完整展示资源站点的文件与目录组织方式，降低新贡献者的理解成本。
- **文档自动拼装**：基于预定义章节模板，将项目简介、功能列表、安装要求、常见问题等内容与资源列表组合为完整 README 文档。
- **依赖与环境检查**：提供安装要求表格，明确列出运行所需的所有依赖项及其版本约束，减少环境配置错误。
- **多场景适配**：支持个人博客、团队知识库、技术导航站等多种输出形态，可根据实际需求调整页面布局与样式。

## 应用场景

- **个人技术博客的外链整理页**：技术博主可在博客中单独开设“推荐资源”页面，将长期积累的参考链接通过 ResourceBridge 进行结构化展示，方便读者按类别浏览。
- **开源项目文档的参考资料附录**：开源项目的维护者可以将项目中引用的外部文档、依赖项目、社区讨论等链接统一整理为 ResourceBridge 资源列表，作为项目 README 或 Wiki 的补充附录。
- **企业内部技术团队的共享知识库**：团队可将日常开发中用到的 API 文档、规范说明、最佳实践文章等外链集中收录，通过 ResourceBridge 生成内部导航页面，减少重复搜索时间。
- **技术社区的活动资源汇总**：社区组织者可将线上或线下活动中提及的所有参考资料、演讲幻灯片链接、示例代码仓库等统一汇总，形成活动专属资源页。

## 快速开始

以下命令演示如何获取 ResourceBridge 项目代码、安装必要依赖并启动本地预览服务。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run build
```

执行完成后，生成的静态页面默认输出至 `dist/` 目录，可直接部署至任何支持静态文件托管的平台。

## 安装要求

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Node.js | >= 18.0.0 | 项目构建与依赖管理运行环境 |
| npm | >= 9.0.0 | 包管理器，用于安装项目依赖 |
| Git | >= 2.30.0 | 版本控制工具，用于克隆仓库与提交变更 |
| Markdown 解析器 | 无强制版本 | 用于将源数据渲染为 README 文档，推荐使用 remark 或 marked |
| 静态服务器 | 无强制版本 | 本地预览使用，推荐 serve 或 http-server |
| 可选：Pandoc | >= 2.0 | 如需输出 PDF 或 DOCX 格式文档时使用 |
| 可选：Shell 环境 | Bash 4.0+ | 运行辅助脚本时使用，Windows 用户可使用 WSL 或 Git Bash |

## 贡献指南

ResourceBridge 欢迎来自社区的各种形式贡献，包括但不限于新增分类规则、优化渲染逻辑、补充测试用例以及完善文档。请遵循以下步骤提交贡献：

1. 在 GitHub 上 Fork 本仓库，并克隆至本地开发环境。确保本地 Node.js 版本满足安装要求中列出的版本约束。
2. 新建一个以 `feature/` 或 `fix/` 为前缀的分支，例如 `feature/add-rust-classifier`。在该分支上进行代码修改或文档更新。
3. 提交变更前，请运行 `npm test` 确保所有现有测试用例通过。若新增功能，请同步添加对应的测试用例至 `tests/` 目录。
4. 编写清晰的 Commit Message，遵循 Conventional Commits 规范，简要说明变更内容与动机。
5. 推送分支至远程仓库，并通过 GitHub 界面发起 Pull Request。PR 描述中请说明变更的背景、实现方式以及影响范围。

项目维护者会在收到 PR 后的五个工作日内进行 Review，并给出合并或修改意见。

## 常见问题

**问：ResourceBridge 是否支持除 Markdown 之外的其他输出格式？**

答：当前版本默认输出为标准 Markdown 文档。若需要输出 PDF、HTML 或 DOCX 格式，可借助 Pandoc 等外部工具进行格式转换。后续版本计划原生支持多格式导出。

**问：如何更新已收录的资源列表？**

答：资源列表以 `data/raw/links.json` 文件作为输入源。您可以直接编辑该 JSON 文件，添加或删除 URL 条目，然后重新运行 `npm run build` 即可生成更新后的 README 文档。

**问：分类规则是否可以自定义？**

答：可以。您可以在 `config/custom.json` 中覆盖默认分类规则，支持基于 URL 路径关键字、文件扩展名或自定义正则表达式进行分类。具体配置格式请参考 `docs/configuration.md` 中的说明。
