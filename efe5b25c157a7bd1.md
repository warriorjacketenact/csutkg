# LinkVault

LinkVault 是一个面向技术研究者和信息分析人员的结构化外链资源归集与导航系统。该项目不生产内容，而是通过严谨的索引机制将分散于互联网各处的优质长尾资源进行统一收束、分类与可复现的本地化管理，解决个人知识管理中"收藏即遗忘"和"链接失效无回溯"两大核心痛点。

LinkVault 以纯静态 Markdown 为数据载体，配合轻量级元数据提取工具，帮助用户在本地建立一份可检索、可审计、可版本控制的外链资产表。目标用户包括独立研究员、技术文档工程师、开源项目维护者以及需要系统化整理浏览器书签的深度互联网使用者。

## 功能概览

**批量链接摄入学** 支持从文本、HTML 渲染页及常见书签导出文件（HTML 格式）中批量提取 URL，自动去重并归一化协议头。

**元数据自动补全** 对摄入的每个 URL 执行可配置的元数据抓取任务，包括页面标题、响应状态码、内容类型（Content-Type）及最后修改时间。

**标签与分类引擎** 基于 URL 路径特征、域名分段及可自定义的正则规则集，为每个链接自动生成一级分类标签（如 /m/ 路径下的数字 ID 归入"长尾文档"类）。

**本地全文检索** 通过内置的 SQLite 全文索引虚拟表（FTS5）支持对链接标题、备注及分类标签的快速布尔检索与短语匹配。

**快照引用管理** 记录每个链接的归档状态，支持关联本地离线快照文件路径或外部 Wayback Machine 存档链接，形成多级备份策略。

**结构化导出** 支持将索引库中的链接列表按指定分类或标签筛选后导出为 Markdown 表格、JSON 数组或纯文本 URL 清单，便于嵌入其他文档或工作流。

## 应用场景

**技术文献外部引用整理** 技术文档编写者在完成一篇综述或技术方案时，通常需要引用数十个外部来源。LinkVault 允许将这些引用链接统一录入并记录访问状态，防止因链接变更导致文档引用失效。

**浏览器书签库的定期审计** 用户可将浏览器导出的书签 HTML 文件导入 LinkVault，系统自动检测其中返回 4xx 或 5xx 状态码的失效链接，生成一份待清理报告，协助维持书签库的健康度。

**项目外部依赖溯源** 开源项目维护者可以使用 LinkVault 记录项目文档、README 或 Wiki 中出现的所有外部链接，建立一份独立的依赖资源映射表，便于在新版本发布前核对所有引用的可访问性。

**特定领域信息监控** 研究人员可将若干关注的信息源 URL 收录至 LinkVault，通过定时任务运行链接状态检查脚本，获取这些资源的更新频率和可用性变化趋势，为信息获取策略调整提供数据依据。

## 快速开始

以下操作序列可在 Ubuntu 22.04 LTS 或 macOS Monterey 及以上版本中完整执行。

```bash
# 克隆项目仓库
git clone https://github.com/example-org/linkvault.git
cd linkvault

# 安装 Python 依赖（建议使用虚拟环境）
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并运行完整摄入流程（示例数据）
python linkvault.py --init --import samples/import_list.txt
```

上述命令执行完毕后，系统会在 ./data/ 目录下生成 linkvault.db 数据库文件与 logs/ 运行日志。用户可通过 linkvault.py --search "关键词" 进行本地检索。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9 及以上 | 是 | 核心运行时环境，用于执行摄入、索引与导出逻辑 |
| SQLite 3.35.0 及以上 | 是 | 内置数据库引擎，需支持 FTS5 扩展以提供全文检索能力 |
| requests 2.28.0 | 是 | 用于发起 HTTP/HTTPS 请求以检测链接状态与获取元数据 |
| beautifulsoup4 4.11.0 | 否 | 在需要解析 HTML 页面标题与摘要时作为可选增强，可通过 --no-html-parser 禁用 |
| lxml 4.9.0 | 否 | 作为 beautifulsoup4 的解析器后端，提供更快的 HTML 解析速度 |
| pytest 7.2.0 | 否 | 仅当需要运行项目自带单元测试套件时安装，生产环境可不装 |

## 项目结构

```
linkvault/
├── linkvault.py               # 主入口脚本，聚合命令行接口与调度逻辑
├── config/
│   └── linkvault.ini          # 配置文件模板，包含 user_agent、超时、并发数等参数
├── core/
│   ├── __init__.py
│   ├── loader.py              # 链接加载器：实现从纯文本、HTML 书签、CSV 中解析 URL
│   ├── indexer.py             # 索引引擎：调用 SQLite FTS5 构建全文索引
│   ├── checker.py             # 状态检查器：并发 HEAD/GET 请求，记录状态码与响应耗时
│   └── exporter.py            # 导出模块：支持 markdown、json、csv 格式输出
├── utils/
│   ├── __init__.py
│   ├── network.py             # 网络请求封装，含重试策略与代理适配
│   └── parser.py              # HTML 元数据解析辅助函数（标题、编码、关键词）
├── data/
│   ├── linkvault.db           # SQLite 数据库主文件（由 linkvault.py --init 生成）
│   └── snapshots/             # 可选离线快照存储目录（按 URL 哈希分片存放）
├── tests/
│   ├── test_loader.py         # 针对不同输入格式的 URL 提取单元测试
│   ├── test_checker.py        # HTTP 状态检查的模拟响应测试（依赖 pytest）
│   └── fixtures/              # 测试用静态数据（模拟 HTML 书签文件等）

│   ├── user-guide.md          # 用户手册（安装、配置、日常使用流程）
│   ├── config-reference.md    # 完整配置项字典表

│   └── operations.md          # 运维手册：数据库维护、日志管理与故障诊断
├── scripts/
│   ├── cron_check.sh          # 用于 crontab 调用的链接状态定期巡检脚本
│   └── import_from_browser.sh # 从常见浏览器书签导出文件自动触发摄入的辅助脚本
├── requirements.txt           # Python 依赖清单（pip freeze 格式）
└── README.md                  # 当前文档
```

## 常见问题

**问：导入大量链接时程序报错 "OperationalError: database is locked"，应如何解决？**

答：此错误通常出现在 SQLite 并发写入场景。LinkVault 默认使用 WAL 模式（Write-Ahead Logging）以改善并发性能，但若用户同时运行多个 linkvault.py 进程，仍可能触发锁冲突。解决方案为：检查系统进程列表，确保没有遗留的 linkvault.py 后台任务；若问题持续，可在配置文件中将 database.pool_size 设置为 1，强制串行化数据库操作。同时建议在导入大规模链接列表（超过 5000 条）时，使用 --batch-size 参数将单次提交控制在 500 条以内。

**问：自动补全的页面标题与人工标注的备注发生冲突时，系统如何取舍？**

答：LinkVault 的元数据补全流程设计为"非侵入式"。当用户通过 linkvault.py --update 命令触发元数据刷新时，系统仅更新数据库的 metadata_updated_at 时间戳及自动提取的标题字段，而用户手动维护的 remark 字段（备注）和 custom_tag 字段（自定义标签）不会被自动覆盖。若用户希望重置备注为当前页面标题，需显式使用 --overwrite-remark 参数。这一设计旨在保护用户的人工整理成果。

**问：能否将 LinkVault 索引库与团队其他成员共享？**

答：可以。LinkVault 的数据库文件（data/linkvault.db）为单一文件，可直接复制分发。但需注意，若数据库中含有绝对路径类型的本地快照引用（snapshot_path 字段），在其他机器上这些路径可能无效。建议团队协作时统一使用相对路径，并将 snapshots/ 目录纳入版本控制或共享存储。此外，LinkVault 支持通过 export 命令生成纯文本格式的链接清单，该清单可直接用于跨团队分发，不包含本地路径依赖。
