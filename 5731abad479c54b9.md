# VODLink Indexer

VODLink Indexer 是一个面向技术开发者和内容研究人员的结构化视频点播资源索引系统。该项目不对任何视频内容进行存储、转码或分发，而是提供一套标准化的外链采集、分类与检索框架，帮助用户在海量分散的 VOD 资源中快速定位目标条目。项目定位为纯静态资源导航工具，适用于个人媒体库整理、学术数据采集、爬虫规则测试以及自动化链接巡检等场景。

## 功能概览

- **资源链接归一化采集**：基于确定性规则对原始 URL 进行去重、规整与批次管理，确保每条外链具备可追溯的唯一标识。
- **多维度元数据提取**：从 URL 路径中自动解析资源类型、编号后缀及潜在分类线索，为后续索引提供结构化字段。
- **批量状态巡检**：支持自定义周期对全部已收录链接发送 HEAD 请求，快速识别失效、重定向或内容变更的条目。
- **标签体系与全文检索**：允许用户为每条链接添加自定义标签（如地区、年份、清晰度），并基于标题、编号、标签进行布尔检索。
- **导入导出接口**：提供 JSON、CSV 和 Markdown 表格三种导出格式，便于与其他数据分析工具或文档系统对接。
- **访问统计看板**：内置轻量级计数模块，记录每条链接的点击次数与最后访问时间，辅助评估资源热度。
- **权限分级管理**：支持多用户只读与可写角色分离，适用于团队协作场景下的链接维护。

## 应用场景

- **个人影音库整理**：用户可将分散在多个页面的 VOD 外链统一收录至本索引系统，通过标签与检索功能快速筛选想看的条目，避免重复查找。
- **学术研究与内容分析**：研究人员利用批量巡检功能定期检测大量 VOD 链接的有效性，统计资源存活率，分析不同编号段的内容分布规律。
- **自动化监控与告警**：运维人员配置定时任务，当链接状态码异常或响应时间超过阈值时，系统通过 Webhook 发送通知，实现被动资源监控。
- **文档站点外链管理**：技术博客或项目文档中常引用大量外部资源，本系统可作为独立的外链中间层，统一管理引用目标，便于批量更新与替换。
- **爬虫规则调试**：开发者使用本项目提供的编号生成规律与路径模式，验证爬虫解析逻辑的覆盖率与准确性，降低重复劳动。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，确保已安装 Git 与 Node.js 16+。

```bash
# 克隆仓库
git clone https://github.com/your-org/vodlink-indexer.git
cd vodlink-indexer

# 安装依赖
npm install

# 使用示例数据运行索引服务
npm run build
npm start
```

访问控制台输出的本地地址（默认 http://127.0.0.1:3000）即可查看已收录链接列表。首次启动自动加载 `data/sample.json` 中的示例条目，您可通过管理界面上传自定义 CSV 或 JSON 文件替换数据源。

## 常见问题

Q: 项目启动后无法加载数据，控制台提示 `SQLITE_ERROR: no such table: links`，如何解决？

A: 该错误表示数据库尚未初始化。请先执行 `npm run migrate` 命令，该脚本会根据 `config/schema.sql` 自动创建所需数据表。若使用示例数据启动，可运行 `npm run seed` 加载 `data/sample.json`。确认数据库文件所在目录具有写入权限。

Q: 批量巡检功能运行缓慢，如何处理大量链接（超过 1000 条）的并发检查？

A: 系统默认并发数为 10，您可以通过环境变量 `CHECKER_CONCURRENCY` 调整该值（例如 `CHECKER_CONCURRENCY=20`）。同时建议配合 `CHECKER_TIMEOUT` 设置单次请求超时时间（单位毫秒），避免因个别链接响应过慢阻塞整体队列。对于超过 5000 条的批次，建议分片执行并使用 `--offset` 与 `--limit` 参数分段巡检。

Q: 导入 CSV 文件时出现字段映射错误，如何自定义列名？

A: 在管理界面上传 CSV 时，系统会自动检测表头并尝试匹配 `id`、`url`、`title`、`tags` 等标准字段。若您的 CSV 列名不同，可在上传前勾选“自定义映射”选项，手动将文件列与系统字段一一对应。映射配置会保存在用户会话中，便于重复导入相同格式的文件。
