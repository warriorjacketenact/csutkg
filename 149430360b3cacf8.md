# LinkVault — 技术资源导航与外部链接治理系统

LinkVault 是一个面向技术团队、开源社区和个人开发者的外部链接聚合、分类、审计与监控平台。项目定位于解决技术文档、项目 README、社区 Wiki 中外部链接散乱、失效、不可追溯的问题，提供统一的链接入库、标签管理、可用性检测与访问统计能力。目标用户包括开源项目维护者、技术内容运营人员、以及需要长期维护外部资源索引的开发者。

项目不依赖复杂后端架构，基于静态站点生成与轻量级数据库索引，可部署于任意 Web 服务器或云存储服务。LinkVault 关注链接的长期可访问性，通过定期巡检机制自动标记失效链接，并生成可视化报表，帮助用户及时修复或替换外部资源。

## 功能概览

**链接批量导入与解析**：支持从 Markdown、CSV、JSON 等格式文件中批量导入外部链接，自动提取标题、描述、域名及路径信息。

**链接分类与标签管理**：用户可自定义分类体系与标签组，对链接进行多维度标记，支持按项目、领域、文档章节等维度组织资源。

**可用性自动巡检**：内置定时任务引擎，按照可配置周期对全部链接发起 HEAD/GET 请求，检测 HTTP 状态码、响应时间及内容哈希变化。

**访问统计与热度分析**：记录每个链接的点击次数、来源页面和用户代理，生成访问趋势图与热门资源排行榜。

**失效链接告警通知**：当检测到链接返回 4xx/5xx 状态码或超时时，通过邮件、Webhook 或 RSS 推送告警信息。

**链接白名单与忽略规则**：支持配置域名或路径级别的白名单，跳过特定链接的可用性检测，适用于内网地址或临时性资源。

**数据导出与集成支持**：提供 RESTful API 和 JSON 导出功能，可将链接库输出为 Markdown 列表、HTML 目录或 sitemap 格式。

## 应用场景

开源项目文档维护：开源项目在 README 和 Wiki 中通常引用大量第三方依赖、教程和工具链接。LinkVault 可定期扫描这些文档中的外部链接，自动检测失效资源，帮助维护者及时更新文档，避免用户因访问死链而产生困惑。

技术博客资源管理：技术博客作者在文章中引用外部资料时，可使用 LinkVault 建立个人链接书签库。当某篇引用链接失效时，系统自动提醒作者替换或移除，保证博客内容的质量与可信度。

社区知识库质量保障：大型技术社区的知识库或 FAQ 页面包含数百条外部链接。LinkVault 提供分类统计和巡检报表，社区运营者可直观了解整体链接健康度，优先处理高频访问页面的失效链接。

内部技术文档中心：企业内部的 Confluence 或 GitLab Wiki 中存储大量技术方案、运维手册和工具地址。LinkVault 部署在内网环境，定期监控内部文档链接可用性，降低因链接失效导致的信息获取中断风险。

## 快速开始

以下命令可在 Linux / macOS / Windows WSL 环境下快速启动 LinkVault 服务。

```bash
git clone https://github.com/your-org/linkvault.git
cd linkvault
pip install -r requirements.txt
python linkvault.py --init --watch ./docs --output ./reports
```

执行上述命令后，LinkVault 会初始化 SQLite 数据库，扫描 `./docs` 目录下的所有 Markdown 文件并提取外部链接，随后启动 Web 控制台（默认监听 8000 端口）和后台巡检进程。巡检报告默认输出至 `./reports` 目录，包含链接状态汇总表和失效链接清单。

## 常见问题

Q：LinkVault 支持检测需要登录或带有动态参数的链接吗？

A：LinkVault 默认仅检测 HTTP 状态码与响应时间，不支持执行 JavaScript 或处理登录态。对于需要认证的链接，您可以在配置文件中将该域名或路径加入白名单，跳过自动检测。如需检测动态内容，建议配合独立的健康检查服务使用。

Q：巡检任务会影响源站性能吗？如何控制检测频率？

A：LinkVault 默认将并发请求数限制为 10，且每个请求带有 5 秒超时和 1 秒间隔延迟。您可以在 `config.yaml` 中调整 `max_workers` 和 `request_delay` 参数。建议生产环境巡检周期设置为每周一次，避免对源站造成额外负载。

Q：如何从旧版 LinkVault 迁移数据到新版？

A：项目提供了 `scripts/migrate_v1_to_v2.py` 迁移脚本，支持从 SQLite 旧表结构导出数据并转换至新格式。执行前请备份数据库文件，迁移完成后运行 `linkvault.py --verify` 校验数据完整性。跨大版本升级前请查阅 `docs/operations.md` 中的迁移指南。
