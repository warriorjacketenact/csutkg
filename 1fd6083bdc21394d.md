# EON Resource Navigator

EON Resource Navigator 是一个面向技术研究者、数据采集工程师和信息分析人员的高密度外链资源汇总系统。本项目并非传统的内容管理系统，而是一个精心编排的导航索引，将分散于网络各处的深度技术文档、行业分析报告、数据看板与专业工具入口整合为统一的访问平面。

项目定位为"技术资源的索引之索引"，专注于收录那些具有长期参考价值但缺乏有效发现路径的深层网页。通过本项目，用户可以快速定位到特定领域的技术备忘、行业基准数据、配置模板以及各类专业计算工具，大幅降低信息检索的时间成本。项目本身不存储任何实质内容，仅提供指向原始资源的稳定引用，确保内容的权威性与时效性完全由源站保证。

## 功能概览

- **分类索引浏览** 按技术领域、文档类型与数据层级对全部资源链接进行多维度归类，支持快速筛选与定位。

- **裸域直链访问** 所有外链均以原始URL形式呈现，不添加任何重定向、跟踪参数或中间跳转页面，确保访问路径最短。

- **资源状态标记** 对每个索引条目记录其指向内容类型，包括技术规范、数据快照、配置模板与分析报告等。

- **批量导入导出** 支持以结构化格式导入外部链接列表，并可导出为多种数据交换格式供下游工具使用。

- **本地快照缓存** 可选开启本地元数据缓存功能，在源站不可达时仍可查看资源的基本描述与最后已知状态。

- **访问日志审计** 记录所有外链的点击访问行为，为资源热度分析与链接可用性监控提供数据基础。

- **标签体系关联** 每个资源可绑定多个自定义标签，形成灵活的交叉索引网络，支持复杂查询条件下的资源发现。

- **定期可用性探测** 后台定时任务对收录链接进行HTTP可达性检查，自动标记异常链接并生成告警通知。

## 应用场景

- **技术文档归档与检索** 技术团队可将项目部署为内部知识库的前端导航层，将散落在各个Wiki、代码仓库和云笔记中的文档链接统一索引，团队成员通过标签或关键词快速找到所需的部署手册、接口规范或故障处理指南。

- **数据采集管道配置参考** 数据工程师在进行爬虫规则编写或API对接时，可将本项目作为配置模板与数据结构示例的查找入口。收录的各类数据格式说明与接口定义文档能够帮助快速完成采集适配工作。

- **行业基准数据查询** 分析师与研究人员可通过本项目快速定位到各类行业报告、统计年鉴与市场数据看板的原始发布页面，避免因搜索引擎结果变动而无法找到关键数据来源。

- **运维故障处置辅助** 运维人员在处理系统异常时，可通过本项目索引的故障案例库与配置参数参考页面，快速比对当前系统状态与已知问题场景，缩短故障定位时间。

## 快速开始

以下步骤指导您在三分钟内完成项目的本地部署与初始运行。

```bash
# 克隆项目仓库
git clone https://github.com/eon-resource/navigator.git
cd navigator

# 安装项目依赖
pip install -r requirements.txt

# 初始化本地索引数据库并启动服务
python manage.py migrate
python manage.py loaddata initial_resources.json
python manage.py runserver 0.0.0.0:8000
```

服务启动后，访问本地端口8000即可进入索引浏览界面。首次启动时将自动创建SQLite数据库文件并导入预置的资源链接元数据，包含全部收录URL的标题、描述与分类信息。如需更新索引内容，可执行`python manage.py fetch_metadata`命令触发批量元数据抓取。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 核心运行环境，用于后端服务与索引管理脚本 |
| SQLite | 3.35 及以上 | 默认元数据存储引擎，支持JSON字段类型 |
| Redis | 6.0 及以上 | 可选依赖，用于缓存加速与后台任务队列 |
| requests | 2.28.0 及以上 | HTTP客户端库，用于资源可达性探测与元数据抓取 |
| beautifulsoup4 | 4.11.0 及以上 | HTML解析库，用于从目标页面提取标题与描述信息 |
| django | 4.2.0 及以上 | Web框架，提供管理界面与RESTful API接口 |
| django-cors-headers | 4.0.0 及以上 | 跨域资源共享中间件，支持前端独立部署场景 |
| celery | 5.3.0 及以上 | 分布式任务队列，用于执行周期性可用性检查任务 |
| gunicorn | 21.0.0 及以上 | 生产环境WSGI服务器，推荐用于多进程部署 |

## 常见问题

**问：收录的链接出现无法访问时应该如何处理？**

本项目作为外链索引不托管实际内容，当用户发现某个链接不可达时，可通过项目管理后台的"报告链接异常"功能提交反馈。系统后台的可用性探测任务会定期自动重试，若连续三次探测均失败，该链接将被标记为"异常"状态并从默认视图中隐藏，但不会从数据库中删除。用户亦可手动触发单条链接的即时探测，以获取最新的可达性状态。

**问：如何请求新增资源链接到索引中？**

用户可通过项目提供的 RESTful API 端点提交新链接，提交时需附带资源标题、简要描述以及至少一个分类标签。提交的链接将由维护团队进行人工审核，审核标准包括内容相关性、资源质量与链接稳定性。审核通过后新链接将在下一个索引更新周期（每日凌晨）并入主库。普通用户亦可直接使用项目提供的浏览器书签工具快速提交当前浏览页面。

**问：项目是否支持私有化部署与离线使用？**

项目完全支持私有化部署，用户可在内网环境完成全部安装流程。离线使用场景下，需提前将所有外链的元数据缓存至本地 SQLite 数据库，具体操作方式参见运维手册中的"离线部署指南"章节。注意离线模式下可用性探测功能将自动禁用，且无法获取源站内容的实时更新，建议定期通过合规途径同步元数据。
