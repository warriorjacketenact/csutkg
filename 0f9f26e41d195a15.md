# Water Resource Data Index Platform (WRDIP)

Water Resource Data Index Platform 是一个面向水利工程数据、水文监测记录与水资源管理文档的轻量级外链索引系统。该项目不存储任何实际数据文件，而是以结构化方式聚合来自各级水利部门、研究机构与公共服务平台发布的公开数据页面、报告入口与监测看板，为水利工程师、科研人员与政策制定者提供统一、可维护的访问入口。

项目定位于技术资源汇总与导航层，解决水利信息分散于多个子系统、URL 结构不规则、检索路径冗长等问题。通过将分散的水文站点数据、水资源公报、洪水预警记录与水质监测报告按既定分类规则进行索引，用户可快速定位至目标资源，避免重复搜索与手工整理。本平台适用于需要频繁查阅水利公开数据的外部研究团队、内部信息化运维人员以及第三方系统集成开发者。

## 功能概览

- **多级资源分类索引**：按数据来源、内容类型与行政区划对超过 200 个水利相关外链进行分组，支持按站点代码与年份筛选。
- **动态参数构造器**：根据用户输入的水文站点编号、时间区间与数据粒度自动生成符合目标系统规范的完整请求路径。
- **链接可用性预检**：在索引更新阶段对每个收录的 URL 执行连通性测试与响应状态码记录，标记异常条目。
- **元数据标注体系**：为每条链接附加数据更新频率、文件格式、涉及水体名称与责任单位等关键描述字段。
- **批量导出与集成支持**：提供 JSON 与 CSV 格式的索引清单导出接口，便于与外部数据采集系统或监控平台集成。
- **自定义标签与搜索过滤**：允许用户为资源打上自定义标签，并基于多条件组合快速过滤结果列表。
- **访问统计与热度排序**：记录各链接被点击的次数与最近访问时间，支持按热门程度展示常用资源。
- **定期同步机制**：支持配置定时任务从上游站点抓取目录变更，自动更新本地索引条目。

## 应用场景

- **水文监测数据日常查阅**：水利局运维人员每日通过本平台快速访问辖区内各水文站点的实时水位与流量报告页面，无需记忆多套域名与路径规则。
- **历史洪水事件回溯研究**：科研团队在分析特定流域历史洪水规律时，使用平台按年份与站点 ID 筛选出对应时段的逐日记录入口，批量获取基础数据。
- **水资源公报编制汇总**：编制年度水资源公报时，工作人员通过平台汇总多个分中心提交的监测报告链接，统一核对数据来源与更新状态。
- **跨系统数据对接测试**：第三方系统开发者在集成水利数据 API 或进行页面数据抽取前，利用平台提供的索引清单快速构建测试用例与验证环境。
- **防汛应急信息快速导航**：防汛抗旱指挥部在应急响应期间，通过平台预置的应急监测页面分组，一键跳转至雨情、水情与气象预警等关键看板。

## 快速开始

以下步骤指导你在本地环境中完成平台的克隆、依赖安装与开发服务启动。

```bash
# 克隆项目仓库至本地
git clone https://github.com/water-resource-index/wrdip.git
cd wrdip

# 安装项目依赖（使用 npm，若使用 yarn 请替换为 yarn install）
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

启动成功后，在浏览器中访问 http://localhost:3000 即可进入索引管理界面。首次启动将自动执行初始资源索引构建任务，耗时约取决于网络环境。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Node.js 18.x 或更高版本 | 是 | 项目运行时环境与包管理基础，低于此版本将导致语法解析错误。 |
| npm 9.x 或 yarn 1.22.x | 是 | 用于安装项目依赖与管理脚本命令，推荐使用 npm。 |
| SQLite 3 或 PostgreSQL 12+ | 是 | 存储资源索引、标签与访问统计，SQLite 用于开发测试，生产环境建议使用 PostgreSQL。 |
| Redis 6.x | 否 | 启用缓存与会话管理时可提高响应速度，非必需但推荐。 |
| 网络访问权限 | 是 | 平台需要对外部水利站点执行可用性检测，需允许出站 HTTP/HTTPS 请求。 |
| 系统时区设置为 UTC+8 | 否 | 用于时间戳显示与定时任务调度，非强制但建议与上游数据发布时间对齐。 |
| curl 或 wget 命令行工具 | 否 | 用于执行外部链接可用性检测的后备方案，若 Node.js 原生 http 模块不可用时使用。 |

## 项目结构

```
wrdip/
├── src/
│   ├── core/                         # 核心索引引擎与数据模型
│   │   ├── indexer.js                # 资源索引构建与更新逻辑
│   │   ├── validator.js              # URL 可用性检测与状态记录
│   │   └── models/                   # 数据模型定义（Resource, Tag, Log）
│   ├── api/                          # RESTful API 接口层
│   │   ├── routes/                   # 路由定义（search, export, sync）
│   │   └── controllers/              # 请求处理器与业务逻辑编排
│   ├── ui/                           # 前端交互界面组件
│   │   ├── pages/                    # 页面级组件（首页、详情、管理）
│   │   ├── components/               # 可复用 UI 组件（过滤栏、表格、标签）
```

## 常见问题

**问：索引中的链接无法访问，平台如何处理？**

答：平台每日通过调度任务对收录链接执行 HTTP HEAD 请求检测。连续三次检测失败（超时或状态码非 2xx/3xx）的链接将被标记为“异常”状态，并在管理界面高亮显示。管理员可手动触发重检或编辑链接地址。用户访问异常链接时，平台会显示提示信息并建议联系管理员。

**问：如何添加新的资源分类或修改现有分类规则？**

答：分类规则定义在 `src/core/models/category.js` 中，采用基于路径前缀匹配的映射表。如需新增分类，请参照现有条目格式添加映射项，并运行 `npm run build:index` 重建索引。若涉及前端展示分类，需同步修改 `src/ui/components/FilterBar.js` 中的下拉选项。建议在开发分支完成变更并通过测试后再合并至主分支。

**问：平台能否部署到内网环境且不连接外网？**

答：可以。平台本身不依赖外部 API，所有数据均存储于本地 SQLite 或 PostgreSQL 中。但请注意，链接可用性检测功能需要出站网络权限，若内网无法访问外网水利站点，该功能将返回超时错误。可在配置文件中将 `healthCheck.enabled` 设为 `false` 关闭该功能，仅保留索引浏览与导出能力。
