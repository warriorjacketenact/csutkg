# LinkVault Core

LinkVault Core 是一个面向技术团队与内容运营者的轻量级外链资源聚合与导航系统。该项目定位于解决多源异构链接的集中管理、快速检索与状态监控问题，特别适用于需要维护大量外部参考链接、文档入口或版本发布地址的技术文档库与知识中台场景。目标用户包括技术文档工程师、DevOps 团队、开源项目维护者以及需要系统化管理外部依赖链接的开发者。

LinkVault Core 本身不存储任何实际内容，仅提供结构化的链接索引、元数据标签以及可扩展的校验机制。通过简单的 Markdown 配置与脚本工具，用户可以将大量分散的 URL 纳入统一的管理框架，并自动生成可浏览的导航页面或 API 响应。第 87/1241 批资源导入工作已证明该系统能够稳定处理超过 250 个外部链接的批量入库与分类标记。

## 功能概览

- **批量链接导入解析**：支持从纯文本、CSV 或 Markdown 列表中批量读取 URL，自动识别协议与路径结构，并提取路径参数作为初步分类依据。

- **元数据标签系统**：允许用户为每个链接添加自定义标签（如 "video-source", "api-doc", "archive"），并支持按标签组合进行快速筛选与分组。

- **链接状态健康检查**：内置 HTTP 状态码探测模块，可定时检测链接可达性，并输出异常报告，帮助维护者及时发现失效资源。

- **多格式导航输出**：支持将链接集输出为静态 HTML 导航页、JSON API 结构或纯 Markdown 目录，适应不同的发布与集成需求。

- **版本差异对比**：提供批次导入记录功能，可在新批次导入时自动对比新旧链接列表，生成新增、删除或变更的差异报告。

- **路径规则映射**：支持基于 URL 路径模式（如 /voddetail/{id}）自动提取资源标识符，并生成规范化的内部引用键值。

## 应用场景

- **技术文档库外部参考管理**：技术文档中常包含大量外部参考链接。LinkVault Core 可将这些链接统一收录并定期检查可用性，避免文档中出现死链，提升用户体验。

- **运营活动资源聚合页生成**：运营团队在策划内容专题时，需要快速汇总多个视频、文章或下载入口。通过标签分类与 HTML 输出功能，可在数分钟内生成一个清晰的外部资源导航页。

- **多环境配置链接集中维护**：开发团队在测试、预发布、生产环境中使用不同地址的外部服务。LinkVault Core 的路径映射与版本对比功能可以帮助追踪各环境链接的差异，减少配置错误。

- **开源项目外部依赖清单管理**：开源项目常引用第三方库文档、工具站点或社区论坛。将这类链接纳入 LinkVault Core 管理后，项目维护者可快速获知哪些外部资源已迁移或失效，及时更新项目 README 或文档站。

## 快速开始

以下命令演示了如何获取 LinkVault Core 源码、安装依赖并运行一个示例导入任务。

```bash
# 克隆仓库
git clone https://github.com/linkvault/core.git linkvault-core

# 进入项目目录
cd linkvault-core

# 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行导入示例（包含 250 个链接的测试批次）
python linkvault.py --input samples/batch_87.txt --output dist/nav.html
```

| Python 3.8+ | 是 | 核心运行环境，用于执行链接解析、校验与输出生成 |

| Docker 20.10+ | 否 | 可选容器化运行方式，适用于持续集成环境 |

## 常见问题

**问：LinkVault Core 是否支持批量导入超过 1000 个链接？**

是。系统设计上对批次大小没有硬性上限，但建议单批次不超过 2000 个链接以保证内存与网络请求的合理开销。对于超大批次，可使用 --chunk 参数分块处理，并将中间结果缓存至本地 SQLite 数据库。

**问：链接状态检查是否会影响目标站点的正常访问？**

不会。检查模块默认使用 HEAD 请求获取状态码，仅消耗极小的服务器资源。若目标站点不支持 HEAD，系统会自动降级为 GET 请求并设置超时（默认 5 秒）与重试策略（最多 2 次）。用户可在配置文件中调整超时时间与并发数。

**问：是否可以在不联网的环境中使用 LinkVault Core？**

可以。在不联网环境中，链接状态检查功能将自动跳过或标记为 "unreachable"，其余解析、分类与输出生成功能完全正常工作。用户可通过配置文件设置 check_enabled: false 彻底关闭检查模块。
