# LinkTheater

LinkTheater 是一个面向技术内容策展人与开源社区运营者的轻量化外链资源聚合系统。该项目并非传统的爬虫或采集器，而是一个用于手动维护、分类展示与版本化托管技术阅读清单、工具导航与案例参考库的静态站点生成框架。LinkTheater 帮助团队将散落在浏览器书签、团队文档与即时通讯消息中的高价值外部链接，转化为结构清晰、可公开访问、便于协作维护的技术资源目录，适用于开源项目官网的生态页、技术周刊的推荐列表或内部知识库的入口导航。

## 功能概览

- **按主题分区的链接目录**：支持通过目录配置文件将链接归入不同技术领域或使用场景分类，每个分类可单独设置说明文本与图标标识。
- **链接元数据注解系统**：为每条外链附加作者、发布日期、技术栈标签与简要摘要，便于访客在点击前判断内容相关性。
- **静态站点生成流水线**：基于配置文件与模板引擎，在构建阶段生成完整的 HTML 页面集合，无需运行时数据库，可直接部署至对象存储或 CDN。
- **链接存活健康检查**：内置基于 HTTP 状态码的链接可用性检测脚本，支持定期扫描并生成失效链接报告，帮助维护者及时更新或移除死链。
- **RSS 订阅源生成**：自动为链接更新列表生成 RSS/Atom Feed，方便关注者通过阅读器获取新增资源通知。
- **全文搜索支持**：集成客户端全文搜索能力，访客可通过关键词在已收录的链接标题、摘要与标签中进行检索。
- **暗色主题与响应式布局**：默认提供两套配色方案，并针对移动端阅读优化排版，确保在不同设备上的浏览体验一致。

## 应用场景

1. **开源项目生态资源导航**：开源项目维护者可使用 LinkTheater 构建项目官网的 Community Resources 页面，集中展示第三方工具、插件、教程文章与视频讲解，降低新用户的上手门槛，同时为贡献者提供生态现状的快速认知。

2. **技术团队内部知识入口**：研发团队可将 LinkTheater 部署在内网，作为团队技术决策参考库的前端界面，统一存放架构设计文档链接、性能测试报告、事故复盘记录以及外部技术博客的推荐阅读清单，减少重复沟通成本。

3. **技术周刊与资讯聚合站**：内容创作者或社区运营人员可利用 LinkTheater 的 RSS 生成能力，定期发布某一技术领域（如 Rust 生态、云原生、前端工程化）的优质外链合集，取代传统的邮件列表或社交媒体长图，提供更便于检索与回溯的存档形式。

4. **技术会议与黑客松资源手册**：活动组织者可在会前将演讲幻灯片链接、代码示例仓库、线上协作工具地址及 Wi-Fi 配置说明统一收录至 LinkTheater 生成的页面中，通过短链接分发给参会者，避免纸质材料浪费并方便会后查阅。

## 快速开始

以下步骤帮助您在本地环境中快速启动 LinkTheater 开发实例。

```bash
# 克隆项目仓库
git clone https://github.com/linktheater/linktheater.git
cd linktheater

# 安装依赖（使用 npm）
npm install

# 构建静态站点并启动本地预览服务
npm run build
npm run serve
```

执行完上述命令后，打开浏览器访问 `http://localhost:8080` 即可查看默认示例站点。您可以通过编辑 `config/links.yaml` 文件来替换示例链接数据，修改 `config/site.yaml` 调整站点标题与导航结构。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | v18.17.0 或更高 | 运行时环境，用于执行构建脚本与依赖管理 |
| npm | v9.0.0 或更高 | 包管理器，用于安装项目依赖 |
| Git | v2.25.0 或更高 | 版本控制工具，用于克隆仓库及后续更新 |
| YAML 编辑器 | 任意 | 推荐 VS Code 或 JetBrains 系 IDE，用于编辑链接配置文件 |
| 现代浏览器 | Chrome 90+ / Firefox 88+ / Edge 90+ | 用于预览生成的静态站点，支持 ES Module 与 CSS Grid |
| Python 3.9+（可选） | 仅当启用链接健康检查脚本时需要 | 用于运行独立的链接存活检测工具 |
| curl / wget（可选） | 仅当使用 CLI 方式触发健康检查时需要 | 用于向检查服务发送 HTTP 请求 |

## 贡献指南

LinkTheater 遵循开源社区协作模式，欢迎各类形式的贡献。请按照以下步骤参与项目开发。

1. 查阅 issue 列表与项目看板，确认当前迭代周期的待办事项，或提交新的 issue 描述您希望解决的问题或新增功能。对于较大规模的变更，建议先通过 issue 与维护者沟通设计思路，避免无效开发。

2. 从主仓库 fork 项目至个人账户，并在本地创建特性分支（如 `feat/add-rss-feed` 或 `fix/health-check-timeout`），遵循项目已有的代码风格与提交信息规范（使用语义化提交消息格式）。

3. 完成代码变更后，确保通过全部单元测试，并为新增功能补充对应的测试用例与文档说明（更新 `docs/` 目录下的相关文件）。若涉及配置格式变更，请同步更新 `config/` 下的示例文件与 schema 校验规则。

4. 提交 pull request 至主仓库的 `main` 分支，在 PR 描述中清晰说明变更内容、测试覆盖情况以及是否涉及破坏性改动。PR 将通过 CI 流水线自动执行构建与测试，维护者会在 3 个工作日内进行审查。

5. 对于非代码类贡献（如新增推荐链接、完善文档、报告失效链接），可直接通过 issue 或 discussions 板块提交，维护者将定期整理并合并至示例数据或文档中。

## 常见问题

**Q: LinkTheater 是否提供在线演示站点？如何快速预览示例链接的分类效果？**

A: 项目自带示例配置数据（即资源列表中的全部链接），您只需按照快速开始章节执行 `npm install && npm run build && npm run serve` 即可在本地启动演示站点。演示站点会展示所有示例链接按其分类标签组织的页面，并启用全文搜索与 RSS 订阅功能，方便您评估项目能力。若需部署至公网，请参考部署手册中的云平台部署指引。

**Q: 链接健康检查脚本的检测频率如何控制？是否会对外部站点造成过大请求压力？**

A: 健康检查脚本默认采用串行请求方式，且每个请求之间设置 200 毫秒的延迟间隔，单次完整扫描约耗时 2-5 分钟（取决于网络状况）。您可以通过环境变量 `CHECK_INTERVAL_MS` 调整延迟间隔，或通过 `--concurrency` 参数控制并行请求数（建议不超过 3）。脚本仅发送 HEAD 请求以获取状态码，不会下载完整页面内容，对外部站点的资源消耗极小。生产环境中建议通过 cron 或 CI 定时任务每周触发一次扫描，避免过于频繁的检测。

**Q: 是否支持导入现有的书签导出文件（如 Chrome HTML 书签或 Firefox JSON 备份）？**

A: 项目在 `scripts/` 目录下提供了 `import-bookmarks.js` 辅助脚本，支持解析 Chrome 书签导出 HTML 文件并转换为 LinkTheater 的 YAML 配置格式。执行 `node scripts/import-bookmarks.js --input bookmarks.html --output config/links.yaml` 即可完成导入。目前对 Firefox JSON 格式的支持仍在实验中，建议先转换为 Chrome HTML 格式后再进行导入。
