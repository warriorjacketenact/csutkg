# WeiKe Net Series Index

WeiKe Net Series Index 是一个面向技术研究、学术资源整理与知识体系构建的开源外链汇总工具。本项目并非传统的爬虫或采集系统，而是一套结构化、可维护的系列资源索引方案，旨在帮助研究人员、开发者及内容策展人高效管理分散在网络各处的系列化专题内容。

项目定位为“系列资源的知识图谱入口”，通过标准化的索引格式与目录树组织，将散落的系列文章、教程或文档聚合成可被程序化读取、可被人工浏览的静态知识库。目标用户包括技术博客作者、开源文档维护者、在线教育内容运营者以及需要长期追踪特定主题系列更新的研究者。本项目解决的核心问题是：当优质内容以系列形式分散发布时，如何通过一个轻量级、无锁定的索引层实现统一发现与持久引用。

## 功能概览

**系列资源索引聚合**：将来自同一数据源的大量系列入口以扁平化列表形式集中呈现，支持按系列标识符快速定位特定专题。

**结构化元数据提取**：从每条系列链接中解析出系列唯一标识符，作为后续分类、检索与版本管理的基础键值。

**静态站点生成友好**：索引数据完全基于 Markdown 与纯文本，无需数据库或后端运行时，可直接挂载为静态站点的内容目录。

**外链校验与可用性追踪**：提供可选的链接状态检查工具链，支持定期批量检测系列链接的有效性，减少死链污染。

**分类标签系统预置**：内置基于系列标识符前缀特征的自动分类建议，辅助人工打标与专题分组。

**多级目录树浏览**：通过 ASCII 目录树可视化展示索引库的物理文件组织，便于贡献者理解项目结构。

**纯前端可消费的数据格式**：支持将索引导出为 JSON 或 YAML 格式，方便嵌入 VuePress、Docusaurus、Hugo 等文档框架。

**贡献者友好的维护流程**：提供标准化的新增、更新、废弃系列链接的 Git 工作流，支持多人协作维护索引库。

## 应用场景

技术博客系列文章归档与导航：技术博主可借助本项目结构，将历年发布的系列教程（如“XX 入门系列”、“YY 源码解析系列”）的入口链接统一收录，并为读者提供清晰的系列总览页面，替代零散的“上一篇/下一篇”导航。

开源项目文档版本库索引：大型开源项目的文档团队可将不同版本、不同语言、不同模块的文档系列入口纳入索引，通过标识符区分稳定版、候选版与开发版，便于用户按需切换。

在线教育平台课程系列外链管理：教育内容运营者可将入驻讲师发布的系列视频或图文课程链接按专题组织，通过本索引层快速生成课程目录页，降低学员的查找成本。

企业内部知识库系列化整理：企业技术团队可将内部 Wiki 中分散的系列化运维文档、故障排查手册、架构决策记录等通过本索引统一暴露，形成可检索、可追溯的知识资产清单。

## 快速开始

以下命令演示如何将本项目克隆至本地、安装基础依赖并启动静态索引服务。

```bash
git clone https://github.com/your-org/wei-ke-series-index.git
cd wei-ke-series-index
npm install
npm run build
npm run serve
```

若使用 Yarn 或 pnpm，请将对应命令替换为 `yarn install` / `yarn build` / `yarn serve` 或 `pnpm install` / `pnpm build` / `pnpm serve`。构建产物默认输出至 `dist` 目录，可直接部署至任意静态托管服务。

## 贡献指南

新增系列链接：在 `data/index.json` 中按照既有的 JSON Schema 追加新条目，必须包含系列标识符（从 URL 中提取）、标题（若已知）及添加日期。若标题未知，可暂留空字符串，但标识符不得重复。

更新已有系列信息：若某系列链接的目标页面标题或分类发生变化，请直接在 `data/index.json` 中修改对应条目的 `title` 或 `category` 字段，并更新 `updated_at` 时间戳。修改后需运行 `npm run validate` 确保 JSON 格式合法。

移除失效系列链接：当某系列链接经多次校验确认返回 HTTP 404 或 410 状态码，且持续不可用超过 30 天，可将其从 `data/index.json` 中移除，并在 `data/removed.log` 中记录移除原因与日期。移除操作需通过 Pull Request 提交，并在描述中附上最后一次校验的时间。

提交变更流程：所有修改必须在独立分支上进行，分支命名遵循 `feat/`、`fix/`、`chore/` 前缀规范。提交信息须符合 Conventional Commits 格式。发起 Pull Request 后，需通过 CI 中的链接校验与 JSON Schema 验证两道检查，至少一名项目维护者审查后方可合并。

## 常见问题

问：索引中的系列链接对应的具体内容是什么？我是否需要额外配置才能查看？

答：本项目仅提供链接索引，不托管系列内容本身。每条链接指向的外部页面由 `wei-ke.net` 域名下的独立系列页面承载。用户无需任何额外配置，直接点击链接即可在浏览器中访问对应的系列专题。项目本身不设置访问代理或内容缓存。

问：如何批量检查索引中所有链接的可用性？

答：项目根目录下提供了 `scripts/validate-links.js` 脚本，执行 `npm run check-links` 即可启动批量校验。该脚本会并发发送 HEAD 请求，默认超时时间为 10 秒，最终生成一份包含状态码、响应时间及错误信息的 CSV 报告，存放于 `reports/` 目录下。建议每周运行一次，以维持索引质量。

问：我能否将本索引数据嵌入到我自己的静态站点生成器中？

答：可以。构建产物 `dist/api/index.json` 提供了纯净的 JSON 数据接口，支持跨域请求。你可以在任何静态站点生成器的数据加载阶段通过 HTTP 请求或本地文件引用方式获取该 JSON，然后按照你的模板结构渲染系列列表。项目本身不限制数据的使用场景，但请注意遵守外部链接源站的使用条款。
