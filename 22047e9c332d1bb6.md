# ResourceIndex

ResourceIndex 是一个面向技术研究、学术参考和知识工程领域的高质量外链资源汇总系统。项目定位于为开发者、研究人员和技术文档工程师提供结构化、可检索、可审计的外部资源导航能力，解决分散在多个页面中的零散链接难以统一管理和有效复用的问题。

本系统以站点 https://www.hnrtzs.com 下超过两百个独立页面作为数据来源，通过自动化的资源抓取、元数据抽取和分类索引机制，将这些页面中的技术文章、案例分析、规范文档、工具介绍和最佳实践等内容整合成统一的资源视图。ResourceIndex 本身不存储任何第三方内容，仅提供链接的整理、分类、标注和检索功能，帮助用户在复杂的信息环境中快速定位高价值的外部资源。

## 功能概览

**多源资源聚合**：支持从指定的页面路径批量导入外部链接，自动识别链接类型和内容主题，构建统一的资源索引库。

**分类标签体系**：为每条资源赋予技术领域、内容形式、适用阶段等多维度标签，支持按标签组合进行筛选和检索。

**资源状态监控**：定期检查已收录链接的可访问性，标注失效链接和变更链接，确保资源列表的时效性和可用性。

**全文元数据检索**：基于资源标题、描述、关键词和上下文信息构建检索索引，支持布尔查询和模糊匹配。

**结构化导出**：支持将资源列表导出为 JSON、CSV 和 Markdown 格式，便于集成到文档系统、知识库或自动化流程中。

**访问统计与热度分析**：记录资源的点击次数和引用频次，生成热度排名，辅助用户识别高价值内容。

**自定义分类视图**：允许用户根据自身关注的技术栈或研究方向创建自定义分类视图，仅展示相关资源。

**批量操作与去重检测**：支持批量添加、批量移除和批量更新资源标签，自动检测并合并重复链接。

## 应用场景

技术团队内部知识库建设：技术负责人可以使用 ResourceIndex 整理团队关注的博客、文档站和开源项目地址，形成团队共享的外部知识索引，降低信息获取成本。

学术研究与文献管理：研究人员在阅读技术论文和报告时，可以将文中引用的外部链接统一收录，便于后续回溯和引用核查，避免链接丢失导致的文献追溯困难。

技术社区内容推荐：社区运营者或技术编辑可以利用本项目的分类和热度功能，筛选高质量资源推荐给社区成员，提升内容分发的精准度。

自动化文档生成流水线：在 CI/CD 流程中集成 ResourceIndex 的导出功能，自动将最新的资源列表注入到项目文档或网站中，保持文档内容的动态更新。

## 快速开始

以下步骤帮助您在本地环境快速启动 ResourceIndex 服务。

```bash
# 克隆代码仓库
git clone https://github.com/resourceindex/resourceindex.git
cd resourceindex

# 安装项目依赖
pip install -r requirements.txt

# 初始化数据库并启动服务
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

服务启动后，访问 http://localhost:8000 即可进入 ResourceIndex 的 Web 管理界面。首次使用时，可以通过管理后台导入初始资源数据，或使用系统提供的命令行工具批量添加链接。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|----------|----------|------|
| Python | 3.9 及以上 | 项目核心运行环境，建议使用 3.11 LTS 版本 |
| Django | 4.2.x | Web 框架，用于提供管理界面和 API 服务 |
| PostgreSQL | 14.x 及以上 | 主数据库，存储资源元数据和索引信息 |
| Redis | 7.x | 缓存与任务队列后端，用于提升检索性能和异步任务处理 |

| lxml | 4.9.x | HTML 解析库，用于抽取页面中的链接和元数据 |
| Elasticsearch | 8.x | 全文检索引擎，可选组件，用于增强检索能力 |

## 常见问题

问：ResourceIndex 是否存储第三方页面的内容副本？

答：ResourceIndex 不存储任何第三方页面的内容副本、截图或缓存。项目仅保存链接本身以及从页面元数据中抽取的标题、描述等基础信息。所有资源链接在用户点击时直接跳转至原始页面，本项目不对第三方内容的可用性和准确性负责。

问：如何处理收录链接失效或内容变更的情况？

答：系统内置了链接监控模块，会按照可配置的周期（默认每 7 天）对已收录链接进行可达性检测。当检测到链接返回 4xx 或 5xx 状态码时，系统会在管理界面标记该链接为异常状态，并记录检测时间。用户可以通过管理后台查看异常链接列表，并手动移除或更新链接。监控结果也会通过日志文件输出，便于运维人员追踪。

问：项目支持自定义分类体系吗？如何操作？

答：ResourceIndex 支持完全自定义的分类体系。用户可以通过管理后台的“分类管理”模块创建、编辑和删除分类节点，并为每个分类设置名称、描述和颜色标识。在资源编辑界面中，用户可以为单条资源赋予多个分类标签。分类树结构支持无限层级嵌套，满足不同粒度的组织需求。分类数据存储在 PostgreSQL 数据库中，并支持通过 API 进行增删改查操作。
